# 17장. 통합 프로젝트 — 영화 검색 앱 (독립 AI-Native)

> **Phase 3**

> 💡 **실습 흐름**: 가이드를 참고하며 **혼자** 진행 → 막히면 AI에게 질문 → 강사는 최종 검토만

> **이 장의 AI 사용 규칙 — 독립 AI-Native 실행**
>
> 16장까지 배운 전체 AI-Native 워크플로우를 **혼자** 수행합니다.
>
> 요구사항은 내가 쓰고, AI가 구현하고, 테스트로 검증합니다.
>
> 강사의 시연 없이 가이드만 보고 진행하는 것이 목표입니다.

---

## 학습 목표

- AI-Native 워크플로우(설계 → Instructions → TDD → 구현 → 검증)를 **독립적으로** 수행할 수 있다
- 배열 형태의 API 응답을 **카드 그리드로 반복 렌더링**할 수 있다
- 객체 배열을 localStorage에 저장하고 관리할 수 있다


## 진행 순서

1. 프로젝트 소개 — 무엇을 만드나요?
2. Step 1: 요구사항 작성 — 설계 먼저
3. Step 2: 프로젝트 구조 + Instructions — 뼈대 세우기
4. Step 3: API 키 준비 — TMDB API
5. Step 4: TDD로 핵심 로직 구현 — 테스트 먼저
6. Step 5: UI 구현 — 카드 그리드 만들기
7. Step 6: 통합 테스트 + 디버깅 — 연결하고 확인하기
8. 최종 산출물 확인 — 완성 체크리스트
9. 마무리 — 전체 과정 돌아보기

## 1️⃣ 프로젝트 소개

### 무엇을 만드나요?

영화 제목을 검색하면 결과를 카드 그리드로 보여주는 웹 앱입니다.

![영화 검색 앱 완성 화면](/docs/12_ai_native_javascript/img/movie-app-mockup.png)

### 16장(날씨 앱)과 무엇이 다른가요?

| 비교 | 날씨 앱 | 영화 검색 앱 |
|------|---------|-------------|
| API 응답 | 하나의 도시 정보 | **여러 영화 목록 (배열)** |
| 화면 표시 | 텍스트 1개 | **카드 여러 개 (그리드)** |
| 이미지 | 작은 아이콘 (32px) | **큰 포스터 (200px+)** |
| 저장 데이터 | 도시 이름만 | **영화 정보 전체 (객체)** |
| 강사 도움 | 시연 + 따라하기 | **혼자 + 가이드 참조** |

### 이번 장에서 새로 배우는 것

1. **배열 반복 렌더링** — 검색 결과 여러 개를 화면에 표시하기
2. **카드 그리드 레이아웃** — CSS Grid로 카드 배치하기
3. **외부 이미지 처리** — 포스터 이미지 표시 + 이미지 없을 때 처리
4. **객체 배열 localStorage** — 복잡한 데이터 저장하기
5. **독립 AI-Native 실행** — 강사 없이 혼자 전체 워크플로우 수행

### copilot-instructions.md에 요구사항 작성하기

`.github/copilot-instructions.md` 파일에 아래 내용을 **직접 작성**하세요.

```markdown
# 영화 검색 앱 프로젝트 규칙 및 요구사항

## 기능 요구사항

### 1. 영화 검색
- 영화 제목을 한국어 또는 영어로 입력할 수 있다
- 검색 버튼 클릭 또는 Enter 키로 검색한다
- 검색 중에는 로딩 표시를 보여준다

### 2. 검색 결과 표시
- 검색 결과를 카드 그리드로 표시한다
- 각 카드에는 포스터 이미지, 제목, 평점, 개봉연도를 표시한다
- 포스터 이미지가 없는 영화는 대체 이미지를 표시한다
- 검색 결과 총 건수를 표시한다
- 평점은 소수점 1자리까지 표시한다 (예: 8.5)

### 3. 에러 처리
- 검색 결과가 없으면: "검색 결과가 없습니다. 다른 제목으로 검색해보세요."
- 네트워크 오류: "영화 정보를 불러오지 못했습니다. 인터넷 연결을 확인해주세요."
- 빈 검색어: 검색하지 않고 "영화 제목을 입력해주세요." 안내

### 4. 즐겨찾기 (워치리스트)
- 각 영화 카드에 즐겨찾기 버튼이 있다
- 즐겨찾기한 영화는 localStorage에 저장된다
- 즐겨찾기 목록을 별도 영역에 표시한다
- 이미 즐겨찾기한 영화는 버튼 스타일이 다르다 (♥ vs ♡)

## 제약 조건
- 파일 구조: src/api.js, src/favorites.js, src/app.js
- 테스트: tests/api.test.js, tests/favorites.test.js
- API 키는 환경변수(.env)에 저장 (import.meta.env.VITE_TMDB_API_KEY)
- ESM(import/export) 사용

## 범위 밖 (Out of Scope)
- 영화 상세 페이지 (모달이나 별도 화면)
- 페이지네이션 / 무한 스크롤
- 정렬 기능 (평점순, 개봉일순)
- 검색 기록 저장
- 다크 모드
- 반응형 디자인

## 금지 사항 (Don'ts)
- var 사용 금지 (let/const만)
- 전역 변수 사용 금지
- innerHTML에 사용자 입력 직접 삽입 금지 (XSS 방지)
- API 키를 코드에 하드코딩 금지
- jQuery 등 외부 라이브러리 금지
- console.log 디버깅 코드를 최종 코드에 남기지 않는다

## 기술 스택
- JavaScript ES2022+ (ESM import/export)
- Vitest + jsdom 테스트
- Vite 개발 서버
- TMDB API
- HTML5 + CSS Grid
```

## 4️⃣ ⭐ **핵심** — Step 3: API 키 준비

> 이미 API 키를 발급받았다면 바로 다음 단계로 넘어가세요.

### TMDB 가입 및 API 키 발급

1. [themoviedb.org](https://www.themoviedb.org)에 접속합니다
2. **가입하기**를 클릭하고 무료 계정을 만듭니다
3. 이메일 인증을 완료합니다
4. 로그인 후 **설정 > API** 메뉴로 이동합니다
5. API 키(v3 auth)를 복사합니다

### .env 파일에 API 키 저장

```
VITE_TMDB_API_KEY=여기에_발급받은_키를_붙여넣기
```

> **왜 `VITE_` 접두사를 붙이나요?**
> Vite는 보안을 위해 `VITE_` 접두사가 있는 환경변수만 브라우저에 노출합니다.
> 접두사가 없으면 브라우저에서 사용할 수 없습니다.
> 16장 날씨 앱에서도 같은 방식이었습니다.

### API 테스트 (브라우저에서 확인)

아래 URL의 `API키` 부분을 본인 키로 바꿔서 브라우저에 입력합니다:

```
https://api.themoviedb.org/3/search/movie?query=기생충&api_key=API키&language=ko-KR
```

아래와 비슷한 JSON 응답이 오면 성공입니다:

```json
{
  "results": [
    {
      "id": 496243,
      "title": "기생충",
      "poster_path": "/igw938inb6Fy...",
      "vote_average": 8.5,
      "release_date": "2019-05-30",
      "overview": "전원백수로 살아가는 기택 가족에게..."
    },
    ...
  ],
  "total_results": 15
}
```

**날씨 앱과의 차이를 확인하세요:**
- 날씨 앱: 응답이 **하나의 객체** (`{ name: "Seoul", main: { temp: 25 } }`)
- 영화 앱: 응답이 **배열** (`{ results: [영화1, 영화2, 영화3, ...] }`)

이 차이가 이번 프로젝트의 핵심입니다!

### a) favorites.js — 즐겨찾기 관리

날씨 앱의 `storage.js`와 비슷하지만, 이번에는 **문자열이 아닌 객체**를 저장합니다.

#### 🔴 Red — 테스트 먼저 작성

`tests/favorites.test.js`:

```javascript
// @vitest-environment jsdom
import { describe, it, expect, beforeEach } from 'vitest';
import { addFavorite, removeFavorite, getFavorites, isFavorite } from '../src/favorites.js';

beforeEach(() => {
  localStorage.clear();
});

describe('addFavorite', () => {
  it('영화 객체를 즐겨찾기에 추가한다', () => {
    const movie = { id: 496243, title: '기생충', poster_path: '/abc.jpg', vote_average: 8.5 };
    addFavorite(movie);

    const favorites = getFavorites();
    expect(favorites).toHaveLength(1);
    expect(favorites[0].title).toBe('기생충');
  });

  it('같은 id의 영화는 중복 추가하지 않는다', () => {
    const movie = { id: 496243, title: '기생충', poster_path: '/abc.jpg', vote_average: 8.5 };
    addFavorite(movie);
    addFavorite(movie);

    expect(getFavorites()).toHaveLength(1);
  });

  it('다른 영화는 각각 추가된다', () => {
    const movie1 = { id: 1, title: '기생충', poster_path: '/a.jpg', vote_average: 8.5 };
    const movie2 = { id: 2, title: '올드보이', poster_path: '/b.jpg', vote_average: 8.3 };
    addFavorite(movie1);
    addFavorite(movie2);

    expect(getFavorites()).toHaveLength(2);
  });
});

describe('removeFavorite', () => {
  it('id로 즐겨찾기에서 제거한다', () => {
    const movie = { id: 496243, title: '기생충', poster_path: '/abc.jpg', vote_average: 8.5 };
    addFavorite(movie);
    removeFavorite(496243);

    expect(getFavorites()).toHaveLength(0);
  });

  it('존재하지 않는 id를 제거해도 에러가 나지 않는다', () => {
    removeFavorite(99999);
    expect(getFavorites()).toHaveLength(0);
  });
});

describe('getFavorites', () => {
  it('저장된 즐겨찾기가 없으면 빈 배열을 반환한다', () => {
    expect(getFavorites()).toEqual([]);
  });
});

describe('isFavorite', () => {
  it('즐겨찾기에 있는 영화는 true를 반환한다', () => {
    addFavorite({ id: 1, title: '기생충', poster_path: '/a.jpg', vote_average: 8.5 });
    expect(isFavorite(1)).toBe(true);
  });

  it('즐겨찾기에 없는 영화는 false를 반환한다', () => {
    expect(isFavorite(999)).toBe(false);
  });
});
```

Copilot이 생성한 `src/favorites.js` 예시:

```javascript
const STORAGE_KEY = 'movie_favorites';

/**
 * 즐겨찾기 목록을 반환합니다.
 * @returns {Array<{id: number, title: string, poster_path: string, vote_average: number}>}
 */
export const getFavorites = () => {
  try {
    const data = localStorage.getItem(STORAGE_KEY);
    return data ? JSON.parse(data) : [];
  } catch {
    return [];
  }
};

/**
 * 영화를 즐겨찾기에 추가합니다. 중복은 무시합니다.
 * @param {Object} movie - {id, title, poster_path, vote_average}
 */
export const addFavorite = (movie) => {
  const favorites = getFavorites();
  if (favorites.some((fav) => fav.id === movie.id)) return;

  favorites.push(movie);
  localStorage.setItem(STORAGE_KEY, JSON.stringify(favorites));
};

/**
 * 영화를 즐겨찾기에서 제거합니다.
 * @param {number} id
 */
export const removeFavorite = (id) => {
  const favorites = getFavorites().filter((fav) => fav.id !== id);
  localStorage.setItem(STORAGE_KEY, JSON.stringify(favorites));
};

/**
 * 해당 id의 영화가 즐겨찾기에 있는지 확인합니다.
 * @param {number} id
 * @returns {boolean}
 */
export const isFavorite = (id) => {
  return getFavorites().some((fav) => fav.id === id);
};
```

```javascript
const BASE_URL = 'https://api.themoviedb.org/3';
const IMAGE_BASE_URL = 'https://image.tmdb.org/t/p/w500';

/**
 * TMDB API로 영화를 검색합니다.
 * @param {string} query - 검색할 영화 제목
 * @returns {Promise<{movies: Array, totalResults: number}>}
 */
export const searchMovies = async (query) => {
  if (!query || !query.trim()) {
    throw new Error('영화 제목을 입력해주세요');
  }

  const apiKey = import.meta.env.VITE_TMDB_API_KEY;
  const url = `${BASE_URL}/search/movie?query=${encodeURIComponent(query)}&api_key=${apiKey}&language=ko-KR`;

  try {
    const response = await fetch(url);

    if (!response.ok) {
      throw new Error(`서버 오류: ${response.status}`);
    }

    const data = await response.json();

    const movies = data.results.map((movie) => ({
      id: movie.id,
      title: movie.title,
      posterUrl: movie.poster_path
        ? `${IMAGE_BASE_URL}${movie.poster_path}`
        : null,
      voteAverage: movie.vote_average,
      releaseDate: movie.release_date,
      overview: movie.overview,
    }));

    return {
      movies,
      totalResults: data.total_results,
    };
  } catch (error) {
    if (error.message.includes('영화 제목')) {
      throw error;
    }
    throw new Error('영화 정보를 불러오지 못했습니다. 인터넷 연결을 확인해주세요.');
  }
};
```

**핵심 CSS 패턴을 확인하세요:**

```css
/* 이 3줄이 카드 그리드의 핵심입니다 */
.movie-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
```

> **이 패턴이 왜 중요한가요?**
> `repeat(auto-fill, minmax(200px, 1fr))`는 "카드 너비를 최소 200px으로 하되, 공간이 남으면 균등하게 늘려라"는 뜻입니다. 화면 크기가 바뀌면 카드 수가 자동으로 조절됩니다. 이것이 **반응형 그리드**입니다.
> 쇼핑몰 상품 목록, 넷플릭스 콘텐츠 목록, 인스타그램 피드가 모두 이 패턴입니다.
