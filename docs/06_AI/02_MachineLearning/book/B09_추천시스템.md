## 2️⃣ 콘텐츠 기반 필터링 — TMDB [↑](#toc)

> 책 9.5절. TMDB 5000 영화 데이터로 유사 영화 추천.

### 흐름

1. 영화의 **장르/키워드/캐스트/감독** 등 텍스트 결합
2. **CountVectorizer / TfidfVectorizer** 로 벡터화
3. **코사인 유사도** 계산
4. 특정 영화 → 가장 유사한 N개 추천

```python
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.metrics.pairwise import cosine_similarity

movies['features'] = movies['genres'] + ' ' + movies['keywords']
cv_matrix = CountVectorizer(ngram_range=(1, 2)).fit_transform(movies['features'])
sim = cosine_similarity(cv_matrix)

def recommend(title, top_n=10):
    idx = movies[movies['title'] == title].index[0]
    similar = sim[idx].argsort()[::-1][1:top_n+1]
    return movies.iloc[similar]['title']
```

> 💡 **평점 정보 가중치** 추가하면 "유사 + 평점 높은" 추천 가능.

## 3️⃣ 협업 필터링 개요 [↑](#toc)

> "당신과 비슷한 사람들이 좋아한 것을 추천."

### 데이터 구조

사용자-아이템 평점 행렬 (대부분 NaN):

```
         영화A 영화B 영화C 영화D
사용자1   5    -    3    -
사용자2   -    4    -    5
사용자3   4    -    3    4
```

### 두 갈래

| 방식 | 유사도 측정 |
|---|---|
| 사용자 기반 | 사용자끼리 유사 |
| 아이템 기반 | 아이템끼리 유사 (책 권장) |

## 5️⃣ 잠재 요인 협업 필터링 [↑](#toc)

> 책 9.7절. SVD 기반. 평점 행렬 R을 **R ≈ P × Q^T** 로 분해.

- **P** (사용자 × 잠재 요인) 
- **Q** (아이템 × 잠재 요인)
- 잠재 요인 = "장르 취향", "분위기 선호" 같은 숨겨진 축

### SGD 학습 (책의 핵심 코드)

```python
def matrix_factorization(R, K=8, lr=0.01, steps=200):
    n_users, n_items = R.shape
    P = np.random.normal(size=(n_users, K), scale=1/K)
    Q = np.random.normal(size=(n_items, K), scale=1/K)
    non_zeros = [(i, j, R[i, j]) for i in range(n_users)
                 for j in range(n_items) if R[i, j] > 0]
    for step in range(steps):
        for i, j, r in non_zeros:
            err = r - np.dot(P[i], Q[j])
            P[i] += lr * err * Q[j]
            Q[j] += lr * err * P[i]
    return P, Q
```

> 💡 **Netflix Prize 우승 알고리즘의 핵심**. 6장 SVD/NMF와 연결.

## 7️⃣ 정리와 체크리스트 [↑](#toc)

### 한 문장 요약

> **콘텐츠는 아이템 속성, 협업은 평점 이력, 잠재 요인은 둘 사이 숨은 축.** 콜드 스타트엔 콘텐츠, 대용량엔 잠재 요인.

### 자기 점검

- [ ] 추천 3대 유형 비교
- [ ] 콜드 스타트 문제와 해법
- [ ] TF-IDF + 코사인 유사도 → 콘텐츠 기반 코드 작성
- [ ] 아이템 기반과 사용자 기반 차이 / 왜 아이템 기반이 안정적인지
- [ ] R ≈ P × Q^T 분해의 의미
- [ ] Surprise 설치가 Python 3.12에서 실패하는 경우 대처

### 흔한 오해

| 오해 | 사실 |
|---|---|
| "추천은 항상 딥러닝(Wide&Deep, 2-tower)" | 콜드 스타트와 해석 가능성 때문에 협업/콘텐츠는 여전히 표준 |
