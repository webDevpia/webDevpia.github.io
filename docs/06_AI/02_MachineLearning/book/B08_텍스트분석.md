## 2️⃣ 텍스트 전처리 [↑](#toc)

### 4단계 표준 흐름

1. **클렌징** — 특수문자/HTML 제거
2. **토큰화** — 문장 → 단어
3. **Stopword 제거** — 'the', 'is', 'and' 등 의미 적은 단어
4. **표제어 추출(Lemmatization)** 또는 **어간 추출(Stemming)** — 'running'/'ran' → 'run'

```python
import nltk
nltk.download(['punkt', 'punkt_tab', 'stopwords', 'wordnet'])

from nltk.tokenize import word_tokenize
from nltk.corpus import stopwords
from nltk.stem import WordNetLemmatizer

tokens = word_tokenize(text.lower())
stops = set(stopwords.words('english'))
filtered = [t for t in tokens if t.isalpha() and t not in stops]
lem = WordNetLemmatizer()
final = [lem.lemmatize(t) for t in filtered]
```

## 3️⃣ BOW & TF-IDF [↑](#toc)

### Bag of Words

문장을 "단어 가방"으로 — 순서 무시, **단어 개수만**.

```python
from sklearn.feature_extraction.text import CountVectorizer
cv = CountVectorizer(min_df=2, ngram_range=(1, 2))
X_bow = cv.fit_transform(corpus)   # 희소 행렬
```

### TF-IDF

흔한 단어 페널티 + 희소 단어 가산.

| 약어 | 의미 |
|---|---|
| TF | Term Frequency (해당 문서 내 빈도) |
| IDF | Inverse Document Frequency (전체 중 희귀할수록 큼) |

```python
from sklearn.feature_extraction.text import TfidfVectorizer
tfv = TfidfVectorizer(min_df=2, max_df=0.95, stop_words='english',
                      ngram_range=(1, 2))
X_tfidf = tfv.fit_transform(corpus)
```

> 💡 **거의 항상 TF-IDF가 BOW보다 우수**. BOW는 개념 학습용.

## 5️⃣ 감성 분석 [↑](#toc)

### 두 가지 접근

| 방식 | 학습 데이터 | 도구 |
|---|---|---|
| **지도학습** | 라벨링된 리뷰 (positive/negative) | LR + TF-IDF |
| **비지도(어휘 기반)** | 단어별 감성 점수 사전 | VADER, SentiWordNet |

### VADER 예

```python
from nltk.sentiment import SentimentIntensityAnalyzer
sia = SentimentIntensityAnalyzer()
sia.polarity_scores("This movie is amazing!")
# {'neg': 0.0, 'neu': 0.5, 'pos': 0.5, 'compound': 0.65}
```

## 6️⃣ 토픽 모델링 (LDA) [↑](#toc)

> Latent Dirichlet Allocation. **문서에 숨은 주제를 자동 발견**.

> ⚠️ **이름 혼동 주의**: 6장의 LinearDiscriminantAnalysis(LDA) 와 다름. 토픽 모델링의 LDA는 별개.

```python
from sklearn.decomposition import LatentDirichletAllocation
lda = LatentDirichletAllocation(n_components=8, random_state=11)
lda.fit(X_bow)

# 각 토픽의 상위 단어 출력
for idx, topic in enumerate(lda.components_):
    top_words = [feature_names[i] for i in topic.argsort()[-10:]]
    print(f"Topic {idx}: {top_words}")
```

## 8️⃣ 문서 유사도 [↑](#toc)

> 코사인 유사도: 두 벡터의 각도. 0~1, 클수록 유사.

```python
from sklearn.metrics.pairwise import cosine_similarity
sim = cosine_similarity(X_tfidf[0:1], X_tfidf)
top_idx = sim[0].argsort()[-5:][::-1]
print("가장 비슷한 문서:", top_idx)
```

> 💡 **검색 엔진, 추천 시스템의 핵심 함수**. 9장과 연결.

## 🔟 실습: Mercari Price Suggestion [↑](#toc)

> 책 8.10절. **책 텍스트 분석의 백미**. 캐글 회귀 대회 데이터.

### 핵심 난제: 희소 행렬 결합

여러 종류의 피처:
- `name` (텍스트, CountVectorizer)
- `item_description` (텍스트, TF-IDF)
- `brand_name`, `category` (범주, LabelBinarizer + sparse)
- `item_condition` (수치)

→ 각각 희소 행렬로 만든 뒤 **scipy.sparse.hstack** 으로 결합.

```python
from scipy.sparse import hstack
X_features = hstack([X_name, X_desc, X_brand, X_cat]).tocsr()
print(X_features.shape)   # (행수, 전체 컬럼수)
```

### 회귀 모델

- **Ridge** (희소 입력에 빠름)
- **LightGBM** (대용량 + 희소)
- 가중 평균 앙상블

> 💡 **이 패턴은 실무에서 자주 등장**. 텍스트 + 범주 + 수치 혼합 데이터의 표준 처리.
