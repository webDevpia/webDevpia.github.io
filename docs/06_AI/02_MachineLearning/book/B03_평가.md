## 1️⃣ 정확도의 함정 [↑](#toc)

> "정확도 95%" 만 보고 좋은 모델이라 단정하면 위험합니다.

### 책 3.1절 핵심 예시

- 사기 거래 탐지: 전체 거래의 1% 만 사기
- **모든 거래를 "정상"으로 예측**해도 정확도는 99%
- 정작 잡아야 할 사기는 0% 잡음

> 📌 **결론**: **불균형 데이터(imbalanced data)** 에서 정확도는 무용지물. 다른 지표가 필요.

## 2️⃣ 혼동 행렬 [↑](#toc)

> 모든 분류 지표의 출발점. 외울 것은 4개 칸의 이름.

```
                예측
              양성    음성
실제 양성   [ TP  ] [ FN  ]   ← 진짜 양성 중 놓친 것 = FN
실제 음성   [ FP  ] [ TN  ]   ← 진짜 음성 중 잘못 양성으로 본 것 = FP
```

| 약어 | 한국어 | 의미 |
|---|---|---|
| **TP** | True Positive | 양성을 양성으로 (정답) |
| **FP** | False Positive | 음성을 양성으로 (1종 오류) |
| **FN** | False Negative | 양성을 음성으로 (2종 오류) |
| **TN** | True Negative | 음성을 음성으로 (정답) |

### 코드

```python
from sklearn.metrics import confusion_matrix
print(confusion_matrix(y_test, pred))
# [[TN FP]
#  [FN TP]]   ← sklearn 기본 정렬
```

> ⚠️ **순서 주의**: 사이킷런은 `[음성, 양성]` 순서이지만 책의 그림은 `[양성, 음성]` 순서. 헷갈리지 말 것.

## 3️⃣ 정밀도와 재현율 [↑](#toc)

### 정의

| 지표 | 식 | 해석 |
|---|---|---|
| **Precision (정밀도)** | TP / (TP + FP) | 양성 예측 중 진짜 양성 비율 |
| **Recall (재현율)** | TP / (TP + FN) | 실제 양성 중 잡아낸 비율 |

### 어느 쪽을 더 중요시할까?

| 도메인 | 우선 지표 | 이유 |
|---|---|---|
| 스팸 필터 | **정밀도** | 정상 메일을 스팸함에 넣으면 안 됨 (FP 비용 큼) |
| 암 진단 | **재현율** | 환자를 정상이라 진단하면 안 됨 (FN 비용 큼) |
| 사기 탐지 | **재현율** | 사기를 놓치면 큰 손해 |
| 추천 시스템 | **정밀도** | 추천한 게 별로면 사용자 이탈 |

> 💡 **수업 시연 팁**: 학생들에게 "당신이 의사라면? / 스팸 필터 개발자라면?" 질문 던지고 직관 끌어내기.

## 4️⃣ F1 스코어 [↑](#toc)

> 정밀도와 재현율의 **조화 평균**. 한쪽이 낮으면 F1도 낮음.

$$ F_1 = 2 \times \frac{\text{precision} \times \text{recall}}{\text{precision} + \text{recall}} $$

```python
from sklearn.metrics import precision_score, recall_score, f1_score
print(precision_score(y, pred))
print(recall_score(y, pred))
print(f1_score(y, pred))
```

### 왜 산술 평균이 아니고 조화 평균?

- 산술: 정밀도=1, 재현율=0 → 평균 0.5 (높아 보임)
- 조화: 같은 경우 → 0 (한쪽이 0이면 0)

**한쪽으로 치우친 모델을 자동 페널티**.

## 5️⃣ 임곗값 조정 [↑](#toc)

> `predict()` 의 결과는 사실 **임곗값 0.5** 기준의 이진 변환. 임곗값을 바꾸면 정밀도·재현율이 트레이드오프.

### `predict_proba` 사용

```python
proba = model.predict_proba(X_test)[:, 1]   # 양성 클래스 확률
pred_custom = (proba > 0.3).astype(int)     # 임곗값 0.3
```

### Precision-Recall 곡선

```python
from sklearn.metrics import precision_recall_curve
prec, rec, thr = precision_recall_curve(y_test, proba)
```

> 💡 **비즈니스 결정 예**: 재현율 90% 이상 보장이 필요하다면? 곡선에서 재현율=0.9 인 점의 임곗값을 채택.

## 6️⃣ ROC 곡선과 AUC [↑](#toc)

> **임곗값을 0~1로 모두 바꿔보며 그린 곡선**. 모델 자체의 분별력 평가.

### ROC 곡선

- X축: **FPR** (False Positive Rate) = FP / (FP + TN)
- Y축: **TPR** (True Positive Rate, 재현율과 동일)
- 좌상단으로 갈수록 좋음

### AUC (Area Under Curve)

곡선 아래 면적. 0.5 = 동전 던지기, 1.0 = 완벽.

| AUC 구간 | 해석 |
|---|---|
| 0.9 ~ 1.0 | 매우 우수 |
| 0.8 ~ 0.9 | 우수 |
| 0.7 ~ 0.8 | 양호 |
| 0.6 ~ 0.7 | 빈약 |
| ~ 0.5 | 무의미 |

```python
from sklearn.metrics import roc_auc_score, roc_curve
auc = roc_auc_score(y_test, proba)
fpr, tpr, thr = roc_curve(y_test, proba)
```

> ⚠️ **AUC가 만능은 아님**: 극단적으로 불균형한 데이터에서는 **PR-AUC**가 더 정직.

## 7️⃣ 실습: 피마 인디언 당뇨병 [↑](#toc)

> 책 3.6절. 768개 환자 데이터로 당뇨병 여부 예측.

### 데이터 특이점

| 컬럼 | 값 0 의 의미 |
|---|---|
| Glucose (혈당) | 0은 불가능 — **결측치** |
| BloodPressure | 0은 불가능 — **결측치** |
| SkinThickness, Insulin, BMI | 0은 의심 — 결측치로 처리 |

> 책의 핵심 메시지: **"0 = 결측치"** 라는 도메인 지식을 모르면 평가 지표가 거짓말. 데이터 이해의 중요성.

### 실습 흐름 (`3.6_colab.ipynb`)

| 단계 | 학습 포인트 |
|---|---|
| 1. 데이터 로드 + `describe` | 0이 들어간 컬럼 발견 |
| 2. 0 → NaN 대치 → 평균 대치 | 결측치 처리 전후 비교 |
| 3. 로지스틱 회귀 학습 | 분류 기본 알고리즘 |
| 4. 정확도 / 정밀도 / 재현율 / F1 | 모두 출력 |
| 5. 임곗값 0.3, 0.4, 0.5, 0.6 비교 | 트레이드오프 체감 |
| 6. ROC AUC 출력 + 곡선 시각화 | 임곗값 독립 평가 |

## 8️⃣ 정리와 체크리스트 [↑](#toc)

### 한 문장 요약

> **정확도는 균형 데이터에서만 신뢰. 불균형 데이터엔 정밀도·재현율·F1·AUC** — 도메인이 어느 오류를 더 비싸게 보는지에 따라 지표를 고른다.

### 자기 점검

- [ ] 사기 탐지 시나리오에서 정확도 99%가 왜 위험한지 설명
- [ ] 혼동 행렬 4칸을 한 번에 그릴 수 있다
- [ ] 스팸 필터 vs 암 진단 — 어느 지표를 우선할지 즉답
- [ ] `predict_proba` 와 `predict` 의 관계를 안다
- [ ] ROC 곡선의 X/Y축이 무엇인지 안다
- [ ] AUC 0.5 와 1.0 의 의미를 안다

### 흔한 오해

| 오해 | 사실 |
|---|---|
| "정확도가 95% 이상이면 좋은 모델" | 클래스 비율 확인 필수 |
| "F1이 항상 최고 지표" | 도메인이 어느 오류를 더 싫어하느냐가 우선 |
| "AUC 0.99는 거의 완벽" | 극단적 불균형 데이터에선 함정 — PR-AUC 같이 보기 |
| "임곗값 0.5는 절대 불변" | 비즈니스 요구에 맞춰 조정 가능 |
