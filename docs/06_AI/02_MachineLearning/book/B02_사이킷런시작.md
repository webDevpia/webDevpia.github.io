## 1️⃣ 사이킷런 통일 API [↑](#toc)

> 사이킷런의 가장 큰 강점은 **모든 알고리즘이 같은 인터페이스**를 따른다는 점.

### Estimator 패턴 (3줄)

```python
model = SomeAlgorithm(...)     # 1. 모델 선택
model.fit(X_train, y_train)    # 2. 학습
predictions = model.predict(X_test)   # 3. 예측
```

LinearRegression, RandomForestClassifier, KMeans 등 **이름만 바뀌고 패턴은 동일**.

### Transformer 패턴 (전처리)

```python
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)    # 학습 + 변환
X_test_scaled  = scaler.transform(X_test)         # 변환만 (학습 X — 중요!)
```

### 주요 모듈 한눈에

| 모듈 | 역할 |
|---|---|
| `sklearn.datasets` | 내장 예제 데이터 (iris, breast_cancer 등) |
| `sklearn.model_selection` | train_test_split, KFold, GridSearchCV |
| `sklearn.preprocessing` | Scaler, Encoder |
| `sklearn.feature_selection` | 피처 선택 |
| `sklearn.linear_model` / `tree` / `ensemble` / ... | 알고리즘 |
| `sklearn.metrics` | accuracy_score, f1_score, ROC AUC 등 |
| `sklearn.pipeline` | 전처리+모델 묶기 |

## 2️⃣ 붓꽃 첫 분류기 [↑](#toc)

> **iris 데이터** — 150개 꽃, 4개 피처(꽃잎/꽃받침 길이·너비), 3종(품종) — ML 입문의 고전.

### 미니 코드 — 풀버전은 `2.2_colab.ipynb`

```python
from sklearn.datasets import load_iris
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

iris = load_iris()
X, y = iris.data, iris.target

X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=11)

clf = DecisionTreeClassifier(random_state=11)
clf.fit(X_tr, y_tr)
pred = clf.predict(X_te)
print('정확도:', accuracy_score(y_te, pred))   # 약 0.93
```

### 책 2.2절 강조점

- `random_state` 고정 → **재현 가능한 실험** 의 출발
- 정확도 ≠ 모델 평가의 전부 (3장 평가에서 다룸)

## 3️⃣ train_test_split 의 필요성 [↑](#toc)

> 같은 데이터로 학습·평가하면 **과적합(overfitting)** 을 못 잡습니다. 시험문제를 학생에게 미리 주는 셈.

### 책 2.3절 핵심 메시지

```python
# 잘못된 방식 — 같은 데이터로 학습 + 평가
clf.fit(X, y); clf.score(X, y)   # 항상 거의 1.0 (의미 없음)

# 올바른 방식
clf.fit(X_tr, y_tr); clf.score(X_te, y_te)
```

### 주요 인자

| 인자 | 의미 |
|---|---|
| `test_size=0.2` | 테스트 20% (보통 0.2 ~ 0.3) |
| `random_state=N` | 분할 재현용 시드 |
| `stratify=y` | **클래스 비율 유지** (불균형 데이터 필수) |
| `shuffle=True` | 분할 전 섞기 (기본값) |

> ⚠️ **분류 문제에서 `stratify=y` 는 사실상 필수**. 안 쓰면 클래스가 한쪽으로 쏠릴 수 있음.

## 4️⃣ 교차 검증 [↑](#toc)

> "한 번 분할로 평가하면 운에 좌우된다" — 책 2.3절 인용.

### K-Fold 직관

데이터를 K개 조각으로 나누고, **K번 학습/평가**를 돌려서 평균을 본다.

```
[1] [2] [3] [4] [5]    ← 5-Fold
↑테스트=1, 나머지=학습  → 점수1
↑테스트=2, 나머지=학습  → 점수2
... 5회 반복 → 평균
```

### 3가지 API

| API | 특징 | 사용 시점 |
|---|---|---|
| `KFold(n_splits=5)` | 단순 K분할 | 회귀, 균형 분류 |
| `StratifiedKFold(n_splits=5)` | **클래스 비율 유지** | 분류 (기본 선택) |
| `cross_val_score(model, X, y, cv=5)` | 위 둘 자동 + 한 줄 평가 | 빠른 실험 |

```python
from sklearn.model_selection import cross_val_score
scores = cross_val_score(clf, X, y, cv=5, scoring='accuracy')
print(scores, scores.mean())
```

> 💡 **수업 시연 팁**: `KFold`로 직접 루프 돌려 보여준 뒤 `cross_val_score` 한 줄로 줄여서 "사이킷런이 얼마나 편한지" 체감시키기.

<a id="part5"></a>

## 5️⃣ GridSearchCV — 하이퍼파라미터 탐색 [↑](#toc)

> 손으로 `n_estimators=100, 200, 300...` 바꿔보지 말고 **격자(grid)로 자동 탐색**.

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    'max_depth': [3, 5, 7],
    'min_samples_split': [2, 5, 10],
}
gs = GridSearchCV(DecisionTreeClassifier(random_state=11),
                  param_grid, cv=5, scoring='accuracy')
gs.fit(X_tr, y_tr)

print('best:', gs.best_params_, gs.best_score_)
best_model = gs.best_estimator_
```

| 속성 | 의미 |
|---|---|
| `best_params_` | 최적 파라미터 조합 |
| `best_score_` | 최적 조합의 CV 평균 점수 |
| `best_estimator_` | 학습된 최적 모델 |
| `cv_results_` | 모든 조합의 결과(DataFrame 변환 가능) |

> ⚠️ **격자 폭발 주의**: 5종 파라미터 × 각 5개 값 × 5-fold = 3125번 학습. 4장에서 더 효율적인 `RandomizedSearchCV`, `HyperOpt`로 발전.

## 6️⃣ 데이터 인코딩 [↑](#toc)

> ML 모델은 **숫자만** 받습니다. 'male', 'female' → 숫자로 변환 필요.

### LabelEncoder

```python
from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
df['Sex_enc'] = le.fit_transform(df['Sex'])    # male=1, female=0
```

> ⚠️ **위험**: `'서울'=0, '부산'=1, '대구'=2` 처럼 **순서 없는 범주에 LabelEncoder를 쓰면** 모델이 잘못된 대소 관계를 학습. → **OneHotEncoder** 가 정답.

### OneHotEncoder (또는 pd.get_dummies)

```python
from sklearn.preprocessing import OneHotEncoder
oh = OneHotEncoder(sparse_output=False)   # sklearn 1.2+ 키워드
X_oh = oh.fit_transform(df[['Embarked']])
```

**판다스 한 줄 버전**:
```python
df_oh = pd.get_dummies(df, columns=['Embarked', 'Sex'])
```

> 💡 **언제 LabelEncoder vs OneHotEncoder?** 트리 기반 모델(결정트리/RF)은 LabelEncoder도 잘 작동. 선형 모델은 반드시 OneHotEncoder.

## 7️⃣ 피처 스케일링 [↑](#toc)

> "Age는 0~100, Fare는 0~500" — 단위가 다르면 거리 기반 모델(KNN, SVM)이 큰 값에 끌려감.

### 3종 스케일러 비교

| Scaler | 변환 후 분포 | 이상치 영향 |
|---|---|---|
| **StandardScaler** | 평균 0, 표준편차 1 | 큼 |
| **MinMaxScaler** | 0 ~ 1 사이 | 매우 큼 (max에 좌우) |
| **RobustScaler** | 중앙값 0, IQR=1 | 작음 (이상치 강함) |

```python
from sklearn.preprocessing import StandardScaler
sc = StandardScaler()
X_tr_s = sc.fit_transform(X_tr)
X_te_s = sc.transform(X_te)        # ⚠️ fit_transform 아님!
```

> ⚠️ **반드시 학습 데이터로만 fit**. 테스트 데이터로 fit하면 **데이터 누수**.

## 8️⃣ 데이터 누수와 Pipeline [↑](#toc)

> 책 2.5절 후반의 핵심 — 초보자가 가장 자주 실수.

### 누수의 전형적 패턴

```python
# ❌ 잘못 — 전체 데이터로 스케일링
X_all_scaled = scaler.fit_transform(X)
X_tr, X_te = train_test_split(X_all_scaled, ...)   # 테스트 정보가 train 통계에 섞임
```

### Pipeline 로 자동 차단

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.tree import DecisionTreeClassifier

pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', DecisionTreeClassifier(random_state=11)),
])
pipe.fit(X_tr, y_tr)        # 학습 데이터로만 fit
pipe.predict(X_te)          # 자동으로 transform → predict
```

`cross_val_score(pipe, X, y)` 처럼 쓰면 **각 폴드마다 train 통계로만 fit** 되어 누수 자동 방지.

## 🔟 정리와 체크리스트 [↑](#toc)

### 한 문장 요약

> **사이킷런의 모든 ML 도구는 `fit/predict` 또는 `fit_transform/transform` 패턴이다.** 그 위에 train/test 분리, 교차 검증, GridSearchCV, Pipeline이 얹힌다.

### 자기 점검

- [ ] Estimator와 Transformer 의 차이를 설명할 수 있다
- [ ] `train_test_split(stratify=y)` 의 `stratify` 가 무엇을 보장하는지 안다
- [ ] `KFold` 와 `StratifiedKFold` 를 언제 골라 써야 하는지 설명할 수 있다
- [ ] `GridSearchCV` 의 `best_params_` 와 `best_estimator_` 의 차이를 안다
- [ ] LabelEncoder를 명목형 다범주에 적용했을 때 발생할 문제를 예시로 든다
- [ ] StandardScaler 를 테스트 데이터에 fit하면 안 되는 이유를 설명한다
- [ ] Pipeline 으로 누수를 막는 방식 한 줄로 코드 작성

### 흔한 오해

| 오해 | 사실 |
|---|---|
| "train_test_split 만 잘하면 평가는 끝" | 한 번 분할은 운에 좌우. 교차 검증으로 확인 |
| "GridSearchCV는 만능" | 격자가 크면 시간 폭발. RandomizedSearch / HyperOpt 필요 (4장) |
| "스케일링은 항상 해야 함" | 트리 기반 모델은 스케일 무관. 거리/그래디언트 모델만 필수 |
| "OneHotEncoder는 메모리 낭비" | `sparse_output=True` 로 희소 행렬 사용하면 절감. 8장 텍스트에서 다시 봄 |
