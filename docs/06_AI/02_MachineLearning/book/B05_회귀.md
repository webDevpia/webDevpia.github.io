## 1️⃣ 회귀 개요 [↑](#toc)

> **연속 값 예측**. 집값, 매출, 기온 등. 분류는 "어느 종류"였다면 회귀는 "얼마".

### 회귀 평가 지표

| 지표 | 식 | 특징 |
|---|---|---|
| **MAE** | 평균 절대 오차 | 이상치에 강함 |
| **MSE** | 평균 제곱 오차 | 이상치에 민감 |
| **RMSE** | √MSE | 단위가 원본과 같음 (해석 쉬움) |
| **R²** | 1 - SS_res/SS_tot | 0~1 (1=완벽), 음수도 가능 |

> ⚠️ **sklearn 1.6+ 호환**: `mean_squared_error(y, p, squared=False)` 인자 제거됨. `np.sqrt(mean_squared_error(y, p))` 또는 `root_mean_squared_error(y, p)` 사용.

## 2️⃣ 경사 하강법 [↑](#toc)

> "비용 함수의 산을 내려가며 최저점을 찾는다" — 책 5.3절.

### 비용 함수 (MSE)

$$ J(w, b) = \frac{1}{n} \sum (y_i - (wx_i + b))^2 $$

### 업데이트 규칙

$$ w := w - \alpha \frac{\partial J}{\partial w} $$
$$ b := b - \alpha \frac{\partial J}{\partial b} $$

- α (learning rate): 너무 크면 발산, 너무 작으면 느림

### 직접 구현 (책의 핵심)

```python
def gradient_descent(X, y, lr=0.01, iters=1000):
    w, b = 0, 0
    n = len(X)
    for _ in range(iters):
        pred = w * X + b
        dw = -(2/n) * np.sum(X * (y - pred))
        db = -(2/n) * np.sum(y - pred)
        w -= lr * dw
        b -= lr * db
    return w, b
```

> 💡 **수업 시연 팁**: `iters=10, 100, 1000` 으로 바꿔가며 w/b 가 수렴해가는 모습을 그래프로.

## 3️⃣ LinearRegression [↑](#toc)

> 사이킷런은 **정규방정식(Normal Equation)** 해법 사용 (작은 데이터엔 GD보다 빠르고 정확).

```python
from sklearn.linear_model import LinearRegression
lr = LinearRegression()
lr.fit(X_tr, y_tr)
print('계수:', lr.coef_, ' / 절편:', lr.intercept_)
```

> ⚠️ **`load_boston` 제거됨 (sklearn 1.2+)**: 책은 보스턴 주택 데이터로 시작하지만 sklearn에서 윤리적 이유로 삭제. Colab 노트북에선 `fetch_california_housing` 으로 자동 대체.

### 계수 해석

`coef_[i]` = i번째 피처가 1 증가할 때 y가 얼마나 변하는가. **피처 스케일이 다르면 비교 불가** → 스케일링 필수.

## 4️⃣ 다항 회귀 [↑](#toc)

> 직선이 아닌 곡선. `x²`, `x³` 등을 피처로 추가하여 선형 모델로 풀기.

```python
from sklearn.preprocessing import PolynomialFeatures
from sklearn.pipeline import Pipeline

poly_lr = Pipeline([
    ('poly', PolynomialFeatures(degree=3)),
    ('lr', LinearRegression()),
])
poly_lr.fit(X_tr, y_tr)
```

> ⚠️ **degree가 너무 크면 과적합**. 책은 degree=3 이상에서 R²가 학습에서 1.0 가까이 가도 테스트는 추락하는 예제로 경고.

## 5️⃣ 규제: Ridge / Lasso / ElasticNet [↑](#toc)

> 과적합을 막기 위해 **계수가 커지는 것에 페널티**.

### 비교

| 모델 | 페널티 | 효과 |
|---|---|---|
| **Ridge** | $\alpha \sum w_i^2$ (L2) | 계수가 작아짐 (0은 안 됨) |
| **Lasso** | $\alpha \sum \|w_i\|$ (L1) | **계수가 0이 됨 (피처 선택 효과)** |
| **ElasticNet** | L1 + L2 | 두 효과 절충 |

```python
from sklearn.linear_model import Ridge, Lasso, ElasticNet
Ridge(alpha=10).fit(X_tr, y_tr)
Lasso(alpha=0.1).fit(X_tr, y_tr)
ElasticNet(alpha=0.1, l1_ratio=0.5).fit(X_tr, y_tr)
```

> 💡 **alpha 직관**: 클수록 페널티 강함 → 계수가 더 작아짐. `LassoCV`, `RidgeCV` 로 자동 탐색.

## 6️⃣ 로지스틱 회귀 [↑](#toc)

> 이름은 회귀지만 **분류** 알고리즘. 선형 출력을 **시그모이드**로 0~1 확률로.

$$ p = \sigma(wx + b) = \frac{1}{1 + e^{-(wx+b)}} $$

```python
from sklearn.linear_model import LogisticRegression
log_clf = LogisticRegression(C=1.0, solver='liblinear')
log_clf.fit(X_tr, y_tr)
proba = log_clf.predict_proba(X_te)[:, 1]
```

| 파라미터 | 의미 |
|---|---|
| `C` | **1/alpha** (작을수록 강한 규제) |
| `solver` | 최적화 방법 (작은 데이터 `'liblinear'`, 큰 데이터 `'lbfgs'`) |
| `penalty` | `'l1'`, `'l2'`, `'elasticnet'`, `'none'` |

> 💡 **사실상 선형 모델 분류의 디폴트**. 빠르고 해석 가능.

## 7️⃣ 회귀 트리 [↑](#toc)

> 분류 트리와 같은 구조, **리프 값이 평균(연속값)**.

```python
from sklearn.tree import DecisionTreeRegressor

models = [DecisionTreeRegressor(), RandomForestRegressor(),
```

## 9️⃣ 실습: Kaggle House Price [↑](#toc)

> 책 5.10절. **책의 회귀 백미**. 79개 피처, 1460개 행. 캐글 실제 대회 데이터.

### 7단계 전처리 흐름

1. **타깃 로그 변환** — `np.log1p(SalePrice)`
2. **수치형 결측치** — 평균/중앙값 대치
3. **범주형 결측치** — `'None'` 또는 최빈값
4. **범주형 인코딩** — `pd.get_dummies`
5. **수치형 왜도 조정** — `abs(skew) > 1` 컬럼만 `log1p`
6. **이상치 제거** — `GrLivArea` 가 비정상 큰 행 제거
7. **모델 학습** — Ridge / Lasso / ElasticNet + XGBoost / LightGBM

### 앙상블 마무리

- 단순 가중평균
- 스태킹 (4.11과 연결)

> 💡 **이 챕터를 끝내면 캐글 회귀 대회에 처음 도전할 수 있는 수준**.

### 흔한 오해

| 오해 | 사실 |
|---|---|
| "선형 회귀는 옛날 알고리즘이라 안 씀" | 해석 가능성·속도로 여전히 디폴트. 규제 버전이 산업 표준 |

| "log 변환은 왜도가 클 때만" | 가격·매출 같은 양수 타깃엔 사실상 표준 |
