## 2️⃣ 결정 트리 [↑](#toc)

> "스무고개 게임" — 질문을 차례로 던져 답을 좁힌다.

### 분할 기준 (불순도)

| 척도 | 식 | 의미 |
|---|---|---|
| 지니 불순도 | $1 - \sum p_i^2$ | 한 노드에서 잘못 분류될 확률 |
| 엔트로피 | $-\sum p_i \log_2 p_i$ | 정보 이론 기반 |

> 책의 핵심 메시지: **계산은 컴퓨터가 하지만 직관적으로 "한쪽으로 쏠릴수록 좋다"**.

### 하이퍼파라미터

| 파라미터 | 의미 | 권장 시작값 |
|---|---|---|
| `max_depth` | 최대 깊이 | 5 ~ 10 (과적합 방지) |
| `min_samples_split` | 분할 최소 샘플 | 5 ~ 20 |
| `min_samples_leaf` | 리프 최소 샘플 | 1 ~ 10 |
| `max_features` | 분할 시 고려할 피처 수 | sqrt(n) (분류) |

### 시각화

```python
from sklearn.tree import plot_tree
plot_tree(clf, feature_names=feature_names, class_names=['0','1'], filled=True)
```

> 🛠 **수업 활용**: 깊이 2 트리를 그려서 "이 트리가 어떻게 결정하는가" 학생이 직접 따라가게.

### 피처 중요도

```python
import pandas as pd
pd.Series(clf.feature_importances_, index=feature_names).sort_values(ascending=False)
```

## 3️⃣ 앙상블 학습 개요 [↑](#toc)

> "여러 약한 학습기를 모아 강한 학습기를 만든다" — 책 4.3절.

### 3대 방식 비교

| 방식 | 직관 | 대표 알고리즘 | 학습 방식 |
|---|---|---|---|
| **Bagging** | 비슷한 모델 여러 개를 **병렬**로 학습, 평균 | Random Forest | 부트스트랩 샘플링 |
| **Boosting** | 앞 모델의 오답에 집중하며 **순차** 학습 | GBM / XGBoost / LightGBM | 잔차 학습 |
| **Stacking** | 여러 모델의 예측을 **메타 모델의 입력**으로 사용 | StackingClassifier | 2단계 학습 |

### 직관 비유

- Bagging: **반장 선거 다수결** — 각 위원이 독립적으로 결정 후 평균
- Boosting: **시험 오답 노트** — 틀린 문제만 다시 풀이
- Stacking: **재판소의 판사단** — 각자 판결 후 대법관이 최종 결정

## 4️⃣ Random Forest [↑](#toc)

> "랜덤"이 두 군데 들어감.
> 1. **샘플 랜덤** (Bootstrap) — 데이터에서 복원 추출
> 2. **피처 랜덤** — 분할 시 일부 피처만 사용

### 코드

```python
from sklearn.ensemble import RandomForestClassifier
rf = RandomForestClassifier(n_estimators=100, max_depth=10,
                            random_state=11, n_jobs=-1)
rf.fit(X_tr, y_tr); rf.score(X_te, y_te)
```

### 하이퍼파라미터

| 파라미터 | 의미 | 권장 |
|---|---|---|
| `n_estimators` | 트리 개수 | 100~500 (많을수록 안정, 시간 증가) |
| `max_features` | 분할 피처 수 | `'sqrt'` (분류 기본) |
| `n_jobs=-1` | 모든 CPU 사용 | 권장 |

> 💡 **장점**: 거의 튜닝 없이 좋은 성능. 정형 데이터의 **기본 선택**.

## 5️⃣ GBM [↑](#toc)

> Gradient Boosting — 앞 트리의 **잔차(오차)** 를 다음 트리가 학습.

```python
from sklearn.ensemble import GradientBoostingClassifier
gbm = GradientBoostingClassifier(n_estimators=100, learning_rate=0.1, max_depth=3)
```

### 핵심 파라미터 트리오

| 파라미터 | 의미 |
|---|---|
| `n_estimators` | 트리 개수 = 부스팅 반복 횟수 |
| `learning_rate` | 한 트리의 영향력 (작을수록 보수적) |
| `max_depth` | 트리 깊이 (보통 3~5의 얕은 트리) |

> ⚠️ **단점**: 사이킷런 GBM은 느리고 메모리 많이 씀. XGBoost / LightGBM 대안 사용 권장.

## 6️⃣ XGBoost [↑](#toc)

> "Extreme Gradient Boosting" — GBM의 산업용 최적화 버전.

### 사이킷런 래퍼

```python
from xgboost import XGBClassifier

xgb_clf = XGBClassifier(
    n_estimators=400, learning_rate=0.1, max_depth=3,
    early_stopping_rounds=50, eval_metric='logloss'  # ⚠️ XGBoost 2.x: 생성자에서 지정
)
xgb_clf.fit(X_tr, y_tr, eval_set=[(X_val, y_val)], verbose=True)
```

> ⚠️ **Colab 마이그레이션 안내**: 원래 책 코드는 `.fit(early_stopping_rounds=50, eval_metric='logloss', ...)` 였지만 XGBoost 2.x부터 생성자로 이동. Colab 노트북은 이미 자동 수정됨.

### 핵심 하이퍼파라미터 (책 권장)

| 파라미터 | 의미 | 시작값 |
|---|---|---|
| `max_depth` | 트리 깊이 | 3 ~ 6 |
| `min_child_weight` | 분할 최소 가중치 | 1 ~ 6 |
| `subsample` | 행 샘플링 비율 | 0.7 ~ 1.0 |
| `colsample_bytree` | 트리별 피처 샘플링 | 0.7 ~ 1.0 |
| `reg_alpha` (L1), `reg_lambda` (L2) | 규제 | 0 / 1 부터 |

### Early Stopping

검증 점수가 N라운드 동안 안 개선되면 멈춤 → **과적합 방지 + 시간 절약**.

## 7️⃣ LightGBM [↑](#toc)

> XGBoost와 거의 같은 성능, **더 빠름** (히스토그램 기반 + 리프 중심 트리 성장).

### 사이킷런 래퍼 (callbacks 사용)

```python
from lightgbm import LGBMClassifier
import lightgbm as lgb

lgbm = LGBMClassifier(n_estimators=400, learning_rate=0.05)
lgbm.fit(X_tr, y_tr,
         eval_set=[(X_val, y_val)],
         callbacks=[lgb.early_stopping(stopping_rounds=50),
                    lgb.log_evaluation(period=100)])  # ⚠️ LightGBM 4.x
```

> ⚠️ **Colab 마이그레이션 안내**: 원래는 `.fit(early_stopping_rounds=50, eval_metric='logloss', verbose=True)` 였지만 LightGBM 4.x부터 callbacks 사용. Colab 노트북은 이미 자동 수정됨.

### XGBoost vs LightGBM 선택 가이드

| 기준 | XGBoost | LightGBM |
|---|---|---|
| 속도 | 보통 | 매우 빠름 |
| 메모리 | 보통 | 적게 사용 |
| 작은 데이터 | 무난 | 과적합 위험 ↑ |
| 대용량 데이터 | 무거움 | 권장 |
| GPU | `device='cuda'` | `device='gpu'` |

## 8️⃣ HyperOpt — 베이지안 최적화 [↑](#toc)

> GridSearchCV는 격자 폭발. RandomizedSearchCV는 운에 의존. **HyperOpt는 과거 결과를 학습해 다음 후보를 똑똑하게 고름**.

### 4단계 흐름 (책 4.8절)

1. **`search_space` 정의**: `hp.quniform`, `hp.loguniform` 등으로 범위
2. **`objective_func` 작성**: `search_space` 한 세트 → 모델 학습 → **음수 점수** 반환
3. **`fmin` 호출**: `algo=tpe.suggest`, `max_evals=50` 등
4. **best 파라미터로 재학습**: `int(best['max_depth'])` 형변환 주의

```python
from hyperopt import hp, fmin, tpe, Trials

search_space = {
    'max_depth':        hp.quniform('max_depth', 3, 10, 1),
    'learning_rate':    hp.loguniform('learning_rate', -3, -1),
}

def objective(space):
    clf = XGBClassifier(
        max_depth=int(space['max_depth']),
        learning_rate=space['learning_rate'],
        early_stopping_rounds=50, eval_metric='auc',
    )
    clf.fit(X_tr, y_tr, eval_set=[(X_val, y_val)], verbose=False)
    return -1 * roc_auc_score(y_val, clf.predict_proba(X_val)[:, 1])

best = fmin(objective, search_space, algo=tpe.suggest, max_evals=50,
            trials=Trials())
print(best)
```

> 💡 **음수 반환 트릭**: HyperOpt는 최소화 알고리즘. AUC 같은 "높을수록 좋은" 지표는 음수로 뒤집어 반환.

## 9️⃣ 실습: 산탄데르 / 신용카드 사기 검출 [↑](#toc)

### 4.9 산탄데르 고객 만족 예측

- **불균형 데이터** (만족 vs 불만족 비율 차이)
- XGBoost / LightGBM 기본 → HyperOpt 튜닝 → AUC 비교

### 4.10 신용카드 사기 검출 (책의 백미)

- 30만 거래, 사기 비율 **0.17%** (극단적 불균형)
- 다양한 기법:
  - 결측치 없음 (이미 PCA 변환된 데이터)
  - **언더샘플링 / 오버샘플링 (SMOTE)** 시도
  - **로그 변환** (Amount 컬럼 왜도)
  - **이상치 제거** (IQR 기반)
- **Recall이 핵심 지표** (사기를 놓치면 안 됨)

> 🛠 **수업 시연 흐름**: 데이터 불균형 → 평가지표 선택 → 샘플링 → 모델 → 비교. 약 2시간 시연.

## 🔟 스태킹 앙상블 [↑](#toc)

> "여러 모델의 예측을 새 피처로 만들어 메타 모델이 학습."

### CV 기반 스태킹 (책 4.11절 후반)

오버피팅을 막기 위해 **K-Fold 안에서 예측값 생성**.

```python
from sklearn.ensemble import StackingClassifier
from sklearn.linear_model import LogisticRegression

estimators = [
    ('rf', RandomForestClassifier(...)),
    ('xgb', XGBClassifier(...)),
    ('lgbm', LGBMClassifier(...)),
]
stk = StackingClassifier(estimators=estimators,
                         final_estimator=LogisticRegression(),
                         cv=5, n_jobs=-1)
stk.fit(X_tr, y_tr)
```

> 💡 **스태킹은 마지막 1% 짜내기 용**. 처음부터 스태킹 쓰지 말고, 단일 모델 튜닝 끝낸 뒤 시도.

## 1️⃣1️⃣ 정리와 체크리스트 [↑](#toc)

### 한 문장 요약

> **트리 → RF → GBM/XGB/LGBM 으로 성능 + 속도가 점차 개선되고, HyperOpt 로 튜닝하며 스태킹으로 마무리한다.**

### 자기 점검

- [ ] 결정 트리의 지니/엔트로피 의미 설명
- [ ] Bagging / Boosting / Stacking 한 줄로 비유 가능
- [ ] Random Forest의 두 가지 랜덤성을 말한다
- [ ] XGBoost / LightGBM 차이를 표로 정리
- [ ] LightGBM 4.x 에서 `early_stopping_rounds` 를 어디에 어떻게 넣는지 안다
- [ ] HyperOpt 의 4단계 흐름 + 음수 트릭 설명
- [ ] 신용카드 사기 검출에서 왜 Recall 이 핵심인지 안다
- [ ] 스태킹은 언제 쓸지 (그리고 언제 안 쓸지) 안다

### 흔한 오해

| 오해 | 사실 |
|---|---|
| "XGBoost가 LightGBM보다 항상 우수" | 대용량 데이터에선 LightGBM이 더 빠르고 더 좋음 |
| "HyperOpt 가 GridSearchCV 보다 항상 빠름" | 함수 평가 1회가 비싸야 의미. 가벼운 모델은 Grid도 OK |
| "스태킹 = 무조건 성능 향상" | 1~3% 개선 기대. 단일 모델 튜닝이 우선 |
| "n_estimators는 클수록 좋음" | 늘릴수록 시간 ↑, early_stopping 으로 자동 결정 |
