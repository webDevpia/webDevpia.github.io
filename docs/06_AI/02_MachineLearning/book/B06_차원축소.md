## 학습 목표

- **차원의 저주(Curse of Dimensionality)** 가 무엇이고 왜 문제인지 설명할 수 있다
- **PCA** 의 핵심 아이디어 ("가장 분산이 큰 방향") 를 시각화로 설명할 수 있다
- PCA 적용 시 **반드시 스케일링이 선행** 되어야 하는 이유를 안다
- **LDA / SVD / NMF** 의 PCA 대비 차이를 구분할 수 있다
- 설명 분산 비율(`explained_variance_ratio_`) 로 차원 결정 방법을 안다
- 4가지 알고리즘을 적용 후 분류·시각화 성능을 비교한다

## 1️⃣ 차원의 저주 [↑](#toc)

> 피처가 많아질수록 모델이 학습할 데이터 공간이 **기하급수적으로 커짐**.

| 차원 | 단위 격자 한 변에 필요한 샘플 수 (10개 균등 분포 가정) |
|---|---|
| 1 | 10 |
| 2 | 100 |
| 5 | 100,000 |
| 10 | 10 billion |

### 부작용

- 모델 학습 시간 ↑
- 거리 기반 알고리즘(KNN)이 무의미해짐 (모든 점이 비슷한 거리)
- 시각화 불가 (4차원 이상)

### 해법 두 갈래

| 방법 | 설명 |
|---|---|
| **피처 선택 (Feature Selection)** | 원본 피처 중 중요한 것만 선택 |
| **피처 추출 (Feature Extraction)** | 원본을 변환해 **새 피처(주성분)** 생성 ← 이 장의 주제 |

## 2️⃣ PCA — 가장 분산 큰 방향 [↑](#toc)

> "데이터가 가장 퍼져 있는 방향이 가장 정보가 많다."

### 직관

2차원 점 구름이 대각선으로 길게 늘어졌다면:
- **새 축 1** = 대각선 방향 (정보 거의 다)
- **새 축 2** = 수직 방향 (잔여)

→ 새 축 1만으로도 데이터를 거의 표현. 차원 축소.

### 코드

```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

X_scaled = StandardScaler().fit_transform(X)   # ⚠️ 스케일링 필수
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

print(pca.explained_variance_ratio_)
# [0.72, 0.23] → 두 축으로 95% 설명
```

### `n_components` 결정법

| 방법 | 코드 |
|---|---|
| 정수 (절대 개수) | `PCA(n_components=10)` |
| 0~1 비율 (분산 보존 목표) | `PCA(n_components=0.95)` ← 95% 분산 자동 |

> ⚠️ **반드시 스케일링 선행**: PCA는 분산 기반이라 단위 큰 피처에 끌려감.

## 3️⃣ LDA — 분류를 위한 축 [↑](#toc)

> PCA는 "분산"만 봄. **LDA**는 **클래스를 가장 잘 분리하는 축**.

### PCA vs LDA

| 기준 | PCA | LDA |
|---|---|---|
| 학습 | 비지도 (y 없음) | 지도 (y 필요) |
| 목적 | 분산 최대화 | **클래스 간 분산 / 클래스 내 분산** 최대화 |
| 사용 | 시각화, 비지도 전처리 | 분류 전처리 |

```python
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
lda = LinearDiscriminantAnalysis(n_components=2)
X_lda = lda.fit_transform(X_scaled, y)   # y 필요!
```

> 💡 **언제 LDA?** 클래스 정보가 있고, 분류기를 그 뒤에 둘 때.

## 4️⃣ SVD — 행렬 분해 일반화 [↑](#toc)

> Singular Value Decomposition. PCA의 수학적 일반화.

$$ A = U \Sigma V^T $$

- 임의의 m×n 행렬 분해
- **TruncatedSVD**: 희소 행렬(텍스트 TF-IDF 등)에도 적용 가능

```python
from sklearn.decomposition import TruncatedSVD
svd = TruncatedSVD(n_components=100)
X_svd = svd.fit_transform(X_tfidf)   # 희소 행렬도 OK
```

> 💡 **추천 시스템에서 핵심**. 9장 잠재 요인 협업 필터링과 직결.

## 5️⃣ NMF — 음수 없는 분해 [↑](#toc)

> Non-negative Matrix Factorization. **모든 값이 ≥ 0** 인 분해.

$$ A = W H, \quad W, H \geq 0 $$

### 어디 쓰나?

- 텍스트 토픽 모델링 (8장)
- 이미지 분해 (얼굴 → 부분)
- 추천 시스템

```python
from sklearn.decomposition import NMF
nmf = NMF(n_components=10, init='random', random_state=11)
W = nmf.fit_transform(X)   # X >= 0
H = nmf.components_
```

> ⚠️ **X에 음수가 있으면 에러**. 카운트/TF-IDF 등 자연스럽게 양수인 데이터에 적합.

## 7️⃣ 정리와 체크리스트 [↑](#toc)

### 한 문장 요약

> **PCA는 분산 최대화, LDA는 클래스 분리, SVD는 일반화, NMF는 음수 없는 분해.** 모두 "원본보다 적은 차원으로 정보 보존".

### 자기 점검

- [ ] 차원의 저주 한 줄로 설명
- [ ] PCA 적용 전 왜 스케일링이 필수인지 안다
- [ ] `explained_variance_ratio_` 가 무엇인지 안다
- [ ] PCA / LDA 차이 (지도 vs 비지도)
- [ ] TruncatedSVD가 PCA와 다른 이유 (희소 행렬 지원)
- [ ] NMF가 텍스트 토픽 모델링에 쓰이는 이유

### 흔한 오해

| 오해 | 사실 |
|---|---|
| "PCA는 항상 성능 향상" | 분류 성능 떨어지기도 함. 시각화/속도 목적 우선 |
| "n_components 는 항상 2 (시각화용)" | 0.95 비율 자동 결정 + 모델 입력으로 사용이 더 흔함 |
| "LDA는 분류 알고리즘" | 분류기로도, **차원 축소기로도** 모두 사용 가능 |
