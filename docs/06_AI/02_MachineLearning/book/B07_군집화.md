## 2️⃣ K-Means 직관과 알고리즘 [↑](#toc)

> **K개 깃발(센트로이드) 을 꽂고, 가까운 점끼리 팀을 만든다.**

### 알고리즘 4단계

1. K개 중심점 무작위 초기화
2. 각 점을 가장 가까운 중심에 배정
3. 각 그룹의 평균으로 중심 갱신
4. 변화 없을 때까지 반복

### 코드 (sklearn 1.4+ 호환)

```python
from sklearn.cluster import KMeans
km = KMeans(n_clusters=3, n_init=10, random_state=11)   # ⚠️ n_init=10 명시
km.fit(X_scaled)
labels = km.labels_
centers = km.cluster_centers_
```

> ⚠️ **`n_init=10` 명시**: sklearn 1.4부터 기본값이 변경되어 경고 발생. Colab 노트북은 이미 자동 추가됨.

> 💡 **반드시 스케일링 후 적용** (거리 기반이라 단위에 민감).

## 3️⃣ K 결정: 엘보우와 실루엣 [↑](#toc)

> "K를 몇으로 할까?" — 두 가지 접근.

### 엘보우 방법

`inertia_` (군집 내 거리 합) 가 급감 후 완만해지는 지점.

```python
inertias = []
for k in range(2, 11):
    km = KMeans(n_clusters=k, n_init=10, random_state=11).fit(X)
    inertias.append(km.inertia_)
plt.plot(range(2, 11), inertias, 'o-')
```

### 실루엣 분석

각 점에 대해 (다른 군집 평균 거리 - 같은 군집 평균 거리) / 큰 값. **1에 가까울수록 좋음**.

```python
from sklearn.metrics import silhouette_score
silhouette_score(X, labels)   # 전체 평균
```

> 💡 **두 가지 같이 보기**: 엘보우로 후보 K → 실루엣으로 검증.

## 4️⃣ Mean Shift [↑](#toc)

> **K를 안 정해도 됨**. 데이터 밀도가 높은 곳을 자동 찾기.

```python
from sklearn.cluster import MeanShift, estimate_bandwidth
bandwidth = estimate_bandwidth(X, quantile=0.2)
ms = MeanShift(bandwidth=bandwidth)
ms.fit(X)
```

- `bandwidth` = 영향력 반경 (작을수록 군집 많아짐)
- 이미지 세분화에 자주 쓰임

## 5️⃣ GMM (Gaussian Mixture Model) [↑](#toc)

> "데이터는 여러 정규분포의 혼합이다" 가정.

### KMeans vs GMM

| 기준 | KMeans | GMM |
|---|---|---|
| 군집 모양 | 원형 (구) | **타원** 가능 |
| 할당 | 하드 (한 군집) | **소프트** (확률) |
| 가정 | 거리 | 분포 |

```python
from sklearn.mixture import GaussianMixture
gmm = GaussianMixture(n_components=3, random_state=11)
gmm.fit(X_scaled)
labels = gmm.predict(X_scaled)
probs = gmm.predict_proba(X_scaled)   # 군집별 확률
```

> 💡 **GMM이 KMeans보다 나은 경우**: 군집이 길쭉하거나 회전된 모양일 때.

## 6️⃣ DBSCAN [↑](#toc)

> Density-Based Spatial Clustering. **밀도 기반** — 형태가 자유롭고 노이즈 자동 분리.

### 두 파라미터

| 파라미터 | 의미 |
|---|---|
| `eps` | 한 점의 영향 반경 |
| `min_samples` | 한 군집 최소 점 수 |

```python
from sklearn.cluster import DBSCAN
db = DBSCAN(eps=0.5, min_samples=5)
labels = db.fit_predict(X_scaled)
# label == -1 인 점은 노이즈 (이상치)
```

### KMeans vs DBSCAN

| 기준 | KMeans | DBSCAN |
|---|---|---|
| K 지정 | 필요 | 불필요 |
| 군집 모양 | 원형 | **임의의 모양** |
| 노이즈 | 강제 배정 | -1로 분리 |
| 파라미터 튜닝 | K | eps, min_samples |

> 💡 **반달/원형 데이터에서 KMeans는 실패, DBSCAN은 성공**. 책의 핵심 예제.

## 7️⃣ 실습: 고객 세분화 [↑](#toc)

> 책 7.6절. Online Retail 데이터 (영국 온라인 쇼핑몰 트랜잭션).

### RFM 분석

| R | Recency | 최근성 (마지막 구매로부터) |
| F | Frequency | 구매 빈도 |
| M | Monetary | 총 구매 금액 |

### 흐름

1. 데이터 로드, 결측·이상치 제거
2. 고객 단위로 R/F/M 집계
3. 로그 변환 (왜도 큰 분포)
4. StandardScaler
5. KMeans 클러스터링
6. 클러스터별 R/F/M 평균 해석 → "VIP / 활성 / 휴면" 등 라벨링

> 💡 **이 패턴은 e-커머스 데이터 분석 표준**. 실무에서 그대로 재사용 가능.

## 8️⃣ 정리와 체크리스트 [↑](#toc)

### 한 문장 요약

> **KMeans = 거리 기반 K개. MeanShift = 밀도 자동 K. GMM = 확률 분포 가정. DBSCAN = 임의 형태 + 노이즈 분리.**

### 자기 점검

- [ ] K-Means 4단계 알고리즘 설명
- [ ] `n_init=10` 을 왜 명시해야 하는지 안다 (sklearn 1.4+)
- [ ] 엘보우와 실루엣 차이
- [ ] GMM이 KMeans보다 나은 데이터 형태 예시
- [ ] DBSCAN의 `eps` 과 `min_samples` 의미
- [ ] 반달 모양 데이터에서 KMeans가 실패하는 이유
- [ ] RFM 분석의 3가지 변수 의미

### 흔한 오해

| 오해 | 사실 |
|---|---|
| "KMeans만 알면 됨" | 데이터 형태 따라 다른 알고리즘 필수 (DBSCAN 등) |
| "실루엣 점수 0.5 이상이면 좋은 군집" | 데이터 특성마다 다름. **상대 비교**가 핵심 |
| "DBSCAN 파라미터는 알아서 맞춰짐" | eps 튜닝이 가장 까다로움. k-distance 그래프 활용 |
| "스케일링 없이도 KMeans 돌아감" | 결과가 큰 단위 피처에 좌우됨 — 의미 없음 |
