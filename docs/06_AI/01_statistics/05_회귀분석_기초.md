# 05장. 회귀분석 기초

<a id="part1"></a>

## 1. 회귀란 — 점들 사이로 직선 긋기 [↑](#toc)

### 비유: 공부시간과 시험점수

> 학생 10명의 데이터를 모았습니다.
> - 공부시간(x): 1, 2, 3, ..., 10
> - 시험점수(y): 점수가 대체로 공부시간에 따라 올라감
>
> 산점도를 그리면 우상향 점들이 보입니다. **이 점들을 가장 잘 가로지르는 직선**을 찾는 게 회귀분석입니다.

```
점수
 100│              ●
    │          ●
  80│       ●  ┄┄┄┄┄ "이 직선을 찾자"
    │   ●  ┄┄┄
  60│ ●┄┄┄┄
    │
  40└─────────────── 공부시간
       2    4    6    8
```

### 회귀의 목적

| 용도 | 예 |
|------|---|
| **예측** | 8시간 공부하면 점수는? |
| **관계 분석** | 공부시간이 1시간 늘면 점수는 얼마 오르나? |
| **변수 효과 측정** | 공부시간이 점수에 미치는 영향은 학원수강보다 큰가? |

### 회귀식의 모양

```
y = β₀ + β₁ × x + ε
└─ 절편   └─ 기울기  └─ 오차(잔차)
```

| 기호 | 이름 | 의미 |
|------|------|------|
| **β₀ (베타 0)** | 절편(intercept) | x=0일 때 y값 (직선이 y축과 만나는 지점) |
| **β₁ (베타 1)** | 기울기(coefficient) | x가 1 늘면 y가 얼마나 변하는지 |
| **ε (엡실론)** | 잔차(residual) | 직선이 설명 못 한 차이 |

> 📌 변수가 여러 개면(다중회귀) `y = β₀ + β₁x₁ + β₂x₂ + ...` 식으로 늘어납니다. 본질은 같음.

## 2. 최소제곱법 — 왜 "제곱"인가 [↑](#toc)

### 직선을 어떻게 "가장 잘" 그을까?

각 점에서 직선까지의 **수직 거리(잔차)** 의 합을 최소화하면 됩니다. 그런데 그냥 합하면 +와 -가 상쇄됩니다.

```
점 위쪽: 잔차 +3
점 아래쪽: 잔차 -3
합: 0 ← 좋은 직선이 아닌데 합이 0?!
```

### 해결: 제곱

```
오차 제곱합 SSE = Σ (y_실제 - y_예측)²
                   └─ 각 점의 잔차를 제곱해서 모두 더함
                       (제곱하면 모두 양수)
```


> 💡 **"제곱"의 부수효과**: 큰 오차는 더 큰 페널티를 받게 됩니다. 작은 오차 여러 개보다 큰 오차 한 개를 모델이 더 무겁게 다룹니다. **아웃라이어에 민감해지는 양날의 검**.

### 최소제곱법의 결과

`SSE`를 최소화하는 β₀, β₁을 **수학적 공식**으로 직접 구할 수 있습니다 (정규방정식). 그래서 선형회귀는 **반복 학습 없이 한 번에** 정답을 찾을 수 있습니다.

> 💡 ML의 다른 모델들(신경망 등)은 정답 공식이 없어 **경사하강법으로 반복 학습**합니다. 선형회귀는 그런 점에서 특별합니다.

## 3. 회귀계수 β의 해석 [↑](#toc)

### 단순 예

> 회귀식: `시험점수 = 50 + 4.5 × 공부시간`

| 해석 | 의미 |
|------|------|
| β₀ = 50 | 공부시간 0일 때 기본 점수 50점 |
| β₁ = 4.5 | **공부시간 1시간 늘 때마다 점수 4.5점 ↑** |

> 💡 **계수 = "x가 1 단위 늘 때 y가 얼마나 변하는지"**. 이 한 줄이 회귀 해석의 전부입니다.

### 다중회귀의 계수 해석

```
시험점수 = 30 + 3.0 × 공부시간 + 8.5 × 학원수강 + 1.2 × 부모학력
```

| 변수 | β | 해석 |
|------|---|------|
| 공부시간 | 3.0 | **다른 변수가 같다면**, 공부 1시간↑ → 점수 3점↑ |
| 학원수강 | 8.5 | 다른 변수가 같다면, 학원 1단위↑ → 점수 8.5점↑ |
| 부모학력 | 1.2 | 다른 변수가 같다면, 부모학력 1단위↑ → 점수 1.2점↑ |

> 📌 **"다른 변수가 같다면"** (ceteris paribus) — 다중회귀 계수 해석의 핵심 조건. 변수들이 서로 영향을 안 미친다는 가정 (모듈 8 다중공선성과 연결).

### 표준화 회귀계수

원본 계수는 단위에 따라 달라집니다. **각 변수를 표준화한 뒤 회귀**하면 계수의 크기로 **상대 중요도**를 비교할 수 있습니다.

> 💡 ML의 **피처 중요도(feature importance)** 의 원시 형태가 표준화 회귀계수입니다.

## 4. R² — 모델이 얼마나 잘 설명하나 [↑](#toc)

### 직관

> **R² = "전체 변동 중 모델이 설명한 비율"**

```
R² = 1 - (SSE / SST)
        └─ 잔차 제곱합   └─ 총 제곱합 (y의 흩어짐 전체)
```

| R² | 의미 |
|----|------|
| 1.0 | 모델이 데이터를 **완벽히** 설명 (잔차 0) |
| 0.8 | 데이터 변동의 80%를 설명 |
| 0.5 | 절반 설명 |
| 0.0 | 평균 예측보다 못함 |
| **음수** | 평균 예측보다도 못함 (매우 나쁨) |

### 함정: R²가 높다고 무조건 좋은 모델은 아니다

- **과적합(overfitting)** : 학습 데이터에서 R²=0.99인데 새 데이터에선 0.3
- **변수가 많을수록 자동으로 올라감** → Adjusted R²로 보정

> 💡 ML 평가는 **테스트 데이터의 R²**(또는 RMSE, MAE)를 봅니다. 학습 R²만 보면 속을 수 있음.

## 5. 실습 — statsmodels로 회귀와 결과 읽기 [↑](#toc)

### Step 1: 데이터 + 회귀 모델

```python
import numpy as np
import pandas as pd
import statsmodels.api as sm

np.random.seed(42)
n = 100
df = pd.DataFrame({
    "study_hours": np.random.uniform(0, 10, n),
    "private_classes": np.random.randint(0, 5, n),
})
# 실제 관계: 점수 = 40 + 4*공부 + 6*학원 + 노이즈
df["score"] = 40 + 4*df["study_hours"] + 6*df["private_classes"] + np.random.normal(0, 5, n)

X = sm.add_constant(df[["study_hours", "private_classes"]])  # 절편 추가
y = df["score"]

model = sm.OLS(y, X).fit()
print(model.summary())
```

### Step 2: summary() 핵심 출력 읽기

```
                            OLS Regression Results
==============================================================================
Dep. Variable:                  score   R-squared:                       0.875
Model:                            OLS   Adj. R-squared:                  0.872
Method:                 Least Squares   F-statistic:                     338.5
Prob (F-statistic):            7.4e-46
==============================================================================
                      coef    std err          t      P>|t|      [0.025      0.975]
------------------------------------------------------------------------------
const               40.1245      1.234     32.51      0.000      37.676      42.573
study_hours          3.9876      0.187     21.32      0.000       3.616       4.359
private_classes      6.0234      0.382     15.76      0.000       5.265       6.781
==============================================================================
```

### 결과 해석 — 어디만 보면 되나

| 위치 | 의미 | 합격 기준 |
|------|------|---------|
| **R-squared** | 모델 설명력 | 분야마다 다름. 0.7 이상이면 보통 양호 |
| **coef** | 회귀계수 β | 식의 직접 결과 |
| **P>|t|** | 그 계수가 0인지 검정 | < 0.05면 "그 변수가 의미 있다" |
| **[0.025, 0.975]** | 95% 신뢰구간 | 0을 포함하지 않으면 유의함 |

### 우리 결과 해석

| 발견 | 해석 |
|------|------|
| const = 40.12 | 공부 0, 학원 0일 때 기본 점수 약 40 |
| study_hours = 3.99 | 공부 1시간 ↑ → 점수 약 4점 ↑ |
| private_classes = 6.02 | 학원 1단위 ↑ → 점수 약 6점 ↑ |
| 모든 p-value ≈ 0 | 세 효과 모두 통계적으로 유의 |
| R² = 0.875 | 점수 변동의 87.5%를 두 변수로 설명 |

> 💡 **실제 관계 (40, 4, 6)** 와 추정치 (40.12, 3.99, 6.02)가 거의 일치. **회귀가 우리 데이터에서 진실을 잘 회복**했습니다.

### Step 3: scikit-learn 버전 (ML 스타일)

```python
from sklearn.linear_model import LinearRegression

X = df[["study_hours", "private_classes"]]
y = df["score"]

model = LinearRegression().fit(X, y)
print("절편:", model.intercept_)
print("계수:", model.coef_)
print("R²:", model.score(X, y))
```

**`statsmodels` vs `sklearn`**:

| | statsmodels | sklearn |
|---|------------|---------|
| 목적 | 통계적 해석 | ML 예측 |
| p-value, 신뢰구간 | ✅ | ❌ |
| 파이프라인, 교차검증 | ❌ | ✅ |
| **이 과정** | **사용** | (ML 과정에서) |

## 6. ML/DL 연결 — MSE는 최소제곱법 [↑](#toc)

> 🔗 **이 모듈이 ML/DL에서 어떻게 쓰이나**

### 1) MSE 손실함수 = 최소제곱법

```
회귀 손실 (MSE)  = (1/N) × Σ (y_실제 - y_예측)²
최소제곱법 (SSE) =         Σ (y_실제 - y_예측)²
                          └─ 같은 식 (상수배만 차이)
```

ML이 "MSE를 최소화한다"고 할 때, 그건 **이 모듈에서 배운 최소제곱법을 똑같이 하는 것**입니다.

### 2) 회귀계수 = 모델의 학습 결과

선형회귀에서는 정답 공식(정규방정식)으로 한 번에 구함.
신경망에서는 같은 개념을 **경사하강법으로 반복 학습**해서 찾음.

### 3) 한계 → 비선형으로 가는 다리

- 선형회귀: y와 x가 직선 관계만 가능
- **실세계는 곡선** → 다항회귀 / 트리 / 신경망
- **이진 분류(합격/불합격)** → 로지스틱 회귀 (모듈 6)

### 4) R² → ML 평가 지표

| 통계 | ML |
|-----|-----|
| R² (학습 데이터) | train R² |
| (없음) | **test R²** ← 진짜 중요 |
| Adjusted R² | (변수 많을수록 페널티) |

### 5) "선형회귀가 ML의 출발점"

대부분의 ML 코스가 선형회귀부터 시작하는 이유:
- **공식이 있어 직관적**
- 손실(MSE), 최적화(정규방정식 vs 경사하강법), 평가(R²) 모든 ML 흐름을 압축해서 보여줌

> 💡 본 과정의 5번 모듈은 통계적 관점(계수 해석, p-value, R²)에 집중하고, **ML 과정의 회귀 모듈에서 코드/규제/교차검증을 깊게** 다룹니다.

## 7. 정리 [↑](#toc)

### 이 장 한 줄 요약
> **회귀 = 점들 사이로 잔차 제곱합이 최소인 직선 긋기.** 이게 ML의 MSE 손실의 정체.

### 자가 진단 체크리스트

| 항목 | 확인 |
|------|:---:|
| 산점도 + 회귀선의 그림을 직접 그릴 수 있다 | ☐ |
| "제곱"하는 이유를 두 가지 들 수 있다 | ☐ |
| 회귀계수 β를 한 문장으로 해석할 수 있다 | ☐ |
| R²의 의미와 함정 둘을 안다 | ☐ |
| `model.summary()`에서 p-value 위치를 안다 | ☐ |
| 최소제곱법 = MSE 손실임을 안다 | ☐ |
