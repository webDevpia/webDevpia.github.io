## 2. 운영 계획

### 환경 설정 특별 주의사항 (JavaScript 과정 대비)

Flutter 환경 설정은 JavaScript보다 **10배 복잡**합니다. Day1 일정 계획에 반드시 반영해야 합니다.

| 항목 | JavaScript | Flutter |
|------|-----------|---------|
| 설치 시간 | 15분 | **1-2시간** |
| 필요 도구 | VS Code + Node.js | VS Code + Flutter SDK + Android Studio + 에뮬레이터 |
| 트러블슈팅 빈도 | 낮음 | **매우 높음** |
| 백업 환경 | CodeSandbox (항상 작동) | **DartPad + Flutter Web** |
| OS별 차이 | 거의 없음 | Windows/macOS 경험 크게 다름 |

#### 주요 트러블슈팅 시나리오

| 문제 | OS | 대응 |
|------|-----|------|
| `flutter doctor`에서 빨간색 X | 공통 | 해당 항목만 해결, 나머지 경고는 무시해도 됨 |
| Android 에뮬레이터 시작 안 됨 | Windows | HAXM 또는 Hyper-V 설정 확인. 안 되면 실기기 USB 연결 |
| 에뮬레이터 너무 느림 | 공통 | `flutter run -d chrome`으로 Flutter Web 대체 |
| Xcode 버전 불일치 | macOS | `sudo xcode-select --switch /Applications/Xcode.app` |
| gradle 빌드 실패 | 공통 | `flutter clean && flutter pub get && flutter run` |
| Hot Reload 작동 안 함 | 공통 | Hot Restart(`R`) 시도, 안 되면 `flutter run` 재시작 |
| pub.dev 패키지 다운로드 실패 | 공통 | 네트워크 확인 또는 사전 다운로드된 starter 사용 |
| `null check operator` 런타임 오류 | 공통 | Null Safety 개념 간단 설명 후 `?` 또는 `!` 처리 |

#### 백업 환경 전략

```
1순위: Android 에뮬레이터
  ↓ 안 되면
2순위: 실기기 USB 연결 (Android)
  ↓ 안 되면
3순위: Flutter Web (flutter run -d chrome)
  ↓ 안 되면 (Dart 기초 학습 시)
4순위: DartPad (dartpad.dev)
```

### Windows vs macOS 학생 혼재 시 운영

- **공통 타겟: Android + Web** — 모든 학생이 사용 가능
- **iOS 관련 내용**: macOS 학생만 선택적 진행. Windows 학생에게는 "macOS에서만 가능"이라고 명시
- **터미널 명령어**: `flutter` 명령어는 OS 동일. 경로 구분자 차이(`\` vs `/`)만 주의
- **Android Studio**: 에뮬레이터 관리용으로만 사용. 코드 편집은 VS Code
- **강의실 PC가 Windows인 경우**: 사전에 HAXM 또는 Hyper-V 중 어느 것이 활성화되어 있는지 확인 필수

## 6. 학생 상태 판단 신호

### "잘 따라오고 있다" 신호
- Hot Reload 후 변경 결과를 기대하며 화면을 봄
- 위젯 속성을 자발적으로 변경해봄 (색상, 크기, 텍스트)
- "이 위젯 말고 다른 건 없나요?" 질문
- 에러 메시지를 읽고 스스로 시도해봄
- "왜 이렇게 되는 거예요?"라는 이해 기반 질문
- AI가 생성한 코드를 보고 "이 부분이 뭐예요?"라고 능동적으로 질문

### "어려워하고 있다" 신호
- 에뮬레이터 화면만 보고 코드를 안 씀
- 괄호가 맞지 않아 빨간 줄이 가득한 상태로 멈춤
- "그냥 넘어가도 되나요?" 질문
- Dart 문법 챕터에서 "이걸 왜 배우는 건지 모르겠어요"
- AI 코드를 붙여넣고 작동하면 더 이상 읽으려 하지 않음
- 쉬는 시간에 혼자 조용히 있으려 함

### 대응 전략
- 어려워하는 학생이 **30% 이상**: 속도 줄이기, "도전" 건너뛰기
- 어려워하는 학생이 **50% 이상**: 다음 챕터 진입 보류, 현재 내용 보충 실습 추가
- 빠른 학생이 지루해할 때: 💪 보너스 챌린지 안내 또는 느린 학생 페어링

## 7. Dart vs Flutter 개념 혼동 방지

비전공자 학생들이 가장 자주 혼동하는 Dart/Flutter 경계를 명확히 안내하세요.

| 혼동 상황 | 올바른 설명 |
|-----------|-----------|
| "Dart랑 Flutter가 같은 건가요?" | "Dart는 언어, Flutter는 그 언어로 만든 UI 프레임워크. 비유: 한국어(Dart)로 쓴 소설(Flutter 앱)" |
| "`main()` 함수가 왜 있어요?" | "모든 Dart 프로그램의 시작점. Flutter도 `main()` → `runApp()`으로 시작" |
| "위젯이 클래스예요?" | "네. 모든 Flutter UI 요소는 클래스입니다. `Text('안녕')` = Text 클래스의 인스턴스" |
| "`build()` 메서드가 뭐예요?" | "위젯이 화면에 어떻게 그려질지 정의하는 메서드. 상태가 바뀌면 다시 호출됨" |
| "StatefulWidget이 너무 복잡해요" | "껍데기(StatefulWidget)와 두뇌(State)를 분리한 것. 두뇌에만 집중하면 됨" |
