## 1️⃣ ⭐ **핵심** — 네비게이션이란? [↑](#toc)

### 책의 페이지 비유

> Flutter의 네비게이션은 **책에서 다른 장으로 이동하는 것**과 같습니다.
>
> - **Navigator** = 책의 목차 및 북마크 관리자
> - **화면(Route)** = 책의 한 장(Chapter)
> - **push** = 다음 페이지로 넘기기
> - **pop** = 이전 페이지로 돌아가기
> - **스택(Stack)** = 지금까지 읽은 페이지 순서

### 스택(Stack) 구조

Flutter의 네비게이션은 **스택** 자료구조를 사용합니다.

```
처음:          push 홈→상세:      push 상세→설정:    pop:
┌─────────┐    ┌─────────┐       ┌─────────┐      ┌─────────┐
│  (없음)  │    │  상세화면 │       │  설정화면 │      │  상세화면 │
└─────────┘    ├─────────┤       ├─────────┤      ├─────────┤
               │  홈화면  │       │  상세화면 │      │  홈화면  │
               └─────────┘       ├─────────┤      └─────────┘
                                 │  홈화면  │
                                 └─────────┘
```

- `push`: 화면을 스택에 쌓음 (위에 올리기)
- `pop`: 맨 위 화면을 제거 (돌아가기)

## 2️⃣ ⭐ **핵심** — Navigator.push [↑](#toc)

### 기본 push

```dart
// 버튼을 누르면 상세 화면으로 이동
ElevatedButton(
  onPressed: () {
    Navigator.push(
      context,
      MaterialPageRoute(
        builder: (context) => const DetailScreen(),
      ),
    );
  },
  child: const Text('상세 보기'),
),
```

### push의 매개변수

```dart
Navigator.push(
  context,      // 현재 화면의 BuildContext
  route,        // 이동할 화면을 만드는 Route
)
```

- `context`: 현재 화면의 위치 정보 (항상 필요)
- `route`: `MaterialPageRoute`로 만든 화면 경로

<a id="part3"></a>
## 3️⃣ Navigator.pop — 이전으로 돌아가기 [↑](#toc)

### 기본 pop

```dart
// "뒤로 가기" 버튼 구현
ElevatedButton(
  onPressed: () {
    Navigator.pop(context);    // 현재 화면을 스택에서 제거
  },
  child: const Text('돌아가기'),
),
```

> AppBar에 자동으로 생기는 ← 뒤로가기 버튼도 내부적으로 `Navigator.pop(context)`를 호출합니다.

### pop 가능 여부 확인

```dart
// 돌아갈 화면이 있을 때만 pop
if (Navigator.canPop(context)) {
  Navigator.pop(context);
}
```

## 4️⃣ MaterialPageRoute — 전환 애니메이션 [↑](#toc)

`MaterialPageRoute`는 Material Design 스타일의 화면 전환 애니메이션을 제공합니다.

```dart
// 기본 슬라이드 애니메이션
MaterialPageRoute(
  builder: (context) => const NextScreen(),
)

// 전체 화면으로 열기 (모달처럼)
MaterialPageRoute(
  builder: (context) => const NextScreen(),
  fullscreenDialog: true,    // 아래에서 위로 슬라이드
)
```

### 다양한 Route 종류

```dart
// 1. MaterialPageRoute — 플랫폼별 기본 애니메이션
Navigator.push(context, MaterialPageRoute(builder: (ctx) => const Screen()));

// 2. PageRouteBuilder — 커스텀 애니메이션
Navigator.push(
  context,
  PageRouteBuilder(
    pageBuilder: (ctx, anim1, anim2) => const Screen(),
    transitionsBuilder: (ctx, anim1, anim2, child) {
      return FadeTransition(opacity: anim1, child: child);
    },
  ),
);
```

## 5️⃣ ⭐ **핵심** — 화면 간 데이터 전달 [↑](#toc)

### 생성자로 데이터 전달

다음 화면의 생성자에 필요한 데이터를 매개변수로 추가합니다.

```dart
// 상세 화면 클래스 — 데이터를 받을 준비
class DetailScreen extends StatelessWidget {
  final String title;
  final String description;
  final int id;

  const DetailScreen({
    super.key,
    required this.title,
    required this.description,
    required this.id,
  });

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text(title)),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: Text(description),
      ),
    );
  }
}
```

```dart
// 이전 화면에서 데이터와 함께 push
Navigator.push(
  context,
  MaterialPageRoute(
    builder: (context) => DetailScreen(
      title: '뉴스 제목',
      description: '뉴스 본문 내용...',
      id: 42,
    ),
  ),
);
```

### 모델 클래스로 전달

여러 데이터를 전달할 때는 모델 클래스를 만들면 깔끔합니다.

```dart

class Product {

// 전달
Navigator.push(

// 받기
class ProductDetailScreen extends StatelessWidget {
  final Product product;
  const ProductDetailScreen({super.key, required this.product});
  // ...
}
```

<a id="part6"></a>
## 6️⃣ 📖 **더 알아보기** — Named Routes [↑](#toc)

앱이 커질수록 화면이 많아집니다. **Named Routes**는 각 화면에 이름을 부여해 관리합니다.

### Named Routes 설정

```dart
// main.dart
class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      initialRoute: '/',          // 처음 화면
      routes: {
        '/': (context) => const HomeScreen(),
        '/detail': (context) => const DetailScreen(),
        '/settings': (context) => const SettingsScreen(),
        '/profile': (context) => const ProfileScreen(),
      },
    );
  }
}
```

### Named Routes로 이동

```dart
// 이름으로 이동
Navigator.pushNamed(context, '/detail');

// 이전으로 돌아가기 (동일)
Navigator.pop(context);

// 현재 화면을 교체 (홈으로 돌아갈 때 스택 정리)
Navigator.pushReplacementNamed(context, '/home');

// 특정 화면까지 모두 제거하고 이동
Navigator.pushNamedAndRemoveUntil(
  context,
  '/home',
  (route) => false,    // 모든 이전 화면 제거
);
```

### Named Routes로 인수(arguments) 전달

```dart
// 보내기
Navigator.pushNamed(
  context,
  '/detail',
  arguments: {'id': 42, 'title': '제품명'},
);

// 받기
class DetailScreen extends StatelessWidget {
  const DetailScreen({super.key});

  @override
  Widget build(BuildContext context) {
    final args = ModalRoute.of(context)!.settings.arguments
        as Map<String, dynamic>;
    final id = args['id'] as int;
    final title = args['title'] as String;

    return Scaffold(
      appBar: AppBar(title: Text(title)),
      body: Text('ID: $id'),
    );
  }
}
```

> **주의**: Named Routes로 인수를 전달할 때는 타입 안전성이 낮습니다. 생성자 방식이 더 안전합니다.

## 7️⃣ 결과 받아오기 — pop에 데이터 담기 [↑](#toc)

화면 B에서 화면 A로 돌아올 때 데이터를 함께 보낼 수 있습니다.

```
화면 A                    화면 B
  │                         │
  │ push(B)                 │
  │────────────────────────>│
  │                         │ 사용자가 선택
  │ pop(선택된 색상)          │
  │<────────────────────────│
  │ 받은 색상으로 업데이트     │
```

### 화면 B에서 결과와 함께 pop

```dart
// 색상 선택 화면 (B)
class ColorPickerScreen extends StatelessWidget {
  const ColorPickerScreen({super.key});

  @override
  Widget build(BuildContext context) {
    final colors = [Colors.red, Colors.green, Colors.blue, Colors.purple];

    return Scaffold(
      appBar: AppBar(title: const Text('색상 선택')),
      body: GridView.builder(
        padding: const EdgeInsets.all(16),
        gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
          crossAxisCount: 2,
          crossAxisSpacing: 8,
          mainAxisSpacing: 8,
        ),
        itemCount: colors.length,
        itemBuilder: (context, index) {
          return GestureDetector(
            onTap: () {
              // 선택된 색상과 함께 pop
              Navigator.pop(context, colors[index]);
            },
            child: Container(
              decoration: BoxDecoration(
                color: colors[index],
                borderRadius: BorderRadius.circular(12),
              ),
            ),
          );
        },
      ),
    );
  }
}
```

### 화면 A에서 결과 받기

```dart
// 이전 화면 (A)
class _HomeState extends State<HomeScreen> {
  Color _selectedColor = Colors.blue;

  Future<void> _openColorPicker() async {
    // push로 이동 + 결과 기다리기
    final result = await Navigator.push<Color>(
      context,
      MaterialPageRoute(builder: (ctx) => const ColorPickerScreen()),
    );

    // 사용자가 선택했다면 (pop 없이 뒤로가기 누르면 null)
    if (result != null) {
      setState(() {
        _selectedColor = result;
      });
    }
  }

  // ...
}
```

## 8️⃣ 💪 **보너스 챌린지** — 실습: 3개 화면 앱 만들기 [↑](#toc)

홈 → 상세 → 설정, 3개 화면을 가진 앱을 만들어 봅시다.

### 구조

```
HomeScreen
  ├── 아이템 목록 (이름, 가격)
  └── 아이템 클릭 → DetailScreen(아이템 데이터 전달)
          └── "설정 열기" 버튼 → SettingsScreen()
                  └── 뒤로가기 → DetailScreen → HomeScreen
```

### 데이터 모델

```dart
// lib/models/item.dart

  final String name;
  final double price;
  final String category;

  const Item({
    required this.id,
    required this.name,
    required this.price,
    required this.category,
  });

// 샘플 데이터
const sampleItems = [

  Item(id: 2, name: '카페라떼', price: 5000, category: '음료'),
  Item(id: 3, name: '크로아상', price: 3500, category: '베이커리'),
  Item(id: 4, name: '치즈케이크', price: 6000, category: '디저트'),
```

```dart

import 'models/item.dart';
import 'screens/detail_screen.dart';
```

## 9️⃣ 🚀 **도전** — BottomNavigationBar [↑](#toc)

대부분의 실제 앱은 하단에 탭 네비게이션이 있습니다.

```
┌────────────────────────────┐
│         메인 화면            │
│                            │
│                            │
│                            │
├────────────────────────────┤
│  🏠홈   🔍검색  ❤️즐겨찾기  👤프로필 │
└────────────────────────────┘
```

### BottomNavigationBar 기본 구조

```dart
class MainScreen extends StatefulWidget {
  const MainScreen({super.key});

  @override
  State<MainScreen> createState() => _MainScreenState();
}

class _MainScreenState extends State<MainScreen> {
  int _selectedIndex = 0;

  // 각 탭에 표시할 화면들
  static const List<Widget> _screens = [
    HomeTab(),
    SearchTab(),
    FavoriteTab(),
    ProfileTab(),
  ];

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: _screens[_selectedIndex],    // 현재 탭 화면
      bottomNavigationBar: BottomNavigationBar(
        currentIndex: _selectedIndex,
        onTap: (index) {
          setState(() {
            _selectedIndex = index;
          });
        },
        type: BottomNavigationBarType.fixed,    // 4개 이상일 때 필요
        selectedItemColor: Colors.blue,
        items: const [
          BottomNavigationBarItem(
            icon: Icon(Icons.home),
            label: '홈',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.search),
            label: '검색',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.favorite),
            label: '즐겨찾기',
          ),
          BottomNavigationBarItem(
            icon: Icon(Icons.person),
            label: '프로필',
          ),
        ],
      ),
    );
  }
}
```

### IndexedStack — 탭 전환 시 상태 유지

기본 `_screens[_selectedIndex]` 방식은 탭을 바꿀 때마다 위젯을 새로 만듭니다.
`IndexedStack`을 사용하면 모든 탭을 메모리에 유지합니다.

```dart
body: IndexedStack(
  index: _selectedIndex,
  children: const [
    HomeTab(),
    SearchTab(),
    FavoriteTab(),
    ProfileTab(),
  ],
),
```

| | 기본 방식 | IndexedStack |
|---|---|---|
| 성능 | 탭 전환 빠름 | 초기 로딩 느림 |
| 상태 유지 | 초기화됨 | 유지됨 |
| 사용 경우 | 간단한 앱 | 스크롤 위치, 입력값 유지 |

<a id="part10"></a>
## 🔟 정리 [↑](#toc)

### 네비게이션 핵심 메서드

| 메서드 | 동작 |
|--------|------|
| `Navigator.push` | 새 화면을 스택에 추가 |
| `Navigator.pop` | 현재 화면 제거 (돌아가기) |
| `Navigator.pushReplacement` | 현재 화면을 새 화면으로 교체 |
| `Navigator.pushAndRemoveUntil` | 스택 정리 후 새 화면으로 |
| `Navigator.pushNamed` | 이름으로 화면 이동 |

### 선택 가이드

```
화면 전환 방법 선택:

소규모 앱 (화면 3~5개)
  → MaterialPageRoute + 생성자로 데이터 전달

중규모 앱 (화면 10개 이상)
  → Named Routes 사용

실제 앱 (앱 스토어 출시 수준)
  → go_router 패키지 사용 (Phase 3에서 배웁니다)
```

### 데이터 전달 방법 비교

```dart
// 방법 1: 생성자로 전달 (추천 — 타입 안전)
Navigator.push(ctx,
  MaterialPageRoute(builder: (_) => DetailScreen(item: item)));

// 방법 2: Named Routes arguments (타입 안전성 낮음)
Navigator.pushNamed(ctx, '/detail', arguments: item);
```
