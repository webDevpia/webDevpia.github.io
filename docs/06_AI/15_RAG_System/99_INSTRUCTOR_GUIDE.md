## 3. 교육장 셋업

### 3.1 물리 환경

- **좌석 배치**: 4-5인 그룹 4개. 그룹 내 진도 빠른 사람이 자연 멘토 역할.
- **프로젝터/모니터**: 강사 화면 1, 보조 화면 1 권장 (시연 + 코드 동시 표시).
- **인터넷**: 최소 100Mbps 권장. WiFi가 부족하면 유선 우선.
- **전원**: 노트북 20대 동시 사용 → 멀티탭·차단기 용량 사전 확인.

### 3.2 네트워크 화이트리스트 (사내망 시 필수)

다음 도메인을 IT팀에 사전 화이트리스트 요청:

```
api.openai.com           # OpenAI Chat/Embeddings
api.anthropic.com        # Claude API
api.cohere.ai            # Cohere Rerank
api.smith.langchain.com  # LangSmith
api.tavily.com           # Tavily Web Search
huggingface.co           # HuggingFace 모델/임베딩
*.huggingface.co         # CDN
pypi.org                 # 패키지 설치
files.pythonhosted.org   # pip wheel
astral.sh                # uv 설치
github.com               # 저장소 클론
*.googleapis.com         # Colab 사용 시
*.colab.research.google.com
```

> ⚠️ 회사 보안 정책상 사내 문서를 외부 API로 보내는 게 금지면, 실습 데이터는 **공개 자료**(공공정책, 공개 법령)만 사용하고 사내 문서는 종합 프로젝트에서만 본인 환경에서 다루도록 안내.

### 3.3 사전 모델 다운로드

20명이 동시에 HuggingFace에서 568MB 모델을 받으면 1~2시간 걸립니다. 다음 중 하나를 선택:

**옵션 A (권장)** — D-1에 안내 메일로 사전 다운로드:

```bash
# 수강생이 D-1 저녁에 한 번 실행
huggingface-cli download BAAI/bge-reranker-v2-m3
huggingface-cli download Dongjin-kr/ko-reranker
huggingface-cli download jhgan/ko-sroberta-multitask
```

**옵션 B** — 교육장 NAS·USB에 모델 캐시(`~/.cache/huggingface/`) 통째로 배포.

**옵션 C** — Ch.04에서 로컬 reranker는 데모만 보여주고 실습은 Cohere API로 진행.

### 3.4 공유 폴더

교육장 NAS 또는 강사 노트북의 공유 폴더에 다음을 비치:

## 4. API 키·비용 관리

### 4.1 키 분배 전략

| 전략 | 장점 | 단점 | 권장 시나리오 |
|:---|:---|:---|:---|
| **개인 키 사용** | 운영 단순, 보안 우수 | 비용 분산, 가입 부담 | 개인 학습자 위주, 비용 본인 부담 |
| **팀 단위 공유 키** | 비용 통제, 가입 부담 ↓ | rate limit 충돌 위험 | 5명/조 × 4조 |
| **조직 단일 키 + 사용자 쿼터** | 비용 통제 최강 | OpenAI 조직 설정 필요 | 회사 차원 환급과정 (권장) |

### 4.2 OpenAI 조직 단일 키 설정 (권장)

1. OpenAI Dashboard → Organization Settings → Limits
2. **Daily limit**: $100 (인당 $5 × 20명)
3. **Per-user rate limit**: 분당 100K tokens (20명 동시 호출 대비)
4. 강사가 시작 전에 잔여 크레딧 확인, Day 3 종료 시 중간 점검.

### 4.3 비용 모니터링

매일 종료 시 강사가 OpenAI Dashboard에서 일별 사용량 확인:

```
Day 1 예상: $20-30   (Ch.01-02 임베딩 + 청킹)
Day 2 예상: $30-40   (Ch.03-05 임베딩 다수 + reranker)
Day 3 예상: $20-30   (Ch.06-07 메모리 누적)
Day 4 예상: $40-60   (Ch.08-09 평가 LLM 호출 ↑↑)
Day 5 예상: $10-20   (Ch.10 모니터링 + 발표)
```

> ⚠️ Day 4 Ragas 평가가 가장 비싸다. judge LLM이 모든 답변·컨텍스트를 평가하므로 호출 수 폭증. 사전에 인당 sample을 5개로 제한하면 비용 50% 절감.

### 4.4 rate limit 폭주 방지

20명이 동시에 SemanticChunker를 돌리면 분당 60K~100K 임베딩 요청 → 429 폭발.

**해결 패턴**:

```python
# Tier 1 OpenAI 계정: 3000 RPM
# 20명 동시 → 분당 인당 150 요청 한도 → 1초당 2.5회

import time
import random

def safe_embed_with_jitter(texts, embedder, batch=10):
    results = []
    for i in range(0, len(texts), batch):
        time.sleep(random.uniform(0.5, 1.5))  # 인원별 분산
        results.extend(embedder.embed_documents(texts[i:i+batch]))
    return results
```

또는 강사가 **5명씩 4조 시차 시작** (조마다 30초 간격) 안내.
