## 1. 공통 — uv·Python 버전 문제

### 1.1 `uv: command not found`

uv가 설치 안 됐거나 PATH에 없습니다.

```bash
# 재설치 (macOS/Linux)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 새 셸 열거나
source ~/.zshrc      # zsh
source ~/.bashrc     # bash

# 직접 PATH 추가
export PATH="$HOME/.local/bin:$PATH"
```

Windows는 PowerShell을 **관리자 권한**으로 다시 실행 또는:

```powershell
$env:Path = "$env:USERPROFILE\.local\bin;" + $env:Path
```

### 1.2 Python 3.11이 없다

```bash
uv python install 3.11
uv python list   # 설치된 버전 확인
```

uv가 Python을 자동 설치합니다. 시스템 Python이 3.9 이하라도 무관.

### 1.3 `pyproject.toml` 인식 안 됨

```bash
# 새 폴더에서 시작했는지 확인
pwd
ls   # pyproject.toml이 보여야 함

# 없으면 초기화
uv init --python 3.11
```

### 1.4 `uv add` 실패 — 네트워크 오류

```bash
# pypi mirror 변경 (한국)
export UV_INDEX_URL=https://pypi.org/simple
uv add <package>

# 또는 회사망이면 사내 미러
export UV_INDEX_URL=https://nexus.company.com/repository/pypi/simple
```

## 2. macOS — Intel·Apple Silicon

### 2.1 `kiwipiepy` 빌드 실패 (M1/M2/M3)

가장 흔한 문제. C++ 컴파일러 의존성.

**증상**:
```
ERROR: Failed building wheel for kiwipiepy
fatal error: 'string.h' file not found
```

**해결**:

```bash
# Xcode Command Line Tools (필수)
xcode-select --install

# Homebrew (없으면)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# CMake + 의존성
brew install cmake

# 환경 변수 설정 후 재설치
export ARCHFLAGS="-arch arm64"
export CFLAGS="-I$(brew --prefix)/include"
export LDFLAGS="-L$(brew --prefix)/lib"
uv pip install --no-cache-dir kiwipiepy
```

여전히 실패하면 prebuilt wheel 시도:

```bash
uv pip install kiwipiepy --no-build-isolation
```

### 2.2 `unstructured` 의존성 (PDF 표 인식)

```
ImportError: libgthread-2.0.so.0: cannot open shared object
```

**해결**:

```bash
brew install poppler tesseract libmagic
brew install --cask libreoffice   # HWP 변환용
uv add "unstructured[pdf]"
```

### 2.3 `chromadb` 빌드 실패

ARM Mac에서 `onnxruntime` 호환성 문제.

```bash
brew install protobuf
uv pip install chromadb --no-cache-dir
```

### 2.4 `sentence-transformers` 메모리 부족

8GB RAM Mac에서 로컬 reranker 시 OOM.

**해결**:
- Cohere API로 전환 (Ch.04 §3)
- 또는 작은 모델: `BAAI/bge-reranker-base` (278MB)

## 3. Windows (10/11)

### 3.1 `kiwipiepy` 빌드 실패 — VC++ Tools 없음

**증상**:
```
error: Microsoft Visual C++ 14.0 or greater is required
```

**해결**:

1. [Visual Studio Build Tools 2022](https://visualstudio.microsoft.com/visual-cpp-build-tools/) 다운로드.
2. 설치 시 **"Desktop development with C++"** 워크로드 선택.
3. 재부팅 후:

```powershell
uv pip install --no-cache-dir kiwipiepy
```

또는 **Chocolatey** 사용:

```powershell
choco install visualstudio2022-workload-vctools -y
```

### 3.2 경로 길이 한도 (260자)

Windows 기본 경로 한도 때문에 일부 패키지 실패.

**해결**:

PowerShell 관리자 권한:

```powershell
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" `
                 -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force
```

재부팅.

### 3.3 한글 경로 인코딩 문제

```python
UnicodeDecodeError: 'cp949' codec can't decode byte ...
```

**해결**: 항상 `encoding="utf-8"` 명시.

```python
loader = TextLoader("policy.txt", encoding="utf-8")
```

### 3.4 PowerShell 실행 정책

```
.ps1 cannot be loaded because running scripts is disabled
```

**해결**:

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

### 3.5 LibreOffice (HWP 변환)

```powershell
choco install libreoffice-fresh -y

# HWP → PDF
soffice --headless --convert-to pdf C:\path\to\doc.hwp
```

## 4. WSL / Linux (Ubuntu·Debian)

### 4.1 시스템 패키지 사전 설치

WSL 또는 Linux는 다음을 시스템에 설치:

```bash
sudo apt update && sudo apt install -y \
  build-essential \
  cmake \
  libxml2-dev libxslt-dev \
  libpoppler-cpp-dev \
  poppler-utils \
  tesseract-ocr tesseract-ocr-kor \
  libreoffice \
  fonts-nanum
```

### 4.2 WSL에서 GPU 사용

WSL2 + Windows 11 → NVIDIA GPU 패스스루 가능.

```bash
# GPU 확인
nvidia-smi

# CUDA 토치 설치
uv add torch --index https://download.pytorch.org/whl/cu121
```

WSL1은 GPU 미지원 → CPU만.

### 4.3 메모리 부족

WSL은 기본 메모리가 시스템의 50%. 늘리려면 `~/.wslconfig`:

```ini
[wsl2]
memory=16GB
swap=8GB
```

PowerShell에서 `wsl --shutdown` 후 재시작.

## 5. 한글 폰트 깨짐 (matplotlib)

### 5.1 증상

```
matplotlib에서 한글이 □□□로 표시됨
```

### 5.2 macOS 해결

```python
import matplotlib.pyplot as plt
plt.rcParams['font.family'] = 'AppleGothic'   # 또는 'Apple SD Gothic Neo'

```

### 5.3 Windows 해결

```python
plt.rcParams['font.family'] = 'Malgun Gothic'
plt.rcParams['axes.unicode_minus'] = False
```

### 5.4 Linux/WSL/Colab 해결

```python
import subprocess
subprocess.run(['apt-get', '-qq', 'install', '-y', 'fonts-nanum'], check=True)

import matplotlib.font_manager as fm
for f in fm.findSystemFonts(fontpaths=['/usr/share/fonts/truetype/nanum']):
    fm.fontManager.addfont(f)
plt.rcParams['font.family'] = 'NanumGothic'
plt.rcParams['axes.unicode_minus'] = False
```

### 5.5 Jupyter 캐시 문제

폰트가 설치됐는데도 안 보이면 Jupyter 캐시 클리어:

```python
import matplotlib
matplotlib.font_manager._rebuild()
```

또는 **커널 재시작**.

## 7. 패키지 의존성 충돌

### 7.1 `pip` vs `uv` 혼용 금지

**증상**: 일부 패키지가 `.venv` 밖 시스템에 설치되고, 다른 건 `.venv` 안에 있어 import 실패.

**해결**:

```bash
# 깨끗이 다시
rm -rf .venv uv.lock
uv venv --python 3.11
uv sync
```

### 7.2 `pydantic v1 vs v2`

LangChain은 v2, 일부 옛 코드는 v1. 충돌 시:

```bash
uv add "pydantic>=2.0"
uv pip list | grep pydantic   # 단일 버전 확인
```

### 7.3 `langchain` vs `langchain-community` import 오류

```python
# 옛 import (deprecated)
from langchain.document_loaders import PyPDFLoader

# 새 import (권장)
from langchain_community.document_loaders import PyPDFLoader
```

본 핸드북은 모두 새 import 패턴을 사용.

### 7.4 `chromadb` ↔ `numpy` 충돌

```
ValueError: numpy.dtype size changed
```

```bash
uv pip install --upgrade --force-reinstall chromadb numpy
```

### 7.5 의존성 lockfile 동기화

```bash
# pyproject.toml만 수동 수정했을 때
uv lock              # uv.lock 재생성
uv sync              # .venv를 lock 기준으로 맞춤
```

## 8. API 호출 오류

### 8.1 OpenAI 401 Authentication

```
openai.AuthenticationError: Incorrect API key provided
```

**원인**: `.env` 미로드 또는 키 오타.

```python
from dotenv import load_dotenv
load_dotenv()

import os
print(os.environ.get("OPENAI_API_KEY", "NOT SET")[:10])  # 'sk-...' 일부만 출력
```

### 8.2 OpenAI 429 Rate Limit

```
openai.RateLimitError: Rate limit reached
```

**원인**: 분당 RPM 초과 (Tier 1: 3000 RPM).

**해결**:

```python
from tenacity import retry, wait_exponential, stop_after_attempt

@retry(wait=wait_exponential(min=1, max=60), stop=stop_after_attempt(5))
def safe_invoke(llm, prompt):
    return llm.invoke(prompt)
```

또는 batch 크기 줄이기. 강사는 §4.4 인원 시차 시작 안내.

### 8.3 OpenAI Quota Exceeded

```
openai.error.RateLimitError: You exceeded your current quota
```

**원인**: 결제 카드 등록 안 됐거나 한도 초과.

**해결**: OpenAI 대시보드 → Billing → Add payment method. 또는 강사 백업 키.

### 8.4 Anthropic API 키 형식

```
ANTHROPIC_API_KEY=sk-ant-api03-...
```

`sk-`로만 시작하는 OpenAI 키와 다름. 헷갈리지 말 것.

### 8.5 Cohere 429

무료 티어 분당 한도 초과. 1분 대기 후 재시도 또는 유료 전환.

### 8.6 Tavily 401

```
TavilyClientUnauthorizedException
```

`TAVILY_API_KEY` 미설정 또는 무효. tavily.com에서 재발급.

## 9. ChromaDB 관련 문제

### 9.1 영속화 vs 인메모리

```python
# 인메모리 (재시작 시 사라짐)
db = Chroma.from_documents(chunks, emb)

# 영속화 (재시작 후에도 유지)
db = Chroma.from_documents(chunks, emb, persist_directory="./chroma_db")
```

### 9.2 컬렉션 중복

같은 이름 재사용 시 에러.

```python
import shutil
shutil.rmtree("./chroma_db", ignore_errors=True)   # 깨끗이 시작
```

또는 컬렉션 이름 바꾸기:

```python
db = Chroma.from_documents(chunks, emb, collection_name=f"ch{i}")
```

### 9.3 동시 접근 락

같은 `persist_directory`를 두 프로세스가 열면 락 충돌. 노트북 여러 개 띄울 때 주의.

```python
# 세션 시작 시 한 번만
db = Chroma(persist_directory="./chroma_db", embedding_function=emb)
```

### 9.4 임베딩 차원 불일치

기존 컬렉션과 다른 차원의 임베딩으로 추가 시 에러.

**해결**: 컬렉션 삭제 후 재생성, 또는 별도 컬렉션 사용.

## 10. LangGraph·LangChain 버전 이슈

### 10.1 권장 버전 (2026년 5월 기준)

```toml
# pyproject.toml의 [project.dependencies] 권장 핀
"langchain>=1.2,<2.0"           # 최신 stable: 1.2.17
"langchain-core>=1.3,<2.0"      # 최신 stable: 1.3.x
"langchain-community>=0.3"      # 0.3.x 유지 (1.x로 이동 중)
"langchain-openai>=0.3"
"langchain-anthropic>=0.3"
"langchain-chroma>=0.2"
"langchain-text-splitters>=0.3"
"langchain-experimental>=0.3"
"langchain-huggingface>=0.1"
"langchain-cohere>=0.4"
"langgraph>=1.1,<2.0"           # 최신 stable: 1.1.10
"langgraph-checkpoint>=2.0,<3.0"
"langgraph-checkpoint-sqlite>=2.0"
"langsmith>=0.2"
"ragas>=0.2,<0.3"
```

> 📌 **버전 정합성**:
> - LangChain `1.2.x` ↔ LangGraph `1.1.x` (현재 LangGraph는 `1.2`가 미발매; `1.1`이 최신 stable)
> - LangChain `1.0` 메이저 릴리스(2025-10-20) 이후 RAG 핵심 API(LCEL, EnsembleRetriever, BM25Retriever, ContextualCompressionRetriever, RunnableWithMessageHistory)는 시그니처 호환을 유지합니다.
> - LangGraph `1.x`에서 **`MemorySaver` → `InMemorySaver`** 이름이 권장됩니다. (구 이름도 alias로 유지.)

### 10.2 `RunnableWithMessageHistory` 동작 안 함

LangChain 0.2 이전 코드와 호환 안 됨. 1.2+로 업그레이드:

```bash
uv add "langchain>=1.2"
```

### 10.3 `create_react_agent` import 위치

```python
# 권장 (LangGraph)
from langgraph.prebuilt import create_react_agent

# 옛 위치 (LangChain — deprecated)
# from langchain.agents import create_react_agent
```

### 10.4 LangGraph 0.x → 1.x 마이그레이션

본 핸드북은 LangGraph **1.1.x** 기준입니다. 핵심 변경:

| 영역 | 0.x | 1.x (권장) |
|:---|:---|:---|
| 그래프 import | `from langgraph.graph import StateGraph, START, END` | (동일) |
| 인메모리 체크포인터 | `MemorySaver` | **`InMemorySaver`** (구 이름은 alias로 유지) |
| 체크포인터 패키지 | `langgraph` 내장 | `langgraph-checkpoint` (분리), SQLite는 `langgraph-checkpoint-sqlite` |
| `create_react_agent` 시그니처 | `state_modifier=...` | **`prompt=...`** (state_modifier는 deprecated) |
| 모듈 분리 | 단일 패키지 | `langgraph-prebuilt`로 prebuilt agent 분리 가능 |

```python
# 1.x 권장 패턴
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver   # ← 새 이름
from langgraph.prebuilt import create_react_agent

agent = create_react_agent(
    model="anthropic:claude-haiku-4-5",
    tools=[...],
    prompt="You are a helpful assistant",   # ← state_modifier 대신 prompt
    checkpointer=InMemorySaver(),
)
```

> 💡 본 핸드북 Ch.06·Ch.08의 코드 예제 중 `MemorySaver`로 표기된 부분은 1.x에서도 alias로 동작하지만, 새 코드는 `InMemorySaver`를 사용하세요.

### 10.5 Ragas v0.1 코드가 v0.2에서 안 돌 때

대표 변경:

| v0.1 | v0.2 |
|:---|:---|
| `TestsetGenerator.from_langchain(...)` | `TestsetGenerator(llm=..., embedding_model=...)` |
| `evaluate(ds, metrics=[faithfulness], llm=judge)` | `evaluate(ds, metrics=[Faithfulness(llm=judge)])` |
| `question`/`answer`/`contexts`/`ground_truth` | `user_input`/`response`/`retrieved_contexts`/`reference` |

본 Ch.09는 v0.2 기준.

## 11. GPU·CUDA 관련

### 11.1 GPU 사용 가능 확인

```python
import torch
print(torch.cuda.is_available())          # True여야 함
print(torch.cuda.device_count())
print(torch.cuda.get_device_name(0))
```

### 11.2 PyTorch CUDA 버전 매칭

NVIDIA 드라이버와 CUDA 버전이 안 맞으면 `cuda.is_available()`이 False.

```bash
# CUDA 12.1용
uv pip install torch --index https://download.pytorch.org/whl/cu121

# CUDA 11.8용
uv pip install torch --index https://download.pytorch.org/whl/cu118

# CPU 전용
uv pip install torch --index https://download.pytorch.org/whl/cpu
```

### 11.3 메모리 부족 (OOM)

```python
torch.cuda.empty_cache()
# 또는
import gc; gc.collect(); torch.cuda.empty_cache()
```

작은 모델로 변경:
- `BAAI/bge-reranker-v2-m3` (568MB) → `BAAI/bge-reranker-base` (278MB)

### 11.4 Apple Silicon MPS

M1/M2/M3에서 GPU 가속:

```python
device = "mps" if torch.backends.mps.is_available() else "cpu"
```

일부 모델은 MPS 미지원 → CPU fallback.
