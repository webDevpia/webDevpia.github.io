## 1️⃣ 이 과정에서 만들 것 [↑](#toc)

> AI 비서에게 일하는 방법을 가르치는 것을 상상해보세요.
> Copilot이라는 AI 비서에게 우리 프로젝트의 규칙, 반복 작업 양식, 전문 역할을 알려주면,
> 훨씬 더 정확하게 도와줍니다. 이 과정은 그 "가르치는 방법"을 익히는 과정입니다.

### 전체 흐름

```
환경 구축 → AI-Native 이해 → Custom Instructions → Prompt Files → Custom Agents → TDD → 통합
```

### 실습 프로젝트: Todo Manager

이 과정 전체에서 하나의 프로젝트를 사용합니다. Python으로 작성된 할 일 관리 애플리케이션입니다.

```
AN.1.1/
├── src/todo/
│   └── manager.py    # 핵심 비즈니스 로직
├── tests/
│   └── test_manager.py
├── .github/          # ← 이 폴더를 과정을 통해 채워나갑니다
└── pyproject.toml
```

### 최종 산출물

과정이 끝나면 `.github/` 폴더에 Instructions, Prompts, Agents 파일이 갖춰지고,
Copilot은 이 규칙들을 자동으로 따르며 Todo Manager를 개발하게 됩니다.

## 2️⃣ uv 설치 [↑](#toc)

`uv`는 Python 환경과 패키지를 관리하는 도구입니다. 이 과정에서는 Python 설치부터 프로젝트 실행까지 모두 `uv`를 기준으로 진행합니다.

> 기존에 Python이 설치되어 있어도 이 과정에서는 `uv` 기반으로 통일합니다.
> 로컬 PC의 전역 Python 환경에 의존하지 않으면 "내 컴퓨터에서만 안 됩니다" 문제를 크게 줄일 수 있습니다.

### 설치 명령

**Windows (PowerShell)**:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```
설치 후 터미널 재실행합니다.

```powershell
# 보안 오류 발생 시:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope currentuser
```

**macOS / Linux (터미널)**:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 설치 확인

```bash
uv --version
```

예상 출력:

```
uv 0.6.x (또는 최신 버전)
```

> 🤔 **`uv: command not found` 오류가 나면?**
> 터미널을 완전히 닫고 새로 여세요. 설치 후 환경 변수가 현재 터미널에는 적용되지 않은 것입니다.
> 그래도 안 되면 설치 명령을 다시 실행해보세요.

### 업데이트 확인 (선택)

```bash
uv self update
```

## 3️⃣ Python 3.12 설치 [↑](#toc)

`uv`로 Python 3.12를 설치합니다. 시스템 Python과 별도로 관리되므로 기존 환경에 영향을 주지 않습니다.

### 설치

```bash
uv python install 3.12
```

### 설치 확인

```bash
uv python list
```

예상 출력 (일부):

```
cpython-3.12.x-...   <설치된 경로>
```

목록에서 `3.12`가 보이면 정상입니다.

### 실행 확인

```bash
uv run python --version
```

예상 출력:

```
Python 3.12.x
```

> 🤔 **`python3.12: command not found` 오류가 나면?**
> `python3.12` 명령을 직접 쓰지 않아도 됩니다. 이 과정에서는 `uv run python ...` 형태로 실행합니다.
> `uv python list`에서 3.12가 보이면 설치는 완료된 것입니다.

## 4️⃣ VS Code + GitHub Copilot 설정 [↑](#toc)

### VS Code 설치

1. [https://code.visualstudio.com](https://code.visualstudio.com) 접속 후 운영체제에 맞는 설치 파일 다운로드
2. 설치 후 터미널에서 확인:

```bash
code --version
```

> 🤔 **`code: command not found` 오류가 나면?**
> VS Code를 User setup으로 재설치하세요. 또는 VS Code 안에서 `Ctrl+Shift+P` → `Shell Command: Install 'code' command in PATH` 를 실행하세요.

### GitHub Copilot Chat 확장 설치

1. VS Code 왼쪽 사이드바에서 Extensions(확장) 아이콘 클릭
2. 검색창에 `GitHub Copilot Chat` 입력
3. 첫 번째 결과 설치
4. VS Code 하단 Copilot 아이콘 클릭 → `Use AI Features with Copilot for free` 선택
5. GitHub 계정으로 로그인

### Copilot Pro 30일 평가판 (구독이 없는 경우)

1. [https://github.com/features/copilot/plans](https://github.com/features/copilot/plans) 접속
2. `Pro` 카드에서 `Try for 30 days free` 클릭
3. 결제 정보 입력 후 활성화
4. VS Code로 돌아와 Copilot Chat 재로그인

> **평가판 취소 방법**: GitHub `Settings` → `Billing & licensing` → Copilot 섹션 → `Cancel trial`
> 30일 내 취소하면 과금되지 않습니다.

### Agent 모드 확인

Copilot Chat에서 Agent 모드는 파일을 직접 읽고 수정할 수 있는 강력한 실행 모드입니다. 이 과정의 실습은 Agent 모드를 기준으로 진행합니다.

1. `Ctrl+Alt+I` 또는 Chat 아이콘으로 Copilot Chat 패널 열기
2. 입력창 위쪽 드롭다운에서 `Agent` 선택

> 🤔 **Agent 모드가 드롭다운에 안 보이면?**
> VS Code와 GitHub Copilot Chat 확장을 최신 버전으로 업데이트하세요.
> VS Code: `Help` → `Check for Updates`
> 확장: Extensions 패널에서 Copilot Chat 옆 업데이트 버튼

### 연동 확인

Copilot Chat 입력창에 아래를 입력해봅니다.

```
이 저장소의 구조를 3줄로 요약해줘.
```

응답이 돌아오면 연동 정상입니다.

## 5️⃣ 프로젝트 열기 + 검증 [↑](#toc)

### 프로젝트 폴더 열기

VS Code에서 `AN.1.1` 폴더를 열거나, 터미널에서 이동 후 VS Code로 엽니다.

```bash
code AN.1.1
```

### 의존성 설치

```bash
uv sync
```

예상 출력:

```
Resolved N packages in ...
Installed N packages in ...
```

### 테스트 실행

```bash
uv run pytest -v
```

예상 출력:

```
collected X items

tests/test_manager.py::test_... PASSED
...

X passed in X.XXs
```

> 🤔 **`ModuleNotFoundError` 또는 `ImportError` 오류가 나면?**
> `uv sync`를 먼저 실행했는지 확인하세요. `uv sync` 후 다시 `uv run pytest -v`를 실행합니다.

> 🤔 **`uv run pytest: command not found` 오류가 나면?**
> 프로젝트 폴더 안(`pyproject.toml`이 있는 위치)에서 실행하는지 확인하세요.

## 6️⃣ 여기까지 되셨나요? [↑](#toc)

아래 항목을 순서대로 확인합니다. 모두 체크되면 다음 장으로 넘어갈 준비가 된 것입니다.

| 항목 | 확인 명령 | 기대 결과 | 확인 |
|------|-----------|-----------|:------:|
| uv 설치 | `uv --version` | 버전 번호 출력 | ☐ |
| Python 3.12 | `uv python list` | 목록에 3.12.x 표시 | ☐ |
| VS Code | `code --version` | 버전 번호 출력 | ☐ |
| Copilot Chat | Chat에 메시지 입력 | AI 응답 반환 | ☐ |
| Agent 모드 | Chat 드롭다운 | `Agent` 선택 가능 | ☐ |
| 의존성 동기화 | `uv sync` | 오류 없이 완료 | ☐ |
| 테스트 통과 | `uv run pytest -v` | `X passed` 출력 | ☐ |
