## 1️⃣ AI 에이전트란? [↑](#toc)

### 만능 비서 비유

> 챗봇이 **"질문에 답해주는 직원"**이라면,
> AI 에이전트는 **"직접 검색하고, 계산하고, 파일을 만들고, 이메일을 보내는 만능 비서"**입니다.

ChatGPT에게 "서울 3박4일 여행 계획을 짜줘"라고 하면 그럴듯한 답변을 줍니다. 하지만 그 답변은 모델이 학습한 과거 데이터를 기반으로 생성된 텍스트일 뿐, 실제로 **지금 이 순간** 숙소 가격을 검색하거나 예약을 하지는 않습니다.

AI 에이전트는 다릅니다. 목표를 주면 스스로 생각하고, 필요한 도구를 선택하고, 행동하고, 결과를 확인하며 목표를 달성합니다.

### 챗봇 vs AI 에이전트

| | 챗봇 | AI 에이전트 |
|--|------|------------|
| **동작 방식** | 질문 → 답변 (1회성) | 질문 → 생각 → 행동 → 관찰 → 반복 |
| **도구 사용** | ❌ | ✅ (검색, 계산, 파일 읽기/쓰기 등) |
| **자율성** | 없음 — 지시한 대로만 | 스스로 판단하여 다음 행동 결정 |
| **메모리** | 대화 내 제한적 | 단기 메모리 + 장기 메모리 |
| **목표 달성** | 답변 제공으로 끝 | 목표를 달성할 때까지 계속 시도 |

## 2️⃣ 에이전트의 4가지 구성 요소 [↑](#toc)

```
AI Agent = LLM (두뇌) + Tools (도구) + Memory (기억) + Planning (계획)
```

### LLM (두뇌) — 생각하고 판단하는 핵심

> "다음에 뭘 해야 하지? 어떤 도구를 써야 하지?"

LLM(대형 언어 모델, Large Language Model)은 에이전트의 중앙 처리 장치입니다. GPT-4o, Claude, Gemini 같은 모델이 여기에 해당합니다. 도구를 쓸지 말지, 어떤 도구를 쓸지, 결과를 어떻게 해석할지 모두 LLM이 판단합니다.

### Tools (팔과 손) — 실제로 행동하는 능력

> "검색하고, 계산하고, 이메일을 보내자"

LLM 혼자서는 학습된 지식만 사용할 수 있습니다. 도구(Tools)를 통해 에이전트는 **외부 세계와 상호작용**할 수 있게 됩니다.

- 웹 검색 도구 — 최신 정보 수집
- 계산기 도구 — 정확한 수치 계산
- 파일 읽기/쓰기 도구 — 문서 처리
- 이메일/캘린더 도구 — 실제 업무 자동화

### Memory (수첩) — 이전 작업을 기억하는 능력

> "아까 검색한 결과가 뭐였더라?"

- **단기 메모리 (Short-term)**: 현재 진행 중인 작업의 맥락 — 이번 대화에서 무엇을 했는지
- **장기 메모리 (Long-term)**: 이전 세션의 학습 내용 — "이 사용자는 채식주의자다" 같은 정보

### Planning (계획표) — 큰 목표를 단계로 나누는 능력

> "여행 계획 세우기를 한 번에 할 수 없으니 → 관광지 조사 → 맛집 조사 → 교통 조사 → 통합 정리 순서로 나누자"

복잡한 목표를 받으면 에이전트는 이를 작은 단계들로 분해합니다. 각 단계를 순서대로 처리하거나, 병렬로 처리하거나, 결과에 따라 다음 단계를 동적으로 결정합니다.

## 3️⃣ 에이전트 루프(Agent Loop) [↑](#toc)

### 업무 루프 비유

> 직장인의 하루를 떠올려보세요.
> "지시 받기 → 무엇을 해야 할지 생각 → 행동 → 결과 확인 → 다시 생각..."
> 에이전트도 목표가 달성될 때까지 이 루프를 반복합니다.

### ReAct 패턴 (Reasoning + Acting)

현대 AI 에이전트 대부분은 **ReAct 패턴**을 사용합니다. "추론(Reasoning)하고 행동(Acting)한다"는 의미입니다.

```
사용자: "삼성전자의 오늘 주가를 알려줘"

[Thought]  "주가는 실시간 정보라서 내 학습 데이터에 없다. 검색 도구를 써야겠다."
[Action]   웹 검색 도구 실행: "삼성전자 주가 오늘"
[Observation] 검색 결과: "삼성전자 2025-04-07 종가: 61,400원 (전일 대비 +1.2%)"
[Thought]  "원하는 정보를 얻었다. 이제 답변할 수 있다."
[Final Answer] "삼성전자의 오늘(2025년 4월 7일) 종가는 61,400원으로, 전일 대비 1.2% 상승했습니다."
```

### verbose=True 로 루프 관찰하기

CrewAI에서 `verbose=True`를 설정하면 에이전트의 생각 과정을 콘솔에서 직접 볼 수 있습니다. 실습할 때 꼭 켜두세요!

```
> Entering new CrewAgentExecutor chain...
Thought: 검색이 필요하다...
Action: web_search
Action Input: {"query": "삼성전자 주가 오늘"}
Observation: 61,400원...
Thought: 답변할 수 있다.
Final Answer: 삼성전자의 오늘 종가는 61,400원...
> Finished chain.
```

## 4️⃣ AI 에이전트 프레임워크 생태계 [↑](#toc)

### 주요 프레임워크 비교

CrewAI만 있는 것이 아닙니다. 다양한 에이전트 프레임워크가 존재합니다.

| 프레임워크 | 핵심 특징 | 적합한 상황 | 난이도 |
|-----------|---------|-----------|--------|
| **CrewAI** | 역할 기반 멀티에이전트, 가장 직관적 | 팀 협업 시뮬레이션, 입문 | ★★☆☆☆ |
| **LangGraph** | 상태 기반 그래프, 정밀한 제어 | 복잡한 분기 로직, 루프 제어 | ★★★★☆ |
| **OpenAI Agents SDK** | Handoff / Guardrails 내장 | OpenAI 생태계 활용 | ★★☆☆☆ |
| **AutoGen** | 대화 기반 멀티에이전트 | 에이전트 간 자유로운 토론 | ★★★☆☆ |
| **Agno** | 경량 고성능, 멀티모달 지원 | 빠른 프로토타이핑 | ★★☆☆☆ |

> **이 수업에서는 CrewAI를 사용합니다.**
> 역할(role), 목표(goal), 배경(backstory)으로 에이전트를 정의하는 방식이 가장 직관적이고,
> 멀티에이전트 협업을 빠르게 실습할 수 있기 때문입니다.

### MCP (Model Context Protocol) 간단 소개

**MCP**는 Anthropic이 제안한 표준 프로토콜로, AI 모델과 외부 도구를 연결하는 방식을 통일합니다.

### USB 규격 비유

> USB가 등장하기 전에는 장치마다 연결 규격이 달랐습니다.
> MCP는 AI 도구 연결의 **USB 규격**입니다 — 어떤 AI 모델이든 MCP를 지원하면 같은 방식으로 도구에 연결됩니다.

Day 2에서 MCP를 CrewAI와 연동하는 실습을 진행합니다.
