## 2️⃣ 실무 적용 팁 [↑](#toc)

### 좋은 Agent backstory 작성법

backstory는 단순한 소개글이 아니라 에이전트의 **행동 방식과 전문성**을 결정합니다.

**나쁜 예:**

```python
backstory="당신은 여행 전문가입니다."
```

**좋은 예:**

```python
backstory=(
    "15년 경력의 여행 전문 컨설턴트로, 아시아 30개국을 직접 방문했습니다. "
    "예산 제약이 있는 여행자에게 최고의 경험을 제공하는 데 특화되어 있으며, "
    "현지인만 아는 숨겨진 명소를 발굴하는 것으로 유명합니다. "
    "항상 검증된 정보만 제공하며, 불확실한 정보는 반드시 '확인 필요'라고 표시합니다."
)
```

**핵심 요소:**

| 요소 | 설명 | 예시 |
|------|------|------|
| 구체적 경력 | 연차, 전문 분야 | "15년 경력", "아시아 30개국 방문" |
| 행동 원칙 | 어떻게 일하는지 | "검증된 정보만 제공" |
| 특기/강점 | 다른 에이전트와 구분 | "예산 최적화 특화" |
| 한계 인정 | 신뢰도 향상 | "불확실하면 '확인 필요' 표시" |

### 비용 최적화 전략

```python
from crewai import Agent, Crew, LLM

# 전략 1: 에이전트 역할에 따라 모델 분리
# - 단순 정리/요약: 저렴한 모델
# - 핵심 분석/판단: 고성능 모델

cheap_llm = LLM(model="gpt-4o-mini")    # 저렴 + 빠름
smart_llm = LLM(model="gpt-4o")        # 비쌈 + 정확

# 단순 작업 에이전트 → gpt-4o-mini
formatter = Agent(
    role="결과 정리 담당",
    llm=cheap_llm,
    # ...
)

# 핵심 분석 에이전트 → gpt-4o
analyst = Agent(
    role="전략 분석가",
    llm=smart_llm,
    # ...
)

# 전략 2: max_iter 제한으로 무한 루프 방지
agent_with_limit = Agent(
    role="리서치 담당",
    max_iter=5,      # 최대 5번 반복 후 종료 (기본값: 15)
    llm=cheap_llm
)

# 전략 3: verbose=False로 불필요한 토큰 사용 줄이기
# (verbose는 터미널 출력용이며 API 토큰에는 영향 없음,
#  하지만 Task description을 간결하게 유지하는 것이 핵심)

# 전략 4: cache 활용 (동일 요청 반복 시 캐시 사용)
crew = Crew(
    agents=[analyst],
    tasks=[task],

)
```

### 에러 처리 패턴

```python
from crewai import Crew
from crewai.flow.flow import Flow, listen, start
from pydantic import BaseModel

class SafeFlowState(BaseModel):
    topic: str = ""
    error: str = ""
    result: str = ""

class SafeFlow(Flow[SafeFlowState]):

    @start()
    def initialize(self):
        self.state.topic = "AI 트렌드"
        return self.state.topic

    @listen(initialize)
    def run_with_error_handling(self):
        """에러 처리가 포함된 Crew 실행"""
        try:
            # Crew 실행 시도
            crew = Crew(
                agents=[agent],
                tasks=[task],
                verbose=False
            )
            result = crew.kickoff()
            self.state.result = str(result)
            return self.state.result

        except Exception as e:
            # 에러 기록 후 기본값 반환
            self.state.error = str(e)
            self.state.result = "에러로 인해 결과를 생성할 수 없습니다."
            print(f"[경고] Crew 실행 실패: {e}")
            return self.state.result

# ─── Flow 실행 시 에러 처리 ────────────────────────────────
def run_flow_safely(inputs: dict) -> str:
    """Flow를 안전하게 실행"""
    try:
        flow = SafeFlow()
        result = flow.kickoff(inputs=inputs)
        return str(result)
    except KeyboardInterrupt:
        return "사용자가 실행을 중단했습니다."
    except Exception as e:
        return f"실행 실패: {e}"
```

## 4️⃣ Q&A + 피드백 [↑](#toc)

### 자주 묻는 질문

**Q. CrewAI와 LangGraph 중 어떤 것을 선택해야 하나요?**

> CrewAI는 설정이 간단하고 **역할 기반 멀티에이전트**에 적합합니다. LangGraph는 **복잡한 상태 머신과 루프**가 필요한 경우에 유리합니다. 처음에는 CrewAI로 시작하고, 더 정밀한 흐름 제어가 필요할 때 LangGraph를 배우는 순서를 권장합니다.

**Q. 로컬 LLM으로 GPT-4o 수준의 결과를 얻을 수 있나요?**

> 현재는 어렵습니다. Llama 3.2, Gemma 3 등 최신 오픈소스 모델은 많이 발전했지만, GPT-4o 수준의 복잡한 추론에는 아직 격차가 있습니다. 개인정보 처리나 비용 절감이 목적이라면 충분히 활용할 수 있습니다.

**Q. 에이전트가 환각(hallucination)을 일으키면 어떻게 하나요?**

> 1. Guardrail로 출력 검증 후 재시도
> 2. backstory에 "불확실한 정보는 반드시 표시하라" 명시
> 3. Task에 "정보 출처를 함께 제시하라" 추가
> 4. 검색 도구(SerperDev, DuckDuckGo)를 연결하여 실제 데이터 조회

**Q. API 비용이 걱정됩니다. 어떻게 관리하나요?**

> 1. 개발 중에는 `gpt-4o-mini` 사용 (약 20~30배 저렴)
> 2. `max_iter`를 낮게 설정 (기본 15 → 5)
> 3. OpenAI 플랫폼에서 사용량 알림 설정
> 4. 테스트 시 `verbose=True`로 실행 흐름 확인 후 불필요한 태스크 제거
