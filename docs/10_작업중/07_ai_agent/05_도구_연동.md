## 1️⃣ 도구(Tool)란? [↑](#toc)

### 팔과 손 비유

> LLM은 뛰어난 두뇌를 가졌지만, 혼자서는 실제 세계와 상호작용할 수 없습니다.
> 도구(Tools)는 에이전트에게 **실제로 행동할 수 있는 팔과 손**을 부여합니다.

```
LLM 단독:
  학습된 지식만 사용 → 최신 정보 없음, 계산 오류 가능, 파일 접근 불가

LLM + Tools:
  웹 검색 → 최신 뉴스, 실시간 주가, 날씨
  계산기   → 정확한 수치 계산
  파일 도구 → 문서 읽기/쓰기
  이메일   → 실제 이메일 발송
```

### 도구가 필요한 상황

| 상황 | 필요한 도구 |
|------|-----------|
| "오늘 삼성전자 주가 알려줘" | 웹 검색 도구 |
| "1024 × 768 × 3을 계산해줘" | 계산기 도구 |
| "이 PDF 파일을 요약해줘" | 파일 읽기 도구 |
| "회의 일정을 캘린더에 추가해줘" | 캘린더 API 도구 |

## 2️⃣ 내장 도구 사용 [↑](#toc)

`crewai-tools` 패키지에는 자주 쓰이는 도구들이 미리 구현되어 있습니다.

### SerperDevTool — 구글 검색 도구

```bash
# Serper API 키 발급: https://serper.dev (무료 2,500회/월)
# .env에 추가
SERPER_API_KEY=여기에_serper_api_키_입력
```

```python
from crewai_tools import SerperDevTool
from crewai import Agent

# 검색 도구 생성
search_tool = SerperDevTool()

# Agent에 도구 연결
researcher = Agent(
    role="리서치 전문가",
    goal="최신 정보를 검색하여 정확한 리서치 수행",
    backstory="최신 정보 검색이 특기인 전문 리서처입니다.",
    tools=[search_tool],   # ← 도구를 리스트로 전달
    verbose=True
)
```

### ScrapeWebsiteTool — 웹페이지 내용 추출

```python
from crewai_tools import ScrapeWebsiteTool

# 특정 URL의 내용을 읽어오는 도구
scrape_tool = ScrapeWebsiteTool()

# Agent에 추가
researcher = Agent(
    role="웹 분석 전문가",
    goal="웹페이지의 핵심 정보를 추출하고 분석한다",
    backstory="웹 콘텐츠 분석 전문가입니다.",
    tools=[search_tool, scrape_tool],  # 여러 도구 동시 연결 가능
    verbose=True
)
```

### FileReadTool — 파일 읽기

```python
from crewai_tools import FileReadTool

file_tool = FileReadTool(file_path="./data/report.txt")

analyst = Agent(
    role="문서 분석가",
    goal="문서의 핵심 내용을 요약하고 인사이트를 도출한다",
    backstory="문서 분석 전문가입니다.",
    tools=[file_tool],
    verbose=True
)
```

| 도구 | 패키지 | 역할 | API 키 필요 |

## 3️⃣ 커스텀 도구 만들기 [↑](#toc)

### BaseTool 상속 방식

내장 도구에 없는 기능이 필요하면 직접 만들 수 있습니다. `BaseTool`을 상속하고 `_run` 메서드를 구현합니다.

```python
from crewai.tools import BaseTool

class CalculatorTool(BaseTool):
    name: str = "계산기"                               # 도구 이름 (LLM이 이 이름으로 인식)
    description: str = "수학 계산을 수행합니다. 입력: 수식 문자열 (예: '10 * 20 + 5')"

    def _run(self, expression: str) -> str:
        """수학 계산을 수행하고 결과를 반환합니다."""
        try:
            result = eval(expression)
            return f"계산 결과: {expression} = {result}"
        except Exception as e:
            return f"계산 오류: {str(e)}"
```

### 커스텀 도구 사용

```python
calc_tool = CalculatorTool()

analyst = Agent(
    role="데이터 분석가",
    goal="정확한 수치 계산을 통해 데이터를 분석한다",
    backstory="정확한 계산과 데이터 분석이 특기인 전문가입니다.",
    tools=[calc_tool],
    verbose=True
)
```

### 좀 더 실용적인 커스텀 도구: 환율 계산기

```python
from crewai.tools import BaseTool

class CurrencyConverterTool(BaseTool):
    name: str = "환율 계산기"
    description: str = (
        "달러(USD)를 원화(KRW)로 변환합니다. "
        "입력 형식: '숫자 USD' (예: '100 USD')"
    )

    def _run(self, amount_usd: str) -> str:
        """달러를 원화로 변환합니다. (고정 환율 1370원 사용)"""
        try:
            # 입력에서 숫자 추출
            amount = float(amount_usd.replace(" USD", "").replace(",", "").strip())
            rate = 1370  # 실습용 고정 환율 (실제 서비스는 API 사용)
            krw = amount * rate
            return f"{amount:,.0f} USD = {krw:,.0f} 원 (환율: 1 USD = {rate} 원)"
        except Exception as e:
            return f"변환 오류: {str(e)}. 올바른 형식: '100 USD'"
```

### `@tool` 데코레이터 방식 (간단한 경우)

BaseTool보다 간단하게 도구를 만드는 방법도 있습니다.

```python
from crewai.tools import tool

@tool("텍스트 글자 수 세기")
def count_characters(text: str) -> str:
    """텍스트의 글자 수(공백 포함/제외)를 계산합니다."""
    total = len(text)
    without_spaces = len(text.replace(" ", ""))
    return f"전체: {total}자, 공백 제외: {without_spaces}자"
```

## 4️⃣ 도구 선택 원리 [↑](#toc)

### LLM이 어떻게 도구를 고르는가

LLM은 도구의 `name`과 `description`을 읽고, 현재 상황에서 어떤 도구를 써야 할지 결정합니다.

```python
# 에이전트에게 이런 도구들이 제공됩니다
tools = [
    SerperDevTool(),       # name: "Search", description: "구글에서 최신 정보 검색"
    CalculatorTool(),      # name: "계산기",  description: "수학 계산을 수행"
    CurrencyConverterTool()# name: "환율 계산기", description: "달러를 원화로 변환"
]

# 에이전트가 "오늘 환율로 100달러는 얼마야?" 라는 Task를 받으면:
# LLM 추론: "현재 환율 정보가 필요하다 → Search 도구로 검색"
# LLM 추론: "100달러 → 원화 변환이 필요하다 → 환율 계산기 도구 사용"
```

### description 작성의 중요성

```python
# 나쁜 예 — LLM이 언제 써야 할지 모름
description: str = "계산"

# 좋은 예 — 언제, 어떻게 사용하는지 명확
description: str = (
    "수학 계산을 수행합니다. "
    "덧셈, 뺄셈, 곱셈, 나눗셈, 거듭제곱 등이 가능합니다. "
    "입력: 파이썬 수식 문자열 (예: '(1500 * 0.1) + 200')"
)
```

> **핵심 원칙**: description이 명확할수록 LLM이 올바른 도구를 선택합니다.
> 도구가 제대로 사용되지 않는다면, 먼저 description을 개선해보세요.

## 5️⃣ 실습: 검색+계산 에이전트 [↑](#toc)

웹 검색으로 현재 환율을 찾고, 계산기로 금액을 변환하는 에이전트를 만들어봅니다.

> **참고**: Serper API 키가 없는 경우, `SerperDevTool` 대신 커스텀 `CurrencyConverterTool`만 사용하는 버전으로도 실습 가능합니다.

```python
# 파일: currency_agent.py
from dotenv import load_dotenv
load_dotenv()

from crewai import Agent, Task, Crew, Process, LLM
from crewai.tools import BaseTool

# ── 커스텀 도구 정의 ───────────────────────────────────
class CalculatorTool(BaseTool):
    name: str = "계산기"
    description: str = (
        "수학 계산을 수행합니다. "
        "입력: 파이썬 수식 문자열 (예: '1370 * 100', '(500 + 200) * 1370')"
    )

    def _run(self, expression: str) -> str:
        try:
            result = eval(expression)
            return f"계산 결과: {expression} = {result:,.2f}"
        except Exception as e:
            return f"계산 오류: {str(e)}"


class ExchangeRateTool(BaseTool):
    name: str = "환율 정보 조회"
    description: str = (
        "현재 주요 통화의 원화 환율을 제공합니다. "
        "입력: 통화 코드 (USD, EUR, JPY, CNY 중 하나)"
    )

    # 실습용 고정 환율 (실제 서비스는 외부 API 사용)
    _rates = {
        "USD": 1370,
        "EUR": 1510,
        "JPY": 9.2,    # 100엔 기준
        "CNY": 189,
    }

    def _run(self, currency: str) -> str:
        currency = currency.strip().upper()
        if currency in self._rates:
            rate = self._rates[currency]
            if currency == "JPY":
                return f"현재 환율: 100 {currency} = {rate * 100:,} 원"
            return f"현재 환율: 1 {currency} = {rate:,} 원"
        return f"지원하지 않는 통화입니다. 지원 통화: {', '.join(self._rates.keys())}"


# ── 에이전트 설정 ──────────────────────────────────────
llm = LLM(model="gpt-4o-mini")

currency_agent = Agent(
    role="환율 계산 전문가",
    goal="정확한 환율 정보를 제공하고 다양한 통화 간 금액을 계산한다",
    backstory=(
        "당신은 외환 딜러 출신의 환율 전문가입니다. "
        "여러 통화의 실시간 환율 정보를 파악하고 "
        "정확한 환산 금액을 제공하는 것이 특기입니다."
    ),
    llm=llm,
    tools=[ExchangeRateTool(), CalculatorTool()],
    verbose=True
)

# ── Task 설정 ──────────────────────────────────────────
currency_task = Task(
    description=(
        "다음 환전 요청에 대해 정확한 정보를 제공하세요:\n"
        "1. 현재 USD/KRW, EUR/KRW 환율을 조회하세요.\n"
        "2. 500달러(USD)를 원화로 변환하세요.\n"
        "3. 300유로(EUR)를 원화로 변환하세요.\n"
        "4. 두 금액의 합계를 계산하세요."
    ),
    expected_output=(
        "각 통화의 현재 환율, 변환된 원화 금액, 그리고 합계를 포함한 환전 보고서. "
        "마크다운 형식으로 표를 사용하면 더 좋습니다."
    ),
    agent=currency_agent
)

# ── Crew 실행 ──────────────────────────────────────────
crew = Crew(
    agents=[currency_agent],
    tasks=[currency_task],
    process=Process.sequential,
    verbose=True
)

result = crew.kickoff()

print("\n" + "=" * 60)
print("환율 계산 결과:")
print("=" * 60)
print(result)
```

## 6️⃣ 정리 [↑](#toc)

### 주요 내장 도구 요약

| 도구 | 임포트 | 주요 용도 |
|------|-------|---------|
| `SerperDevTool` | `from crewai_tools import SerperDevTool` | 구글 검색 |
| `ScrapeWebsiteTool` | `from crewai_tools import ScrapeWebsiteTool` | 웹 스크래핑 |
| `FileReadTool` | `from crewai_tools import FileReadTool` | 파일 읽기 |
| `FileWriterTool` | `from crewai_tools import FileWriterTool` | 파일 쓰기 |
| `PDFSearchTool` | `from crewai_tools import PDFSearchTool` | PDF 검색 |

### 커스텀 도구 작성 템플릿

```python
from crewai.tools import BaseTool

class MyCustomTool(BaseTool):
    name: str = "도구 이름 (LLM이 읽는 이름)"
    description: str = "언제, 어떻게 사용하는지 설명 (입력 형식 포함)"

    def _run(self, input_data: str) -> str:
        # 도구 로직 구현
        result = ...
        return str(result)
```

### 도구 설계 핵심 원칙

1. `name`은 짧고 직관적으로
2. `description`은 **언제** 사용하는지 + **입력 형식** 명시
3. `_run`은 항상 문자열(str) 반환
4. 예외 처리 필수 — 오류 메시지도 문자열로 반환
