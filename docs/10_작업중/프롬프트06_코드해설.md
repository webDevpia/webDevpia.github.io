### Code 1-1: 첫 번째 API 호출 [↑](#toc)

```python
!pip install openai -q

import os
from openai import OpenAI
from google.colab import userdata

# Colab Secrets에서 API 키 가져오기
os.environ['OPENAI_API_KEY'] = userdata.get('OPENAI_API_KEY')

# OpenAI 클라이언트 생성
client = OpenAI()

# 첫 번째 API 호출
response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": "프롬프트 엔지니어링이 뭔지 한 문장으로 설명해줘"}]
)

# 결과 출력
print(response.choices[0].message.content)
```

**줄별 해설**

① `!pip install openai -q`
- `pip`는 파이썬의 **앱스토어**입니다. 스마트폰에서 앱을 설치하듯, 파이썬에서 외부 도구(라이브러리)를 설치할 때 씁니다.
- `install openai` = "openai 라는 도구를 설치해줘"
- `-q`(quiet) = "설치하는 동안 너무 많은 메시지를 출력하지 마"
- 맨 앞의 `!`는 구글 코랩(Colab) 환경에서 터미널 명령어를 실행할 때 붙이는 표시입니다.

② `import os`
- `import`는 **"이미 만들어진 도구 상자를 꺼내 쓰겠다"**는 뜻입니다. `os`는 운영체제(Operating System)와 소통하는 도구 상자로, 여기서는 환경 변수를 설정하는 데 쓰입니다.

③ `from openai import OpenAI`
- `from openai` = "openai 도구 상자에서"
- `import OpenAI` = "OpenAI 라는 특정 도구만 꺼내온다"
- 도구 상자 전체가 아니라 필요한 도구 하나만 골라 쓰는 것입니다.

④ `from google.colab import userdata`
- 구글 코랩의 비밀번호 보관함(`userdata`)을 꺼내옵니다. API 키처럼 외부에 노출되면 안 되는 값을 안전하게 저장하고 꺼내는 용도입니다.

⑤ `# Colab Secrets에서 API 키 가져오기`
- `#`으로 시작하는 줄은 **주석**입니다. 파이썬이 실행하지 않는 줄로, 코드를 읽는 사람을 위한 메모입니다.

⑥ `os.environ['OPENAI_API_KEY'] = userdata.get('OPENAI_API_KEY')`
- `os.environ`은 프로그램이 실행되는 환경의 **비밀 서랍**입니다. 다른 도구들이 이 서랍을 열어 API 키를 가져갑니다.
- `['OPENAI_API_KEY']` = 서랍 안의 `OPENAI_API_KEY`라는 이름의 칸
- `=` = **대입 연산자**. "오른쪽 값을 왼쪽에 저장해라"는 뜻입니다. 수학의 등호와 다릅니다.
- `userdata.get('OPENAI_API_KEY')` = 코랩 비밀번호 보관함에서 `OPENAI_API_KEY` 값을 꺼낸다

⑦ `client = OpenAI()`
- `client`라는 **변수**를 만들고, OpenAI API에 연결하는 객체를 저장합니다.
- **변수**란 값을 담는 **이름표가 붙은 상자**입니다. `client`라는 이름표를 붙인 상자에 OpenAI 연결 도구를 넣은 것입니다.
- `OpenAI()`처럼 이름 뒤에 `()`가 붙으면 "이 도구를 실제로 꺼내 사용할 준비를 해라"는 뜻입니다.

⑧ `response = client.chat.completions.create(...)`
- `response` = AI의 대답을 담을 상자
- `client.chat.completions.create(...)` = "연결된 OpenAI API에게 대화(chat) 완성(completions) 요청을 생성(create)해라"
- `.`(점)은 도구 상자에서 세부 기능을 꺼내는 기호입니다. `client` → `chat` → `completions` → `create` 순서로 점점 깊이 들어가는 것입니다.

⑨ `model="gpt-4.1-nano"`
- 어떤 AI 모델을 쓸지 지정합니다. 메뉴판에서 음식 이름을 고르는 것과 같습니다.
- `"..."` 처럼 큰따옴표로 감싸인 값을 **문자열**이라고 합니다. 텍스트 데이터를 표현하는 방식입니다.

⑩ `messages=[{"role": "user", "content": "프롬프트 엔지니어링이 뭔지 한 문장으로 설명해줘"}]`
- `messages=` = AI에게 보낼 대화 내역 목록
- `[...]` = **리스트(목록)**입니다. 여러 항목을 순서대로 나열할 때 쓰는 대괄호 묶음입니다.
- `{...}` = **딕셔너리**입니다. `"키": "값"` 쌍으로 정보를 저장하는 중괄호 묶음입니다. 이름표가 붙은 서랍장과 같습니다.
- `"role": "user"` = "이 메시지의 역할은 사용자(user)가 보낸 것"
- `"content": "프롬프트 엔지니어링이..."` = "메시지 내용은 이 문장"

⑪ `print(response.choices[0].message.content)`
- `print(...)` = 괄호 안의 값을 화면에 출력합니다.
- `response.choices` = AI가 돌려준 대답 후보들의 목록
- `[0]` = 목록에서 첫 번째 항목을 꺼냄 (파이썬에서 번호는 0부터 시작합니다)
- `.message.content` = 그 대답의 실제 텍스트 내용

<a id="code1-2"></a>
### Code 1-2: temperature 비교 [↑](#toc)

```python
prompt = "파이썬에서 for문과 while문의 차이를 한 문장으로 설명해줘."

for temp in [0.0, 0.7, 1.5]:
    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[{"role": "user", "content": prompt}],
        temperature=temp,
        max_tokens=100
    )
    print(f"\n[temperature={temp}]")
    print(response.choices[0].message.content)
```

**줄별 해설**

① `prompt = "파이썬에서 for문과 while문의..."`
- `prompt`라는 이름표 상자에 질문 문장을 저장합니다. 나중에 이 상자 이름만 써도 이 문장 전체를 불러올 수 있습니다.

② `for temp in [0.0, 0.7, 1.5]:`
- **for문**은 목록 안의 항목을 하나씩 꺼내며 같은 작업을 반복합니다. 공장 컨베이어 벨트에서 물건을 하나씩 처리하는 것과 같습니다.
- `temp`는 매 반복마다 목록에서 꺼낸 값(0.0, 그 다음엔 0.7, 마지막엔 1.5)을 담는 임시 상자입니다.
- `[0.0, 0.7, 1.5]`는 앞에서 배운 리스트로, 세 개의 숫자를 순서대로 담고 있습니다.
- 줄 끝의 `:`는 "이 아래부터 반복할 내용이 시작된다"는 신호입니다.

③~⑦ `response = client.chat.completions.create(...)` (들여쓰기된 부분 전체)
- 앞에서 배운 API 호출과 동일합니다. 단, 이번에는 `for`문 안에 있으므로 `temp` 값을 바꿔가며 세 번 반복합니다.
- **들여쓰기(indentation)**: 파이썬에서는 줄 앞의 공백(들여쓰기)이 매우 중요합니다. 들여쓰기된 줄들은 `for`문의 반복 범위에 속한다는 뜻입니다.

⑤ `temperature=temp`
- `temperature`는 AI 응답의 **창의성/무작위성** 수준을 조절합니다. 요리사에게 비유하면, 0에 가까울수록 "레시피대로만 만들어", 높을수록 "마음대로 창의적으로 만들어"에 해당합니다.
- `temp`는 위의 `for`문에서 매 반복마다 바뀌는 값을 담은 임시 상자입니다.

⑥ `max_tokens=100`
- AI가 응답으로 생성할 최대 토큰(단어 조각) 수입니다. 100으로 제한하면 짧게 답합니다. 글자 수 제한과 비슷한 개념입니다.

⑧ `print(f"\n[temperature={temp}]")`
- `f"..."` = **f-문자열**입니다. 따옴표 안에서 `{변수명}` 형태로 변수의 값을 바로 끼워 넣을 수 있습니다. 마치 빈칸 채우기 양식과 같습니다.
- `\n`은 줄바꿈 기호입니다. 이 기호가 있는 위치에서 줄이 바뀝니다.
- 결과 예시: `[temperature=0.7]`

⑨ `print(response.choices[0].message.content)`
- 앞에서 배운 것과 동일하게 AI의 응답 텍스트를 출력합니다.

### Code 2-1: CRAFT API 코드 예시 [↑](#toc)

```python
response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[
        {"role": "system", "content": "당신은 친절한 파이썬 튜터입니다. 비전공자 학생이 이해할 수 있게 비유를 활용해 설명합니다."},
        {"role": "user", "content": """파이썬을 배우기 시작한 비전공자 학생입니다.
리스트와 딕셔너리의 차이를 설명하고, 각각 언제 쓰는지 코드 예시와 함께 비교표로 정리해주세요.

| 항목 | 리스트 | 딕셔너리 |"""}
    ]
)
print(response.choices[0].message.content)
```

**줄별 해설**

① `response = client.chat.completions.create(...)` — 앞에서 배운 API 호출 구조입니다.

② `messages=[...]` — 앞에서는 메시지가 하나였지만, 이번에는 두 개입니다.

③ 첫 번째 메시지: `{"role": "system", "content": "당신은 친절한..."}`
- `"role": "system"` = 이 메시지는 AI에게 역할을 부여하는 시스템 지시입니다. 사용자가 보내는 `"user"` 메시지와 달리, 대화 무대 뒤에서 AI에게 "넌 이런 역할이야"라고 알려주는 연출 지시서입니다.
- `"content"` 안의 내용이 AI의 역할과 태도를 결정합니다.

④ 두 번째 메시지: `{"role": "user", "content": """..."""}`
- `"""..."""` = **삼중 따옴표**입니다. 여러 줄에 걸친 긴 문장을 하나의 문자열로 표현할 때 씁니다. 일반 따옴표 `"..."`는 한 줄만 가능하지만, `"""..."""`은 줄바꿈을 포함할 수 있습니다.

⑤ 표 형식 `| 항목 | 리스트 | 딕셔너리 |`
- 프롬프트 안에 마크다운 표 머리글을 미리 써두면, AI가 이 표를 이어서 완성하는 방향으로 응답합니다. AI에게 출력 형식을 "시범"으로 보여주는 기법입니다.

### Code 2-2: Zero-shot API 코드 [↑](#toc)

```python
prompt = """다음 파이썬 에러 메시지의 원인을 '문법 오류', '타입 오류', '인덱스 오류' 중 하나로 분류해주세요.

분류:"""

    messages=[{"role": "user", "content": prompt}]
)
print(response.choices[0].message.content)
```

**줄별 해설**

① `prompt = """..."""`
- 앞에서 배운 삼중 따옴표로 여러 줄 프롬프트를 작성합니다. `prompt` 상자에 저장합니다.

② 프롬프트 내용의 마지막 줄 `분류:`
- 이렇게 프롬프트를 완성되지 않은 채로 끝내면, AI는 자연스럽게 `분류:` 다음을 채워 넣습니다. 마치 빈칸 채우기 문제의 빈칸을 AI가 메우는 것과 같습니다. **Zero-shot**이란 예시 없이 바로 질문하는 방식입니다.

③~⑤ API 호출과 출력 — 앞에서 배운 것과 동일합니다.

### Code 2-3: Few-shot API 코드 [↑](#toc)

```python
few_shot_prompt = """다음 파이썬 에러 메시지를 분류해주세요.

에러: "SyntaxError: invalid syntax"
분류: 문법 오류

에러: "IndexError: list index out of range"
분류: 인덱스 오류

에러: "NameError: name 'x' is not defined"
분류: 이름 오류

에러: "TypeError: unsupported operand type(s) for +: 'int' and 'str'"
분류:"""

response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": few_shot_prompt}]
)
print(response.choices[0].message.content)
```

**줄별 해설**

① `few_shot_prompt = """..."""`
- `few_shot_prompt` = "소수(few)의 예시(shot)가 담긴 프롬프트"라는 의미의 변수 이름입니다. 프롬프트 내용은 삼중 따옴표로 감싼 여러 줄 문자열입니다.

② 프롬프트 본문 (예시 세 쌍)
- **Few-shot 기법**의 핵심은 "질문-정답" 쌍을 여러 개 미리 보여줘서 AI가 패턴을 파악하게 하는 것입니다. 마치 학생에게 문제 유형을 예시 풀이로 보여준 뒤 새 문제를 주는 것과 같습니다.
- 세 번의 `에러: ... / 분류: ...` 쌍이 예시 역할을 합니다.

③ 마지막 줄 `분류:`
- 앞에서 배운 것처럼 AI가 여기를 채워 넣도록 유도합니다.

<a id="code2-4"></a>
### Code 2-4: Role Prompting — 역할별 응답 비교 [↑](#toc)

```python
def ask_with_role(role_desc, question):
    """역할을 부여하고 질문하는 함수"""
    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[
            {"role": "system", "content": role_desc},
            {"role": "user", "content": question}
        ]
    )
    return response.choices[0].message.content

question = "CNN(합성곱 신경망)이 이미지를 인식하는 원리를 설명해주세요."

roles = {
    "신입 교육 담당 사수": "당신은 3년차 ML 엔지니어이고 신입 교육을 담당합니다. 쉬운 비유와 그림을 활용해 설명합니다.",
    "대학교 교수": "당신은 컴퓨터공학과 교수입니다. 정확한 수학적 개념과 논문 레퍼런스를 포함하여 설명합니다.",
    "유튜브 크리에이터": "당신은 IT 유튜버입니다. 재미있는 예시와 밈을 활용해 누구나 이해할 수 있게 설명합니다.",
}

for name, role in roles.items():
    print(f"\n{'='*50}")
    print(f"[{name}]")
    print(ask_with_role(role, question))
```

**줄별 해설**

① `def ask_with_role(role_desc, question):`
- `def`는 **함수(function)를 정의**하는 키워드입니다. 함수란 "자주 쓰는 작업을 묶어서 이름을 붙인 레시피"입니다.
- `ask_with_role`이 이 레시피의 이름입니다.
- `(role_desc, question)`는 **매개변수(파라미터)**로, 레시피를 실행할 때 필요한 재료 목록입니다. `role_desc`에는 역할 설명을, `question`에는 질문을 넣어야 합니다.

② `"""역할을 부여하고 질문하는 함수"""`
- 삼중 따옴표로 감싼 이 줄은 **독스트링(docstring)**이라고 합니다. 함수가 어떤 역할을 하는지 설명하는 메모로, 파이썬이 실행하지는 않습니다.

③~⑧ `response = client.chat.completions.create(...)` — 앞에서 배운 API 호출과 동일합니다. 다만 `role_desc`와 `question`은 함수를 호출할 때 전달받은 재료 값으로 채워집니다.

⑨ `return response.choices[0].message.content`
- `return`은 **"이 함수를 호출한 곳에 이 값을 돌려줘라"**는 뜻입니다. 음식점에서 요리를 완성해 손님 테이블로 가져다주는 것과 같습니다.

⑩ `question = "CNN(합성곱 신경망)이..."`
- 모든 역할에 공통으로 쓸 질문을 하나의 변수에 저장합니다.

⑪ `roles = {"신입 교육 담당 사수": "당신은...", "대학교 교수": "...", "유튜브 크리에이터": "..."}`
- **딕셔너리**입니다. `"역할 이름": "역할 설명"` 형태로 세 가지 역할을 저장합니다. 서랍장의 각 칸에 이름표를 붙여둔 것과 같습니다.

⑫ `for name, role in roles.items():`
- 앞에서 배운 for문입니다. `roles.items()`는 딕셔너리의 "이름표-내용" 쌍을 모두 꺼내줍니다.
- 매 반복마다 `name`에는 역할 이름(예: "신입 교육 담당 사수"), `role`에는 역할 설명이 담깁니다.

⑬ `print(f"\n{'='*50}")`
- `'='*50`은 `=` 기호를 50번 반복하라는 뜻입니다. 출력 결과에서 구분선 역할을 합니다.
- 앞에서 배운 f-문자열 안에서 `{...}`로 계산 결과를 바로 넣었습니다.

⑭ `print(ask_with_role(role, question))`
- 위에서 정의한 `ask_with_role` 레시피를 `role`과 `question` 재료를 넣어 실행하고, 돌아온 결과를 출력합니다.

```python
import json

response = client.chat.completions.create(

    messages=[
        {"role": "system", "content": "응답은 반드시 JSON 형식으로만 출력하세요."},

    ],

data = json.loads(response.choices[0].message.content)

print(f"분류: {data['category']}")

```

**줄별 해설**

① `import json`
- `json`은 파이썬 기본 도구 상자로, JSON 형식의 텍스트를 파이썬이 다룰 수 있는 딕셔너리/리스트로 변환하거나 반대로 변환하는 기능을 담고 있습니다.
- **JSON**이란 데이터를 교환하기 위한 표준 텍스트 형식으로, `{"키": "값"}` 구조입니다. 컴퓨터들이 서로 데이터를 주고받을 때 사용하는 공통 언어입니다.

② `response_format={"type": "json_object"}`
- AI에게 "반드시 JSON 형식으로만 답해라"고 강제하는 설정입니다. 일반 텍스트 대신 딕셔너리처럼 생긴 구조화된 텍스트를 반환하도록 합니다.

③ `data = json.loads(response.choices[0].message.content)`
- `json.loads(...)` = JSON 형식의 **문자열**을 파이썬 **딕셔너리**로 변환합니다. 택배 박스(JSON 문자열)를 열어서 내용물(딕셔너리)을 꺼내는 것과 같습니다.
- 변환된 결과를 `data` 상자에 저장합니다.

④ `print(f"이름: {data['name']}")`
- `data['name']`은 `data` 딕셔너리에서 `'name'` 이름표가 붙은 칸의 값을 꺼냅니다. 서랍장에서 `'name'` 서랍을 여는 것과 같습니다.

⑤ `print(json.dumps(data, indent=2, ensure_ascii=False))`
- `json.dumps(...)` = 파이썬 딕셔너리를 JSON 형식의 **문자열**로 변환합니다. `loads`의 반대 방향입니다.
- `indent=2` = 들여쓰기를 2칸 적용해서 사람이 읽기 좋게 보기 좋게 출력합니다.
- `ensure_ascii=False` = 한글 등 비영어 문자를 `\uc548\ub155` 같은 코드로 바꾸지 말고 원래 글자 그대로 출력합니다.

### Code 3-1: CoT vs 일반 비교 [↑](#toc)

```python
import time

problem = """딥러닝 모델 A는 학습 시간 2시간에 정확도 92%,
모델 B는 학습 시간 8시간에 정확도 95%입니다.
어떤 모델이 효율적인가요?"""

response_no_cot = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": problem + "\n답:"}]
)

response_cot = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": problem + "\n단계별로 계산해서 답해주세요."}]
)

print("=== CoT 없이 ===")
print(response_no_cot.choices[0].message.content)
print("\n=== CoT 적용 ===")
print(response_cot.choices[0].message.content)
```

**줄별 해설**

① `import time`
- 시간 관련 기능을 담은 도구 상자를 불러옵니다. 이 코드에서는 실제로 쓰이지 않지만, 원래 실행 시간을 측정할 목적으로 포함된 것입니다.

② `problem = """..."""`
- 앞에서 배운 삼중 따옴표로 여러 줄 문제를 저장합니다.

③ `messages=[{"role": "user", "content": problem + "\n답:"}]`
- `problem + "\n답:"` = 문자열 더하기입니다. `problem`에 저장된 문장 뒤에 `\n답:`을 붙입니다. 결과는 "...어떤 모델이 효율적인가요?\n답:"이 됩니다.
- `\n`은 앞에서 배운 줄바꿈 기호입니다.

④ CoT 적용 버전: `problem + "\n단계별로 계산해서 답해주세요."`
- **CoT(Chain-of-Thought)**는 "단계별로 생각해라"고 유도하는 프롬프트 기법입니다. 이 한 문장을 붙이는 것만으로 AI의 추론 품질이 높아집니다.
- "단계별로 계산해서 답해주세요"라는 문구가 CoT 트리거입니다.

### Code 3-3: 멀티턴 대화 구현 [↑](#toc)

```python
messages = [
    {"role": "system", "content": "당신은 친절한 파이썬/딥러닝 튜터입니다. 코드 예시를 포함하여 쉽게 설명합니다."}
]

def chat(user_input):
    """대화 히스토리를 유지하며 AI와 대화합니다."""
    messages.append({"role": "user", "content": user_input})

    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=messages,
        temperature=0.7,
    )

    answer = response.choices[0].message.content
    messages.append({"role": "assistant", "content": answer})
    return answer

print("[나]:", "YOLO로 객체 탐지 프로젝트를 시작하려는데 뭐부터 해야 해?")
print("[AI]:", chat("YOLO로 객체 탐지 프로젝트를 시작하려는데 뭐부터 해야 해?"))

print("\n[나]:", "방금 말한 것 중에서 데이터셋 준비를 좀 더 자세히 알려줘")
print("[AI]:", chat("방금 말한 것 중에서 데이터셋 준비를 좀 더 자세히 알려줘"))

print(f"\n현재 대화 턴 수: {len(messages)}개 메시지")
```

**줄별 해설**

① `messages = [{"role": "system", "content": "..."}]`
- `messages`라는 리스트를 만들고, 처음부터 system 메시지(AI 역할 지시)를 담아 초기화합니다.
- 이 리스트가 전체 대화 기록 노트가 됩니다.

② `def chat(user_input):`
- 앞에서 배운 `def`로 함수를 정의합니다. `user_input`이라는 재료(사용자가 입력한 텍스트)를 받습니다.

③ `messages.append({"role": "user", "content": user_input})`
- `append()`는 리스트의 **맨 뒤에 항목을 추가**하는 기능입니다. 대화 노트에 사용자 발언을 한 줄 추가하는 것과 같습니다.
- `{"role": "user", "content": user_input}` = 사용자가 방금 입력한 내용을 딕셔너리 형태로 만든 것

④ `response = client.chat.completions.create(model=..., messages=messages, temperature=0.7)`
- `messages=messages` = 지금까지 쌓인 전체 대화 기록을 통째로 AI에게 전송합니다. AI가 이전 대화 맥락을 기억할 수 있는 이유입니다.

⑤ `answer = response.choices[0].message.content`
- AI의 응답 텍스트를 `answer` 상자에 저장합니다.

⑥ `messages.append({"role": "assistant", "content": answer})`
- AI의 응답도 대화 노트에 추가합니다. `"role": "assistant"`는 AI(어시스턴트)가 한 말임을 나타냅니다.

⑦ `return answer`
- 앞에서 배운 `return`으로 AI 응답을 함수 밖으로 돌려줍니다.

⑧ `print("[나]:", "YOLO로 객체 탐지...")`
- `print()`에 쉼표로 구분된 여러 값을 넣으면 공백으로 연결하여 출력합니다.

⑨ `print("[AI]:", chat("YOLO로 객체 탐지..."))`
- `chat(...)` 함수를 호출하여 대화하고, 돌아온 응답을 출력합니다.

⑩ `print(f"\n현재 대화 턴 수: {len(messages)}개 메시지")`
- `len(messages)` = `messages` 리스트 안의 항목 개수를 반환합니다. `len`은 길이(length)의 줄임말입니다. 서랍 개수를 세는 것과 같습니다.

### Code 3-5: 배치 처리 [↑](#toc)

```python
concepts = [
    "Overfitting",
    "Batch Normalization",
    "Dropout",
    "Transfer Learning",
]

results = []
for concept in concepts:
    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[{"role": "user", "content": f"'{concept}'을(를) 비전공자 학생이 이해할 수 있게 한 문장으로 설명하고, 실생활 비유를 한 줄 추가해줘."}],
        max_tokens=150
    )
    results.append({
        "개념": concept,
        "설명": response.choices[0].message.content.strip()
    })

for r in results:
    print(f"[{r['개념']}]")
    print(f"{r['설명']}\n")
```

**줄별 해설**

① `concepts = ["Overfitting", "Batch Normalization", "Dropout", "Transfer Learning"]`
- 처리할 딥러닝 개념들을 리스트에 저장합니다. 앞에서 배운 대괄호 `[...]` 리스트입니다.

② `results = []`
- 결과를 모아둘 **빈 리스트**를 만듭니다. 빈 바구니를 준비하는 것과 같습니다.

③ `for concept in concepts:`
- 앞에서 배운 for문으로 `concepts` 목록을 순서대로 하나씩 꺼내며 반복합니다.

④ `f"'{concept}'을(를)..."`
- 앞에서 배운 f-문자열로 `concept`의 실제 값을 프롬프트 안에 끼워 넣습니다. 예: `'Overfitting'을(를) 비전공자...`

⑤ `results.append({"개념": concept, "설명": response.choices[0].message.content.strip()})`
- 앞에서 배운 `append()`로 결과를 바구니에 추가합니다.
- 저장하는 값은 `"개념"`과 `"설명"` 두 항목을 가진 딕셔너리입니다.
- `.strip()` = 문자열 앞뒤의 불필요한 공백이나 줄바꿈을 제거합니다. 답변을 깔끔하게 정리하는 도구입니다.

⑥ `for r in results:` (두 번째 for문)
- 모든 API 호출이 끝난 뒤, 결과 바구니(`results`)를 순서대로 꺼내며 출력합니다. `r`은 매 반복마다 꺼낸 딕셔너리 하나입니다.

⑦ `print(f"[{r['개념']}]")`
- `r['개념']`은 `r` 딕셔너리에서 `'개념'` 키의 값을 꺼냅니다. 결과: `[Overfitting]`

### Code 3-6: 토큰 사용량과 비용 확인 [↑](#toc)

```python
response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": "CNN과 RNN의 차이를 비교표로 정리하고, 각각 어떤 프로젝트에 적합한지 설명해줘"}]
)

usage = response.usage
print(f"입력 토큰:  {usage.prompt_tokens}")
print(f"출력 토큰:  {usage.completion_tokens}")
print(f"총 토큰:    {usage.total_tokens}")

input_cost = usage.prompt_tokens * 0.0000001
output_cost = usage.completion_tokens * 0.0000004
total_cost = input_cost + output_cost
print(f"예상 비용:  ${total_cost:.6f} (약 {total_cost * 1400:.2f}원)")
```

**줄별 해설**

① API 호출 — 앞에서 배운 것과 동일합니다.

② `usage = response.usage`
- `response.usage`는 API가 이번 호출에서 사용한 토큰 정보를 담고 있는 객체입니다. `usage` 상자에 저장합니다.

③ `usage.prompt_tokens` / `usage.completion_tokens` / `usage.total_tokens`
- 각각 입력 토큰 수, 출력 토큰 수, 전체 토큰 수를 꺼냅니다.
- **토큰**은 AI가 텍스트를 처리하는 단위입니다. 한국어 한 글자가 약 1~2 토큰, 영어 단어 하나가 약 1 토큰 정도입니다.

④ `input_cost = usage.prompt_tokens * 0.0000001`
- `*`는 **곱셈 기호**입니다. 토큰 수에 1토큰당 가격을 곱해 비용을 계산합니다.

⑤ `output_cost = usage.completion_tokens * 0.0000004`
- 출력 토큰은 입력보다 4배 비쌉니다.

⑥ `total_cost = input_cost + output_cost`
- `+`는 **덧셈 기호**입니다. 입력 비용과 출력 비용을 합산합니다.

⑦ `print(f"예상 비용:  ${total_cost:.6f} (약 {total_cost * 1400:.2f}원)")`
- `:.6f` = 소수점 6자리까지 출력 (f는 float, 소수의 약자)
- `:.2f` = 소수점 2자리까지 출력
- `* 1400` = 달러를 원화로 환산 (1달러 ≈ 1400원 기준)

### Code 4-2: 평가자 AI에게 채점 요청 [↑](#toc)

```python
evaluator_system = """당신은 프롬프트 응답 품질을 평가하는 전문 평가자입니다.
주어진 응답을 다음 4가지 기준으로 1~5점 채점하고, 반드시 JSON 형식으로만 출력하세요.

평가 기준:
- 정확성: 사실에 맞는가? 환각(거짓 정보)이 없는가?
- 일관성: 역할(튜터)에 맞는 톤을 처음부터 끝까지 유지하는가?
- 톤적절성: 학생 수준에 맞게 쉽게 설명하는가?
- 포맷적합성: 요청한 형식(표, 비유 포함)을 지켰는가?

출력 형식:
{
  "정확성": <1~5 정수>,
  "일관성": <1~5 정수>,
  "톤적절성": <1~5 정수>,
  "포맷적합성": <1~5 정수>,
  "총평": "<한 문장 평가>"
}"""

eval_response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[
        {"role": "system", "content": evaluator_system},
        {"role": "user", "content": f"""다음 응답을 평가해주세요.

[역할]: 딥러닝 전문 튜터
[요청]: CNN 핵심 구성 요소 3가지, 표 형식, 역할+비유
[응답]:
{answer}"""}
    ],
    temperature=0,
    response_format={"type": "json_object"},
)

scores = json.loads(eval_response.choices[0].message.content)

print("\n=== 평가 결과 ===")
for key, value in scores.items():
    print(f"  {key}: {value}")

score_keys = ["정확성", "일관성", "톤적절성", "포맷적합성"]
avg = sum(scores[k] for k in score_keys) / len(score_keys)
print(f"\n  평균 점수: {avg:.1f}/5.0")
```

**줄별 해설**

① `evaluator_system = """..."""`
- 평가자 AI에게 줄 역할 지시문을 삼중 따옴표 문자열로 저장합니다. 이 변수는 이후 Code 4-3에서도 재사용됩니다.

② 지시문 안의 JSON 출력 형식 예시
- `{...}` 안에 원하는 출력 구조를 미리 보여줌으로써 AI가 정확히 같은 형식으로 답하도록 유도합니다.

③ `temperature=0`
- 앞에서 배운 `temperature`를 0으로 설정합니다. 평가는 창의성이 아닌 일관성이 중요하므로, 매번 같은 결과가 나오도록 0을 씁니다.

④ `response_format={"type": "json_object"}` — 앞에서 배운 것과 동일하게 JSON 형식을 강제합니다.

⑤ `scores = json.loads(eval_response.choices[0].message.content)`
- 앞에서 배운 `json.loads()`로 JSON 텍스트를 딕셔너리로 변환하여 `scores` 상자에 저장합니다.

⑥ `for key, value in scores.items():`
- 앞에서 배운 딕셔너리 `.items()` 순회입니다. 매 반복마다 `key`에는 항목 이름(예: `"정확성"`), `value`에는 점수(예: `4`)가 담깁니다.

⑦ `score_keys = ["정확성", "일관성", "톤적절성", "포맷적합성"]`
- 평균을 계산할 네 항목의 이름을 리스트로 저장합니다.

⑧ `avg = sum(scores[k] for k in score_keys) / len(score_keys)`
- `sum(scores[k] for k in score_keys)` = 네 항목의 점수를 모두 더합니다. `for ... in ...` 형태를 사용해 리스트 안을 순회하며 합산하는 방식입니다. 이를 **제너레이터 표현식**이라고 합니다.
- `/ len(score_keys)` = 앞에서 배운 `len()`으로 항목 개수(4)를 구하고 나누어 평균을 냅니다.
- `/`는 **나눗셈 기호**입니다.

### Code 4-3: 프롬프트 3단계 개선 [↑](#toc)

```python
prompts = {
    "v1 (모호함)": "파이썬 에러 해결해줘",

    "v2 (구조화)": "파이썬에서 IndexError가 발생했어. 원인과 해결법을 알려줘.",

    "v3 (CRAFT)": """당신은 3년차 백엔드 개발자이고, 신입 교육을 담당하고 있습니다.

아래 파이썬 코드에서 IndexError가 발생합니다. 원인을 분석하고 수정된 코드를 보여주세요.

```python
fruits = ["사과", "바나나", "딸기"]
print(fruits[3])
```

조건:
- 에러 원인을 비유를 들어 쉽게 설명
- 수정된 코드 + 왜 그렇게 수정하는지 설명
- 비슷한 실수를 방지하는 팁 1개"""
}

for version, prompt in prompts.items():
    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[{"role": "user", "content": prompt}]
    )
    answer = response.choices[0].message.content

    eval_resp = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[
            {"role": "system", "content": evaluator_system},
            {"role": "user", "content": f"[요청]: IndexError 원인 분석 및 수정\n[응답]: {answer}"}
        ],
        temperature=0,
        response_format={"type": "json_object"},
    )
    scores = json.loads(eval_resp.choices[0].message.content)

    avg = sum(scores[k] for k in score_keys) / len(score_keys)
    print(f"\n{'='*50}")
    print(f"[{version}] 평균: {avg:.1f}/5.0")
    print(f"총평: {scores.get('총평', '')}")
    print(f"응답 미리보기: {answer[:100]}...")
```

**줄별 해설**

① `prompts = {"v1 (모호함)": "...", "v2 (구조화)": "...", "v3 (CRAFT)": """..."""}`
- 세 가지 버전의 프롬프트를 하나의 딕셔너리에 저장합니다. 버전 이름이 키(이름표), 실제 프롬프트가 값(내용)입니다.

② `v3` 안의 코드 블록 (백틱 세 개로 감싸진 부분)
- 프롬프트 안에 파이썬 코드 예시를 포함시키는 방법입니다. AI가 이 코드를 읽고 분석합니다.

③ `for version, prompt in prompts.items():`
- 앞에서 배운 딕셔너리 순회입니다. 세 버전을 순서대로 처리합니다.

④ 첫 번째 API 호출: 응답 생성 — 앞에서 배운 것과 동일합니다.

⑤ 두 번째 API 호출: 자동 평가
- `evaluator_system`은 Code 4-2에서 정의한 평가자 지시문을 그대로 재사용합니다.
- `f"[요청]: ...\n[응답]: {answer}"` = f-문자열로 평가 요청 메시지를 조립합니다.

⑥ `scores.get('총평', '')`
- `.get(키, 기본값)` = 딕셔너리에서 키를 찾되, 키가 없으면 기본값(여기서는 빈 문자열 `''`)을 반환합니다. `scores['총평']`과 비슷하지만, 키가 없을 때 오류 대신 기본값을 줘서 더 안전합니다.

⑦ `answer[:100]`
- **슬라이싱**입니다. 문자열에서 0번째부터 99번째까지 앞 100글자만 잘라냅니다. 긴 응답을 미리보기로 줄이는 용도입니다.

<a id="code5-1"></a>
### Code 5-1: app_chatbot.py — 역할 전환 AI 챗봇 [↑](#toc)

```python
# app_chatbot.py
# 실행: streamlit run app_chatbot.py
import streamlit as st
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

ROLES = {
    "파이썬 튜터": """...""",
    "딥러닝 멘토": """...""",
    "코드 리뷰어": """...""",
}

st.set_page_config(page_title="역할 전환 AI 챗봇", page_icon="💬", layout="centered")
st.title("💬 프로그래밍 학습 AI 챗봇")
st.caption("역할을 선택하면 해당 전문가로 대화합니다.")

with st.sidebar:
    st.header("설정")
    선택_역할 = st.radio("대화 상대 선택", list(ROLES.keys()))

    if "현재_역할" not in st.session_state:
        st.session_state["현재_역할"] = 선택_역할
    if st.session_state["현재_역할"] != 선택_역할:
        st.session_state["현재_역할"] = 선택_역할
        st.session_state["messages"] = []
        st.rerun()

    st.divider()
    st.markdown(f"**현재 역할**: {선택_역할}")
    st.markdown(f"**대화 수**: {len(st.session_state.get('messages', []))} 턴")

    if st.button("대화 초기화", use_container_width=True):
        st.session_state["messages"] = []
        st.rerun()

if "messages" not in st.session_state:
    st.session_state["messages"] = []

for msg in st.session_state["messages"]:
    with st.chat_message(msg["role"]):
        st.write(msg["content"])

if user_input := st.chat_input(f"{선택_역할}에게 말을 걸어보세요..."):
    with st.chat_message("user"):
        st.write(user_input)
    st.session_state["messages"].append({"role": "user", "content": user_input})

    api_messages = [
        {"role": "system", "content": ROLES[선택_역할]},
    ] + st.session_state["messages"]

    with st.chat_message("assistant"):
        with st.spinner("생각 중..."):
            response = client.chat.completions.create(
                model="gpt-4.1-nano",
                messages=api_messages,
                temperature=0.7,
                stream=True,
            )
            전체_답변 = st.write_stream(response)

    st.session_state["messages"].append({"role": "assistant", "content": 전체_답변})
```

**줄별 해설**

① `import streamlit as st`
- `streamlit`은 파이썬 코드만으로 웹 페이지를 만들 수 있는 도구입니다.
- `as st` = 앞으로 `streamlit`을 `st`라는 짧은 별명으로 부르겠다는 뜻입니다. 긴 이름을 매번 쓰지 않아도 되게 합니다.

② `from dotenv import load_dotenv`
- `dotenv`는 `.env` 파일에서 환경 변수를 읽어오는 도구입니다.

③ `load_dotenv()`
- `.env` 파일을 읽어서 그 안의 값(예: `OPENAI_API_KEY`)을 환경 변수로 등록합니다. 이후 `OpenAI()`가 자동으로 이 키를 가져갑니다.

④ `client = OpenAI()` — 앞에서 배운 것과 동일합니다. 단, Colab이 아닌 로컬 환경이므로 `load_dotenv()`로 키를 미리 불러왔습니다.

⑤ `ROLES = {...}`
- 앞에서 배운 딕셔너리로 세 가지 역할의 이름과 지시문을 저장합니다. 변수 이름이 모두 대문자인 것은 "이 값은 바뀌지 않는 상수"임을 관행적으로 알리는 방식입니다.

⑥ `st.set_page_config(...)`
- 웹 페이지의 탭 제목, 아이콘, 레이아웃을 설정합니다. 브라우저 탭에 표시되는 이름을 정하는 것입니다.

⑦ `st.title(...)` / `st.caption(...)`
- `st.title()` = 페이지 최상단의 큰 제목을 출력합니다.
- `st.caption()` = 작은 글씨의 설명 텍스트를 출력합니다.

⑧ `with st.sidebar:`
- **with 문**은 "이 블록 안의 내용은 특정 맥락에서 실행해라"는 문법입니다. 여기서는 들여쓰기된 내용 전체가 사이드바(화면 왼쪽 패널) 안에 배치됩니다.

⑨ `선택_역할 = st.radio("대화 상대 선택", list(ROLES.keys()))`
- `st.radio()` = 라디오 버튼(하나만 선택 가능한 버튼 목록)을 화면에 그립니다. 사용자가 선택한 값이 `선택_역할` 상자에 저장됩니다.
- `list(ROLES.keys())` = 딕셔너리의 모든 키(역할 이름들)를 리스트로 변환합니다.

⑩ `if "현재_역할" not in st.session_state:`
- `if`문은 **조건문**입니다. "만약 ~이라면 이 블록을 실행해라"는 뜻입니다. 계산기에서 특정 버튼이 눌렸을 때만 작동하는 기능과 같습니다.
- `st.session_state` = Streamlit이 페이지 새로고침 사이에도 값을 유지하는 저장소입니다. 서랍장처럼 키-값 쌍으로 저장합니다.
- `not in` = "~에 없다면"이라는 조건입니다.

⑪ `if st.session_state["현재_역할"] != 선택_역할:`
- `!=` = "같지 않다면"이라는 비교 연산자입니다. 역할이 바뀌었을 때만 대화를 초기화합니다.

⑫ `st.rerun()`
- 페이지를 처음부터 다시 실행합니다. 역할이 바뀌었을 때 화면을 갱신하기 위해 씁니다.

⑬ `st.divider()` — 수평 구분선을 그립니다.

⑭ `st.markdown(f"**현재 역할**: {선택_역할}")`
- `**텍스트**`는 마크다운 굵은 글씨 문법입니다. Streamlit이 이를 해석해 굵게 표시합니다.

⑮ `st.session_state.get('messages', [])`
- 앞에서 배운 `.get(키, 기본값)` 패턴입니다. 아직 `messages`가 없을 수도 있으므로, 없으면 빈 리스트 `[]`를 반환합니다.

⑯ `if st.button("대화 초기화", use_container_width=True):`
- `st.button()` = 클릭 가능한 버튼을 화면에 그립니다. 버튼이 클릭되면 `True`를 반환하므로 `if`문이 실행됩니다.
- `use_container_width=True` = 버튼 너비를 사이드바 너비에 맞춥니다.

⑰ `if "messages" not in st.session_state:` (두 번째)
- 페이지 처음 실행 시 `messages` 저장소가 없으므로 빈 리스트로 초기화합니다.

⑱ `for msg in st.session_state["messages"]:` / `with st.chat_message(msg["role"]):`
- 저장된 대화 기록을 순서대로 꺼내 화면에 표시합니다.
- `st.chat_message(role)` = 역할에 따라 사용자 또는 AI 스타일의 대화 말풍선을 만듭니다.

⑲ `if user_input := st.chat_input(...):`
- `st.chat_input()` = 화면 하단에 채팅 입력창을 만듭니다. 사용자가 메시지를 입력하고 Enter를 누르면 값이 반환됩니다.
- `:=` = **바다코끼리 연산자(walrus operator)**입니다. `st.chat_input()`의 반환값을 `user_input`에 저장하면서 동시에 `if` 조건도 확인합니다. 입력값이 있을 때만 if 블록이 실행됩니다.

⑳ `api_messages = [{"role": "system", "content": ROLES[선택_역할]}] + st.session_state["messages"]`
- 리스트끼리 `+`로 합칩니다. system 메시지(역할 지시)를 맨 앞에, 이후 전체 대화 기록을 뒤에 붙입니다.

㉑ `stream=True`
- AI의 응답을 한 번에 받는 대신 실시간으로 글자씩 스트리밍하여 받습니다. 타이핑하는 것처럼 응답이 흘러나오는 효과를 냅니다.

㉒ `전체_답변 = st.write_stream(response)`
- 스트리밍된 응답을 화면에 실시간으로 표시하면서, 완성된 전체 텍스트를 `전체_답변` 상자에 저장합니다.

### Code 5-2: app_resume.py — 자기소개서 첨삭 서비스 [↑](#toc)

```python
# app_resume.py
# 실행: streamlit run app_resume.py
import streamlit as st
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

EXAMPLES = {
    "마케팅": [
        {"role": "user", "content": "자소서: '마케팅에 관심이 있고 열심히 하겠습니다.'\n직무: 마케팅"},
        {"role": "assistant", "content": """**강점**
...(생략)...

**수정된 자소서**
...(생략)..."""},
    ],
}

def review_resume(job: str, content: str) -> str:
    system = """당신은 IT 기업 인사 담당자이며 10년 경력의 자기소개서 전문 멘토입니다.
항상 다음 형식으로 출력하세요:
**강점** (3개) → **개선점** (3개) → **수정된 자소서** (완성본)"""

    few_shot = EXAMPLES.get(job, EXAMPLES["마케팅"])
    messages = [
        {"role": "system", "content": system},
        *few_shot,
        {"role": "user", "content": f"자소서: '{content}'\n직무: {job}"},
    ]

    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=messages,
        temperature=0.5,
    )
    return response.choices[0].message.content

st.set_page_config(page_title="자소서 첨삭", page_icon="📝", layout="centered")
st.title("📝 자기소개서 첨삭 서비스")
st.caption("직무를 선택하고 자소서를 입력하면 맞춤 피드백을 드립니다.")

job_role = st.selectbox("지원 직무", ["마케팅", "기획/PM", "디자인", "개발", "영업"])
resume_input = st.text_area("자기소개서 입력", height=200, placeholder="첨삭받을 내용을 입력하세요.")

if resume_input:
    st.caption(f"입력 글자 수: {len(resume_input)}자")

if st.button("첨삭하기", type="primary", use_container_width=True):
    if not resume_input or len(resume_input.strip()) < 20:
        st.warning("최소 20자 이상 입력해주세요.")
    else:
        with st.spinner(f"{job_role} 직무 기준으로 첨삭 중..."):
            result = review_resume(job_role, resume_input)

        st.divider()
        st.subheader(f"첨삭 결과 — {job_role}")
        st.markdown(result)

        st.download_button(
            label="결과 저장 (.txt)",
            data=result,
            file_name=f"자소서_첨삭_{job_role}.txt",
        )
```

**줄별 해설**

① 파일 상단의 `import`, `load_dotenv()`, `client = OpenAI()` — 앞에서 배운 것과 동일합니다.

② `EXAMPLES = {"마케팅": [...]}`
- Few-shot 예시를 딕셔너리에 저장합니다. 직무 이름이 키, 예시 대화 리스트가 값입니다.
- `{"role": "assistant", "content": "..."}` = AI(어시스턴트)가 한 모범 답변 예시입니다. Few-shot에서 AI의 이상적인 응답을 미리 보여주는 역할을 합니다.

③ `def review_resume(job: str, content: str) -> str:`
- 앞에서 배운 `def`로 함수를 정의합니다.
- `job: str` = `job` 재료는 문자열(str) 타입이어야 한다는 **타입 힌트**입니다. 파이썬이 강제하지는 않지만, 코드를 읽는 사람에게 어떤 값을 넣어야 하는지 알려줍니다.
- `-> str` = 이 함수가 문자열을 반환한다는 표시입니다.

④ `few_shot = EXAMPLES.get(job, EXAMPLES["마케팅"])`
- 앞에서 배운 `.get(키, 기본값)` 패턴입니다. `job`에 해당하는 예시가 없으면 마케팅 예시를 기본값으로 씁니다.

⑤ `messages = [{"role": "system", ...}, *few_shot, {"role": "user", ...}]`
- `*few_shot` = **언패킹 연산자**입니다. 리스트 앞에 `*`를 붙이면 그 리스트의 항목들이 펼쳐져 삽입됩니다. 봉투를 열어 편지들을 직접 꺼내 나란히 놓는 것과 같습니다.
- 결과적으로 system 메시지, Few-shot 예시들, 사용자 메시지가 하나의 리스트에 순서대로 담깁니다.

⑥ `temperature=0.5`
- 앞에서 배운 temperature를 0.5로 설정합니다. 완전히 고정(0)도 아니고 너무 창의적(1.5)도 아닌 중간 값으로, 일관성 있으면서도 자연스러운 첨삭 결과를 냅니다.

⑦ `job_role = st.selectbox("지원 직무", ["마케팅", "기획/PM", "디자인", "개발", "영업"])`
- `st.selectbox()` = 드롭다운 선택 메뉴를 화면에 그립니다. 사용자가 선택한 값이 `job_role`에 저장됩니다.

⑧ `resume_input = st.text_area("자기소개서 입력", height=200, placeholder="...")`
- `st.text_area()` = 여러 줄을 입력할 수 있는 텍스트 박스를 화면에 그립니다.
- `height=200` = 텍스트 박스의 세로 높이를 200 픽셀로 설정합니다.
- `placeholder="..."` = 아무것도 입력하지 않았을 때 연한 글씨로 보이는 안내 문구입니다.

⑨ `if resume_input:` / `st.caption(f"입력 글자 수: {len(resume_input)}자")`
- 앞에서 배운 `if`문으로 `resume_input`에 내용이 있을 때만 글자 수를 표시합니다.
- `len(resume_input)` = 앞에서 배운 `len()`으로 입력된 문자열의 길이(글자 수)를 가져옵니다.

⑩ `if st.button("첨삭하기", type="primary", use_container_width=True):`
- 앞에서 배운 `st.button()` 패턴입니다.
- `type="primary"` = 버튼을 강조 색상(파란색 등)으로 표시합니다.

⑪ `if not resume_input or len(resume_input.strip()) < 20:`
- `not resume_input` = 입력값이 비어있다면
- `or` = "또는"이라는 논리 연산자입니다.
- `len(resume_input.strip()) < 20` = 공백 제거 후 글자 수가 20 미만이라면
- 두 조건 중 하나라도 참이면 경고를 표시합니다.
- `<` = "보다 작다"는 비교 연산자입니다.

⑫ `st.warning("최소 20자 이상 입력해주세요.")`
- 노란색 경고 메시지 박스를 화면에 표시합니다.

⑬ `with st.spinner(f"{job_role} 직무 기준으로 첨삭 중..."):`
- 앞에서 배운 `with` 문입니다. 이 블록이 실행되는 동안 화면에 로딩 스피너(회전하는 원)와 메시지를 표시합니다.

⑭ `result = review_resume(job_role, resume_input)`
- 위에서 정의한 `review_resume` 함수를 호출합니다. 직무와 자소서 내용을 재료로 넣고, 첨삭 결과를 받아 `result`에 저장합니다.

⑮ `st.subheader(f"첨삭 결과 — {job_role}")`
- 중간 크기의 소제목을 화면에 표시합니다.

⑯ `st.markdown(result)`
- AI가 돌려준 마크다운 형식의 첨삭 결과를 렌더링(예쁘게 변환)하여 표시합니다. `**강점**`이 굵은 글씨로, `-`가 글머리 기호로 변환됩니다.

⑰ `st.download_button(label="결과 저장 (.txt)", data=result, file_name=f"자소서_첨삭_{job_role}.txt")`
- 클릭하면 파일을 다운로드하는 버튼을 만듭니다.
- `label=` = 버튼에 표시할 텍스트
- `data=` = 다운로드할 내용
- `file_name=` = 저장될 파일 이름. f-문자열로 직무 이름이 파일명에 포함됩니다.
