클라우드 컴퓨팅의 기본 개념과 AWS를 활용한 서버 구축 실습입니다.

## ⏱ 권장 시간 분배 (총 4~5시간 기준)

본 과정은 **AWS 가입이 완료된 상태**를 전제로 약 **순수 실습 3.5시간 + 설명·여유 1~1.5시간**으로 운영합니다. **02장(가입 따라하기)는 수업 전날까지 사전 과제**로 마치는 것을 권장합니다.

| 챕터 | 분량 | 비고 |
|:---:|---:|:---|
| 02. AWS 가입 따라하기 | (사전 과제) | 수업 전날까지 완료 |
| 01. 클라우드와 AWS 소개 | 30분 | 강의 위주 (개념 이해) |
| 03. AWS 콘솔과 비용 관리 | 25분 | Zero-Spend Budget 필수 |
| 04. EC2 생성과 접속 | 40분 | 보안 그룹·Instance Connect 트러블슈팅 여유 |
| 05. 리눅스 기초 명령어 | 30분 | 명령어 시연 + 실습 |
| 06. 웹서버 Nginx 설치 | 20분 | 짧음 — 설치 + http 확인 |
| 07. GitHub 정적 사이트 배포 | 30분 | git clone·복사·재로드 |
| 08. 리소스 정리와 다음 단계 | **30분** | **Terminate에 최소 20분 사수** |
| 휴식·예비 | 30~60분 | 점심·중간 휴식 포함 |
| **합계** | **약 4~5시간** | (사전 과제 02장 제외) |

> 💡 **사전 과제 미완료자 대응**: 수업 시작 시 02장 사전 과제(가입) 미완료자가 있으면 별도 좌석에서 02장 가이드를 따라가도록 안내하고, 강사는 03장부터 정상 진행. 가입은 15~25분 소요되므로 본 수업 시간을 잡아먹지 않게 분리.

> 💡 **압축 운영 (3시간)**: 03장 비용 알림 + 04장 EC2 + 06장 Nginx + 08장 Terminate만 진행. 05·07장 생략.

> 💡 **확장 운영 (1일, 7~8시간)**: 위 표대로 진행하되 점심 1시간 + Stretch Goal(HTTPS 안내, S3 업로드 등) 1시간 추가.

## 🎤 강사 사전 준비 체크리스트

> 이 섹션은 **강사 전용**입니다. 수업을 원활히 진행하기 위해 **수업 1주 전~당일 아침**까지 아래 항목을 모두 점검하세요.

### 📅 수업 1주 전

- [ ] **GitHub 강사 레포 동작 확인** — `https://github.com/seonjo0217/todoApp.git` (07장 §6 참조)
  - `index.html`, `src/app.js`, `src/todo.js`, `README.md` 푸시 상태 확인
  - 테스트 EC2에서 `git clone` → `sudo cp -r ~/todoApp/* /var/www/html/` 후 브라우저에서 항목 추가/체크/필터/드래그 모두 정상 동작 확인
- [ ] **AWS 프리티어/요금 정책 최신화 확인**
  - 신규 Free Plan 정책($200 크레딧, 6개월) 변동 여부
  - t3.micro 서울 리전 시간당 단가 ($0.0104 기준)
  - 퍼블릭 IPv4 시간당 $0.005 정책(2024.2~) 유지 여부
- [ ] **수강생 사전 과제 안내 발송**
  - 이메일·신용/체크카드·휴대폰 준비
  - **02장 가입 따라하기** 링크 공유, **계정 가입은 수업 전날까지 완료** 권장

### 📅 수업 전날

- [ ] 강사 본인 계정으로 **03~08장 전체 흐름 1회 리허설** (인스턴스 생성→접속→Nginx→배포→Terminate)
- [ ] 강의장 네트워크에서 **EC2 Instance Connect 접속 가능** 확인 (회사·학원 방화벽이 막는 경우 있음)
- [ ] 강사 GitHub 레포 `git clone`이 새 EC2에서 정상 동작하는지 확인
- [ ] **수업 후 Terminate 안 한 리허설용 인스턴스** 직접 종료

### 📅 수업 당일

- [ ] 시작 전: 수강생 **계정 가입 완료 여부** 거수 확인. 미완료자는 02장 가입 따라하기 페이지로 안내하며 따라잡기
- [ ] 03장 종료 시점: 전원 **Zero-Spend Budget 생성 완료** 시각 확인 (이게 안 되면 이후 진행 위험)
- [ ] 04장 종료 시점: 전원 **인스턴스 running + 2/2 status check** 확인
- [ ] **08장 Terminate에 최소 20분 확보** — 04~07장이 늦어지면 Nginx 파일 수정 실습은 줄여서라도 Terminate 시간 사수
- [ ] 수업 종료 직전: 수강생 **한 명 한 명 `terminated` 상태 직접 확인** (강사 옆에서 같이 보기)

### 📅 수업 다음 날

- [ ] 수강생 채널/단톡방에 "**결제 대시보드에서 청구 금액 확인**" 리마인더 발송
- [ ] Zero-Spend Budget 알림 이메일을 받은 수강생이 있으면 1:1 확인
