## 1️⃣ 나만의 웹 페이지 배포 [↑](#toc)

> "인터넷에 내 페이지를 올리는 순간!"

Nginx는 `/var/www/html/` 폴더에 있는 파일을 브라우저에 제공합니다.
이 폴더의 `index.html`을 교체하면 브라우저에서 바로 확인할 수 있습니다.

---

### HTML 파일 작성

```bash
# Nginx 기본 페이지를 우리 페이지로 교체
sudo nano /var/www/html/index.html
```

기존 내용을 모두 지우고(Ctrl+K 반복 또는 전체 선택 후 삭제) 아래 HTML을 입력합니다.

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>나의 첫 클라우드 웹서버</title>
    <style>
        body {
            font-family: 'Pretendard', sans-serif;
            max-width: 800px;
            margin: 0 auto;
            padding: 40px;
            background: #f0f9ff;
        }
        h1 { color: #1e40af; }
        .card {
            background: white;
            border-radius: 12px;
            padding: 24px;
            margin: 16px 0;
            box-shadow: 0 2px 8px rgba(0,0,0,0.1);
        }
        .info { color: #6b7280; font-size: 14px; }
    </style>
</head>
<body>
    <h1>🎉 나의 첫 클라우드 웹서버!</h1>
    <div class="card">
        <h2>안녕하세요!</h2>
        <p>이 페이지는 AWS EC2 서버에서 Nginx 웹서버를 통해 서비스되고 있습니다.</p>
        <p class="info">서버: Ubuntu 22.04 | 웹서버: Nginx | 위치: AWS 서울 리전</p>
    </div>
    <div class="card">
        <h2>오늘 배운 것</h2>
        <ul>
            <li>☁️ 클라우드와 AWS의 개념</li>
            <li>🖥️ EC2 인스턴스 생성과 접속</li>
            <li>🐧 리눅스 기초 명령어</li>
            <li>🌐 Nginx 웹서버 설치</li>
            <li>🗄️ MySQL 데이터베이스 설치</li>
            <li>🚀 웹 페이지 배포</li>
        </ul>
    </div>
    <div class="card">
        <h2>방명록</h2>
        <p>여기에 메시지를 남겨주세요! (심화 과제: PHP로 MySQL 연동)</p>
    </div>
</body>
</html>
```

nano에서 저장하는 방법:

```
Ctrl + O   →   파일 이름 확인 (Enter)   →   Ctrl + X
```

---

### 브라우저에서 확인

```bash
# Nginx 설정을 다시 불러옵니다 (파일만 교체했으면 불필요하지만 습관적으로 실행)
sudo systemctl reload nginx
```

브라우저 주소창에서 `http://퍼블릭IP`를 **새로고침(F5)** 하면 방금 작성한 HTML 페이지가 나타납니다.

> "브라우저를 새로고침하면 방금 작성한 HTML 페이지가 전 세계 어디서나 접근 가능한 상태가 됩니다!"

---

### 수정 내용 실시간 반영

HTML 내용을 수정하고 저장하면 브라우저 새로고침만으로 변경 사항이 즉시 반영됩니다.
서버를 재시작할 필요가 없습니다.

```bash
sudo nano /var/www/html/index.html
# 수정 후 Ctrl+O → Enter → Ctrl+X
# 브라우저에서 F5 (새로고침)
```

## 2️⃣ (보너스) PHP로 MySQL 연동 [↑](#toc)

> 시간 여유가 있을 때만 진행합니다. **시간이 부족하면 이 섹션은 건너뛰어도 됩니다.**

PHP를 설치하면 서버 측에서 MySQL 데이터를 읽어 HTML로 동적으로 출력할 수 있습니다.

---

### PHP 설치

```bash
sudo apt install php-fpm php-mysql -y
```

- `php-fpm`: PHP FastCGI Process Manager — Nginx와 PHP를 연결하는 브릿지
- `php-mysql`: PHP에서 MySQL에 접속할 수 있게 해주는 확장

---

### Nginx PHP 설정

`/etc/nginx/sites-available/default` 파일에서 PHP 처리 부분의 주석을 해제합니다.

```bash
sudo nano /etc/nginx/sites-available/default
```

파일에서 아래 부분을 찾아 주석(`#`)을 제거합니다.

```nginx
# Pass PHP scripts to FastCGI server
location ~ \.php$ {
    include snippets/fastcgi-php.conf;
    fastcgi_pass unix:/var/run/php/php8.1-fpm.sock;
}
```

저장 후 Nginx를 재시작합니다.

```bash
sudo nginx -t              # 설정 문법 검사
sudo systemctl reload nginx
```

---

### 방명록 PHP 페이지 예시

```bash
sudo nano /var/www/html/guestbook.php
```

```php
<?php
// MySQL 접속 정보
$host = 'localhost';
$user = 'root';
$pass = 'Cloud2026!';
$db   = 'mywebsite';

// 데이터베이스 연결
$conn = new mysqli($host, $user, $pass, $db);

if ($conn->connect_error) {
    die("연결 실패: " . $conn->connect_error);
}

// 방명록 데이터 조회
$result = $conn->query("SELECT name, message, created_at FROM guestbook ORDER BY id DESC");
?>
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>방명록</title>
    <style>
        body { font-family: sans-serif; max-width: 700px; margin: 0 auto; padding: 40px; background: #f9fafb; }
        .entry { background: white; border-radius: 8px; padding: 16px; margin: 12px 0; box-shadow: 0 1px 4px rgba(0,0,0,0.08); }
        .name { font-weight: bold; color: #1e40af; }
        .date { color: #9ca3af; font-size: 12px; }
    </style>
</head>
<body>
    <h1>방명록</h1>
    <?php while ($row = $result->fetch_assoc()): ?>
    <div class="entry">
        <p class="name"><?= htmlspecialchars($row['name']) ?></p>
        <p><?= htmlspecialchars($row['message']) ?></p>
        <p class="date"><?= $row['created_at'] ?></p>
    </div>
    <?php endwhile; ?>
</body>
</html>
<?php $conn->close(); ?>
```

브라우저에서 `http://퍼블릭IP/guestbook.php`로 접속하면 MySQL에 저장된 방명록 데이터가 표시됩니다.

## 3️⃣ 수업 완료 — 성취한 것 정리 [↑](#toc)

수업을 통해 구축한 전체 아키텍처를 확인합니다.

```
[내 컴퓨터] → 인터넷 → [AWS 서울 리전 (ap-northeast-2)]
                              │
                         [EC2 인스턴스 (t2.micro)]
                         ├── Ubuntu 22.04 LTS (운영체제)
                         ├── Nginx (웹서버 · 포트 80)
                         ├── MySQL (데이터베이스 · 포트 3306)
                         └── /var/www/html/index.html (내 웹페이지)
```

**여러분은 지금 전 세계 누구나 접속할 수 있는 웹 서버를 운영하고 있습니다!**

| 구성 요소 | 역할 |
|----------|------|
| EC2 인스턴스 | 물리 서버를 대체하는 가상 컴퓨터 |
| Ubuntu 22.04 | 서버를 운영하는 리눅스 운영체제 |
| Nginx | 브라우저의 요청을 받아 HTML 파일을 응답 |
| MySQL | 사용자 데이터, 방명록 등을 저장하는 데이터베이스 |
| 보안 그룹 | 허용된 포트(22, 80)만 통신하게 하는 방화벽 |

## 4️⃣ ⚠️ 리소스 정리 (매우 중요!) [↑](#toc)

> ⚠️ **인스턴스를 종료하지 않으면 프리 티어 시간이 소진되어 과금될 수 있습니다!**
>
> AWS 프리 티어는 월 750시간의 t2.micro 사용을 무료로 제공합니다.
> 계속 실행 중이면 시간이 소진되고, 초과 시 요금이 청구됩니다.

---

### EC2 인스턴스 종료 절차

```
Step 1: AWS 콘솔 → EC2 대시보드로 이동
Step 2: 왼쪽 메뉴 "인스턴스(Instances)" 클릭
Step 3: 종료할 인스턴스 체크박스 선택
Step 4: 상단 "인스턴스 상태(Instance state)" 버튼 클릭
Step 5: "인스턴스 종료(Terminate instance)" 선택
Step 6: 확인 팝업에서 "종료(Terminate)" 클릭
Step 7: 인스턴스 상태가 "종료됨(terminated)"으로 변경 확인
```

> 상태가 `shutting-down` → `terminated` 순서로 변경됩니다.
> `terminated` 상태의 인스턴스는 몇 분 후 목록에서 자동으로 사라집니다.

---

### 중지(Stop) vs 종료(Terminate) — 반드시 구분!

| 구분 | 중지 (Stop) | 종료 (Terminate) |
|------|------------|-----------------|
| 인스턴스 상태 | 일시 정지 | 영구 삭제 |
| EBS 디스크 | 유지 (비용 계속 발생!) | 삭제 (비용 중단) |
| 퍼블릭 IP | 해제 | 해제 |
| 재시작 여부 | 가능 | 불가능 |
| **수업 후 권장** | ❌ 비추천 | ✅ 권장 |

> ⚠️ **Stop(중지)은 인스턴스는 꺼지지만 EBS 디스크 요금은 계속 발생합니다.**
> 수업 후에는 반드시 **Terminate(종료)** 를 선택하세요.

---

### 추가 정리 체크리스트

수업 후 아래 항목을 모두 확인하세요.

- [ ] EC2 인스턴스 상태가 `terminated`인지 확인
- [ ] **모든 리전 확인** — EC2 > Global View에서 다른 리전에 실행 중인 인스턴스가 없는지 확인
- [ ] **Elastic IP(탄력적 IP)** 할당 받은 것이 있다면 해제 (연결되지 않은 Elastic IP는 요금 발생)
- [ ] **AWS Billing 대시보드** 에서 예상 청구 금액이 $0에 가까운지 확인
  - 상단 계정 이름 클릭 → "결제 대시보드(Billing Dashboard)"

> **프리 티어 알림 설정 (01장에서 설정했다면 이미 완료)**
> 청구금액이 $1을 초과하면 이메일 알림이 오도록 설정되어 있어야 합니다.

## 5️⃣ 다음 단계 안내 [↑](#toc)

오늘 배운 내용을 기반으로 더 깊이 공부하고 싶다면 아래 주제를 추천합니다.

| 주제 | 내용 | 왜 배워야 할까? |
|------|------|----------------|
| **Docker** | 컨테이너로 앱 패키징 | 개발 환경을 그대로 서버에 배포, 환경 차이 문제 해결 |
| **RDS** | AWS 관리형 데이터베이스 | 자동 백업, 자동 확장, 장애 복구 — MySQL을 직접 관리할 필요 없음 |
| **S3** | 파일/이미지 저장소 | 이미지·CSS·JS 파일 저장, 정적 웹사이트 호스팅 가능 |
| **Route 53** | DNS(도메인 네임 서비스) | `example.com` 같은 도메인을 EC2/로드밸런서에 연결 |
| **CI/CD** | 코드 푸시 시 자동 배포 | GitHub에 코드를 올리면 자동으로 서버에 반영 |
| **Load Balancer** | 부하 분산 | 트래픽이 많아지면 여러 서버에 분산, 고가용성 구현 |

### 실습 과제

**기본** — HTML 페이지의 내용을 자기소개로 변경하세요.

```bash
sudo nano /var/www/html/index.html
# h1 태그의 내용을 이름으로, p 태그를 자기소개 문장으로 수정
```

**중급** — Tailwind CSS CDN을 사용하여 페이지 디자인을 개선하세요.

```html
<!-- <head> 안에 추가 -->
<script src="https://cdn.tailwindcss.com"></script>
```

기존 `<style>` 태그를 제거하고 Tailwind 유틸리티 클래스를 사용하여 카드 레이아웃, 색상, 반응형 디자인을 적용합니다.

**심화** — PHP와 MySQL을 연동하여 방명록 기능을 구현하세요.

1. 05장에서 생성한 `guestbook` 테이블을 활용합니다.
2. 이름과 메시지를 입력하는 HTML 폼을 만듭니다.
3. PHP로 폼 데이터를 MySQL에 저장하고, 저장된 목록을 출력합니다.

```
기능 목록:
  - [ ] 이름 + 메시지 입력 폼
  - [ ] 폼 제출 시 MySQL에 INSERT
  - [ ] 저장된 방명록 목록 SELECT + 출력
  - [ ] XSS 방지: htmlspecialchars() 적용
```
