## 1️⃣ 제어 컴포넌트 = 리모컨으로 조종하는 TV — 비유로 시작하기 [↑](#toc)

두 가지 종류의 TV를 상상합니다.

**비제어 TV**: 채널 버튼이 TV 본체에만 있습니다. TV 스스로 채널을 기억하고 관리합니다. 외부에서 "현재 채널이 몇 번이야?"라고 물어보려면 TV 패널을 직접 눌러봐야 합니다.

**제어 TV**: 리모컨으로 모든 것을 조종합니다. 리모컨(React state)이 채널을 기억하고, TV는 리모컨이 시키는 대로만 표시합니다. 현재 채널이 몇 번인지는 리모컨을 보면 바로 알 수 있습니다.

React의 **제어 컴포넌트**는 제어 TV와 같습니다. 입력값이 DOM(TV 본체)이 아닌 React state(리모컨)에 저장됩니다.

```
비제어: DOM → (직접 읽기) → document.getElementById('input').value
제어:  state → (표시)    → input value={name}
       input → (변경)    → onChange → setName → state 업데이트
```

## 2️⃣ 제어 컴포넌트란? — React가 입력을 지배한다 [↑](#toc)

### 비제어 방식 (쓰지 마세요)

```jsx
// ❌ 비제어 방식 — React 방식이 아닙니다
function BadForm() {
  function handleSubmit() {
    const name = document.getElementById("name-input").value; // DOM 직접 접근
    console.log(name);
  }

  return (
    <div>
      <input id="name-input" type="text" />
      <button onClick={handleSubmit}>제출</button>
    </div>
  );
}
// 동작은 하지만 React의 장점을 전혀 활용하지 못합니다
```

### 제어 방식 (React 표준)

```jsx
// ✅ 제어 방식 — React 상태로 입력값을 관리합니다
import { useState } from "react";

function GoodForm() {
  const [name, setName] = useState("");

  function handleSubmit() {
    console.log("입력값:", name); // state에서 직접 가져옵니다
  }

  return (
    <div>
      {/* value와 onChange가 세트입니다 */}
      <input
        type="text"
        value={name}                          // state가 표시값을 제어
        onChange={(e) => setName(e.target.value)} // 입력 시 state 업데이트
      />
      <button onClick={handleSubmit}>제출</button>
    </div>
  );
}
```

**제어 컴포넌트의 데이터 흐름:**

```
사용자 입력 → onChange 이벤트 → setName(새값) → state 업데이트
                                                        ↓
                              React 리렌더 → value={name} → 입력창에 표시
```

이 순환 덕분에 입력값이 항상 React state와 동기화됩니다.

import { useState } from "react";

function NameInput() {

## 4️⃣ textarea와 select — HTML과 차이점 [↑](#toc)

### textarea

HTML에서 textarea는 `<textarea>내용</textarea>` 방식이지만, React에서는 `value` prop을 씁니다.

```jsx
import { useState } from "react";

function TextareaExample() {
  const [bio, setBio] = useState("");
  const maxLength = 200;

  return (
    <div className="p-6 space-y-2">
      <label className="block text-sm font-medium text-gray-700">
        자기소개
      </label>
      {/* ❌ HTML 방식: <textarea>초기값</textarea> */}
      {/* ✅ React 방식: value prop 사용 */}
      <textarea
        value={bio}
        onChange={(e) => setBio(e.target.value)}
        maxLength={maxLength}
        rows={4}
        placeholder="자신을 소개해주세요..."
        className="w-full border border-gray-300 rounded-lg px-3 py-2 resize-none focus:outline-none focus:ring-2 focus:ring-blue-500"
      />
      <p className={`text-right text-xs ${bio.length >= maxLength ? "text-red-500" : "text-gray-400"}`}>
        {bio.length} / {maxLength}자
      </p>
    </div>
  );
}
// maxLength에 도달하면 카운터가 빨간색으로 변합니다
```

### select

import { useState } from "react";

function SelectExample() {

## 5️⃣ 여러 입력 관리 — 객체 상태 [↑](#toc)

입력 필드가 많을 때 각각 useState를 만들면 코드가 길어집니다. **객체 상태**로 한꺼번에 관리하세요.

### 여러 개의 useState (비추천)

```jsx
// ❌ 각각 따로 관리 — 필드가 늘면 useState가 계속 늘어납니다
const [name, setName] = useState("");
const [email, setEmail] = useState("");
const [phone, setPhone] = useState("");
const [address, setAddress] = useState("");
```

### 객체 상태 (추천)

```jsx
import { useState } from "react";

function SignupForm() {
  // 하나의 객체로 모든 필드 관리
  const [form, setForm] = useState({
    name: "",
    email: "",
    phone: "",
  });

  // 범용 핸들러 — 어떤 필드든 처리
  function handleChange(e) {
    const { name, value } = e.target;
    // 스프레드로 불변성 유지
    setForm((prev) => ({
      ...prev,    // 나머지 필드는 그대로
      [name]: value, // 해당 필드만 업데이트 (동적 키)
    }));
  }

  return (
    <form className="p-6 space-y-4 bg-white rounded-xl shadow max-w-sm">
      <div>
        <label className="block text-sm font-medium text-gray-700 mb-1">이름</label>
        <input
          type="text"
          name="name"             // name 속성이 handleChange에서 키로 사용됩니다
          value={form.name}
          onChange={handleChange}
          className="w-full border rounded-lg px-3 py-2 focus:outline-none focus:ring-2 focus:ring-blue-400"
          placeholder="홍길동"
        />
      </div>

      <div>
        <label className="block text-sm font-medium text-gray-700 mb-1">이메일</label>
        <input
          type="email"
          name="email"
          value={form.email}
          onChange={handleChange}
          className="w-full border rounded-lg px-3 py-2 focus:outline-none focus:ring-2 focus:ring-blue-400"
          placeholder="hong@example.com"
        />
      </div>

      <div>
        <label className="block text-sm font-medium text-gray-700 mb-1">전화번호</label>
        <input
          type="tel"
          name="phone"
          value={form.phone}
          onChange={handleChange}
          className="w-full border rounded-lg px-3 py-2 focus:outline-none focus:ring-2 focus:ring-blue-400"
          placeholder="010-0000-0000"
        />
      </div>

      {/* 실시간 미리보기 */}
      <div className="bg-gray-50 rounded-lg p-3 text-xs font-mono text-gray-600">
        {JSON.stringify(form, null, 2)}
      </div>
    </form>
  );
}
// 각 필드 입력 시 미리보기 JSON이 실시간으로 업데이트됩니다
// 출력: { "name": "홍길동", "email": "hong@...", "phone": "010-..." }
```

> **핵심**: `[name]: value` 는 동적 키를 사용해 어떤 필드든 하나의 핸들러로 처리합니다. `name="email"` 인 input이 바뀌면 `form.email`이 업데이트됩니다.

## 6️⃣ 폼 제출 (onSubmit) — e.preventDefault()와 유효성 검사 [↑](#toc)

### 기본 폼 제출

```jsx
import { useState } from "react";

function LoginForm() {
  const [form, setForm] = useState({ email: "", password: "" });
  const [errors, setErrors] = useState({});
  const [submitted, setSubmitted] = useState(false);

  function handleChange(e) {
    const { name, value } = e.target;
    setForm((prev) => ({ ...prev, [name]: value }));
    // 입력 시 해당 필드 에러 제거
    if (errors[name]) {
      setErrors((prev) => ({ ...prev, [name]: "" }));
    }
  }

  function validate() {
    const newErrors = {};

    if (!form.email) {
      newErrors.email = "이메일을 입력해주세요";
    } else if (!/\S+@\S+\.\S+/.test(form.email)) {
      newErrors.email = "올바른 이메일 형식이 아닙니다";
    }

    if (!form.password) {
      newErrors.password = "비밀번호를 입력해주세요";
    } else if (form.password.length < 8) {
      newErrors.password = "비밀번호는 8자 이상이어야 합니다";
    }

    return newErrors;
  }

  function handleSubmit(e) {
    e.preventDefault(); // 페이지 새로고침 방지!

    const newErrors = validate();
    if (Object.keys(newErrors).length > 0) {
      setErrors(newErrors);
      return; // 에러가 있으면 제출 중단
    }

    console.log("제출 데이터:", form);
    setSubmitted(true);
    // 출력: "제출 데이터: { email: 'test@test.com', password: '12345678' }"
  }

  if (submitted) {
    return (
      <div className="p-8 text-center">
        <div className="text-5xl mb-3">✅</div>
        <h2 className="text-xl font-bold">로그인 성공!</h2>
        <p className="text-gray-500 text-sm">{form.email}</p>
      </div>
    );
  }

  return (
    <form
      onSubmit={handleSubmit}
      className="p-6 space-y-4 bg-white rounded-xl shadow max-w-sm"
    >
      <h2 className="text-xl font-bold">로그인</h2>

      <div>
        <label className="block text-sm font-medium text-gray-700 mb-1">이메일</label>
        <input
          type="email"
          name="email"
          value={form.email}
          onChange={handleChange}
          className={`w-full border rounded-lg px-3 py-2 focus:outline-none focus:ring-2 ${
            errors.email ? "border-red-400 focus:ring-red-400" : "border-gray-300 focus:ring-blue-400"
          }`}
          placeholder="email@example.com"
        />
        {errors.email && (
          <p className="text-red-500 text-xs mt-1">{errors.email}</p>
        )}
      </div>

      <div>
        <label className="block text-sm font-medium text-gray-700 mb-1">비밀번호</label>
        <input
          type="password"
          name="password"
          value={form.password}
          onChange={handleChange}
          className={`w-full border rounded-lg px-3 py-2 focus:outline-none focus:ring-2 ${
            errors.password ? "border-red-400 focus:ring-red-400" : "border-gray-300 focus:ring-blue-400"
          }`}
          placeholder="8자 이상"
        />
        {errors.password && (
          <p className="text-red-500 text-xs mt-1">{errors.password}</p>
        )}
      </div>

      <button
        type="submit"
        className="w-full bg-blue-600 text-white py-2 rounded-lg font-medium hover:bg-blue-700 transition-colors"
      >
        로그인
      </button>
    </form>
  );
}
// 빈 폼 제출 시: 빨간 에러 메시지들이 표시됩니다
// 올바른 값 제출 시: "로그인 성공!" 화면으로 전환
```

## 7️⃣ React 19: useActionState 맛보기 [↑](#toc)

React 19에서는 폼 처리가 더욱 간단해졌습니다. `useActionState`는 폼 액션과 상태를 하나로 묶어줍니다.

### 기존 방식 (React 18)

```jsx
// 기존 방식: 로딩, 에러, 결과 상태를 따로 관리
const [isLoading, setIsLoading] = useState(false);
const [error, setError] = useState(null);
const [result, setResult] = useState(null);

async function handleSubmit(e) {
  e.preventDefault();
  setIsLoading(true);
  try {
    const data = await submitForm(form);
    setResult(data);
  } catch (err) {
    setError(err.message);
  } finally {
    setIsLoading(false);
  }
}
```

### React 19 방식 (useActionState)

```jsx
// src/App.jsx
import { useActionState } from "react";

// 액션 함수 — 폼 처리 로직
async function submitContactForm(prevState, formData) {
  // FormData 객체에서 값 추출
  const name = formData.get("name");
  const email = formData.get("email");
  const message = formData.get("message");

  // 간단한 유효성 검사
  if (!name || !email || !message) {
    return { success: false, error: "모든 필드를 입력해주세요" };
  }

  // 실제로는 API 호출이 여기 들어갑니다
  // await fetch('/api/contact', { method: 'POST', body: formData });

  // 성공 응답 시뮬레이션
  return { success: true, name };
}

function ContactForm() {
  const [state, formAction, isPending] = useActionState(
    submitContactForm,
    null // 초기 상태
  );

  if (state?.success) {
    return (
      <div className="p-8 text-center bg-green-50 rounded-xl">
        <div className="text-4xl mb-2">🎉</div>
        <p className="font-bold text-green-800">
          {state.name}님의 메시지가 전송되었습니다!
        </p>
      </div>
    );
  }

  return (
    // action에 formAction을 직접 연결 — onSubmit 불필요
    <form action={formAction} className="space-y-4 max-w-sm p-6 bg-white rounded-xl shadow">
      <h2 className="text-xl font-bold">문의하기</h2>

      {state?.error && (
        <div className="bg-red-50 text-red-600 text-sm p-3 rounded-lg">
          {state.error}
        </div>
      )}

      <input
        name="name"
        type="text"
        placeholder="이름"
        className="w-full border rounded-lg px-3 py-2"
        required
      />
      <input
        name="email"
        type="email"
        placeholder="이메일"
        className="w-full border rounded-lg px-3 py-2"
        required
      />
      <textarea
        name="message"
        rows={4}
        placeholder="메시지를 입력하세요"
        className="w-full border rounded-lg px-3 py-2 resize-none"
        required
      />

      {/* isPending으로 로딩 상태 자동 처리 */}
      <button
        type="submit"
        disabled={isPending}
        className={`w-full py-2 rounded-lg font-medium text-white transition-colors ${
          isPending ? "bg-blue-300 cursor-not-allowed" : "bg-blue-600 hover:bg-blue-700"
        }`}
      >
        {isPending ? "전송 중..." : "전송하기"}
      </button>
    </form>
  );
}

export default ContactForm;
// 제출 중: "전송 중..." 회색 버튼
// 성공 시: "🎉 홍길동님의 메시지가 전송되었습니다!"
// 실패 시: 빨간 에러 메시지
```

> **useActionState의 장점**:
> - `isLoading`, `error`, `result` 를 각각 `useState`로 따로 관리하지 않아도 됩니다
> - 폼의 `action`에 직접 연결하면 `e.preventDefault()`도 필요 없습니다
> - `isPending`으로 로딩 상태를 자동으로 알 수 있습니다

<a id="8"></a>
## 8️⃣ 실습: 회원가입 폼 [↑](#toc)

### 기본 과제

이름, 이메일, 비밀번호, 비밀번호 확인 필드가 있는 회원가입 폼을 만드세요.

**요구사항:**
1. 모든 필드는 제어 컴포넌트로 구현
2. 제출 시 유효성 검사:
   - 이름: 2자 이상
   - 이메일: `@` 포함 여부
   - 비밀번호: 8자 이상
   - 비밀번호 확인: 비밀번호와 일치 여부
3. 에러 메시지는 해당 필드 아래에 빨간 텍스트로 표시
4. 모든 유효성 검사 통과 시 제출 성공 화면 표시

```jsx
// src/App.jsx
import { useState } from "react";

function SignupForm() {
  const [form, setForm] = useState({
    name: "",
    email: "",
    password: "",
    confirmPassword: "",
  });
  const [errors, setErrors] = useState({});
  const [isSuccess, setIsSuccess] = useState(false);

  function handleChange(e) {
    const { name, value } = e.target;
    setForm((prev) => ({ ...prev, [name]: value }));
    if (errors[name]) {
      setErrors((prev) => ({ ...prev, [name]: "" }));
    }
  }

  function validate() {
    const errs = {};
    if (form.name.length < 2) errs.name = "이름은 2자 이상이어야 합니다";
    if (!/\S+@\S+\.\S+/.test(form.email)) errs.email = "올바른 이메일을 입력하세요";
    if (form.password.length < 8) errs.password = "비밀번호는 8자 이상이어야 합니다";
    if (form.password !== form.confirmPassword) errs.confirmPassword = "비밀번호가 일치하지 않습니다";
    return errs;
  }

  function handleSubmit(e) {
    e.preventDefault();
    const newErrors = validate();
    if (Object.keys(newErrors).length > 0) {
      setErrors(newErrors);
      return;
    }
    setIsSuccess(true);
  }

  if (isSuccess) {
    return (
      <div className="min-h-screen bg-gray-100 flex items-center justify-center">
        <div className="bg-white p-10 rounded-2xl shadow text-center">
          <div className="text-6xl mb-4">🎉</div>
          <h2 className="text-2xl font-bold mb-2">가입 완료!</h2>
          <p className="text-gray-500">
            <strong>{form.name}</strong>님, 환영합니다!
          </p>
          <p className="text-gray-400 text-sm mt-1">{form.email}</p>
        </div>
      </div>
    );
  }

  // 필드 컴포넌트 인라인 정의
  function Field({ label, name, type = "text", placeholder }) {
    return (
      <div>
        <label className="block text-sm font-medium text-gray-700 mb-1">
          {label}
        </label>
        <input
          type={type}
          name={name}
          value={form[name]}
          onChange={handleChange}
          placeholder={placeholder}
          className={`w-full border rounded-lg px-3 py-2 focus:outline-none focus:ring-2 transition-colors ${
            errors[name]
              ? "border-red-400 focus:ring-red-300"
              : "border-gray-300 focus:ring-blue-400"
          }`}
        />
        {errors[name] && (
          <p className="text-red-500 text-xs mt-1">{errors[name]}</p>
        )}
      </div>
    );
  }

  return (
    <div className="min-h-screen bg-gray-100 flex items-center justify-center">
      <form
        onSubmit={handleSubmit}
        className="bg-white rounded-2xl shadow-md p-8 w-full max-w-sm space-y-4"
      >
        <h1 className="text-2xl font-bold text-center mb-2">회원가입</h1>

        <Field label="이름" name="name" placeholder="홍길동" />
        <Field label="이메일" name="email" type="email" placeholder="hong@example.com" />
        <Field label="비밀번호" name="password" type="password" placeholder="8자 이상" />
        <Field label="비밀번호 확인" name="confirmPassword" type="password" placeholder="비밀번호 재입력" />

        <button
          type="submit"
          className="w-full bg-blue-600 text-white py-3 rounded-xl font-semibold hover:bg-blue-700 transition-colors mt-2"
        >
          가입하기
        </button>
      </form>
    </div>
  );
}

export default SignupForm;
// 유효성 검사 실패 시: 각 필드 아래 빨간 에러 메시지
// 성공 시: "🎉 가입 완료!" 화면
```

### 도전 과제

1. **실시간 유효성 검사**: `onBlur` (포커스를 잃을 때) 이벤트로 각 필드 검사
2. **비밀번호 강도 표시**: 글자 수에 따라 "약함 / 보통 / 강함" 표시 (색상 바)
3. **체크박스 동의**: "이용약관 동의" 체크박스를 추가하고, 미동의 시 제출 불가

## 9️⃣ 정리 + 브릿지 [↑](#toc)

### 핵심 정리

| 개념 | 요점 |
|------|------|
| 제어 컴포넌트 | `value` + `onChange` 세트로 state가 입력을 제어 |
| 범용 핸들러 | `[e.target.name]: e.target.value` 로 모든 필드 처리 |
| 불변성 유지 | `setForm(prev => ({ ...prev, [name]: value }))` |
| 폼 제출 | `onSubmit` + `e.preventDefault()` |
| 체크박스 | `checked` + `e.target.checked` (`value` 아님!) |
| React 19 | `useActionState`로 로딩/에러/결과 통합 관리 |

### 체크리스트

- [ ] `value={state}` + `onChange={e => setState(e.target.value)}` 를 쓸 수 있다
- [ ] 객체 상태 + 스프레드로 여러 필드를 관리할 수 있다
- [ ] `e.preventDefault()`로 폼 새로고침을 막을 수 있다
- [ ] 간단한 유효성 검사 로직을 작성할 수 있다
- [ ] `useActionState`의 기본 개념을 설명할 수 있다

---

> **브릿지**: 문자열 상태로 폼을 관리할 수 있게 되었습니다. 하지만 쇼핑 카트처럼 **여러 상품의 목록**을 상태로 관리하거나, 중첩된 객체를 업데이트하려면 어떻게 해야 할까요? **다음 장에서 객체와 배열 상태 관리**를 배우면, 할 일 목록 추가/삭제/수정, 장바구니 같은 실제 앱 기능을 만들 수 있습니다.
