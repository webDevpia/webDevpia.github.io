# 06장. 조건부 렌더링

## 학습 목표

이 장을 마치면 다음을 할 수 있습니다.

- `if/else`와 early return 패턴으로 조건부 렌더링을 구현할 수 있다
- 삼항 연산자 `? :`로 두 가지 UI를 전환할 수 있다
- `&&` 연산자로 조건부 표시/숨김을 처리할 수 있다
- `&&`에서 `0`이 렌더링되는 함정을 피할 수 있다
- 리스트 필터링과 조건부 렌더링을 결합할 수 있다

<a id="1"></a>
## 1️⃣ 날씨에 따라 옷 고르기 — 비유로 시작하기 [↑](#toc)

아침에 일어나서 창문을 봅니다.

- 비가 오면 → 우산을 챙깁니다
- 맑으면 → 선글라스를 챙깁니다
- 흐리면 → 아무것도 안 챙깁니다

React의 조건부 렌더링도 같습니다. 어떤 **조건**이냐에 따라 다른 UI를 화면에 보여줍니다.

```
조건 = true  → 이 컴포넌트를 보여줌
조건 = false → 저 컴포넌트를 보여줌 (또는 아무것도 안 보여줌)
```

오늘 배울 도구는 세 가지입니다: `if/else`, `? :` (삼항), `&&`.

<a id="2"></a>
## 2️⃣ if/else로 조건부 렌더링 — early return 패턴 [↑](#toc)

### 기본 if/else

컴포넌트 함수 안에서 일반 `if/else`를 사용할 수 있습니다.

```jsx
function WeatherIcon({ weather }) {
  let icon;
  let message;

  if (weather === "rain") {
    icon = "🌂";
    message = "우산을 챙기세요!";
  } else if (weather === "sunny") {
    icon = "😎";
    message = "선글라스 착용!";
  } else {
    icon = "🌥️";
    message = "평범한 하루입니다";
  }

  return (
    <div className="text-center p-4">
      <div className="text-6xl mb-2">{icon}</div>
      <p className="text-gray-700">{message}</p>
    </div>
  );
}
// 출력 (weather="rain"): 🌂 우산을 챙기세요!
// 출력 (weather="sunny"): 😎 선글라스 착용!
```

### Early Return 패턴 (권장)

특정 조건에서 완전히 다른 UI를 반환할 때는 **early return**이 더 깔끔합니다.

```jsx
function UserProfile({ isLoading, user }) {
  // 로딩 중이면 즉시 로딩 화면 반환
  if (isLoading) {
    return (
      <div className="flex justify-center p-8">
        <div className="text-gray-500">불러오는 중...</div>
      </div>
    );
  }

  // 유저 정보가 없으면 에러 화면 반환
  if (!user) {
    return (
      <div className="text-center p-8 text-red-500">
        사용자를 찾을 수 없습니다
      </div>
    );
  }

  // 정상 케이스 — 프로필 화면
  return (
    <div className="p-4 bg-white rounded-lg shadow">
      <h2 className="font-bold text-xl">{user.name}</h2>
      <p className="text-gray-600">{user.email}</p>
    </div>
  );
}
// 출력 (isLoading=true): "불러오는 중..." 텍스트
// 출력 (user=null): "사용자를 찾을 수 없습니다" 텍스트
// 출력 (정상): 이름과 이메일이 있는 프로필 카드
```

> **Early return의 장점**: 조건들을 위에서 처리하고, 함수 끝에서는 "정상 케이스"만 반환합니다. 코드 읽기가 훨씬 쉬워집니다.

<a id="3"></a>
## 3️⃣ 삼항 연산자 — 두 가지 선택 [↑](#toc)

### 삼항 연산자란?

```
조건 ? 참일때_반환값 : 거짓일때_반환값
```

`if/else`를 **한 줄**로 표현합니다. JSX 안에서 직접 사용할 수 있어서 매우 유용합니다.

### 로그인/비로그인 전환 예시

```jsx
function Header({ isLoggedIn }) {
  return (
    <header className="bg-blue-600 text-white p-4 flex justify-between items-center">
      <h1 className="font-bold text-lg">My App</h1>

      {/* 삼항 연산자로 버튼 전환 */}
      {isLoggedIn ? (
        <button className="bg-white text-blue-600 px-4 py-1 rounded font-medium">
          로그아웃
        </button>
      ) : (
        <button className="bg-blue-500 text-white border border-white px-4 py-1 rounded">
          로그인
        </button>
      )}
    </header>
  );
}
// 출력 (isLoggedIn=true): 파란 헤더에 "로그아웃" 버튼
// 출력 (isLoggedIn=false): 파란 헤더에 "로그인" 버튼
```

### 텍스트만 다를 때

```jsx
function LikeButton({ liked }) {
  return (
    <button
      className={`px-4 py-2 rounded-full font-medium ${
        liked ? "bg-red-100 text-red-600" : "bg-gray-100 text-gray-600"
      }`}
    >
      {liked ? "❤️ 좋아요 취소" : "🤍 좋아요"}
    </button>
  );
}
// 출력 (liked=true):  빨간 배경의 "❤️ 좋아요 취소"
// 출력 (liked=false): 회색 배경의 "🤍 좋아요"
```

### 언제 삼항 연산자를 쓸까?

| 상황 | 추천 방법 |
|------|-----------|
| 두 가지 중 하나 선택 | 삼항 연산자 `? :` |
| 세 가지 이상 | if/else 또는 별도 변수 |
| JSX 안에서 짧게 | 삼항 연산자 |
| 삼항이 중첩될 때 | if/else로 리팩터 |

> **주의**: 삼항 안에 삼항을 쓰면 (`A ? B : C ? D : E`) 읽기 매우 어렵습니다. 세 가지 이상일 때는 if/else를 쓰세요.

<a id="4"></a>
## 4️⃣ && 연산자 — 보여주거나 아무것도 안 보여주거나 [↑](#toc)

### && 연산자의 동작 원리

```js
true && "보여줘"   // → "보여줘"
false && "보여줘"  // → false (아무것도 렌더링 안 됨)
```

React는 `false`, `null`, `undefined`를 렌더링하지 않습니다. 이 성질을 이용해 "보여주거나 아무것도 안 보여주는" 패턴을 만듭니다.

### 알림 배지 예시

```jsx
function NavBar({ notificationCount }) {
  return (
    <nav className="bg-gray-800 text-white p-4 flex items-center gap-4">
      <span className="font-bold">홈</span>

      {/* notificationCount > 0 일 때만 배지 표시 */}
      <div className="relative">
        <span>🔔</span>
        {notificationCount > 0 && (
          <span className="absolute -top-2 -right-2 bg-red-500 text-white text-xs w-5 h-5 rounded-full flex items-center justify-center">
            {notificationCount}
          </span>
        )}
      </div>
    </nav>
  );
}
// 출력 (notificationCount=3): 🔔 에 빨간 배지 "3"
// 출력 (notificationCount=0): 배지 없이 🔔 만
```

### 에러 메시지 표시

```jsx
function InputField({ label, error }) {
  return (
    <div className="mb-4">
      <label className="block text-sm font-medium text-gray-700 mb-1">
        {label}
      </label>
      <input
        className={`w-full border rounded px-3 py-2 ${
          error ? "border-red-400" : "border-gray-300"
        }`}
      />
      {/* error가 있을 때만 에러 메시지 표시 */}
      {error && (
        <p className="text-red-500 text-sm mt-1">{error}</p>
      )}
    </div>
  );
}
// 출력 (error="이메일 형식이 아닙니다"): 빨간 테두리 + 에러 메시지
// 출력 (error=""): 일반 입력 필드만
```

### ⚠️ 함정: count가 0일 때

```jsx
// ❌ 이렇게 쓰면 0이 화면에 그대로 나타납니다!
function CartBadge({ count }) {
  return (
    <div>
      장바구니
      {count && <span className="badge">{count}</span>}
    </div>
  );
}
// count=0 일 때 출력: "장바구니 0"  ← 0이 숫자로 렌더링됨!
```

왜 이런 일이 생길까요? `0 && <span>` 을 평가하면 `0`이 반환되고, React는 `0`을 숫자로 렌더링합니다.

```jsx
// ✅ 명시적으로 boolean으로 변환하세요
{count > 0 && <span className="badge">{count}</span>}
// 또는
{!!count && <span className="badge">{count}</span>}
// count=0 일 때: 아무것도 렌더링 안 됨 (false → 렌더링 안 됨)
```

> **규칙**: `&&` 왼쪽에는 항상 **boolean 표현식**을 사용하세요.

<a id="5"></a>
## 5️⃣ 여러 조건 분기 — 세 가지 이상일 때 [↑](#toc)

세 가지 이상의 상태가 있을 때 삼항 중첩은 피하고, 다른 방법을 씁니다.

### 방법 1: 변수에 미리 저장

```jsx
function OrderStatus({ status }) {
  let badge;

  if (status === "pending") {
    badge = <span className="bg-yellow-100 text-yellow-800 px-2 py-1 rounded text-sm">대기 중</span>;
  } else if (status === "processing") {
    badge = <span className="bg-blue-100 text-blue-800 px-2 py-1 rounded text-sm">처리 중</span>;
  } else if (status === "shipped") {
    badge = <span className="bg-purple-100 text-purple-800 px-2 py-1 rounded text-sm">배송 중</span>;
  } else if (status === "delivered") {
    badge = <span className="bg-green-100 text-green-800 px-2 py-1 rounded text-sm">배달 완료</span>;
  } else {
    badge = <span className="bg-red-100 text-red-800 px-2 py-1 rounded text-sm">취소됨</span>;
  }

  return (
    <div className="p-4 bg-white rounded shadow">
      <span className="font-medium mr-2">주문 상태:</span>
      {badge}
    </div>
  );
}
// 출력 (status="shipped"): "주문 상태: 배송 중" (보라색 배지)
```

### 방법 2: 매핑 객체 (더 간결)

```jsx
function OrderStatus({ status }) {
  const statusConfig = {
    pending:    { label: "대기 중",   className: "bg-yellow-100 text-yellow-800" },
    processing: { label: "처리 중",   className: "bg-blue-100 text-blue-800" },
    shipped:    { label: "배송 중",   className: "bg-purple-100 text-purple-800" },
    delivered:  { label: "배달 완료", className: "bg-green-100 text-green-800" },
    cancelled:  { label: "취소됨",   className: "bg-red-100 text-red-800" },
  };

  const config = statusConfig[status] || { label: "알 수 없음", className: "bg-gray-100 text-gray-800" };

  return (
    <div className="p-4 bg-white rounded shadow">
      <span className="font-medium mr-2">주문 상태:</span>
      <span className={`px-2 py-1 rounded text-sm ${config.className}`}>
        {config.label}
      </span>
    </div>
  );
}
// 출력 (status="delivered"): "주문 상태: 배달 완료" (초록색 배지)
// 출력 (status="unknown"):   "주문 상태: 알 수 없음" (회색 배지)
```

매핑 객체 방법은 조건이 많을수록 훨씬 관리하기 쉽습니다.

## 6️⃣ 리스트 + 조건부 렌더링 결합 [↑](#toc)

`filter()`로 배열을 걸러내고 `map()`으로 렌더링하면 강력한 조합이 됩니다.

```jsx
function ProductList({ products, showOnlyInStock }) {
  // 조건부 필터링
  const displayedProducts = showOnlyInStock
    ? products.filter((p) => p.inStock)
    : products;

  // 필터링 후 아무것도 없을 때
  if (displayedProducts.length === 0) {
    return (
      <div className="text-center py-12 text-gray-400">
        <div className="text-4xl mb-2">📦</div>
        <p>표시할 상품이 없습니다</p>
      </div>
    );
  }

  return (
    <div className="grid grid-cols-2 gap-4">
      {displayedProducts.map((product) => (
        <div key={product.id} className="bg-white rounded-lg p-4 shadow">
          <h3 className="font-semibold">{product.name}</h3>
          <p className="text-blue-600 font-bold">
            {product.price.toLocaleString("ko-KR")}원
          </p>
          {/* 품절 배지 */}
          {!product.inStock && (
            <span className="text-xs bg-gray-200 text-gray-500 px-2 py-0.5 rounded mt-1 inline-block">
              품절
            </span>
          )}
        </div>
      ))}
    </div>
  );
}
// 출력 (showOnlyInStock=false): 전체 상품 (품절 포함)
// 출력 (showOnlyInStock=true):  재고 있는 상품만
// 출력 (빈 배열):               "표시할 상품이 없습니다"
```

## 7️⃣ 실습: 로그인/로그아웃 UI [↑](#toc)

### 기본 과제

로그인 상태에 따라 다른 화면을 보여주는 UI를 만드세요.

```jsx
// src/App.jsx
// 지금은 useState 없이 변수로 상태를 흉내냅니다
// (다음 장에서 useState로 실제로 토글 구현)

function LoginForm() {
  return (
    <div className="bg-white p-8 rounded-xl shadow-md w-full max-w-sm">
      <h2 className="text-2xl font-bold mb-6 text-center">로그인</h2>
      <div className="space-y-4">
        <input
          type="email"
          placeholder="이메일"
          className="w-full border border-gray-300 rounded px-3 py-2"
        />
        <input
          type="password"
          placeholder="비밀번호"
          className="w-full border border-gray-300 rounded px-3 py-2"
        />
        <button className="w-full bg-blue-600 text-white py-2 rounded font-medium hover:bg-blue-700">
          로그인
        </button>
      </div>
    </div>
  );
}

function UserProfile({ user }) {
  return (
    <div className="bg-white p-8 rounded-xl shadow-md w-full max-w-sm text-center">
      <div className="text-5xl mb-3">{user.avatar}</div>
      <h2 className="text-xl font-bold">{user.name}</h2>
      <p className="text-gray-500 text-sm mb-4">{user.email}</p>
      <button className="bg-gray-100 text-gray-700 px-4 py-2 rounded hover:bg-gray-200">
        로그아웃
      </button>
    </div>
  );
}

function App() {
  // 이 값을 바꿔가며 두 화면을 확인하세요
  const isLoggedIn = false;
  const user = { name: "김철수", email: "chulsoo@example.com", avatar: "🧑" };

  return (
    <div className="min-h-screen bg-gray-100 flex items-center justify-center">
      {isLoggedIn ? <UserProfile user={user} /> : <LoginForm />}
    </div>
  );
}

export default App;
// isLoggedIn=false 출력: 로그인 폼
// isLoggedIn=true  출력: 사용자 프로필 카드
```

### 도전 과제

1. **알림 배너**: `hasError` 변수가 `true`일 때 상단에 빨간 에러 배너를 표시
2. **빈 상태(Empty State)**: 할 일 목록이 비어 있을 때 "할 일이 없습니다! 🎉" 메시지 표시
3. **로딩 상태**: `isLoading` 변수가 `true`일 때 스켈레톤 UI(회색 직사각형) 표시

## 8️⃣ 정리 + 브릿지 [↑](#toc)

### 핵심 정리

| 방법 | 사용 상황 | 예시 |
|------|-----------|------|
| `if` / early return | 완전히 다른 화면, 가드 조건 | 로딩/에러/정상 분기 |
| `? :` 삼항 | JSX 안에서 A 아니면 B | `{isOn ? "켜짐" : "꺼짐"}` |
| `&&` | 있거나 없거나 | `{count > 0 && <Badge />}` |
| 매핑 객체 | 조건이 3개 이상 | `statusConfig[status]` |

### 주의사항 모아보기

- `0 && <컴포넌트 />` → `0`이 화면에 출력됨 → `count > 0 && ...` 으로 수정
- 삼항 중첩은 가독성을 해침 → 3개 이상이면 if/else 사용
- Early return은 "정상 케이스"가 함수 끝에 오도록 정리

### 체크리스트

- [ ] Early return 패턴으로 로딩/에러/정상 화면을 분기할 수 있다
- [ ] `isLoggedIn ? <A /> : <B />` 를 쓸 수 있다
- [ ] `count > 0 && <Badge />` 와 `count && <Badge />` 의 차이를 설명할 수 있다
- [ ] 4가지 주문 상태를 매핑 객체로 처리할 수 있다

---

> **브릿지**: 조건에 따라 화면을 다르게 보여줄 수 있게 되었습니다. 하지만 지금은 `isLoggedIn = false`를 코드에서 직접 바꿔야 합니다. 사용자가 **버튼을 클릭했을 때** 화면이 바뀌면 더 좋겠죠? **다음 장에서 이벤트 처리를 배우면**, 버튼 클릭에 반응하는 진짜 인터랙티브 UI를 만들 수 있습니다.
