## 1️⃣ Node.js 설치 [↑](#toc)

### Node.js가 필요한 이유

React 앱을 개발할 때는 여러 라이브러리를 설치하고 관리해야 합니다. Node.js는 JavaScript를 브라우저 밖에서 실행할 수 있게 해주는 환경이고, 함께 오는 **npm(Node Package Manager)**은 라이브러리 설치 도구입니다.

비유하자면: Node.js는 주방이고, npm은 재료를 사다 주는 마트입니다.

### 설치 방법

**[nodejs.org](https://nodejs.org)** 에 접속하여 **LTS(Long Term Support)** 버전을 다운로드하세요.

> LTS = "오래 지원하는 안정적인 버전"입니다. 가장 최신 버전이 아닌 LTS를 선택하세요.

현재 요구 버전: **Node.js 18 이상**

### 설치 확인

터미널(맥: Terminal, 윈도우: PowerShell)을 열고 다음을 입력하세요.

```bash
node -v
```

출력 예시:
```
v22.11.0
```

```bash
npm -v
```

출력 예시:
```
10.9.0
```

숫자가 나오면 설치 성공입니다. 명령어를 찾을 수 없다는 에러가 나오면 터미널을 완전히 닫았다가 다시 열어보세요.

### 자주 발생하는 문제

| 증상 | 해결 방법 |
|------|-----------|
| `node: command not found` | 터미널 재시작 후 시도 |
| 버전이 18 미만으로 표시됨 | nodejs.org에서 최신 LTS 재설치 |
| 윈도우에서 PowerShell 오류 | `npm config set script-shell bash` 시도 |

## 2️⃣ 첫 React 프로젝트 만들기 (Vite) [↑](#toc)

### Vite가 무엇인가요?

이전에는 React 앱을 만들 때 **Create React App(CRA)**을 썼지만, 느리고 무거워서 현재는 거의 쓰이지 않습니다. **Vite**는 훨씬 빠른 개발 서버와 빌드 도구입니다.

비유: CRA가 구형 차라면, Vite는 전기차입니다. 출발이 즉각적이고 조용합니다.

### 프로젝트 생성

원하는 폴더(예: `Documents` 또는 `바탕화면`)로 이동한 뒤 아래 명령어를 실행합니다.

```bash
npm create vite@latest my-weather-app -- --template react
```

명령어 해석:
- `npm create vite@latest` — Vite 최신 버전으로 프로젝트 생성
- `my-weather-app` — 프로젝트 이름 (폴더명이 됩니다)
- `-- --template react` — React 템플릿 사용

### 설치 및 실행

```bash
cd my-weather-app    # 프로젝트 폴더로 이동
npm install          # 필요한 라이브러리 설치 (1-2분 소요)
npm run dev          # 개발 서버 시작
```

터미널에 다음과 비슷한 내용이 나오면 성공입니다.

```
  VITE v6.0.0  ready in 300 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

브라우저에서 **http://localhost:5173** 을 열면 React 기본 화면이 보입니다.

### 프로젝트 구조 이해하기

```
my-weather-app/
├── public/              ← 정적 파일 (이미지, 아이콘 등)
├── src/                 ← 실제 코드를 작성하는 곳
│   ├── assets/          ← 컴포넌트에서 쓰는 이미지
│   ├── App.css          ← App 컴포넌트 스타일 (곧 삭제할 예정)
│   ├── App.jsx          ← 최상위 컴포넌트 ⭐
│   ├── index.css        ← 전역 스타일 (Tailwind 설정에서 수정)
│   └── main.jsx         ← 앱 진입점 (index.html과 연결)
├── index.html           ← 단 하나의 HTML 파일
├── package.json         ← 프로젝트 설정 + 라이브러리 목록
└── vite.config.js       ← Vite 설정 파일
```

### 각 파일의 역할

**index.html** — 브라우저가 처음 로드하는 파일. `<div id="root">`가 React 앱이 들어갈 자리입니다.

```html
<!-- index.html (핵심 부분만) -->
<div id="root"></div>
<script type="module" src="/src/main.jsx"></script>
```

**main.jsx** — `index.html`의 `#root`에 React 앱을 연결합니다.

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import "./index.css";
import App from "./App.jsx";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <App />
  </StrictMode>
);
```

**App.jsx** — 우리가 가장 많이 수정할 파일. 화면에 보이는 것들이 여기서 시작됩니다.

## 3️⃣ Tailwind CSS 설정 [↑](#toc)

### 설치

개발 서버를 실행 중이라면 `Ctrl + C`로 멈추고, 아래 명령어를 실행합니다.

```bash
npm install tailwindcss @tailwindcss/vite
```

### vite.config.js 수정

```javascript
// vite.config.js
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";  // ← 추가

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),  // ← 추가
  ],
});
```

### index.css 수정

기존 내용을 모두 지우고 아래 한 줄만 남깁니다.

```css
/* src/index.css */
@import "tailwindcss";
```

### 동작 확인

개발 서버를 다시 시작합니다.

```bash
npm run dev
```

**App.jsx**를 열고 임시로 Tailwind 클래스를 추가해 확인해봅니다.

```jsx
// src/App.jsx (임시 확인용)
function App() {
  return (
    <div className="bg-blue-500 text-white text-2xl p-8 min-h-screen">
      Tailwind가 적용되었습니다!
    </div>
  );
}

export default App;
```

브라우저에 파란 배경에 흰 글씨가 보이면 Tailwind 설정 완료입니다.

### App.css도 정리

Tailwind를 사용하면 별도 CSS 파일이 거의 필요 없습니다. `App.css` 파일을 삭제하고 `App.jsx`에서 해당 import를 제거하세요.

```jsx
// 이 줄을 삭제하세요
import "./App.css";
```

## 4️⃣ React DevTools 설치 [↑](#toc)

### React DevTools가 필요한 이유

React DevTools는 브라우저 확장 프로그램으로, React 컴포넌트 구조와 상태(state), Props를 실시간으로 확인할 수 있습니다. 개발할 때 필수 도구입니다.

비유: 의사가 청진기 없이 진료하는 것처럼, React DevTools 없이 React를 개발하면 속이 보이지 않습니다.

### 설치 방법

1. Chrome 브라우저를 엽니다
2. Chrome 웹 스토어에서 **"React Developer Tools"**를 검색합니다
3. "Chrome에 추가" 버튼을 클릭합니다
4. 설치 완료 후 [http://localhost:5173](http://localhost:5173)을 새로고침합니다

Firefox 사용자는 Firefox 부가 기능 마켓플레이스에서 동일하게 검색하세요.

### 사용 방법

1. 개발 서버가 실행 중인 상태에서 브라우저를 엽니다
2. `F12` 또는 우클릭 → "검사"로 개발자 도구를 엽니다
3. 탭 목록에서 **"Components"** 탭을 클릭합니다

**Components 탭에서 확인할 수 있는 것들:**

- 현재 화면의 컴포넌트 트리 구조
- 각 컴포넌트의 Props 값
- 각 컴포넌트의 State 값
- 컴포넌트를 클릭하면 소스 코드로 이동

지금은 `App` 컴포넌트 하나만 보입니다. 앞으로 컴포넌트를 추가할수록 트리가 깊어집니다.

## 5️⃣ 첫 컴포넌트 수정하기 [↑](#toc)

### Hot Reload 체험하기

Vite의 강력한 기능 중 하나는 **HMR(Hot Module Replacement)**입니다. 코드를 저장하는 순간 브라우저가 자동으로 업데이트됩니다. 새로고침이 필요 없습니다.

직접 체험해봅시다. `App.jsx`를 다음과 같이 수정합니다.

```jsx
// src/App.jsx
function App() {
  const title = "날씨 앱";
  const description = "오늘의 날씨를 확인하세요";

  return (
    <div className="min-h-screen bg-gradient-to-br from-blue-400 to-indigo-600 flex items-center justify-center">
      <div className="bg-white rounded-2xl shadow-2xl p-10 text-center max-w-md">
        <h1 className="text-4xl font-bold text-gray-800 mb-4">
          {title}
        </h1>
        <p className="text-gray-500 text-lg">
          {description}
        </p>
        <button className="mt-6 bg-blue-500 hover:bg-blue-600 text-white font-semibold px-6 py-3 rounded-lg transition-colors">
          시작하기
        </button>
      </div>
    </div>
  );
}

export default App;
```

파일을 저장하면 브라우저가 즉시 업데이트됩니다.

**확인 포인트:**
- `{title}`과 `{description}` — JavaScript 변수를 JSX 안에서 사용
- `className` — HTML의 `class` 대신 사용 (JSX 규칙)
- Tailwind 클래스들 — 별도 CSS 없이 바로 스타일 적용
- `hover:bg-blue-600` — 마우스를 올리면 색상 변경

텍스트를 바꿔보고, 색상 클래스를 바꿔보면서 핫 리로드를 경험해보세요. 저장할 때마다 즉시 반영되는 것을 확인하세요.

## 6️⃣ VS Code 추천 확장 [↑](#toc)

VS Code를 사용한다면 아래 확장 프로그램이 개발 효율을 크게 높여줍니다.

### 필수 확장

| 확장 이름 | 기능 |
|-----------|------|
| **ES7+ React/Redux/React-Native snippets** | `rafce` 입력 → 컴포넌트 틀 자동 완성 |
| **Tailwind CSS IntelliSense** | Tailwind 클래스 자동완성 + 미리보기 |
| **ESLint** | 코드 오류를 실시간으로 표시 |
| **Prettier** | 코드 자동 정렬 |

### ES7 Snippets 사용법

새 파일을 만들고 `rafce`를 입력한 뒤 Tab을 누르면:

```jsx
// 자동으로 이 틀이 생성됩니다
const ComponentName = () => {
  return (
    <div>ComponentName</div>
  );
};

export default ComponentName;
```

### Prettier 설정

`.prettierrc` 파일을 프로젝트 루트에 만듭니다.

```json
{
  "semi": true,
  "singleQuote": false,
  "tabWidth": 2,
  "trailingComma": "es5"
}
```

VS Code 설정에서 "Format On Save"를 체크하면 저장할 때 자동 정렬됩니다.

현재 단계에서 DevTools를 열어 다음을 확인해보세요.

1. Components 탭에서 `App` 컴포넌트가 보이는지 확인

3. 화면의 요소를 클릭하면 해당 컴포넌트가 트리에서 하이라이트됩니다

## 실습 과제

### 기본 과제

`App.jsx`를 수정하여 자신만의 첫 화면을 만들어보세요.

**조건:**
- 자신의 이름이 표시되어야 합니다
- Tailwind 클래스를 최소 5가지 사용하세요
- 버튼이 하나 이상 있어야 합니다 (기능은 없어도 됩니다)

### 도전 과제

다음 구조를 참고하여 헤더, 메인, 푸터가 있는 레이아웃을 만들어보세요.

```jsx
function App() {
  return (
    <div className="min-h-screen flex flex-col">
      {/* 헤더 */}
      <header className="bg-blue-600 text-white p-4">
        <h1 className="text-xl font-bold">My App</h1>
      </header>

      {/* 메인 — flex-1로 남은 공간 채우기 */}
      <main className="flex-1 p-8 bg-gray-50">
        <p>여기에 내용을 넣으세요</p>
      </main>

      {/* 푸터 */}
      <footer className="bg-gray-800 text-white text-center p-4 text-sm">
        © 2025 My App
      </footer>
    </div>
  );
}
```
