# 03장. 기본 타입과 타입 추론

<a id="part1"></a>

## 1️⃣ 타입 추론 — 먼저! [↑](#toc)

### JS 02장 상자 비유 확장

> **JS 02장**: 상자에 아무거나 넣을 수 있었다
> **TS 03장**: 상자에 처음 넣은 것을 보고, TypeScript가 자동으로 라벨을 붙여줍니다

TypeScript는 여러분이 생각하는 것보다 훨씬 똑똑합니다. **대부분의 타입을 직접 명시하지 않아도 자동으로 파악**합니다.

```typescript
// 타입 추론 — TypeScript가 알아서 타입을 파악합니다
const name = "홍길동";   // TypeScript가 string으로 추론
const age = 25;          // TypeScript가 number로 추론
const isStudent = true;  // TypeScript가 boolean으로 추론
```

VS Code에서 `name` 위에 마우스를 올리면 `const name: string`이 표시됩니다. 직접 쓰지 않았지만 TypeScript가 파악한 것입니다.

---

### 추론이 동작하는 원리

```typescript
// 오른쪽 값으로 타입을 결정합니다
const greeting = "안녕";         // → string
const count = 0;                 // → number
const flag = false;              // → boolean
const items = ["사과", "배"];    // → string[]
const pair = [1, "hello"];       // → (number | string)[]
```

TypeScript가 값을 분석해서 타입을 결정하므로, 단순한 초기화에서는 타입을 직접 쓸 필요가 없습니다.

---

### 추론이 없으면 에러

```typescript
// 초기값 없이 선언하면 any로 추론 — strict 모드에서 에러
// ❌ strict 모드에서 금지
let value;  
// Variable 'value' implicitly has an 'any' type.

// ✅ 초기값을 주거나
let value = "안녕";  // string으로 추론

// ✅ 타입을 명시하거나
let value: string;
```

---

### 추론 우선, 명시는 나중에

TypeScript 공식 스타일 가이드 원칙:

> **타입 추론이 충분하면 타입 주석을 쓰지 마세요.**
> 타입 주석은 추론이 불가능하거나 부정확할 때만 사용합니다.

```typescript
// ❌ 불필요한 타입 주석 (추론이 정확하게 동작함)
const name: string = "홍길동";
const age: number = 25;

// ✅ 추론에 맡기기
const name = "홍길동";
const age = 25;
```

## 2️⃣ 명시적 타입 주석 [↑](#toc)

### 타입 주석이 필요한 경우

타입 추론이 불가능하거나 충분하지 않을 때 명시적으로 작성합니다.

**1) 함수 매개변수** — 항상 명시

```typescript
// ❌ 매개변수는 추론 불가
function greet(name) {  // 에러: Parameter 'name' implicitly has an 'any' type.
  return `안녕하세요, ${name}님!`;
}

// ✅ 매개변수는 항상 명시
function greet(name: string): string {
  return `안녕하세요, ${name}님!`;
}
```

**2) 나중에 값이 할당될 변수**

```typescript
// ✅ 나중에 값을 할당할 때
let userId: number;
// ... 나중에
userId = 12345;
```

**3) 추론 결과가 너무 넓을 때**

```typescript
// 추론: (number | string)[] — 너무 광범위
const mixed = [1, "hello"];

// 명시: 숫자 배열임을 명시
const scores: number[] = [];
scores.push(95);
scores.push(88);
```

---

### 실무 규칙

| 상황 | 권장 |
|------|------|
| 초기값이 있는 변수 | 추론에 맡기기 |
| 함수 매개변수 | 항상 명시 |
| 함수 반환 타입 | 공개 함수는 명시, 내부 함수는 추론 |
| 나중에 할당되는 변수 | 명시 |

## 3️⃣ string, number, boolean [↑](#toc)

### string — 문자열 타입

JS 02장에서 배운 문자열에 타입을 추가합니다.

```typescript
// 추론 사용 (권장)
const productName = "아메리카노";          // string으로 추론
const message = `${productName} 주문됨`;  // string으로 추론

// 명시 (함수 매개변수 등)
function printName(name: string): void {
  console.log(`이름: ${name}`);
}

// ❌ 타입 불일치
// let title: string = 123;  // 에러: number를 string에 할당 불가
```

**string 타입의 메서드들** — VS Code 자동완성으로 확인 가능

```typescript
const greeting: string = "Hello, World!";

console.log(greeting.toUpperCase());   // HELLO, WORLD!
console.log(greeting.toLowerCase());   // hello, world!
console.log(greeting.length);          // 13
console.log(greeting.includes("World")); // true
console.log(greeting.replace("World", "TypeScript")); // Hello, TypeScript!
```

실행 결과:
```
HELLO, WORLD!
hello, world!
13
true
Hello, TypeScript!
```

---

### number — 숫자 타입

```typescript
// 정수, 소수, 음수 모두 number
const price = 15000;         // number
const discount = 0.1;        // number
const temperature = -3;      // number

function calculateFinalPrice(price: number, discountRate: number): number {
  return price * (1 - discountRate);
}

console.log(calculateFinalPrice(15000, 0.1)); // 13500
```

실행 결과:
```
13500
```

---

### boolean — 참/거짓 타입

```typescript
// boolean: true 또는 false만 가능
const isLoggedIn = false;      // boolean으로 추론
const hasPermission = true;    // boolean으로 추론

function checkAccess(isLoggedIn: boolean, hasPermission: boolean): string {
  if (isLoggedIn && hasPermission) {
    return "접근 허용";
  }
  return "접근 거부";
}

console.log(checkAccess(true, true));   // 접근 허용
console.log(checkAccess(true, false));  // 접근 거부
```

실행 결과:
```
접근 허용
접근 거부
```

## 4️⃣ 배열 타입 [↑](#toc)

### 서랍장 비유 확장

> **JS 04장**: 서랍장처럼 여러 항목을 순서대로 저장하는 배열을 배웠습니다.
> **TS 03장**: TypeScript의 배열은 **칸마다 같은 종류만 넣는 서랍장**입니다.
> string 배열 서랍장에는 숫자를 넣을 수 없습니다.

### 두 가지 표현법

```typescript
// 방법 1: 타입[] (권장 — 더 간결)
const names: string[] = ["김철수", "이영희", "박민준"];
const scores: number[] = [95, 87, 92];
const flags: boolean[] = [true, false, true];

// 방법 2: Array<타입> (제네릭 문법 — 8장에서 자세히)
const names2: Array<string> = ["김철수", "이영희"];
```

두 방법은 완전히 동일합니다. 일반적으로 `string[]` 형태를 더 많이 사용합니다.

---

### 배열 타입의 장점

```typescript
const fruits: string[] = ["사과", "배", "포도"];

// ✅ 문자열만 추가 가능
fruits.push("딸기");

// ❌ 다른 타입 추가 시 에러
// fruits.push(42); // Argument of type 'number' is not assignable to parameter of type 'string'.

// 자동완성: fruits[0]. 입력 시 string 메서드 자동 제시
console.log(fruits[0].toUpperCase()); // 사과 → APPLE은 아니고 "사과" 자체 (영문 아님)
```

---

### 추론과 배열

```typescript
// 추론 사용 (권장)
const fruits = ["사과", "배", "포도"];  // string[]으로 추론

// 빈 배열은 타입 명시 필요
// ❌ 추론 실패 — any[]로 추론됨 (strict에서 주의)
const emptyList = [];  

// ✅ 명시 필요
const emptyList: string[] = [];
emptyList.push("사과");  // ✅ 정상
```

## 5️⃣ 튜플 [↑](#toc)

### 좌석 번호 비유

> 튜플은 **고정된 개수의 값을 순서와 타입이 정해진 배열**입니다.
> 좌석 번호를 `[구역, 번호]`로 표현한다면 `["A", 3]`처럼 첫 번째는 항상 문자열, 두 번째는 항상 숫자입니다.

```typescript
// 튜플: [타입1, 타입2] 형태
type Seat = [string, number];  // [구역, 번호]

const mySeat: Seat = ["A", 3];   // ✅ 정상
const yourSeat: Seat = ["B", 12]; // ✅ 정상

// ❌ 순서가 바뀌면 에러
// const wrongSeat: Seat = [3, "A"]; // Type 'number' is not assignable to type 'string'.

// 구조 분해 할당 (JS 08장에서 배운 패턴)
const [area, seatNumber] = mySeat;
console.log(`구역: ${area}, 번호: ${seatNumber}`); // 구역: A, 번호: 3
```

실행 결과:
```
구역: A, 번호: 3
```

---

### 튜플 실전 사용 — React의 useState

튜플은 React의 `useState` 훅에서 많이 사용됩니다.

```typescript
// React useState 반환값 (미리 보기)
// [현재값, 변경함수] 형태의 튜플
function useState<T>(initial: T): [T, (value: T) => void] {
  // 실제 구현은 React 내부에 있음
  let state = initial;
  const setState = (value: T) => { state = value; };
  return [state, setState];
}
```

## 6️⃣ any — 사용 금지 [↑](#toc)

### any는 타입 시스템 탈퇴 선언

> **`any` = 타입 시스템에서 탈퇴**
> any를 사용하면 TypeScript는 그 변수에 대해 아무런 검사도 하지 않습니다.
> "이 상자에 뭐든 마음대로 넣어도 돼"라고 선언하는 것과 같습니다.

```typescript
// ❌ any — TypeScript 포기 선언
let anything: any = "문자열";
anything = 42;        // 에러 없음
anything = true;      // 에러 없음
anything = null;      // 에러 없음
anything.nonExistentMethod(); // 에러 없음 — 런타임에서 터짐!
```

`any`를 사용하면:
1. 타입 에러를 잡을 수 없습니다
2. 자동완성이 동작하지 않습니다
3. TypeScript를 사용하는 의미가 없어집니다

---

### unknown — any의 올바른 대안

> `unknown`은 "타입을 아직 모른다"는 뜻입니다.
> any처럼 어떤 타입이든 할당할 수 있지만, **사용 전에 타입 확인을 강제**합니다.

```typescript
// ✅ unknown — 사용 전 타입 확인 필요
let value: unknown = "안녕하세요";

// ❌ 타입 확인 없이 사용 불가
// console.log(value.toUpperCase()); // 에러: Object is of type 'unknown'.

// ✅ typeof로 타입 확인 후 사용
if (typeof value === "string") {
  console.log(value.toUpperCase()); // 안녕하세요 — 여기서는 string임이 보장됨
}
```

실행 결과:
```
안녕하세요
```

---

### any vs unknown 비교

| | `any` | `unknown` |
|---|-------|-----------|
| 어떤 값이든 할당 | ✅ | ✅ |
| 사용 전 타입 확인 | ❌ 안 해도 됨 | ✅ 필수 |
| 타입 안전성 | ❌ 없음 | ✅ 있음 |
| 사용 권장 | ❌ 지양 | ✅ 외부 데이터에 사용 |

`any`를 보면 `unknown`으로 바꿀 수 있는지 먼저 확인하세요.

## 7️⃣ null과 undefined [↑](#toc)

### strictNullChecks

> **JS 02장**: `null`은 빈 상자를 일부러 보내는 것, `undefined`는 아직 배달되지 않은 것
> **TS 03장**: `strictNullChecks`가 켜지면, null이 들어올 수 있는 상자에는 반드시 `null`을 타입에 포함해야 합니다

`"strict": true`를 사용하면 `strictNullChecks`가 자동으로 켜집니다.

---

### strictNullChecks 없을 때 (위험)

```typescript
// tsconfig에서 strict: false 인 경우 (권장하지 않음)
let name: string = null;  // 에러 없음 — 위험!
console.log(name.toUpperCase()); // 런타임 에러!
```

---

### strictNullChecks 있을 때 (안전)

```typescript
// strict: true — 기본 설정
let name: string = null;
//                 ~~~~
// ❌ 에러: Type 'null' is not assignable to type 'string'.

// ✅ null을 허용하려면 유니언 타입으로 명시
let name: string | null = null;
```

---

### null 안전하게 다루기

```typescript
function getDisplayName(name: string | null): string {
  // null 체크 없이 사용하면 에러
  // return name.toUpperCase(); // ❌ Object is possibly 'null'.

  // ✅ 방법 1: if 체크
  if (name === null) {
    return "이름 없음";
  }
  return name.toUpperCase(); // 여기서는 null이 아님이 보장됨

  // ✅ 방법 2: nullish 합산 연산자 (JS 08장에서 배운 ??)
  // return (name ?? "이름 없음").toUpperCase();

  // ✅ 방법 3: 옵셔널 체이닝 (JS 07장에서 배운 ?.)
  // return name?.toUpperCase() ?? "이름 없음";
}

console.log(getDisplayName("홍길동")); // 홍길동
console.log(getDisplayName(null));     // 이름 없음
```

실행 결과:
```
홍길동
이름 없음
```

---

### undefined vs null 사용 지침

| | `null` | `undefined` |
|---|-------|-------------|
| 의미 | "값이 없음"을 명시적으로 표현 | "아직 할당되지 않음" |
| 주로 사용 | 함수 반환값, API 응답 | 선택적 속성, 미초기화 변수 |
| TS에서 | `T \| null` | `T \| undefined` 또는 `T?` |

## 8️⃣ 정리 [↑](#toc)

### 핵심 개념 요약

| 개념 | 설명 | 비유 |
|------|------|------|
| 타입 추론 | 초기값으로 TypeScript가 타입을 자동 파악 | 내용물로 라벨 자동 생성 |
| 타입 주석 | `: string` 형태로 타입을 직접 명시 | 수동으로 라벨 붙이기 |
| `string` | 문자열 타입 | 글자 상자 |
| `number` | 숫자 타입 (정수, 소수 모두) | 숫자 상자 |
| `boolean` | 참/거짓 타입 | 스위치 상자 |
| `string[]` | 문자열 배열 | 문자열만 넣는 서랍장 |
| 튜플 | 고정 길이, 각 위치의 타입이 정해진 배열 | 좌석 번호 [구역, 번호] |
| `any` | 타입 검사 포기 (지양) | 라벨 제거 |
| `unknown` | 타입 불명 + 사용 전 확인 강제 | "내용물 모름" 라벨 + 봉인 |
| `null \| string` | null을 허용하는 유니언 타입 | null도 들어갈 수 있는 상자 |
