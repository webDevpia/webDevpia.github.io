# 06장. 타입 별칭과 인터페이스

<a id="part1"></a>

## 1️⃣ 왜 타입에 이름이 필요한가? [↑](#toc)

### 계약서 비유

> 레스토랑이 배달앱에 등록하려면 메뉴, 주소, 배달시간을 제출해야 합니다.
> 배달앱은 이 형식을 `인터페이스`로 정의해두고, 모든 레스토랑이 그 형식을 지키도록 강제합니다.
> 객체 타입을 `interface`로 정의하는 것도 같은 개념입니다 — "이 형태를 가진 객체만 받겠다"는 계약서.

---

### 인라인 타입의 문제

```typescript
// ❌ 인라인 타입 — 같은 구조를 반복
function printUser(user: { name: string; age: number; email: string }): void {
  console.log(`${user.name} (${user.age}세) - ${user.email}`);
}

function saveUser(user: { name: string; age: number; email: string }): void {
  // 저장 로직
  console.log(`${user.name} 저장됨`);
}

function validateUser(user: { name: string; age: number; email: string }): boolean {
  return user.age >= 19;
}
```

같은 객체 구조 `{ name: string; age: number; email: string }`를 세 곳에서 반복 작성했습니다. 나중에 `phone` 속성을 추가하면 세 곳을 모두 수정해야 합니다.

---

### 이름을 붙이면 재사용 가능

```typescript
// ✅ 타입에 이름 붙이기 — 한 번 정의, 여러 곳 사용
interface User {
  name: string;
  age: number;
  email: string;
}

function printUser(user: User): void {
  console.log(`${user.name} (${user.age}세) - ${user.email}`);
}

function saveUser(user: User): void {
  console.log(`${user.name} 저장됨`);
}

function validateUser(user: User): boolean {
  return user.age >= 19;
}

// phone 추가 시 interface User 한 곳만 수정
```

## 2️⃣ type (타입 별칭) [↑](#toc)

### type 키워드로 타입 이름 붙이기

`type` 키워드는 어떤 타입이든 이름을 붙일 수 있습니다.

```typescript
// 객체 타입에 이름 붙이기
type User = {
  name: string;
  age: number;
};

// 원시 타입에 이름 붙이기
type UserID = string;
type Score = number;

// 유니언 타입에 이름 붙이기 (5장에서 배운 패턴)
type Status = "active" | "inactive" | "banned";
type StringOrNumber = string | number;

// 함수 타입에 이름 붙이기 (4장에서 배운 패턴)
type GreetFn = (name: string) => string;
```

---

### type으로 객체 타입 사용

```typescript
type Product = {
  id: number;
  name: string;
  price: number;
  category: string;
};

// 타입 사용
const coffee: Product = {
  id: 1,
  name: "아메리카노",
  price: 4500,
  category: "음료"
};

function formatProduct(product: Product): string {
  return `[${product.category}] ${product.name} - ${product.price.toLocaleString()}원`;
}

console.log(formatProduct(coffee)); // [음료] 아메리카노 - 4,500원
```

실행 결과:
```
[음료] 아메리카노 - 4,500원
```

## 3️⃣ interface (인터페이스) [↑](#toc)

### 객체 구조의 계약서

`interface`는 **객체의 구조를 정의**하는 데 특화된 문법입니다.

```typescript
// interface 선언 — 세미콜론 또는 쉼표로 구분
interface User {
  name: string;
  age: number;
  email: string;
}

// interface를 만족하는 객체 생성
const user1: User = {
  name: "홍길동",
  age: 25,
  email: "hong@example.com"
};

// ❌ 필수 속성 누락 시 에러
// const user2: User = { name: "김철수" };
// Property 'age' is missing in type '{ name: string; }' but required in type 'User'.

// ❌ 추가 속성 포함 시 에러 (객체 리터럴의 경우)
// const user3: User = { name: "이영희", age: 20, email: "e@e.com", phone: "010..." };
// Object literal may only specify known properties, and 'phone' does not exist in type 'User'.
```

---

### interface와 함수

```typescript
interface Product {
  id: number;
  name: string;
  price: number;
}

function createProduct(name: string, price: number): Product {
  return {
    id: Math.floor(Math.random() * 1000),
    name,
    price
  };
}

const newProduct = createProduct("라떼", 5000);
console.log(`${newProduct.name}: ${newProduct.price}원`); // 라떼: 5000원
```

실행 결과:
```
라떼: 5000원
```

## 4️⃣ 선택적 속성 [↑](#toc)

### "있을 수도 없을 수도 있는" 속성

속성 이름 뒤에 `?`를 붙이면 선택적 속성이 됩니다. 있을 수도 없을 수도 있습니다.

```typescript
interface User {
  name: string;
  age: number;
  email?: string;   // 선택적 — string | undefined
  phone?: string;   // 선택적 — string | undefined
}

// email, phone 없이도 유효
const minimalUser: User = { name: "홍길동", age: 25 };

// email만 있는 경우
const userWithEmail: User = {
  name: "김철수",
  age: 30,
  email: "kim@example.com"
};

// 모두 있는 경우
const fullUser: User = {
  name: "이영희",
  age: 28,
  email: "lee@example.com",
  phone: "010-1234-5678"
};
```

---

### 선택적 속성 사용 시 null 체크

```typescript
interface User {
  name: string;
  email?: string;
}

function sendEmail(user: User): void {
  // ❌ email이 없을 수 있으므로 직접 사용 불가
  // console.log(user.email.toLowerCase());
  // Object is possibly 'undefined'.

  // ✅ 체크 후 사용
  if (user.email !== undefined) {
    console.log(`이메일 발송: ${user.email.toLowerCase()}`);
  } else {
    console.log(`${user.name}님의 이메일이 없습니다.`);
  }
}

sendEmail({ name: "홍길동", email: "Hong@Example.com" }); // 이메일 발송: hong@example.com
sendEmail({ name: "김철수" });                              // 김철수님의 이메일이 없습니다.
```

실행 결과:
```
이메일 발송: hong@example.com
김철수님의 이메일이 없습니다.
```

## 5️⃣ 읽기 전용 속성 [↑](#toc)

### 변경 불가 속성

`readonly` 키워드를 붙이면 처음 설정 이후 변경할 수 없는 속성이 됩니다.

```typescript
interface User {
  readonly id: number;   // 읽기 전용 — 생성 후 변경 불가
  name: string;
  age: number;
}

const user: User = { id: 1, name: "홍길동", age: 25 };

user.name = "김철수"; // ✅ name은 변경 가능
// user.id = 2;       // ❌ Cannot assign to 'id' because it is a read-only property.
```

---

### readonly 활용 사례

```typescript
interface Config {
  readonly apiUrl: string;
  readonly version: string;
  maxRetries: number;  // 변경 가능
}

const appConfig: Config = {
  apiUrl: "https://api.example.com",
  version: "1.0.0",
  maxRetries: 3
};

appConfig.maxRetries = 5; // ✅ 변경 가능
// appConfig.apiUrl = "..."; // ❌ 읽기 전용
```

---

### ReadonlyArray — 변경 불가 배열

```typescript
// 배열도 readonly로 만들 수 있음
const numbers: readonly number[] = [1, 2, 3, 4, 5];

// ❌ 변경 메서드 사용 불가
// numbers.push(6);       // Property 'push' does not exist on type 'readonly number[]'.
// numbers[0] = 10;       // Cannot assign to '0' because it is a read-only property.

// ✅ 읽기 메서드는 사용 가능
console.log(numbers[0]); // 1
console.log(numbers.includes(3)); // true
```

## 6️⃣ 확장 (extends) [↑](#toc)

### 레스토랑 비유

> 기본 회원(User) 정보를 바탕으로 학생 회원(Student)을 만들 때,
> 기존 정보를 복사하지 않고 **"기본 회원 + 학생 정보"** 형태로 확장합니다.

---

### interface extends

```typescript
interface User {
  name: string;
  age: number;
  email: string;
}

// User의 모든 속성 + grade 추가
interface Student extends User {
  grade: number;       // 학년 (1~4)
  major: string;       // 전공
}

const student: Student = {
  name: "홍길동",
  age: 22,
  email: "hong@university.kr",
  grade: 3,            // Student 추가 속성
  major: "컴퓨터공학"  // Student 추가 속성
};

console.log(`${student.name} (${student.grade}학년, ${student.major})`);
// 홍길동 (3학년, 컴퓨터공학)
```

실행 결과:
```
홍길동 (3학년, 컴퓨터공학)
```

---

### 여러 인터페이스 확장

```typescript
interface Timestamped {
  createdAt: Date;
  updatedAt: Date;
}

interface SoftDeletable {
  deletedAt: Date | null;
}

// 여러 인터페이스 동시 확장
interface Post extends Timestamped, SoftDeletable {
  id: number;
  title: string;
  content: string;
}
```

---

### type의 교차 타입 (Intersection)

`type`으로도 확장과 비슷한 효과를 낼 수 있습니다. `&` 연산자를 사용합니다.

```typescript
type User = {
  name: string;
  age: number;
};

// & 교차 타입 — 두 타입의 모든 속성을 합침
type StudentType = User & {
  grade: number;
  major: string;
};

const student: StudentType = {
  name: "김철수",
  age: 21,
  grade: 2,
  major: "경영학"
};
```

## 7️⃣ interface vs type [↑](#toc)

### 비교표

| 기능 | `interface` | `type` |
|------|-------------|--------|
| 객체 구조 정의 | ✅ | ✅ |
| 원시 타입 별칭 | ❌ | ✅ (`type ID = string`) |
| 유니언 타입 | ❌ | ✅ (`type A = B \| C`) |
| 교차 타입 | ✅ (`extends`) | ✅ (`&`) |
| 선언 병합 | ✅ (같은 이름 재선언 시 합쳐짐) | ❌ (에러 발생) |
| 튜플 | ❌ | ✅ |

---

### 언제 무엇을 쓸까

**공식 TypeScript 가이드라인:**

> "가능하면 `interface`를 사용하고, `interface`로 표현할 수 없을 때만 `type`을 사용하세요."

실무 기준:
- **객체 구조**: `interface` 우선
- **유니언 타입**: `type` 필수
- **함수 타입**: `type` 선호
- **라이브러리 확장**: `interface` (선언 병합 활용)

```typescript
// ✅ 객체는 interface
interface User {
  name: string;
  age: number;
}

// ✅ 유니언/리터럴은 type
type Status = "active" | "inactive";
type ID = string | number;

// ✅ 함수 타입은 type
type Handler = (event: Event) => void;
```

---

### 선언 병합 — interface만의 특징

같은 이름으로 두 번 `interface`를 선언하면 자동으로 합쳐집니다.

```typescript
// 두 번 선언하면 합쳐짐
interface Window {
  title: string;
}

interface Window {
  theme: string;
}

// 두 선언이 합쳐진 Window
const win: Window = {
  title: "My App",
  theme: "dark"
};
```

이는 주로 **서드파티 라이브러리의 타입을 확장**할 때 사용됩니다.

## 8️⃣ 인덱스 시그니처 [↑](#toc)

### 사전처럼 키-값 쌍

> 인덱스 시그니처는 **동적인 키를 가진 객체**를 타입으로 표현합니다.
> 어떤 문자열 키든 올 수 있지만, 그 값은 모두 같은 타입이어야 합니다.

```typescript
// { [키이름: 키타입]: 값타입 } 형태
interface ScoreRecord {
  [subject: string]: number;
}

const scores: ScoreRecord = {
  "수학": 95,
  "영어": 88,
  "과학": 92,
  "국어": 87
};

// 동적으로 접근 가능
const subject = "수학";
console.log(`${subject}: ${scores[subject]}점`); // 수학: 95점

// 새 항목 추가
scores["역사"] = 90;
console.log(scores["역사"]); // 90
```

실행 결과:
```
수학: 95점
90
```

---

### 인덱스 시그니처와 명시적 속성 혼합

```typescript
interface Config {
  [key: string]: string | number; // 모든 추가 속성
  version: string;                // 반드시 있어야 하는 속성
  maxRetries: number;             // 반드시 있어야 하는 속성
}

// 명시된 속성은 인덱스 시그니처의 값 타입과 호환되어야 함
const config: Config = {
  version: "2.0",     // string — 호환됨
  maxRetries: 3,      // number — 호환됨
  timeout: 5000,      // 추가 속성 — 인덱스 시그니처로 처리
  debug: "true"       // 추가 속성
};
```

---

### Record 유틸리티 타입 (미리보기)

인덱스 시그니처보다 간단하게 키-값 타입을 정의하는 방법으로 `Record<K, V>`가 있습니다. (11장 유틸리티 타입에서 자세히)

```typescript
// Record<키 타입, 값 타입>
type ScoreRecord = Record<string, number>;

// interface 인덱스 시그니처와 동일한 효과
const scores: ScoreRecord = {
  "수학": 95,
  "영어": 88
};
```

## 9️⃣ JS 07장 연결 — 객체에 타입 추가 [↑](#toc)

JS 07장에서 작성한 객체 코드를 TypeScript로 변환합니다.

### JS 07장 코드 → TS 인터페이스 추가

```javascript
// JS 07장 스타일 — 타입 없는 객체
const user = {
  name: "홍길동",
  age: 25,
  address: {
    city: "서울",
    district: "강남구"
  },
  hobbies: ["독서", "코딩", "운동"]
};

function greetUser(user) {
  return `${user.name}님, ${user.address.city}에 사시는군요!`;
}
```

```typescript
// ✅ TypeScript — 인터페이스로 구조 정의
interface Address {
  city: string;
  district: string;
}

interface User {
  name: string;
  age: number;
  address: Address;    // 중첩 인터페이스
  hobbies: string[];
}

const user: User = {
  name: "홍길동",
  age: 25,
  address: {
    city: "서울",
    district: "강남구"
  },
  hobbies: ["독서", "코딩", "운동"]
};

function greetUser(user: User): string {
  return `${user.name}님, ${user.address.city}에 사시는군요!`;
}

console.log(greetUser(user)); // 홍길동님, 서울에 사시는군요!
```

실행 결과:
```
홍길동님, 서울에 사시는군요!
```

---

### JS 07장 배열 메서드 + 인터페이스

```typescript
interface Product {
  id: number;
  name: string;
  price: number;
  inStock: boolean;
}

const products: Product[] = [
  { id: 1, name: "아메리카노", price: 4500, inStock: true },
  { id: 2, name: "카페라떼", price: 5000, inStock: false },
  { id: 3, name: "에스프레소", price: 3500, inStock: true }
];

// JS 07장에서 배운 배열 메서드 + 타입 안전성
const available = products.filter(p => p.inStock);
const names = available.map(p => p.name);
const totalValue = products.reduce((sum, p) => sum + p.price, 0);

console.log(`재고 있는 상품: ${names.join(", ")}`);
console.log(`전체 가격 합계: ${totalValue.toLocaleString()}원`);
```

실행 결과:
```
재고 있는 상품: 아메리카노, 에스프레소
전체 가격 합계: 13,000원
```

| `interface` | 객체 구조의 계약서 | 배달앱 등록 양식 |
| 선택적 속성 `?` | 있을 수도 없을 수도 있는 속성 | "선택 사항" 칸 |
| `readonly` | 초기화 후 변경 불가 속성 | 인쇄된 계약서 내용 |
| `extends` | 기존 interface를 확장 | 기본 계약서 + 추가 조항 |
| `&` 교차 타입 | 두 타입을 모두 만족하는 타입 | A이면서 동시에 B |
| 인덱스 시그니처 | 동적 키-값 타입 | 사전 (key: value) |

### interface vs type 요약

```typescript
// 객체 구조 → interface
interface UserProfile { ... }

// 유니언/리터럴 → type
type Status = "on" | "off";

// 원시 타입 별칭 → type
type UserID = string;
```
