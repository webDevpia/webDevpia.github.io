# 04장. 함수와 타입

<a id="part1"></a>

## 1️⃣ 매개변수 타입 [↑](#toc)

### 자판기 비유 확장

> **JS 05장**: 자판기에 동전을 넣으면 음료가 나옵니다
> **TS 04장**: TypeScript 자판기에는 **동전 투입구 크기가 정해져 있습니다**
> 100원짜리 투입구에 1000원짜리를 넣으려 하면 투입 전에 막아줍니다

JS에서는 함수에 어떤 값을 전달해도 런타임 전까지 알 수 없었습니다. TypeScript는 함수 호출 시점에 바로 알려줍니다.

---

### JS 버그 → TS 해결

```javascript
// ❌ JavaScript — 잘못된 인자를 전달해도 에러 없음
function greet(name) {
  return `안녕하세요, ${name}님!`;
}

console.log(greet("홍길동")); // "안녕하세요, 홍길동님!"
console.log(greet(42));       // "안녕하세요, 42님!" — 의도하지 않은 결과
console.log(greet());         // "안녕하세요, undefined님!" — 버그
```

실행 결과:
```
안녕하세요, 홍길동님!
안녕하세요, 42님!
안녕하세요, undefined님!
```

```typescript
// ✅ TypeScript — 매개변수 타입 명시
function greet(name: string): string {
  return `안녕하세요, ${name}님!`;
}

console.log(greet("홍길동")); // ✅ 정상
// greet(42);                 // ❌ Argument of type 'number' is not assignable to parameter of type 'string'.
// greet();                   // ❌ Expected 1 arguments, but got 0.
```

실행 결과:
```
안녕하세요, 홍길동님!
```

---

### 여러 매개변수 타입

```typescript
// 여러 매개변수 각각에 타입 지정
function createOrder(
  productName: string,
  quantity: number,
  price: number
): string {
  const total = quantity * price;
  return `${productName} x ${quantity}개 = ${total.toLocaleString()}원`;
}

console.log(createOrder("아메리카노", 3, 4500)); // 아메리카노 x 3개 = 13,500원

// ❌ 타입이 맞지 않으면 즉시 에러
// createOrder(3, "아메리카노", 4500); // 인자 순서가 바뀌면 에러
```

실행 결과:
```
아메리카노 x 3개 = 13,500원
```

## 2️⃣ 반환 타입 [↑](#toc)

### 함수의 계약서

> 매개변수 타입이 "어떤 재료를 넣어야 하는가"라면,
> 반환 타입은 "어떤 결과물이 나오는가"의 **약속**입니다.

```typescript
// 반환 타입 명시: 콜론 뒤에 작성
function add(a: number, b: number): number {
  return a + b;
}

function getFullName(firstName: string, lastName: string): string {
  return `${lastName}${firstName}`;
}

function isAdult(age: number): boolean {
  return age >= 19;
}
```

---

### 추론 vs 명시

반환 타입도 TypeScript가 추론할 수 있습니다. 하지만 **공개 함수(API)는 명시를 권장**합니다.

```typescript
// 추론 사용 — 내부 헬퍼 함수에서 OK
function double(n: number) {
  return n * 2;  // TypeScript가 number로 추론
}

// 명시 사용 — 외부에서 사용하는 함수는 명시 권장
function calculateDiscount(originalPrice: number, discountRate: number): number {
  return originalPrice * (1 - discountRate);
}
```

명시 이유:
1. 함수 시그니처만 보고도 반환 타입 파악 가능
2. 실수로 반환 타입이 바뀌면 즉시 에러 감지
3. 코드 문서화 효과

---

### 반환 타입 불일치 에러

```typescript
function getScore(): number {
  return "100"; // ❌ Type 'string' is not assignable to type 'number'.
}
```

이 에러는 여러분을 도와주는 에러입니다. 함수를 고치거나 반환 타입을 올바르게 수정하면 됩니다.

## 3️⃣ void [↑](#toc)

### 영수증만 나오는 자판기

> `void`는 **함수가 아무것도 반환하지 않음**을 표시합니다.
> 자판기에서 음료는 안 나오고 영수증만 출력되는 것과 같습니다.
> `console.log()`가 대표적인 void 함수입니다.

```typescript
// void — 반환값이 없는 함수
function printWelcome(name: string): void {
  console.log(`환영합니다, ${name}님!`);
  // return "뭔가"; // ❌ Type 'string' is not assignable to type 'void'.
  // return; 또는 아무것도 없어도 됨
}

printWelcome("홍길동"); // 환영합니다, 홍길동님!
```

실행 결과:
```
환영합니다, 홍길동님!
```

---

### void vs undefined

```typescript
// void: "반환값이 없는 함수"
function logMessage(msg: string): void {
  console.log(msg);
}

// undefined: "undefined를 반환하는 함수" (드물게 사용)
function returnUndefined(): undefined {
  return undefined;
}

// void를 반환하는 함수에 반환값을 사용하려 하면 에러
// const result: string = logMessage("test"); // ❌
```

## 4️⃣ 선택적 매개변수 [↑](#toc)

### 있을 수도 없을 수도 있는 매개변수

> 선택적 매개변수는 **호출 시 전달해도 되고 안 해도 되는 매개변수**입니다.
> `?`를 붙여서 표시하며, 항상 **필수 매개변수 뒤**에 와야 합니다.

```typescript
// ?: 선택적 매개변수 — 전달 안 해도 됨
function greet(name: string, greeting?: string): string {
  // greeting이 없으면 undefined
  const actualGreeting = greeting ?? "안녕하세요";
  return `${actualGreeting}, ${name}님!`;
}

console.log(greet("홍길동"));            // 안녕하세요, 홍길동님!
console.log(greet("홍길동", "반갑습니다")); // 반갑습니다, 홍길동님!
```

실행 결과:
```
안녕하세요, 홍길동님!
반갑습니다, 홍길동님!
```

---

### 선택적 매개변수의 타입

`greeting?: string`은 `greeting: string | undefined`와 동일합니다.

```typescript
function showProfile(
  name: string,      // 필수
  age: number,       // 필수
  city?: string      // 선택: string | undefined
): void {
  console.log(`이름: ${name}, 나이: ${age}`);
  if (city !== undefined) {
    console.log(`도시: ${city}`);
  }
}

showProfile("홍길동", 25);         // 도시 없이
showProfile("김철수", 30, "서울"); // 도시 포함
```

실행 결과:
```
이름: 홍길동, 나이: 25
이름: 김철수, 나이: 30
도시: 서울
```

---

### 잘못된 순서 — 필수 매개변수는 앞에

```typescript
// ❌ 선택적 매개변수가 필수 매개변수보다 앞에 오면 에러
function greet(greeting?: string, name: string): string {
//                                 ~~~~
// A required parameter cannot follow an optional parameter.
  return `${greeting ?? "안녕"}, ${name}님!`;
}
```

## 5️⃣ 기본값 매개변수 [↑](#toc)

### JS 05장 연결

JS 05장에서 배운 기본값 매개변수에 TypeScript 타입이 추가됩니다.

```typescript
// 기본값이 있으면 TypeScript가 타입을 자동 추론
function greet(name: string, greeting = "안녕하세요"): string {
  //                                  ↑ string으로 자동 추론
  return `${greeting}, ${name}님!`;
}

console.log(greet("홍길동"));              // 안녕하세요, 홍길동님!
console.log(greet("홍길동", "반갑습니다")); // 반갑습니다, 홍길동님!
```

실행 결과:
```
안녕하세요, 홍길동님!
반갑습니다, 홍길동님!
```

기본값이 있으면 타입 주석을 생략해도 됩니다. TypeScript가 기본값으로부터 타입을 추론합니다.

---

### 선택적 vs 기본값 매개변수 비교

```typescript
// 선택적 매개변수: 전달 안 하면 undefined
function withOptional(name: string, greeting?: string): string {
  return `${greeting ?? "안녕"}, ${name}님!`; // null 처리 필요
}

// 기본값 매개변수: 전달 안 하면 기본값 사용
function withDefault(name: string, greeting = "안녕"): string {
  return `${greeting}, ${name}님!`; // null 처리 불필요 — greeting은 항상 string
}
```

실무에서는 **기본값 매개변수**를 선택적 매개변수보다 선호합니다. null 처리가 필요 없기 때문입니다.

## 6️⃣ 나머지 매개변수 [↑](#toc)

### JS 05장 확장 — 가변 인자

JS 05장에서 배운 `...` 나머지 매개변수에 타입을 추가합니다.

```typescript
// 나머지 매개변수: 타입은 항상 배열
function sum(...nums: number[]): number {
  return nums.reduce((total, n) => total + n, 0);
}

console.log(sum(1, 2, 3));          // 6
console.log(sum(10, 20, 30, 40));   // 100
console.log(sum());                 // 0
```

실행 결과:
```
6
100
0
```

---

### 일반 매개변수와 혼합

```typescript
// 첫 매개변수는 필수, 나머지는 가변
function formatList(title: string, ...items: string[]): string {
  const list = items.map(item => `  - ${item}`).join("\n");
  return `${title}:\n${list}`;
}

console.log(formatList(
  "오늘 할 일",
  "TypeScript 공부",
  "운동",
  "저녁 요리"
));
```

실행 결과:
```
오늘 할 일:
  - TypeScript 공부
  - 운동
  - 저녁 요리
```

## 7️⃣ 화살표 함수 타입 [↑](#toc)

### JS 05장 화살표 함수 확장

JS 05장에서 배운 화살표 함수에 TypeScript 타입을 추가합니다.

```typescript
// 화살표 함수: 매개변수와 반환 타입 모두 명시
const add = (a: number, b: number): number => a + b;

const multiply = (a: number, b: number): number => {
  return a * b;
};

// 추론 활용 — 반환 타입은 추론에 맡겨도 됨
const subtract = (a: number, b: number) => a - b; // 반환: number로 추론

console.log(add(3, 5));       // 8
console.log(multiply(3, 5));  // 15
console.log(subtract(10, 3)); // 7
```

실행 결과:
```
8
15
7
```

---

### 배열 메서드와 화살표 함수

JS 04장/07장에서 배운 배열 메서드에 타입이 추가됩니다.

```typescript
const numbers = [1, 2, 3, 4, 5];

// TypeScript가 n의 타입을 number로 추론
const doubled = numbers.map(n => n * 2);
const evens = numbers.filter(n => n % 2 === 0);
const total = numbers.reduce((sum, n) => sum + n, 0);

console.log(doubled); // [2, 4, 6, 8, 10]
console.log(evens);   // [2, 4]
console.log(total);   // 15
```

실행 결과:
```
[2, 4, 6, 8, 10]
[2, 4]
15
```

배열이 `number[]`이면 `map`의 콜백 매개변수 `n`도 `number`로 자동 추론됩니다.

## 8️⃣ 함수 타입 표현식 [↑](#toc)

### 함수 자체를 타입으로

> 함수도 타입을 가집니다. 콜백 함수를 매개변수로 받는 함수를 만들 때 꼭 필요합니다.

```typescript
// 함수 타입 표현식
type CalcFn = (a: number, b: number) => number;

// CalcFn 타입의 함수를 매개변수로 받음
function doCalculation(a: number, b: number, calc: CalcFn): number {
  return calc(a, b);
}

// 사용
const result1 = doCalculation(10, 3, (a, b) => a + b); // 13
const result2 = doCalculation(10, 3, (a, b) => a - b); // 7
const result3 = doCalculation(10, 3, (a, b) => a * b); // 30

console.log(result1, result2, result3);
```

실행 결과:
```
13 7 30
```

### 함수 타입 표현식 실전 — 이벤트 핸들러

```typescript
// DOM 이벤트 핸들러 타입 (12장에서 자세히)
type ClickHandler = (event: MouseEvent) => void;
type ChangeHandler = (event: Event) => void;

// 콜백을 받는 함수
type AsyncCallback<T> = (error: Error | null, result: T | null) => void;

function fetchData(url: string, callback: AsyncCallback<string>): void {
  // 실제로는 fetch를 사용하지만, 예시를 위한 간단 구현
  setTimeout(() => {
    callback(null, `${url}의 데이터`);
  }, 100);
}
```

### interface로 함수 타입 표현

```typescript
// interface로 함수 타입 정의 (호출 시그니처)
interface Formatter {
  (value: string): string;
}

const toUpperCase: Formatter = (value) => value.toUpperCase();
const addPrefix: Formatter = (value) => `[INFO] ${value}`;

console.log(toUpperCase("hello"));      // HELLO
console.log(addPrefix("시스템 시작")); // [INFO] 시스템 시작
```

실행 결과:
```
HELLO
[INFO] 시스템 시작
```

### 함수 오버로드 미리보기

TypeScript는 같은 함수 이름으로 다른 타입 조합을 처리하는 **오버로드**도 지원합니다. (심화 과정에서 다룹니다)

```typescript
// 오버로드 시그니처
function formatValue(value: string): string;
function formatValue(value: number): string;
// 구현 시그니처
function formatValue(value: string | number): string {
  if (typeof value === "string") {
    return `"${value}"`;
  }
  return value.toLocaleString();
}

console.log(formatValue("안녕"));  // "안녕"
console.log(formatValue(15000));   // 15,000
```

실행 결과:
```
"안녕"
15,000
```

## 9️⃣ 정리 [↑](#toc)

### 핵심 개념 요약

| 개념 | 설명 | 비유 |
|------|------|------|
| 매개변수 타입 | `function f(x: string)` 형태로 명시 | 투입구 크기 정하기 |
| 반환 타입 | `): number` 형태로 명시 | 출력물 종류 약속 |

| 선택적 매개변수 | `name?: string` — 전달 안 해도 됨 | 추가 옵션 |
| 기본값 매개변수 | `name = "홍길동"` — 미전달 시 기본값 | 기본 설정값 |
| 나머지 매개변수 | `...nums: number[]` — 여러 인자 배열로 수집 | 여러 재료 한꺼번에 |
| 함수 타입 표현식 | `type Fn = (a: number) => number` | 함수 설계도 |

### 실습 과제

**기본**: JS 05장에서 작성한 함수들에 TypeScript 타입을 추가해보세요.

```typescript
// JS 버전에서 TS 버전으로 변환하세요
// 1. 두 수를 더하는 함수
function add(a, b) { return a + b; }

// 2. 이름과 나이로 소개하는 함수
function introduce(name, age) {
  return `저는 ${name}이고 ${age}살입니다.`;
}

// 3. 배열의 합계를 구하는 함수
```

```typescript
// 변환 함수를 받아서 배열의 각 요소를 변환하는 함수
// JavaScript의 Array.map과 유사하지만, 직접 구현합니다
type TransformFn<T, U> = (item: T) => U;

function transform<T, U>(
  items: T[],
  transformFn: TransformFn<T, U>
): U[] {
  return items.map(transformFn);
}

// 테스트
const numbers = [1, 2, 3, 4, 5];
const strings = transform(numbers, (n) => n.toString());
const doubled = transform(numbers, (n) => n * 2);

console.log(strings); // ['1', '2', '3', '4', '5']
console.log(doubled); // [2, 4, 6, 8, 10]
```
