# 09장. 클래스와 타입

## 1️⃣ 클래스 속성 타입 [↑](#toc)

### 보안 등급 비유

> 클래스는 객체의 **설계도**입니다.
> 접근 제어자는 **보안 등급**입니다.
> `public` = 누구나 접근 가능 (공개),
> `private` = 해당 클래스만 접근 (기밀),
> `protected` = 클래스와 자식 클래스만 접근 (내부 공유).
> **JavaScript에는 없던 것 — TypeScript가 제공하는 새로운 도구입니다.**

### 기본 클래스 선언

JavaScript(JS 8장)에서 배운 클래스에 타입 선언을 추가합니다.

```typescript
// ❌ JavaScript 클래스 — 타입 없음
class PersonJS {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}

// ✅ TypeScript 클래스 — 필드를 먼저 선언해야 함
class Person {
  name: string;
  age: number;
  email: string | undefined; // 있을 수도, 없을 수도 있음

  constructor(name: string, age: number, email?: string) {
    this.name = name;
    this.age = age;
    this.email = email;
  }

  greet(): string {
    return `안녕하세요, ${this.name}님 (${this.age}세)입니다.`;
  }
}

const person = new Person("김민수", 25, "kim@example.com");
console.log(person.greet());
console.log(person.name);
```

실행 결과:
```
안녕하세요, 김민수님 (25세)입니다.
김민수
```

### 확실한 할당 단언 (!)

초기화를 나중에 하는 경우 `!`를 사용합니다.

```typescript
class AppService {
  private db!: DatabaseConnection; // 나중에 초기화될 것을 보장
  private config!: Config;

  async initialize(): Promise<void> {
    this.db = await DatabaseConnection.connect();
    this.config = await loadConfig();
  }
}
```

## 2️⃣ constructor 타입 [↑](#toc)

생성자의 매개변수에도 타입을 지정하고, 기본값과 선택적 매개변수를 사용할 수 있습니다.

```typescript
class BankAccount {
  accountNumber: string;
  owner: string;
  balance: number;
  currency: string;

  constructor(
    accountNumber: string,
    owner: string,
    initialBalance: number = 0, // 기본값
    currency: string = "KRW"    // 기본값
  ) {
    this.accountNumber = accountNumber;
    this.owner = owner;
    this.balance = initialBalance;
    this.currency = currency;
  }

  deposit(amount: number): void {
    if (amount <= 0) {
      throw new Error("입금액은 0보다 커야 합니다");
    }
    this.balance += amount;
  }

  withdraw(amount: number): void {
    if (amount > this.balance) {
      throw new Error("잔액이 부족합니다");
    }
    this.balance -= amount;
  }

  getInfo(): string {
    return `[${this.accountNumber}] ${this.owner}: ${this.balance.toLocaleString()}${this.currency}`;
  }
}

const account = new BankAccount("110-1234-5678", "김민수", 100000);
account.deposit(50000);
account.withdraw(30000);
console.log(account.getInfo());
```

실행 결과:
```
[110-1234-5678] 김민수: 120,000KRW
```

## 3️⃣ 접근 제어자 [↑](#toc)

접근 제어자는 클래스 멤버에 대한 **외부 접근 범위**를 제한합니다.

### 세 가지 접근 제어자

| 제어자 | 접근 범위 | 비유 |
|--------|-----------|------|
| `public` (기본값) | 어디서든 | 공개 정보 |
| `private` | 같은 클래스 내부만 | 기밀 문서 |
| `protected` | 클래스 + 자식 클래스 | 내부 공유 자료 |

```typescript
class Employee {
  public name: string;           // 누구나 접근 가능
  private salary: number;        // Employee 클래스 내부만
  protected department: string;  // Employee와 자식 클래스만

  constructor(name: string, salary: number, department: string) {
    this.name = name;
    this.salary = salary;
    this.department = department;
  }

  public getSalarySlip(): string {
    // private 멤버는 클래스 내부에서 접근 가능
    return `${this.name}: ${this.salary.toLocaleString()}원`;
  }

  private calculateTax(): number {
    return this.salary * 0.03; // 내부 계산 로직
  }
}

const emp = new Employee("이영희", 4000000, "개발팀");
console.log(emp.name);             // ✅ public: 접근 가능
console.log(emp.getSalarySlip()); // ✅ public 메서드
// console.log(emp.salary);       // ❌ private: 외부 접근 불가
// console.log(emp.department);   // ❌ protected: 외부 접근 불가
```

실행 결과:
```
이영희
이영희: 4,000,000원
```

### protected — 자식 클래스에서 접근 가능

```typescript
class Manager extends Employee {
  private teamSize: number;

  constructor(name: string, salary: number, teamSize: number) {
    super(name, salary, "관리팀");
    this.teamSize = teamSize;
  }

  public getTeamInfo(): string {
    // ✅ protected 멤버에 자식 클래스에서 접근 가능
    return `${this.department} — 팀원 ${this.teamSize}명`;
  }
}

const manager = new Manager("박철수", 6000000, 8);
console.log(manager.getTeamInfo()); // 관리팀 — 팀원 8명
// console.log(manager.salary);     // ❌ private: 자식도 접근 불가
```

실행 결과:
```
관리팀 — 팀원 8명
```

## 4️⃣ readonly 속성 [↑](#toc)

`readonly`는 **생성자에서 한 번만 설정**할 수 있고, 이후 변경이 불가합니다.
변경되면 안 되는 식별자(ID, 생성 시각 등)에 사용합니다.

```typescript
class Product {
  readonly id: number;
  readonly createdAt: Date;
  name: string;
  price: number;

  constructor(id: number, name: string, price: number) {
    this.id = id;                  // ✅ 생성자에서만 설정 가능
    this.createdAt = new Date();   // ✅ 생성자에서만 설정 가능
    this.name = name;
    this.price = price;
  }

  updatePrice(newPrice: number): void {
    // this.id = 999; // ❌ 컴파일 오류: readonly 수정 불가
    this.price = newPrice; // ✅ price는 readonly가 아님
  }
}

const product = new Product(1, "노트북", 1200000);
console.log(product.id);        // 1
console.log(product.createdAt); // 생성 시각

// product.id = 2; // ❌ 컴파일 오류: Cannot assign to 'id' because it is a read-only property.
```

## 5️⃣ implements — 인터페이스로 계약하기 [↑](#toc)

### 계약서 비유

> 인터페이스는 **계약서**입니다.
> `implements`는 "이 클래스가 해당 계약서를 이행하겠다"는 선언입니다.
> 계약서에 명시된 모든 항목을 빠짐없이 구현해야 합니다.

```typescript
interface Printable {
  print(): void;
}

interface Serializable {
  serialize(): string;
  deserialize(data: string): void;
}

// 여러 인터페이스를 동시에 구현 가능
class Document implements Printable, Serializable {
  private title: string;
  private content: string;

  constructor(title: string, content: string) {
    this.title = title;
    this.content = content;
  }

  // ✅ Printable 계약 이행
  print(): void {
    console.log(`=== ${this.title} ===`);
    console.log(this.content);
  }

  // ✅ Serializable 계약 이행
  serialize(): string {
    return JSON.stringify({ title: this.title, content: this.content });
  }

  deserialize(data: string): void {
    const parsed = JSON.parse(data);
    this.title = parsed.title;
    this.content = parsed.content;
  }
}

const doc = new Document("TypeScript 노트", "타입 좁히기를 배웠습니다.");
doc.print();
console.log(doc.serialize());
```

실행 결과:
```
=== TypeScript 노트 ===
타입 좁히기를 배웠습니다.
{"title":"TypeScript 노트","content":"타입 좁히기를 배웠습니다."}
```

### implements vs extends

```typescript
// extends: 부모 클래스를 상속 (구현 코드도 물려받음)
class SportsCar extends Car { ... }

// implements: 인터페이스를 구현 (구조만 강제, 구현은 직접)
class Dog implements Animal { ... }
class Robot implements Animal { ... }  // Dog와 완전히 다른 구현 가능
```

## 6️⃣ 추상 클래스 (abstract) [↑](#toc)

### 설계 지침서 비유

> 추상 클래스는 **설계 지침서**입니다.
> 공통 뼈대와 공통 구현은 제공하지만,
> 핵심 세부 사항은 "자식 클래스에서 반드시 구현하라"고 강제합니다.
> 추상 클래스 자체는 직접 인스턴스화할 수 없습니다.

```typescript
abstract class Shape {
  color: string;

  constructor(color: string) {
    this.color = color;
  }

  // 추상 메서드: 자식 클래스에서 반드시 구현해야 함
  abstract area(): number;
  abstract perimeter(): number;

  // 공통 구현: 자식 클래스가 그대로 사용 가능
  describe(): string {
    return `${this.color} 도형 — 넓이: ${this.area().toFixed(2)}, 둘레: ${this.perimeter().toFixed(2)}`;
  }
}

// const s = new Shape("red"); // ❌ 추상 클래스는 인스턴스화 불가

class Circle extends Shape {
  constructor(private radius: number, color: string) {
    super(color);
  }

  area(): number {
    return Math.PI * this.radius ** 2;
  }

  perimeter(): number {
    return 2 * Math.PI * this.radius;
  }
}

class Rectangle extends Shape {
  constructor(
    private width: number,
    private height: number,
    color: string
  ) {
    super(color);
  }

  area(): number {
    return this.width * this.height;
  }

  perimeter(): number {
    return 2 * (this.width + this.height);
  }
}

const shapes: Shape[] = [
  new Circle(5, "빨강"),
  new Rectangle(4, 6, "파랑"),
  new Circle(3, "초록")
];

shapes.forEach(shape => console.log(shape.describe()));
```

실행 결과:
```
빨강 도형 — 넓이: 78.54, 둘레: 31.42
파랑 도형 — 넓이: 24.00, 둘레: 20.00
초록 도형 — 넓이: 28.27, 둘레: 18.85
```

### 추상 클래스 vs 인터페이스

| 항목 | 추상 클래스 | 인터페이스 |
|------|------------|-----------|
| 인스턴스화 | 불가 | 불가 |
| 구현 코드 포함 | 가능 | 불가 (TypeScript 5.x 기준) |
| 상태(필드) 포함 | 가능 | 타입 선언만 |
| 다중 상속 | 불가 (단일 extends) | 가능 (여러 implements) |
| 사용 목적 | 공통 구현 공유 | 구조 계약 |

## 7️⃣ getter / setter [↑](#toc)

`get`/`set` 접근자에도 타입을 지정할 수 있습니다.
private 필드를 외부에서 안전하게 읽고 쓸 수 있도록 합니다.

```typescript
class Temperature {
  private _celsius: number;

  constructor(celsius: number) {
    this._celsius = celsius;
  }

  // getter: 읽기 전용 접근
  get celsius(): number {
    return this._celsius;
  }

  // setter: 유효성 검사 포함 쓰기
  set celsius(value: number) {
    if (value < -273.15) {
      throw new RangeError("절대 영도(-273.15°C) 이하로 내려갈 수 없습니다");
    }
    this._celsius = value;
  }

  // getter만 있는 경우 — 읽기 전용 파생 값
  get fahrenheit(): number {
    return (this._celsius * 9) / 5 + 32;
  }

  get kelvin(): number {
    return this._celsius + 273.15;
  }
}

const temp = new Temperature(100);
console.log(temp.celsius);    // 100
console.log(temp.fahrenheit); // 212
console.log(temp.kelvin);     // 373.15

temp.celsius = 37;
console.log(temp.celsius);    // 37

// temp.fahrenheit = 100; // ❌ setter가 없으면 컴파일 오류
// temp.celsius = -300;   // ❌ RangeError 발생
```

실행 결과:
```
100
212
373.15
37
```

## 8️⃣ 매개변수 속성 [↑](#toc)

생성자 매개변수 앞에 접근 제어자를 붙이면 **속성 선언과 초기화를 한 번에** 처리합니다.
반복 코드를 크게 줄여줍니다.

```typescript
// ❌ 기존 방식 — 선언, 생성자 매개변수, 할당 세 번 반복
class PersonVerbose {
  public name: string;
  private age: number;
  protected email: string;

  constructor(name: string, age: number, email: string) {
    this.name = name;
    this.age = age;
    this.email = email;
  }
}

// ✅ 매개변수 속성 — 한 번에 선언 + 초기화
class Person {
  constructor(
    public name: string,
    private age: number,
    protected email: string,
    readonly id: number
  ) {}

  introduce(): string {
    return `${this.name} (${this.age}세, ID: ${this.id})`;
  }
}

const p = new Person("최지수", 28, "choi@example.com", 42);
console.log(p.introduce());  // 최지수 (28세, ID: 42)
console.log(p.name);        // ✅ public
// console.log(p.age);      // ❌ private
// p.id = 99;               // ❌ readonly
```

실행 결과:
```
최지수 (28세, ID: 42)
최지수
```

### 실용 예제: 매개변수 속성 + implements 조합

```typescript
interface Identifiable {
  id: number;
  createdAt: Date;
}

interface Displayable {
  display(): string;
}

class User implements Identifiable, Displayable {
  readonly createdAt: Date = new Date();

  constructor(
    public readonly id: number,
    public name: string,
    private email: string,
    private role: "admin" | "user" = "user"
  ) {}

  display(): string {
    return `[${this.role.toUpperCase()}] ${this.name} <${this.email}>`;
  }

  isAdmin(): boolean {
    return this.role === "admin";
  }
}

const user = new User(1, "홍길동", "hong@example.com");
const admin = new User(2, "관리자", "admin@example.com", "admin");

console.log(user.display());
console.log(admin.display());
console.log(admin.isAdmin());
```

실행 결과:
```
[USER] 홍길동 <hong@example.com>
[ADMIN] 관리자 <admin@example.com>
true
```

## 9️⃣ 정리 [↑](#toc)

### 핵심 개념 요약

| 개념 | 문법 | 역할 |
|------|------|------|
| 클래스 속성 타입 | `name: string;` | 필드 선언 (생성자 전에 필요) |
| 접근 제어자 — public | `public name: string` | 어디서나 접근 가능 (기본값) |
| 접근 제어자 — private | `private salary: number` | 같은 클래스 내부만 |
| 접근 제어자 — protected | `protected dept: string` | 클래스 + 자식 클래스 |
| readonly | `readonly id: number` | 생성 후 변경 불가 |
| implements | `class A implements B` | 인터페이스를 계약으로 구현 |
| abstract class | `abstract class Shape` | 인스턴스화 불가, 자식에 구현 강제 |
| abstract method | `abstract area(): number` | 자식 클래스에서 반드시 구현 |
| getter / setter | `get value()` / `set value(v)` | 유효성 검사 포함 접근자 |
| 매개변수 속성 | `constructor(public name: string)` | 선언 + 초기화 단축 문법 |

### 실습 과제

#### 기본 — 접근 제어자로 도서관 Book 클래스 만들기

```typescript
// 아래 조건을 만족하는 Book 클래스를 작성하세요
// - readonly id: number
// - public title: string
// - public author: string
// - private _borrowCount: number = 0
// - borrow(): void — _borrowCount를 1 증가
// - get borrowCount(): number — 대출 횟수 반환 (setter 없음)
// - describe(): string — "[id] 제목 (저자) - 대출 {n}회"

const book = new Book(1, "TypeScript 기초", "김민수");
book.borrow();
```
