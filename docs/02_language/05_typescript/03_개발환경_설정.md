## 1️⃣ TypeScript Playground [↑](#toc)

### 오늘 첫 TS 코드는 설치 없이 실행합니다!

> TypeScript Playground는 브라우저에서 바로 TypeScript를 실행할 수 있는 공식 도구입니다.
> 아무것도 설치하지 않아도, 아이디 없이도 바로 사용할 수 있습니다.

**접속**: [https://www.typescriptlang.org/play](https://www.typescriptlang.org/play)

### Playground 화면 구성

```
┌─────────────────────────────────────────────┐
│  TypeScript Playground                       │
├───────────────────┬─────────────────────────┤
│  TypeScript 코드  │   컴파일된 JavaScript   │
│  (왼쪽)           │   (오른쪽 .JS 탭)       │
│                   │                         │
│  Errors 탭        │   타입 에러 목록        │
│  Logs 탭          │   실행 결과 출력        │
└───────────────────┴─────────────────────────┘
```

### Playground 첫 실습

왼쪽 편집기에 아래 코드를 입력해보세요.

```typescript
// TypeScript Playground 첫 실습
// 타입 주석이 있는 함수
function greet(name: string): string {
  return `안녕하세요, ${name}님!`;
}

// greet(42); // 이 줄의 주석을 풀면 Errors 탭에 에러가 표시됩니다
console.log(greet("홍길동"));
```

오른쪽 `.JS` 탭에서 컴파일 결과를 확인하면:

```javascript
// 타입 주석(: string)이 모두 제거됩니다
function greet(name) {
    return `안녕하세요, ${name}님!`;
}
console.log(greet("홍길동"));
```

**Logs 탭**에서 `Run` 버튼을 누르면 실행 결과를 볼 수 있습니다.

### Playground 활용 팁

| 기능 | 위치 | 용도 |
|------|------|------|
| `.JS` 탭 | 오른쪽 상단 | 컴파일된 JavaScript 확인 |
| `Errors` 탭 | 오른쪽 상단 | 타입 에러 목록 |
| `Logs` 탭 | 오른쪽 상단 | Run 버튼 실행 결과 |
| 설정 아이콘 | 상단 | strict 등 tsconfig 옵션 조정 |
| 공유 버튼 | 상단 | URL로 코드 공유 |

## 3️⃣ tsx 설치 [↑](#toc)

### tsx란?

> **tsx = TypeScript eXecute**, 제로 설정 TypeScript 실행기입니다.
> `ts-node`의 대안으로, 설정 없이 `.ts` 파일을 바로 실행할 수 있습니다.
> JS 과정에서 `node hello.js`를 쓴 것처럼, `tsx hello.ts`로 실행합니다.

### 설치

```bash
npm install -g tsx
```

실행 결과:
```
added 1 package in 3s
```

### 설치 확인

```bash
tsx --version
```

실행 결과:
```
4.x.x
```

### ts-node 대신 tsx를 사용하는 이유

| | ts-node | tsx |
|---|---------|-----|
| 설정 | tsconfig 필요 | 제로 설정 |
| 속도 | 느림 | 빠름 (esbuild 기반) |
| ESM 지원 | 복잡 | 기본 지원 |
| 학습용 추천 | △ | ✅ |

## 4️⃣ 첫 TypeScript 파일 [↑](#toc)

### 프로젝트 폴더 준비

```bash
mkdir ts-practice
cd ts-practice
```

### hello.ts 작성

VS Code에서 `hello.ts` 파일을 만들고 아래 내용을 입력합니다.

```typescript
// hello.ts — 나의 첫 TypeScript 파일

// 타입이 있는 변수 선언
const name: string = "홍길동";
const age: number = 25;
const isStudent: boolean = true;

// 타입이 있는 함수
function introduce(name: string, age: number): string {
  return `저는 ${name}이고, ${age}살입니다.`;
}

// 잘못된 타입을 전달하면 에러 발생 (저장 즉시 VS Code가 알려줌)
// introduce(25, "홍길동"); // ❌ 타입 에러

console.log(introduce(name, age));
console.log(`학생 여부: ${isStudent}`);
```

### tsx로 실행

```bash
tsx hello.ts
```

실행 결과:
```
저는 홍길동이고, 25살입니다.
학생 여부: true
```

### .ts와 .js 파일 비교

| 항목 | hello.js | hello.ts |
|------|----------|----------|
| 타입 주석 | 없음 | `: string`, `: number` 등 |
| 에러 감지 | 실행 시 | 저장 즉시 |
| 실행 명령 | `node hello.js` | `tsx hello.ts` |
| 빌드 결과 | 그대로 | `.js` 파일로 컴파일 |

## 5️⃣ tsconfig.json 기초 [↑](#toc)

### tsconfig.json이란?

> tsconfig.json은 TypeScript 컴파일러에게 "이 프로젝트를 어떻게 처리해줘"라고 알려주는 설정 파일입니다.
> 100개가 넘는 옵션이 있지만, **8개만 알면 시작할 수 있습니다**.

### 생성

```bash
npx tsc --init
```

실행 결과:
```
Created a new tsconfig.json with:                 TS
  target: es2016
  module: commonjs
  strict: true
  esModuleInterop: true
  skipLibCheck: true
  forceConsistentCasingInFileNames: true

You can learn more at https://aka.ms/tsconfig
```

생성된 `tsconfig.json`에서 핵심 옵션만 남기면:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "strict": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "skipLibCheck": true
  }
}
```

### 핵심 8개 옵션 설명

| 옵션 | 기본값 | 설명 | 비유 |
|------|--------|------|------|
| `target` | `"ES2016"` | 컴파일 결과물 JS 버전 | "어느 나라 말로 번역할까" |
| `module` | `"commonjs"` | 모듈 시스템 | "짐을 어떻게 포장할까" |
| `strict` | `true` | 엄격한 타입 검사 모두 활성화 | "꼼꼼하게 확인 모드 ON" |
| `outDir` | 없음 | 컴파일된 JS 출력 폴더 | "번역본을 어디에 저장할까" |
| `rootDir` | 없음 | TypeScript 소스 파일 위치 | "원본은 어디 있나" |
| `esModuleInterop` | `true` | CommonJS 모듈 import 간편화 | "다른 형식 짐 쉽게 열기" |
| `forceConsistentCasingInFileNames` | `true` | 파일명 대소문자 일관성 강제 | "맥/윈도우 호환성 확보" |
| `skipLibCheck` | `true` | 외부 라이브러리 타입 검사 생략 | "남의 짐은 검사 안 함" |

### strict 옵션의 중요성

`"strict": true`는 아래 옵션들을 한 번에 켜줍니다.

```
strict: true =
  strictNullChecks: true     ← null/undefined 안전 처리 강제
  + noImplicitAny: true      ← any 타입 자동 할당 금지
  + strictFunctionTypes: true ← 함수 타입 엄격 검사
  + ... (총 8개 옵션)
```

**이 과정에서는 항상 `"strict": true`를 사용합니다.** 처음에는 에러가 많아 불편하지만, 이것이 TypeScript를 올바르게 사용하는 방법입니다.

## 6️⃣ TS 에러 읽기 [↑](#toc)

TypeScript 에러 메시지는 처음에는 당황스럽게 느껴집니다. **읽는 방법**을 알면 금방 익숙해집니다.

> TypeScript 에러는 **아래에서 위로** 읽으세요.
> 가장 마지막 줄이 가장 구체적인 원인입니다.

### 에러 읽기 3단계

**예시 에러:**

```
Argument of type 'string' is not assignable to parameter of type 'number'.
```

**1단계 — 에러 코드 확인**: `TS2345` 같은 코드로 구글 검색 가능

**2단계 — 핵심 단어 파악**:
- `type 'string'` → 내가 전달한 타입
- `type 'number'` → 함수가 기대하는 타입
- `not assignable` → 호환되지 않음

**3단계 — 위치 확인**: 에러 아래 `at line X` 정보로 코드 위치 찾기

---

### 자주 보는 에러 메시지 사전

| 에러 메시지 | 의미 | 해결 방법 |
|------------|------|-----------|
| `Type 'X' is not assignable to type 'Y'` | X 타입을 Y 타입 자리에 넣으려 함 | 올바른 타입의 값 전달 |
| `Property 'X' does not exist on type 'Y'` | Y 타입에 X 속성이 없음 | 오타 확인, 타입 정의 확인 |
| `Argument of type 'X' is not assignable to parameter of type 'Y'` | 함수 호출 시 잘못된 타입 전달 | 올바른 타입 인자 전달 |
| `Object is possibly 'null'` | null일 수 있는 값에 접근 | null 체크 추가 (`if (x !== null)`) |
| `Cannot find name 'X'` | X 변수/함수가 정의되지 않음 | 선언 확인, import 확인 |
| `Implicitly has an 'any' type` | 타입 추론 불가 (strict 모드) | 명시적 타입 주석 추가 |

---

### 에러 실습 — 읽고 고쳐보기

```typescript
// 아래 코드에는 에러가 있습니다. 에러 메시지를 읽고 고쳐보세요.

function multiply(a: number, b: number): number {
  return a * b;
}

const result = multiply("5", 3);
//                       ~~~
// Argument of type 'string' is not assignable to parameter of type 'number'.

console.log(result);
```

해결:
```typescript
const result = multiply(5, 3); // ✅ 문자열 "5" → 숫자 5
```

## 7️⃣ VS Code 설정 [↑](#toc)

VS Code는 TypeScript를 기본으로 지원합니다. 별도 설정 없이도 `.ts` 파일을 열면 타입 검사가 동작합니다.

### TypeScript 기본 기능

**인라인 에러 표시**: 에러가 있는 코드에 빨간 물결선 표시, 마우스를 올리면 에러 메시지 팝업

**타입 호버**: 변수나 함수 위에 마우스를 올리면 타입 정보 팝업

```typescript
const name = "홍길동";
//    ^^^^
// 마우스를 올리면: const name: string
```

**자동완성**: 객체의 `.` 뒤에서 사용 가능한 속성/메서드 목록 자동 제시

```typescript
const text = "안녕하세요";
text.  // 여기서 자동완성: toUpperCase, toLowerCase, split, trim ...
```

**타입 오류 패널**: `Ctrl+Shift+M` (Mac: `Cmd+Shift+M`)으로 프로젝트 전체 에러 목록 확인

---

### 추천 VS Code 확장 프로그램

| 확장 | 용도 | 설치 필수 여부 |
|------|------|-------------|
| TypeScript Hero | import 자동 정렬 | 추천 |
| Error Lens | 에러를 줄 옆에 인라인 표시 | 추천 |
| Pretty TypeScript Errors | 에러 메시지를 읽기 쉽게 변환 | 추천 |

**Pretty TypeScript Errors** 확장을 설치하면 이런 에러가:

```
Argument of type '{ nme: string; age: number; }' is not assignable to parameter of type 'User'. Object literal may only specify known properties, and 'nme' does not exist in type 'User'.
```

이렇게 바뀝니다:

```
Argument of type
  { nme: string; age: number }
is not assignable to parameter of type
  User
Object literal may only specify known properties,
'nme' does not exist in type 'User'
```

### 실습 과제

**기본**: `ts-practice` 폴더에 `hello.ts`를 만들고 `tsx hello.ts`로 실행해보세요.

**중급**: `npx tsc --init`으로 `tsconfig.json`을 생성하고, `strict`, `outDir`, `rootDir` 세 옵션만 남기고 나머지를 주석 처리해보세요.

**심화**: 아래 코드를 `error-practice.ts`로 저장하고, 에러 메시지를 읽어서 3개의 에러를 모두 수정해보세요.

```typescript
// 이 파일에는 3개의 타입 에러가 있습니다
function formatPrice(price: number, currency: string): string {
  return `${price}${currency}`;
}

const total = formatPrice("15000", "원");  // 에러 1

let userName: string = "홍길동";
userName = 42;  // 에러 2

function getFirstItem(items: string[]): string {
  return items[0];
}
getFirstItem("사과");  // 에러 3
```
