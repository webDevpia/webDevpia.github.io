# 05장. 함수 기초

<a id="part1"></a>

## 1️⃣ 함수란? [↑](#toc)

프로그래밍을 하다 보면 **같은 코드를 여러 곳에서 반복**해야 할 때가 생깁니다.
함수는 이 반복을 없애고, **한 번 정의하고 여러 번 재사용**할 수 있게 해줍니다.

### 자판기 비유

> 자판기에 동전(입력)을 넣고 버튼을 누르면 음료(출력)가 나옵니다.
> **한 번 설치**해두면 필요할 때마다 **반복해서 사용**할 수 있습니다.
> 함수도 마찬가지로, 한 번 정의하면 원할 때마다 호출해서 씁니다.

```javascript
// 함수 없이 — 인사 코드를 반복해서 써야 함
console.log("안녕하세요, 김철수님!");
console.log("안녕하세요, 이영희님!");
console.log("안녕하세요, 박민준님!");

// 함수 사용 — 한 번 정의하고 세 번 재사용
function greet(name) {
    console.log(`안녕하세요, ${name}님!`);
}

greet("김철수");
greet("이영희");
greet("박민준");
```

### 함수의 기본 구조

| 구성 요소 | 역할 | 비유 |
|-----------|------|------|
| `function` 키워드 | 함수임을 선언 | 자판기 설치 시작 |
| 함수 이름 | 나중에 호출할 때 사용 | 자판기 모델명 |
| 매개변수(parameter) | 함수 안으로 전달하는 입력값 | 동전 투입구 |
| `return` | 함수가 돌려주는 결과값 | 음료 배출구 |

## 2️⃣ 함수 선언식 [↑](#toc)

`function` 키워드로 시작하는 가장 기본적인 함수 정의 방식입니다.

**매개변수(parameter)**: 함수를 **정의할 때** 사용하는 변수 이름
**인자(argument)**: 함수를 **호출할 때** 실제로 전달하는 값

```javascript
// name이 매개변수, "김철수"가 인자
function greet(name) {
    return `안녕하세요, ${name}님!`;
}

console.log(greet("김철수"));  // 안녕하세요, 김철수님!
```

### return문

`return`이 없으면 `undefined`를 반환하고, `return`을 만나는 순간 함수 실행이 종료됩니다.

```javascript
function add(a, b) {
    return a + b;   // 결과를 반환하고 종료
}

function sayHi() {
    console.log("안녕!");
    // return 없음 → undefined 반환
}

console.log(add(3, 5));    // 8
console.log(sayHi());      // "안녕!" 출력 후 undefined 반환
```

실행 결과:
```
8
안녕!
undefined
```

### 기본값 매개변수 (ES6)

ES6(ECMAScript 2015)부터 매개변수에 **기본값**을 지정할 수 있습니다.
인자를 전달하지 않으면 기본값이 사용됩니다.

```javascript
// 세금 계산 함수 — 세율 기본값은 10%
function calculatePrice(price, taxRate = 0.1) {
    return price + price * taxRate;
}

console.log(calculatePrice(10000));        // 기본값 10% 적용 → 11000
console.log(calculatePrice(10000, 0.05)); // 5% 적용 → 10500
```

실행 결과:
```
11000
10500
```

## 3️⃣ 함수 표현식 [↑](#toc)

함수도 값(value)이기 때문에 **변수에 담을 수** 있습니다.

```javascript
const greet = function(name) {
    return `안녕하세요, ${name}님!`;
};

console.log(greet("이영희"));  // 안녕하세요, 이영희님!
```

### 선언식 vs 표현식: 호이스팅(hoisting) 차이

> **호이스팅(hoisting)**: 코드가 실행되기 전, 선언부가 맨 위로 끌어올려지는 현상.

> 선언식은 **"미리 등록된 전화번호"** — 전화번호부에 있어서 어디서든 바로 찾을 수 있습니다.
> 표현식은 **"그 자리에서 받은 명함"** — 명함을 받은 이후에만 연락할 수 있습니다.

```javascript
// 함수 선언식 — 정의보다 먼저 호출 가능 (호이스팅)
console.log(sayHi("철수"));       // 정상 동작

function sayHi(name) {
    return `안녕, ${name}!`;
}

// 함수 표현식 — 정의보다 먼저 호출하면 오류
console.log(sayHello("철수"));    // ReferenceError 발생!

const sayHello = function(name) {
    return `안녕, ${name}!`;
};
```

실행 결과:
```
안녕, 철수!
ReferenceError: Cannot access 'sayHello' before initialization
```

## 4️⃣ 화살표 함수 [↑](#toc)

ES6에서 도입된 **간결한 함수 문법**입니다. `function` 키워드 대신 `=>`(화살표)를 사용합니다.

### 3단계 변환: function → 화살표 함수

```javascript
// 1단계: 함수 선언식
function add(a, b) {
    return a + b;
}

// 2단계: 함수 표현식으로 변환
const add = function(a, b) {
    return a + b;
};

// 3단계: 화살표 함수 — 본문이 한 줄이면 중괄호와 return 생략
const add = (a, b) => a + b;

console.log(add(3, 7));    // 10
```

### 괄호·중괄호 생략 규칙

```javascript
const double  = x => x * 2;          // 매개변수 1개 — 괄호 생략 가능
const sayHi   = () => "안녕하세요!"; // 매개변수 없음 — 빈 괄호 필수
const greetFull = (name) => {         // 본문 여러 줄 — 중괄호와 return 필요
    const msg = `안녕하세요, ${name}님!`;
    return msg;
};

console.log(double(5));          // 10
console.log(sayHi());           // 안녕하세요!
console.log(greetFull("박민준")); // 안녕하세요, 박민준님!
```

실행 결과:
```
10
안녕하세요!
안녕하세요, 박민준님!
```

### 세 가지 함수 문법 비교표

| 구분 | 문법 | 호이스팅 | 주요 용도 |
|------|------|----------|-----------|
| 함수 선언식 | `function 이름() {}` | ✅ 가능 | 일반적인 함수 정의 |
| 함수 표현식 | `const 이름 = function() {}` | ❌ 불가 | 콜백, 조건부 함수 정의 |
| 화살표 함수 | `const 이름 = () => {}` | ❌ 불가 | 짧은 함수, 콜백 |

## 5️⃣ 콜백 함수 [↑](#toc)

**콜백 함수(callback function)**란 다른 함수의 **인자로 전달되는 함수**입니다.

### 배달 주문 비유

> "음식이 도착하면 이 번호로 전화해주세요" — 배달부는 도착 시점에 그 번호로 전화합니다.
> 함수를 넘기면서 **"이 작업이 끝나면 이 함수를 실행해주세요"** 라고 예약하는 것과 같습니다.

```javascript
// setTimeout(콜백, 지연ms) — 2초 후 콜백 실행
setTimeout(() => {
    console.log("배달이 도착했습니다!");
}, 2000);

console.log("주문 완료! 기다리는 중...");
```

실행 결과:
```
주문 완료! 기다리는 중...
(2초 후)
배달이 도착했습니다!
```

### 배열 메서드와 콜백 (4장 연결)

```javascript
const scores = [85, 92, 78, 95, 60];

// map — 각 요소를 변환해 새 배열 반환
const graded = scores.map(s => s >= 90 ? "A" : s >= 80 ? "B" : "C");
console.log(graded);   // ['B', 'A', 'C', 'A', 'C']

// filter — 조건을 만족하는 요소만 추려 새 배열 반환
const passed = scores.filter(s => s >= 80);
console.log(passed);   // [85, 92, 95]
```

실행 결과:
```
['B', 'A', 'C', 'A', 'C']
[85, 92, 95]
```

### sort()에 비교 함수 전달

```javascript
const numbers = [40, 1, 5, 200, 30];

// 비교 함수 없이 — 문자열 기준 정렬 (의도와 다름)
console.log([...numbers].sort());            // [1, 200, 30, 40, 5]

// 비교 함수(콜백) 전달 — 숫자 오름차순
console.log([...numbers].sort((a, b) => a - b)); // [1, 5, 30, 40, 200]
```

실행 결과:
```
[1, 200, 30, 40, 5]
[1, 5, 30, 40, 200]
```

## 6️⃣ 실용 예제: 계산기 함수 모음 [↑](#toc)

사칙연산 함수를 분리 정의하고, 3장(조건문 switch)과 연결하는 종합 예제입니다.

```javascript
// 사칙연산 함수 — 화살표 함수로 간결하게 정의
const add      = (a, b) => a + b;
const subtract = (a, b) => a - b;
const multiply = (a, b) => a * b;
const divide   = (a, b) => {
    if (b === 0) return "0으로 나눌 수 없습니다.";
    return a / b;
};

// 계산기 함수 — 연산자를 받아 적절한 함수를 호출 (3장 switch 연결)
function calculate(a, operator, b) {
    switch (operator) {
        case "+": return add(a, b);
        case "-": return subtract(a, b);
        case "*": return multiply(a, b);
        case "/": return divide(a, b);
        default : return "알 수 없는 연산자입니다.";
    }
}

console.log(calculate(10, "+", 5));  // 15
console.log(calculate(10, "*", 5));  // 50
console.log(calculate(10, "/", 0));  // 0으로 나눌 수 없습니다.
console.log(calculate(10, "%", 5));  // 알 수 없는 연산자입니다.
```

실행 결과:
```
15
50
0으로 나눌 수 없습니다.
알 수 없는 연산자입니다.
```

> **핵심 패턴**: 각 연산을 별도 함수로 분리하면 코드를 읽기 쉽고,
> 한 연산에 문제가 생겨도 다른 연산에 영향을 주지 않습니다.
> 이를 **관심사 분리(separation of concerns)** 라고 합니다.

## 7️⃣ 정리 [↑](#toc)

### 핵심 개념 요약

| 개념 | 설명 | 비유 |
|------|------|------|
| 함수 | 재사용 가능한 코드 묶음 | 자판기 — 한 번 설치, 반복 사용 |
| 매개변수 / 인자 | 정의 시 변수 이름 / 호출 시 실제 값 | 버튼 종류 / 실제로 누르는 버튼 |
| return | 함수가 돌려주는 결과값 | 자판기에서 나오는 음료 |
| 기본값 매개변수 | 인자 생략 시 사용할 기본값 | 버튼 안 누르면 가장 인기 있는 음료 |
| 호이스팅 | 선언이 코드 상단으로 끌어올려지는 현상 | 미리 등록된 전화번호 |
| 콜백 함수 | 인자로 전달되는 함수 | "도착하면 이 번호로 전화" |

### 함수 문법 비교표

| 문법 | 예시 | 호이스팅 | 특징 |
|------|------|----------|------|
| 함수 선언식 | `function add(a, b) { return a + b; }` | ✅ | 전통적, 어디서나 호출 가능 |
| 함수 표현식 | `const add = function(a, b) { return a + b; };` | ❌ | 변수에 담아 유연하게 사용 |
| 화살표 함수 | `const add = (a, b) => a + b;` | ❌ | 간결한 현대 문법, 콜백에 자주 사용 |

### 다음 장 미리보기

**6장 스코프와 클로저** — 함수를 더 깊이 이해하는 데 핵심적인 두 개념을 다룹니다.

- **스코프(scope)**: 변수가 어디에서 접근 가능한지 정의하는 범위 (지역 vs 전역)
- **클로저(closure)**: 함수가 자신이 만들어진 환경을 기억하는 특성

### 실습 과제

**1단계 (기초)**: 두 수를 받아 더 큰 수를 반환하는 `max(a, b)` 함수를 함수 선언식으로 작성하세요.

**2단계 (응용)**: 숫자 배열을 받아 최솟값과 최댓값을 객체로 반환하는 `getRange(numbers)` 함수를 화살표 함수로 작성하세요. (힌트: `Math.min(...numbers)`, `Math.max(...numbers)`)

```javascript
console.log(getRange([3, 1, 7, 2, 9]));
// { min: 1, max: 9 }
```

**3단계 (심화)**: 이름 배열을 받아 각 이름에 "님"을 붙인 새 배열을 반환하는 `addTitle(names)` 함수를 `map()`과 화살표 함수를 조합해 작성하세요.

```javascript
console.log(addTitle(["김철수", "이영희", "박민준"]));
// ['김철수님', '이영희님', '박민준님']
```
