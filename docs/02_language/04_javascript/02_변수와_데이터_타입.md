# 02장. 변수와 데이터 타입

<a id="part1"></a>

## 1️⃣ 변수란? [↑](#toc)

**변수(Variable)**는 데이터를 저장하는 이름 붙은 공간입니다.

### 이름표가 붙은 상자 비유

> 변수는 **이름표가 붙은 상자**입니다.
> 상자 안에 데이터를 넣고, 이름표를 부르면 안의 데이터를 꺼낼 수 있습니다.

```javascript
// 이름표 "name"이 붙은 상자에 "심선조"를 넣음
let name = "심선조";
console.log(name);  // 상자를 열어서 내용물을 확인
```

실행 결과:
```
심선조
```

---

### 변수가 왜 필요할까?

같은 값을 여러 곳에서 사용할 때, 변수 없이 작성하면 반복 입력이 필요하고 수정도 번거롭습니다.

```javascript
// 변수 없이 — 같은 값을 반복 입력해야 함
console.log("심선조님, 안녕하세요!");
console.log("심선조님의 주문이 접수되었습니다.");

// 변수 사용 — 한 번만 저장하고 재사용
let userName = "심선조";
console.log(userName + "님, 안녕하세요!");
console.log(userName + "님의 주문이 접수되었습니다.");
```

실행 결과:
```
심선조님, 안녕하세요!
심선조님의 주문이 접수되었습니다.
```

> 변수를 사용하면 `userName` 값 하나만 바꿔도 모든 출력이 한꺼번에 바뀝니다.
> 코드 수정이 훨씬 편리해집니다.

## 2️⃣ let과 const [↑](#toc)

JavaScript에서 변수를 선언(만들다)하는 방법은 `let`과 `const` 두 가지입니다.

### 비유

> - `let` = "내용물을 교체할 수 있는 상자" (값 변경 가능)
> - `const` = "봉인된 상자" (한 번 넣으면 교체 불가)

```javascript
// let — 값을 나중에 바꿀 수 있음
let score = 0;
console.log(score);   // 0
score = 100;          // 값 변경 가능!
console.log(score);   // 100

// const — 값을 바꿀 수 없음 (상수)
const PI = 3.14159;
console.log(PI);      // 3.14159
// PI = 3.15;         // ❌ TypeError: Assignment to constant variable.
```

실행 결과:
```
0
100
3.14159
```

---

### 실무 규칙

| 키워드 | 값 변경 | 재선언 | 사용 시기 |
|--------|---------|--------|-----------|
| `const` | ❌ | ❌ | **기본값** — 변경이 필요 없는 대부분의 경우 |
| `let` | ✅ | ❌ | 값이 변경되어야 하는 경우 (카운터, 상태 등) |

> 💡 **실무 팁:** 변수 선언 시 **const를 기본으로 사용**하고, 값을 바꿔야 할 때만 let을 사용하세요. 이렇게 하면 실수로 값을 바꾸는 버그를 예방할 수 있습니다.

---

### 변수 이름 규칙

변수 이름을 지을 때는 몇 가지 규칙이 있습니다.

```javascript
// ✅ 좋은 이름
let userName = "심선조";        // camelCase (첫 단어 소문자, 이후 단어 첫글자 대문자)
let totalPrice = 15000;
const MAX_RETRY = 3;           // 상수는 UPPER_SNAKE_CASE도 사용

// ❌ 잘못된 이름
// let 1name = "에러";          // 숫자로 시작 불가
// let my-name = "에러";        // 하이픈 불가
// let let = "에러";            // 예약어 사용 불가
```

> **camelCase(카멜 케이스)**란 단어를 붙여 쓰되 두 번째 단어부터 첫 글자를 대문자로 쓰는 방식입니다.
> 낙타의 등처럼 중간이 솟아있는 모양에서 이름이 붙었습니다.

## 3️⃣ var는 왜 쓰지 않을까? [↑](#toc)

> ⚠️ **var는 오래된 문법입니다.** 2015년(ES6) 이전에 사용하던 방식이며, 현재는 let과 const 사용을 권장합니다. 오래된 코드에서 var를 볼 수 있으므로 알아두되, 새 코드에서는 사용하지 마세요.

### 문제 1: 재선언 허용 (실수를 잡아주지 않음)

```javascript
// var는 같은 이름으로 재선언 가능 — 실수를 잡아주지 않음
var name = "심선조";
var name = "다른사람";    // 에러 없이 덮어씀! 위험!
console.log(name);        // "다른사람"

// let은 재선언 불가 — 실수를 미리 잡아줌
let age = 25;
// let age = 30;          // ❌ SyntaxError: Identifier 'age' has already been declared
```

실행 결과:
```
다른사람
```

---

### 문제 2: 블록 스코프 무시

**스코프(Scope)**란 변수가 유효한 범위를 뜻합니다. `var`는 `if`, `for` 같은 블록(중괄호 `{}`) 안에서 선언해도 블록 밖에서 접근할 수 있어 예상치 못한 버그를 만듭니다.

```javascript
// var는 블록 스코프를 무시 (if 안에서 선언해도 밖에서 접근 가능)
if (true) {
    var leaked = "나는 밖에서도 보여요!";
    let contained = "나는 안에서만 보여요!";
}
console.log(leaked);       // "나는 밖에서도 보여요!" (의도치 않은 동작)
// console.log(contained); // ❌ ReferenceError (정상 — if 블록 안에서만 존재)
```

실행 결과:
```
나는 밖에서도 보여요!
```

---

### var vs let vs const 비교

| | `var` | `let` | `const` |
|---|-----|-----|-------|
| 재선언 | ✅ 가능 (위험) | ❌ 불가 | ❌ 불가 |
| 값 변경 | ✅ | ✅ | ❌ |
| 블록 스코프 | ❌ 무시 | ✅ 존중 | ✅ 존중 |
| 사용 권장 | ❌ 비추천 | ✅ 값 변경 시 | ✅ 기본값 |

## 4️⃣ 데이터 타입 [↑](#toc)

### 상자 안에 넣는 내용물 비유

> 데이터 타입(Data Type)은 **상자 안에 넣는 내용물의 종류**입니다.
> 숫자, 문자, 참/거짓 등 다양한 타입이 있으며,
> JavaScript는 상자에 넣는 순간 자동으로 타입을 판단합니다.

---

### 기본(원시) 타입 7가지

| 타입 | 설명 | 예시 | `typeof` 결과 |
|------|------|------|------------|
| `string` | 문자열 (텍스트) | `"안녕"`, `'하세요'` | `"string"` |
| `number` | 숫자 (정수, 소수 모두) | `42`, `3.14`, `-10` | `"number"` |
| `boolean` | 참/거짓 | `true`, `false` | `"boolean"` |
| `null` | "비어있음"을 명시적으로 표현 | `null` | `"object"` (역사적 버그) |
| `undefined` | 값이 할당되지 않은 상태 | `undefined` | `"undefined"` |
| `bigint` | 매우 큰 정수 | `9007199254740991n` | `"bigint"` |
| `symbol` | 고유한 식별자 | `Symbol("id")` | `"symbol"` |

> 💡 **입문 단계에서는 string, number, boolean, null, undefined** 5가지만 확실히 이해하면 됩니다. bigint와 symbol은 실무에서 드물게 사용됩니다.

---

### string (문자열)

```javascript
// 문자열은 따옴표로 감싸야 합니다 (작은따옴표, 큰따옴표 모두 가능)
let greeting = "안녕하세요";
let name = '심선조';

// 문자열 길이 확인
console.log(greeting.length);      // 5
// 문자열 합치기 (연결)
console.log(greeting + ", " + name + "님!");  // "안녕하세요, 심선조님!"
```

실행 결과:
```
5
안녕하세요, 심선조님!
```

---

### number (숫자)

```javascript
let price = 15000;       // 정수
let tax = 0.1;           // 소수
let negative = -5;       // 음수

// 기본 연산
console.log(price * tax);           // 1500
console.log(price + (price * tax)); // 16500

// 주의: 특수 숫자 값
console.log(1 / 0);        // Infinity (무한대)
console.log("문자" * 2);   // NaN (Not a Number — 숫자가 아님)
```

실행 결과:
```
1500
16500
Infinity
NaN
```

> **NaN(Not a Number)**이란 숫자가 아닌 값을 숫자 연산에 사용했을 때 나타나는 특수 값입니다.
> 역설적으로 `typeof NaN`은 `"number"`입니다. JavaScript의 또 다른 특이한 점입니다.

---

### boolean (참/거짓)

```javascript
let isLoggedIn = true;
let isAdmin = false;

// 비교 연산의 결과는 boolean
console.log(10 > 5);          // true
console.log(10 < 5);          // false
console.log("abc" === "abc"); // true

// if문에서 사용 (3장에서 자세히 다룸)
if (isLoggedIn) {
    console.log("로그인 상태입니다.");
}
```

실행 결과:
```
true
false
true
로그인 상태입니다.
```

---

### null vs undefined

이 두 가지는 모두 "값이 없음"을 나타내지만 뉘앙스가 다릅니다.

```javascript
// undefined: 값이 아직 할당되지 않음 (JavaScript가 자동으로 설정)
let notYet;
console.log(notYet);       // undefined

// null: "비어있음"을 프로그래머가 명시적으로 설정
let emptyValue = null;
console.log(emptyValue);   // null
```

실행 결과:
```
undefined
null
```

> 비유: **undefined = "택배가 아직 안 옴"**, **null = "빈 상자를 일부러 보냄"**
> undefined는 JavaScript가 자동으로 설정하는 값이고,
> null은 프로그래머가 "여기는 비어있어"라고 직접 지정하는 값입니다.

---

### typeof로 타입 확인

`typeof` 연산자(Operator, 연산을 수행하는 기호나 키워드)를 사용하면 값의 타입을 문자열로 확인할 수 있습니다.

```javascript
console.log(typeof "안녕");     // "string"
console.log(typeof 42);         // "number"
console.log(typeof true);       // "boolean"
console.log(typeof undefined);  // "undefined"
console.log(typeof null);       // "object" — JavaScript의 유명한 역사적 버그!
console.log(typeof {});         // "object" — 객체 (7장에서 자세히)
console.log(typeof []);         // "object" — 배열도 객체의 일종 (4장에서 자세히)
```

실행 결과:
```
string
number
boolean
undefined
object
object
object
```

> `typeof null`이 `"object"`를 반환하는 것은 JavaScript 초기 설계의 버그입니다.
> 오랫동안 수정되지 않은 이유는 이미 수많은 코드가 이 동작에 의존하고 있기 때문입니다.

## 5️⃣ 타입 변환 [↑](#toc)

JavaScript는 상황에 따라 타입을 **자동으로 변환**합니다. 이것이 편리하지만 예상치 못한 버그를 만들기도 합니다.

---

### 자동 변환 (암묵적 변환)

```javascript
// 문자열 + 숫자 → 문자열로 변환 (문자열이 이김!)
console.log("나이: " + 25);        // "나이: 25" (숫자 25가 문자열 "25"로 변환)
console.log("3" + 5);              // "35" (숫자 5가 문자열로 → 문자열 합치기)

// 문자열 - 숫자 → 숫자로 변환 (뺄셈은 숫자만 가능하므로)
console.log("10" - 3);             // 7 (문자열 "10"이 숫자 10으로 변환)
console.log("10" * 2);             // 20

// ⚠️ 이런 동작이 버그를 만들 수 있습니다!
console.log("5" + 3);    // "53" (문자열 연결 — 의도하지 않은 결과!)
console.log("5" - 3);    // 2 (숫자 연산 — + 와 - 가 다르게 동작!)
```

실행 결과:
```
나이: 25
35
7
20
53
2
```

---

### 명시적 변환 (직접 변환)

예상치 못한 자동 변환을 피하려면 직접 타입을 변환하는 것이 좋습니다.

```javascript
// 문자열 → 숫자
let str = "123";
let num = Number(str);      // 123 (숫자)
console.log(typeof num);    // "number"

// 숫자 → 문자열
let price = 15000;
let text = String(price);   // "15000" (문자열)
console.log(typeof text);   // "string"

// 불리언 변환 — 참(truthy) / 거짓(falsy)
console.log(Boolean(0));         // false
console.log(Boolean(""));        // false (빈 문자열)
console.log(Boolean(null));      // false
console.log(Boolean(undefined)); // false
console.log(Boolean("안녕"));    // true (내용이 있는 문자열)
console.log(Boolean(42));        // true (0이 아닌 숫자)
```

실행 결과:
```
number
string
false
false
false
false
true
true
```

---

### falsy 값 — Boolean으로 변환하면 false가 되는 값들

| 값 | Boolean 변환 | 설명 |
|----|-------------|------|
| `0` | `false` | 숫자 0 |
| `""` (빈 문자열) | `false` | 내용 없는 문자열 |
| `null` | `false` | 빈 값 |
| `undefined` | `false` | 미할당 |
| `NaN` | `false` | 숫자가 아님 |
| **그 외 모든 값** | `true` | "비어있지 않은 값"은 모두 참 |

> **truthy(트루시)**와 **falsy(폴시)**란 Boolean으로 변환했을 때 각각 `true`, `false`가 되는 값들을 가리키는 용어입니다.

---

### === vs == 비교

```javascript
// == (느슨한 비교) — 타입을 변환해서 비교 (예상치 못한 결과 발생!)
console.log(5 == "5");      // true (문자열 "5"를 숫자로 변환 후 비교)
console.log(0 == false);    // true
console.log("" == false);   // true

// === (엄격한 비교) — 타입까지 같아야 true ✅ 권장!
console.log(5 === "5");     // false (숫자 vs 문자열 → 타입이 다름)
console.log(0 === false);   // false
console.log("" === false);  // false
```

실행 결과:
```
true
true
true
false
false
false
```

> 💡 **실무 규칙: 항상 `===`(엄격한 비교)를 사용하세요.** `==`는 타입 변환 때문에 예상치 못한 결과를 만들 수 있습니다. ESLint에서도 `===` 사용을 권장합니다.

## 6️⃣ 템플릿 리터럴 [↑](#toc)

### 빈칸 채우기 비유

> 템플릿 리터럴(Template Literal)은 **문자열 안에 변수를 직접 끼워넣는 방법**입니다.
> 마치 "\_\_\_님, 안녕하세요!" 같은 빈칸 채우기 문장처럼,
> `${}` 안에 변수를 넣으면 자동으로 값이 채워집니다.

---

### 기본 사용법

```javascript
let name = "심선조";
let age = 25;

// ❌ 기존 방식 — + 로 연결 (복잡하고 실수하기 쉬움)
console.log("안녕하세요, " + name + "님! " + age + "살이시군요.");

// ✅ 템플릿 리터럴 — 백틱(`)과 ${} 사용 (깔끔하고 직관적)
console.log(`안녕하세요, ${name}님! ${age}살이시군요.`);
```

실행 결과 (동일):
```
안녕하세요, 심선조님! 25살이시군요.
```

> 💡 **백틱(`)은 키보드 어디에?** 숫자 1 왼쪽, ESC 아래에 있는 키입니다. 작은따옴표(')와 다르니 주의하세요!

---

### 연산식과 여러 줄 문자열

```javascript
// ${} 안에는 변수뿐 아니라 연산식도 넣을 수 있습니다
let price = 15000;
let quantity = 3;
console.log(`총 금액: ${price * quantity}원`);   // "총 금액: 45000원"

// 여러 줄 문자열도 가능 (기존 방식에서는 \n 필요)
let message = `
주문 내역:
- 상품: 커피
- 수량: ${quantity}개
- 합계: ${price * quantity}원
`;
console.log(message);
```

실행 결과:
```
총 금액: 45000원

주문 내역:
- 상품: 커피
- 수량: 3개
- 합계: 45000원

```

## 7️⃣ 정리 [↑](#toc)

### 핵심 개념 요약

| 개념 | 설명 | 비유 |
|------|------|------|
| 변수 | 데이터를 저장하는 이름 붙은 공간 | 이름표가 붙은 상자 |
| `let` | 값을 바꿀 수 있는 변수 | 내용물 교체 가능한 상자 |
| `const` | 값을 바꿀 수 없는 상수 | 봉인된 상자 |
| 데이터 타입 | 값의 종류 (string, number, boolean...) | 상자에 넣는 내용물의 종류 |
| `typeof` | 값의 타입을 확인하는 연산자 | "이 상자에 뭐가 들어있지?" |
| 템플릿 리터럴 | 백틱 + `${}`로 문자열에 변수 삽입 | 빈칸 채우기 문장 |
| `===` | 타입까지 비교하는 엄격한 비교 | "겉모양과 내용물 모두 같은지" |
