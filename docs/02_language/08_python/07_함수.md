> ⭐ **상세 챕터** — 학생이 가장 자주 막히는 부분. 특히 **`return` vs `print`** 와 **스코프**.

## 학습 목표

- `def` 로 함수를 정의하고 호출할 수 있다
- **`return` 과 `print` 의 차이**를 설명하고 적절히 사용한다

- **스코프**(지역 / 전역) 를 이해하고 의도치 않은 부작용을 피한다
- `lambda` 함수를 `sorted`, `map`, `filter` 에 활용한다
- **가변 기본값 함정**을 인지하고 회피한다

## 1️⃣ 왜 함수를 쓰는가 [↑](#toc)

```python
# ❌ 반복 — 비효율, 수정 어려움
print(f"이름: 철수, 나이: {2025 - 1995}")
print(f"이름: 영희, 나이: {2025 - 1990}")
print(f"이름: 민수, 나이: {2025 - 1988}")

# ✅ 함수 — 한 번 정의, 여러 번 호출
def get_age(birth_year):
    return 2025 - birth_year

print(f"이름: 철수, 나이: {get_age(1995)}")
print(f"이름: 영희, 나이: {get_age(1990)}")
print(f"이름: 민수, 나이: {get_age(1988)}")
```

### 함수가 해결하는 것

1. **반복 제거** (DRY: Don't Repeat Yourself)
2. **이름 짓기** (코드 의미 명확)
3. **테스트 가능** (작은 단위로 검증)
4. **재사용** (다른 곳에서 호출)

## 2️⃣ 기본 문법 [↑](#toc)

```python
def 함수이름(매개변수1, 매개변수2):
    """(선택) 함수 설명 (docstring)"""
    # 함수 본문
    결과 = ...
    return 결과
```

### 예시

```python
def greet(name):
    return f"안녕하세요, {name}님!"

message = greet("Alice")        # 함수 호출
print(message)                  # 안녕하세요, Alice님!
```

### 매개변수가 없거나 반환이 없는 함수

```python
def say_hi():                   # 매개변수 X
    print("Hi!")                # return X (None 반환)

say_hi()                        # Hi!
result = say_hi()               # 호출은 되지만
print(result)                   # None
```

## 3️⃣ `return` vs `print` ⚠️ (가장 자주 실수) [↑](#toc)

| | `print` | `return` |
|---|---|---|
| 역할 | **화면에 출력** | **값을 반환** |
| 함수 종료? | 아니오 | **예** (즉시 종료) |
| 다른 코드에서 활용? | 못 함 | 변수에 저장 가능 |

### ❌ 흔한 실수

```python
def add(a, b):
    print(a + b)        # 화면 출력만, 함수는 None 반환

result = add(3, 4)      # 화면에 7 출력됨
print(result + 1)
# TypeError: unsupported operand type(s) for +: 'NoneType' and 'int'
```

### ✅ 올바름

```python
def add(a, b):
    return a + b        # 값을 돌려줌

result = add(3, 4)      # 7
print(result + 1)       # 8 (계산 가능)
```

### 둘 다 필요할 때

```python
def add(a, b):
    result = a + b
    print(f"계산: {a} + {b} = {result}")    # 사용자에게 알림
    return result                           # 값 반환

x = add(3, 4)        # 화면: "계산: 3 + 4 = 7"
y = x * 2            # x를 활용 가능
```

> 💡 **외워두기**: 함수가 다른 코드에 쓰일 거면 **`return` 필수**. `print` 만 있으면 그 함수는 "보기 좋게 출력하는 일"만 함.

### 다중 반환 (튜플)

```python
def stats(numbers):
    return min(numbers), max(numbers), sum(numbers) / len(numbers)

mn, mx, avg = stats([1, 2, 3, 4, 5])
print(mn, mx, avg)    # 1 5 3.0
```

`return a, b, c` 는 사실 **튜플 `(a, b, c)` 반환**.

## 4️⃣ 매개변수 4종 [↑](#toc)

```python
def order(menu, size, ice=True, extra=None):
    """
    menu  : 위치 인자 (필수)
    size  : 위치 인자 (필수)
    ice   : 기본값 인자
    extra : 기본값 인자
    """
    print(f"{size} {menu}, 얼음={ice}, 추가={extra}")
```

### 호출 방법 비교

```python
# 1. 위치 인자 (순서대로)
order("아메리카노", "L")

# 2. 키워드 인자 (이름으로)
order(size="L", menu="아메리카노")    # 순서 무관

# 3. 섞기 (위치 → 키워드)
order("아메리카노", "L", extra="시럽")

# ❌ 키워드 다음에 위치 — 에러
# order(menu="아메리카노", "L")     # SyntaxError
```

### 기본값 인자

```python
def greet(name, greeting="안녕"):
    return f"{greeting}, {name}!"

greet("Alice")                      # "안녕, Alice!"
greet("Bob", "Hello")               # "Hello, Bob!"
greet("Carol", greeting="반가워")    # "반가워, Carol!"
```

> ⚠️ **기본값은 함수 정의 시 1회만 평가됨** → 가변 기본값 함정 (7장 §7).

## 5️⃣ `*args` 와 `**kwargs` [↑](#toc)

> **임의 개수의 인자**를 받는 방법.

### `*args` — 임의 개수의 위치 인자 (튜플)

```python
def total(*nums):
    return sum(nums)

total(1, 2)                # 3
total(1, 2, 3, 4, 5)       # 15
total()                    # 0
```

`*nums` 안에 모든 위치 인자가 **튜플로** 들어옵니다.

### `**kwargs` — 임의 개수의 키워드 인자 (dict)

```python
def show_info(**person):
    for key, value in person.items():
        print(f"{key}: {value}")

show_info(name="Alice", age=30, city="Seoul")
# name: Alice
# age: 30
# city: Seoul
```

### 함께 사용

```python
def universal(*args, **kwargs):
    print("위치 인자:", args)
    print("키워드 인자:", kwargs)

universal(1, 2, 3, x=10, y=20)
# 위치 인자: (1, 2, 3)
# 키워드 인자: {'x': 10, 'y': 20}
```

### 리스트/딕셔너리 풀어서 전달 (반대 방향)

```python
def show(a, b, c):
    print(a, b, c)

nums = [1, 2, 3]
show(*nums)              # 풀어서 → show(1, 2, 3)

params = {"a": 10, "b": 20, "c": 30}
show(**params)           # 풀어서 → show(a=10, b=20, c=30)
```

> 💡 **외워두기**:
> - 함수 **정의** 에서 `*args` `**kwargs` = "**모아라**"
> - 함수 **호출** 에서 `*list` `**dict` = "**풀어라**"

## 6️⃣ 스코프 (지역 vs 전역) [↑](#toc)

> 변수가 **어디서 보이는가**.

### 기본 규칙

```python
total = 0           # 전역 변수

def add_to_total(n):
    total = n + 10  # ← 이건 함수 안의 새로운 지역 변수!
    print("함수 안:", total)

add_to_total(5)     # 함수 안: 15
print("함수 밖:", total)    # 함수 밖: 0
```

함수 안의 `total = ...` 은 **새로운 지역 변수**를 만듭니다. 전역 `total` 은 그대로.

### 함수 안에서 전역 읽기는 OK

```python
PI = 3.14            # 전역 상수

def area(r):
    return PI * r * r    # 읽기는 가능 (지역에 없으니 전역 봄)

print(area(5))    # 78.5
```

### 전역 변수 수정하려면 `global` (권장 X)

```python
counter = 0

def increment():
    global counter         # "전역 counter를 쓰겠다"
    counter += 1

increment()
increment()
print(counter)    # 2
```

> ⚠️ `global` 은 **버그 온상**. 가능하면 함수가 값을 받고 결과를 `return` 하는 방식으로:
>
> ```python
> def increment(c):
>     return c + 1
>
> counter = 0
> counter = increment(counter)
> ```

### LEGB 규칙 (이름 검색 순서)

```
L (Local)    — 현재 함수 안
E (Enclosing) — 바깥 함수 (중첩 함수)
G (Global)   — 모듈 전체
B (Built-in) — len, print, range 등
```

이름을 만나면 위 순서로 찾고, **가장 안쪽에서 발견된 것**을 사용.

## 7️⃣ 가변 기본값 함정 ⚠️ [↑](#toc)

> **이것 하나 모르면 추적 불가능한 버그를 만나게 됩니다.**

### 함정 코드

```python
def add_item(item, items=[]):
    items.append(item)
    return items

print(add_item("apple"))       # ['apple']
print(add_item("banana"))      # ['apple', 'banana']  ← 응?!
print(add_item("cherry"))      # ['apple', 'banana', 'cherry']
```

**왜?** Python은 함수 정의 시 기본값을 **한 번만 평가**합니다.
→ 모든 호출이 **같은 리스트**를 공유.

### 안전한 패턴

```python
def add_item(item, items=None):
    if items is None:
        items = []             # 호출할 때마다 새 리스트
    items.append(item)
    return items

print(add_item("apple"))       # ['apple']
print(add_item("banana"))      # ['banana']  ← 독립
```

### 같은 함정 — dict, set 도 마찬가지

```python
# ❌ 위험
def cache_value(key, val, cache={}):
    cache[key] = val
    return cache

# ✅ 안전
def cache_value(key, val, cache=None):
    if cache is None:
        cache = {}
    cache[key] = val
    return cache
```

> 💡 **외워두기**: **mutable 객체(list/dict/set)는 기본값으로 쓰면 안 됨**. 항상 `None` 으로 두고 함수 안에서 새로 만들기.

## 8️⃣ `lambda` 와 고차 함수 [↑](#toc)

### lambda — 한 줄짜리 익명 함수

```python
# def 로 쓰면
def square(x):
    return x * x

# lambda 로 쓰면 (한 줄)
square = lambda x: x * x

print(square(5))    # 25
```

> 💡 **언제 lambda?** **다른 함수에 인자로 한 번만 넘길 때**. 그 외엔 `def` 가 가독성 더 좋음.

### 자주 만나는 곳 1 — `sorted(key=...)`

```python
words = ["banana", "fig", "cherry"]

sorted(words, key=lambda w: len(w))          # 길이순
# ['fig', 'banana', 'cherry']

sorted(words, key=lambda w: w[-1])            # 마지막 글자 기준
# ['banana', 'cherry', 'fig']

# 객체 정렬
people = [{"name": "A", "age": 30}, {"name": "B", "age": 25}]
sorted(people, key=lambda p: p["age"])
# [{'name': 'B', 'age': 25}, {'name': 'A', 'age': 30}]
```

### 자주 만나는 곳 2 — `map` / `filter` (대부분 컴프리헨션으로 대체)

```python
nums = [1, 2, 3, 4]

# map: 모든 원소에 함수 적용
list(map(lambda x: x * 2, nums))      # [2, 4, 6, 8]
# 컴프리헨션이 더 Pythonic
[x * 2 for x in nums]                  # [2, 4, 6, 8]

# filter: 조건 통과한 것만
list(filter(lambda x: x > 2, nums))    # [3, 4]
# 컴프리헨션 버전
[x for x in nums if x > 2]              # [3, 4]
```

> 💡 **Python 관행**: `map` / `filter` 보다 **컴프리헨션 권장**. 다음 챕터 참조.

## 9️⃣ 자기 점검 5문항 [↑](#toc)

### Q1

```python
def multiply(a, b):
    print(a * b)

result = multiply(3, 4)
print(result)
```
출력 두 줄은? 무엇이 잘못됐나?

<details><summary>정답</summary>
`12` / `None`. 함수가 `print` 만 하고 `return` 이 없어서 결과를 다른 곳에 못 씀.
```python
def multiply(a, b):
    return a * b    # 수정
```
</details>

### Q2

```python
def f(*args, **kwargs):
    print(args)
    print(kwargs)

f(1, 2, 3, x=10, y=20)
```
출력은?

<details><summary>정답</summary>
```
(1, 2, 3)
{'x': 10, 'y': 20}
```
</details>

### Q3

```python
def add(a, b, c):
    return a + b + c

nums = [1, 2, 3]
print(add(nums))
```
무슨 에러? 어떻게 고치나?

<details><summary>정답</summary>
`TypeError: add() missing 2 required positional arguments`. 리스트 하나만 전달됨.

언패킹으로 해결: `print(add(*nums))`.
</details>

### Q4

```python
def append_log(msg, logs=[]):
    logs.append(msg)
    return logs

print(append_log("a"))
print(append_log("b"))
```
두 출력은? 왜?

<details><summary>정답</summary>
`['a']` / `['a', 'b']`. **가변 기본값 함정**. 기본 리스트가 호출 사이에 공유됨. 수정:
```python
def append_log(msg, logs=None):
    if logs is None:
        logs = []
    logs.append(msg)
    return logs
```
</details>

### Q5

`students = [{"name":"A","score":80},{"name":"B","score":92},{"name":"C","score":75}]` 을 점수 높은 순으로 정렬해 이름만 출력하세요.

<details><summary>정답</summary>
```python
sorted_students = sorted(students, key=lambda s: -s["score"])
for s in sorted_students:
    print(s["name"])
# B / A / C
```
</details>

## 정리 카드

```
def 함수(매개변수):       함수 정의
return 값                값 반환 (없으면 None)
print(값)                화면 출력 (≠ 반환)

매개변수
  위치       def f(a, b)
  기본값      def f(a, b=10)
  *args       임의 개수의 위치 인자 (튜플)
  **kwargs    임의 개수의 키워드 인자 (dict)
  *list / **dict  호출 시 풀어서 전달

스코프
  지역(함수 안) → enclosing → 전역 → built-in
  전역 수정 = global (가급적 피하고 return 사용)

⚠️ 가변 기본값 함정
  def f(x, items=[])  ← 같은 리스트 공유!
  def f(x, items=None): if items is None: items = []

lambda x: x * 2       한 줄짜리 익명 함수
sorted(list, key=lambda x: ...)  정렬 기준 함수
```
