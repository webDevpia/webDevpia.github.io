## 1️⃣ 변수와 동적 타이핑 [↑](#toc)

> 변수 = **이름표가 붙은 메모리 자리**. Python은 타입을 미리 선언하지 않습니다.

```python
x = 10          # 이 순간 x는 정수
x = "hello"     # 다음 순간 x는 문자열 (에러 없음)
x = [1, 2, 3]   # 그 다음은 리스트
```

> 💡 다른 언어(Java, C)는 `int x = 10;` 처럼 타입을 미리 적습니다. Python은 **값에 따라 타입이 자동 결정**됩니다 = 동적 타이핑.

### 변수 명명 규칙

- 영문자·숫자·`_` 사용. **숫자로 시작 금지**
- 대소문자 구분 (`age` ≠ `Age`)
- 예약어 사용 금지 (`if`, `for`, `class` 등)
- 관행: `snake_case` (소문자 + 언더스코어)

```python
# 좋음
user_name = "Alice"
total_score = 95

# 안 좋음 (동작은 하지만 관행 위배)
UserName = "Bob"    # 클래스명 같음
totalScore = 95     # camelCase는 JS 스타일
```

## 2️⃣ 기본 타입 4종 [↑](#toc)

| 타입 | 예 | `type()` 결과 |
|---|---|---|
| `int` | `42`, `-7`, `0` | `<class 'int'>` |
| `float` | `3.14`, `-0.5`, `1.0` | `<class 'float'>` |
| `str` | `"hi"`, `'안녕'` | `<class 'str'>` |
| `bool` | `True`, `False` | `<class 'bool'>` |

```python
print(type(42))          # <class 'int'>
print(type(3.14))        # <class 'float'>
print(type("hello"))     # <class 'str'>
print(type(True))        # <class 'bool'>
```

### bool 의 숨겨진 정체

`True` 와 `False` 는 사실 **`int` 의 자식**:

```python
print(True + 1)      # 2  (True는 1처럼 동작)
print(False + 10)    # 10
print(True == 1)     # True
```

> ⚠️ **단, 가독성을 위해 `True + 1` 같은 코드는 쓰지 마세요.** 알아두기만.

## 3️⃣ 형변환 [↑](#toc)

> 다른 타입으로 바꿔서 사용. **`int()`, `float()`, `str()`** 3개만 알면 됨.

```python
# 문자열 → 숫자
int("42")         # 42
float("3.14")     # 3.14

# 숫자 → 문자열
str(42)           # "42"
str(3.14)         # "3.14"

# float → int (소수점 버림)
int(3.9)          # 3 (반올림이 아니라 버림)
int(-3.9)         # -3 (0 방향으로 버림)
```

### ⚠️ 흔한 실수 비교

```python
# ❌ 잘못 — 타입 에러
age = "25"
next_year = age + 1
# TypeError: can only concatenate str (not "int") to str

# ✅ 올바름 — 명시적 형변환
age = "25"
next_year = int(age) + 1     # 26
```

```python
# ❌ 잘못 — 사용자 입력은 항상 str
n = input("숫자 입력: ")     # "5"
print(n * 2)                 # "55" (문자열 반복)

# ✅ 올바름
n = int(input("숫자 입력: "))
print(n * 2)                 # 10
```

> 💡 `input()` 은 **항상 문자열을 반환**. 숫자가 필요하면 즉시 `int()` / `float()` 로 감싸세요.

## 4️⃣ 연산자 [↑](#toc)

### 산술 연산자

| 연산자 | 의미 | 예 | 결과 |
|---|---|---|---|
| `+ - * /` | 사칙 | `7 / 2` | `3.5` (항상 float) |
| `//` | 몫 (정수 나눗셈) | `7 // 2` | `3` |
| `%` | 나머지 | `7 % 2` | `1` |
| `**` | 거듭제곱 | `2 ** 10` | `1024` |

> 💡 `7 / 2` 은 `3.5`, `7 // 2` 은 `3`. **`/` 는 항상 실수, `//` 는 몫만**.

### 비교 연산자

`>`, `<`, `>=`, `<=`, `==`, `!=` — 결과는 **항상 `bool`**.

```python
3 < 5          # True
"a" == "a"     # True
[1] == [1]     # True (값 비교)
```

### 논리 연산자 — 단락 평가 (short-circuit)

```python
True and False   # False
True or False    # True
not True         # False
```

**단락 평가**: `and` 는 첫 False에서 멈춤, `or` 는 첫 True에서 멈춤.

```python
# 흔한 패턴: None 체크 후 메서드 호출
name = None
result = name and name.upper()   # name이 None이면 .upper() 호출 안 함
print(result)                    # None (에러 안 남)
```

> 💡 다른 언어의 `&&`, `||` 가 Python에선 `and`, `or`. 영어 단어 그대로.

### 할당 연산자

```python
x = 10
x += 5    # x = x + 5  (15)
x -= 3    # x = x - 3  (12)
x *= 2    # x = x * 2  (24)
x //= 5   # x = x // 5 (4)
```

## 5️⃣ `==` vs `is` (가장 자주 헷갈리는 부분) [↑](#toc)

| 연산자 | 의미 | 비유 |
|---|---|---|
| `==` | **값**이 같은가? | "이름이 같은 사람" |
| `is` | **메모리 주소**가 같은가? | "같은 사람 (동일인)" |

```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)   # True  (값이 같음)
print(a is b)   # False (서로 다른 리스트 객체)
print(a is c)   # True  (같은 객체를 두 이름으로 참조)
```

### 언제 `is` 를 쓰나?

거의 한 가지 경우뿐: **`None` 비교**.

```python
# ✅ 권장
if x is None:
    ...

# ❌ 동작은 하지만 관행 위배
if x == None:
    ...
```

> 💡 외울 규칙: **`is` 는 `None`·`True`·`False` 비교에만 쓰고, 나머지는 `==`**.

### ⚠️ 작은 정수의 함정 (참고만)

```python
a = 100
b = 100
print(a is b)    # True ← 보통 True (작은 정수는 캐싱)

a = 1000
b = 1000
print(a is b)    # False ← 큰 정수는 다를 수 있음
```

이런 동작에 의존하면 안 되니 **숫자 비교는 무조건 `==`**.

## 6️⃣ 자기 점검 5문항 [↑](#toc)

### Q1

다음 코드의 출력은?

```python
x = "10"
y = 3
print(x * y)
```

<details><summary>정답</summary>
`101010`. 문자열 `*` 정수는 반복. (숫자 30을 원했다면 `int(x) * y`)
</details>

### Q2

다음 코드의 출력 두 줄은?

```python
print(7 / 2)
print(7 // 2)
```

<details><summary>정답</summary>
`3.5` / `3`. `/` 는 항상 float, `//` 는 몫.
</details>

### Q3

다음 중 `True` 인 것을 모두 고르세요.

```python
[1, 2] == [1, 2]
[1, 2] is [1, 2]
None == None
None is None
"" == False
```

<details><summary>정답</summary>
`True, False, True, True, False`. 빈 문자열과 False는 둘 다 falsy이지만 값 자체는 다름.
</details>

### Q4

코드를 고치세요.

```python
age = input("나이: ")
if age > 18:
    print("성인")
```

<details><summary>정답</summary>
`input()` 반환은 str이라 정수와 비교 불가.

```python
age = int(input("나이: "))
if age > 18:
    print("성인")
```
</details>

### Q5

다음 코드의 출력은? 왜?

```python
x = None
result = x or "기본값"
print(result)
```

<details><summary>정답</summary>
`기본값`. `or` 는 첫 truthy 값을 반환. `None` 은 falsy → 다음으로 진행 → `"기본값"` 반환.

이 패턴은 "기본값 지정"에 자주 씁니다. (단, `0`이나 `""` 같이 falsy인 정상 값도 무시되므로 주의 — 그 경우 `if x is None`)
</details>
