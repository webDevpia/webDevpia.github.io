# 03장. Path & Query 파라미터

> Path 파라미터는 **건물 층수** 같습니다. "3층 가주세요"라고 말하면 목적지가 명확합니다.
> Query 파라미터는 **검색 필터** 같습니다. "빨간색, L 사이즈만 보여주세요" 처럼 조건을 추가합니다.
> 두 가지를 조합하면 "3층 중에서 빨간색 L 사이즈 상품만" 같은 정밀한 요청이 가능합니다.

<a id="part1"></a>

## 1️⃣ Path 파라미터 [↑](#toc)

URL 경로의 일부를 변수로 받는 방식입니다. `{변수명}` 형태로 URL 안에 직접 포함됩니다.

```
GET /items/5     → item_id = 5
GET /items/42    → item_id = 42
GET /items/999   → item_id = 999
```

### 기본 Path 파라미터

```python
# main.py
from fastapi import FastAPI

app = FastAPI()


@app.get("/items/{item_id}")
def get_item(item_id: int):
    # item_id는 URL에서 자동으로 추출되고, int로 변환됩니다
    return {"item_id": item_id, "message": f"{item_id}번 아이템입니다"}
```

실행 결과 (`GET /items/5`):
```json
{
  "item_id": 5,
  "message": "5번 아이템입니다"
}
```

### 타입 힌트가 하는 일

`item_id: int`라고 선언하면 FastAPI가 자동으로:

1. URL 문자열 `"5"`를 정수 `5`로 변환
2. 변환이 불가능하면 422 에러 반환

```
GET /items/abc   →  422 Unprocessable Entity
GET /items/5.5   →  422 Unprocessable Entity
GET /items/5     →  200 OK
```

### 식당 비유 적용

```python
# 3번 테이블 정보 조회
@app.get("/tables/{table_number}")
def get_table(table_number: int):
    return {"table": table_number, "status": "사용중"}

# 특정 메뉴 조회
@app.get("/menu/{menu_id}")
def get_menu(menu_id: int):
    return {"menu_id": menu_id, "name": f"{menu_id}번 메뉴"}
```

### 문자열 Path 파라미터

숫자뿐 아니라 문자열도 받을 수 있습니다.

```python
@app.get("/users/{username}")
def get_user_profile(username: str):
    # username: "홍길동", "gildong", "user123" 등 모든 문자열 허용
    return {"username": username, "profile_url": f"/users/{username}/profile"}
```

## 2️⃣ 여러 Path 파라미터 [↑](#toc)

> "2번 고객의 3번 주문을 보여주세요" — 두 가지 정보가 URL에 포함됩니다.

```python
@app.get("/users/{user_id}/orders/{order_id}")
def get_user_order(user_id: int, order_id: int):
    return {
        "user_id": user_id,
        "order_id": order_id,
        "message": f"{user_id}번 사용자의 {order_id}번 주문"
    }
```

URL 예시:
```
GET /users/2/orders/3
→ {"user_id": 2, "order_id": 3, "message": "2번 사용자의 3번 주문"}
```

### URL 설계 원칙: 계층 관계 표현

계층 관계가 있는 리소스는 URL에서 중첩으로 표현합니다.

```
/users/2                    → 2번 사용자 정보
/users/2/orders             → 2번 사용자의 전체 주문 목록
/users/2/orders/3           → 2번 사용자의 3번 주문
/users/2/orders/3/items     → 그 주문의 아이템 목록
```

### 고정 경로와 변수 경로의 우선순위

```python
# ⚠️ 순서 주의! 고정 경로를 변수 경로보다 먼저 정의해야 합니다

@app.get("/users/me")        # ✅ 먼저 정의
def get_current_user():
    return {"user": "현재 로그인한 사용자"}


@app.get("/users/{user_id}") # ✅ 나중에 정의
def get_user(user_id: int):
    return {"user_id": user_id}
```

```python
# ❌ 잘못된 순서 — /users/me가 user_id=me로 처리될 수 있습니다
@app.get("/users/{user_id}")  # 이게 먼저 정의되면
def get_user(user_id: int):   # "me"를 int로 변환하려다 422 에러
    return {"user_id": user_id}


@app.get("/users/me")          # 이건 위에서 이미 처리되어버림
def get_current_user():
    return {"user": "현재 로그인한 사용자"}
```

## 3️⃣ Query 파라미터 [↑](#toc)

> "빨간색 L 사이즈 상품을 0번부터 10개만 보여주세요"
> URL로 표현하면: `/items?color=red&size=L&skip=0&limit=10`

URL에서 `?` 뒤에 오는 `key=value` 형식의 파라미터입니다. 여러 개를 `&`로 연결할 수 있습니다.

### 기본 Query 파라미터

```python
@app.get("/items")
def get_items(skip: int, limit: int):
    # skip과 limit은 URL의 ?skip=0&limit=10 에서 자동으로 추출됩니다
    return {
        "skip": skip,
        "limit": limit,
        "message": f"{skip}번부터 {limit}개 조회"
    }
```

테스트:
```
GET /items?skip=0&limit=10
→ {"skip": 0, "limit": 10, "message": "0번부터 10개 조회"}

GET /items?skip=20&limit=5
→ {"skip": 20, "limit": 5, "message": "20번부터 5개 조회"}

GET /items
→ 422 에러 (skip, limit이 필수인데 없음)
```

### Path vs Query — 언제 무엇을 쓰나?

| 상황 | 방식 | 예시 |
|------|------|------|
| 특정 리소스를 식별 | Path | `/items/5` (5번 아이템) |
| 조건/필터/정렬 | Query | `/items?category=food` |
| 페이지네이션 | Query | `/items?page=2&size=20` |
| 검색 | Query | `/search?q=아메리카노` |

## 4️⃣ 선택적 Query 파라미터 [↑](#toc)

> "검색어는 없어도 되고, 있으면 필터링합니다."

기본값을 지정하면 선택적 파라미터가 됩니다. `None`을 기본값으로 하면 없어도 됩니다.

### 기본값 있는 Query 파라미터

```python
@app.get("/items")
def get_items(skip: int = 0, limit: int = 10):
    # skip과 limit은 생략 가능. 생략하면 기본값 사용
    return {
        "skip": skip,
        "limit": limit,
    }
```

테스트:
```
GET /items
→ {"skip": 0, "limit": 10}  # 기본값 사용

GET /items?limit=5
→ {"skip": 0, "limit": 5}   # limit만 지정, skip은 기본값

GET /items?skip=20&limit=5
→ {"skip": 20, "limit": 5}  # 둘 다 지정
```

### None을 기본값으로 — 진짜 선택적 파라미터

```python
@app.get("/items")
def get_items(
    skip: int = 0,
    limit: int = 10,
    q: str | None = None,        # 검색어: 없어도 됨
    category: str | None = None, # 카테고리 필터: 없어도 됨
):
    result: dict = {"skip": skip, "limit": limit}

    if q is not None:
        result["search"] = f"'{q}' 검색 중"

    if category is not None:
        result["filter"] = f"카테고리: {category}"

    return result
```

테스트:
```
GET /items
→ {"skip": 0, "limit": 10}

GET /items?q=커피
→ {"skip": 0, "limit": 10, "search": "'커피' 검색 중"}

GET /items?q=커피&category=음료
→ {"skip": 0, "limit": 10, "search": "'커피' 검색 중", "filter": "카테고리: 음료"}
```

### Python 3.12 타입 힌트 — `X | None` vs `Optional[X]`

```python
# ✅ Python 3.10+ 방식 (이 과정에서 사용)
q: str | None = None

# ✅ 구버전 호환 방식 (같은 의미)
from typing import Optional
q: Optional[str] = None
```

이 과정에서는 Python 3.12를 사용하므로 `str | None` 방식을 사용합니다.

## 5️⃣ Path + Query 조합 [↑](#toc)

> "3번 카테고리에서, 가격 낮은 순으로, 0번부터 10개"
> `/categories/3/items?sort=price&skip=0&limit=10`

실제 API에서는 Path와 Query를 함께 사용하는 경우가 많습니다.

```python
@app.get("/categories/{category_id}/items")
def get_items_by_category(
    category_id: int,      # Path 파라미터 — 어떤 카테고리인지
    skip: int = 0,         # Query 파라미터 — 몇 번째부터
    limit: int = 10,       # Query 파라미터 — 몇 개
    q: str | None = None,  # Query 파라미터 — 검색어 (선택)
    sort: str = "id",      # Query 파라미터 — 정렬 기준
):
    return {
        "category_id": category_id,
        "skip": skip,
        "limit": limit,
        "search": q,
        "sort": sort,
    }
```

테스트:
```
GET /categories/3/items
→ {"category_id": 3, "skip": 0, "limit": 10, "search": null, "sort": "id"}

GET /categories/3/items?q=아메리카노&sort=price&limit=5
→ {"category_id": 3, "skip": 0, "limit": 5, "search": "아메리카노", "sort": "price"}
```

### 실제 프로젝트에서 자주 보는 패턴

```python
# 특정 사용자의 주문 목록 (페이지네이션 포함)
@app.get("/users/{user_id}/orders")
def get_user_orders(
    user_id: int,
    page: int = 1,
    page_size: int = 20,
    status: str | None = None,  # "pending", "completed", "cancelled"
):
    skip = (page - 1) * page_size
    return {
        "user_id": user_id,
        "page": page,
        "page_size": page_size,
        "skip": skip,
        "status_filter": status,
    }
```

## 6️⃣ 타입 검증 자동화 [↑](#toc)

> "주방에 이상한 주문이 들어가면 사고가 납니다."
> FastAPI는 웨이터 역할을 하면서 이상한 주문을 자동으로 걸러냅니다.

FastAPI는 타입 힌트를 기반으로 입력값을 자동 검증합니다. 잘못된 타입이 들어오면 422 에러를 자동으로 반환합니다.

### 잘못된 타입 보내기

```python
@app.get("/items/{item_id}")
def get_item(item_id: int):
    return {"item_id": item_id}
```

```
# ❌ 문자열로 보내면?
GET /items/abc

# 자동 응답:
HTTP 422 Unprocessable Entity
{
  "detail": [
    {
      "type": "int_parsing",
      "loc": ["path", "item_id"],
      "msg": "Input should be a valid integer, unable to parse string as an integer",
      "input": "abc"
    }
  ]
}
```

### 검증이 일어나는 위치 확인

```
"loc": ["path", "item_id"]   → Path 파라미터에서 에러
"loc": ["query", "limit"]    → Query 파라미터에서 에러
"loc": ["body", "price"]     → 요청 바디에서 에러 (다음 장에서)
```

### 직접 검증하지 않아도 됩니다

```python
# ❌ 이렇게 직접 검증할 필요가 없습니다
@app.get("/items/{item_id}")
def get_item(item_id: str):
    try:
        id_int = int(item_id)
    except ValueError:
        return {"error": "숫자를 입력해주세요"}
    return {"item_id": id_int}


# ✅ 타입 힌트만 써주면 FastAPI가 자동으로 처리합니다
@app.get("/items/{item_id}")
def get_item(item_id: int):
    return {"item_id": item_id}
```

### 숫자 범위 제약 — Query 파라미터에도 적용

```python
from fastapi import FastAPI, Query

app = FastAPI()


@app.get("/items")
def get_items(
    limit: int = Query(default=10, ge=1, le=100),  # 1 이상, 100 이하
    skip: int = Query(default=0, ge=0),             # 0 이상
):
    return {"limit": limit, "skip": skip}
```

테스트:
```
GET /items?limit=0    → 422 에러 (ge=1 위반)
GET /items?limit=200  → 422 에러 (le=100 위반)
GET /items?limit=50   → 200 OK
```

## 7️⃣ Swagger에서 파라미터 확인 [↑](#toc)

**Swagger UI에서 확인해보세요!** 파라미터를 정의하면 `/docs`에서 자동으로 입력 폼이 생성됩니다.

아래 코드를 `main.py`에 작성하고 `/docs`를 열어봅니다.

```python
# main.py
from fastapi import FastAPI, Query

app = FastAPI()


@app.get("/search")
def search_items(
    q: str = Query(description="검색어를 입력하세요"),
    category: str | None = Query(default=None, description="카테고리 필터"),
    min_price: float | None = Query(default=None, ge=0, description="최소 가격"),
    max_price: float | None = Query(default=None, ge=0, description="최대 가격"),
    sort: str = Query(default="name", description="정렬 기준: name, price, date"),
    skip: int = Query(default=0, ge=0),
    limit: int = Query(default=10, ge=1, le=100),
):
    return {
        "q": q,
        "category": category,
        "price_range": [min_price, max_price],
        "sort": sort,
        "pagination": {"skip": skip, "limit": limit},
    }
```

`/docs`에서 `GET /search` 엔드포인트를 클릭하면:
- 각 파라미터의 이름, 타입, 필수/선택 여부가 표시됩니다
- `description`에 입력한 설명이 문서에 그대로 나타납니다
- "Try it out"으로 직접 값을 넣고 테스트할 수 있습니다

## 8️⃣ 정리 [↑](#toc)

### 핵심 개념 요약

| 개념 | 선언 방법 | URL 예시 | 설명 |
|------|----------|---------|------|
| Path 파라미터 (필수) | `{item_id}` + 함수 인자 | `/items/5` | URL 경로에 포함, 필수값 |
| Query 파라미터 (필수) | 기본값 없는 함수 인자 | `?skip=0` | `?` 뒤에 오는 값, 필수 |
| Query 파라미터 (선택) | `= 기본값` | `?q=검색어` | 없으면 기본값 사용 |
| Optional Query | `str \| None = None` | `?q=검색어` | 없어도 되는 파라미터 |
| Query + 제약 | `Query(ge=0, le=100)` | `?limit=50` | 범위 검증 포함 |
| 자동 타입 변환 | 타입 힌트 | `/items/5` → `int` | URL 문자열을 지정 타입으로 변환 |
| 자동 에러 응답 | FastAPI 내장 | 잘못된 값 → 422 | 검증 실패 시 자동 에러 |
