# 14장. 인증 — OAuth2 + JWT

> "JWT = 콘서트 팔찌 — 입장 시 한 번 받으면 공연 내내 유효합니다. 위조가 어렵고, 공연이 끝나면 만료됩니다."

지금까지 만든 ToDo API는 누구나 접근할 수 있었습니다. 실제 서비스라면 "내 할 일은 나만 볼 수 있어야" 합니다. 이번 장에서는 **로그인**, **JWT 토큰**, **보호된 라우트**를 구현합니다.

<a id="1"></a>

## 1️⃣ 인증이란? [↑](#toc)

### 인증 vs 인가 — 문 앞의 비유

> 클럽에 입장할 때를 생각해보세요.
> - **인증(Authentication)**: 신분증을 보여줍니다. "당신이 누구인지 확인합니다."
> - **인가(Authorization)**: VIP 팔찌를 확인합니다. "당신이 무엇을 할 수 있는지 결정합니다."

```
사용자 → 신분증 제시 (로그인) → 인증 통과 → 팔찌 발급 (토큰)
                                              ↓
                              팔찌 제시 (토큰 첨부) → 인가 확인 → VIP 입장
```

API에서는:
- **인증**: 아이디+비밀번호로 로그인 → JWT 토큰 발급
- **인가**: 매 요청마다 JWT 토큰을 Header에 담아 전송 → 서버가 검증

### 이번 장에서 만드는 흐름

```
POST /auth/register  → 회원가입 (비밀번호 해싱 후 저장)
POST /auth/token     → 로그인 (비밀번호 검증 → JWT 발급)
GET  /todos          → 보호된 라우트 (JWT 검증 → 내 할 일만 반환)
```

## 2️⃣ 비밀번호 해싱 [↑](#toc)

### 절대 하면 안 되는 것

# ❌ 절대 안 됩니다 — 평문 비밀번호 저장
```python
user.password = "비밀번호123"  # DB에 그대로 저장 — 해킹 시 즉시 노출
```

### 해싱이란?

> 해싱 = 고기를 분쇄기에 넣는 것. 결과물(해시)로 원래 고기를 복원할 수 없습니다.
> 비밀번호를 해시로 변환하면, DB가 유출되어도 원래 비밀번호를 알 수 없습니다.

# ✅ 올바른 방법 — 해시 저장
```python
hashed = password_hash.hash("비밀번호123")
# DB에는 "$argon2id$v=19$m=65536,t=2,p=1$..." 이런 문자열이 저장됩니다
```

### pwdlib 설치 및 사용

```bash
uv add "pwdlib[argon2]"
```

```python
from pwdlib import PasswordHash

# 앱 전역에서 한 번만 초기화
password_hash = PasswordHash.recommended()

# 비밀번호 해싱
hashed = password_hash.hash("비밀번호123")
print(hashed)
# $argon2id$v=19$m=65536,t=2,p=1$xxxx...

# 비밀번호 검증
is_valid = password_hash.verify("비밀번호123", hashed)
print(is_valid)  # True

is_valid_wrong = password_hash.verify("틀린비밀번호", hashed)
print(is_valid_wrong)  # False
```

실행 결과:
```
$argon2id$v=19$m=65536,t=2,p=1$...
True
False
```

> `PasswordHash.recommended()`는 현재 가장 안전한 알고리즘(Argon2id)을 자동으로 선택합니다.
> 예전 강의에서 많이 보이는 `passlib`은 **유지보수가 중단**되었습니다. 반드시 `pwdlib`을 사용하세요.

## 3️⃣ JWT 토큰이란? [↑](#toc)

### JWT = JSON Web Token

JWT는 세 부분으로 구성된 문자열입니다:

```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ1c2VyQGV4YW1wbGUuY29tIiwiZXhwIjoxNzE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
     ↑ Header               ↑ Payload                                          ↑ Signature
```

| 부분 | 내용 | 역할 |
|------|------|------|
| **Header** | `{"alg": "HS256", "typ": "JWT"}` | 알고리즘 명시 |
| **Payload** | `{"sub": "user@example.com", "exp": 1716239022}` | 사용자 정보, 만료 시간 |
| **Signature** | `HMACSHA256(header + payload, SECRET_KEY)` | 위조 방지 서명 |

> 팔찌에 비유하면:
> - Header = 팔찌 재질 ("이것은 공식 팔찌입니다")
> - Payload = 팔찌에 적힌 정보 (이름, 날짜, VIP 등급)
> - Signature = 위조 방지 홀로그램 (비밀 열쇠로만 만들 수 있습니다)

### JWT의 특징

- **Stateless**: 서버가 토큰 내용을 DB에 저장하지 않아도 됩니다
- **Self-contained**: 필요한 정보(사용자 ID, 만료 시간)가 토큰 안에 있습니다
- **위조 불가**: SECRET_KEY 없이는 유효한 서명을 만들 수 없습니다
- **만료 가능**: `exp` 클레임으로 만료 시간을 지정합니다

## 4️⃣ JWT 생성과 검증 [↑](#toc)

### 설치

```bash
uv add PyJWT
```

### 토큰 생성

```python
import jwt
from datetime import datetime, timedelta, timezone

# 실제로는 환경변수에서 읽어야 합니다 — 아래 섹션에서 다룹니다
SECRET_KEY = "임시-비밀-키"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30


def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    encoded_jwt = jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
    return encoded_jwt
```

### 토큰 검증

```python
from jwt.exceptions import InvalidTokenError


def verify_token(token: str) -> dict:
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        return payload
    except InvalidTokenError:
        return {}
```

### 직접 실행해보기

```python
# 토큰 생성
token = create_access_token({"sub": "user@example.com"})
print(token)

# 토큰 검증
payload = verify_token(token)
print(payload)
# {'sub': 'user@example.com', 'exp': 1716239022}

# 잘못된 토큰
bad_payload = verify_token("이것은.잘못된.토큰")
print(bad_payload)
# {}
```

실행 결과:
```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ1c2VyQGV4YW1wbGUuY29tIiwiZXhwIjoxN...
{'sub': 'user@example.com', 'exp': 1716239022}
{}
```

### 프로젝트 구조

13장까지 만든 ToDo 프로젝트에 인증을 추가합니다:

app/

### User 모델 (`app/models/user.py`)

```python
from sqlmodel import SQLModel, Field
from typing import Optional


class User(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    email: str = Field(unique=True, index=True)
    hashed_password: str
    is_active: bool = Field(default=True)
```

### User 스키마 (`app/schemas/user.py`)

```python
from pydantic import BaseModel, EmailStr, ConfigDict


class UserCreate(BaseModel):
    email: EmailStr
    password: str


class UserResponse(BaseModel):
    id: int
    email: EmailStr
    is_active: bool

    model_config = ConfigDict(from_attributes=True)


class Token(BaseModel):
    access_token: str
    token_type: str = "bearer"
```

## 6️⃣ 회원가입 엔드포인트 [↑](#toc)

### `app/routers/auth.py` 작성

```python
from fastapi import APIRouter, HTTPException, Depends, status
from sqlmodel import Session, select
from pwdlib import PasswordHash

from app.database import get_session
from app.models.user import User
from app.schemas.user import UserCreate, UserResponse

router = APIRouter(prefix="/auth", tags=["인증"])

password_hash = PasswordHash.recommended()


@router.post(
    "/register",
    response_model=UserResponse,
    status_code=status.HTTP_201_CREATED,
)
def register(user_data: UserCreate, session: Session = Depends(get_session)):
    # 이메일 중복 확인
    existing_user = session.exec(
        select(User).where(User.email == user_data.email)
    ).first()

    if existing_user:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="이미 등록된 이메일입니다.",
        )

    # 비밀번호 해싱 후 저장
    hashed = password_hash.hash(user_data.password)
    new_user = User(email=user_data.email, hashed_password=hashed)

    session.add(new_user)
    session.commit()
    session.refresh(new_user)
    return new_user
```

### Swagger에서 테스트

```json
POST /auth/register
{
  "email": "test@example.com",
  "password": "비밀번호123"
}
```

실행 결과:
```json
{
  "id": 1,
  "email": "test@example.com",
  "is_active": true
}
```

> 응답에 `hashed_password`가 없는 것을 확인하세요. `UserResponse` 스키마 덕분에 민감한 필드가 자동으로 제외됩니다.

## 7️⃣ 로그인 엔드포인트 [↑](#toc)

### OAuth2PasswordRequestForm

FastAPI는 OAuth2 표준을 지원하는 `OAuth2PasswordRequestForm`을 내장하고 있습니다. Swagger UI의 "Authorize" 버튼과 연동됩니다.

```python
from fastapi.security import OAuth2PasswordRequestForm
import jwt
from datetime import datetime, timedelta, timezone
from jwt.exceptions import InvalidTokenError

from app.schemas.user import Token

SECRET_KEY = "임시-비밀-키"  # 곧 환경변수로 이동합니다
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30


def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


@router.post("/token", response_model=Token)
def login(
    form_data: OAuth2PasswordRequestForm = Depends(),
    session: Session = Depends(get_session),
):
    # 사용자 조회
    user = session.exec(
        select(User).where(User.email == form_data.username)
    ).first()

    if not user:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="이메일 또는 비밀번호가 올바르지 않습니다.",
            headers={"WWW-Authenticate": "Bearer"},
        )

    # 비밀번호 검증
    if not password_hash.verify(form_data.password, user.hashed_password):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="이메일 또는 비밀번호가 올바르지 않습니다.",
            headers={"WWW-Authenticate": "Bearer"},
        )

    # JWT 발급
    access_token = create_access_token(data={"sub": user.email})
    return Token(access_token=access_token)
```

실행 결과:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ0ZXN0QGV4YW1wbGUuY29tIiwiZXhwIjoxNzE2MjM5MDIyfQ...",
  "token_type": "bearer"
}
```

> 로그인 실패 메시지를 일부러 모호하게 적었습니다 ("이메일 또는 비밀번호가 올바르지 않습니다").
> "이메일이 존재하지 않습니다"처럼 구체적으로 적으면 공격자가 어떤 이메일이 등록됐는지 알 수 있습니다.

## 8️⃣ 보호된 라우트 [↑](#toc)

### get_current_user 의존성 함수

```python
from fastapi.security import OAuth2PasswordBearer

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")


def get_current_user(
    token: str = Depends(oauth2_scheme),
    session: Session = Depends(get_session),
) -> User:
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="인증 정보를 확인할 수 없습니다.",
        headers={"WWW-Authenticate": "Bearer"},
    )

    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        email: str | None = payload.get("sub")
        if email is None:
            raise credentials_exception
    except InvalidTokenError:
        raise credentials_exception

    user = session.exec(select(User).where(User.email == email)).first()
    if user is None:
        raise credentials_exception

    return user
```

### 보호된 엔드포인트 적용

기존 `app/routers/todos.py`에 인증을 추가합니다:

```python
from app.routers.auth import get_current_user
from app.models.user import User


@router.get("/", response_model=list[TodoResponse])
def read_todos(
    session: Session = Depends(get_session),
    current_user: User = Depends(get_current_user),  # 인증 추가
):
    todos = session.exec(select(Todo)).all()
    return todos


@router.post("/", response_model=TodoResponse, status_code=201)
def create_todo(
    todo: TodoCreate,
    session: Session = Depends(get_session),
    current_user: User = Depends(get_current_user),  # 인증 추가
):
    new_todo = Todo(**todo.model_dump())
    session.add(new_todo)
    session.commit()
    session.refresh(new_todo)
    return new_todo
```

### main.py에 라우터 등록

```python
from app.routers import todos, auth

app.include_router(todos.router)
app.include_router(auth.router)
```

## 9️⃣ Swagger에서 인증 테스트 [↑](#toc)

FastAPI는 JWT 인증을 자동으로 Swagger UI에 통합합니다.

### 단계별 테스트

**1단계 — 회원가입**

`POST /auth/register`에서 이메일과 비밀번호를 입력하여 계정을 생성합니다.

**2단계 — 로그인 (Authorize)**

Swagger UI 오른쪽 상단의 **"Authorize 🔓"** 버튼을 클릭합니다.

```
Username: test@example.com
Password: 비밀번호123
```

**"Authorize"** 버튼을 클릭하면 내부적으로 `POST /auth/token`이 호출되고,
발급된 JWT가 이후 모든 요청의 `Authorization: Bearer <token>` 헤더에 자동으로 첨부됩니다.

**3단계 — 보호된 엔드포인트 테스트**

자물쇠 아이콘(🔒)이 닫힌 상태에서 `GET /todos`를 호출하면 정상 응답이 옵니다.

> Swagger UI의 자물쇠 아이콘은 OAuth2PasswordBearer를 선언하는 것만으로 자동 생성됩니다.
> 별도 설정이 필요하지 않습니다.

## 🔟 보안 주의사항 [↑](#toc)

### 1. SECRET_KEY 절대 하드코딩 금지

# ❌ 절대 안 됩니다 — 코드에 직접 작성
```python
SECRET_KEY = "my-super-secret-key-12345"
```

이 코드가 GitHub에 올라가면 누구나 볼 수 있습니다.

# ✅ 환경변수에서 읽기 (다음 장에서 상세히 다룹니다)
```python
import os
SECRET_KEY = os.environ["SECRET_KEY"]  # 최소한 이렇게
```

15장에서 `pydantic-settings`를 사용한 더 체계적인 방법을 배웁니다.

### 2. 강력한 SECRET_KEY 생성

```bash
# 터미널에서 실행
openssl rand -hex 32
# a3f8c2e1d4b7a6f9e2c5d8b1a4f7c3e6d9b2a5f8c1e4d7b0a3f6c9e2d5b8a1f4
```

이렇게 생성된 랜덤 문자열을 `.env` 파일에 저장하세요.

### 3. HTTPS 사용 (프로덕션)

JWT 토큰이 HTTP로 전송되면 중간에서 가로챌 수 있습니다. 프로덕션 환경에서는 반드시 HTTPS를 사용하세요.

### 4. 토큰 만료 시간 설정

```python
# ❌ 만료 없음 — 토큰이 유출되면 영구적으로 사용 가능
to_encode = {"sub": email}

# ✅ 만료 시간 설정
expire = datetime.now(timezone.utc) + timedelta(minutes=30)
to_encode = {"sub": email, "exp": expire}
```

### 5. SQL 인젝션 방지 — f-string 절대 사용 금지

# ❌ SQL 인젝션 취약점
```python
# 이렇게 하면 악의적인 사용자가 DB를 마음대로 조작할 수 있습니다
query = f"SELECT * FROM user WHERE email = '{user_input}'"
session.exec(query)
```

# ✅ SQLModel이 자동으로 파라미터 바인딩 처리
```python
user = session.exec(
    select(User).where(User.email == user_input)
).first()
```

SQLModel과 SQLAlchemy는 `where(User.email == user_input)` 구문을 내부적으로 파라미터 바인딩으로 변환합니다. f-string으로 직접 SQL을 만드는 방식은 절대 사용하지 마세요.
