## 2. 즉시 수정 필요 — Critical (Tailwind)

### C1. CDN URL 불일치 — 02장이 Tailwind v3를 로드

02_Tailwind_핵심패턴.md의 모든 실습 코드가 `https://cdn.tailwindcss.com` (v3)을 사용합니다. 01장과 03장은 `https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4` (v4)를 올바르게 사용합니다.

```
01장: v4 CDN ✅
02장: v3 CDN ❌  ← 학생이 따라치면 다른 버전 로드
03장: v4 CDN ✅
```

> **위험:** 학생이 02장 코드를 복사-붙여넣기하면 v3가 로드되어, v4 전용 기능 사용 시 디버깅 불가

**수정:** 02장의 모든 `<script src="https://cdn.tailwindcss.com">` → `<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4">`

### C2. 01장 "다음 장 미리보기"가 02장과 불일치

01장 끝: *"02장에서는 **반응형 디자인**을 다룹니다"*  
실제 02장: Position, Z-index, Group Hover, Transition (반응형 없음)  
반응형은 **03장**(프로젝트)에서 등장

> **위험:** 학생이 02장을 열면 예고와 완전히 다른 내용 → "파일을 잘못 열었나?" 혼란

**수정:** 01장의 "다음 장 미리보기"를 02장 실제 내용(Position, Group Hover, Transition)으로 교체

### C3. 가짜 CSS 속성 `ring`

01_Tailwind_시작하기.md의 CSS 대응 코드에 `ring: 2px solid #3b82f6;`이 있습니다. **CSS에 `ring` 속성은 존재하지 않습니다.** Tailwind의 `ring`은 `box-shadow`로 구현됩니다.

> **위험:** CSS 기초를 막 배운 학생이 `ring`을 실제 CSS 속성으로 오인

**수정:**
```css
/* 수정 전 (잘못된 코드) */
ring: 2px solid #3b82f6;

/* 수정 후 */
box-shadow: 0 0 0 2px #3b82f6;
```

## 5. 다중 페르소나 검토

### 페르소나 A: 비전공자 수강생

> *"코드를 따라 치는데 처음 보는 속성이 나오면 당황해요."*

- CSS 02장 카드 예제의 `transition: transform 0.2s`를 타이핑하면서 "이게 뭔가요?" 질문 확실
- CSS 03장 반응형 포트폴리오의 `position: sticky`도 마찬가지
- **핵심:** 예제에 사용된 모든 속성은 최소한 1줄 설명이 필요
- Tailwind 01장의 CSS-Tailwind 대응표는 매우 유용 — "CSS에서 이렇게 쓰던 걸 Tailwind에서는 이렇게" 가 직관적

### 페르소나 B: 프론트엔드 개발자

> *"CDN 버전 불일치는 수업 현장에서 '왜 내 코드만 안 되죠?'를 유발합니다."*

- Tailwind v3과 v4는 대부분 호환되지만, v4 전용 기능(color-mix, 4자리 opacity 등) 사용 시 즉시 깨짐
- `ring`이 CSS 속성이라고 가르치면 나중에 순수 CSS 프로젝트에서 혼란
- CSS Grid를 CSS 과정에서 안 다루고 Tailwind 03장에서 `grid grid-cols-3`가 갑자기 등장 — CSS 기초 지식 없이 유틸리티 클래스만 따라치게 됨

### 페르소나 C: 교육공학 전문가

> *"선수지식 ↔ 본문 ↔ 다음 장 안내가 정합하지 않으면 학습 흐름이 깨집니다."*

- 01장 "다음 장 미리보기"가 실제 02장과 불일치하는 것은 교재 구조적 결함
- 02장 사전준비에 "반응형 접두어를 이해한 상태"라고 했는데 01장에서 안 다룸 → 선수지식 갭
- **강점:** CSS ↔ Tailwind 대응표, 비유(레고, 물감 농도), 3단계 실습(기본/중급/심화)은 우수

### 페르소나 D: 강사 (대면 수업 운영)

> *"02장 핵심패턴(Group Hover, Peer, Animation)을 비전공자가 2시간에 소화하는 건 과부하입니다."*

- 01장(830줄) + 02장(680줄) + 03장(940줄) = 총 2,450줄 — 2교시(100분)에 전부 소화 불가
- **현실적 운영:** 01장(대응표)은 참고자료, 02장(핵심패턴)은 선택적, 03장(프로젝트)이 핵심 실습
- CSS 과정도 마찬가지: 01장 정리표에 예상 소요 시간이 없어 시간 배분 판단 어려움

### 페르소나 E: QA/테스터

> *"코드를 실제로 실행해봤을 때 깨지는 부분이 있습니다."*

- `ring: 2px solid #3b82f6;` → 브라우저에서 무시됨 (존재하지 않는 속성)
- `display: hidden` → 브라우저에서 무시됨 (`display: none`이 올바름)
- 02장 v3 CDN으로 로드 시 v4 전용 클래스 미작동 가능
- 03장 단계별 코드의 `hover:-translate-y-0.5`가 전체 코드에서 누락 → 학생이 비교하면 차이 발견

## 9. 결론

**CSS 과정**: 구조와 내용이 전반적으로 우수합니다. position/transition을 예제에서 사용하면서 설명하지 않는 문제만 보완하면 슬라이드 제작에 충분합니다.

**Tailwind 과정**: CDN 버전 불일치, 가짜 CSS 속성, 챕터 간 네비게이션 오류 등 **기술적 정확성 이슈 3건**을 반드시 수정해야 합니다. 내용 자체의 교육적 품질은 우수하므로, 정합성 수정 후 슬라이드 제작에 사용 가능합니다.

> **작성 방법론**: 5개 독립 페르소나(비전공자 수강생, 프론트엔드 개발자, 교육공학 전문가, 대면 수업 강사, QA 테스터)가 각자의 관점에서 6개 교재 파일을 독립 분석 후, 교차 합의를 통해 수정 우선순위를 도출하였습니다.
