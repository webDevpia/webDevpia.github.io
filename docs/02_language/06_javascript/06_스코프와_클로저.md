# 06장. 스코프와 클로저

<a id="part1"></a>

## 1️⃣ 스코프(Scope)란? [↑](#toc)

**스코프(Scope)**란 변수에 접근할 수 있는 **유효 범위**입니다.
변수가 어디서 선언되었느냐에 따라 사용 가능한 위치가 결정됩니다.

### 건물 비유

> **1층 로비에 놓인 물건**은 건물 어디서나 접근할 수 있습니다.
> 하지만 **3층 사무실 안의 물건**은 그 방에 들어온 사람만 사용할 수 있습니다.
> 스코프는 "변수에 접근할 수 있는 구역"입니다.

```javascript
// 전역 스코프 — 코드 어디서든 접근 가능
let buildingName = "청사빌딩";

function office3F() {
    // 지역(함수) 스코프 — 이 함수 안에서만 접근 가능
    let printer = "레이저프린터";
    console.log(buildingName);  // 로비 물건: 접근 가능
    console.log(printer);       // 같은 방 물건: 접근 가능
}

office3F();
console.log(buildingName);  // 전역 변수: 접근 가능
console.log(printer);       // ❌ ReferenceError: printer is not defined
```

실행 결과:
```
청사빌딩
레이저프린터
청사빌딩
ReferenceError: printer is not defined
```

### let/const(블록 스코프) vs var(함수 스코프)

2장에서 배운 `let`/`const`는 `{ }` 블록 안에서만 유효합니다. `var`는 블록을 무시하고 함수 전체에서 유효합니다.

```javascript
if (true) {
    let blockLet = "블록 안에서만";
    var funcVar  = "함수 전체에서";
}
// console.log(blockLet);  // ❌ ReferenceError
console.log(funcVar);      // "함수 전체에서" — var는 블록 밖에서도 접근 가능

// 같은 이름의 변수가 다른 스코프에 공존
let name = "전역";
function introduce() {
    let name = "지역";     // 전역과 이름 같지만 별개의 변수
    console.log(name);    // 가장 가까운 스코프 우선
}
introduce();
console.log(name);
```

실행 결과:
```
함수 전체에서
지역
전역
```

## 2️⃣ 스코프 체인 [↑](#toc)

**스코프 체인(Scope Chain)**은 변수를 찾을 때 **안쪽 → 바깥쪽** 순서로 탐색하는 규칙입니다.

### 러시아 인형(마트료시카) 비유

> 안쪽 인형은 자신을 감싼 바깥 인형들을 모두 볼 수 있습니다.
> 하지만 **바깥 인형은 안쪽 인형을 볼 수 없습니다.**
> 스코프 체인도 안쪽 함수는 바깥 변수에 접근할 수 있지만, 반대는 불가능합니다.

```javascript
let level1 = "전역";

function outer() {
    let level2 = "outer 함수";

    function inner() {
        let level3 = "inner 함수";
        // 안쪽 함수는 바깥 모든 변수에 접근 가능
        console.log(level3);  // 자신의 스코프
        console.log(level2);  // 바로 바깥 스코프
        console.log(level1);  // 전역 스코프
    }

    inner();
    // console.log(level3);  // ❌ 안쪽 변수는 접근 불가
}

outer();
```

실행 결과:
```
inner 함수
outer 함수
전역
```

변수를 찾는 순서: **현재 스코프 → 바깥 스코프 → … → 전역 스코프 → ReferenceError**

## 3️⃣ 호이스팅(Hoisting) [↑](#toc)

**호이스팅(Hoisting)**이란 JavaScript가 실행 전에 변수·함수 **선언을 스코프 맨 위로 끌어올리는** 동작입니다.

### 출석부 비유

> 선생님이 수업 전에 **출석부를 미리 작성**하는 것과 같습니다.
> 이름(선언)은 미리 적히지만, 실제 학생이 착석(값 할당)하는 것은 나중입니다.

```javascript
// var — 선언만 위로, 값은 undefined
console.log(score);  // undefined (선언은 올라갔지만 값은 없음)
var score = 100;
console.log(score);  // 100
```

```javascript
// let — TDZ(일시적 사각지대) 적용, 선언 전 접근 시 오류
console.log(myVar);  // ❌ ReferenceError: 초기화 전 접근 불가
let myVar = "값";
```

```javascript
// 함수 선언식 — 전체가 호이스팅 → 선언 전 호출 가능
sayHello();                       // 정상 동작
function sayHello() { console.log("안녕!"); }

// 함수 표현식 — 변수만 호이스팅 → 선언 전 호출 불가
// sayBye();                       // ❌ TypeError
const sayBye = function() { console.log("잘 가!"); };
sayBye();
```

실행 결과:
```
undefined
100
안녕!
잘 가!
```

| 선언 방식 | 호이스팅 결과 | 선언 전 사용 |
|-----------|--------------|-------------|
| `var` | 선언 + `undefined` 초기화 | 가능 (`undefined`) |
| `let`/`const` | 선언만 (TDZ 적용) | 불가 (ReferenceError) |
| 함수 선언식 | 선언 + 함수 전체 | 가능 |
| 함수 표현식 | 변수 선언만 | 불가 (TypeError) |

## 4️⃣ 클로저(Closure) [↑](#toc)

**클로저(Closure)**란 함수가 **자신이 생성될 때의 외부 변수를 기억**하는 현상입니다.
함수가 어디서 실행되든, 태어난 환경의 변수에 계속 접근할 수 있습니다.

### 배낭 비유

> 함수가 만들어질 때 주변 변수들을 **배낭에 챙겨** 넣습니다.
> 이 함수가 나중에 어디서 실행되든, 배낭 속 변수를 꺼내 쓸 수 있습니다.

```javascript
function makeGreeter(name) {
    // name이 배낭에 담김
    return function greet() {
        console.log(`안녕하세요, ${name}님!`);  // makeGreeter 종료 후에도 name 기억
    };
}

const greetAlice = makeGreeter("앨리스");
const greetBob   = makeGreeter("밥");

greetAlice();  // makeGreeter는 이미 종료됐지만 name을 기억
greetBob();
```

실행 결과:
```
안녕하세요, 앨리스님!
안녕하세요, 밥님!
```

### 데이터 은닉(private 변수)

클로저를 사용하면 외부에서 직접 접근할 수 없는 **private 변수**를 만들 수 있습니다.

```javascript
function createCounter() {
    let count = 0;          // 외부에서 직접 수정 불가

    return {
        increment() { count++; console.log(`카운트: ${count}`); },
        decrement() { count--; console.log(`카운트: ${count}`); },
        getCount()  { return count; }
    };
}

const counter = createCounter();
counter.increment();
counter.increment();
counter.decrement();
// console.log(count);  // ❌ ReferenceError — 직접 접근 불가
```

실행 결과:
```
카운트: 1
카운트: 2
카운트: 1
```

## 5️⃣ 클로저 활용 패턴 [↑](#toc)

### 패턴 1 — 함수 팩토리 (설정을 기억하는 함수 생성)

```javascript
function createMultiplier(factor) {
    return function(number) {
        return number * factor;  // 배낭 속 factor를 기억
    };
}

const double   = createMultiplier(2);
const triple   = createMultiplier(3);
const tenTimes = createMultiplier(10);

console.log(double(5));    // 10
console.log(triple(5));    // 15
console.log(tenTimes(7));  // 70
```

실행 결과:
```
10
15
70
```

### 패턴 2 — 모듈 패턴 (private/public 구분)

```javascript
const userManager = (function() {
    let users  = [];   // private
    let nextId = 1;    // private

    return {
        addUser(name)  { users.push({ id: nextId++, name }); console.log(`추가: ${name}`); },
        getUsers()     { return [...users]; },  // 원본 대신 복사본 반환
        count()        { return users.length; }
    };
})();  // 즉시 실행 함수(IIFE)

userManager.addUser("김철수");
userManager.addUser("이영희");
console.log("사용자 수:", userManager.count());
```

실행 결과:
```
추가: 김철수
추가: 이영희
사용자 수: 2
```

### ⚠️ 주의 — 반복문 안의 클로저 문제

```javascript
// ❌ var — 모든 함수가 같은 i를 공유 (마지막 값 3)
const funcs1 = [];
for (var i = 0; i < 3; i++) {
    funcs1.push(function() { console.log(i); });
}
funcs1[0]();  // 3 (예상: 0)
funcs1[1]();  // 3 (예상: 1)

// ✅ let — 반복마다 독립적인 스코프 생성
const funcs2 = [];
for (let j = 0; j < 3; j++) {
    funcs2.push(function() { console.log(j); });
}
funcs2[0]();  // 0
funcs2[1]();  // 1
```

실행 결과:
```
3
3
0
1
```

> `var`는 함수 스코프이므로 반복문의 모든 클로저가 **동일한 `i`를 공유**합니다.
> `let`은 블록 스코프이므로 **각 반복마다 새로운 변수**가 생성됩니다.

## 6️⃣ 정리 [↑](#toc)

### 핵심 개념 요약

| 개념 | 설명 | 핵심 포인트 |
|------|------|------------|
| **전역 스코프** | 코드 전체에서 접근 가능 | 어디서든 사용 가능 |
| **지역 스코프** | 함수 내부에서만 접근 가능 | 함수 밖에서 접근 불가 |
| **블록 스코프** | `{ }` 안에서만 유효 (`let`/`const`) | `var`는 블록 무시 |
| **스코프 체인** | 안 → 밖 순서로 변수 탐색 | 바깥 변수 접근 가능 |
| **호이스팅** | 선언이 스코프 맨 위로 이동 | 값은 원래 위치에서 할당 |
| **TDZ** | `let`/`const` 선언 전 접근 금지 구간 | 안전한 코드 작성 |
| **클로저** | 함수가 외부 변수를 기억하는 현상 | 데이터 은닉, 상태 유지 |

### 실습 과제

**기본** — 아래 코드의 출력 결과를 예측한 후 직접 실행하여 확인하세요.

```javascript
let x = "전역";
function outer() {
    let x = "outer";
    function inner() { console.log(x); }
    inner();
}
outer();
console.log(x);
```

**중급** — 주어진 배수를 곱하는 함수를 반환하는 `createMultiplier(factor)`를 구현하세요.

```javascript
function createMultiplier(factor) { /* 여기를 채우세요 */ }

const double = createMultiplier(2);
console.log(double(4));   // 8
console.log(double(10));  // 20
```

**심화** — `deposit`(입금), `withdraw`(출금), `getBalance`(잔액 조회)를 반환하는 클로저 기반 은행 계좌를 구현하세요. `balance`는 외부에서 직접 접근할 수 없어야 하며, 잔액이 부족한 경우 경고를 출력합니다.

```javascript
function createBankAccount(initialBalance = 0) { /* 여기를 채우세요 */ }

const account = createBankAccount(1000);
account.deposit(500);      // 잔액: 1500원
account.withdraw(200);     // 잔액: 1300원
account.withdraw(9999);    // ⚠️ 잔액 부족
console.log(account.getBalance());  // 1300
```
