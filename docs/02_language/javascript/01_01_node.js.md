# 📌 Node.js 

## Node.js란?
- **정의:**  
  - Chrome V8 엔진을 기반으로 만든 **자바스크립트 런타임 환경**
  - 브라우저 밖에서 자바스크립트를 실행할 수 있도록 만든 프로그램
- **특징:**  
  - 싱글 스레드, 이벤트 기반, 논블로킹 I/O  
  - 서버, CLI 툴, 자동화 스크립트 등 다양한 용도로 사용 가능

---

## Node.js의 주요 특징

| 특징 | 설명 |
|------|------|
| **비동기 I/O** | 요청이 완료될 때까지 기다리지 않고 다음 코드 실행 (논블로킹) |
| **싱글 스레드** | 한 개의 스레드로 요청 처리 → 이벤트 루프로 동시성 지원 |
| **이벤트 루프** | 콜백, Promise, async/await을 통해 비동기 작업 관리 |
| **크로스 플랫폼** | Windows, macOS, Linux에서 동일하게 작동 |
| **npm 지원** | 전 세계 개발자들이 만든 오픈소스 패키지 200만+ 개 사용 가능 |

---

## Node.js의 구성 요소

| 구성요소 | 설명 |
|------|------|
| **V8 엔진** | 구글이 만든 JS 엔진 → 자바스크립트를 빠르게 실행 |
| **libuv** | 이벤트 루프, 비동기 I/O, 스레드풀 제공 |
| **모듈 시스템 (CommonJS/ESM)** | `require()` 또는 `import`로 모듈 사용 |
| **내장 모듈** | `fs`, `http`, `path`, `os` 등 기본 제공 API |

## Node.js 설치 & 실행
### 설치
- [공식 사이트](https://nodejs.org)에서 LTS 버전 설치
- 터미널에서 확인:
```bash
node -v
npm -v
```

### 실행
- 파일 만들기 (example.js)
```js
console.log("Hello Node.js!");
```
- 터미널 실행
```bash
node example.js
```

## 모듈 시스템
Node.js에서는 파일 단위로 코드를 분리 가능

```js
// math.js
export function add(a, b) {
  return a + b;
}

// main.js
import { add } from "./math.js";
console.log(add(2, 3)); // 5
```

> ✅ Node.js 14+부터 ES Module (`import/export`) 공식 지원  
> (package.json에 `"type": "module"` 필요)

## npm (Node Package Manager)
- **세계 최대의 패키지 레지스트리**
- 주요 명령어:
```bash
npm init -y        # package.json 생성
npm install axios  # 패키지 설치
npm uninstall axios # 패키지 삭제
npx create-react-app my-app # 패키지 실행
```

## 자주 쓰는 내장 모듈

| 모듈 | 용도 | 예시 |
|:-----|:-----|:-----|
| **fs** | 파일 시스템 읽기/쓰기 | `fs.readFileSync("file.txt")` |
| **http** | 서버 생성 | `http.createServer((req, res)=>{})` |
| **path** | 경로 처리 | `path.join(__dirname, "file.txt")` |
| **os** | OS 정보 확인 | `os.cpus(), os.platform()` |

---

## Node.js로 간단한 서버 만들기
```js
import http from "http";

const server = http.createServer((req, res) => {
  res.writeHead(200, {"Content-Type": "text/plain"});
  res.end("Hello, Node.js Server!");
});

server.listen(3000, () => {
  console.log("Server running at http://localhost:3000");
});
```

## 장단점

| 장점 | 단점 |
|:-----|:-----|
| 빠른 비동기 처리 (실시간 앱에 강함) | CPU 집약적 작업에는 부적합 |
| npm 패키지로 생산성 높음 | 의존성 관리 필요 |
| JS 하나로 프론트·백엔드 가능 | 콜백 지옥 가능 (Promise/async로 해결) |

---

## 요약
- **Node.js = 브라우저 밖에서 JS 실행할 수 있는 런타임**
- **이벤트 기반, 논블로킹 I/O** → 빠르고 효율적인 서버 개발 가능
- **npm 생태계** → 다양한 라이브러리와 프레임워크 사용 가능 (Express, Nest.js 등)
- **백엔드, CLI, 자동화 스크립트, 실시간 서비스** 등에 적합
