## 1️⃣ ⭐ **핵심** — "상태"란 무엇인가? [↑](#toc)

### 전등 스위치 비유

> 전등에는 **상태**가 있습니다: "켜짐(ON)" 또는 "꺼짐(OFF)"
> 스위치를 누르면 상태가 바뀌고, 전등이 달라집니다.

Flutter의 **상태(State)**도 같습니다:
- 상태 = 앱이 기억하는 값 (숫자, 텍스트, 켜짐/꺼짐 등)
- 상태가 바뀌면 → 화면이 자동으로 바뀜

### 상태가 있는 것들

```
카운터 앱       → 현재 숫자 (0, 1, 2, ...)
좋아요 버튼     → 눌렀는지 안 눌렀는지 (true/false)
입력 폼        → 사용자가 입력한 텍스트
토글 스위치     → 켜짐/꺼짐
쇼핑 카트       → 담은 상품 목록
```

### 상태가 없는 것들

```
앱 제목 텍스트    → 항상 같음
아이콘 이미지     → 항상 같음
고정 배경색      → 항상 같음
```

## 2️⃣ ⭐ **핵심** — StatelessWidget vs StatefulWidget [↑](#toc)

### StatelessWidget — "조각상"

```dart
class MyLabel extends StatelessWidget {
  const MyLabel({super.key});

  @override
  Widget build(BuildContext context) {
    return const Text('절대 안 바뀌는 텍스트');
  }
}
```

> 대리석으로 만든 조각상처럼, **한 번 만들면 변하지 않습니다.**
> `build()` 메서드가 딱 한 번 실행되고 끝.

### StatefulWidget — "전자 게시판"

```dart
class MyCounter extends StatefulWidget {
  const MyCounter({super.key});

  @override
  State<MyCounter> createState() => _MyCounterState();
}

class _MyCounterState extends State<MyCounter> {
  int _count = 0;    // 상태 변수

  @override
  Widget build(BuildContext context) {
    return Text('카운트: $_count');
  }
}
```

> 전자 게시판처럼, **내용이 바뀔 수 있습니다.**
> 상태가 바뀔 때마다 `build()`가 다시 실행되어 화면이 업데이트됩니다.

### 비교표

| | StatelessWidget | StatefulWidget |
|---|---|---|
| 상태 | 없음 | 있음 |
| build 호출 | 처음 1번 | 상태가 바뀔 때마다 |
| 사용 예 | 텍스트, 아이콘, 고정 레이아웃 | 카운터, 폼, 토글 |
| 성능 | 더 빠름 | 상태 관리 가능 |

**규칙**: 화면이 바뀌어야 한다면 `StatefulWidget`, 아니면 `StatelessWidget`.

<a id="part3"></a>
## 3️⃣ ⭐ **핵심** — StatefulWidget 구조 해부 [↑](#toc)

`StatefulWidget`은 항상 **2개의 클래스**로 이루어집니다.

```dart
// 클래스 1: 위젯 자체 (변하지 않는 껍데기)
class MyCounter extends StatefulWidget {
  const MyCounter({super.key});

  // createState: "나의 상태 객체를 만들어줘"
  @override
  State<MyCounter> createState() => _MyCounterState();
}

// 클래스 2: 상태 (변하는 내용물)
class _MyCounterState extends State<MyCounter> {
  // 상태 변수 (화면에 표시할 값들)
  int _count = 0;

  // build: 현재 상태를 기반으로 화면을 그린다
  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        Text(
          '$_count',
          style: const TextStyle(fontSize: 48),
        ),
      ],
    );
  }
}
```

### 작명 규칙

```
위젯 클래스: MyCounter         → 공개 (대문자 시작)
상태 클래스: _MyCounterState   → 비공개 (언더스코어 시작)
```

`_` (언더스코어)는 Dart에서 "이 파일 안에서만 쓴다"는 의미입니다.

<a id="part4"></a>
## 4️⃣ ⭐ **핵심** — setState() [↑](#toc)

### "화면아, 다시 그려!"

`setState()`는 Flutter에게 이렇게 말하는 것과 같습니다:

> "상태가 바뀌었어. 화면을 다시 그려줘!"

```dart
class _MyCounterState extends State<MyCounter> {
  int _count = 0;

  void _increment() {
    setState(() {
      _count++;    // 이 안에서 상태를 변경한다
    });
    // setState 호출 → Flutter가 build()를 다시 호출 → 화면 업데이트
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('$_count'),
        ElevatedButton(
          onPressed: _increment,    // 버튼 누르면 _increment 실행
          child: const Text('+1'),
        ),
      ],
    );
  }
}
```

### 틀린 방법 vs 올바른 방법

```dart
// 틀린 방법 — setState 없이 변경하면 화면이 안 바뀜!
void _wrongWay() {
  _count++;    // 값은 바뀌지만 Flutter가 모른다
}

// 올바른 방법 — setState 안에서 변경해야 화면이 바뀜
void _rightWay() {
  setState(() {
    _count++;    // Flutter에게 "바뀌었어!" 알림
  });
}
```

> `setState`는 "변경 알림 + 화면 재그리기 요청"을 한 번에 처리합니다.

## 5️⃣ ⭐ **핵심** — 실습: 카운터 앱 만들기 [↑](#toc)

새 Flutter 프로젝트에서 `lib/main.dart`를 다음과 같이 작성하세요.
**복붙 금지 — 직접 타이핑!**

```dart
import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return const MaterialApp(
      title: '카운터 앱',
      home: CounterPage(),
    );
  }
}

class CounterPage extends StatefulWidget {
  const CounterPage({super.key});

  @override
  State<CounterPage> createState() => _CounterPageState();
}

class _CounterPageState extends State<CounterPage> {
  int _count = 0;

  void _increment() {
    setState(() {
      _count++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('카운터'),
        backgroundColor: Colors.blue,
        foregroundColor: Colors.white,
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            const Text(
              '버튼을 누른 횟수:',
              style: TextStyle(fontSize: 18),
            ),
            const SizedBox(height: 16),
            Text(
              '$_count',
              style: const TextStyle(
                fontSize: 72,
                fontWeight: FontWeight.bold,
                color: Colors.blue,
              ),
            ),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _increment,
        child: const Icon(Icons.add),
      ),
    );
  }
}
```

## 6️⃣ 💪 **보너스 챌린지** — 카운터 앱 확장 [↑](#toc)

기본 카운터에 기능을 추가해 봅시다.

### 감소 버튼 + 리셋 버튼 추가

```dart
class _CounterPageState extends State<CounterPage> {
  int _count = 0;

  void _increment() {
    setState(() {
      _count++;
    });
  }

  void _decrement() {
    setState(() {
      if (_count > 0) _count--;    // 0 미만으로 내려가지 않게
    });
  }

  void _reset() {
    setState(() {
      _count = 0;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('카운터'),
        actions: [
          IconButton(
            onPressed: _reset,
            icon: const Icon(Icons.refresh),
            tooltip: '리셋',
          ),
        ],
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            Text(
              '$_count',
              style: TextStyle(
                fontSize: 72,
                fontWeight: FontWeight.bold,
                // 색상도 상태로 변경!
                color: _count == 0
                    ? Colors.grey
                    : _count > 0
                        ? Colors.blue
                        : Colors.red,
              ),
            ),
            const SizedBox(height: 32),
            Row(
              mainAxisAlignment: MainAxisAlignment.center,
              children: [
                FloatingActionButton(
                  heroTag: 'decrement',    // 같은 화면에 FAB 2개 이상이면 필요
                  onPressed: _decrement,
                  backgroundColor: Colors.red,
                  child: const Icon(Icons.remove),
                ),
                const SizedBox(width: 24),
                FloatingActionButton(
                  heroTag: 'increment',
                  onPressed: _increment,
                  backgroundColor: Colors.blue,
                  child: const Icon(Icons.add),
                ),
              ],
            ),
          ],
        ),
      ),
    );
  }
}
```

### 색상 변경 — 상태로 색상 제어

상태는 숫자뿐 아니라 **색상**, **텍스트**, **불리언** 등 무엇이든 될 수 있습니다.

```dart
class _ColorCounterState extends State<ColorCounter> {
  int _count = 0;

  Color get _color {
    if (_count <= 0) return Colors.grey;
    if (_count <= 3) return Colors.green;
    if (_count <= 7) return Colors.orange;
    return Colors.red;
  }

  String get _message {
    if (_count <= 0) return '시작해 보세요!';
    if (_count <= 3) return '잘 하고 있어요!';
    if (_count <= 7) return '열정적이네요!';
    return '대단한 열정!';
  }

  // ...build 에서 _color와 _message 사용
}
```

## 7️⃣ 📖 **더 알아보기** — 상태의 범위 [↑](#toc)

### 위젯 단위 상태

각 StatefulWidget은 **자기 자신의 상태만** 가집니다.

```dart
// Counter1과 Counter2는 독립적인 상태를 가짐
class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return Column(
      children: const [
        Counter(),    // 이 카운터는 이 카운터만의 상태
        Counter(),    // 이 카운터는 위 카운터와 별개
      ],
    );
  }
}
```

실행하면 두 카운터가 서로 독립적으로 동작합니다.

### 상태를 위로 올리기 (Lifting State Up)

두 위젯이 **같은 상태를 공유**해야 한다면, 공통 부모로 상태를 올립니다.

```dart
// 부모가 상태를 가지고, 자식에게 전달
class ParentWidget extends StatefulWidget {
  const ParentWidget({super.key});

  @override
  State<ParentWidget> createState() => _ParentWidgetState();
}

class _ParentWidgetState extends State<ParentWidget> {
  bool _isFavorite = false;

  void _toggle() {
    setState(() {
      _isFavorite = !_isFavorite;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        FavoriteButton(
          isFavorite: _isFavorite,  // 상태 전달
          onToggle: _toggle,        // 변경 함수 전달
        ),
        FavoriteStatus(isFavorite: _isFavorite),  // 같은 상태 공유
      ],
    );
  }
}
```

> setState는 소규모 앱에 적합합니다. 앱이 커지면 Provider, Riverpod 같은 전역 상태 관리 도구를 사용합니다. (Phase 3에서 배웁니다)

## 8️⃣ 🚀 **도전** — 좋아요/싫어요 투표 앱 [↑](#toc)

지금까지 배운 것을 종합해서 투표 앱을 만들어 보세요.

### 요구사항

```
기능:
1. "좋아요" 버튼 (엄지 올림 아이콘) — 누르면 카운터 +1
2. "싫어요" 버튼 (엄지 내림 아이콘) — 누르면 카운터 +1
3. 각 버튼의 눌린 횟수 표시
4. 총 투표수와 좋아요 비율(%) 표시
5. 비율에 따른 게이지 바 색상 변경
   - 70% 이상: 초록
   - 40~69%: 노랑
   - 40% 미만: 빨강

제약:
- StatefulWidget 사용
- 상태 변수: _likes, _dislikes
- 나머지는 getter로 계산
```

### 힌트: 상태와 계산 분리

```dart
class _VoteState extends State<VoteApp> {
  int _likes = 0;
  int _dislikes = 0;

  // 상태를 기반으로 계산 (setState 불필요)
  int get _total => _likes + _dislikes;
  double get _likeRatio => _total == 0 ? 0.0 : _likes / _total;
  String get _likePercent => '${(_likeRatio * 100).toStringAsFixed(1)}%';

  Color get _barColor {
    if (_likeRatio >= 0.7) return Colors.green;
    if (_likeRatio >= 0.4) return Colors.yellow;
    return Colors.red;
  }

  // ...
}
```

## 9️⃣ 정리 [↑](#toc)

### 핵심 개념 3가지

| 개념 | 한 줄 설명 |
|------|-----------|
| 상태(State) | 앱이 기억하는 값. 바뀌면 화면이 바뀐다. |
| StatefulWidget | 상태를 가진 위젯. 2개 클래스로 구성. |
| setState() | "상태가 바뀌었어, 화면 다시 그려!" 명령 |

### StatefulWidget 구조 템플릿

```dart
class MyWidget extends StatefulWidget {
  const MyWidget({super.key});

  @override
  State<MyWidget> createState() => _MyWidgetState();
}

class _MyWidgetState extends State<MyWidget> {
  // 1. 상태 변수 선언
  int _value = 0;

  // 2. 상태 변경 메서드
  void _changeValue() {
    setState(() {
      _value++;
    });
  }

  // 3. 화면 구성
  @override
  Widget build(BuildContext context) {
    return Text('$_value');
  }
}
```

### setState 규칙

1. 상태 변경은 **반드시 setState() 안에서**
2. setState 안에서 **비동기 작업** 금지 (`await` 사용 불가)
3. 상태 변수는 **언더스코어(\_)로 시작** (관례)
