## 1️⃣ ⭐ **핵심** — 모바일 앱이란? [↑](#toc)

### 앱의 세 가지 종류

스마트폰에서 실행되는 앱은 크게 세 가지로 나뉩니다.

**비유로 이해하기**: 음식점 세 종류를 생각해 보세요.

| 종류 | 음식점 비유 | 예시 | 장점 | 단점 |
|------|-----------|------|------|------|
| **네이티브 앱** | 각 지역 전통 음식점 | 카카오톡(Android), Gmail(iOS) | 가장 빠름, 기기 기능 100% 활용 | Android/iOS 따로 개발 필요 |
| **크로스플랫폼 앱** | 프랜차이즈 음식점 | Flutter 앱, React Native 앱 | 코드 하나로 두 플랫폼 지원 | 약간의 성능 차이 |
| **웹앱 (PWA)** | 배달 음식 | 모바일 웹사이트 | 설치 불필요, 어디서나 접속 | 기기 기능 제한적 |

### 왜 크로스플랫폼인가?

한국에는 Android 사용자와 iPhone 사용자가 함께 있습니다. 네이티브로 만들려면:

- Android 앱: Kotlin/Java 언어로 개발
- iOS 앱: Swift/Objective-C 언어로 개발
- **인력 2배, 비용 2배, 시간 2배**

크로스플랫폼은 하나의 코드로 두 플랫폼을 동시에 지원합니다.

```
하나의 Flutter 코드
        │
        ├── Android 앱 (.apk)
        ├── iOS 앱 (.ipa)
        └── Web 앱 (선택)
```

## 2️⃣ ⭐ **핵심** — Flutter란? [↑](#toc)

### Flutter의 정체

Flutter는 **Google이 만든 오픈소스 UI 프레임워크**입니다. 2018년에 처음 발표되었고, 지금은 전 세계 수백만 명의 개발자가 사용합니다.

> **핵심 철학**: "모든 것은 위젯(Widget)이다"

Flutter 앱에서 화면에 보이는 모든 요소 — 텍스트, 버튼, 이미지, 여백까지 — 는 전부 **위젯**이라고 부릅니다. 레고 블록처럼 위젯을 쌓아서 화면을 만듭니다.

### Flutter가 빠른 이유

일반적인 크로스플랫폼 앱은 OS의 UI 컴포넌트를 빌려 씁니다. 중간에 번역 과정이 생기는 셈입니다.

Flutter는 다릅니다. **Skia/Impeller 엔진**으로 직접 픽셀을 그립니다. OS에게 "이 버튼 그려줘"라고 요청하는 게 아니라, 직접 캔버스에 그림을 그리는 방식입니다.

```
기존 방식:  Flutter 코드 → OS 브리지 → Android/iOS 위젯 → 화면
Flutter:    Flutter 코드 → Skia/Impeller 엔진 → 직접 픽셀 → 화면
```

### Flutter를 쓰는 실제 앱들

- **알리익스프레스** — 쇼핑 앱
- **BMW 앱** — 차량 제어 앱
- **Google Pay** — 결제 앱
- **Nubank** — 브라질 최대 디지털 뱅크

## 3️⃣ ⭐ **핵심** — Dart 언어 소개 [↑](#toc)

### Dart가 뭔가요?

Dart는 Flutter가 사용하는 프로그래밍 언어입니다. Google이 만들었으며, Java나 JavaScript를 알고 있다면 매우 친숙하게 느껴집니다.

> **비유**: Flutter가 자동차라면, Dart는 그 자동차를 움직이는 엔진입니다.

### Dart의 특징

| 특징 | 설명 |
|------|------|
| **타입 안전성** | 변수에 타입을 지정해서 실수를 미리 방지 |
| **Null Safety** | "없는 값" 오류를 컴파일 시점에 잡아냄 |
| **AOT + JIT 컴파일** | 개발 중엔 빠른 Hot Reload, 배포 시엔 빠른 실행 |
| **현대적 문법** | 화살표 함수, 스프레드 연산자 등 편리한 문법 |

### 첫 Dart 코드 맛보기

지금 당장 이해 못해도 괜찮습니다. 나중에 하나씩 배울 내용입니다.

```dart
void main() {
  String name = '김민준';
  int age = 25;
  print('안녕하세요, $name입니다. 나이는 $age살입니다.');
}
```

출력 결과:
```
안녕하세요, 김민준입니다. 나이는 25살입니다.
```

## 4️⃣ ⭐ **핵심** — 환경 구축 (Windows) [↑](#toc)

> Windows 10 이상 기준으로 설명합니다. macOS를 사용한다면 [다음 섹션](#part5)으로 건너뛰세요.

### 필요한 것들

| 도구 | 역할 | 용량 |
|------|------|------|
| Flutter SDK | Flutter + Dart 실행 환경 | ~1GB |
| VS Code | 코드 편집기 | ~100MB |
| Android Studio | Android 에뮬레이터 제공 | ~4GB |
| Git | 소스 코드 버전 관리 | ~50MB |

### Step 1: Git 설치

Git이 없으면 Flutter SDK를 설치할 수 없습니다.

1. [git-scm.com](https://git-scm.com) 방문
2. "Download for Windows" 클릭
3. 설치 마법사에서 기본값 유지하며 Next 클릭

설치 확인:
```bash
git --version
# git version 2.x.x 출력되면 성공
```

### Step 2: Flutter SDK 설치

1. [flutter.dev/docs/get-started/install/windows](https://flutter.dev/docs/get-started/install/windows) 방문
2. "flutter_windows_x.x.x-stable.zip" 다운로드 (약 1GB)
3. `C:\flutter` 폴더에 압축 해제

> **주의**: `C:\Program Files\flutter` 처럼 **공백이 있는 경로**는 피하세요.

### Step 3: PATH 환경 변수 설정

Flutter 명령어를 어디서나 쓸 수 있도록 PATH에 추가합니다.

1. **시작 메뉴** → "환경 변수" 검색 → "시스템 환경 변수 편집" 클릭
2. "환경 변수" 버튼 클릭
3. "시스템 변수" 목록에서 `Path` 선택 → "편집" 클릭
4. "새로 만들기" 클릭 → `C:\flutter\bin` 입력
5. 확인 → 확인 → 확인

설치 확인 (새 명령 프롬프트 열고):
```bash
flutter --version
# Flutter x.x.x • channel stable 출력되면 성공
```

### Step 4: Android Studio 설치

1. [developer.android.com/studio](https://developer.android.com/studio) 방문
2. "Download Android Studio" 클릭
3. 설치 마법사 진행 (기본값 유지)
4. 설치 완료 후 Android Studio 실행
5. "SDK Platforms" 탭 → Android 14 (API 34) 체크 → Apply

### Step 5: Android 에뮬레이터 설정

1. Android Studio 우측 상단 → **Device Manager** 클릭
2. "Create Device" 클릭
3. **Pixel 6** (또는 원하는 기기) 선택 → Next
4. **API 34** 이미지 다운로드 → Next → Finish
5. 에뮬레이터 옆 ▶ 버튼 클릭하여 실행

> **에뮬레이터가 느릴 때**: BIOS에서 Intel VT-x (또는 AMD-V) 가상화 설정이 켜져 있는지 확인하세요. HAXM 설치가 필요할 수 있습니다 (자세한 내용은 [트러블슈팅 섹션](#part7)).

### Step 6: VS Code + Flutter 확장 설치

1. [code.visualstudio.com](https://code.visualstudio.com) 방문 → 다운로드 및 설치
2. VS Code 실행
3. 왼쪽 **Extensions** 아이콘 클릭 (또는 `Ctrl+Shift+X`)
4. "Flutter" 검색 → Flutter (by Dart Code) 설치
5. 자동으로 Dart 확장도 함께 설치됩니다

## 5️⃣ ⭐ **핵심** — 환경 구축 (macOS) [↑](#toc)

> macOS 12 이상 기준으로 설명합니다. Windows를 사용한다면 [이전 섹션](#part4)을 따라주세요.

### Step 1: Homebrew 설치 (권장)

Homebrew는 macOS용 패키지 관리자입니다. 없으면 설치합니다.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### Step 2: Flutter SDK 설치

**방법 A — Homebrew 사용 (권장)**:
```bash
brew install --cask flutter
```

**방법 B — 직접 다운로드**:
1. [flutter.dev/docs/get-started/install/macos](https://flutter.dev/docs/get-started/install/macos) 방문
2. Apple Silicon (M1/M2/M3) 또는 Intel에 맞는 파일 다운로드
3. `~/development/flutter` 에 압축 해제

**방법 B를 선택한 경우 PATH 설정** (`~/.zshrc` 파일에 추가):
```bash
echo 'export PATH="$HOME/development/flutter/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

설치 확인:
```bash
flutter --version
```

### Step 3: Android Studio 설치 (Android 앱 개발 시)

Windows와 동일합니다:
1. [developer.android.com/studio](https://developer.android.com/studio) 방문
2. macOS용 다운로드 (.dmg)
3. 설치 후 Android SDK 및 에뮬레이터 설정

### Step 4: Xcode 설치 (iOS 앱 개발 시, Mac 전용)

iOS 앱은 macOS에서만 빌드할 수 있습니다.

```bash
# App Store에서 Xcode 설치 또는:
xcode-select --install
```

Xcode 설치 후 라이선스 동의:
```bash
sudo xcodebuild -license accept
```

iOS 시뮬레이터 실행:
```bash
open -a Simulator
```

### Step 5: VS Code + Flutter 확장

Windows와 동일하게 설치합니다. (`Cmd+Shift+X`로 확장 마켓 열기)

## 6️⃣ 📖 **더 알아보기** — DartPad (백업 환경) [↑](#toc)

### DartPad란?

[dartpad.dev](https://dartpad.dev)는 브라우저에서 바로 Dart/Flutter 코드를 실행할 수 있는 온라인 환경입니다.

- 설치 없이 즉시 사용 가능
- Dart 문법 연습에 최적화
- 간단한 Flutter UI도 미리보기 가능

### 언제 쓰나요?

| 상황 | DartPad |
|------|---------|
| 노트북 사양 부족 | ✅ 사용 |
| 인터넷 없는 환경 | ❌ 사용 불가 |
| Dart 문법 빠른 연습 | ✅ 매우 편리 |
| 완성된 앱 개발 | ❌ 한계 있음 |

> **추천**: Dart 기초 챕터(2~6장)에서는 DartPad를 적극 활용하세요. 에뮬레이터 없이도 Dart 코드를 바로 실행해볼 수 있습니다.

## 7️⃣ ⭐ **핵심** — flutter doctor 실행 및 트러블슈팅 [↑](#toc)

### flutter doctor 실행

`flutter doctor`는 Flutter 개발 환경을 진단해주는 명령어입니다. 의사(doctor)가 건강 검진하는 것처럼 모든 구성 요소를 체크합니다.

```bash
flutter doctor
```

### 출력 결과 읽기

```
Doctor summary (to see all details, run flutter doctor -v):
[✓] Flutter (Channel stable, 3.x.x, ...)
[✓] Android toolchain - develop for Android devices
[✓] Chrome - develop for the web
[✓] VS Code (version 1.x.x)
[✓] Connected device (2 available)
[✓] Network resources
```

- **[✓]**: 정상
- **[!]**: 경고 (동작하지만 설정 권장)
- **[✗]**: 오류 (해결 필요)

### 트러블슈팅

#### 문제 1: Android 라이선스 동의 안 됨

```
[!] Android toolchain
    ✗ Android licenses not accepted.
```

**해결책**:
```bash
flutter doctor --android-licenses
# 모든 질문에 y 입력 후 Enter
```

---

#### 문제 2: HAXM 설치 실패 (Windows)

증상: 에뮬레이터 실행 시 "HAXM is not installed" 오류

**해결책 A — Intel HAXM 설치**:
1. Android Studio → SDK Manager → SDK Tools 탭
2. "Intel x86 Emulator Accelerator (HAXM installer)" 체크 → Apply

**해결책 B — BIOS 가상화 활성화**:
1. 컴퓨터 재시작 → BIOS 진입 (제조사마다 F2, F12, DEL 등)
2. "Virtualization Technology" 또는 "VT-x" → Enabled 설정

**해결책 C — AMD 프로세서라면**:
- HAXM 대신 **Windows Hypervisor Platform** 사용
- 제어판 → 프로그램 → Windows 기능 켜기/끄기 → "Windows 하이퍼바이저 플랫폼" 체크

---

#### 문제 3: PATH 설정 문제 (Windows)

증상: `flutter` 명령어를 찾을 수 없음

```
'flutter'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는
배치 파일이 아닙니다.
```

**해결책**:
1. 새 명령 프롬프트/PowerShell을 **관리자 권한**으로 열기
2. PATH에 Flutter bin 경로가 있는지 확인:
   ```bash
   echo %PATH%
   ```
3. 없다면 환경 변수 설정 재확인 ([Step 3](#part4) 참조)
4. 컴퓨터 재시작 후 재시도

---

#### 문제 4: 에뮬레이터가 너무 느림

**해결책 A**: 에뮬레이터 RAM 늘리기
- Android Studio → Device Manager → 에뮬레이터 편집(연필 아이콘)
- Advanced → RAM: 2048 MB 이상 설정

**해결책 B**: x86_64 이미지 사용
- API 이미지 선택 시 "x86_64" 아키텍처 선택 (ARM보다 빠름)

**해결책 C**: 실제 Android 기기 사용 (가장 빠름)
- USB 디버깅 활성화 → USB 연결 → `flutter run`

---

#### 문제 5: macOS에서 CocoaPods 오류

```
[!] CocoaPods not installed
```

**해결책**:
```bash
sudo gem install cocoapods
# 또는
brew install cocoapods
```
