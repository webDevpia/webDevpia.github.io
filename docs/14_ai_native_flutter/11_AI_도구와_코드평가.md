## 1️⃣ ⭐ **핵심** — GitHub Copilot 설치 [↑](#toc)

### 사전 준비

- GitHub 계정 (없으면 github.com에서 무료 가입)
- GitHub Copilot 구독 (학생은 GitHub Student Pack으로 무료)
- VS Code 설치 완료

### 단계별 설치

**Step 1: VS Code에서 Copilot 확장 설치**

1. VS Code 왼쪽 사이드바에서 **확장(Extensions)** 아이콘 클릭 (또는 `Ctrl+Shift+X`)
2. 검색창에 `GitHub Copilot` 입력
3. **GitHub Copilot** (Microsoft 공식) 클릭 → **Install** 클릭
4. 설치 완료 후 **GitHub Copilot Chat** 도 함께 설치

**Step 2: GitHub 계정 연결**

1. VS Code 왼쪽 하단 계정 아이콘 클릭
2. **Sign in with GitHub** 선택
3. 브라우저가 열리면 GitHub 로그인 및 권한 허용
4. VS Code로 돌아오면 연결 완료

**Step 3: 연결 확인**

VS Code 오른쪽 하단에 Copilot 아이콘(비행기)이 나타나면 성공!

```
상태 표시:
✅ Copilot 아이콘이 활성화 상태 → 정상
⚠️ 아이콘에 x 표시 → 로그인 필요
```

### Copilot 기본 단축키

| 동작 | 단축키 (Mac) | 단축키 (Windows) |
|------|-------------|-----------------|
| 제안 수락 | `Tab` | `Tab` |
| 제안 거부 | `Esc` | `Esc` |
| 다음 제안 | `Option+]` | `Alt+]` |
| 이전 제안 | `Option+[` | `Alt+[` |
| Copilot Chat 열기 | `Ctrl+Shift+I` | `Ctrl+Shift+I` |

## 3️⃣ AI에게 위젯 트리 요청하기 [↑](#toc)

### 좋은 프롬프트 vs 나쁜 프롬프트

**나쁜 프롬프트:**
```
카드 만들어줘
```

AI가 무엇을 원하는지 모릅니다.

---

**좋은 프롬프트:**
```
Flutter StatelessWidget으로 다음 조건의 프로필 카드를 만들어줘:

- 가로 폭: 화면 전체
- 구성: 왼쪽에 CircleAvatar(반지름 30), 오른쪽에 이름(굵게)과 이메일(회색)
- 카드 배경: 흰색, 둥근 모서리(12), 그림자
- 패딩: 16px
- const 생성자 사용
- null safety 준수
```

### 프롬프트 작성 패턴

```
[위젯 종류] + [레이아웃 설명] + [스타일 조건] + [기술적 제약]
```

| 항목 | 예시 |
|------|------|
| 위젯 종류 | StatelessWidget, StatefulWidget |
| 레이아웃 | Row 안에 아이콘과 텍스트, Column으로 세로 정렬 |
| 스타일 | 파란색 배경, 흰색 텍스트, 둥근 모서리 |
| 기술 제약 | const 사용, Dart null safety 준수 |

## 4️⃣ ⭐ **핵심** — AI 코드 평가 체크리스트 (Flutter 버전) [↑](#toc)

AI가 코드를 생성하면, **이 체크리스트로 평가**하세요.

### Flutter AI 코드 체크리스트

```
□ 1. const 사용 여부
   - 변하지 않는 위젯에 const가 붙어 있는가?
   - const Text('안녕') ✅   Text('안녕') ⚠️

□ 2. null safety 준수
   - ? 없이 null이 들어올 수 있는 변수가 있는가?
   - String? name = null; 처리가 되어 있는가?

□ 3. 위젯 분리
   - build()가 너무 길지 않은가? (50줄 이상이면 분리 필요)
   - 반복되는 위젯을 별도 메서드/클래스로 분리했는가?

□ 4. dispose() 호출
   - TextEditingController, AnimationController 등을 사용했다면
     dispose()에서 정리하는가?

□ 5. StatefulWidget 사용이 적절한가?
   - 변하는 값이 없는데 StatefulWidget을 쓰고 있지 않은가?

□ 6. import 확인
   - 사용하지 않는 import가 있는가?
   - flutter/material.dart 외에 필요한 패키지가 있는가?

□ 7. 에러 처리
   - 비어있을 수 있는 목록 접근 시 bounds 체크가 있는가?
   - async 코드에 try-catch가 있는가?
```

### 즉시 교체해야 할 패턴

```dart
// ❌ 나쁜 패턴 1: const 빠진 고정 위젯
Text('제목')             // → const Text('제목')
Icon(Icons.star)         // → const Icon(Icons.star)

// ❌ 나쁜 패턴 2: 너무 긴 build()
Widget build(BuildContext context) {
  return Column(
    children: [
      // 100줄의 코드...
    ],
  );
}
// → 별도 _buildHeader(), _buildBody() 메서드로 분리

// ❌ 나쁜 패턴 3: controller 정리 안 함
class _MyFormState extends State<MyForm> {
  final _controller = TextEditingController();
  // dispose() 없음!  ← 메모리 누수!
}
// → dispose() 추가 필수
```

## 5️⃣ 🚀 **도전** — Bug Hunt: AI 코드에서 버그 찾기 [↑](#toc)

> 탐정이 되어보세요. AI가 만든 아래 코드에서 버그를 찾으세요.

### 레벨 1 — 기초 버그 (3개 찾기)

```dart
// AI가 생성한 코드 — 버그가 있습니다!
class ProfileCard extends StatelessWidget {
  final String name;
  final String email;
  final int age;

  ProfileCard({required this.name, required this.email, required this.age});

  Widget build(BuildContext context) {    // (A)
    return Card(
      child: Padding(
        padding: EdgeInsets.all(16),      // (B)
        child: Column(
          children: [
            Text(name, style: TextStyle(fontSize: 18, fontWeight: FontWeight.bold)),  // (C)
            Text(email, style: TextStyle(color: Colors.grey)),
            Text('나이: $age세'),
          ],
        ),
      ),
    );
  }
}
```

**힌트:** (A), (B), (C) 표시 근처를 집중적으로 봐보세요.

<details markdown="0">
<summary>정답 보기</summary>

```
(A) build() 앞에 @override가 없음
    → @override Widget build(BuildContext context) 로 수정

(B) EdgeInsets.all(16) 앞에 const 없음
    → const EdgeInsets.all(16) 으로 수정

(C) Text, TextStyle 위젯들 앞에 const 없음
    → const Text(name, ...) 불가 (name이 변수), 하지만
       const TextStyle(fontSize: 18, fontWeight: FontWeight.bold) 는 가능

    그리고! ProfileCard 생성자에 const 키워드 없음
    → const ProfileCard({...}) 로 수정
```

</details>

### 레벨 2 — 상태 버그 (2개 찾기)

```dart
class ToggleButton extends StatefulWidget {
  const ToggleButton({super.key});

  @override
  State<ToggleButton> createState() => _ToggleButtonState();
}

class _ToggleButtonState extends State<ToggleButton> {
  bool isOn = false;    // (A)

  void _toggle() {
    isOn = !isOn;       // (B) setState 없음!
  }

  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: _toggle,
      style: ElevatedButton.styleFrom(
        backgroundColor: isOn ? Colors.green : Colors.grey,
      ),
      child: Text(isOn ? 'ON' : 'OFF'),
    );
  }
}
```

<details markdown="0">
<summary>정답 보기</summary>

```
(A) 상태 변수 이름이 _isOn이어야 함 (Flutter 관례: 언더스코어)
    → bool _isOn = false;

(B) setState() 없이 상태 변경 — 화면이 절대 안 바뀜!
    → void _toggle() {
         setState(() {
           _isOn = !_isOn;
         });
       }
```

</details>

### 레벨 3 — 메모리 버그 (1개 찾기)

```dart
class SearchField extends StatefulWidget {
  const SearchField({super.key});

  @override
  State<SearchField> createState() => _SearchFieldState();
}

class _SearchFieldState extends State<SearchField> {
  final _controller = TextEditingController();

  @override
  Widget build(BuildContext context) {
    return TextField(
      controller: _controller,
      decoration: const InputDecoration(
        hintText: '검색어를 입력하세요',
        prefixIcon: Icon(Icons.search),
      ),
    );
  }
  // dispose() 없음!
}
```

<details markdown="0">
<summary>정답 보기</summary>

```dart
// dispose() 반드시 추가!
@override
void dispose() {
  _controller.dispose();    // 메모리 반환
  super.dispose();
}
```

TextEditingController는 메모리를 사용합니다.
위젯이 사라질 때 dispose()를 호출하지 않으면 메모리 누수가 발생합니다.

</details>

## 6️⃣ "Explain It Back" — AI 코드 한 줄씩 설명하기 [↑](#toc)

> 이해한 것만 사용할 수 있습니다. 이해 못 한 코드는 쓰지 마세요.

### 방법

Copilot이 코드를 생성하면, 각 줄을 소리 내어 설명하세요.

**예시 — AI가 생성한 코드:**

```dart
@override
Widget build(BuildContext context) {
  return AnimatedContainer(
    duration: const Duration(milliseconds: 300),
    curve: Curves.easeInOut,
    color: _isSelected ? Colors.blue : Colors.white,
    child: ListTile(
      title: Text(widget.title),
      onTap: _handleTap,
    ),
  );
}
```

**Explain It Back 적용:**

```
"@override Widget build(BuildContext context)"
  → Flutter가 이 위젯을 그릴 때 호출하는 메서드

"AnimatedContainer"
  → 색상이나 크기가 바뀔 때 자동으로 애니메이션이 되는 Container

"duration: const Duration(milliseconds: 300)"
  → 애니메이션이 0.3초 동안 진행됨

"curve: Curves.easeInOut"
  → 애니메이션이 처음과 끝에서 천천히, 중간에 빠르게 진행됨

"color: _isSelected ? Colors.blue : Colors.white"
  → _isSelected가 true면 파란색, false면 흰색
  → 삼항 연산자 사용

"child: ListTile(title: Text(widget.title), onTap: _handleTap)"
  → ListTile 안에 제목 텍스트, 탭하면 _handleTap 실행
```

### 막히면 어떻게?

```
1. Flutter 공식 문서에서 해당 위젯 검색
2. Copilot Chat에 "AnimatedContainer를 한 줄로 설명해줘" 요청
3. 이해가 될 때까지 설명 반복
4. 이해 안 되면 사용 금지 — 더 간단한 방법으로 교체
```

## 7️⃣ Phase 2 AI 규칙 정리 [↑](#toc)

### Phase별 AI 사용 규칙

| Phase | 규칙 | 이유 |
|-------|------|------|
| Phase 1 (01~08장) | AI 사용 금지 | 기초 문법 체득 |
| Phase 2 초반 (09~10장) | AI 사용 금지 | 레이아웃·상태 감각 체득 |
| Phase 2 중반~ (11장~) | AI 생성 + 검증 | AI 협업 방식 훈련 |
| Phase 3 | AI 설계 + 자유 사용 | AI-Native 개발자 |

### Phase 2 "생성 + 검증" 규칙

```
✅ 허용:
- Copilot 인라인 제안 수락 (Tab)
- Copilot Chat으로 위젯 생성 요청
- AI 제안 코드를 이해한 후 수정

❌ 금지:
- AI 코드를 이해 없이 복붙
- 체크리스트 확인 없이 AI 코드 사용
- "AI가 만들었으니까 맞겠지" 믿음
```

### 황금률

> AI는 **코드를 생성**하고, 사람은 **코드를 책임**진다.
> AI가 만든 버그의 책임은 프롬프트를 쓴 사람에게 있습니다.
