| 판정 | 공통과정 | 전문과정 | 합계 |

| ✅ 충분 | 4차시 | 3차시 | **7차시 (23%)** |

## 2. 다중 페르소나 비판적 검토

### 페르소나 A: 현직 부트캠프 운영자 (박진수, 7년 경력)

> *"집중 교육에서 9시간을 채우는 것은 콘텐츠가 아니라 '활동 설계'의 문제입니다."*

#### 핵심 지적

1. **"자유 실습 = 빈 시간" 함정**
   - 현재 프로젝트 세션(Day4-5)이 "팀별 구현" → "멘토링"으로만 구성
   - **경험상 20-30분마다 구조화된 체크포인트가 없으면 학습자의 50%가 멈춤**
   - 특히 No-Code 플랫폼은 시행착오가 빠르게 끝나서 "다 했는데요?" 상황 발생
   - **해결**: 30분 단위 마이크로 스프린트(계획→구현→데모) 도입

2. **에너자이저/참여 활동 완전 부재**
   - 하루 9시간에 아이스브레이커, 에너자이저, 경쟁 활동이 **단 하나도 없음**
   - 오후 3시(7시간차) 집중도 급락 구간에 대한 대책 없음
   - **해결**: 매일 오후 첫 세션 시작 시 15분 에너자이저 + Day1 아침 아이스브레이커 30분

3. **발표 시간 과소 산정**
   - 공통과정 15차시: 팀당 10분 + 5분 Q&A = 15분 × 5팀 = 75분 + 시상 15분 = 90분
   - 실제로는 세팅/전환 시간 포함하면 **120분 이상** 소요 — 2시간 45분 세션에 비해 나머지 시간 부족
   - **해결**: 발표를 오후 전체(2세션)에 걸쳐 배치하고, 발표 전 "리허설 피드백 세션" 추가

4. **트러블슈팅 버퍼 시간 부재**
   - OAuth 크리덴셜 설정만으로 팀당 20-40분 소요 가능
   - API 호출 실패, 네트워크 문제 등 돌발 상황 버퍼 없음
   - **해결**: 외부 연동 세션에 30분 트러블슈팅 버퍼 확보

### 페르소나 B: DeepLearning.AI 강의 설계자 관점 (Sarah Chen)

> *"해외 상위 프로그램과 비교했을 때 누락된 핵심 토픽이 4가지 있습니다."*

#### 누락 토픽 분석 (벤치마크 대비)

| 토픽 | Hopkins | Cornell | Stanford | DeepLearning.AI | Udacity | 우리 과정 |
|------|---------|---------|---------|----------------|---------|----------|
| Reflection 패턴 | ✅ | ✅ | - | ✅ | ✅ | **❌ 없음** |
| Agent Evaluation | ✅ | - | ✅ | ✅ | - | **❌ 없음** |
| MCP (Model Context Protocol) | - | - | - | ✅ | - | **❌ 없음** |
| 멀티에이전트 시스템 | ✅ | ✅ | ✅ | ✅ | ✅ | **❌ 없음** |
| 벡터 DB / Dense Retrieval | - | ✅ | - | - | ✅ | **△ 피상적** |
| AI 윤리/거버넌스 | ✅ | ✅ | ✅ | - | - | **❌ 없음** |
| 디버깅/Observability | - | - | - | ✅ | - | **❌ 없음** |
| Planning 패턴 | ✅ | ✅ | - | ✅ | ✅ | **❌ 없음** |

#### 핵심 보강 제안

1. **Reflection 패턴 (4시간)** — DeepLearning.AI가 독립 모듈로 운영
   - AI가 자신의 출력을 비판하고 개선하는 루프
   - 실습: "LLM이 생성한 이메일 → 검증 LLM이 품질 점검 → 피드백 반영 → 개선된 이메일"
   - Agentria에서 LLM→검증 LLM→Branch(통과/재생성)→루프 워크플로우로 구현 가능

2. **Agent Evaluation Framework (3시간)** — 대부분의 상위 프로그램에 포함
   - 에이전트 성능을 정량적으로 측정하는 방법론
   - 실습: Bulk Run + 평가 루브릭 설계 + Before/After 비교

3. **Planning 패턴 (2시간)** — Agentic AI의 4대 패턴 중 하나인데 빠져있음
   - 복잡한 작업을 하위 단계로 분해하는 패턴
   - 실습: "여행 계획 에이전트" — 목적지→항공편→숙소→일정을 단계별로 계획

4. **AI 윤리와 책임 있는 AI (1.5시간)** — Hopkins, Cornell, Stanford 모두 포함
   - 에이전트 자율성의 위험, 편향, 프라이버시
   - 토론: "에이전트가 잘못된 의료 조언을 하면 누구의 책임인가?"

### 페르소나 C: 대학생 동아리 회장 관점 (정유진, CS 4학년)

> *"솔직히 프로젝트 시간에 뭘 해야 할지 막막할 것 같아요."*

#### 핵심 지적

1. **프로젝트 세션이 너무 비구조적**
   - Day4 11차시 "1차 구현" — "노드 배치 + 프롬프트 작성 + 테스트" 이것만으로 3시간?
   - 경험 많은 팀은 1시간이면 끝나고, 느린 팀은 3시간 내내 막힘
   - **제안**: 마이크로 스프린트 + 팀 간 교차 리뷰 + 보너스 챌린지 계층화

2. **경쟁/게임 요소 전혀 없음**
   - 해커톤 형식의 시간 제한 챌린지가 하나도 없음
   - "가장 빠르게 3노드 연결하기", "가장 정확한 분류 에이전트 만들기" 같은 미니 대회 요소
   - **제안**: Day2, Day3에 "스피드 챌린지"(30분 제한) 도입

3. **산업 사례가 너무 일반적**
   - "고객 리뷰 분류", "회의록 요약"이 반복됨
   - 금융, 의료, 이커머스, 교육 등 **산업별 특화 시나리오**가 있으면 훨씬 몰입감 높음
   - **제안**: Day별로 다른 산업 시나리오 적용 (Day1: 이커머스, Day2: 금융, Day3: 교육)

4. **쉬는 시간 후 복귀 활동 없음**
   - 점심 후, 쉬는 시간 후 바로 실습에 들어가면 워밍업 시간 필요
   - **제안**: 오후 첫 시작 5분 "어제/오전 핵심 퀴즈" (Kahoot! 스타일)

### 페르소나 D: 교수법(Pedagogy) 전문가 (Dr. 김은정)

> *"인지 부하 관리와 간격 반복 원칙이 적용되지 않았습니다."*

#### 핵심 지적

1. **간격 반복(Spaced Repetition) 부재**
   - Day1에 배운 프롬프트 기법이 Day3 이후로 명시적 복습 없이 프로젝트에서만 암묵적으로 사용
   - **제안**: 매일 아침 10분 "전일 핵심 3문항 퀴즈" + 정답 해설

2. **인지 부하 불균형**
   - Day1-3: 매 차시 새로운 개념 + 실습 (높은 인지 부하)
   - Day4-5: 거의 새 개념 없이 프로젝트만 (낮은 인지 부하)
   - **제안**: Day4-5에도 30분 "신규 고급 팁" 미니 강의 삽입 (예: 프롬프트 디버깅 기법, 성능 최적화 팁)

3. **메타인지 활동 부재**
   - "무엇을 배웠는지" 돌아보는 구조화된 성찰 시간이 없음
   - **제안**: 매일 종료 15분 "오늘의 TIL(Today I Learned) 카드" 작성 + 1인 1문장 공유

4. **다양한 학습 양식(Learning Modality) 미활용**
   - 현재: 읽기(이론) + 하기(실습) 2가지만 사용
   - 누락: 듣기(토론), 가르치기(동료 교수), 보기(데모 관찰)
   - **제안**: "짝 설명하기(Pair Explain)" — 새 개념 학습 후 짝에게 2분 설명
