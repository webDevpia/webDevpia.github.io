## 2. 다중 페르소나 비판적 분석

### 페르소나 A: 교육공학 전문가 (Dr. 이수진)

> *"커리큘럼의 학습 설계 구조, 블룸의 분류 체계 정렬, 스캐폴딩, 형성 평가 전략을 검토합니다."*

#### 긍정적 평가

1. **점진적 복잡도 설계**: 공통과정이 단일 노드 → 다중 노드 조합 → 팀 프로젝트로 이어지는 나선형 커리큘럼 구조를 따르고 있어 학습 부담을 분산시킴
2. **실습 중심 ABL/PBL 채택**: 45시간 집중 교육에 적합한 방법론으로, 이론-실습 비율이 적절
3. **차시별 학습 목표-기대효과-학습활동 3요소 명시**: 교수설계의 기본 구조를 갖춤

#### 비판적 지적

1. **블룸 분류 체계 역전 현상**
   - 공통과정 7차시에 "업무 분석 및 자동화 기획"(분석/평가 수준)이 등장하는데, 이는 1-6차시의 "이해/적용" 수준 학습만으로는 도달하기 어려운 인지 단계
   - **제안**: 1일차에 간단한 업무 분석 프레임을 먼저 소개하고, 각 실습마다 "이 기술이 어떤 업무에 적용 가능한가?"를 반복 연결하여 7차시의 종합 분석으로 자연스럽게 이어지게 설계

2. **형성 평가(Formative Assessment) 부재**
   - 15차시 중 어디에서도 퀴즈, 체크리스트, 자기평가, 동료평가 도구가 설계되어 있지 않음
   - 5일 집중 과정에서 학습자의 이해도 격차가 3일차부터 급격히 벌어질 위험
   - **제안**: 매일 마지막 30분을 "Daily Check-in"으로 편성하여 핵심 개념 5문항 자가진단 + 막힌 부분 공유

3. **두 과정 간 내용 중복률 약 40%**
   - 프롬프트 엔지니어링, 조건 분기, 구글 시트, Gmail/Slack 연동이 양 과정에 거의 동일하게 등장
   - 동일 대상이 두 과정을 순차 수강한다면 Day2-Day3이 거의 복습에 해당
   - **제안**: 공통과정에서 기초 체득 → 전문과정에서는 "복습"이 아닌 "심화 확장" 관점으로 재설계. 예: 공통에서 단일 시트 쓰기 → 전문에서 다중 시트 조건부 읽기/쓰기 + 에러 핸들링

4. **대상 불일치 문제**
   - 공통과정이 "비전공/전공 혼합"으로 설계되어 있으나 실제 수강자는 CS 3-4학년 전공자
   - 1차시의 "Agentic AI 개념" 설명에 3시간은 전공자에게 과다
   - **제안**: 전공자 대상으로 1차시를 1.5시간으로 압축하고, 남은 시간을 심화 실습에 배분

5. **프로젝트 시간 배분 불균형**
   - 공통과정: 15차시 중 프로젝트 5차시(33%) — 기획부터 발표까지 5×3시간=15시간
   - 전문과정: 프로젝트 관련 8차시(53%) — 적절
   - 공통과정의 프로젝트 15시간 내에 기획→구현→외부연동→테스트→발표를 모두 해야 하는데, 팀 빌딩과 아이디어 선정에 최소 3시간이 소요되면 실제 구현 시간은 6-9시간으로 제한적
   - **제안**: 공통과정 Day3의 업무 분석(7차시)을 프로젝트 기획 시간과 통합하여 실질 구현 시간 확보

### 페르소나 B: AI/ML 현업 엔지니어 (김태현, 5년차 MLOps)

> *"실무 현장에서 실제로 쓰이는 AI 에이전트 기술과의 정합성, 기술적 깊이를 검토합니다."*

#### 긍정적 평가

1. **No-Code에서 시작하는 접근**: AI 에이전트 첫 경험자에게 코드 없이 핵심 개념(워크플로우, 프롬프트, 분기, 연동)을 체득시키는 전략은 실무적으로 유효
2. **RAG 기반 지식베이스 포함**: 현업에서 가장 수요가 높은 패턴 중 하나를 다루고 있음
3. **외부 서비스 연동 실습**: Gmail, Slack, Google Sheets는 실제 기업 업무 자동화의 핵심 연동 대상

#### 비판적 지적

1. **Agentria 3계층 아키텍처(Node → Ability → Agent) 미반영**
   - Agentria는 Node(실행 단위) → Ability(워크플로우) → Agent(자율 판단 주체)의 3계층 구조인데, 커리큘럼이 이 구분을 명확히 다루지 않음
   - 공통과정은 대부분 Ability 수준에서 끝나고, Agent(ReAct 노드 기반 자율 판단)까지 도달하지 않음
   - 전문과정에서도 ReAct 노드, 장기 기억, 에이전트 자율 판단 흐름이 빠져 있음
   - **제안**: 전문과정에 "Ability vs Agent 차이 이해" + "ReAct 노드 기반 자율형 에이전트 구축" 차시를 반드시 추가

2. **멀티 에이전트 패턴 부재**
   - 2025-2026년 AI 에이전트 트렌드의 핵심은 단일 에이전트가 아닌 **멀티 에이전트 오케스트레이션**
   - 현행 설계안에는 에이전트 간 협업/위임 패턴이 전혀 없음
   - **제안**: 전문과정 후반에 "여러 Ability를 조합한 Agent 구현" + "에이전트 간 데이터 파이프라인" 실습 추가

3. **Python 노드 활용도 극히 낮음**
   - Agentria는 Python 노드(pandas, numpy, datetime 지원)를 제공하는데, CS 전공자임에도 Python 코드 노드를 활용한 데이터 전처리/후처리가 커리큘럼에 없음
   - **제안**: 전공자 강점을 살려 "Python 노드로 데이터 가공 → LLM 노드로 분석 → 결과 시트 저장" 같은 하이브리드 파이프라인 실습 추가

4. **프롬프트 엔지니어링의 기술적 깊이 부족**
   - Persona, 출력 형식, 분류 정도만 다루고 있으며 Chain-of-Thought, Few-shot, Self-consistency 등 고급 기법 미포함
   - 전공자라면 "왜 이 프롬프트가 더 나은가?"를 실험적으로 검증하는 A/B 테스트 관점이 필요
   - **제안**: 전문과정에 "프롬프트 A/B 테스트 + Bulk Run으로 정량 평가" 차시 추가

5. **모델 선택/비용 최적화 관점 부재**
   - Agentria가 Gemini, GPT 등 다중 모델을 지원하는데, 어떤 모델을 왜 선택하는지, 토큰 비용 대비 성능 트레이드오프를 다루지 않음
   - 실무에서 모델 선택은 핵심 의사결정 중 하나
   - **제안**: "동일 프롬프트를 GPT-4, GPT-3.5, Gemini로 실행하여 품질/속도/비용 비교" 실습 추가

6. **배포/운영 관점 전무**
   - Agentria는 Cloud API 배포, 버전 관리, 스케줄러 기능을 제공하는데 커리큘럼에 배포 실습이 없음
   - 만들고 끝나는 것이 아니라 "서비스로 배포"까지 경험하는 것이 실무 관점에서 핵심
   - **제안**: 전문과정 12차시 "리팩토링" 후 "API 배포 + Postman 테스트" 단계를 추가

### 페르소나 C: 대학교 CS 4학년 학생 (박민지)

> *"수강자 관점에서 학습 동기, 난이도 적절성, 지루함/과부하 지점, 포트폴리오 가치를 평가합니다."*

#### 긍정적 평가

1. **No-Code라서 진입 장벽이 낮음**: 코딩만 하다가 처음 접하는 AI 에이전트를 시각적으로 배울 수 있어 흥미로울 것
2. **팀 프로젝트 + 발표**: 포트폴리오 소재가 될 수 있음
3. **실제 서비스 연동(이메일, Slack)**: 직접 동작하는 것을 보면 성취감이 높을 것

#### 비판적 지적

1. **Day1 전반이 지루할 위험이 높음**
   - CS 전공자에게 "LLM이란 무엇인가", "AI vs 에이전트 차이점" 같은 개념 설명을 3시간 동안 하면 집중력이 급격히 떨어짐
   - 우리는 이미 AI, ML 수업을 들었기 때문에 기본 개념은 30분이면 충분
   - **제안**: Day1 1차시를 "개념 30분 + Agentria 환경 셋업 + 첫 에이전트 완성까지" 압축형으로 재설계. "만들면서 배우기(Learning by Doing)" 방식이 훨씬 효과적

2. **"비전공자도 할 수 있는" 난이도가 오히려 동기 저하 요인**
   - 공통과정이 비전공자 기준으로 설계되어 있어서, CS 전공자 입장에서는 "이걸 왜 5일이나 배워야 하지?" 느낌
   - 특히 Day2의 텍스트 분류, 조건 분기가 프로그래밍 경험자에게는 너무 단순
   - **제안**: 전공자용으로 난이도를 올리고, 동일 시간에 더 복잡한 시나리오(다단계 분기, 에러 핸들링 포함)를 다루기

3. **프로젝트 자유도 제한**
   - "구글 시트 저장 + 이메일 알림" 조합이 거의 고정 패턴으로 느껴짐
   - 좀 더 창의적인 프로젝트(챗봇 서비스, 데이터 분석 파이프라인, 자동화 대시보드 등)를 선택할 수 있으면 좋겠음
   - **제안**: 프로젝트 카테고리를 3-4개 제시하고 팀별로 선택하게 하여 다양성 확보

4. **취업 포트폴리오 관점에서의 가치**
   - No-Code 플랫폼으로 만든 결과물이 개발자 포트폴리오에 얼마나 어필될지 우려
   - **제안**: 전문과정에서 "Agentria API를 호출하는 프론트엔드 앱 연동" 또는 "Python 코드 + Agentria 하이브리드" 프로젝트를 포함하여 코딩 역량도 보여줄 수 있게

5. **하루 9시간 집중도 관리**
   - 하루 3차시×3시간 = 9시간은 체력적으로 매우 힘듬
   - 오후 마지막 차시에서 집중도가 급격히 떨어질 것
   - **제안**: 오후 마지막 차시는 팀 활동/토론/프로젝트 작업처럼 활동적인 세션으로 배치. 강의형 이론은 오전에 집중

### 페르소나 D: 커리큘럼 품질 감사관 (QA Auditor, 정혜원)

> *"과정 간 일관성, 중복, 누락, 평가 기준, 시간 배분의 합리성을 정밀 감사합니다."*

#### 감사 결과

1. **[CRITICAL] 일정 충돌**
   - 두 과정 모두 동일한 날짜(2026-07-06~07-10)로 설정되어 있음
   - 동일 학생이 순차 수강이라면 총 10일(07-06~07-17)이어야 함
   - 다른 그룹이 동시 수강이라면 강사가 2명 필요
   - **시정 필요**: 일정을 명확히 분리하거나 운영 방식을 명시

2. **[CRITICAL] 중복 콘텐츠 상세 매핑**

   | 토픽 | 공통과정 | 전문과정 | 중복도 |
   |------|---------|---------|--------|
   | 프롬프트 엔지니어링(Persona/출력형식) | 2-3차시 (6h) | 3차시 (3h) | 높음 |
   | 조건 분기(Branch 노드) | 5차시 (3h) | 5차시 (3h) | 동일 |
   | 구글 시트 연동 | 9차시 (3h) | 6차시 (3h) | 동일 |
   | Gmail/Slack 알림 | 6차시 (3h) | 7차시 (3h) | 동일 |
   | PDF/지식베이스(RAG) | 8차시 (3h) | 4차시 (3h) | 높음 |

   - **총 중복: ~18시간/45시간 = 40%**
   - 순차 수강 시 학습자가 18시간을 사실상 복습에 사용하게 됨

3. **[MAJOR] 평가 체계 부재**
   - 두 과정 모두 "3학점"으로 표기되어 있으나 평가 기준(배점, 루브릭)이 전혀 없음
   - 대학교 과정이라면 성적 평가 기준이 필수
   - **제안**: 출석(10%) + Daily 미니과제(20%) + 중간 산출물(20%) + 최종 프로젝트(40%) + 동료평가(10%) 등 명시

4. **[MAJOR] 전문과정 차별성 불명확**
   - 공통과정 "중급" / 전문과정 "고급"으로 표기되어 있으나, 실제 내용 난이도 차이가 미미
   - 전문과정의 고유 심화 콘텐츠: UX/예외처리(10차시), 외부 API(11차시), 리팩토링(12차시) = 9시간뿐
   - **제안**: 전문과정에 Agent(ReAct), 멀티 에이전트, API 배포, 성능 최적화 등 공통과정에 없는 고급 주제를 최소 15시간 이상 편성

5. **[MINOR] 용어 불일치**
   - Agentria의 공식 용어는 "Ability"(워크플로우)와 "Agent"(자율형)인데, 커리큘럼에서 "에이전트"라는 용어를 Ability 수준의 워크플로우에도 혼용
   - 학습자 혼란 유발 가능
   - **제안**: 공통과정은 "Ability(워크플로우) 중심", 전문과정은 "Agent(자율형) 중심"으로 용어와 학습 범위를 일치시키기

3. **산업 연계 사례 부족**
   - "고객 리뷰 분류", "회의록 요약" 같은 예시가 반복되며 산업별 특화 사례가 없음

## 6. 결론

현행 교육설계안은 Agentria 플랫폼의 핵심 기능을 체계적으로 전달하는 기본 골격을 갖추고 있으나, **3가지 핵심 문제**의 해결이 필요합니다:

1. **두 과정 간 40% 콘텐츠 중복을 제거**하고, 공통과정은 "Ability(워크플로우) 마스터", 전문과정은 "Agent(자율형) + 고급 통합"으로 명확히 차별화
2. **CS 전공자 대상에 맞는 난이도 조정**: 기초 개념 압축, Python 노드 활용, 프롬프트 A/B 테스트, 모델 비교 등 전공자 강점을 살리는 콘텐츠 추가
3. **Agentria 3계층(Node → Ability → Agent) 완전 정복**: 현행 설계안이 Ability 수준에 머무르는 한계를 극복하고, ReAct 기반 자율형 Agent + API 배포까지 도달
