# 02장. 개발 환경 설정 + 첫 API

> 식당을 열기 전에 주방을 세팅해야 합니다. 가스레인지, 냉장고, 조리도구를 준비하는 과정이 개발 환경 설정입니다.
> 한 번만 제대로 세팅하면 이후 모든 챕터에서 그냥 코드만 작성하면 됩니다.

<a id="part1"></a>

## 1️⃣ Python 버전 확인 [↑](#toc)

FastAPI 0.135.x는 **Python 3.10 이상**이 필요합니다. 이 과정은 **3.12를 권장**합니다.

```bash
# 현재 Python 버전 확인
python --version
# 또는
python3 --version
```

실행 결과:
```
Python 3.12.3
```

버전이 3.10 미만이라면 [python.org/downloads](https://www.python.org/downloads/)에서 최신 버전을 설치하세요.

### 문제 → 해결: `python` 명령어가 없는 경우

macOS/Linux에서는 `python3`를 사용해야 하는 경우가 있습니다.

```bash
# ❌ 에러가 나는 경우
python --version
# command not found: python

# ✅ macOS/Linux에서는 python3로 시도
python3 --version
```

이 과정에서는 앞으로 `python`으로 통일하겠습니다. macOS 사용자는 `python3`로 대체해서 입력하세요.

## 2️⃣ 가상환경 만들기 [↑](#toc)

> 식당에 비유하면, 가상환경은 **각 요리사의 개인 조리 공간**입니다.
> 프로젝트 A에서 쓰는 재료(라이브러리)가 프로젝트 B에 섞이지 않도록 격리합니다.

### 방법 1 — 표준 방법: `python -m venv`

```bash
# 1. 프로젝트 폴더 만들기
mkdir my-fastapi && cd my-fastapi

# 2. 가상환경 생성 (.venv라는 폴더에 생성)
python -m venv .venv

# 3. 가상환경 활성화
# macOS/Linux:
source .venv/bin/activate

# Windows (Command Prompt):
.venv\Scripts\activate.bat

# Windows (PowerShell):
.venv\Scripts\Activate.ps1
```

활성화되면 터미널 앞에 `(.venv)` 표시가 나타납니다:

```
(.venv) ➜ my-fastapi $
```

### 방법 2 — 더 빠른 방법: `uv` (권장)

`uv`는 Rust로 만들어진 초고속 Python 패키지 관리자입니다. `pip`보다 10~100배 빠릅니다.

```bash
# uv 설치 (한 번만)
curl -LsSf https://astral.sh/uv/install.sh | sh
# 또는 Windows:
# powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# uv로 프로젝트 초기화 (가상환경 자동 생성)
uv init my-fastapi
cd my-fastapi

# 가상환경 활성화
source .venv/bin/activate
```

이 과정에서는 두 방법 모두 사용할 수 있습니다. 명령어는 `pip` 기준으로 작성하지만, `uv`를 사용한다면 `pip install` 대신 `uv add`를 사용하면 됩니다.

## 3️⃣ FastAPI 설치 [↑](#toc)

```bash
# fastapi[standard] — FastAPI + uvicorn + 추가 도구 모두 설치
pip install "fastapi[standard]"

# uv를 사용하는 경우:
# uv add "fastapi[standard]"
```

`fastapi[standard]`는 다음을 한 번에 설치합니다:

| 패키지 | 역할 |
|--------|------|
| `fastapi` | 웹 프레임워크 본체 |
| `uvicorn[standard]` | ASGI 서버 (API를 실행하는 서버) |
| `pydantic` | 데이터 검증 |
| `python-multipart` | 파일 업로드 지원 |
| `email-validator` | 이메일 형식 검증 |

설치 확인:

```bash
python -c "import fastapi; print(fastapi.__version__)"
```

실행 결과:
```
0.135.0
```

## 4️⃣ 첫 API 작성 [↑](#toc)

> 식당의 첫 번째 메뉴를 등록하는 순간입니다. 단 몇 줄로 실제 작동하는 API가 생깁니다.

프로젝트 폴더 안에 `main.py` 파일을 만들고 아래 코드를 작성합니다.

```python
# main.py
from fastapi import FastAPI  # FastAPI 프레임워크 가져오기

app = FastAPI()  # 앱 인스턴스 생성 (식당 오픈!)


@app.get("/")  # GET 요청을 "/" URL에서 처리
def read_root():
    # 딕셔너리를 반환하면 자동으로 JSON으로 변환됩니다
    return {"message": "안녕하세요! 첫 번째 API입니다"}
```

### 개발 서버 실행

```bash
# fastapi dev — 자동 재시작이 되는 개발 모드
fastapi dev main.py
```

실행 결과:
```
INFO     Using path main.py
INFO     Loaded module from ...
INFO     Using import string: main:app

 ╭────────── FastAPI CLI - Development mode ───────────╮
 │                                                     │
 │  Serving at: http://127.0.0.1:8000                  │
 │                                                     │
 │  API docs: http://127.0.0.1:8000/docs               │
 │                                                     │
 │  Running in development mode, for production use:   │
 │                                                     │
 │  fastapi run                                        │
 │                                                     │
 ╰─────────────────────────────────────────────────────╯

INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process using WatchFiles
```

브라우저에서 `http://localhost:8000`에 접속하면 아래 JSON을 볼 수 있습니다:

```json
{"message": "안녕하세요! 첫 번째 API입니다"}
```

### 코드 분석

```python
app = FastAPI()
```

`FastAPI()` 인스턴스가 바로 애플리케이션입니다. 이 객체에 모든 엔드포인트(URL 경로)를 등록합니다.

```python
@app.get("/")
```

데코레이터입니다. `GET /` 요청이 들어오면 아래 함수를 실행하라고 FastAPI에게 알려줍니다.

```python
def read_root():
    return {"message": "안녕하세요! 첫 번째 API입니다"}
```

파이썬 딕셔너리를 반환하면 FastAPI가 자동으로 JSON으로 변환해서 응답합니다.

### 엔드포인트 추가해보기

`main.py`를 수정해서 엔드포인트를 더 추가해봅니다. 파일을 저장하면 서버가 자동으로 재시작됩니다.

```python
# main.py
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
def read_root():
    return {"message": "안녕하세요! 첫 번째 API입니다"}


@app.get("/hello/{name}")
def say_hello(name: str):
    # name 파라미터를 URL에서 받아서 사용합니다
    return {"message": f"안녕하세요, {name}님!"}


@app.get("/health")
def health_check():
    # 서버 상태 확인용 엔드포인트 (모든 API에 넣는 관례)
    return {"status": "ok"}
```

- `http://localhost:8000/hello/홍길동` → `{"message": "안녕하세요, 홍길동님!"}`
- `http://localhost:8000/health` → `{"status": "ok"}`

## 5️⃣ Swagger UI 체험 [↑](#toc)

> 이것이 FastAPI의 마법입니다!
> 코드를 작성했을 뿐인데, 전문적인 API 문서가 자동으로 만들어졌습니다.

브라우저에서 `http://localhost:8000/docs`에 접속합니다.

### Swagger UI 화면 구성

```
┌─────────────────────────────────────────────┐
│  FastAPI  0.1.0  OAS 3.1                   │
├─────────────────────────────────────────────┤
│  GET  /              read_root              │
│  GET  /hello/{name}  say_hello              │
│  GET  /health        health_check           │
└─────────────────────────────────────────────┘
```

각 엔드포인트를 클릭하면 세부 정보가 펼쳐집니다.

### "Try it out" 버튼으로 직접 테스트

1. `GET /hello/{name}` 행을 클릭합니다
2. 오른쪽 **"Try it out"** 버튼을 클릭합니다
3. `name` 파라미터 칸에 `홍길동`을 입력합니다
4. **"Execute"** 버튼을 클릭합니다

응답 섹션에서 아래를 확인할 수 있습니다:

```
# Request URL
http://localhost:8000/hello/홍길동

# Response body
{
  "message": "안녕하세요, 홍길동님!"
}

# Response headers
content-type: application/json
```

**Swagger UI에서 확인해보세요!** — 앞으로 새 엔드포인트를 만들 때마다 `/docs`에서 즉시 테스트하는 습관을 만드세요. Postman 같은 별도 도구 없이 모든 테스트가 가능합니다.

### ReDoc — 읽기 전용 문서

`http://localhost:8000/redoc`에 접속하면 더 깔끔한 형태의 읽기 전용 문서를 볼 수 있습니다. 이쪽은 외부에 API 명세를 공유할 때 적합합니다.

## 6️⃣ async def vs def [↑](#toc)

> 식당 주방에서 요리사가 한 요리를 기다리는 동안 다른 요리를 할 수 있다면 효율적입니다.
> `async def`가 바로 그런 방식으로 작동합니다.

FastAPI는 두 가지 함수 형태를 모두 지원합니다.

```python
# 일반 함수 — 동기(Synchronous)
@app.get("/sync-example")
def sync_endpoint():
    return {"type": "동기 함수"}


# 비동기 함수 — 비동기(Asynchronous)
@app.get("/async-example")
async def async_endpoint():
    return {"type": "비동기 함수"}
```

### 언제 무엇을 쓰나요?

| 상황 | 권장 |
|------|------|
| 지금 (인메모리 CRUD) | `def` |
| DB 쿼리 (SQLModel async) | `async def` |
| 외부 API 호출 (httpx async) | `async def` |
| 단순 계산만 하는 경우 | `def` |

지금 단계에서는 `def`를 사용합니다. DB를 배울 9장부터 `async def`를 본격적으로 사용합니다.

### 중요한 주의사항

```python
# ✅ 올바른 사용 — async 함수 안에서 await
@app.get("/items/{item_id}")
async def get_item(item_id: int):
    item = await db.get(item_id)  # await 필수!
    return item


# ❌ 잘못된 사용 — async 함수인데 await 없이 동기 DB 라이브러리 호출
@app.get("/items/{item_id}")
async def get_item_wrong(item_id: int):
    item = db.get(item_id)  # await 없음 — 성능 문제 발생
    return item
```

지금은 이 개념을 완전히 이해하지 않아도 됩니다. `def`로 시작하고, DB 챕터에서 자세히 배웁니다.

## 7️⃣ VS Code 설정 [↑](#toc)

VS Code를 사용한다면 아래 확장을 설치하면 개발이 훨씬 편해집니다.

### 필수 확장

**Python** (Microsoft 공식)
- 확장 ID: `ms-python.python`
- 역할: Python 파일 실행, 디버깅, 가상환경 선택

**Pylance** (Microsoft 공식)
- 확장 ID: `ms-python.vscode-pylance`
- 역할: 타입 힌트 기반 자동완성, 에러 표시

### 설정 방법

1. VS Code에서 `Ctrl+Shift+X` (Mac: `Cmd+Shift+X`)로 확장 패널 열기
2. "Python" 검색 후 Microsoft 공식 확장 설치
3. "Pylance" 검색 후 설치

### 인터프리터 선택

VS Code에서 Python 인터프리터를 가상환경으로 지정해야 합니다.

1. `Ctrl+Shift+P` (Mac: `Cmd+Shift+P`)
2. "Python: Select Interpreter" 입력
3. `.venv`가 포함된 항목 선택

올바르게 설정되면 VS Code 하단 상태바에 Python 버전이 표시됩니다.

### settings.json 권장 설정

프로젝트 루트에 `.vscode/settings.json` 파일을 만들고 아래를 추가합니다.

```json
{
  "python.defaultInterpreterPath": ".venv/bin/python",
  "python.analysis.typeCheckingMode": "basic"
}
```

### 실습 과제

**기본**: `main.py`에 `GET /about` 엔드포인트를 추가하고, `{"author": "여러분의 이름", "version": "1.0.0"}`을 반환하는 API를 만들어보세요. Swagger UI에서 확인해보세요!

**중급**: `GET /add/{a}/{b}` 엔드포인트를 만들어서 두 숫자를 더한 결과를 반환해보세요. URL에서 `a`와 `b`를 `int` 타입으로 받고, `{"result": 합계}`를 반환하세요. `/add/3/7`에 접속하면 `{"result": 10}`이 나와야 합니다.

**심화**: `fastapi run main.py`(운영 모드)와 `fastapi dev main.py`(개발 모드)의 차이를 공식 문서에서 찾아보세요. 개발 모드에서는 어떤 추가 기능이 활성화되는지 정리해보세요.
