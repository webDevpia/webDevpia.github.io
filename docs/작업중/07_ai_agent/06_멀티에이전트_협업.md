## 1️⃣ 왜 멀티에이전트인가? [↑](#toc)

### 전문가 팀 비유

> 혼자서 모든 일을 하는 것보다, **전문가 팀이 나눠서 하는 것**이 빠르고 정확합니다.
>
> 신문사를 예로 들면: 한 기자가 취재 + 집필 + 편집 + 팩트체크를 모두 하면 품질이 떨어집니다.
> 취재기자 + 작가 + 편집자 + 팩트체커가 분업하면 더 좋은 결과물이 나옵니다.

### 단일 에이전트 vs 멀티에이전트

```
단일 에이전트:
  "뉴스 기사 작성해줘"
  → 에이전트 1명이 조사 + 작성 + 검토 모두 처리
  → 각 단계에 최적화되지 않음

멀티에이전트:
  뉴스 검색 전문가 → 최신 뉴스 수집
         ↓
  기사 작성 전문가 → 수집된 정보로 기사 작성
         ↓
  편집 전문가     → 문법, 스타일, 사실 검토
  → 각 단계가 전문화되어 품질 향상
```

### 멀티에이전트가 유리한 상황

| 상황 | 이유 |
|------|------|
| 작업을 명확하게 단계로 나눌 수 있을 때 | 각 에이전트가 한 단계에 집중 가능 |
| 서로 다른 전문성이 필요할 때 | 리서처, 작가, 번역가 등 역할 분리 |
| 결과물에 검증/검토가 필요할 때 | 별도의 검토자 에이전트 추가 |
| 긴 컨텍스트 처리가 필요할 때 | 단계별로 정보를 전달하여 컨텍스트 분산 |

## 2️⃣ Sequential Process [↑](#toc)

### 릴레이 경주 비유

> **Sequential(순차) 방식**은 릴레이 경주와 같습니다.
> 첫 번째 주자(Agent 1)가 달리고 바통을 넘기면, 두 번째 주자(Agent 2)가 이어받아 달립니다.
> 각 Task는 이전 Task가 완료된 후에 시작합니다.

```
Task 1 (리서치)  →  Task 2 (작성)  →  Task 3 (편집)
   Agent 1           Agent 2           Agent 3
   완료 후              완료 후              완료
   결과 전달           결과 전달
```

### Sequential 코드

```python
from crewai import Crew, Process

crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, writing_task, editing_task],
    process=Process.sequential,  # 순서대로 실행
    verbose=True
)
```

### Sequential의 특징

| 항목 | 내용 |
|------|------|
| **실행 순서** | tasks 리스트 순서대로 |
| **결과 전달** | `context` 파라미터로 명시적 연결 |
| **사용 시기** | 단계가 명확하고 순서가 중요할 때 |
| **난이도** | ★★☆☆☆ (가장 기본, 예측 가능) |

## 3️⃣ Hierarchical Process [↑](#toc)

### 팀장 회의 비유

> **Hierarchical(계층적) 방식**은 팀장이 업무를 분배하는 회의와 같습니다.
> 매니저 에이전트가 전체 목표를 보고, 각 팀원 에이전트에게 알맞은 Task를 배분합니다.
> 팀원들이 완료한 결과를 수집하고, 필요하면 재배분도 합니다.

```
         Manager Agent
        /      |      \
   Agent 1  Agent 2  Agent 3
   (조사)   (작성)   (번역)
        \      |      /
         최종 결과물
```

### Hierarchical 코드

```python
from crewai import Agent, Crew, Process, LLM

llm = LLM(model="gpt-4o-mini")

# 방법 1: manager_llm 지정 (CrewAI가 자동으로 매니저 에이전트 생성)
crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, writing_task, editing_task],
    process=Process.hierarchical,
    manager_llm=llm,             # 매니저가 사용할 LLM
    verbose=True
)

# 방법 2: manager_agent 직접 지정
manager = Agent(
    role="콘텐츠 팀 매니저",
    goal="팀원들의 작업을 조율하여 최고 품질의 콘텐츠를 완성한다",
    backstory=(
        "당신은 10년 경력의 콘텐츠 팀 디렉터입니다. "
        "팀원들의 강점을 파악하고 적절한 업무를 배분하는 능력이 탁월합니다."
    ),
    llm=llm,
    allow_delegation=True   # 다른 에이전트에게 위임 허용
)

crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, writing_task, editing_task],
    process=Process.hierarchical,
    manager_agent=manager,   # 직접 만든 매니저 사용
    verbose=True
)
```

### Sequential vs Hierarchical 비교

| 항목 | Sequential | Hierarchical |
|------|-----------|-------------|
| **제어 방식** | Task 순서대로 자동 실행 | 매니저가 동적으로 배분 |
| **유연성** | 낮음 — 순서 고정 | 높음 — 상황에 따라 변경 |
| **예측 가능성** | 높음 | 낮음 (매니저 판단 의존) |
| **적합한 상황** | 단계가 명확할 때 | 복잡하고 동적인 작업 |
| **API 비용** | 낮음 | 높음 (매니저 호출 추가) |
| **난이도** | ★★☆☆☆ | ★★★★☆ |

> **입문 추천**: Sequential Process로 시작하세요. 동작을 이해한 뒤 Hierarchical로 넘어가세요.

## 4️⃣ 실습: 콘텐츠 제작 팀 [↑](#toc)

리서처 + 작가 + 편집자 3인 팀으로 블로그 포스팅을 제작합니다.

```python
# 파일: content_team.py
from dotenv import load_dotenv
load_dotenv()

from crewai import Agent, Task, Crew, Process, LLM

llm = LLM(model="gpt-4o-mini")

# ── 에이전트 3명 설정 ──────────────────────────────────
researcher = Agent(
    role="IT 트렌드 리서치 전문가",
    goal="AI 관련 최신 트렌드와 핵심 정보를 수집하고 정리한다",
    backstory=(
        "당신은 IT 전문 미디어에서 10년간 근무한 테크 저널리스트입니다. "
        "복잡한 기술 동향을 핵심 포인트로 압축하는 것이 특기입니다."
    ),
    llm=llm,
    verbose=True
)

writer = Agent(
    role="테크 블로그 전문 작가",
    goal="비개발자도 이해할 수 있는 쉽고 재미있는 기술 블로그 글을 작성한다",
    backstory=(
        "당신은 IT 비전공자 독자를 위한 테크 블로그를 운영합니다. "
        "어려운 개념을 일상적인 비유로 쉽게 설명하는 것이 특기입니다. "
        "구독자 30만 명의 신뢰받는 작가입니다."
    ),
    llm=llm,
    verbose=True
)

editor = Agent(
    role="콘텐츠 편집장",
    goal="글의 품질, 가독성, 사실 정확성을 검토하고 최종 완성도를 높인다",
    backstory=(
        "당신은 15년 경력의 편집장으로, 수백 편의 기술 아티클을 편집했습니다. "
        "문법 오류, 논리 흐름, 독자 친화성을 동시에 검토하는 능력이 있습니다."
    ),
    llm=llm,
    verbose=True
)

# ── Task 3개 설정 ──────────────────────────────────────
research_task = Task(
    description=(
        "'AI 에이전트'가 실생활에서 활용되는 최신 사례 3가지를 조사하세요.\n"
        "각 사례마다: 어떤 회사/서비스인지, 어떻게 에이전트를 활용하는지, "
        "어떤 결과를 얻었는지를 포함해주세요."
    ),
    expected_output=(
        "AI 에이전트 활용 사례 3가지를 포함한 리서치 보고서 (마크다운). "
        "각 사례마다 회사명, 활용 방식, 성과를 구조적으로 정리."
    ),
    agent=researcher
)

writing_task = Task(
    description=(
        "리서치 결과를 바탕으로 'AI 에이전트, 이미 우리 곁에 있다' 라는 주제의 "
        "블로그 포스팅을 작성하세요.\n"
        "대상 독자: AI에 관심 있는 직장인 (비개발자)\n"
        "톤: 친근하고 이해하기 쉽게, 비유 적극 활용\n"
        "길이: 800자 이상"
    ),
    expected_output=(
        "제목, 도입부, 본문(3가지 사례 설명), 마무리를 포함한 블로그 포스팅. "
        "비유와 예시로 이해하기 쉽게 작성된 800자 이상의 글."
    ),
    agent=writer,
    context=[research_task]
)

editing_task = Task(
    description=(
        "작성된 블로그 글을 편집하고 최종 완성본을 작성하세요.\n"
        "다음 항목을 검토하고 개선하세요:\n"
        "1. 문법 및 맞춤법 오류\n"
        "2. 문장 흐름과 가독성\n"
        "3. 도입부가 독자의 관심을 끄는지\n"
        "4. 핵심 메시지가 명확한지\n"
        "5. 마무리가 인상적인지"
    ),
    expected_output=(
        "편집이 완료된 최종 블로그 포스팅. "
        "원본과 달라진 주요 수정 사항 요약 포함."
    ),
    agent=editor,
    context=[writing_task]
)

# ── Crew 실행 ──────────────────────────────────────────
crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, writing_task, editing_task],
    process=Process.sequential,
    verbose=True
)

result = crew.kickoff()

print("\n" + "=" * 60)
print("최종 편집 완료 포스팅:")
print("=" * 60)
print(result)
```


## 5️⃣ 정리 [↑](#toc)

### Sequential vs Hierarchical 선택 가이드

```
단계가 명확하고 순서가 중요한가?
  ↓ YES                  ↓ NO
Sequential          동적 배분이 필요한가?
(가장 적합)           ↓ YES          ↓ NO
               Hierarchical    Sequential
                              (기본값 사용)
```

### 멀티에이전트 설계 체크리스트

- [ ] 각 에이전트의 역할이 명확하게 분리되어 있는가?
- [ ] Task의 순서가 논리적인가? (조사 → 작성 → 검토)
- [ ] 이전 Task 결과가 필요한 Task에 `context`가 연결되어 있는가?
- [ ] 에이전트 수가 필요 이상으로 많지 않은가? (3~5명 권장)

### 핵심 패턴 요약

```python
# Sequential: 3인 팀 기본 패턴
crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, writing_task, editing_task],
    process=Process.sequential
)

# Hierarchical: 매니저 추가
crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, writing_task, editing_task],
    process=Process.hierarchical,
    manager_llm=LLM(model="gpt-4o-mini")
)
```
