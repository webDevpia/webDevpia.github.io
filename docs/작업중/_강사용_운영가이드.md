## 1. 핵심 운영 원칙

### 유연한 진행 전략

16챕터를 5일에 배치하되, **학생 상태에 따라 속도를 조절**합니다.

- **진행 판단 기준**: 실습 완료율 70% 이상이면 다음 챕터로 이동
- **느릴 때**: 현재 챕터의 "핵심" 부분만 마무리하고, "도전/더 알아보기"는 건너뜀
- **빠를 때**: "도전" 과제를 진행하거나, 다음 Day 챕터를 선행

### 학생 자료에서의 우선순위 표현 방식

학생이 보는 자료에서는 직접적으로 "건너뛰세요"라고 하지 않습니다. 대신:

| 실제 의미 | 학생 자료에 쓰는 표현 |
|-----------|---------------------|
| 반드시 해야 함 | ⭐ **핵심** |
| 시간 없으면 건너뛸 수 있음 | 🚀 **도전** — "여유가 있다면 도전해 보세요!" |
| 자습으로 전환 가능 | 📖 **더 알아보기** — "수업 후에 더 탐구해 보세요" |
| 빠른 학생용 추가 과제 | 💪 **보너스 챌린지** |

## 4. 학생 상태 판단 신호

### "잘 따라오고 있다" 신호
- 실습 후 "더 할 거 없나요?" 질문
- 코드 수정 시 자신감 있게 시도
- 팀 토론에서 적극적으로 의견 제시
- 미니과제를 시간 내 제출

### "어려워하고 있다" 신호
- 강사 화면만 보고 자기 화면에서 시도 안 함
- "그냥 넘어가도 되나요?" 질문
- 에러 메시지를 보고 바로 손 듬 (시도 없이)
- 쉬는 시간에 혼자 있으려 함

### 대응 전략
- 어려워하는 학생이 **30% 이상**: 현재 챕터 속도를 줄이고, "도전" 부분 건너뛰기
- 어려워하는 학생이 **50% 이상**: 다음 챕터 진입 보류, 현재 내용 보충 실습 추가
- 빠른 학생이 **지루해할 때**: "보너스 챌린지" 안내 또는 느린 학생 페어링
