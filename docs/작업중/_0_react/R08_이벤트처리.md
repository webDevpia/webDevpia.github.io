# 07장. 이벤트 처리

## 학습 목표

이 장을 마치면 다음을 할 수 있습니다.

- `onClick`에 이벤트 핸들러 함수를 올바르게 연결할 수 있다
- 함수를 호출하지 않고 전달하는 차이를 설명할 수 있다
- 이벤트 객체 `e`에서 필요한 정보를 꺼낼 수 있다
- 핸들러에 매개변수를 전달하는 방법을 알 수 있다
- `onChange`, `onSubmit`, `onKeyDown` 등 다양한 이벤트를 사용할 수 있다
- 이벤트 버블링을 이해하고 `e.stopPropagation()`을 사용할 수 있다

<a id="1"></a>
## 1️⃣ 이벤트 = 초인종 — 비유로 시작하기 [↑](#toc)

아파트 초인종을 상상합니다.

- 초인종이 눌리면(이벤트 발생) → 집 안에서 "딩동" 소리가 납니다(핸들러 실행)
- 초인종을 설치할 때 "눌리면 이 함수를 실행해" 라고 등록해둡니다

React 이벤트도 같습니다.

```
버튼(초인종) → 클릭(이벤트) → handleClick(함수 실행)
```

HTML에서는 이렇게 썼습니다:

```html
<!-- HTML 방식 (문자열로 전달) -->
<button onclick="handleClick()">클릭</button>
```

React에서는 이렇게 씁니다:

```jsx
// React 방식 (함수 레퍼런스로 전달)
<button onClick={handleClick}>클릭</button>
```

두 가지 차이가 보입니다. `onclick` → `onClick` (camelCase), 문자열 → 함수 레퍼런스.

## 2️⃣ onClick — 클릭 이벤트의 기본 [↑](#toc)

### 가장 중요한 규칙: 함수를 호출하지 말고 전달하세요

```jsx
// ✅ 올바른 방법 — 함수를 "전달"합니다
<button onClick={handleClick}>클릭</button>

// ❌ 틀린 방법 — 함수를 "즉시 호출"합니다
<button onClick={handleClick()}>클릭</button>
```

왜 `handleClick()`이 틀릴까요?

```
handleClick()  → 컴포넌트가 렌더링될 때 즉시 실행됩니다
handleClick   → 클릭될 때 실행됩니다
```

`handleClick()`을 쓰면 버튼을 클릭하지도 않았는데 컴포넌트가 그려지는 순간 함수가 실행됩니다. 그리고 클릭해도 아무 반응이 없습니다.

### 기명 함수 vs 인라인 함수

```jsx
function MyButton() {
  // 방법 1: 기명 함수 (이름이 있는 함수) — 추천
  function handleClick() {
    console.log("버튼이 클릭되었습니다!");
  }

  // 방법 2: 화살표 함수로 정의
  const handleHover = () => {
    console.log("마우스가 올라갔습니다!");
  };

  return (
    <div className="space-x-3 p-4">
      {/* 기명 함수 전달 */}
      <button
        onClick={handleClick}
        className="bg-blue-500 text-white px-4 py-2 rounded"
      >
        클릭 (기명)
      </button>

      {/* 방법 3: 인라인 화살표 함수 — 짧은 로직에 적합 */}
      <button
        onClick={() => console.log("인라인 클릭!")}
        className="bg-green-500 text-white px-4 py-2 rounded"
      >
        클릭 (인라인)
      </button>

      <button
        onMouseEnter={handleHover}
        className="bg-purple-500 text-white px-4 py-2 rounded"
      >
        마우스 올리기
      </button>
    </div>
  );
}
// 출력: 세 개의 버튼
// 각 버튼 클릭/호버 시 console.log가 실행됩니다
```

### 언제 기명 함수, 언제 인라인 함수?

| 상황 | 추천 방법 |
|------|-----------|
| 로직이 2줄 이상 | 기명 함수 (`handleClick`) |
| 단순한 한 줄 | 인라인 `() => ...` |
| 재사용해야 할 때 | 기명 함수 |
| 테스트가 필요할 때 | 기명 함수 |

## 3️⃣ 이벤트 객체 (e) — 이벤트의 세부 정보 [↑](#toc)

이벤트 핸들러 함수는 자동으로 **이벤트 객체**를 첫 번째 인자로 받습니다. 이 객체에는 이벤트에 대한 모든 정보가 담겨 있습니다.

### e.target — 이벤트가 발생한 요소

```jsx
function InputLogger() {
  function handleChange(e) {
    console.log(e.target.value);       // 입력된 텍스트
    console.log(e.target.type);        // "text"
    console.log(e.target.placeholder); // 플레이스홀더 텍스트
  }

  return (
    <input
      type="text"
      placeholder="무엇이든 입력하세요"
      onChange={handleChange}
      className="border border-gray-300 rounded px-3 py-2 w-64"
    />
  );
}
// 출력: "안녕" 입력 시 콘솔에 "안녕" 출력
```

### e.preventDefault() — 기본 동작 막기

폼을 제출하면 브라우저가 기본적으로 **페이지를 새로고침**합니다. React 앱에서는 이를 막아야 합니다.

```jsx
function SearchForm() {
  function handleSubmit(e) {
    e.preventDefault(); // 페이지 새로고침 방지!
    const query = e.target.elements.search.value;
    console.log(`검색어: ${query}`);
    // 출력: "검색어: React 19" (입력값에 따라 다름)
  }

  return (
    <form onSubmit={handleSubmit} className="flex gap-2">
      <input
        name="search"
        type="text"
        placeholder="검색어 입력"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        type="submit"
        className="bg-blue-600 text-white px-4 py-2 rounded"
      >
        검색
      </button>
    </form>
  );
}
// e.preventDefault()가 없으면: 제출 시 페이지 새로고침
// e.preventDefault()가 있으면: 새로고침 없이 console.log 실행
```

### 자주 사용하는 이벤트 객체 속성

| 속성/메서드 | 설명 | 예시 |
|-------------|------|------|
| `e.target` | 이벤트가 발생한 요소 | `<input>` 요소 |
| `e.target.value` | 입력 요소의 현재 값 | `"안녕하세요"` |
| `e.target.checked` | 체크박스 체크 여부 | `true / false` |
| `e.preventDefault()` | 기본 동작 방지 | 폼 새로고침, 링크 이동 |
| `e.stopPropagation()` | 이벤트 버블링 중단 | 부모로의 전파 막기 |
| `e.key` | 눌린 키 | `"Enter"`, `"Escape"` |

## 4️⃣ 매개변수 전달 — 어떤 항목을 클릭했는지 알기 [↑](#toc)

목록에서 특정 항목의 삭제 버튼을 클릭할 때, **어떤 항목인지** 핸들러에 전달해야 합니다.

### 문제

```jsx
// ❌ 이렇게는 어떤 항목인지 알 수 없습니다
<button onClick={handleDelete}>삭제</button>
```

### 해결: 화살표 함수로 감싸기

```jsx
function TodoList() {
  const todos = [
    { id: 1, text: "React 공부하기" },
    { id: 2, text: "운동하기" },
    { id: 3, text: "장보기" },
  ];

  function handleDelete(id) {
    console.log(`${id}번 항목을 삭제합니다`);
    // 출력: "1번 항목을 삭제합니다"
  }

  function handleEdit(id, text) {
    console.log(`${id}번 항목 "${text}" 수정`);
    // 출력: '2번 항목 "운동하기" 수정'
  }

  return (
    <ul className="space-y-2 p-4">
      {todos.map((todo) => (
        <li
          key={todo.id}
          className="flex items-center justify-between bg-white p-3 rounded shadow-sm"
        >
          <span>{todo.text}</span>
          <div className="flex gap-2">
            {/* 화살표 함수로 id를 전달 */}
            <button
              onClick={() => handleEdit(todo.id, todo.text)}
              className="text-sm text-blue-600 hover:underline"
            >
              수정
            </button>
            <button
              onClick={() => handleDelete(todo.id)}
              className="text-sm text-red-600 hover:underline"
            >
              삭제
            </button>
          </div>
        </li>
      ))}
    </ul>
  );
}
// "삭제" 버튼 클릭 시 콘솔에 해당 항목 id 출력
```

> **핵심 패턴**: `onClick={() => handleDelete(todo.id)}`
>
> 화살표 함수로 감싸면 "클릭될 때" 인자를 넘겨서 함수를 호출합니다. 이게 가능한 이유는 `() =>` 자체가 함수(전달)이고, 내부의 `handleDelete(todo.id)`는 클릭 시에 실행되기 때문입니다.

## 5️⃣ 다양한 이벤트 — onChange, onSubmit, 키보드, 마우스 [↑](#toc)

### onChange — 입력값 실시간 감지

```jsx
function LiveInput() {
  // (다음 장에서 useState를 배웁니다 — 지금은 개념만)
  function handleChange(e) {
    console.log("현재 입력값:", e.target.value);
  }

  return (
    <div className="p-4">
      <input
        type="text"
        onChange={handleChange}
        placeholder="입력해보세요"
        className="border rounded px-3 py-2"
      />
    </div>
  );
}
// 글자를 입력할 때마다 콘솔에 현재 값 출력
// 출력: "현재 입력값: R" → "현재 입력값: Re" → "현재 입력값: Rea" ...
```

### 체크박스 onChange

```jsx
function CheckboxExample() {
  function handleCheck(e) {
    console.log("체크됨:", e.target.checked);
    console.log("값:", e.target.value);
  }

  return (
    <label className="flex items-center gap-2 cursor-pointer">
      <input
        type="checkbox"
        value="marketing"
        onChange={handleCheck}
        className="w-4 h-4"
      />
      <span>마케팅 이메일 수신 동의</span>
    </label>
  );
}
// 체크 시 콘솔: "체크됨: true"  "값: marketing"
// 해제 시 콘솔: "체크됨: false" "값: marketing"
```

### onKeyDown — 특정 키 감지

```jsx
function EnterSearch() {
  function handleKeyDown(e) {
    console.log("눌린 키:", e.key);

    if (e.key === "Enter") {
      console.log("검색 실행!");
      // 출력: "눌린 키: Enter" 후 "검색 실행!"
    }

    if (e.key === "Escape") {
      console.log("검색 취소");
    }
  }

  return (
    <input
      type="text"
      placeholder="검색 후 Enter"
      onKeyDown={handleKeyDown}
      className="border rounded px-3 py-2 w-64"
    />
  );
}
```

### onMouseEnter / onMouseLeave — 호버 효과

```jsx
function HoverCard() {
  function handleMouseEnter() {
    console.log("마우스 들어옴");
  }

  function handleMouseLeave() {
    console.log("마우스 나감");
  }

  return (
    <div
      onMouseEnter={handleMouseEnter}
      onMouseLeave={handleMouseLeave}
      className="p-6 bg-gray-100 rounded-lg cursor-pointer hover:bg-blue-50 hover:shadow-md transition-all"
    >
      마우스를 올려보세요
    </div>
  );
}
// 마우스 진입 시: "마우스 들어옴"
// 마우스 이탈 시: "마우스 나감"
```

### 자주 사용하는 React 이벤트 목록

| 이벤트 | 발생 시점 | 주요 사용처 |
|--------|-----------|-------------|
| `onClick` | 마우스 클릭 | 버튼, 카드 |
| `onChange` | 값 변경 | 입력, 체크박스, select |
| `onSubmit` | 폼 제출 | form 태그 |
| `onFocus` | 포커스 획득 | 입력 필드 |
| `onBlur` | 포커스 잃음 | 유효성 검사 |
| `onMouseEnter` | 마우스 진입 | 호버 효과 |
| `onMouseLeave` | 마우스 이탈 | 호버 효과 |
| `onKeyDown` | 키 누름 | 단축키, Enter 처리 |
| `onKeyUp` | 키 뗌 | 실시간 검색 |
| `onScroll` | 스크롤 | 무한 스크롤 |

## 6️⃣ 이벤트 버블링 — 클릭이 퍼져나가는 현상 [↑](#toc)

### 이벤트 버블링이란?

물에 돌을 던지면 파문이 바깥쪽으로 퍼집니다. 이벤트도 마찬가지로, 안쪽 요소에서 발생한 이벤트가 바깥 요소로 전파됩니다.

```
<div onClick={handleOuterClick}>          ← 3. 마지막으로 여기까지 전파
  <div onClick={handleMiddleClick}>       ← 2. 그 다음 여기로 전파
    <button onClick={handleButtonClick}>  ← 1. 버튼에서 시작
      클릭
    </button>
  </div>
</div>
```

버튼을 클릭하면 `handleButtonClick` → `handleMiddleClick` → `handleOuterClick` 순서로 모두 실행됩니다.

### 문제: 의도치 않은 부모 클릭

```jsx
function CardWithButton() {
  function handleCardClick() {
    console.log("카드 클릭됨 — 상세 페이지 이동");
  }

  function handleDeleteClick(e) {
    e.stopPropagation(); // 버블링 중단!
    console.log("삭제 버튼 클릭됨 — 카드 클릭은 실행 안 됨");
  }

  return (
    <div
      onClick={handleCardClick}
      className="bg-white p-4 rounded shadow cursor-pointer hover:shadow-md"
    >
      <h3 className="font-bold">상품 이름</h3>
      <p className="text-gray-600 text-sm">클릭하면 상세 페이지로...</p>

      {/* e.stopPropagation()이 없으면 삭제 버튼 클릭 시 카드도 클릭됨 */}
      <button
        onClick={handleDeleteClick}
        className="mt-2 text-xs text-red-500 hover:underline"
      >
        삭제
      </button>
    </div>
  );
}
// 삭제 버튼 클릭 시:
// stopPropagation 없으면: "삭제 버튼 클릭됨" → "카드 클릭됨" (둘 다 실행)
// stopPropagation 있으면: "삭제 버튼 클릭됨" (카드 클릭은 실행 안 됨)
```

> **요약**:
> - `e.stopPropagation()`: 이벤트가 부모로 전파되는 것을 막습니다
> - `e.preventDefault()`: 브라우저의 기본 동작을 막습니다 (새로고침, 링크 이동 등)
> - 두 개는 전혀 다른 것입니다. 목적에 맞게 사용하세요.

## 7️⃣ 실습: 좋아요 버튼 + 이벤트 로그 [↑](#toc)

### 기본 과제

버튼 클릭 이벤트를 연습하는 UI를 만드세요.

```jsx
// src/App.jsx
function EventLogger() {
  const logs = []; // 나중에 useState로 교체할 예정

  function handleLike() {
    console.log(`[${new Date().toLocaleTimeString()}] 좋아요 클릭`);
  }

  function handleShare() {
    console.log(`[${new Date().toLocaleTimeString()}] 공유 클릭`);
  }

  function handleBookmark() {
    console.log(`[${new Date().toLocaleTimeString()}] 북마크 클릭`);
  }

  return (
    <div className="min-h-screen bg-gray-100 flex items-center justify-center">
      <div className="bg-white rounded-xl shadow-md p-6 w-80">
        {/* 게시물 카드 */}
        <div className="mb-4">
          <div className="flex items-center gap-2 mb-3">
            <div className="w-8 h-8 rounded-full bg-blue-200 flex items-center justify-center text-sm">
              👤
            </div>
            <div>
              <div className="font-semibold text-sm">김철수</div>
              <div className="text-xs text-gray-400">방금 전</div>
            </div>
          </div>
          <p className="text-gray-800 text-sm leading-relaxed">
            React 이벤트 처리를 배우고 있습니다! 정말 재미있네요 🎉
          </p>
        </div>

        {/* 이벤트 버튼 영역 */}
        <div className="border-t pt-3 flex justify-around">
          <button
            onClick={handleLike}
            className="flex items-center gap-1 text-sm text-gray-600 hover:text-red-500 transition-colors px-3 py-1 rounded hover:bg-red-50"
          >
            🤍 좋아요
          </button>
          <button
            onClick={handleShare}
            className="flex items-center gap-1 text-sm text-gray-600 hover:text-blue-500 transition-colors px-3 py-1 rounded hover:bg-blue-50"
          >
            🔗 공유
          </button>
          <button
            onClick={handleBookmark}
            className="flex items-center gap-1 text-sm text-gray-600 hover:text-yellow-500 transition-colors px-3 py-1 rounded hover:bg-yellow-50"
          >
            🔖 저장
          </button>
        </div>

        {/* 이벤트 로그 (콘솔 대신 화면에) */}
        <div className="mt-4 bg-gray-900 text-green-400 rounded p-3 font-mono text-xs">
          <div className="text-gray-500 mb-1">// 버튼을 클릭하면 콘솔에서 로그를 확인하세요</div>
          <div>&gt; 버튼을 클릭해보세요...</div>
        </div>
      </div>
    </div>
  );
}

export default EventLogger;
// 각 버튼 클릭 시 브라우저 DevTools 콘솔에 로그가 출력됩니다
```

### 도전 과제

1. **호버 카운터**: 버튼에 `onMouseEnter` 이벤트를 추가해 마우스를 올린 횟수를 콘솔에 로그
2. **키보드 단축키**: `document`에 `keydown` 이벤트를 달아 `L` 키 → 좋아요, `S` 키 → 저장 처리 (힌트: `useEffect` 미리 맛보기 또는 `onKeyDown`을 `div`에)
3. **이벤트 위임**: 버튼마다 핸들러를 달지 말고, 버튼 영역 `div` 하나에 `onClick`을 달고 `e.target.dataset.action`으로 분기

## 8️⃣ 정리 + 브릿지 [↑](#toc)

### 핵심 정리

| 개념 | 요점 |
|------|------|
| 핸들러 전달 | `onClick={handleClick}` (O), `onClick={handleClick()}` (X) |
| 인라인 함수 | `onClick={() => handleDelete(id)}` |
| 이벤트 객체 | 첫 번째 인자로 자동 전달 (`e`) |
| `e.target.value` | 입력 요소의 현재 값 |
| `e.preventDefault()` | 브라우저 기본 동작 방지 |
| `e.stopPropagation()` | 부모로의 이벤트 전파 방지 |

### 체크리스트

- [ ] `onClick={handleClick}` 과 `onClick={handleClick()}` 의 차이를 설명할 수 있다
- [ ] `e.target.value`로 입력값을 꺼낼 수 있다
- [ ] `e.preventDefault()`로 폼 새로고침을 막을 수 있다
- [ ] `onClick={() => handleDelete(todo.id)}`로 매개변수를 전달할 수 있다
- [ ] 버블링을 막을 때 `e.stopPropagation()`을 쓸 수 있다

> **브릿지**: 버튼을 클릭하면 `console.log`가 출력됩니다. 그런데 좋아요 숫자가 실제로 화면에서 바뀌면 훨씬 좋겠죠? 지금은 클릭해도 UI는 변하지 않습니다. **다음 장에서 `useState`를 배우면**, 클릭할 때마다 숫자가 올라가고, 버튼 텍스트가 바뀌는 진짜 인터랙티브 UI를 만들 수 있습니다.
