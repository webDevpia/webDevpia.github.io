# 05장. 리스트 렌더링과 key

## 학습 목표

이 장을 마치면 다음을 할 수 있습니다.

- 배열 데이터를 `map()`으로 JSX 요소 목록으로 변환할 수 있다
- `key` prop이 왜 필요한지 설명할 수 있다
- 인덱스를 key로 쓰면 안 되는 상황을 알 수 있다
- 객체 배열을 컴포넌트로 분리해서 렌더링할 수 있다

## 1️⃣ map() = 출석부 한 명씩 부르기 [↑](#toc)

실제 세계에서 선생님이 출석부를 보면서 한 명씩 이름을 부릅니다.

> "김철수 — 있음. 이영희 — 있음. 박민준 — 있음..."

JavaScript의 `map()` 함수가 바로 이 역할을 합니다. 배열의 각 항목을 하나씩 꺼내서, 새로운 무언가(여기서는 JSX 요소)로 변환합니다.

```js
// 출석부 배열
const students = ["김철수", "이영희", "박민준"];

// 한 명씩 불러서 <li> 태그로 변환
const listItems = students.map((name) => <li>{name}</li>);
// 결과: [<li>김철수</li>, <li>이영희</li>, <li>박민준</li>]
```

이것이 React 리스트 렌더링의 핵심입니다. **배열 → map() → JSX 배열 → 화면**.

<a id="2"></a>
## 2️⃣ 배열 데이터 렌더링하기 [↑](#toc)

### 문제

쇼핑몰에 과일 10개를 화면에 보여줘야 합니다. 하드코딩하면 코드가 너무 길어집니다.

```jsx
// 이렇게 쓰면 안 됩니다 — 10개면 10줄, 100개면 100줄
function FruitList() {
  return (
    <ul>
      <li>사과</li>
      <li>바나나</li>
      <li>딸기</li>
      {/* ... 계속 */}
    </ul>
  );
}
```

### 고통

데이터가 바뀌면(예: 서버에서 받아온 목록) 코드도 전부 수정해야 합니다. 유지보수가 악몽입니다.

### 해결: map()으로 렌더링

```jsx
// src/components/FruitList.jsx
function FruitList() {
  const fruits = ["사과", "바나나", "딸기", "포도", "수박"];

  return (
    <ul className="list-disc pl-6 space-y-1">
      {fruits.map((fruit) => (
        <li key={fruit} className="text-gray-700">
          {fruit}
        </li>
      ))}
    </ul>
  );
}

export default FruitList;
// 출력: • 사과  • 바나나  • 딸기  • 포도  • 수박
```

> **핵심 패턴**: `{배열.map((항목) => <JSX요소 key={...}>{항목}</JSX요소>)}`

중괄호 `{}` 안에서 map()을 호출하고, 각 항목마다 JSX를 반환합니다. 바깥쪽 `()` 는 화살표 함수가 객체를 반환할 때처럼 JSX를 감쌀 때 사용합니다.

<a id="3"></a>
## 3️⃣ key prop이 필요한 이유 [↑](#toc)

### 경고 메시지

`key` 없이 map()을 쓰면 브라우저 콘솔에 이런 경고가 뜹니다.

```
Warning: Each child in a list should have a unique "key" prop.
```

이 경고를 무시하면 어떤 일이 생길까요?

### React의 재조정(Reconciliation) 원리

React는 화면을 다시 그릴 때 이전 목록과 새 목록을 비교해서 **바뀐 부분만** 업데이트합니다. 이때 각 항목이 "어떤 항목인지" 알기 위해 `key`를 사용합니다.

```
이전 목록:       새 목록:
[사과]           [바나나]  ← key로 이게 '사과'가 바뀐 건지,
[바나나]         [사과]       새 항목인지 알 수 있음
[딸기]           [딸기]
```

key가 없으면 React는 순서만 보고 판단합니다. 목록을 재정렬하거나 삭제할 때 오작동의 원인이 됩니다.

### 올바른 key 사용

```jsx
// ✅ 고유한 ID를 key로 사용
const fruits = [
  { id: 1, name: "사과" },
  { id: 2, name: "바나나" },
  { id: 3, name: "딸기" },
];

{fruits.map((fruit) => (
  <li key={fruit.id}>{fruit.name}</li>
))}
```

### ⚠️ WARNING: index를 key로 쓰면 안 되는 경우

처음 배우면 이렇게 쓰고 싶어집니다.

```jsx
// ❌ 인덱스를 key로 쓰는 것은 위험합니다
{fruits.map((fruit, index) => (
  <li key={index}>{fruit.name}</li>
))}
```

**왜 위험한가?** 다음 시나리오를 봅니다.

```
초기 상태:           항목 삭제 후 (바나나 삭제):
key=0: 사과          key=0: 사과
key=1: 바나나        key=1: 딸기  ← 원래 key=2였던 딸기가 key=1로 바뀜!
key=2: 딸기
```

React는 key=1이 바뀌었다고 생각해서, 딸기를 바나나가 변경된 것으로 인식합니다. 체크박스나 입력 필드가 있는 목록에서 상태가 엉킵니다.

**인덱스 key를 써도 안전한 경우 (이 경우에만 허용):**
- 목록의 순서가 절대 바뀌지 않는다
- 항목을 추가하거나 삭제하지 않는다
- 항목에 내부 상태(input, checkbox)가 없다

> **결론: 항상 데이터 고유 ID를 key로 사용하세요.** DB에서 가져온 데이터는 대부분 `id` 필드가 있습니다.

<a id="4"></a>
## 4️⃣ 객체 배열 렌더링 [↑](#toc)

실제 앱의 데이터는 단순 문자열이 아니라 객체 배열입니다.

```jsx
// src/components/StudentList.jsx
function StudentList() {
  const students = [
    { id: 1, name: "김철수", grade: "A", avatar: "🧑" },
    { id: 2, name: "이영희", grade: "B+", avatar: "👩" },
    { id: 3, name: "박민준", grade: "A+", avatar: "🧒" },
    { id: 4, name: "최지은", grade: "B", avatar: "👧" },
  ];

  return (
    <div className="p-4">
      <h2 className="text-xl font-bold mb-4">학생 목록</h2>
      <div className="grid grid-cols-2 gap-4">
        {students.map((student) => (
          <div
            key={student.id}
            className="bg-white border rounded-lg p-4 shadow-sm"
          >
            <div className="text-3xl mb-2">{student.avatar}</div>
            <div className="font-semibold">{student.name}</div>
            <div className="text-sm text-gray-500">성적: {student.grade}</div>
          </div>
        ))}
      </div>
    </div>
  );
}

export default StudentList;
// 출력: 2×2 그리드로 4명의 학생 카드가 나타납니다
```

**구조 분해 할당**을 쓰면 더 읽기 좋습니다.

```jsx
{students.map(({ id, name, grade, avatar }) => (
  <div key={id} className="bg-white border rounded-lg p-4 shadow-sm">
    <div className="text-3xl mb-2">{avatar}</div>
    <div className="font-semibold">{name}</div>
    <div className="text-sm text-gray-500">성적: {grade}</div>
  </div>
))}
```

<a id="5"></a>
## 5️⃣ 컴포넌트로 분리하기 [↑](#toc)

map() 안의 JSX가 길어지면 별도 컴포넌트로 분리해야 합니다.

### 분리 전 (읽기 어려움)

```jsx
{students.map((student) => (
  <div key={student.id} className="bg-white border rounded-lg p-4 shadow-sm flex items-center gap-3">
    <div className="text-3xl">{student.avatar}</div>
    <div>
      <div className="font-semibold text-gray-900">{student.name}</div>
      <div className="text-sm text-gray-500">학년: {student.year}학년</div>
      <div className="text-sm text-gray-500">성적: {student.grade}</div>
      {student.grade === "A+" && (
        <span className="text-xs bg-yellow-100 text-yellow-800 px-2 py-0.5 rounded">우수</span>
      )}
    </div>
  </div>
))}
```

### 분리 후 (명확하고 재사용 가능)

```jsx
// src/components/StudentCard.jsx
function StudentCard({ id, name, grade, avatar, year }) {
  const isExcellent = grade === "A+";

  return (
    <div className="bg-white border rounded-lg p-4 shadow-sm flex items-center gap-3">
      <div className="text-3xl">{avatar}</div>
      <div>
        <div className="font-semibold text-gray-900">{name}</div>
        <div className="text-sm text-gray-500">학년: {year}학년</div>
        <div className="text-sm text-gray-500">성적: {grade}</div>
        {isExcellent && (
          <span className="text-xs bg-yellow-100 text-yellow-800 px-2 py-0.5 rounded">
            우수
          </span>
        )}
      </div>
    </div>
  );
}

export default StudentCard;
```

```jsx
// src/components/StudentList.jsx
import StudentCard from "./StudentCard";

function StudentList() {
  const students = [
    { id: 1, name: "김철수", grade: "A+", avatar: "🧑", year: 2 },
    { id: 2, name: "이영희", grade: "B+", avatar: "👩", year: 3 },
    { id: 3, name: "박민준", grade: "A", avatar: "🧒", year: 1 },
  ];

  return (
    <div className="p-4 space-y-3">
      <h2 className="text-xl font-bold">학생 목록</h2>
      {students.map((student) => (
        // key는 부모(map이 있는 곳)에서 지정합니다
        <StudentCard key={student.id} {...student} />
      ))}
    </div>
  );
}

export default StudentList;
```

> **중요**: `key`는 `StudentCard` 컴포넌트 안에서 지정하는 것이 아닙니다. **map()을 사용하는 쪽에서** 지정해야 합니다. `StudentCard` 내부에서 `props.key`로 접근할 수 없습니다 — key는 React 내부용입니다.

## 6️⃣ 실습: 메뉴판 만들기 [↑](#toc)

### 기본 과제

다음 데이터를 받아서 메뉴판을 만드세요.

```jsx
// src/App.jsx
const menuItems = [
  { id: 1, name: "아메리카노", price: 4500, category: "커피" },
  { id: 2, name: "카페라떼", price: 5000, category: "커피" },
  { id: 3, name: "녹차 라떼", price: 5500, category: "논커피" },
  { id: 4, name: "딸기 스무디", price: 6000, category: "논커피" },
  { id: 5, name: "치즈 케이크", price: 6500, category: "디저트" },
  { id: 6, name: "티라미수", price: 7000, category: "디저트" },
];
```

**요구사항:**
1. 각 메뉴를 카드로 렌더링
2. 이름, 가격(원 단위 포맷), 카테고리 배지 표시
3. `MenuItem` 컴포넌트로 분리

**힌트: 가격 포맷**
```js
// 4500 → "4,500원"
price.toLocaleString("ko-KR") + "원"
```

**기본 구현 예시:**

```jsx
// src/components/MenuItem.jsx
function MenuItem({ name, price, category }) {
  const categoryColors = {
    커피: "bg-amber-100 text-amber-800",
    논커피: "bg-green-100 text-green-800",
    디저트: "bg-pink-100 text-pink-800",
  };

  return (
    <div className="bg-white rounded-xl shadow p-4 flex justify-between items-center">
      <div>
        <span
          className={`text-xs px-2 py-0.5 rounded-full font-medium ${
            categoryColors[category] || "bg-gray-100 text-gray-700"
          }`}
        >
          {category}
        </span>
        <h3 className="font-semibold mt-1 text-gray-900">{name}</h3>
      </div>
      <div className="text-blue-600 font-bold">
        {price.toLocaleString("ko-KR")}원
      </div>
    </div>
  );
}

export default MenuItem;
```

```jsx
// src/App.jsx
import MenuItem from "./components/MenuItem";

const menuItems = [
  { id: 1, name: "아메리카노", price: 4500, category: "커피" },
  { id: 2, name: "카페라떼", price: 5000, category: "커피" },
  { id: 3, name: "녹차 라떼", price: 5500, category: "논커피" },
  { id: 4, name: "딸기 스무디", price: 6000, category: "논커피" },
  { id: 5, name: "치즈 케이크", price: 6500, category: "디저트" },
  { id: 6, name: "티라미수", price: 7000, category: "디저트" },
];

function App() {
  return (
    <div className="min-h-screen bg-gray-50 p-6">
      <h1 className="text-2xl font-bold text-center mb-6">☕ 카페 메뉴</h1>
      <div className="max-w-md mx-auto space-y-3">
        {menuItems.map((item) => (
          <MenuItem key={item.id} {...item} />
        ))}
      </div>
    </div>
  );
}

export default App;
// 출력: 6개의 메뉴 카드가 세로로 나열됩니다
```

### 도전 과제

1. **카테고리별 그룹핑**: 커피 / 논커피 / 디저트 섹션을 나눠서 표시
2. **가격순 정렬**: 버튼을 눌러 오름차순/내림차순 정렬 (힌트: `[...menuItems].sort(...)`)
3. **검색 필터**: 이름 검색 input을 추가해 일치하는 항목만 표시

## 7️⃣ 정리 + 브릿지 [↑](#toc)

### 핵심 정리

| 개념 | 요점 |
|------|------|
| `map()` | 배열 → JSX 배열 변환 |
| `key` prop | React 재조정을 위한 고유 식별자 |
| key 규칙 | 항상 고유 ID 사용, index는 지양 |
| 컴포넌트 분리 | map 안의 JSX가 길면 별도 컴포넌트로 |
| key 위치 | map을 사용하는 쪽(부모)에서 지정 |

### 체크리스트

- [ ] `fruits.map((fruit) => <li key={fruit.id}>...</li>)` 를 직접 작성할 수 있다
- [ ] key를 빠뜨렸을 때 콘솔에 뜨는 경고를 설명할 수 있다
- [ ] index를 key로 쓰면 안 되는 이유를 설명할 수 있다
- [ ] `StudentCard` 같은 별도 컴포넌트로 분리할 수 있다

---

> **브릿지**: 목록을 렌더링하는 법을 배웠습니다. 그런데 지금은 모든 항목이 **항상** 화면에 보입니다. 특정 조건에서만 보여주고 싶다면 어떻게 할까요? **다음 장에서 조건부 렌더링을 배우면**, 로그인 상태일 때만 대시보드를 보여주거나, 알림이 있을 때만 배지를 표시하는 UI를 만들 수 있습니다.
