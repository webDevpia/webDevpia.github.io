<a id="1"></a>
## 1️⃣ 선형 회귀란? [↑](#toc)

### 예측이란 무엇인가

매일 공부한 시간과 시험 점수 데이터가 있다고 가정합니다.

| 공부 시간 | 시험 점수 |
|---|---|
| 1시간 | 55점 |
| 2시간 | 63점 |
| 3시간 | 72점 |
| 5시간 | 85점 |
| 7시간 | 91점 |

"8시간 공부하면 몇 점을 받을까?"를 예측하고 싶습니다.

> 이 점들을 가장 잘 설명하는 **직선 하나**를 찾는 것이 선형 회귀입니다.

산점도를 그려보면 점들이 대략 직선 위에 놓여 있습니다. 그 직선을 활용하면 새로운 공부 시간에 대한 점수를 예측할 수 있습니다.

### 회귀(Regression)란?

**분류(Classification)** 는 "어떤 종류?"를 맞히고,  
**회귀(Regression)** 는 **"얼마?"** 를 예측합니다.

- 집값 예측, 주가 예측, 온도 예측, 매출 예측 → 모두 회귀 문제입니다.

<a id="2"></a>
## 2️⃣ 수식 y = Wx + b [↑](#toc)

직선의 방정식은 중학교에서 배웠습니다.

```
y = Wx + b
```

- **x** : 입력값 (공부 시간)
- **y** : 예측값 (시험 점수)
- **W** (Weight, 가중치) : 기울기 — x가 1 늘면 y가 얼마나 오르는가
- **b** (bias, 편향) : 절편 — x가 0일 때 y의 시작값

> "W = 5" 라면, 1시간 더 공부할 때마다 점수가 5점씩 오른다는 뜻입니다.

머신러닝이 하는 일은 딱 하나입니다.  
**데이터를 가장 잘 설명하는 W와 b를 찾아라.**

<a id="3"></a>
## 3️⃣ 손실 함수 MSE [↑](#toc)

### 직관 — "틀린 정도를 하나의 숫자로"

W와 b를 아무 값으로 시작해봅니다. 예측값이 실제값과 다를 것입니다.  
이 차이(오차)를 하나의 숫자로 표현한 것이 **손실 함수(Loss Function)** 입니다.

> 손실 함수가 작을수록 직선이 데이터를 잘 설명합니다.

가장 많이 쓰이는 손실 함수는 **MSE(Mean Squared Error, 평균 제곱 오차)** 입니다.

- 오차 = 실제값 - 예측값
- 오차를 제곱합니다 (음수 처리 + 큰 오차에 벌점)
- 전체 평균을 냅니다

직관: "모든 점에서 직선까지의 거리를 제곱해서 평균낸 값"이 MSE입니다.  
이 값을 최소화하는 W, b를 찾는 것이 선형 회귀의 목표입니다.

<details markdown="1">
<summary>📐 더 알아보기 — MSE 수식</summary>

$$
\text{MSE} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2
$$

- $y_i$: 실제값 (i번째 샘플)
- $\hat{y}_i = Wx_i + b$: 예측값
- $n$: 데이터 개수

MSE를 최소화하는 W, b의 **해석적 해(Closed-Form Solution)** 도 존재합니다.

$$
W = \frac{\sum (x_i - \bar{x})(y_i - \bar{y})}{\sum (x_i - \bar{x})^2}, \quad b = \bar{y} - W\bar{x}
$$

하지만 데이터가 많거나 변수가 많아지면 이 공식을 직접 계산하는 것보다  
**경사하강법**으로 반복해서 찾는 것이 훨씬 효율적입니다.

</details>

<a id="4"></a>
## 4️⃣ 경사하강법 [↑](#toc)

### 등산객 비유

> 눈을 감은 채 산 정상에서 출발했습니다. 가장 빠르게 골짜기(MSE 최솟값)로 내려가야 합니다.  
> 발아래 경사를 느끼면서 경사가 가파른 쪽으로 한 발씩 내려갑니다.  
> 이것이 **경사하강법(Gradient Descent)** 입니다.

### 작동 원리

1. W, b를 임의값으로 초기화합니다.
2. 현재 MSE를 계산합니다.
3. W, b에 대한 기울기(gradient)를 계산합니다.
4. 기울기의 반대 방향으로 W, b를 조금씩 움직입니다.
5. 2~4를 반복합니다.

이때 "조금씩"의 크기를 **학습률(Learning Rate, lr)** 이라고 합니다.

> 학습률이 너무 크면 골짜기를 건너뛰고, 너무 작으면 너무 오래 걸립니다.

<details markdown="1">
<summary>📐 더 알아보기 — 경사하강법 수식</summary>

MSE에 대한 W, b의 편미분:

$$
\frac{\partial \text{MSE}}{\partial W} = \frac{-2}{n} \sum x_i (y_i - \hat{y}_i)
$$

$$
\frac{\partial \text{MSE}}{\partial b} = \frac{-2}{n} \sum (y_i - \hat{y}_i)
$$

업데이트 규칙 (lr = 학습률):

$$
W \leftarrow W - lr \cdot \frac{\partial \text{MSE}}{\partial W}
$$

$$
b \leftarrow b - lr \cdot \frac{\partial \text{MSE}}{\partial b}
$$

이 과정을 수백~수천 번 반복하면 W, b가 최적값에 수렴합니다.

</details>

<a id="5"></a>
## 5️⃣ NumPy로 직접 구현 [↑](#toc)

> 원리를 이해하기 위해 NumPy로 선형 회귀를 직접 만들어봅니다.

```python
# Google Colab에서 바로 실행하세요
import numpy as np
import matplotlib.pyplot as plt

# ── 1. 데이터 준비 ──────────────────────────────────────
np.random.seed(42)
X = np.array([1, 2, 3, 4, 5, 6, 7, 8], dtype=float)
y = np.array([55, 63, 72, 75, 85, 88, 91, 95], dtype=float)

# ── 2. 파라미터 초기화 ──────────────────────────────────
W = 0.0   # 기울기 (가중치)
b = 0.0   # 절편 (편향)
lr = 0.01 # 학습률
epochs = 500  # 반복 횟수

# ── 3. 경사하강법 학습 루프 ─────────────────────────────
n = len(X)
loss_history = []

for epoch in range(epochs):
    # 예측
    y_pred = W * X + b

    # 손실 계산 (MSE)
    loss = np.mean((y - y_pred) ** 2)
    loss_history.append(loss)

    # 기울기(gradient) 계산
    dW = (-2 / n) * np.sum(X * (y - y_pred))
    db = (-2 / n) * np.sum(y - y_pred)

    # 파라미터 업데이트
    W -= lr * dW
    b -= lr * db

    if epoch % 100 == 0:
        print(f"Epoch {epoch:4d} | Loss: {loss:.4f} | W: {W:.4f} | b: {b:.4f}")

print(f"\n최종 결과 → W: {W:.4f}, b: {b:.4f}")
print(f"8시간 공부 시 예측 점수: {W * 8 + b:.1f}점")
```

실행 결과:
```
Epoch    0 | Loss: 6076.0000 | W: 10.9300 | b: 1.5575
Epoch  100 | Loss: 15.8322 | W: 5.1823 | b: 48.3241
Epoch  200 | Loss: 13.9418 | W: 4.8755 | b: 50.9871
Epoch  300 | Loss: 13.8752 | W: 4.8187 | b: 51.4742
Epoch  400 | Loss: 13.8730 | W: 4.8082 | b: 51.5644

최종 결과 → W: 4.8059, b: 51.5817
8시간 공부 시 예측 점수: 89.9점
```

500번 반복하면서 손실이 점점 줄어드는 것을 확인합니다.

<a id="6"></a>
## 6️⃣ scikit-learn으로 구현 [↑](#toc)

> 실무에서는 이렇게 합니다. 단 3줄입니다.

```python
from sklearn.linear_model import LinearRegression
import numpy as np

# 데이터 준비
X = np.array([1, 2, 3, 4, 5, 6, 7, 8]).reshape(-1, 1)  # 2D 배열 필요
y = np.array([55, 63, 72, 75, 85, 88, 91, 95])

# 모델 학습 (fit/predict/score 패턴)
model = LinearRegression()       # 1. 모델 생성
model.fit(X, y)                  # 2. 학습 (데이터로 W, b 찾기)

# 예측
X_new = np.array([[8]])          # 8시간 공부
y_pred = model.predict(X_new)   # 3. 예측

print(f"W (기울기): {model.coef_[0]:.4f}")
print(f"b (절편):   {model.intercept_:.4f}")
print(f"8시간 공부 시 예측 점수: {y_pred[0]:.1f}점")
print(f"R² 점수: {model.score(X, y):.4f}")
```

실행 결과:
```
W (기울기): 4.9167
b (절편):   48.9167
8시간 공부 시 예측 점수: 88.3점
R² 점수: 0.9733
```

> scikit-learn은 경사하강법 대신 해석적 최적해를 사용하므로 결과가 약간 다릅니다.  
> 하지만 두 방법 모두 "데이터를 가장 잘 설명하는 직선"을 찾습니다.

## 7️⃣ 50줄 vs 3줄 비교 [↑](#toc)

| 항목 | NumPy 직접 구현 | scikit-learn |
|---|---|---|
| 코드 양 | ~50줄 | ~3줄 |
| 목적 | **원리 이해** | **실무 사용** |
| 최적화 방법 | 경사하강법 (반복) | 해석적 해 (정확) |
| 학습률 조정 | 직접 설정 필요 | 자동 |
| 정규화/스케일링 | 직접 처리 | 내장 지원 |
| 다변수 확장 | 수식 수정 필요 | 자동 |

> 원리를 이해하면 scikit-learn이 "왜 이렇게 동작하는지" 납득할 수 있습니다.

<a id="8"></a>
## 8️⃣ 시각화 [↑](#toc)

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression

# 한글 폰트 설정 (Colab)
plt.rcParams['font.family'] = 'NanumGothic'
plt.rcParams['axes.unicode_minus'] = False

# 데이터
X_raw = np.array([1, 2, 3, 4, 5, 6, 7, 8], dtype=float)
y = np.array([55, 63, 72, 75, 85, 88, 91, 95], dtype=float)
X = X_raw.reshape(-1, 1)

# 모델 학습
model = LinearRegression()
model.fit(X, y)

# 예측선 생성
X_line = np.linspace(0, 9, 100).reshape(-1, 1)
y_line = model.predict(X_line)

# 시각화
fig, axes = plt.subplots(1, 2, figsize=(12, 5))

# 왼쪽: 회귀선
ax1 = axes[0]
ax1.scatter(X_raw, y, color='steelblue', s=80, label='실제 데이터')
ax1.plot(X_line, y_line, color='tomato', linewidth=2, label='회귀선')
ax1.scatter([8], model.predict([[8]]), color='gold', s=150, marker='*', 
            zorder=5, label=f'예측값 ({model.predict([[8]])[0]:.1f}점)')
# 잔차 시각화
for xi, yi in zip(X_raw, y):
    y_hat = model.predict([[xi]])[0]
    ax1.plot([xi, xi], [yi, y_hat], 'gray', linewidth=1, linestyle='--')
ax1.set_xlabel('공부 시간')
ax1.set_ylabel('시험 점수')
ax1.set_title('선형 회귀: 직선과 잔차')
ax1.legend()

# 오른쪽: 손실 그래프 (NumPy 구현에서)
W, b, lr = 0.0, 0.0, 0.01
loss_history = []
for _ in range(500):
    y_pred = W * X_raw + b
    loss = np.mean((y - y_pred) ** 2)
    loss_history.append(loss)
    dW = (-2 / len(X_raw)) * np.sum(X_raw * (y - y_pred))
    db = (-2 / len(X_raw)) * np.sum(y - y_pred)
    W -= lr * dW
    b -= lr * db

ax2 = axes[1]
ax2.plot(loss_history, color='coral')
ax2.set_xlabel('에폭(반복 횟수)')
ax2.set_ylabel('MSE 손실')
ax2.set_title('학습 과정: 손실 감소 곡선')

plt.tight_layout()
plt.show()
```

왼쪽 그래프에서 회색 점선은 실제값과 예측값의 차이(잔차)를 나타냅니다.  
오른쪽 그래프에서 에폭이 늘수록 손실이 줄어드는 것을 볼 수 있습니다.

## 9️⃣ 정리 및 실습 [↑](#toc)

### 핵심 개념 정리

| 개념 | 설명 | 비유 |
|---|---|---|
| 선형 회귀 | 데이터를 가장 잘 설명하는 직선을 찾음 | 산점도에 자 대기 |
| W (가중치) | 직선의 기울기, 입력 변화에 대한 출력 변화율 | 속도계 |
| b (편향) | 직선의 절편, 입력이 0일 때 출력값 | 기본 체력 |
| MSE | 예측과 실제의 차이를 제곱 평균한 손실 | 과녁에서 떨어진 거리 |
| 경사하강법 | 손실을 줄이는 방향으로 W, b를 반복 업데이트 | 눈 감고 산 내려오기 |
| 학습률 (lr) | 업데이트 보폭 크기 | 발걸음 크기 |
