### 2.2 효과적인 ML 비유 모음

| 개념 | 비유 | 출처 |
|------|------|------|
| 지도학습 | **선생님이 채점하는 숙제** — 정답이 있는 데이터로 학습 | Andrew Ng |
| 비지도학습 | **분류 기준 없이 책을 정리하는 사서** — 자연스러운 그룹 발견 | 커뮤니티 |
| 과적합(Overfitting) | **시험 문제를 통째로 외운 학생** — 연습문제는 만점, 새 문제는 0점 | Medium/Towards AI |
| 과소적합(Underfitting) | **공부를 안 한 학생** — 기본 패턴도 못 찾음 | 커뮤니티 |
| 경사하강법 | **안개 낀 산에서 하산하는 등산객** — 발밑 경사만 보고 한 걸음씩 | Google ML, MIT |
| 학습률(Learning Rate) | **등산객의 보폭** — 너무 크면 골짜기를 지나침, 너무 작으면 도착 못함 | Analytics Vidhya |
| 결정 트리 | **20 Questions 게임** — 예/아니오 질문으로 답 좁혀가기 | 커뮤니티 |
| 랜덤 포레스트 | **전문가 다수결 투표** — 한 명은 틀릴 수 있지만, 다수는 대체로 맞음 | 업계 표준 |
| 특성 스케일링 | **mm와 km 비교** — 단위 통일 없이 거리 계산하면 왜곡 | 커뮤니티 |
| 훈련/테스트 분할 | **연습 문제 vs 실전 시험** — 연습을 시험지로 쓰면 실력을 속임 | Google ML |
| 혼동 행렬 | **화재 감지기** — TP=화재+경보, FP=오경보, FN=화재 미감지, TN=정상 | 커뮤니티 |
| 정밀도(Precision) | **"경보를 울렸을 때, 실제 화재 비율"** | Google ML |
| 재현율(Recall) | **"실제 화재 중, 감지기가 잡은 비율"** | Google ML |
| 데이터 누수 | **시험 전에 답안지를 본 것** — 점수는 좋지만 실력이 아님 | IBM |
| 파이프라인 | **공장 컨베이어 벨트** — 원료 투입 → 가공 → 제품 출력 자동화 | 커뮤니티 |
| K-Means | **무작위로 깃발 꽂고 가장 가까운 사람끼리 팀 만들기** | 커뮤니티 |
| PCA | **3D 물체의 2D 그림자** — 핵심 정보는 유지하면서 차원 축소 | r2d3.us |

## 3. 다중 페르소나 분석

### 3.1 Dr. 데이터 — 교육공학 연구자의 분석

#### 핵심 주장: "직관 → 코드 → 수학 순서가 비전공자에게 가장 효과적"

**Andrew Ng 모델 (가장 검증된 접근):**

```
1단계: 직관 (Intuition)
  "비가 오면 우산을 가져간다" → 이것이 분류입니다

2단계: 코드 (Implementation)
  from sklearn.tree import DecisionTreeClassifier
  model.fit(X_train, y_train)

3단계: 수학 (Optional Math)
  정보 이득(Information Gain) = H(parent) - Σ(w_i × H(child_i))
  → "더 알고 싶으면" 박스에 배치
```

**비전공자에게 수학을 다루는 방법:**

```
❌ "MSE의 편미분을 구하면..."
✅ "오차를 줄이려면 기울기의 반대 방향으로 조금씩 이동합니다.
    등산객이 안개 속에서 발밑을 더듬어 내려가는 것과 같습니다."
```

수식은 **참고 상자**에 넣고, 본문은 **직관과 코드**로 진행한다.

**파이프라인을 초반에 가르쳐야 하는 이유:**

```python
# ❌ 데이터 누수 발생 코드 (초보자 흔한 실수)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)  # 전체 데이터에 fit!
X_train, X_test = train_test_split(X_scaled)

# ✅ 파이프라인으로 자동 방지
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('model', LogisticRegression())
])
pipe.fit(X_train, y_train)  # scaler가 X_train에만 fit됨
```

파이프라인을 3장(전처리)부터 가르치면, 데이터 누수를 **구조적으로 방지**할 수 있다.

### 3.2 김 사이언티스트 — 시니어 데이터 과학자의 분석

#### 핵심 주장: "scikit-learn의 통일 API를 처음부터 가르치면 모든 알고리즘이 동일 패턴"

**scikit-learn 통일 API 패턴:**

```python
# 모든 알고리즘이 이 패턴을 따름
model = AlgorithmName()    # 1. 모델 생성
model.fit(X_train, y_train) # 2. 학습
predictions = model.predict(X_test) # 3. 예측
score = model.score(X_test, y_test) # 4. 평가
```

이 패턴을 한 번 배우면, 새로운 알고리즘을 배울 때마다 **"이름만 바꾸면 된다"**는 강력한 일관성을 제공한다.

| 5 | 랜덤 포레스트 | 앙상블 첫 경험, 트리 자연 확장 |

### 3.3 박 비평가 — 교육 비평가의 분석

#### 핵심 주장: "수학 공포와 '블랙박스' 사용이 가장 큰 위험"

**위험 1: 수학 공포 (Mathematical Anxiety)**

비전공자가 수식을 보는 순간 **학습 동기가 급락**한다.

```
❌ "MSE = (1/n) Σᵢ₌₁ⁿ (yᵢ - ŷᵢ)²"
   → "이 기호들이 뭐지?" → 포기

✅ "오차 = 예측값 - 실제값. 오차를 제곱해서 평균 내면 평균 오차.
    이걸 줄이는 게 학습의 목표입니다."
   → "아, 오차를 줄이는 거구나" → 이해
```

**완화**: 수식은 항상 **"더 알아보기"** 접이식 블록에 넣고, 본문은 자연어와 코드로.

**위험 2: 블랙박스 사용 — "왜 이 결과가 나왔는지 모름"**

scikit-learn의 `model.fit()` 3줄이면 결과가 나오지만, 학생은 **"무슨 일이 일어났는지"** 모른다.

**완화**: 첫 알고리즘(선형 회귀)은 **NumPy로 직접 구현** 후 scikit-learn 등가 코드를 보여준다. "여러분이 50줄로 한 것을 scikit-learn은 3줄로 합니다. 하지만 내부는 같은 원리입니다."

**위험 3: 평가 지표를 "숫자만 보는" 습관**

```
❌ "accuracy = 0.95입니다. 좋은 모델이네요!"
   (실제로는 95%가 정상인 불균형 데이터 → 모든 것을 '정상'으로 예측해도 95%)

✅ "정확도가 95%인데, 실제 사기를 하나도 못 잡았습니다.
    정밀도와 재현율을 같이 봐야 합니다."
```

**완화**: 불균형 데이터 트랩을 **실습으로 직접 체험**시킨다.

**위험 4: 데이터 전처리를 "지루한 사전 작업"으로 인식**

> **"데이터 과학자는 업무 시간의 80%를 데이터 정리에 쓴다."** — Forbes/Kaggle 설문

**완화**: EDA를 **"탐정 수사"**로 프레이밍. "이 데이터에 숨겨진 단서를 찾아라!"

#### 평가

| 항목 | 판정 |
|------|------|
| 최대 위험 | 수학 공포 → 접이식 블록으로 완화 |
| 두 번째 위험 | 블랙박스 사용 → 첫 알고리즘은 직접 구현 |
| 평가 지표 | 불균형 트랩 직접 체험 필수 |

### 5.3 "직접 구현 → sklearn 등가" 패턴

05장(선형 회귀)에서만 사용:

```python
# 1단계: 50줄로 직접 구현 (기존 내용 활용)
def gradient_descent(X, y, lr=0.01, epochs=1000):
    W, b = 0.0, 0.0
    for _ in range(epochs):
        y_pred = W * X + b
        dW = -2 * np.mean(X * (y - y_pred))
        db = -2 * np.mean(y - y_pred)
        W -= lr * dW
        b -= lr * db
    return W, b

# 2단계: sklearn으로 3줄
from sklearn.linear_model import LinearRegression
model = LinearRegression()
model.fit(X_train, y_train)

# "여러분이 50줄로 한 것을 sklearn은 3줄로 합니다!"
```
