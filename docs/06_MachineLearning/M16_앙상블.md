## 1️⃣ 앙상블이란? [↑](#toc)

> **앙상블 = 전문가 위원회. 한 명은 틀릴 수 있지만, 여러 전문가의 합의는 더 정확합니다.**

**"집단 지성(Wisdom of the Crowd)"**의 원리:

유리병에 든 젤리빈 수를 맞추는 실험에서, 개인의 추측은 크게 빗나가지만 **수백 명의 평균은 정답에 매우 가까웠습니다.**

머신러닝에서도 마찬가지입니다:
- 단일 모델(전문가 1명): 편향될 수 있음
- 앙상블(전문가 위원회): 오류가 상쇄되어 더 정확

```python
# 직관적 데모: 동전 던지기로 앙상블 이해
import numpy as np

np.random.seed(42)

# 각 모델이 70% 확률로 정답을 맞춘다고 가정
model_accuracy = 0.70
n_simulations  = 10000

def single_model_vote():
    """단일 모델의 정답 여부"""
    return np.random.random() < model_accuracy

def ensemble_vote(n_models):
    """n개 모델의 다수결 결과"""
    votes = [single_model_vote() for _ in range(n_models)]
    return sum(votes) > n_models / 2  # 과반수가 정답이면 정답

# 시뮬레이션
for n in [1, 3, 5, 11, 21, 51, 101]:
    ensemble_correct = sum(ensemble_vote(n) for _ in range(n_simulations))
    print(f"모델 {n:3d}개: 앙상블 정확도 = {ensemble_correct/n_simulations:.4f}")
```

실행 결과:
```
모델   1개: 앙상블 정확도 = 0.6998
모델   3개: 앙상블 정확도 = 0.7837
모델   5개: 앙상블 정확도 = 0.8368
모델  11개: 앙상블 정확도 = 0.8869
모델  21개: 앙상블 정확도 = 0.9121
모델  51개: 앙상블 정확도 = 0.9389
모델 101개: 앙상블 정확도 = 0.9524
```

개별 모델이 70%일 때, 101개 모델의 앙상블은 **95%** 정확도를 달성합니다!

**앙상블이 효과적인 조건:**
1. 각 모델이 어느 정도 정확해야 함 (50% 이상)
2. 각 모델이 **서로 다른 실수**를 해야 함 (다양성 필요)

## 2️⃣ Bagging — 서로 다른 시각으로 배우기 [↑](#toc)

> **Bagging = 같은 문제집을 여러 버전으로 복사해서, 각 학생이 다른 버전을 풀게 한 뒤 합산**

**Bagging(Bootstrap Aggregating)**의 핵심:

1. **부트스트랩 샘플링**: 훈련 데이터에서 중복을 허용해 무작위로 샘플 추출
2. **병렬 학습**: 각 샘플로 독립적으로 모델을 학습
3. **집계**: 분류→다수결, 회귀→평균

```
원본 데이터: [A B C D E F G H]

Bootstrap 1: [A C C F G A B H] → 모델 1 학습
Bootstrap 2: [B D A E E G C F] → 모델 2 학습
Bootstrap 3: [G A H B B D F E] → 모델 3 학습

새 데이터 예측:
  모델 1 → 고양이
  모델 2 → 고양이
  모델 3 → 개
  다수결 → 고양이 ✅
```

**Bagging의 특징:**
- 모델들이 **병렬**로 학습 (독립적)
- 과적합을 줄이는 효과 (분산 감소)
- 대표 알고리즘: **랜덤 포레스트**

## 3️⃣ Boosting — 실수에서 배우기 [↑](#toc)

> **Boosting = 학생이 틀린 문제를 표시해서, 다음 선생님이 그 문제를 더 집중적으로 가르침**

**Boosting**의 핵심:

1. 첫 번째 모델 학습
2. 첫 번째 모델이 **틀린 데이터**에 더 높은 가중치 부여
3. 가중치를 반영해 두 번째 모델 학습
4. 이 과정을 반복하여 오차를 순차적으로 줄임

```
1번 모델: [○ × ○ ○ × ○] ← 틀린 것(×)에 가중치↑
2번 모델: [○ ○ ○ × ○ ○] ← 이전에 틀린 것 집중
3번 모델: [○ ○ ○ ○ ○ ×]

최종 예측 = 모든 모델의 가중합
```

**Boosting의 특징:**
- 모델들이 **순차**로 학습 (이전 모델에 의존)
- 편향을 줄이는 효과
- 과적합 가능성 있음 (데이터에 민감)
- 대표 알고리즘: **AdaBoost, Gradient Boosting, XGBoost, LightGBM**

**Bagging vs Boosting 요약:**

| | Bagging | Boosting |
|--|---------|----------|
| 학습 방식 | 병렬 | 순차 |
| 목표 | 분산 감소 | 편향 감소 |
| 과적합 | 강함 | 약할 수 있음 |
| 속도 | 빠름 | 느림 |
| 대표 | 랜덤 포레스트 | XGBoost, LightGBM |

## 4️⃣ Random Forest 복습과 심화 [↑](#toc)

랜덤 포레스트 = Bagging + 피처 랜덤 선택

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import cross_val_score
from sklearn.preprocessing import StandardScaler

cancer = load_breast_cancer()
X, y = cancer.data, cancer.target

# 파라미터별 성능 비교
n_estimators_list = [10, 50, 100, 200, 300]
scores = []

for n in n_estimators_list:
    rf = RandomForestClassifier(n_estimators=n, random_state=42, n_jobs=-1)
    cv_scores = cross_val_score(rf, X, y, cv=5, scoring='accuracy')
    scores.append(cv_scores.mean())
    print(f"n_estimators={n:4d}: {cv_scores.mean():.4f} (±{cv_scores.std():.4f})")

plt.figure(figsize=(8, 5))
plt.plot(n_estimators_list, scores, 'bo-', linewidth=2, markersize=8)
plt.xlabel('트리 수 (n_estimators)')
plt.ylabel('5-Fold 교차 검증 정확도')
plt.title('랜덤 포레스트: 트리 수와 성능')
plt.grid(True, alpha=0.3)
plt.show()
```

실행 결과:
```
n_estimators=  10: 0.9508 (±0.0140)
n_estimators=  50: 0.9631 (±0.0112)
n_estimators= 100: 0.9666 (±0.0098)
n_estimators= 200: 0.9666 (±0.0101)
n_estimators= 300: 0.9666 (±0.0094)
```

100개 이후에는 성능 향상이 거의 없습니다. 속도 vs 성능 트레이드오프를 고려해 n_estimators를 선택하세요.

```python
# 피처 중요도 시각화
rf_best = RandomForestClassifier(n_estimators=100, random_state=42)
rf_best.fit(X, y)

importances = pd.Series(rf_best.feature_importances_, index=cancer.feature_names)
top10 = importances.nlargest(10)

plt.figure(figsize=(9, 6))
top10.sort_values().plot(kind='barh', color='#4ECDC4')
plt.title('랜덤 포레스트 피처 중요도 (상위 10개)')
plt.xlabel('중요도')
plt.tight_layout()
plt.show()
```

## 5️⃣ Gradient Boosting — 순차적 개선 [↑](#toc)

**Gradient Boosting**은 Boosting의 가장 강력한 버전입니다.

> **비유**: 조각가가 대리석을 깎을 때, 처음에는 큰 덩어리를 제거하고, 점점 더 세밀한 부분을 다듬어갑니다. 각 단계에서 "남은 오차(잔차)"를 줄여나갑니다.

**핵심 아이디어: 잔차(Residual) 학습**

```
실제값: 100만원
모델 1 예측: 70만원 → 오차(잔차): 30만원
모델 2 예측: 25만원 (잔차 학습) → 누적 예측: 95만원 → 잔차: 5만원
모델 3 예측: 4만원 (잔차 학습)  → 누적 예측: 99만원 → 잔차: 1만원
...
```

각 트리가 이전 트리의 **남은 오차(잔차)**를 학습합니다!

```python
from sklearn.ensemble import GradientBoostingClassifier
import time

# Gradient Boosting 학습
gb = GradientBoostingClassifier(
    n_estimators=100,
    learning_rate=0.1,    # 학습률 (각 트리의 기여도 조절)
    max_depth=3,           # 각 트리의 깊이 (작을수록 과소적합 방지)
    random_state=42
)

start = time.time()
gb_scores = cross_val_score(gb, X, y, cv=5, scoring='accuracy')
elapsed = time.time() - start

print(f"Gradient Boosting 결과:")
print(f"  정확도: {gb_scores.mean():.4f} (±{gb_scores.std():.4f})")
print(f"  학습 시간: {elapsed:.2f}초")
```

실행 결과:
```
Gradient Boosting 결과:
  정확도: 0.9701 (±0.0112)
  학습 시간: 4.23초
```

**주요 하이퍼파라미터:**

| 파라미터 | 역할 | 기본값 | 팁 |
|---------|------|--------|----|
| `n_estimators` | 트리 수 | 100 | 높을수록 좋지만 느림 |
| `learning_rate` | 학습률 | 0.1 | 낮으면 n_estimators 늘려야 |
| `max_depth` | 트리 깊이 | 3 | 3~5가 일반적 |
| `subsample` | 데이터 샘플 비율 | 1.0 | 0.8로 줄이면 과적합 방지 |

{::nomarkdown}
<details>
<summary>📐 더 알아보기 — Gradient Boosting의 수학적 배경</summary>
<p>Gradient Boosting은 <strong>경사 하강법(Gradient Descent)</strong>을 함수 공간에 적용한 것입니다.</p>
<p>일반 경사 하강법:</p>
<pre><code>θ ← θ - η × ∇L(θ)   (파라미터를 손실의 기울기 방향으로 업데이트)
</code></pre>
<p>Gradient Boosting:</p>
<pre><code>F_m(x) = F_{m-1}(x) + η × h_m(x)
</code></pre>
<p>여기서 <code>h_m</code>은 잔차(음의 기울기)를 맞추는 새 트리입니다.</p>
<p>핵심: 손실 함수의 종류를 바꾸면 (MSE → Log Loss 등) 다양한 문제에 적용 가능합니다.</p>
</details>
{:/nomarkdown}

## 7️⃣ Voting과 Stacking — 모델 결합 [↑](#toc)

서로 **다른 종류**의 모델들을 결합하면 더 강력해집니다.

### Voting Classifier

```python
from sklearn.ensemble import VotingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.svm import SVC
from sklearn.ensemble import RandomForestClassifier

# 개별 모델
lr  = LogisticRegression(max_iter=1000, random_state=42)
svm = SVC(probability=True, random_state=42)  # probability=True 필요
rf  = RandomForestClassifier(n_estimators=100, random_state=42)

# Hard Voting: 다수결
hard_voting = VotingClassifier(
    estimators=[('lr', lr), ('svm', svm), ('rf', rf)],
    voting='hard'
)

# Soft Voting: 확률 평균 (더 좋을 때가 많음)
soft_voting = VotingClassifier(
    estimators=[('lr', lr), ('svm', svm), ('rf', rf)],
    voting='soft'
)

# 성능 비교
for name, model in [('Hard Voting', hard_voting), ('Soft Voting', soft_voting)]:
    scores = cross_val_score(model, X, y, cv=5, scoring='accuracy')
    print(f"{name}: {scores.mean():.4f} (±{scores.std():.4f})")
```

실행 결과:
```
Hard Voting: 0.9736 (±0.0091)
Soft Voting: 0.9771 (±0.0086)
```

### Stacking (개념 소개)

> **Stacking = 여러 전문가(1레벨 모델)의 예측을 입력으로 받아, 최종 심판(메타 모델)이 결정하는 방식**

```python
from sklearn.ensemble import StackingClassifier

# 1레벨 모델들 (다양한 알고리즘)
estimators = [
    ('lr',  LogisticRegression(max_iter=1000, random_state=42)),
    ('rf',  RandomForestClassifier(n_estimators=100, random_state=42)),
    ('svm', SVC(probability=True, random_state=42))
]

# 메타 모델 (1레벨 예측을 입력으로 받음)
stacking = StackingClassifier(
    estimators=estimators,
    final_estimator=LogisticRegression(max_iter=1000),
    cv=5,           # 교차 검증으로 1레벨 예측 생성
    n_jobs=-1
)

stacking_scores = cross_val_score(stacking, X, y, cv=5, scoring='accuracy')
print(f"Stacking: {stacking_scores.mean():.4f} (±{stacking_scores.std():.4f})")
```

실행 결과:
```
Stacking: 0.9789 (±0.0082)
```

## 8️⃣ 실전: 주택 가격 예측 모델 비교 [↑](#toc)

```python
from sklearn.datasets import fetch_california_housing
from sklearn.model_selection import train_test_split, cross_val_score

                               GradientBoostingRegressor,

# 캘리포니아 주택 가격 데이터
housing = fetch_california_housing()
X_h = housing.data
y_h = housing.target

print("데이터 shape:", X_h.shape)
print("피처 이름:", housing.feature_names)
print(f"타겟: 중간 주택 가격 (단위: $100,000)")
print(f"평균 가격: ${y_h.mean():.2f} × $100,000")
```

```python
# 데이터 분할 및 스케일링
X_train, X_test, y_train, y_test = train_test_split(
    X_h, y_h, test_size=0.2, random_state=42
)

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled  = scaler.transform(X_test)

# 비교할 모델들
models = {
    '선형 회귀':           (LinearRegression(), True),
    'Ridge 회귀':          (Ridge(alpha=1.0), True),
    '의사결정나무':         (DecisionTreeRegressor(max_depth=10, random_state=42), True),
    'Bagging':             (BaggingRegressor(n_estimators=50, random_state=42), True),
    '랜덤 포레스트':        (RandomForestRegressor(n_estimators=100, random_state=42, n_jobs=-1), False),
    'Gradient Boosting':   (GradientBoostingRegressor(n_estimators=100, random_state=42), False),
    'XGBoost':             (XGBRegressor(n_estimators=100, random_state=42, verbosity=0), False),
    'LightGBM':            (LGBMRegressor(n_estimators=100, random_state=42, verbose=-1), False),
}

# 모든 모델 평가
results = {}

for name, (model, use_scaled) in models.items():
    X_tr = X_train_scaled if use_scaled else X_train
    X_te = X_test_scaled  if use_scaled else X_test

    start = time.time()
    model.fit(X_tr, y_train)
    y_pred = model.predict(X_te)
    elapsed = time.time() - start

    rmse = np.sqrt(mean_squared_error(y_test, y_pred))
    r2   = r2_score(y_test, y_pred)

    results[name] = {'RMSE': rmse, 'R²': r2, '시간(s)': elapsed}

# 결과 출력
results_df = pd.DataFrame(results).T.sort_values('R²', ascending=False)
results_df = results_df.round({'RMSE': 4, 'R²': 4, '시간(s)': 2})

print("=== 주택 가격 예측 모델 비교 ===")
print(results_df.to_string())
```

## 9️⃣ 정리 [↑](#toc)

| 방법 | 핵심 아이디어 | 장점 | 단점 |
|------|-------------|------|------|
| Bagging | 다른 데이터로 병렬 학습 | 과적합 방지, 안정적 | 개별 모델보다 약간 느림 |
| Random Forest | Bagging + 피처 랜덤 선택 | 강력, 해석 가능 | 메모리 많이 사용 |
| Gradient Boosting | 잔차를 순차 학습 | 높은 정확도 | 느림, 과적합 가능 |
| XGBoost | GB 최적화 버전 | 빠름, 강력한 정규화 | 파라미터 많음 |
| LightGBM | 리프 중심 분할 | 가장 빠름, 대용량 | 소규모에서 과적합 |
| Voting | 다수결/확률 평균 | 간단, 다양성 활용 | 성능 향상 폭 제한적 |
| Stacking | 예측을 피처로 사용 | 높은 성능 | 복잡, 느림 |

**실전 앙상블 선택 가이드:**

```
데이터 크기?
├── 소규모 (< 10만 행)
│   ├── 정확도 중시: XGBoost / Random Forest
│   └── 해석 중시: Random Forest
└── 대규모 (> 10만 행)
    ├── 빠른 학습 필요: LightGBM
    └── 정확도 극대화: XGBoost + LightGBM 앙상블
```
