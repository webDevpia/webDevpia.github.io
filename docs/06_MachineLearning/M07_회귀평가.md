<a id="1"></a>
## 1️⃣ 왜 여러 지표가 필요한가? [↑](#toc)

### "숫자 하나로는 부족하다"

두 모델을 비교합니다.

| 예측 | 실제 | 오차 |
|---|---|---|
| 모델 A: 100, 200, 300 | 95, 205, 295 | -5, +5, -5 |
| 모델 B: 100, 200, 300 | 85, 205, 310 | -15, +5, +10 |

평균 오차(부호 포함)는 두 모델 모두 비슷해 보입니다.  
하지만 **오차의 성질**은 다릅니다.

- 모델 A: 오차가 고르게 분포
- 모델 B: 가끔 큰 오차 발생

어떤 문제냐에 따라 어느 모델이 더 좋은지 달라집니다.  
이 때문에 여러 평가 지표가 존재합니다.

---

<a id="2"></a>
## 2️⃣ MAE — 평균 절대 오차 [↑](#toc)

### 직관 — "평균적으로 얼마나 빗나갔나"

> "예측이 평균적으로 5만원 틀렸습니다" — MAE가 5만원이라는 뜻입니다.

MAE는 가장 이해하기 쉬운 지표입니다. 단위가 원래 데이터와 같습니다.

```python
import numpy as np
from sklearn.metrics import mean_absolute_error, mean_squared_error, r2_score
from sklearn.datasets import fetch_california_housing
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

# 데이터 준비
housing = fetch_california_housing()
X_train, X_test, y_train, y_test = train_test_split(
    housing.data, housing.target, test_size=0.2, random_state=42
)

model = Pipeline([
    ('scaler', StandardScaler()),
    ('lr', LinearRegression())
])
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

# MAE
mae = mean_absolute_error(y_test, y_pred)
print(f"MAE:  {mae:.4f} ($100,000 단위)")
print(f"      → 평균 예측 오차: 약 ${mae * 100000:,.0f}")
```

실행 결과:
```
MAE:  0.5318 ($100,000 단위)
      → 평균 예측 오차: 약 $53,180
```

<details markdown="1">
<summary>📐 더 알아보기 — MAE 수식</summary>

$$
\text{MAE} = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|
$$

- 부호를 제거(절댓값)하고 평균을 냅니다
- 이상값(outlier)에 상대적으로 강건합니다
- 큰 오차와 작은 오차를 동등하게 취급합니다

</details>

<a id="3"></a>
## 3️⃣ MSE / RMSE [↑](#toc)

### 직관 — "큰 실수는 더 크게 벌점"

> 집값을 5억 틀리는 것은 5천만 원 틀리는 것보다 **훨씬** 심각합니다.  
> MSE는 오차를 제곱해서 큰 실수에 더 큰 벌점을 줍니다.

**RMSE** 는 MSE의 제곱근으로, 단위를 원래 데이터와 같게 만듭니다.

```python
# MSE와 RMSE
mse  = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)  # 또는 mean_squared_error(..., squared=False)

print(f"MSE:  {mse:.4f}")
print(f"RMSE: {rmse:.4f} ($100,000 단위)")
print(f"      → 약 ${rmse * 100000:,.0f} 오차")

# MAE vs RMSE 비교
print(f"\nMAE  < RMSE? {mae:.4f} < {rmse:.4f} = {mae < rmse}")
print("→ 항상 MAE ≤ RMSE. 차이가 클수록 큰 오차가 많다는 뜻입니다.")
```

실행 결과:
```
MSE:  0.5559
RMSE: 0.7456 ($100,000 단위)
      → 약 $74,560 오차

MAE  < RMSE? 0.5318 < 0.7456 = True
→ 항상 MAE ≤ RMSE. 차이가 클수록 큰 오차가 많다는 뜻입니다.
```

<details markdown="1">
<summary>📐 더 알아보기 — MSE / RMSE 수식</summary>

$$
\text{MSE} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2
$$

$$
\text{RMSE} = \sqrt{\text{MSE}}
$$

- 오차를 제곱하므로 이상값에 민감합니다
- RMSE는 MSE를 단위 해석 가능하게 변환한 것입니다
- 경사하강법 학습 시 MSE가 수학적으로 다루기 편합니다 (미분 가능)

**언제 무엇을 사용?**

| 상황 | 추천 지표 |
|---|---|
| 이상값 존재 | MAE |
| 큰 오차가 특히 중요 | RMSE |
| 경진대회/논문 | 보통 RMSE |

</details>

<a id="4"></a>
## 4️⃣ R² — 설명력 [↑](#toc)

### 직관 — "이 모델이 데이터를 얼마나 잘 설명하는가"

> "R² = 0.75" 는 "이 모델이 데이터 변동의 75%를 설명한다"는 뜻입니다.

- R² = 1.0 : 완벽한 예측 (현실에선 불가능)
- R² = 0.0 : 그냥 평균을 예측하는 것과 동일
- R² < 0 : 평균 예측보다도 나쁜 모델

```python
r2 = r2_score(y_test, y_pred)
print(f"R²:   {r2:.4f}")
print(f"      → 데이터 분산의 {r2*100:.1f}%를 설명합니다")

# 비교를 위한 기준 모델 (항상 평균 예측)
y_mean_pred = np.full_like(y_test, y_train.mean())
r2_baseline = r2_score(y_test, y_mean_pred)
print(f"\n기준 모델(평균 예측) R²: {r2_baseline:.4f}")
```

실행 결과:
```
R²:   0.5758
      → 데이터 분산의 57.6%를 설명합니다

기준 모델(평균 예측) R²: -0.0023
```

<details markdown="1">
<summary>📐 더 알아보기 — R² 수식</summary>

$$
R^2 = 1 - \frac{\text{SS}_{\text{res}}}{\text{SS}_{\text{tot}}}
$$

$$
\text{SS}_{\text{res}} = \sum (y_i - \hat{y}_i)^2 \quad \text{(모델의 오차)}
$$

$$
\text{SS}_{\text{tot}} = \sum (y_i - \bar{y})^2 \quad \text{(평균 대비 전체 변동)}
$$

- $\text{SS}_{\text{res}}$: 모델이 줄이지 못한 오차
- $\text{SS}_{\text{tot}}$: 원래 데이터의 전체 변동
- 분자가 0이면 (완벽 예측) R² = 1
- 분자 = 분모이면 (평균 예측과 동일) R² = 0

</details>

## 5️⃣ 잔차 분석 [↑](#toc)

### 잔차 = 실제값 - 예측값

잔차(Residual)를 분석하면 모델의 숨겨진 문제를 발견할 수 있습니다.

> **이상적인 잔차:** 예측값에 상관없이 0 근방에서 무작위로 분포

잔차에 패턴이 있다면 모델이 뭔가를 놓치고 있다는 신호입니다.

```python
import matplotlib.pyplot as plt

# 잔차 계산
residuals = y_test - y_pred

fig, axes = plt.subplots(1, 3, figsize=(15, 5))

# 1) 잔차 vs 예측값 (패턴 확인)
axes[0].scatter(y_pred, residuals, alpha=0.3, s=10, color='steelblue')
axes[0].axhline(0, color='red', linewidth=1.5)
axes[0].set_xlabel('예측값')
axes[0].set_ylabel('잔차')
axes[0].set_title('잔차 vs 예측값\n(패턴 없어야 좋음)')

# 2) 잔차 분포 히스토그램 (정규분포 확인)
axes[1].hist(residuals, bins=50, color='steelblue', edgecolor='white', alpha=0.8)
axes[1].axvline(0, color='red', linewidth=1.5)
axes[1].set_xlabel('잔차')
axes[1].set_ylabel('빈도')
axes[1].set_title('잔차 분포\n(정규분포에 가까울수록 좋음)')

# 3) 실제값 vs 예측값 (대각선이 이상적)
axes[2].scatter(y_test, y_pred, alpha=0.3, s=10, color='coral')
axes[2].plot([y_test.min(), y_test.max()],
             [y_test.min(), y_test.max()], 'k--', linewidth=1.5)
axes[2].set_xlabel('실제값')
axes[2].set_ylabel('예측값')
axes[2].set_title('실제값 vs 예측값\n(대각선에 가까울수록 좋음)')

plt.tight_layout()
plt.show()

# 잔차 기본 통계
print(f"\n잔차 통계:")
print(f"  평균:   {residuals.mean():.4f}  (0에 가까울수록 좋음)")
print(f"  표준편차: {residuals.std():.4f}")
print(f"  최솟값: {residuals.min():.4f}")
print(f"  최댓값: {residuals.max():.4f}")
```

잔차 vs 예측값 그래프에서 깔때기 모양(fan shape)이 보이면  
**이분산성(heteroscedasticity)** 문제가 있다는 신호입니다.

## 6️⃣ 교차 검증 [↑](#toc)

### "한 번 평가는 운일 수 있다"

> 시험을 한 번만 본 점수보다 여러 번 본 평균 점수가 더 신뢰할 수 있습니다.

**K-Fold 교차 검증**:
1. 데이터를 K개 조각(fold)으로 나눕니다
2. K-1개로 훈련, 나머지 1개로 평가를 K번 반복합니다
3. K개의 점수 평균을 최종 성능으로 사용합니다

```python
from sklearn.model_selection import cross_val_score, KFold

model_cv = Pipeline([
    ('scaler', StandardScaler()),
    ('lr', LinearRegression())
])

# 5-Fold 교차 검증
kf = KFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(
    model_cv, housing.data, housing.target,
    cv=kf, scoring='r2'
)

print("5-Fold 교차 검증 결과:")
for i, score in enumerate(scores, 1):
    print(f"  Fold {i}: R² = {score:.4f}")
print(f"\n평균 R²: {scores.mean():.4f} ± {scores.std():.4f}")
```

실행 결과:
```
5-Fold 교차 검증 결과:
  Fold 1: R² = 0.6025
  Fold 2: R² = 0.5962
  Fold 3: R² = 0.6087
  Fold 4: R² = 0.6073
  Fold 5: R² = 0.5834

평균 R²: 0.5996 ± 0.0093
```

```python
# 여러 지표로 동시에 교차 검증
from sklearn.model_selection import cross_validate

results = cross_validate(
    model_cv, housing.data, housing.target,
    cv=5,
    scoring=['r2', 'neg_mean_absolute_error', 'neg_root_mean_squared_error'],
    return_train_score=True
)

print("\n교차 검증 종합 결과:")
print(f"  R²:   {results['test_r2'].mean():.4f} ± {results['test_r2'].std():.4f}")
mae_cv = -results['test_neg_mean_absolute_error'].mean()
rmse_cv = -results['test_neg_root_mean_squared_error'].mean()
print(f"  MAE:  {mae_cv:.4f}")
print(f"  RMSE: {rmse_cv:.4f}")
```

## 7️⃣ 학습 곡선 [↑](#toc)

### 과적합과 과소적합을 한눈에

학습 곡선은 훈련 데이터 크기에 따른 성능 변화를 보여줍니다.

> - **과소적합(Underfitting):** 훈련/검증 모두 나쁨 → 모델이 너무 단순
> - **과적합(Overfitting):** 훈련은 좋은데 검증이 나쁨 → 모델이 너무 복잡

```python
from sklearn.model_selection import learning_curve

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

model_names = ['선형 회귀(정상)', '10차 다항 회귀(과적합 위험)']

from sklearn.preprocessing import PolynomialFeatures
models_lc = [
    Pipeline([('scaler', StandardScaler()), ('lr', LinearRegression())]),
    Pipeline([
        ('poly', PolynomialFeatures(degree=10, include_bias=False)),
        ('scaler', StandardScaler()),
        ('lr', LinearRegression())
    ])
]

for ax, model_lc, name in zip(axes, models_lc, model_names):
    train_sizes, train_scores, val_scores = learning_curve(
        model_lc, housing.data, housing.target,
        train_sizes=np.linspace(0.1, 1.0, 10),
        cv=3, scoring='r2', n_jobs=-1
    )
    
    train_mean = train_scores.mean(axis=1)
    val_mean   = val_scores.mean(axis=1)
    train_std  = train_scores.std(axis=1)
    val_std    = val_scores.std(axis=1)
    
    ax.plot(train_sizes, train_mean, 'o-', label='훈련', color='steelblue')
    ax.plot(train_sizes, val_mean, 's--', label='검증', color='tomato')
    ax.fill_between(train_sizes, train_mean - train_std, train_mean + train_std, alpha=0.1, color='steelblue')
    ax.fill_between(train_sizes, val_mean - val_std, val_mean + val_std, alpha=0.1, color='tomato')
    ax.set_title(name)
    ax.set_xlabel('훈련 데이터 크기')
    ax.set_ylabel('R²')
    ax.legend()
    ax.set_ylim([-0.5, 1.05])

plt.tight_layout()
plt.show()
```

두 곡선이 **수렴**하면 적절한 모델,  
두 곡선 사이 **간격이 크면** 과적합 신호입니다.

## 8️⃣ 정리 및 실습 [↑](#toc)

### 핵심 개념 정리

| 개념 | 설명 | 비유 |
|---|---|---|
| MAE | 오차 절댓값의 평균, 단위 동일 | "평균 5km 빗나감" |
| RMSE | MSE의 제곱근, 큰 오차에 민감 | "가끔 크게 틀리면 더 크게 감점" |
| R² | 모델이 설명하는 분산의 비율 | "이 모델로 80%는 설명된다" |
| 잔차 | 실제값 - 예측값, 패턴이 없어야 이상적 | "남은 오차의 흔적" |
| 교차 검증 | 데이터를 나눠 여러 번 평가, 신뢰성 향상 | "시험 여러 번 보기" |
| 학습 곡선 | 데이터 크기에 따른 성능 변화 시각화 | "연습량-실력 그래프" |
