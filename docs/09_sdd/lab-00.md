# SDD(Software Design Description)
## 1) SDD란?

SDD = Software Design Description
→ 소프트웨어를 어떻게 설계했는지를 문서로 정리한 설계 명세서

즉,

“요구사항(SRS)을 바탕으로, 시스템을 어떤 구조로 만들지 설계한 결과물”

⸻

## 2) SDD의 목적

SDD를 작성하는 이유는 다음과 같습니다.
	•	✅ 개발자들이 공통 이해를 갖고 개발하도록 함
	•	✅ 설계 내용을 검토/검증 가능 (리뷰 가능)
	•	✅ 구현(코딩) 전에 구조를 확정해 오류 예방
	•	✅ 유지보수 시 설계 근거를 남김
	•	✅ 테스트/QA 설계 기준 제공
