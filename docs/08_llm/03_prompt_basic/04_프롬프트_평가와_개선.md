## 1. 왜 평가가 필요한가? [↑](#toc)

### 프롬프트도 "시험"을 봐야 한다

여러분이 코드를 작성하면 테스트를 돌려봅니다. 프롬프트도 마찬가지입니다. **"이 프롬프트가 좋은 결과를 내는지"** 체계적으로 확인해야 합니다.

> **비유**: 요리 레시피를 만들 때 "한 번에 완벽한 레시피"는 없습니다. 만들어보고, 맛보고, 조정하는 과정을 반복합니다. 프롬프트도 **작성 → 실행 → 평가 → 수정**을 반복해야 좋아집니다.

### 같은 프롬프트인데 왜 다른 결과가 나올까?

| 원인 | 쉬운 설명 | 예시 |
|------|---------|------|
| **모델 차이** | AI마다 성격이 다름 | 같은 질문도 GPT와 Claude에서 다른 답변 |
| **temperature** | 같은 AI도 매번 조금씩 다르게 답함 | "CNN 설명해줘"를 3번 물으면 3번 다른 설명 |
| **엣지 케이스** (극단적 상황) | 보통은 잘 되지만 특이한 입력에서 실패 | 영어로 물으면 갑자기 영어로 답변, 빈 입력 시 오류 |

### 프롬프트 개선 사이클

```
1. 작성 — CRAFT로 프롬프트 구성
2. 실행 — AI에게 전달
3. 평가 — 4가지 기준으로 채점 (이번 장에서 배움!)
4. 분석 — 어디가 부족한지 파악
5. 수정 — 프롬프트 개선
6. 다시 실행 — 개선된 결과 확인
```

## 2. 4가지 평가 기준 [↑](#toc)

AI의 응답을 **4가지 관점**에서 채점합니다. 마치 시험 채점 루브릭처럼 사용합니다.

### 평가 기준 한눈에 보기

| 기준 | 쉬운 의미 | 이런 걸 확인합니다 |
|------|---------|-----------------|
| **정확성** | 사실에 맞는가? | AI가 지어낸 거짓 정보(환각)가 없는가? |
| **일관성** | 처음부터 끝까지 같은 톤인가? | 존댓말로 시작했는데 중간에 반말로 바뀌지 않는가? |
| **톤 적절성** | 대상에 맞게 말하는가? | 초보자에게 너무 어려운 용어를 쓰지 않는가? |
| **포맷 적합성** | 요청한 형식을 지켰는가? | 표를 달라고 했는데 줄글로 답하지 않는가? |

> **환각(Hallucination)이란?**: AI가 **그럴듯하지만 사실이 아닌 내용을 지어내는 현상**입니다. 예를 들어 "파이썬 3.15의 새 기능"처럼 존재하지 않는 버전의 정보를 마치 사실인 것처럼 답하는 것입니다. AI는 "다음에 올 확률이 높은 단어"를 예측할 뿐이라, 사실 확인을 스스로 하지 못합니다.

### 1~5점 채점 기준 예시

**정확성 루브릭:**

| 점수 | 기준 | 프로그래밍 학습 예시 |
|:---:|------|-----------------|
| 5 | 모든 사실이 정확 | "리스트는 순서가 있고 인덱스로 접근" ✅ |
| 4 | 대부분 정확, 사소한 오류 1~2개 | 설명은 맞지만 코드 예시에 오타 |
| 3 | 핵심은 맞지만 부수 정보에 오류 | "리스트는 중괄호로 만든다" (대괄호가 맞음) |
| 2 | 여러 사실 오류 | 리스트와 딕셔너리 설명이 뒤바뀜 |
| 1 | 주요 사실이 틀림 (환각) | 존재하지 않는 함수를 소개 |

### 실습: 직접 채점해보기

아래 프롬프트와 AI 응답을 보고 4가지 기준으로 채점해보세요.

```
[프롬프트]
당신은 딥러닝 전문 튜터입니다.
CNN의 핵심 구성 요소를 설명해주세요.
3가지 레이어를 표 형식으로, 역할과 비유를 포함해주세요.

[AI 응답 — 나쁜 예시]
CNN은 사진을 분석하는 AI예요 ㅎㅎ
레이어가 있고 좋습니다~
```

**채점해보세요:**

| 기준 | 점수 (?/5) | 이유 |
|------|:---:|------|
| 정확성 | ? | CNN 구성 요소에 대한 구체적 설명이 있는가? |
| 일관성 | ? | "튜터" 역할에 맞는 톤인가? |
| 톤 적절성 | ? | 전문적이면서 이해하기 쉬운가? |
| 포맷 적합성 | ? | 요청한 "표 형식"을 지켰는가? |

> **정답 예시**: 정확성 1(구체적 설명 없음), 일관성 2(너무 가벼운 톤), 톤 2(이모티콘 남발), 포맷 1(표 없음) → 평균 1.5/5.0. 이 프롬프트는 개선이 필요합니다!

## 3. AI가 AI를 채점한다 — 자동 평가 [↑](#toc)

위에서 사람이 직접 채점했는데, 프롬프트가 수십 개면 일일이 채점하기 어렵습니다. **AI에게 "평가자" 역할을 부여**하여 자동으로 채점하게 할 수 있습니다.

> **비유**: 시험을 채점하는 조교를 AI가 대신하는 것입니다. 채점 기준(루브릭)을 미리 알려주면, AI가 일관된 기준으로 빠르게 채점합니다.

### 실습: AI가 AI를 채점하기

**Step 1 — 평가할 응답 생성하기:**

```python
import json
from openai import OpenAI
client = OpenAI()

# AI에게 CNN 설명을 요청
response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[
        {"role": "system", "content": "당신은 딥러닝 전문 튜터입니다."},
        {"role": "user", "content": "CNN의 핵심 구성 요소 3가지를 설명해주세요. 표 형식으로, 각 레이어의 역할과 실생활 비유를 포함해주세요."}
    ]
)
answer = response.choices[0].message.content
print("=== 평가 대상 응답 ===")
print(answer)
```

**Step 2 — 평가자 AI에게 채점 요청:**

```python
# 평가자에게 줄 채점 기준 (루브릭)
evaluator_system = """당신은 프롬프트 응답 품질을 평가하는 전문 평가자입니다.
주어진 응답을 다음 4가지 기준으로 1~5점 채점하고, 반드시 JSON 형식으로만 출력하세요.

평가 기준:
- 정확성: 사실에 맞는가? 환각(거짓 정보)이 없는가?
- 일관성: 역할(튜터)에 맞는 톤을 처음부터 끝까지 유지하는가?
- 톤적절성: 학생 수준에 맞게 쉽게 설명하는가?
- 포맷적합성: 요청한 형식(표, 비유 포함)을 지켰는가?

출력 형식:
{
  "정확성": <1~5 정수>,
  "일관성": <1~5 정수>,
  "톤적절성": <1~5 정수>,
  "포맷적합성": <1~5 정수>,
  "총평": "<한 문장 평가>"
}"""

# 평가 실행
eval_response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[
        {"role": "system", "content": evaluator_system},
        {"role": "user", "content": f"""다음 응답을 평가해주세요.

[역할]: 딥러닝 전문 튜터
[요청]: CNN 핵심 구성 요소 3가지, 표 형식, 역할+비유
[응답]:
{answer}"""}
    ],
    temperature=0,   # 평가는 매번 같은 결과가 나오도록 temperature=0
    response_format={"type": "json_object"},  # JSON 형식 강제
)

# 결과 파싱 및 출력
# json.loads: JSON 문자열 → 파이썬 딕셔너리로 변환
scores = json.loads(eval_response.choices[0].message.content)

print("\n=== 평가 결과 ===")
for key, value in scores.items():
    print(f"  {key}: {value}")

# 평균 점수 계산
score_keys = ["정확성", "일관성", "톤적절성", "포맷적합성"]
avg = sum(scores[k] for k in score_keys) / len(score_keys)
print(f"\n  평균 점수: {avg:.1f}/5.0")
```

### 결과 해석

- **4.0 이상**: 좋은 프롬프트! 미세 조정만 하면 됩니다
- **3.0~4.0**: 괜찮지만 개선 여지가 있습니다
- **3.0 미만**: CRAFT 프레임워크로 다시 구조화해보세요

### LLM-as-a-Judge 장단점

| 장점 | 단점 |
|------|------|
| 수백 건도 수 분 내 처리 가능 | 평가자 AI 자체도 실수할 수 있음 |
| 같은 기준을 일관되게 적용 | 아주 미세한 품질 차이는 감지 어려움 |
| 사람이 일일이 채점하는 것보다 저렴 | 평가 API 호출도 토큰 비용 발생 |

> **팁**: LLM-as-a-Judge는 "대략적인 품질 추세"를 파악하는 데 좋습니다. 중요한 결과는 반드시 **사람이 최종 확인**하세요.

## 4. 반복 개선 실습 [↑](#toc)

### 프롬프트 3단계 개선 — 점수가 올라가는 걸 직접 확인!

같은 작업에 대해 **v1(모호) → v2(구조화) → v3(CRAFT)** 3가지 프롬프트를 비교합니다. 각각의 응답을 AI가 자동 채점하여, 프롬프트를 잘 쓸수록 점수가 올라가는 것을 확인합니다.

```python
# 3가지 버전의 프롬프트
prompts = {
    "v1 (모호함)": "파이썬 에러 해결해줘",

    "v2 (구조화)": "파이썬에서 IndexError가 발생했어. 원인과 해결법을 알려줘.",

    "v3 (CRAFT)": """당신은 3년차 백엔드 개발자이고, 신입 교육을 담당하고 있습니다.

아래 파이썬 코드에서 IndexError가 발생합니다. 원인을 분석하고 수정된 코드를 보여주세요.

```python
fruits = ["사과", "바나나", "딸기"]
print(fruits[3])
```

조건:
- 에러 원인을 비유를 들어 쉽게 설명
- 수정된 코드 + 왜 그렇게 수정하는지 설명
- 비슷한 실수를 방지하는 팁 1개"""
}

# 각 버전을 실행하고 자동 평가
for version, prompt in prompts.items():
    # 1) 응답 생성
    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[{"role": "user", "content": prompt}]
    )
    answer = response.choices[0].message.content

    # 2) 자동 평가 (위에서 정의한 evaluator_system 재사용)
    eval_resp = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[
            {"role": "system", "content": evaluator_system},
            {"role": "user", "content": f"[요청]: IndexError 원인 분석 및 수정\n[응답]: {answer}"}
        ],
        temperature=0,
        response_format={"type": "json_object"},
    )
    scores = json.loads(eval_resp.choices[0].message.content)

    # 3) 결과 출력
    avg = sum(scores[k] for k in score_keys) / len(score_keys)
    print(f"\n{'='*50}")
    print(f"[{version}] 평균: {avg:.1f}/5.0")
    print(f"총평: {scores.get('총평', '')}")
    print(f"응답 미리보기: {answer[:100]}...")
```

### 기대 결과

```
[v1 (모호함)]   평균: 2.5/5.0  ← 어떤 에러인지도 몰라서 두루뭉술한 답변
[v2 (구조화)]   평균: 3.5/5.0  ← 에러를 특정했지만 설명 방식이 제각각
[v3 (CRAFT)]   평균: 4.5/5.0  ← 비유+수정 코드+팁까지 체계적 답변
```

> **핵심**: v1 → v2 → v3로 갈수록 평균 점수가 올라갑니다. 이것이 **CRAFT 프레임워크가 왜 필요한지** 숫자로 보여주는 증거입니다.

> **확인**: 자신이 학습 중에 자주 사용하는 프롬프트 하나를 골라 v1→v2→v3로 개선해보세요. 평가 점수가 올라가나요?
