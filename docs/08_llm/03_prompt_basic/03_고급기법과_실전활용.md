## 1. Chain-of-Thought (CoT) — 단계별 사고 유도 [↑](#toc)

### CoT란?

**"단계별로 생각해 보세요"**라는 한 마디를 추가하는 것만으로, AI의 추론 정확도가 크게 향상되는 기법입니다.

> **비유**: 수학 시험에서 "답만 쓰세요"라고 하면 실수가 많지만, "풀이 과정을 쓰세요"라고 하면 정확도가 올라가는 것과 같습니다.

### ChatGPT에서 바로 해보기

**CoT 없이:**
```
YOLO 모델에서 mAP50이 0.75이고 mAP50-95가 0.45입니다.
이 모델의 성능은 좋은 편인가요?
```

**CoT 적용:**
```
YOLO 모델에서 mAP50이 0.75이고 mAP50-95가 0.45입니다.
이 모델의 성능은 좋은 편인가요?

단계별로 분석해주세요:
1. mAP50과 mAP50-95의 의미
2. 일반적인 기준과 비교
3. 이 수치의 의미 해석
```

> **관찰**: CoT를 추가하면 중간 계산 과정을 보여주므로, 오류를 발견하기도 쉽습니다.

### API 코드: CoT vs 일반 비교

```python
import time

problem = """딥러닝 모델 A는 학습 시간 2시간에 정확도 92%,
모델 B는 학습 시간 8시간에 정확도 95%입니다.
어떤 모델이 효율적인가요?"""

# CoT 없이
response_no_cot = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": problem + "\n답:"}]
)

# CoT 적용
response_cot = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": problem + "\n단계별로 계산해서 답해주세요."}]
)

print("=== CoT 없이 ===")
print(response_no_cot.choices[0].message.content)
print("\n=== CoT 적용 ===")
print(response_cot.choices[0].message.content)
```

### 구조화된 CoT — 단계를 직접 지정

```python
structured_cot = """다음 두 딥러닝 모델의 학습 결과를 비교 분석해주세요.

모델 A (ResNet-18): 학습 시간 2시간, 정확도 92%, 파라미터 11M, 추론 속도 15ms
모델 B (ResNet-50): 학습 시간 8시간, 정확도 95%, 파라미터 25M, 추론 속도 35ms

다음 순서로 분석해주세요:
1단계 - 정확도 대비 학습 시간 효율 비교:
2단계 - 모델 크기(파라미터)와 추론 속도 비교:
3단계 - 실시간 서비스 vs 배치 처리 관점에서 적합한 모델:
4단계 - 종합 추천:"""

response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": structured_cot}]
)
print(response.choices[0].message.content)
```

### CoT가 효과적인 상황

| 상황 | 예시 |
|------|------|
| 숫자 계산 | 모델 정확도 비교, 학습률 계산 |
| 의사결정 | "CNN vs RNN 어떤 걸 쓸까?" |
| 원인 분석 | "왜 loss가 줄어들지 않을까?" |
| 코드 디버깅 | "이 에러가 발생하는 이유는?" |

## 2. 멀티턴 대화 — 문맥을 유지하는 대화 [↑](#toc)

### 싱글턴 vs 멀티턴

| | 싱글턴 (Single-turn) | 멀티턴 (Multi-turn) |
|---|---|---|
| **형태** | 질문 1개 → 답변 1개 | 여러 차례 주고받기 |
| **문맥** | 없음 | 이전 대화 기억 |
| **적합** | 번역, 요약, 분류 | 상담, 튜터링, 기획 |
| **비용** | 낮음 | 높음 (히스토리 누적) |

### 핵심: LLM은 기억하지 못한다

> **LLM은 무상태(Stateless)**입니다. 이전 대화를 자체적으로 기억하지 않습니다. 매번 **전체 대화 히스토리**를 `messages` 배열에 담아 전송해야 합니다.

### 메시지 배열 구조

```python
messages = [
    {"role": "system", "content": "역할과 규칙 정의"},     # AI의 성격
    {"role": "user", "content": "첫 번째 질문"},           # 사용자
    {"role": "assistant", "content": "첫 번째 답변"},      # AI의 이전 답변
    {"role": "user", "content": "두 번째 질문 (이전 내용 참조)"},  # 문맥 연결
]
```

### 실습: 멀티턴 대화 구현

```python
# 대화 히스토리를 저장할 리스트
messages = [
    {"role": "system", "content": "당신은 친절한 파이썬/딥러닝 튜터입니다. 코드 예시를 포함하여 쉽게 설명합니다."}
]

def chat(user_input):
    """대화 히스토리를 유지하며 AI와 대화합니다."""
    # 1. 사용자 메시지 추가
    messages.append({"role": "user", "content": user_input})

    # 2. 전체 히스토리를 API에 전송
    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=messages,
        temperature=0.7,
    )

    # 3. AI 응답을 히스토리에 추가
    answer = response.choices[0].message.content
    messages.append({"role": "assistant", "content": answer})
    return answer

# 실제 대화
print("[나]:", "YOLO로 객체 탐지 프로젝트를 시작하려는데 뭐부터 해야 해?")
print("[AI]:", chat("YOLO로 객체 탐지 프로젝트를 시작하려는데 뭐부터 해야 해?"))

print("\n[나]:", "방금 말한 것 중에서 데이터셋 준비를 좀 더 자세히 알려줘")
print("[AI]:", chat("방금 말한 것 중에서 데이터셋 준비를 좀 더 자세히 알려줘"))
# → "방금 말한 것"을 AI가 이해함 = 멀티턴 성공!

print(f"\n현재 대화 턴 수: {len(messages)}개 메시지")
```

## 3. 구조화된 프롬프트 작성법 [↑](#toc)

### 마크다운 형식 활용

AI는 마크다운 구조를 잘 이해합니다. 헤더(#), 목록(-), 강조(**) 등을 활용하면 지시가 명확해집니다.

```python
structured_prompt = """# 역할
당신은 딥러닝 강사이며 학습 가이드 전문가입니다.

## 작업
아래 학습 주제에 대한 1주일 학습 계획표를 작성해주세요.

## 학습 주제
- 주제: CNN (합성곱 신경망)
- 수강생 수준: 파이썬 기초만 아는 비전공자
- 목표: YOLO를 활용한 이미지 분류 프로젝트 완성

## 제약 조건
- 반드시 이론, 실습, 복습 시간을 포함할 것
- 하루 학습 시간은 3시간 이내
- 무료 자료(Colab, YouTube)만 활용

## 출력 형식
일별 표 형식으로 작성 (요일 | 주제 | 학습 내용 | 실습)"""

response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": structured_prompt}]
)
print(response.choices[0].message.content)
```

### 조건 명시 패턴

| 패턴 | 예시 |
|------|------|
| **반드시 포함** | "반드시 ~를 포함해야 한다" |
| **금지 사항** | "~하지 마라", "~는 제외한다" |
| **수량 제한** | "3개 이내로", "최소 2개" |
| **우선순위** | "가장 중요한 것부터 나열" |

## 4. API 활용 실전 [↑](#toc)

### 배치 처리 — 여러 작업 한 번에

```python
# 여러 고객 리뷰를 한 번에 분류
concepts = [
    "Overfitting",
    "Batch Normalization",
    "Dropout",
    "Transfer Learning",
]

results = []
for concept in concepts:
    response = client.chat.completions.create(
        model="gpt-4.1-nano",
        messages=[{"role": "user", "content": f"'{concept}'을(를) 비전공자 학생이 이해할 수 있게 한 문장으로 설명하고, 실생활 비유를 한 줄 추가해줘."}],
        max_tokens=150
    )
    results.append({
        "개념": concept,
        "설명": response.choices[0].message.content.strip()
    })

for r in results:
    print(f"[{r['개념']}]")
    print(f"{r['설명']}\n")
```

### 토큰 사용량과 비용 확인

```python
response = client.chat.completions.create(
    model="gpt-4.1-nano",
    messages=[{"role": "user", "content": "CNN과 RNN의 차이를 비교표로 정리하고, 각각 어떤 프로젝트에 적합한지 설명해줘"}]
)

# 토큰 사용량 확인
usage = response.usage
print(f"입력 토큰:  {usage.prompt_tokens}")
print(f"출력 토큰:  {usage.completion_tokens}")
print(f"총 토큰:    {usage.total_tokens}")

# 비용 추정 (gpt-4.1-nano 기준)
# 입력: $0.10 / 1M tokens, 출력: $0.40 / 1M tokens
input_cost = usage.prompt_tokens * 0.0000001
output_cost = usage.completion_tokens * 0.0000004
total_cost = input_cost + output_cost
print(f"예상 비용:  ${total_cost:.6f} (약 {total_cost * 1400:.2f}원)")
```

### 비용 절약 팁

| 방법 | 효과 |
|------|------|
| `max_tokens` 제한 | 불필요하게 긴 응답 차단 |
| 프롬프트 간결화 | 입력 토큰 절감 |
| `gpt-4.1-nano` 선택 | GPT-4o 대비 약 **10배 저렴** |
| Few-shot 예시 최소화 | 예시당 수십~수백 토큰 절약 |

> **프롬프트 길이 최적화**: 연구에 따르면 프롬프트의 최적 길이는 **150~300 단어**입니다. 3,000 토큰을 넘으면 오히려 성능이 저하될 수 있습니다. "더 많이 쓸수록 좋다"가 아닙니다.

> **확인**: 딥러닝 개념 100개를 gpt-4.1-nano로 설명시키면 비용이 약 얼마일까요? (힌트: 개념당 약 150 토큰)
