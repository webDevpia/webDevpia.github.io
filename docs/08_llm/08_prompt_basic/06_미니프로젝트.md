## 1. 프로젝트 A: AI 챗봇 (역할 전환형) [↑](#toc)

**학습목표**: Streamlit과 OpenAI API를 조합하여 역할을 전환할 수 있는 멀티턴 챗봇을 만들 수 있다.

```
목표: 사용자가 역할(Python 튜터/커리어 상담사/기술 면접관)을 선택하면 해당 페르소나로 대화하는 챗봇
기술: Streamlit + OpenAI API + 멀티턴 대화
```

### 사용할 기법

| 기법 | 적용 위치 |
|------|-----------|
| Role Prompting | system 메시지에 역할별 페르소나 설정 |
| 멀티턴 대화 | session_state로 대화 히스토리 관리 |
| 포맷 제어 | 역할 변경 시 히스토리 초기화 |

### 완성 코드 (app_chatbot.py)

```python
# ===================================
# 프로젝트 A: 역할 전환형 AI 챗봇
# 파일명: app_chatbot.py
# 실행: streamlit run app_chatbot.py
# ===================================

import os
import streamlit as st
from openai import OpenAI

# OpenAI 클라이언트 초기화
# 환경 변수에서 API 키 로드 (Streamlit Cloud: st.secrets 사용 권장)
client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY", ""))

# ── 역할별 시스템 프롬프트 정의 ──────────────────────────
ROLES = {
    "Python 튜터": """당신은 친절하고 열정적인 Python 튜터입니다.
신입 개발자의 코딩 질문에 항상 예제 코드를 포함해 답변합니다.
잘못된 코드를 보여주면 부드럽게 교정하고 더 나은 방법을 알려줍니다.
항상 마지막에 '더 궁금한 점이 있나요?' 라고 묻습니다.""",

    "커리어 상담사": """당신은 IT 업계 10년 경력의 커리어 상담사입니다.
신입 개발자의 취업 고민, 기술 스택 선택, 커리어 방향에 대해 조언합니다.
판단 없이 경청하고, 구체적인 액션 아이템을 제시합니다.
필요시 관련 학습 리소스나 커뮤니티를 추천합니다.""",

    "기술 면접관": """당신은 IT 기업의 시니어 기술 면접관입니다.
기술 역량과 문제 해결 능력을 평가하는 질문을 합니다.
답변 후 꼬리 질문으로 깊이를 검증합니다.
면접 후 구체적인 피드백도 제공합니다. 어투는 정중하되 날카롭게 유지합니다.""",
}
역할_프롬프트 = ROLES

# ── 페이지 설정 ───────────────────────────────────────────
st.set_page_config(
    page_title="역할 전환 AI 챗봇",
    page_icon="🤖",
    layout="centered",
)

st.title("역할 전환 AI 챗봇")
st.caption("역할을 선택하면 해당 페르소나로 대화합니다.")

# ── 사이드바: 역할 선택 ──────────────────────────────────
with st.sidebar:
    st.header("설정")

    # 역할 선택 라디오 버튼
    선택_역할 = st.radio(
        "대화 상대 역할 선택",
        options=list(역할_프롬프트.keys()),
        index=0,
    )

    # 역할이 변경되면 대화 히스토리 초기화
    if "현재_역할" not in st.session_state:
        st.session_state["현재_역할"] = 선택_역할

    if st.session_state["현재_역할"] != 선택_역할:
        st.session_state["현재_역할"] = 선택_역할
        st.session_state["messages"] = []  # 역할 변경 시 히스토리 리셋
        st.rerun()

    st.divider()
    st.markdown(f"**현재 역할**: {선택_역할}")
    st.markdown(f"**대화 수**: {len(st.session_state.get('messages', []))} 턴")

    # 대화 초기화 버튼
    if st.button("대화 초기화", use_container_width=True):
        st.session_state["messages"] = []
        st.rerun()

# ── 대화 히스토리 초기화 (최초 실행 시) ────────────────────
if "messages" not in st.session_state:
    st.session_state["messages"] = []

# ── 이전 대화 표시 ────────────────────────────────────────
for msg in st.session_state["messages"]:
    with st.chat_message(msg["role"]):
        st.write(msg["content"])

# ── 사용자 입력 처리 ──────────────────────────────────────
if user_input := st.chat_input(f"{선택_역할}에게 말을 걸어보세요..."):
    # 사용자 메시지를 화면에 표시
    with st.chat_message("user"):
        st.write(user_input)

    # 히스토리에 사용자 메시지 추가
    st.session_state["messages"].append({
        "role": "user",
        "content": user_input,
    })

    # API 호출용 messages 배열 구성 (system + 전체 히스토리)
    api_messages = [
        {"role": "system", "content": 역할_프롬프트[선택_역할]},
    ] + st.session_state["messages"]

    # OpenAI API 스트리밍 호출
    with st.chat_message("assistant"):
        with st.spinner("생각 중..."):
            response = client.chat.completions.create(
                model="gpt-4o-mini",
                messages=api_messages,
                temperature=0.7,
                stream=True,  # 스트리밍으로 타이핑 효과
            )

            # 스트리밍 결과를 화면에 실시간 출력
            전체_답변 = st.write_stream(response)

    # 히스토리에 AI 답변 추가
    st.session_state["messages"].append({
        "role": "assistant",
        "content": 전체_답변,
    })
```

## 2. 프로젝트 B: 자기소개서 첨삭 서비스 [↑](#toc)

**학습목표**: Few-shot + Role Prompting을 조합하여 직무별 맞춤 자소서 피드백 서비스를 만들 수 있다.

```
목표: 자소서를 입력하면 직무별 맞춤 피드백 + 개선안을 생성하는 웹앱
기술: Streamlit + OpenAI API + Few-shot + Role Prompting
```

### 사용할 기법

| 기법 | 적용 위치 |
|------|-----------|
| Role Prompting | "10년 경력 자소서 멘토" system 메시지 |
| Few-shot | 직무별 좋은 예시 / 나쁜 예시 2쌍 제공 |
| 포맷 제어 | 강점 3개, 개선점 3개, 수정안 구조화 출력 |

### 완성 코드 (app_resume.py)

```python
# ===================================
# 프로젝트 B: 자기소개서 첨삭 서비스
# 파일명: app_resume.py
# 실행: streamlit run app_resume.py
# ===================================

import os
import streamlit as st
from openai import OpenAI

client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY", ""))

# ── 직무별 Few-shot 예시 ─────────────────────────────────
# 각 직무마다 (나쁜 예시 → 좋은 예시) 쌍을 제공합니다
직무_예시 = {
    "백엔드 개발자": [
        {
            "role": "user",
            "content": """자소서: '저는 Python과 Django를 사용해 본 적이 있습니다. 서버 개발에 관심이 있고 열심히 배우겠습니다.'
직무: 백엔드 개발자"""
        },
        {
            "role": "assistant",
            "content": """**강점**
1. 핵심 백엔드 기술(Python, Django)을 명시했습니다.
2. 서버 개발 직무에 대한 관심을 표현했습니다.
3. 학습 의지를 보여주었습니다.

**개선점**
1. 구체적인 프로젝트 경험이나 성과 수치가 없어 역량 증명이 부족합니다.
2. '열심히 배우겠습니다'는 모든 지원자가 쓰는 표현으로 차별화가 없습니다.
3. DB 설계, API 설계, 배포 경험 등 백엔드 핵심 역량이 드러나지 않습니다.

**수정된 자소서**
Django REST Framework로 팀 프로젝트의 REST API 15개를 설계·구현하고,
PostgreSQL 기반 DB 스키마를 설계하여 N+1 쿼리 문제를 해결한 경험이 있습니다.
Docker로 개발 환경을 컨테이너화하고 GitHub Actions CI/CD 파이프라인을 구축하며,
'동작하는 코드'를 넘어 '배포 가능한 서비스'를 만드는 역량을 키워왔습니다."""
        },
    ],
    "프론트엔드 개발자": [
        {
            "role": "user",
            "content": """자소서: '저는 HTML, CSS, JavaScript를 할 줄 압니다. 웹 개발에 관심이 있으며 열심히 하겠습니다.'
직무: 프론트엔드 개발자"""
        },
        {
            "role": "assistant",
            "content": """**강점**
1. 핵심 프론트엔드 기술 스택(HTML/CSS/JS)을 명시했습니다.
2. 직무 관심을 분명히 표현했습니다.
3. 성실성을 강조했습니다.

**개선점**
1. 구체적인 프로젝트나 수치가 없어 역량 증명이 부족합니다.
2. "열심히 하겠습니다"는 모든 지원자가 쓰는 표현으로 차별화가 없습니다.
3. React, TypeScript 등 현업에서 요구하는 심화 기술을 언급하지 않았습니다.

**수정된 자소서**
저는 React와 TypeScript를 활용하여 팀 프로젝트에서 사용자 인터랙션 응답 속도를 40% 개선한 경험이 있습니다. Lighthouse 성능 점수를 72점에서 95점으로 끌어올린 리팩토링 작업을 주도하며, 퍼포먼스 최적화에 강점을 갖추었습니다."""
        },
    ],
}

# 직무별 기본 few-shot이 없는 경우 공통 예시 사용
공통_예시 = [
    {
        "role": "user",
        "content": "자소서: '프론트엔드와 백엔드 모두 경험해 보았습니다. 다양한 기술을 배우는 것을 좋아합니다.'\n직무: 풀스택 개발자"
    },
    {
        "role": "assistant",
        "content": """**강점**
1. 프론트엔드·백엔드 양쪽 경험을 어필했습니다.
2. 기술 학습에 대한 열정을 보여줬습니다.
3. 풀스택 직무의 핵심인 넓은 범위를 인식하고 있습니다.

**개선점**
1. 어떤 기술로 무엇을 만들었는지 구체적 프로젝트가 없습니다.
2. '다양한 기술을 배우는 것을 좋아합니다'는 깊이 부족으로 읽힐 수 있습니다.
3. 프론트↔백엔드를 연결한 경험(API 연동, 배포)이 드러나지 않습니다.

**수정된 자소서**
React + Express.js로 할일 관리 웹앱을 개인 프로젝트로 개발하여 Vercel에 배포했습니다.
프론트엔드에서 Axios로 REST API를 호출하고, 백엔드에서 MongoDB 연동 및 JWT 인증을 구현하며,
한 서비스의 전체 흐름을 경험했습니다. 이 과정에서 CORS 설정, 환경 변수 관리 등
실무에서 마주치는 문제를 직접 해결하는 능력을 길렀습니다."""
    },
]

# ── 첨삭 함수 ─────────────────────────────────────────────
def 자소서_첨삭(직무: str, 자소서_내용: str) -> str:
    """직무와 자소서를 받아 첨삭 결과를 반환합니다."""

    # Role Prompting: system 메시지로 전문가 역할 부여
    system_message = """당신은 IT 기업 인사 담당자이며 10년 경력의 자기소개서 전문 멘토입니다.
수백 명의 지원자를 검토한 경험을 바탕으로, 직무에 맞는 핵심 역량이 드러나도록 피드백합니다.
항상 다음 형식으로 출력하세요:

**강점**
1. ...
2. ...
3. ...

**개선점**
1. ...
2. ...
3. ...

**수정된 자소서**
(개선점을 반영한 완성된 자소서 작성)"""

    # Few-shot: 직무별 예시 포함 (없으면 공통 예시 사용)
    few_shot = 직무_예시.get(직무, 공통_예시)

    # 최종 사용자 메시지
    user_message = f"자소서: '{자소서_내용}'\n직무: {직무}"

    messages = [
        {"role": "system", "content": system_message},
        *few_shot,  # Few-shot 예시 삽입
        {"role": "user", "content": user_message},
    ]

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        temperature=0.5,  # 창의적 개선안을 위해 중간값 설정
    )

    return response.choices[0].message.content

# ── Streamlit UI ──────────────────────────────────────────
st.set_page_config(
    page_title="자기소개서 첨삭 서비스",
    page_icon="📝",
    layout="centered",
)

st.title("자기소개서 첨삭 서비스")
st.caption("직무를 선택하고 자소서를 입력하면 맞춤 피드백을 드립니다.")

# 직무 선택 드롭다운
job_role = st.selectbox("지원 직무", ["백엔드 개발자", "프론트엔드 개발자", "풀스택 개발자", "AI 엔지니어"])
선택_직무 = job_role

# 자소서 텍스트 입력
자소서_입력 = st.text_area(
    "자기소개서 입력",
    height=200,
    placeholder="첨삭받을 자기소개서 내용을 입력하세요. (지원 동기, 직무 역량, 경험 등)",
)

# 글자 수 표시
if 자소서_입력:
    st.caption(f"입력 글자 수: {len(자소서_입력)}자")

# 첨삭하기 버튼
if st.button("첨삭하기", type="primary", use_container_width=True):
    if not 자소서_입력.strip():
        st.warning("자기소개서를 입력해 주세요.")
    elif len(자소서_입력.strip()) < 20:
        st.warning("더 구체적인 내용을 입력해 주세요. (최소 20자 이상)")
    else:
        with st.spinner(f"{선택_직무} 직무 기준으로 첨삭 중..."):
            결과 = 자소서_첨삭(선택_직무, 자소서_입력)

        st.divider()
        st.subheader(f"첨삭 결과 - {선택_직무} 직무")
        st.markdown(결과)

        # 다운로드 버튼
        st.download_button(
            label="첨삭 결과 저장 (.txt)",
            data=결과,
            file_name=f"자소서_첨삭_{선택_직무}.txt",
            mime="text/plain",
        )
```

### Google Colab에서 실행 (ngrok 사용)

Colab 환경에서는 외부 접속이 막혀 있으므로 ngrok 또는 localtunnel로 공개 URL을 만듭니다.

```python
# ===================================
# Colab에서 Streamlit 실행하기
# ===================================

# 1. 패키지 설치
# !pip install streamlit openai pyngrok -q

# 2. API 키 설정 (Colab Secrets 권장)
import os
from google.colab import userdata
os.environ["OPENAI_API_KEY"] = userdata.get("OPENAI_API_KEY")

# 3. 앱 파일을 작성한 후 아래 실행
import subprocess
import threading
from pyngrok import ngrok

# ngrok 인증 토큰 설정 (https://dashboard.ngrok.com 에서 발급)
ngrok.set_auth_token("YOUR_NGROK_TOKEN")

# Streamlit을 백그라운드로 실행
def run_streamlit():
    subprocess.run(["streamlit", "run", "app_chatbot.py", "--server.port", "8501"])

thread = threading.Thread(target=run_streamlit)
thread.daemon = True
thread.start()

# ngrok 터널 생성
public_url = ngrok.connect(8501)
print(f"공개 URL: {public_url}")
```

**핵심**: 프로젝트 A(챗봇)는 역할 기반 + 멀티턴의 종합 실습이고, 프로젝트 B(자소서 첨삭)는 Few-shot + Role의 종합 실습이다. 어떤 프로젝트든 "프롬프트 설계 → API 연동 → 웹 서비스"의 전체 흐름을 체험하는 것이 핵심이다.
