## 1. 프롬프트의 4가지 구성 요소 [↑](#toc)

**학습목표**: 좋은 프롬프트를 구성하는 4가지 요소를 설명하고, 각 요소가 포함된 프롬프트를 작성할 수 있다.

### 구성 요소 정리

| 요소 | 영문 | 설명 | 예시 |
|------|------|------|------|
| 명령 | Instruction | 모델에게 수행할 작업 지시 | "다음 문장을 요약해줘" |
| 맥락 | Context | 작업에 필요한 배경 정보 | "이 글은 초등학생 대상 교육 자료입니다" |
| 입력 데이터 | Input Data | 실제 처리할 텍스트나 데이터 | "원문: {텍스트}" |
| 출력 지시문 | Output Indicator | 원하는 응답 형식 안내 | "3줄 이내로 작성해줘" |

### 4가지 요소를 조합한 예시

```python
prompt = """
당신은 IT 스타트업의 시니어 백엔드 개발자입니다.           # 맥락 (Context)
다음 기술 개념을 신입 개발자가 이해할 수 있게 설명해주세요. # 명령 (Instruction)

개념: REST API의 GET과 POST 차이                           # 입력 데이터 (Input Data)

실무 예시를 포함하여 3문장 이내로 답해주세요.              # 출력 지시문 (Output Indicator)
"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": prompt}]
)
print(response.choices[0].message.content)
```

**핵심**: 4가지 요소를 모두 포함할 필요는 없지만, 명령이 모호할수록 맥락과 출력 지시문이 중요해진다.

> **확인**: 위 예시에서 맥락(Context)을 제거하면 응답이 어떻게 달라지는지 비교해보자.

## 2. Zero-shot Prompting [↑](#toc)

**학습목표**: Zero-shot 프롬프트를 작성하고, 장점과 한계를 설명할 수 있다.

### 개념

예시(Example)를 전혀 제공하지 않고 바로 작업을 지시하는 방식입니다. 모델이 학습 데이터에서 쌓은 지식만으로 답변합니다.

| 항목 | 내용 |
|------|------|
| 장점 | 프롬프트가 짧아 토큰 절약, 빠른 작성 |
| 한계 | 복잡하거나 형식이 중요한 작업에서는 품질이 낮을 수 있음 |
| 적합한 경우 | 번역, 요약, 간단한 질문 응답 |

### 감정 분석 실습

```python
# Zero-shot: 예시 없이 바로 지시
zero_shot_prompt = """다음 코드 리뷰 코멘트의 톤을 '칭찬', '개선 요청', '질문' 중 하나로 분류해주세요.

코멘트: '이 부분은 동작하지만, 변수명을 좀 더 명확하게 바꾸면 가독성이 올라갈 것 같아요.'

분류:"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": zero_shot_prompt}]
)
print(response.choices[0].message.content)
```

**핵심**: Zero-shot은 가장 간단한 형태의 프롬프트이다. 결과가 불만족스럽다면 예시를 추가하는 Few-shot으로 전환하는 것을 고려한다.

> **확인**: "정말 잘 구현했네요!"와 "이 부분 왜 이렇게 작성하셨나요?"를 각각 Zero-shot으로 분류해보자. 결과가 예상과 일치하는가?

## 3. Few-shot Prompting [↑](#toc)

**학습목표**: Few-shot 프롬프트에서 예시를 구성하는 방법을 이해하고, Zero-shot 대비 출력 일관성이 향상되는 것을 확인할 수 있다.

### 개념

2~5개의 입력-출력 예시를 프롬프트에 함께 제공하여, 모델이 원하는 출력 패턴을 학습하도록 유도합니다.

| 항목 | 내용 |
|------|------|
| 장점 | 출력 형식과 스타일의 일관성 향상 |
| 단점 | 예시만큼 토큰이 추가로 소비됨 |
| 적합한 경우 | 특정 형식 유지가 중요한 분류·변환 작업 |

### 분류 작업 실습

```python
few_shot_prompt = """다음 코드 리뷰 코멘트의 톤을 '칭찬', '개선 요청', '질문' 중 하나로 분류해주세요.

코멘트: '깔끔하게 잘 정리했네요!'
톤: 칭찬

코멘트: '이 함수는 너무 길어서 분리하는 게 좋겠습니다.'
톤: 개선 요청

코멘트: '여기서 try-except를 쓴 이유가 있나요?'
톤: 질문

코멘트: '이 부분은 동작하지만, 변수명을 좀 더 명확하게 바꾸면 가독성이 올라갈 것 같아요.'
톤:"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": few_shot_prompt}]
)
print(response.choices[0].message.content)
```

### Zero-shot vs Few-shot 비교

```python
comments = [
    "에러 핸들링이 잘 되어 있어서 안정적이네요.",
    "이 로직은 O(n²)인데 O(n)으로 줄일 수 있을 것 같습니다.",
    "이 변수는 어디서 초기화되나요?",
    "테스트 코드도 함께 작성해주시면 좋겠습니다."
]

print("=== Zero-shot ===")
for comment in comments:
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": f"다음 코드 리뷰 코멘트의 톤을 '칭찬'/'개선 요청'/'질문' 중 하나로 분류해주세요.\n코멘트: {comment}\n톤:"}]
    )
    print(f"{comment} → {resp.choices[0].message.content.strip()}")
```

**핵심**: Few-shot은 예시를 통해 모델에게 "이런 형식으로 답해라"고 알려준다. 예시의 품질이 결과의 품질을 결정한다.

> **확인**: 예시를 1개, 3개, 5개로 늘려가며 결과의 일관성이 어떻게 변하는지 확인해보자.

## 4. Role Prompting [↑](#toc)

**학습목표**: system 메시지로 역할을 부여하는 방법을 이해하고, 동일한 질문에 대해 역할에 따른 답변 차이를 비교할 수 있다.

### 개념

`messages`의 `role: "system"` 메시지를 통해 모델에게 특정 역할이나 페르소나를 부여합니다. 이 설정은 이후 모든 대화에 영향을 미칩니다.

```python
def ask_with_role(system_role, question):
    """역할(system)을 지정하여 질문하는 함수"""
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": system_role},
            {"role": "user", "content": question}
        ]
    )
    return response.choices[0].message.content
```

### 역할에 따른 답변 비교

```python
question = "Git 브랜치 전략을 설명해주세요. 우리 팀은 4명이고 2주 단위 스프린트를 씁니다."

# 역할 1: 신입 교육 담당 사수
role_mentor = "당신은 신입 개발자 교육을 담당하는 사수입니다. 쉬운 비유를 써서 설명해주세요."
print("=== 신입 교육 담당 사수 ===")
print(ask_with_role(role_mentor, question))
print()

# 역할 2: 시니어 아키텍트
role_architect = "당신은 대기업 소프트웨어 아키텍트입니다. 정확한 기술 용어와 함께 체계적으로 설명해주세요."
print("=== 시니어 아키텍트 ===")
print(ask_with_role(role_architect, question))
print()

# 역할 3: 스타트업 CTO
role_cto = "당신은 5인 스타트업 CTO입니다. 현실적이고 실용적인 관점으로 답변해주세요."
print("=== 스타트업 CTO ===")
print(ask_with_role(role_cto, question))
```

### 역할 설정 팁

| 구성 요소 | 예시 |
|----------|------|
| 직업/직책 | "당신은 마케팅 전문가입니다" |
| 대상 독자 | "대학생을 위해 설명해주세요" |
| 말투/스타일 | "친근하고 간결하게 답변해주세요" |
| 제약 조건 | "전문 용어 없이, 300자 이내로" |

**핵심**: system 메시지는 모델의 행동 방식 전체를 결정한다. 역할을 구체적으로 설정할수록 더 일관된 답변을 얻을 수 있다.

> **확인**: "당신은 비판적인 에디터입니다"와 "당신은 격려를 잘하는 코치입니다"라는 두 역할로 같은 글을 평가시켜보자. 어떻게 다른가?

## 5. JSON 구조화 출력 [↑](#toc)

**학습목표**: `response_format`을 설정하여 JSON 형식의 응답을 받고, Python에서 파싱할 수 있다.

### 개념

자유 형식 텍스트 대신 JSON으로 응답을 받으면 후속 처리(파싱, 저장, 전송)가 쉬워집니다.

```python
import json

# JSON 출력 형식 지정
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {
            "role": "system",
            "content": "당신은 API 응답을 JSON 형식으로 반환하는 백엔드 서버입니다."
        },
        {
            "role": "user",
            "content": """다음 사용자 정보를 REST API 응답 형식의 JSON으로 반환해주세요.
사용자: 김민수, 28세, 백엔드 개발자, 서울
{
  "status": 200,
  "data": {
    "name": "이름",
    "age": 나이,
    "role": "직무",
    "location": "지역"
  }
}"""
        }
    ],
    response_format={"type": "json_object"}  # JSON 출력 강제
)

# JSON 파싱
api_response = json.loads(response.choices[0].message.content)
data = api_response['data']
print(f"status: {api_response['status']}")
print(f"이름: {data['name']}")
print(f"나이: {data['age']}")
print(f"직무: {data['role']}")
print(f"지역: {data['location']}")
```

### 복수 항목 JSON 출력

```python
# 에러 로그 여러 건을 한 번에 분석
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "JSON 형식으로만 응답해주세요."},
        {"role": "user", "content": """다음 에러 로그 3건을 분석하여 {"errors": [...]} 형태로 반환해주세요.
각 항목은 severity(high/medium/low), category, message, suggested_fix 필드를 포함해야 합니다.

로그 1: TypeError: Cannot read properties of undefined (reading 'map')
로그 2: Warning: Each child in a list should have a unique 'key' prop
로그 3: CORS policy: No 'Access-Control-Allow-Origin' header"""}
    ],
    response_format={"type": "json_object"}
)

result = json.loads(response.choices[0].message.content)
for error in result["errors"]:
    print(f"[{error['severity'].upper()}] {error['category']}: {error['message']}")
    print(f"  -> {error['suggested_fix']}")
```

**핵심**: `response_format={"type": "json_object"}`를 사용할 때는 system 또는 user 메시지에 "JSON" 단어가 반드시 포함되어야 한다.

> **확인**: `response_format`을 제거하고 같은 프롬프트를 실행하면 어떤 형식으로 응답이 오는가? 파싱이 쉬운가?

## 6. 한국어 프롬프트 팁 [↑](#toc)

**학습목표**: 한국어의 언어적 특성을 고려한 프롬프트 작성 원칙을 이해하고 적용할 수 있다.

> **팁**: 한국어는 주어·목적어가 생략되기 쉬운 고맥락 언어입니다. 프롬프트에서는 주어, 대상, 조건을 명시적으로 적는 것이 좋습니다.

### 개선 전후 비교

| 항목 | 모호한 프롬프트 | 명시적 프롬프트 |
|------|---------------|----------------|
| 주어 생략 | "설명해줘" | "AI가 무엇인지 설명해줘" |
| 대상 불명확 | "쉽게 써줘" | "중학생이 이해할 수 있도록 쉽게 써줘" |
| 조건 부재 | "요약해줘" | "핵심 내용만 3문장으로 요약해줘" |

### 명시적 프롬프트 예시

```python
# 모호한 버전
vague = "요약해줘."

# 명시적 버전
explicit = """다음 기술 블로그 포스트를 신입 개발자 관점에서 핵심 내용 3가지를 bullet point 형식으로 요약해주세요. 각 항목에 '실무 적용 포인트'를 한 줄씩 추가해주세요.

블로그: {여기에 블로그 내용 입력}"""

# 명시적 프롬프트로 호출
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": explicit.replace("{여기에 블로그 내용 입력}", "최근 마이크로서비스 아키텍처에서 모놀리식으로 회귀하는 기업들이 늘고 있다. Netflix, Amazon 등이 일부 서비스를 다시 통합하며, 작은 팀에서는 모놀리식이 오히려 개발 속도와 디버깅에 유리하다는 분석이 나오고 있다.")}]
)
print(response.choices[0].message.content)
```

**핵심**: "누가, 무엇을, 어떻게, 어떤 형식으로"를 명시하면 한국어 프롬프트의 품질이 크게 향상된다.

> **확인**: 평소 자주 쓰는 프롬프트를 하나 골라, 주어·대상·형식을 모두 명시한 버전으로 다시 작성해보자. 응답 품질이 어떻게 달라지는가?
