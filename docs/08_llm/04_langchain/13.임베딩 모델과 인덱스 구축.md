# OpenAI 임베딩 모델과 Pinecone 인덱스 구축

<a id="part1"></a>

## 🧠 1️⃣ 임베딩(Embedding)이란? [↑](#toc)

**임베딩(Embedding)**은 텍스트를 **벡터(숫자 배열)**로 변환하여  
컴퓨터가 의미적으로 이해하고 비교할 수 있게 하는 기술입니다.

| 구분 | 예시 | 설명 |
|------|------|------|
| 입력 텍스트 | "기생충은 사회 계급을 다룬 영화이다." | 자연어 문장 |
| 출력 벡터 | [0.11, -0.02, 0.87, ...] | 1536차원 실수 벡터 |
| 활용 | 유사도 계산, 검색, 분류, 추천 | 의미 기반 처리 가능 |

💡 **OpenAI의 `text-embedding-3-small`** 모델은  
- **1536차원 벡터 출력**  
- **다국어(한국어 포함) 지원**  
- **RAG 및 검색 시스템에 최적화** 되어 있습니다.

## 🗃️ 2️⃣ Pinecone 인덱스란? [↑](#toc)

**Pinecone**은 벡터를 효율적으로 저장하고 검색하기 위한 **벡터 데이터베이스**입니다.

| 구성요소 | 설명 |
|----------|------|
| **ID** | 각 벡터를 식별하는 고유 키 |
| **벡터(embedding)** | 텍스트의 수치 표현 |
| **메타데이터(metadata)** | 제목, 장르, 연도 등 부가 정보 |

🧩 **핵심 역할**
- 대용량 벡터 저장
- 유사도 검색 (cosine, dot product 등)
- 메타데이터 필터링 지원

## 🏗️ 4️⃣ Pinecone 인덱스 생성 [↑](#toc)

```python
from dotenv import load_dotenv
from pinecone import Pinecone, ServerlessSpec
import os

# ✅ 1️⃣ 환경변수 로드
# .env 파일에 저장된 API 키를 파이썬 코드에서 읽을 수 있도록 불러옵니다.
# 키를 코드에 직접 쓰면 유출 위험이 있으므로 별도 파일로 관리합니다.
load_dotenv()

# ✅ 2️⃣ Pinecone 초기화
# API 키를 사용해 Pinecone 서비스에 접속합니다.
# os.getenv()는 .env 파일에서 해당 키 이름의 값을 가져옵니다.
pc = Pinecone(api_key=os.getenv("PINECONE_API_KEY"))

# 사용할 인덱스(벡터를 저장하는 공간)의 이름을 지정합니다.
# 인덱스는 엑셀의 시트처럼 데이터를 구분하는 단위입니다.
index_name = "movie-index"

# 같은 이름의 인덱스가 이미 있으면 새로 만들지 않습니다.
# 중복 생성을 막아 비용 낭비와 데이터 충돌을 방지합니다.
if not pc.has_index(index_name):
    pc.create_index(
        name=index_name,
        # dimension: 임베딩 벡터의 길이(차원 수)입니다.
        # OpenAI text-embedding-3-small 모델은 1536개의 숫자로 하나의 문장을 표현합니다.
        # 이 값은 나중에 사용할 임베딩 모델의 출력 크기와 반드시 일치해야 합니다.
        dimension=1536,
        # metric: 두 벡터가 얼마나 비슷한지 측정하는 방식입니다.
        # "cosine"(코사인 유사도)은 벡터의 방향을 비교하며, 텍스트 유사도 측정에 가장 많이 씁니다.
        metric="cosine",
        # spec: 인덱스를 어느 클라우드 서버에 만들지 지정합니다.
        # ServerlessSpec은 서버를 직접 관리하지 않아도 되는 간편한 방식입니다.
        spec=ServerlessSpec(
            cloud="aws",       # 또는 "gcp"
            region="us-east-1" # 사용 중인 리전으로 교체
        ),
    )

# 이미 존재하거나 방금 만든 인덱스에 연결합니다.
# 이후 이 index 객체를 통해 데이터 저장과 검색을 수행합니다.
index = pc.Index(index_name)
print(f"✅ Pinecone 인덱스 준비 완료 → {index_name}")
```

💡 **주의사항**
- `dimension`은 임베딩 모델의 출력 크기(`1536`)와 일치해야 합니다.

## 🧩 6️⃣ 하이브리드 검색을 고려한 필드 설계 [↑](#toc)

| 검색 조건 | 필드 예시 | 필터 예시 |
|------------|------------|------------|
| 장르 필터 | genre | `{"genre": {"$in": ["드라마", "스릴러"]}}` |
| 연도 필터 | year | `{"year": {"$gte": 2020}}` |
| 평점 필터 | rating | `{"rating": {"$gt": 8.0}}` |
| 배우 필터 | actors | `{"actors": {"$in": ["송강호"]}}` |

💡 **하이브리드 검색**
> 벡터 유사도 + 메타데이터 조건을 함께 사용해  
> 더 정확한 검색 결과를 제공합니다.

✅ 결과 확인 — 구축한 인덱스가 잘 동작하는지 검증

> 아래는 인덱스가 올바르게 구축되었는지 **확인**하기 위한 간단한 검색입니다. 본격적인 검색 기법(메타데이터 필터링, 하이브리드 검색 등)은 **14장**에서 자세히 다룹니다.

> 💡 Pinecone의 코사인 유사도(cosine similarity)는 **값이 클수록 더 유사**합니다 (0~1 범위). 아래 결과에서 기생충(0.6398)이 응답하라 1988(0.2984)보다 높으므로 쿼리와 더 유사합니다.

```python
# ✅ 7️⃣ 검색 테스트
# 사용자가 입력할 검색 문장입니다.
# 이 문장도 임베딩 모델로 벡터화된 뒤, Pinecone에 저장된 벡터들과 비교됩니다.
query = "가난과 부유함의 차이를 다룬 영화"

# similarity_search_with_score: 벡터 유사도 기준으로 가장 비슷한 문서를 찾습니다.
# k=3: 상위 3개 결과를 반환합니다 (데이터가 10개이므로 충분히 비교 가능).
# 반환값은 (Document 객체, 유사도 점수) 쌍의 리스트입니다.
results = vector_store.similarity_search_with_score(query, k=3)

print("\n🔍 검색 결과:")
for doc, score in results:
    # doc.metadata: 저장할 때 넣어둔 부가 정보(제목, 연도 등)를 꺼냅니다.
    print(f"🎬 {doc.metadata['title']} ({int(doc.metadata['year'])})")
    # doc.page_content: 임베딩의 원본 텍스트(줄거리)입니다.
    print(f"💬 {doc.page_content}")
    # score: 코사인 유사도 점수로, 1에 가까울수록 검색어와 의미가 비슷합니다.
    # :.4f는 소수점 4자리까지 출력하는 형식 지정입니다.
    print(f"🔢 유사도 점수: {score:.4f}\n")
```

```
🔢 유사도 점수: 0.3104

```

> "가난과 부유함의 차이"라는 쿼리에 기생충(계급 격차)이 가장 높은 유사도로 검색됩니다. 7번방의 선물과 미나리도 "가족"이라는 의미적 연결이 있어 상위에 나타납니다. 10개 영화 중 전쟁, 로맨스, 범죄 장르의 영화는 유사도가 낮아 결과에 포함되지 않았습니다.

### 메타데이터 필터 검색 맛보기

> 위의 검색은 10개 영화 **전체**에서 의미적으로 가장 유사한 것을 찾았습니다.
> 하지만 실제 서비스에서는 "2015년 이후 영화만"처럼 **조건을 걸어 검색 범위를 좁히는** 경우가 많습니다.
> 여기서는 간단한 맛보기만 하고, **다양한 필터 조건과 하이브리드 검색은 14장에서 자세히 다룹니다.**

```python
# 연도 필터 — 2015년 이후 개봉한 영화에서만 검색
# $gte = "greater than or equal" (이상), SQL의 WHERE year >= 2015와 같은 역할
query = "감동적인 가족 이야기"
results = vector_store.similarity_search_with_score(
    query, k=3, filter={"year": {"$gte": 2015}}
)

print("🔍 [연도 필터] 2015년 이후 영화에서 '감동적인 가족 이야기' 검색:")
for doc, score in results:
    print(f"  🎬 {doc.metadata['title']} ({int(doc.metadata['year'])}) — 유사도: {score:.4f}")
```

**실행 결과 (예시):**
```
🔍 [연도 필터] 2015년 이후 영화에서 '감동적인 가족 이야기' 검색:
  🎬 응답하라 1988 (2015) — 유사도: 0.5123
  🎬 미나리 (2020) — 유사도: 0.4856
  🎬 기생충 (2019) — 유사도: 0.3942
```

> `filter={"year": {"$gte": 2015}}`를 추가하자, 2015년 이전 영화(올드보이, 태극기 휘날리며 등)가 결과에서 제외되었습니다. 이처럼 메타데이터 필터를 사용하면 벡터 유사도 검색의 **범위를 좁힐** 수 있습니다. 평점 필터, 감독 필터, 복합 조건 등 더 다양한 필터 기법은 **14장**에서 상세히 실습합니다.
