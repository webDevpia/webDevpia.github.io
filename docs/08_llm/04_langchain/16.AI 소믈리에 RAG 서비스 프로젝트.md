# 프로젝트 개요서: AI 소믈리에 RAG 서비스

---

<a id="part1"></a>

## 1. 프로젝트 개요 [↑](#toc)

* **프로젝트명:**

  AI 소믈리에 RAG 서비스

* **목적:**

  음식 사진(이미지)와 한 줄 설명을 입력받아,
  AI가 음식의 특징을 분석하고, 대규모 와인 리뷰 데이터베이스(벡터DB, Pinecone)에서 음식과 어울리는 와인 정보를 검색(RAG, Retrieval-Augmented Generation)한 뒤
  LLM(GPT-4.1-mini)로 자연어 추천 결과를 생성해 사용자에게 제공하는 웹 서비스 구축

* **기대효과:**

  * 초개인화 와인 추천 경험 제공
  * LLM+RAG 기술(이미지→설명→벡터검색→생성) 전체 파이프라인 실습
  * 최신 생성형 AI/검색 증강/멀티모달 AI 실전 체험

---

<a id="part2"></a>

## 2. 서비스 시나리오 [↑](#toc)

    1. 사용자는 음식 사진을 업로드하고 한 줄 설명을 입력
    2. AI가 이미지를 분석(vision LLM: GPT-4.1-mini)해 음식의 맛과 특징을 한 문장으로 요약
    3. 음식 설명을 임베딩으로 변환해 Pinecone DB에서 유사한 와인 리뷰를 top-K로 검색
    4. 검색된 리뷰와 음식 정보를 LLM(GPT-4.1-mini)에 전달
    5. LLM이 최적의 와인을 한글로 추천하고 그 이유를 자연어로 설명
    6. UI에 추천 와인, 이유, 검색된 리뷰와 유사도 등 표시

## 6. 핵심 코드 구조

### (1) Streamlit app.py

* 두 개 컬럼(입력/출력, 이미지 미리보기) 구성
* 1단계: 이미지/설명 → 맛 분석(LLM Vision)
* 2단계: 벡터DB에서 유사한 와인 리뷰(top-K, 유사도 점수 포함) 검색
* 3단계: LLM을 통한 와인 상세 추천 결과 생성 및 출력

### (2) sommelier.py

* 환경 변수 로드 및 LLM/임베딩/벡터스토어 초기화
* **describe\_dish\_flavor:**

  * 입력 이미지(base64)와 프롬프트로 음식 맛 설명(LLM Vision)
* **search\_wine, search\_wine\_with\_score:**

  * 음식 설명으로 Pinecone에서 유사한 와인 리뷰 및 유사도 점수 검색
* **recommend\_wine:**

  * 음식 설명과 검색 리뷰를 context로 하여 GPT-4.1-mini에 페어링 추천 프롬프트 작성 및 호출

`pinecone 연결하고 index 생성`

```py
from pinecone import Pinecone, ServerlessSpec

# API 키로 Pinecone 서비스에 접속합니다
pc = Pinecone(api_key=PINECONE_API_KEY)

# 이미 같은 이름의 인덱스(데이터 테이블)가 없을 때만 새로 만듭니다
# 중복 생성을 방지하는 안전 장치입니다
if not pc.has_index(PINECONE_INDEX_NAME):
    pc.create_index(
        name=PINECONE_INDEX_NAME,
        dimension=1536,          # 벡터 크기: text-embedding-3-small 모델의 출력 차원수
        metric=PINECONE_INDEX_METRIC,  # 유사도 계산 방식 (cosine)
        spec=ServerlessSpec(
            region=PINECONE_INDEX_REGION,  # 서버 지역
            cloud=PINECONE_INDEX_CLOUD     # 클라우드 제공사
        )
    )
```

`OpenAI에서 제공하는 tiktoken 패키지를 사용해 토큰 수를 계산`

tiktoken은 OpenAI가 만든 토크나이저(tokenizer) 라이브러리로, 텍스트를 LLM이 실제로 처리할 수 있는 "토큰 단위"로 변환하거나 반대로 토큰을 다시 문자열로 복원할 때 사용합니다.
GPT 모델의 토큰 계산, 비용 계산, 입력 제한 확인 등에 꼭 필요한 도구입니다.

```py
import tiktoken

# text-embedding-3-small 모델이 텍스트를 어떻게 토큰으로 쪼개는지 정의한 규칙(인코딩)을 가져옵니다
encoding = tiktoken.encoding_for_model("text-embedding-3-small")
text = "이 문장이 몇 개의 토큰으로 인코딩되는지 확인해보세요."

# 문장을 토큰 ID 숫자들의 리스트로 변환합니다
tokens = encoding.encode(text)
print('전체 토큰 : ', tokens)
print('토큰 수 : ', len(tokens))  # 토큰 수 출력 (API 비용 계산의 기준 단위)
print('복원 : ', encoding.decode(tokens))  # 토큰 → 원래 문자열로 복원

# 각 토큰이 실제로 어떤 문자열에 해당하는지 보기
# 한글은 글자단위로 토큰처리되지 않기 때문에 한글은 깨질수 있음.
for token in tokens:
    print(token, "→", encoding.decode([token]))

```

`Embedding 모델 로드`

```py
from langchain_openai import OpenAIEmbeddings

# 텍스트(문장)를 숫자 벡터(1536차원)로 변환하는 임베딩 모델을 준비합니다
# 벡터로 변환해야 Pinecone에 저장하고 의미 기반으로 유사한 항목을 검색할 수 있습니다
embeddings = OpenAIEmbeddings(model=OPENAI_EMBEDDING_MODEL)
```

 > **참고: 129,971건 전체 업로드 예상**
> - **시간:** 배치 300건 기준 약 430회 반복, 네트워크 상태에 따라 30분~1시간 소요
> - **비용:** `text-embedding-3-small` 기준 약 $0.02~0.05 (토큰 수에 따라 변동)
> - 처음 실행 시 시간이 오래 걸리므로, 이미 업로드한 인덱스가 있다면 이 단계를 건너뛸 수 있습니다

`129971건의 데이터를 300건씩 처리`
`문서의 길이가 가장 긴게 1115, 한번에 처리할 수 있는 단위(8192 tokens)보다 작으므로 문서를 쪼개지 않고 진행 `

```py
from langchain_pinecone import PineconeVectorStore

# 한 번에 처리할 문서 수를 300개로 제한합니다
# 너무 많은 문서를 한 번에 보내면 API 오류가 발생하므로, 나눠서 처리합니다 (배치 처리)
BATCH_SIZE = 300

# 전체 문서를 300개씩 잘라서 반복 처리합니다
for i in range(0, len(docs), BATCH_SIZE):
    batch = docs[i:i + BATCH_SIZE]  # 현재 처리할 300개 묶음 추출
    try:
        # 문서를 임베딩 벡터로 변환한 뒤 Pinecone에 저장(업서트)합니다
        # 업서트(upsert) = 없으면 삽입(insert), 이미 있으면 업데이트(update)
        PineconeVectorStore.from_documents(
            documents=batch,
            index_name=PINECONE_INDEX_NAME,
            embedding=embeddings
        )

        print(f"Indexed documents {i} to {i + BATCH_SIZE - 1}")
    except Exception as e:
        # 오류가 발생해도 전체 작업이 멈추지 않고 다음 배치로 넘어갑니다
        print(f"Error indexing documents {i} to {i + BATCH_SIZE - 1}: {e}")

```

> **참고:** `PineconeVectorStore.from_documents()`는 호출할 때마다 새 VectorStore 객체를 생성합니다. 대량 데이터 업서트 시에는 한 번 생성한 VectorStore에 `add_documents()`를 반복 호출하는 것이 더 효율적입니다.
> ```python

> for i in range(0, len(docs), BATCH_SIZE):
>     batch = docs[i:i + BATCH_SIZE]
>     vector_store.add_documents(batch)
>     print(f"Indexed documents {i} to {i + len(batch) - 1}")
> ```

```py
from langchain_core.prompts import ChatPromptTemplate, HumanMessagePromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 와인 이미지를 보고 어울리는 음식을 추천해 주는 함수입니다
# query: {"text": 질문 문자열, "image_urls": [base64 이미지 URL 리스트]} 형태로 받습니다
def recommend_dishes(query):
    # 시스템 메시지(AI의 역할 설정)와 사용자 메시지(질문+이미지)를 합쳐 프롬프트를 구성합니다
    prompt = ChatPromptTemplate.from_messages([
        ("system", '''
            Persona:
            소믈리에로서 저는 포도 품종, 산지, 테이스팅 노트, 음식 페어링을 포함한 와인에 대한 방대한 지식을 보유하고 있습니다. 개인의 취향, 특별한 행사, 특정 요리에 맞춰 와인을 추천하는 데 탁월한 능력을 갖추고 있습니다. 와인 생산 방법, 풍미 프로필, 그리고 다양한 음식과의 상호작용을 이해하는 것이 제 전문성입니다. 또한 와인계의 최신 트렌드를 꾸준히 파악하며 전통적이면서도 모험적인 와인을 제안할 수 있습니다. 식사 경험을 한층 풍요롭게 하기 위해 세심하고 맞춤형 추천을 제공하기 위해 노력합니다.

            Role:
            1. 와인 & 음식 페어링: 특정 요리와 조화를 이루며 풍미를 균형 있게 조율하고 전체적인 식사 경험을 향상시키는 상세한 와인 추천을 제공합니다. 간단한 안주든 정성스러운 식사든, 음식의 질감, 맛, 스타일을 보완하는 와인을 제안합니다.

            가벼운 바디감과 붉은 베리의 은은한 풍미로 유명한 피노 누아는 구운 오리나 버섯 리조또와 함께 즐기기에 완벽합니다. 그 흙내음이 요리의 풍미를 더욱 돋보이게 하기 때문입니다.

        ''')
    ])

    template = [{"text": query["text"]}]
    # 이미지가 있을 때만 이미지 항목을 추가합니다 (멀티모달 입력)
    if query["image_urls"]:
        template += [{"image_url": url} for url in query["image_urls"]]

    # 프롬프트 → LLM 호출 → 결과를 문자열로 파싱하는 체인을 만들어 반환합니다
    # StrOutputParser(): LLM 응답 객체에서 텍스트만 꺼내주는 변환기

    return chain
```

```py
import base64

# 로컬 이미지 파일을 base64 형식의 데이터 URL로 변환합니다
# OpenAI Vision API는 이미지를 base64 인코딩된 문자열로 받을 수 있습니다
def image_file_to_base64_url(file_path: str):
    # 파일을 바이너리(0,1 데이터) 모드로 읽습니다
    with open(file_path, "rb") as f:
        image_bytes = f.read()
    # 이미지 바이너리를 base64 문자열로 변환합니다
    base64_str = base64.b64encode(image_bytes).decode("utf-8")
    # API 전송에 적합한 데이터 URL 형식으로 조합해서 반환합니다
    return f"data:image/jpeg;base64,{base64_str}"
```

```py
    # 호출 측에서 RunnableLambda로 감싸 다른 체인과 연결할 수 있게 하기 위해서입니다
    chain = prompt | llm | StrOutputParser()
    return chain
```

```py
    - 와인 우선 : 카베르네 소비뇽 → 그릴에 구운 스테이크나 양고기와 조화
    - 행사별 : 기념일 저녁 → 샴페인이나 피노 누아 추천
    - 취향별 : 강렬한 타닌 선호 → 나파 밸리산 카베르네 소비뇽
'''


def describe_dish_flavor(image_bytes, query):
    """이미지와 질문으로 요리의 풍미를 분석"""
    # 업로드된 이미지 파일의 바이너리 데이터를 base64 문자열로 변환합니다
    # OpenAI Vision API는 이미지를 이 형식으로 받아야 합니다
    base64_str = base64.b64encode(image_bytes).decode('utf-8')
    # API 전송용 데이터 URL로 조합합니다 (브라우저의 <img src="data:..."> 와 같은 형식)
    data_url = f"data:image/jpeg;base64,{base64_str}"

    # 시스템 역할(맛 분석 AI)과 사용자 입력(텍스트 질문 + 이미지)을 합쳐 프롬프트를 구성합니다
    prompt = ChatPromptTemplate.from_messages([
        ("system", DISH_FLAVOR_SYSTEM_PROMPT),
        ("human", [
            {"type": "text", "text": query},                           # 텍스트 질문
            {"type": "image_url", "image_url": {"url": data_url}},    # 이미지 (멀티모달 입력)
        ]),
    ])

    # 프롬프트 → LLM 호출 → 문자열 추출의 체인을 실행하고 결과 문자열을 반환합니다
    chain = prompt | llm | StrOutputParser()
    return chain.invoke({})


def search_wine(dish_flavor):
    """풍미 설명으로 유사한 와인 리뷰 검색 (유사도 점수 포함)"""
    # similarity_search_with_score: 유사도 점수(0~1)도 함께 반환합니다
    # 1에 가까울수록 더 유사한 와인입니다 (cosine 유사도 기준)
    results = vector_store.similarity_search_with_score(dish_flavor, k=2)
    return {
        'dish_flavor': dish_flavor,
        # 각 리뷰 앞에 유사도 점수를 붙여 하나의 문자열로 합칩니다
        # :.4f: 소수점 4자리까지 표시 (예: 0.8432)
        'wine_reviews': "\n\n".join(
            f"유사도 점수 : {score:.4f}\n\n{doc.page_content}"
            for doc, score in results
        ),
    }


def recommend_wine(inputs):
    """요리 풍미와 와인 리뷰를 기반으로 와인 페어링 추천"""
    # 검색된 와인 리뷰(RAG의 검색 결과)를 컨텍스트로 삼아 LLM이 추천 글을 생성합니다
    # {dish_flavor}와 {wine_reviews}는 inputs 딕셔너리의 값으로 채워집니다
    prompt = ChatPromptTemplate.from_messages([
        ("system", SOMMELIER_SYSTEM_PROMPT),
        ("human",
         "와인 페어링 추천에 아래 요리의 풍미와 와인 리뷰를 참고해 한글로 답변해 주세요.\n"
         "두 개의 와인에 대해서 설명을 하되, 딱 한 개의 와인만 선택해 추천 이유를 달아주세요\n\n"
         "요리의 풍미:\n{dish_flavor}\n\n"
         "와인 리뷰:\n{wine_reviews}"),
    ])
    chain = prompt | llm | StrOutputParser()
    # inputs에는 {'dish_flavor': ..., 'wine_reviews': ...} 가 담겨 있습니다
    return chain.invoke(inputs)
```

```py
    # 이미지가 업로드된 경우에만 오른쪽 컬럼에 미리보기를 표시합니다

    if st.button("추천 하기"):

            st.markdown("### AI 소믈리에의 추천")
            st.info(wine_recommendation)
```
