# 벡터 검색 (Vector Search) 실습

> **13장과 이 장의 차이**
>
> | | 13장 (고수준 래퍼) | 14장 (네이티브 API) |
> |--|--|--|
> | **저장** | `PineconeVectorStore.from_documents()` — 임베딩+저장 한 번에 | `embeddings.embed_documents()` → `index.upsert()` — 각 단계 수동 |
> | **검색** | `similarity_search_with_score()` — Document 객체 반환 | `index.query()` — 딕셔너리 반환 |
> | **장점** | 코드가 간결, LangChain 체인에 바로 연결 | 내부 동작 이해, 세밀한 제어 가능 |
>
> 13장에서는 LangChain이 제공하는 **고수준 래퍼**로 임베딩+저장+검색을 한 줄로 처리했습니다. 이번 장에서는 Pinecone의 **네이티브 API**를 직접 사용해 각 단계를 수동으로 제어합니다. 이를 통해 "래퍼 안에서 무슨 일이 일어나는지" 이해할 수 있습니다.

## 🧱 1. 임베딩(Embedding) 생성 [↑](#toc)
텍스트를 고차원 숫자 벡터로 표현하여 의미적 유사도를 비교할 수 있게 만듭니다.

```python
# 검색에 사용할 영화 데이터를 직접 정의합니다.
# 각 영화는 딕셔너리(key-value 쌍의 묶음) 형태로 저장되며,
# 나중에 벡터 검색 결과와 함께 보여줄 정보(제목, 연도, 장르)와
# 실제 검색의 기준이 되는 줄거리(description)를 포함합니다.
# → AI는 "description" 문장의 의미를 숫자로 바꿔 검색에 활용합니다.
movies = [
    {
        "id": "movie1",       # 각 영화를 구분하는 고유 식별자 (데이터베이스의 기본 키 역할)
        "title": "7번방의 선물",
        "year": 2013,
        "genre": "드라마",
        "description": "억울한 누명을 쓰고 교도소에 수감된 아빠와 그의 어린 딸의 감동적인 스토리"
    },
    {
        "id": "movie2",
        "title": "미나리",
        "year": 2020,
        "genre": "드라마",
        "description": "한국계 미국인 가족의 따뜻하고 감성적인 성장 이야기"
    },
    {
        "id": "movie3",
        "title": "기생충",
        "year": 2019,
        "genre": "드라마",
        "description": "가난한 가족과 부자 가족 사이의 빈부격차를 그린 사회 풍자 드라마"
    },
    {
        "id": "movie4",
        "title": "범죄도시",
        "year": 2017,
        "genre": "범죄",
        "description": "형사가 범죄 조직을 소탕하는 범죄 액션 영화"
    },
    {
        "id": "movie5",
        "title": "범죄도시 2",
        "year": 2022,
        "genre": "범죄",
        "description": "형사와 범죄 조직의 대결을 그린 범죄 액션 영화의 속편"
    },
    {
        "id": "movie6",
        "title": "헤어질 결심",
        "year": 2022,
        "genre": "범죄",
        "description": "산에서 발생한 의문의 죽음(살인 사건)을 수사하던 형사가 피의자에게 이끌리며 벌어지는 미스터리 멜로 영화"
    },
    {
        "id": "movie7",
        "title": "다만 악에서 구하소서",
        "year": 2020,
        "genre": "범죄",
        "description": "청부 살인업자와 범죄 조직의 마지막 거래를 그린 범죄 액션 영화"
    }
]
```

```python
from dotenv import load_dotenv
from langchain_openai import OpenAIEmbeddings

# .env 파일에 저장된 API 키(비밀번호)를 읽어옵니다.
# API 키를 코드에 직접 적으면 유출 위험이 있으므로, 별도 파일로 관리합니다.
load_dotenv()

# OpenAI의 임베딩(Embedding) 모델을 준비합니다.
# 임베딩이란 문장의 '의미'를 숫자 배열(벡터)로 표현하는 기술입니다.
# 예: "감동적인 영화" → [0.12, -0.45, 0.87, ...] 처럼 1536개의 숫자로 변환됩니다.
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 모든 영화의 줄거리(description)를 한꺼번에 벡터로 변환합니다.
# [m["description"] for m in movies] 는 movies 리스트에서 줄거리 텍스트만 모아 새 리스트를 만드는 구문입니다.
# 결과: movie_vectors[0]은 첫 번째 영화 줄거리의 벡터, movie_vectors[1]은 두 번째 영화, ...
movie_vectors = embeddings.embed_documents([m["description"] for m in movies])

# 첫 번째 영화 벡터의 길이(차원 수)를 출력합니다.
# 1536이 출력되면 임베딩이 정상적으로 생성된 것입니다.
print(len(movie_vectors[0]))  # 1536차원 출력

```
💡 `text-embedding-3-small` 모델은 1536차원 벡터를 생성합니다.

## 🗃️ 2. Pinecone 인덱스 구성 및 업서트 [↑](#toc)

> ⚠️ **주의:** 아래 코드는 인덱스가 이미 존재하면 삭제 후 새로 생성합니다. 기존 데이터가 모두 삭제되므로, 보존이 필요한 인덱스에는 사용하지 마세요. 이 실습에서는 매번 깨끗한 상태에서 시작하기 위해 이 패턴을 사용합니다.

```python
import os
from pinecone import Pinecone, ServerlessSpec

# Pinecone 클라이언트를 초기화합니다.
# Pinecone은 벡터를 저장하고 검색하는 데이터베이스 서비스입니다.
# API 키는 .env 파일에서 읽어옵니다.
pc = Pinecone(api_key=os.getenv("PINECONE_API_KEY"))

# 생성할 인덱스(벡터를 저장하는 공간)의 이름을 지정합니다.
# 인덱스는 관계형 DB의 '테이블'과 비슷한 개념입니다.
index_name = "movie-vector-index"

# 매번 깨끗한 상태로 실습하기 위해, 같은 이름의 인덱스가 이미 존재하면 삭제합니다.
# 실제 서비스에서는 데이터가 지워지므로 주의가 필요합니다.
if pc.has_index(index_name):
    pc.delete_index(index_name)

# 새 인덱스를 생성합니다.
pc.create_index(
    name=index_name,
    dimension=1536,       # 저장할 벡터의 크기 (OpenAI 임베딩 모델과 반드시 일치해야 합니다)
    metric="cosine",      # 두 벡터 사이의 유사도를 계산하는 방식 (코사인 유사도: 방향이 같을수록 1에 가까움)
    spec=ServerlessSpec(
        cloud="aws",      # 인덱스를 호스팅할 클라우드 서비스
        region="us-east-1"  # 서버가 위치한 지역 (무료 플랜은 us-east-1 지원)
    )
)

# 생성된 인덱스에 접근할 수 있는 객체를 가져옵니다.
# 이후 데이터 저장(upsert)과 검색(query)은 이 index 객체를 통해 수행합니다.
index = pc.Index(index_name)
```

이후 영화 데이터를 임베딩과 함께 업서트합니다.

```python
# Pinecone에 저장할 데이터를 준비합니다.
# 각 항목은 (ID, 벡터, 메타데이터) 형태의 튜플(묶음)입니다.
#   - ID: 각 항목을 구분하는 고유 이름 (예: "movie1")
#   - 벡터: 줄거리를 숫자 배열로 변환한 임베딩 값 → 실제 검색에 사용됩니다
#   - 메타데이터: 검색 결과로 보여줄 부가 정보 (제목, 장르, 연도 등) → 필터링에도 사용됩니다
# zip(movies, movie_vectors)는 두 리스트를 순서대로 짝지어 묶어 줍니다.
vector_data = [
    (m["id"], v, {"title": m["title"], "genre": m["genre"], "year": m["year"], "description": m["description"]})
    for m, v in zip(movies, movie_vectors)
]

# 준비한 벡터 데이터를 Pinecone 인덱스에 저장합니다.
# upsert란 'update + insert'의 합성어로, 없으면 추가하고 있으면 덮어씁니다.
# 즉, 같은 ID로 다시 실행해도 안전하게 최신 데이터로 갱신됩니다.
index.upsert(vectors=vector_data)
```

## 🔍 3. 벡터 검색 [↑](#toc)

쿼리와 의미적으로 유사한 항목을 찾습니다.

```python
# 사용자가 입력할 검색어(쿼리)를 정의합니다.
# 키워드 일치가 아닌 '의미'를 기준으로 검색하기 때문에,
# 데이터에 없는 단어가 포함되어도 의미가 비슷한 영화를 찾을 수 있습니다.
query = "감성적인 드라마 영화 추천해줘"

# 검색어도 영화 데이터와 동일한 방식으로 벡터(숫자 배열)로 변환합니다.
# 이렇게 해야 저장된 영화 벡터들과 수치적으로 비교할 수 있습니다.
query_vector = embeddings.embed_query(query)

# Pinecone에서 코사인 유사도 기반으로 가장 비슷한 벡터를 검색합니다.
#   - vector: 비교 기준이 될 검색어 벡터
#   - top_k: 유사도 상위 몇 개를 반환할지 (여기서는 3개)
#   - include_metadata=True: 저장 시 함께 넣은 제목, 장르 등 부가 정보도 결과에 포함
result = index.query(vector=query_vector, top_k=3, include_metadata=True)

# 검색 결과를 순서대로 꺼내 출력합니다.
# result["matches"]는 유사도 점수 순으로 정렬된 결과 목록입니다.
for match in result["matches"]:
    print(match["metadata"]["title"], int(match["metadata"]["year"]), match["metadata"]["genre"])
```

📊 **예시 결과**
```
기생충 2019 드라마
7번방의 선물 2013 드라마
범죄도시 2017 범죄
```
➡️ 의미적으로 "감성적인 드라마"에 가까운 영화들이 추천됨.

## 🎯 4. 메타데이터 필터 검색 [↑](#toc)

특정 조건(예: 연도, 장르 등)을 기반으로 검색 범위를 제한할 수 있습니다.

### 4-1. 연도 필터 ($gte — 이상)

```python
# 메타데이터 필터 조건을 정의합니다.
# "$gte"는 "greater than or equal" (이상)을 의미하는 Pinecone 필터 연산자입니다.
# 즉, year >= 2020 조건을 만족하는 영화만 검색 대상이 됩니다.
# → SQL의 WHERE year >= 2020 과 동일한 역할입니다.
filter_condition = {"year": {"$gte": 2020}}

# 의미 검색에 사용할 기준 벡터를 생성합니다.
# 여기서는 특정 의미를 강조하지 않고 "영화" 라는 중립적인 단어를 사용하여
# 필터 조건 자체가 결과를 결정하도록 합니다.
query_vector = embeddings.embed_query("영화")

# filter 인자를 추가하면, 해당 조건을 만족하는 영화들 안에서만 유사도 검색을 수행합니다.
# 즉, 2020년 이전 영화는 아무리 벡터가 비슷해도 결과에 포함되지 않습니다.
result = index.query(vector=query_vector, top_k=3, filter=filter_condition, include_metadata=True)

# 결과 출력: 각 결과의 제목, 연도, 장르를 표시
for match in result["matches"]:
    print(match["metadata"]["title"], int(match["metadata"]["year"]), match["metadata"]["genre"])
```

📊 **예시 결과**
```
범죄도시 2 2022 범죄
헤어질 결심 2022 범죄
다만 악에서 구하소서 2020 범죄
```
➡️ 필터 조건 `year >= 2020`을 만족하는 결과만 반환.

### 4-2. 장르 필터 ($eq — 정확히 일치)

```python
# $eq는 "equal"(같음)을 의미합니다.
# 장르가 정확히 "드라마"인 영화만 검색 대상이 됩니다.
query_vector = embeddings.embed_query("가족 이야기")
result = index.query(
    vector=query_vector, top_k=3,
    filter={"genre": {"$eq": "드라마"}},
    include_metadata=True
)

print("🔍 [장르 필터] 드라마 장르에서 '가족 이야기' 검색:")
for match in result["matches"]:
    print(f"  🎬 {match['metadata']['title']} ({int(match['metadata']['year'])}) {match['metadata']['genre']}")
```

📊 **예시 결과**
```
🔍 [장르 필터] 드라마 장르에서 '가족 이야기' 검색:
  🎬 7번방의 선물 2013 드라마
  🎬 기생충 2019 드라마
  🎬 미나리 2020 드라마
```
➡️ 범죄, 로맨스 등 다른 장르는 제외되고, 드라마 장르만 반환.

### 4-3. 제외 필터 ($ne — 같지 않음)

```python
# $ne는 "not equal"(같지 않음)을 의미합니다.
# 장르가 "범죄"가 아닌 영화만 검색 대상이 됩니다.
# → 범죄 영화를 빼고 나머지에서 검색하고 싶을 때 유용합니다.
query_vector = embeddings.embed_query("재미있는 영화")
result = index.query(
    vector=query_vector, top_k=3,
    filter={"genre": {"$ne": "범죄"}},
    include_metadata=True
)

print("🔍 [제외 필터] 범죄 장르를 제외하고 '재미있는 영화' 검색:")
for match in result["matches"]:
    print(f"  🎬 {match['metadata']['title']} ({int(match['metadata']['year'])}) {match['metadata']['genre']}")
```

📊 **예시 결과**
```
🔍 [제외 필터] 범죄 장르를 제외하고 '재미있는 영화' 검색:
  🎬 7번방의 선물 2013 드라마
  🎬 기생충 2019 드라마
  🎬 건축학개론 2012 로맨스
```
➡️ 범죄도시, 헤어질 결심 등 범죄 영화는 모두 제외됨.

### 4-4. 다중 값 필터 ($in — 목록 중 하나)

```python
# $in은 "in list"(목록에 포함)를 의미합니다.
# 장르가 "드라마" 또는 "로맨스"인 영화만 검색합니다.
# → 여러 장르를 한꺼번에 지정하고 싶을 때 사용합니다.
query_vector = embeddings.embed_query("감동적인 영화 추천해줘")
result = index.query(
    vector=query_vector, top_k=3,
    filter={"genre": {"$in": ["드라마", "로맨스"]}},
    include_metadata=True
)

print("🔍 [$in 필터] 드라마 또는 로맨스 장르에서 검색:")
for match in result["matches"]:
    print(f"  🎬 {match['metadata']['title']} ({int(match['metadata']['year'])}) {match['metadata']['genre']}")
```

📊 **예시 결과**
```
🔍 [$in 필터] 드라마 또는 로맨스 장르에서 검색:
  🎬 7번방의 선물 2013 드라마
  🎬 미나리 2020 드라마
  🎬 건축학개론 2012 로맨스
```
➡️ `$in`을 사용하면 **OR 조건**처럼 여러 값 중 하나라도 일치하는 문서를 모두 포함.

### 4-5. 범위 필터 ($gte + $lte 조합 — 사이 값)

```python
# $gte(이상)와 $lte(이하)를 같은 필드에 동시 적용하면
# 특정 범위 안의 값만 검색할 수 있습니다.
# → SQL의 WHERE year BETWEEN 2010 AND 2019 와 같은 역할입니다.
query_vector = embeddings.embed_query("추천 영화")
result = index.query(
    vector=query_vector, top_k=5,
    filter={"year": {"$gte": 2010, "$lte": 2019}},
    include_metadata=True
)

print("🔍 [범위 필터] 2010~2019년 사이 영화 검색:")
for match in result["matches"]:
    print(f"  🎬 {match['metadata']['title']} ({int(match['metadata']['year'])}) {match['metadata']['genre']}")
```

📊 **예시 결과**
```
🔍 [범위 필터] 2010~2019년 사이 영화 검색:
  🎬 기생충 2019 드라마
  🎬 범죄도시 2017 범죄
  🎬 응답하라 1988 2015 드라마
  🎬 7번방의 선물 2013 드라마
  🎬 건축학개론 2012 로맨스
```
➡️ 2003년(올드보이), 2004년(태극기 휘날리며), 2020년 이후 영화는 모두 제외됨.

> **필터 연산자 활용 팁:** 같은 필드에 `$gte`와 `$lte`를 함께 쓰면 **범위 검색**이 됩니다. 다른 필드에 여러 조건을 쓰면 **AND 조건**이 됩니다. 이 두 가지를 조합하면 매우 정교한 검색이 가능합니다.

## ⚡ 5. 하이브리드 검색 (쿼리 + 필터) [↑](#toc)

자연어 쿼리와 구조화된 필터를 동시에 적용합니다.

```python
# 자연어 검색 쿼리를 정의합니다.
# 이 문장은 벡터 유사도 계산에 사용됩니다.
# 단순 키워드 매칭이 아닌, "범죄 액션" 같은 의미와 가까운 영화를 찾아줍니다.
hybrid_query = "2020년 이후의 범죄 영화 보여줘"

# 검색 쿼리를 벡터로 변환합니다.
hybrid_vector = embeddings.embed_query(hybrid_query)

# 하이브리드 필터 조건을 정의합니다.
# 두 조건을 동시에 적용합니다 (AND 조건):
#   - "$eq": "범죄" → 장르가 정확히 "범죄"인 영화만 (equals, 같음)
#   - "$gte": 2020   → 2020년 이후 개봉한 영화만 (greater than or equal, 이상)
# → 이 두 조건을 모두 만족하는 영화들 안에서 벡터 유사도 검색을 수행합니다.
hybrid_filter = {"genre": {"$eq": "범죄"}, "year": {"$gte": 2020}}

# 벡터 유사도 검색(쿼리 의미)과 메타데이터 필터(구조적 조건)를 동시에 적용합니다.
# 이처럼 두 방식을 함께 쓰는 것을 '하이브리드 검색'이라고 부릅니다.
result = index.query(vector=hybrid_vector, top_k=3, filter=hybrid_filter, include_metadata=True)

# 결과 출력: 각 결과의 제목, 연도, 장르를 표시
for match in result["matches"]:
    print(match["metadata"]["title"], int(match["metadata"]["year"]), match["metadata"]["genre"])
```

📊 **예시 결과**
```
다만 악에서 구하소서 2020 범죄
범죄도시 2 2022 범죄
헤어질 결심 2022 범죄
```
➡️ "쿼리 의미 + 필터 조건"을 모두 충족하는 검색 결과를 제공.

---

### 유사도 점수 해석 가이드

VectorDB마다 반환하는 점수의 의미가 다르므로 주의가 필요합니다.

| VectorDB | 점수 종류 | 범위 | 해석 |
|----------|-----------|------|------|
| **Pinecone** (cosine) | 코사인 유사도 | 0 ~ 1 | **높을수록 유사** (1 = 완전 일치) |
| **Pinecone** (dotproduct) | 내적 | -1 ~ 1 | **높을수록 유사** |
| **FAISS** (기본 Flat/HNSW) | L2 거리 | 0 ~ ∞ | **낮을수록 유사** (0 = 완전 일치) |

> 이 실습에서 Pinecone은 `metric="cosine"`으로 생성했으므로, **점수가 높을수록 더 유사한 결과**입니다.
> 반면 12장에서 사용한 FAISS는 L2 거리를 반환하므로, **값이 작을수록 더 유사**합니다.

---

### 메타데이터 필터 연산자 정리

13장에서 `$gte` 연산자를 맛보기로 사용해 보았습니다. Pinecone이 지원하는 전체 필터 연산자를 정리하면 다음과 같습니다.

| 필터 연산자 | 의미 | SQL 대응 | 예시 |
|-------------|------|----------|------|
| `$eq` | 같음 (equal) | `WHERE director = '봉준호'` | `{"director": {"$eq": "봉준호"}}` |
| `$ne` | 같지 않음 (not equal) | `WHERE genre != '범죄'` | `{"genre": {"$ne": "범죄"}}` |
| `$gt` | 초과 (greater than) | `WHERE rating > 8.0` | `{"rating": {"$gt": 8.0}}` |
| `$gte` | 이상 (greater than or equal) | `WHERE year >= 2015` | `{"year": {"$gte": 2015}}` |
| `$lt` | 미만 (less than) | `WHERE year < 2010` | `{"year": {"$lt": 2010}}` |
| `$lte` | 이하 (less than or equal) | `WHERE rating <= 7.5` | `{"rating": {"$lte": 7.5}}` |
| `$in` | 목록 중 하나 | `WHERE genre IN (...)` | `{"genre": {"$in": ["드라마", "범죄"]}}` |

### 복합 필터 예시 ($and 조건)

하이브리드 검색(5장)에서 이미 복합 필터를 사용했습니다. 여러 조건을 동시에 지정하면 **모든 조건을 만족하는 문서만** 검색됩니다 (AND 조건).

```python
# 복합 필터: 2020년 이후 AND 범죄 장르인 영화에서 검색
complex_filter = {
    "genre": {"$eq": "범죄"},
    "year": {"$gte": 2020}
}
query_vector = embeddings.embed_query("스릴 넘치는 영화")
result = index.query(
    vector=query_vector, top_k=3,
    filter=complex_filter, include_metadata=True
)

print("🔍 [복합 필터] 2020년 이후 범죄 영화에서 검색:")
for match in result["matches"]:
    print(f"  🎬 {match['metadata']['title']} ({int(match['metadata']['year'])}) {match['metadata']['genre']}")
```

**실행 결과 (예시):**
```
🔍 [복합 필터] 2020년 이후 범죄 영화에서 검색:
  범죄도시 2 2022 범죄
  헤어질 결심 2022 범죄
  다만 악에서 구하소서 2020 범죄
```

> **핵심:** 필터 딕셔너리에 여러 키를 넣으면 자동으로 AND 조건이 됩니다. "2020년 이후"와 "범죄 장르"를 동시에 만족하는 영화만 결과에 나타납니다.
