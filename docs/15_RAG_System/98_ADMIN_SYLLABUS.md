## 1. 과정 개요

### 1.1 기본 정보

| 항목 | 내용 |
|:---|:---|
| 과정명 (한글) | RAG 시스템 구축 실전 |
| 과정명 (영문) | RAG System in Practice |
| 훈련 분야 | 정보통신 > 정보기술개발 |
| 훈련 종류 | 직업능력개발훈련 (집체훈련) |
| 총 훈련시간 | 35시간 (5일, 1일 평균 7시간) |
| 정원 | 20명 |
| 강의·실습 비율 | 강의 20% (7시간) / 실습 80% (28시간) |
| 사용 도구 | Python 3.11, VSCode, OpenAI/Claude API, LangChain, LangGraph, LangSmith, ChromaDB, Ragas |
| 선수 학습 | AI 챗봇 개발 경험 또는 벡터 DB·임베딩 이해 |
| 환급 유형 | 고용보험 환급과정 (출석 80% 이상 수료) |

### 1.2 훈련 목표

수강생은 본 과정 수료 후 다음을 수행할 수 있다.

1. Naive RAG의 한계를 진단하고, 적절한 기법(하이브리드 검색·Re-Ranking·Self-RAG 등)으로 개선할 수 있다.
2. 복잡한 행정 문서를 문맥 보존 청킹으로 파싱하고, 메타데이터를 설계하여 검색 효율을 높일 수 있다.
3. LangGraph로 상태 관리·조건부 라우팅·자기검증을 갖춘 RAG 에이전트를 설계·구현할 수 있다.
4. Ragas로 RAG의 신뢰성을 정량 평가하고, LangSmith로 프로덕션을 모니터링할 수 있다.
5. 자기 도메인 데이터에 RAG MVP를 적용하고 평가 결과를 발표할 수 있다.

## 5. 평가 계획서

### 5.1 평가 도구 구성

| 평가 영역 | 배점 | 도구 | 신뢰도 확보 |

| 챕터 체크포인트 | 30 | 객관식 3 + 주관식 2 (10챕터 × 3점) | 사전 출제 + 정답 공개 |
| 실습 결과물 | 20 | Jupyter 노트북 (10챕터 × 2점) | 완료 체크 기준 명시 (§§ Ch.X) |
| 종합 프로젝트 | 20 | Rubric 4항목 × 5점 | 강사 + 동료 평가 |
| **합계** | **100** | | |

### 5.2 객관식 문항 풀

각 챕터 핸드북의 평가 체크포인트에 객관식 3문항이 수록되어 있습니다 (총 30문항). 본 풀에서 챕터당 1~2문항을 무작위로 선택해 출제하며, 정답은 채점자만 보유.

**문항 출제 원칙**:
- 단순 암기 ❌ — 사례 기반 적용 문제 위주
- 4지선다, 정답 1개
- 한 챕터의 핵심 개념 1개에 1문항

### 5.3 주관식 문항

각 챕터 평가 체크포인트의 주관식 2문항. **자기 도메인에 적용하는 전이 학습**을 측정.

**채점 기준**:
- 5점 (수): 자기 도메인 사례 + 본 과정 챕터 매핑 + 근거 제시
- 3점 (우): 자기 도메인 사례 또는 챕터 매핑 중 1개
- 1점 (가): 일반론적 답변

### 5.4 종합 프로젝트 Rubric

| 항목 | 5점 (수) | 3점 (우) | 1점 (가) |
|:---|:---|:---|:---|
| 데이터 준비 | 5건+ 다양한 형식, 메타데이터 설계 | 3건, 단일 형식 | 1건 |
| 검색 파이프라인 | Hybrid + Re-Ranking 적용 | Dense만 | Naive |
| 평가 | Ragas 4메트릭 + 진단 분석 | Ragas 일부 | 정성 평가만 |
| LangSmith 운영 | 트레이싱 + 데이터셋 + 알림 | 트레이싱만 | 미적용 |

### 5.5 채점자 간 일치도

종합 프로젝트는 **강사 + TA 2명이 독립 채점**, 평균값 적용. 점수 차이 2점 이상 시 합의 후 결정.

### 5.6 수료 기준

- 출석 80% 이상 (총 35시간 중 28시간 이상)
- 총점 60점 이상

두 기준 모두 충족 시 수료증 발급 + 환급 신청 진행.
