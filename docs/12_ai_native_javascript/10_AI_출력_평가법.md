## 1️⃣ ⭐ **핵심** — AI는 왜 틀리는가? [↑](#toc)

AI(Copilot, ChatGPT 등)는 놀라울 정도로 빠르고 그럴듯한 코드를 만들어냅니다. 문제는 **틀릴 때도 자신감 있게 틀린다**는 것입니다.

> 지식이 엄청나게 많지만 틀려도 아무렇지 않게 말하는 동료를 상상해보세요.
> 그 동료의 말을 검토 없이 모두 믿으면 어떻게 될까요?
> AI도 마찬가지입니다. AI의 코드는 **항상 검증이 필요합니다**.

### AI 코드의 흔한 실수 유형

**1. 존재하지 않는 메서드 만들어냄 (Hallucination)**

```javascript
// AI가 생성한 코드
const result = array.findAndRemove(item); // ← 이런 메서드는 존재하지 않습니다!
```

AI가 그럴듯한 이름의 메서드를 창작해버립니다. `MDN Web Docs`에서 검색해보면 존재하지 않는다는 것을 알 수 있습니다.

**2. 보안 취약점 무시**

```javascript
// AI가 생성한 코드 — 위험!
function showMessage(userInput) {
  document.getElementById('output').innerHTML = userInput;
  // ← 사용자가 <script>alert('해킹!')</script>을 입력하면?
}
```

사용자 입력을 `innerHTML`에 직접 넣으면 XSS(크로스 사이트 스크립팅) 공격에 취약합니다.

**3. 최신 문법과 구버전 문법 혼용**

```javascript
// AI가 생성한 코드 — 일관성이 없음
var name = '홍길동'; // 옛날 방식 var
const greet = function() { ... }; // 구식 함수 표현식
let age = 20; // 현대 방식
const greet2 = () => { ... }; // 현대 방식
```

**4. 에러 처리 누락**

```javascript
// AI가 생성한 코드 — 네트워크 실패 처리가 없음
async function getUser(id) {
  const res = await fetch(`/api/users/${id}`);
  const data = await res.json();
  return data;
  // ← fetch가 실패하면? res.json()이 실패하면? 아무 처리가 없습니다
}
```

**5. off-by-one 에러 (경계값 실수)**

```javascript
// AI가 생성한 코드
const numbers = [10, 20, 30];
for (let i = 0; i <= numbers.length; i++) { // ← <= 가 문제!
  console.log(numbers[i]);
}
// 출력:
// 10
// 20
// 30
// undefined  ← 배열 범위를 초과함
```

## 2️⃣ ⭐ **핵심** — AI 코드 평가 체크리스트 [↑](#toc)

> 강사와 함께 체크리스트를 하나씩 따라해 보세요

AI가 코드를 생성해주면 다음 5가지를 순서대로 점검하세요.

```
□ 실행되는가? (syntax error 없는가?)
□ 예상대로 동작하는가? (edge case 포함)
□ 모르는 메서드/문법이 있는가? → MDN에서 확인
□ 보안 문제는 없는가? (innerHTML에 사용자 입력 직접 삽입 등)
□ 더 간단한 방법이 있는가?
```

### 체크리스트 사용 예시

Copilot이 다음 코드를 생성했다고 가정합니다:

```javascript
function reverseString(str) {
  return str.split('').reverse().join('');
}
```

체크리스트 적용:

```
□ 실행되는가?
  → 직접 실행: reverseString('hello') → 'olleh' ✓

□ 예상대로 동작하는가?
  → reverseString('') → '' ✓ (빈 문자열)
  → reverseString('a') → 'a' ✓ (한 글자)
  → reverseString('가나다') → '다나가' ✓ (한국어)

□ 모르는 메서드/문법이 있는가?
  → split, reverse, join → MDN에서 확인 완료 ✓

□ 보안 문제는 없는가?
  → 문자열 처리 함수로 보안 문제 없음 ✓

□ 더 간단한 방법이 있는가?
  → 이 방법이 표준적이고 읽기 쉬움 ✓
```

결론: 이 코드는 안전하게 사용할 수 있습니다.

## 3️⃣ ⭐ **핵심** — Bug Hunt 실습 1 — 쉬운 버그 5개 [↑](#toc)

> 강사 시연을 보면서 따라하세요

아래 각 코드에는 버그가 1개씩 있습니다. 버그를 찾고, 왜 틀렸는지 설명하고, 수정하세요.

---

**버그 #1 — 비교 연산자 혼동**

```javascript
// Copilot이 생성한 코드
function isAdult(age) {
  if (age = 18) {  // ← 버그가 있습니다
    return '성인입니다';
  }
  return '미성년자입니다';
}

console.log(isAdult(15)); // 예상: '미성년자입니다'
                           // 실제: ???
```

<details>
<summary>힌트 보기</summary>
`=`과 `===`의 차이를 생각해보세요.
</details>

<details>
<summary>정답 보기</summary>

**버그**: `age = 18`은 `age`에 18을 **대입**하는 연산입니다. 대입 연산은 항상 truthy이므로 조건이 항상 참이 됩니다.

**수정**: `age = 18`을 `age >= 18`로 바꿔야 합니다.

```javascript
function isAdult(age) {
  if (age >= 18) {  // 수정됨
    return '성인입니다';
  }
  return '미성년자입니다';
}
```
</details>

---

**버그 #2 — await 누락**

```javascript
// Copilot이 생성한 코드
async function getUserName(id) {
  const res = fetch(`https://jsonplaceholder.typicode.com/users/${id}`); // ← 버그
  const data = await res.json();
  return data.name;
}
```

<details>
<summary>힌트 보기</summary>
`fetch`는 Promise를 반환합니다. Promise에 `.json()`을 바로 호출할 수 있을까요?
</details>

<details>
<summary>정답 보기</summary>

**버그**: `fetch()` 앞에 `await`가 빠졌습니다. `await` 없이 `fetch()`를 호출하면 `res`는 데이터가 아니라 Promise 객체가 됩니다. Promise 객체에는 `.json()` 메서드가 없어서 에러가 납니다.

**수정**:
```javascript
async function getUserName(id) {
  const res = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`); // 수정됨
  const data = await res.json();
  return data.name;
}
```
</details>

---

**버그 #3 — 배열 인덱스 오류 (off-by-one)**

```javascript
// Copilot이 생성한 코드
function printAll(items) {
  for (let i = 0; i <= items.length; i++) { // ← 버그
    console.log(items[i]);
  }
}

printAll(['사과', '바나나', '딸기']);
// 예상: 사과, 바나나, 딸기
// 실제: ???
```

<details>
<summary>힌트 보기</summary>
배열의 마지막 인덱스는 `length - 1`입니다. `length`번째 인덱스는 존재할까요?
</details>

<details>
<summary>정답 보기</summary>

**버그**: `i <= items.length`에서 `<=` 때문에 `i`가 3일 때도 반복합니다. `items[3]`은 존재하지 않으므로 `undefined`가 출력됩니다.

**수정**: `<=`를 `<`로 바꿉니다.

```javascript
function printAll(items) {
  for (let i = 0; i < items.length; i++) { // 수정됨
    console.log(items[i]);
  }
}
```
</details>

---

**버그 #4 — undefined 변수 참조**

```javascript
// Copilot이 생성한 코드
function greet(name) {
  const message = `안녕하세요, ${Name}님!`; // ← 버그
  return message;
}

console.log(greet('홍길동')); // 예상: "안녕하세요, 홍길동님!"
```

<details>
<summary>힌트 보기</summary>
JavaScript는 대소문자를 구별합니다.
</details>

<details>
<summary>정답 보기</summary>

**버그**: 매개변수는 `name`(소문자)인데 템플릿 리터럴 안에서 `Name`(대문자 N)을 참조했습니다. JavaScript는 대소문자를 구별하므로 `Name`은 정의되지 않은 변수입니다.

**수정**:
```javascript
function greet(name) {
  const message = `안녕하세요, ${name}님!`; // 수정됨
  return message;
}
```
</details>

---

**버그 #5 — 이벤트 리스너 중복 등록**

```javascript
// Copilot이 생성한 코드
let count = 0;

function setupButton() {
  const btn = document.querySelector('#countBtn');
  btn.addEventListener('click', () => {
    count++;
    console.log('카운트:', count);
  });
}

// 사용자가 페이지를 여러 번 방문할 때마다 setupButton()이 호출됨
setupButton();
setupButton();
setupButton();

// 버튼을 한 번 클릭하면?
// 예상: 카운트: 1
// 실제: ???
```

<details>
<summary>힌트 보기</summary>
`addEventListener`를 3번 호출하면 핸들러가 몇 개 등록될까요?
</details>

<details>
<summary>정답 보기</summary>

**버그**: `setupButton()`을 3번 호출하면 같은 버튼에 클릭 핸들러가 3개 등록됩니다. 버튼을 한 번 클릭하면 `count`가 1이 아니라 3이 됩니다.

**수정**: 이벤트 리스너는 한 번만 등록해야 합니다. 또는 등록 전에 기존 리스너를 제거해야 합니다.

```javascript
// 방법 1: setupButton을 한 번만 호출
setupButton(); // 딱 한 번만!

// 방법 2: 핸들러를 변수에 저장하고 removeEventListener 사용
const handler = () => {
  count++;
  console.log('카운트:', count);
};

function setupButton() {
  const btn = document.querySelector('#countBtn');
  btn.removeEventListener('click', handler); // 기존 핸들러 제거
  btn.addEventListener('click', handler);    // 새로 등록
}
```
</details>

## 4️⃣ ⭐ **핵심** — Bug Hunt 실습 2 — 중간 난이도 [↑](#toc)

아래 각 코드에는 버그가 2개씩 있습니다. 두 개를 모두 찾으세요.

---

**버그 세트 #1 — fetch 에러 처리 누락**

```javascript
// Copilot이 생성한 코드
async function fetchPost(id) {
  const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
  const post = await res.json(); // 버그 1: 조건 없이 바로 json() 파싱
  return post.title;
}

// 호출하는 쪽
fetchPost(999).then((title) => {
  document.querySelector('#result').textContent = title; // 버그 2
});
```

<details>
<summary>정답 보기</summary>

**버그 1**: `res.ok`를 확인하지 않고 바로 `res.json()`을 호출합니다. 존재하지 않는 ID(999)에 요청하면 서버가 404 에러를 반환하지만, `fetch`는 이것을 에러로 처리하지 않습니다. `res.ok`가 `false`일 때 에러를 던져야 합니다.

**버그 2**: `fetchPost`는 async 함수라서 에러가 발생하면 rejected Promise를 반환합니다. 하지만 `.catch()`가 없어서 에러가 완전히 무시됩니다. 또한 `document.querySelector('#result')`가 `null`이면 `null.textContent`에서 에러가 납니다.

**수정**:
```javascript
async function fetchPost(id) {
  const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
  if (!res.ok) { // 버그 1 수정: HTTP 에러 확인
    throw new Error(`HTTP 에러: ${res.status}`);
  }
  const post = await res.json();
  return post.title;
}

fetchPost(1)
  .then((title) => {
    const el = document.querySelector('#result');
    if (el) { // 버그 2 수정: null 체크
      el.textContent = title;
    }
  })
  .catch((err) => {
    console.error('게시물을 불러오지 못했습니다:', err.message);
  });
```
</details>

---

**버그 세트 #2 — DOM 요소 null 접근**

```javascript
// Copilot이 생성한 코드
// 이 스크립트는 <head>에 포함됩니다
document.querySelector('#submitBtn').addEventListener('click', () => { // 버그 1
  const name = document.querySelector('#nameInput').value;
  const result = document.querySelector('#output');
  result.textContent = `${name}님 환영합니다!`; // 버그 2 가능성
});
```

```html
<!-- HTML 구조 -->
<head>
  <script src="app.js"></script> <!-- 스크립트가 여기 있음 -->
</head>
<body>
  <input id="nameInput" type="text">
  <button id="submitBtn">제출</button>
  <p id="output"></p>
</body>
```

<details>
<summary>정답 보기</summary>

**버그 1**: 스크립트가 `<head>`에 있어서 `<body>`의 HTML이 아직 렌더링되지 않은 상태에서 실행됩니다. `document.querySelector('#submitBtn')`은 `null`을 반환하고, `null.addEventListener`에서 에러가 납니다.

**버그 2**: 마찬가지로 `#output` 요소도 아직 렌더링되지 않았다면 `null`이 될 수 있습니다.

**수정**:
```javascript
// 방법 1: 스크립트를 </body> 직전으로 이동
// 방법 2: DOMContentLoaded 이벤트 사용
document.addEventListener('DOMContentLoaded', () => {
  document.querySelector('#submitBtn').addEventListener('click', () => {
    const name = document.querySelector('#nameInput').value;
    const result = document.querySelector('#output');
    if (result) {
      result.textContent = `${name}님 환영합니다!`;
    }
  });
});
```
</details>

---

**버그 세트 #3 — async 함수에서 return 누락**

```javascript
// Copilot이 생성한 코드
async function getTemperature(city) {
  try {
    const res = await fetch(`https://api.example.com/weather?city=${city}`);
    const data = await res.json();
    const temp = data.temperature; // 버그 1: return이 없음
  } catch (err) {
    console.log('에러 발생'); // 버그 2: 에러 정보 손실
  }
}

const temp = await getTemperature('서울');
console.log(temp); // 예상: 온도값. 실제: ???
```

<details>
<summary>정답 보기</summary>

**버그 1**: `temp` 변수에 값을 저장했지만 `return temp`가 없습니다. async 함수에서 명시적으로 반환하지 않으면 `undefined`가 반환됩니다.

**버그 2**: `catch` 블록에서 `console.log('에러 발생')`만 출력하고 에러 객체를 버립니다. 어떤 에러인지, 어디서 났는지 알 수 없습니다. 또한 에러 발생 시 함수는 `undefined`를 반환하게 됩니다.

**수정**:
```javascript
async function getTemperature(city) {
  try {
    const res = await fetch(`https://api.example.com/weather?city=${city}`);
    const data = await res.json();
    const temp = data.temperature;
    return temp; // 버그 1 수정: return 추가
  } catch (err) {
    console.error('날씨 정보 로드 실패:', err.message); // 버그 2 수정: 에러 내용 포함
    throw err; // 호출자에게 에러를 다시 전달
  }
}
```
</details>

## 5️⃣ 🚀 **도전** — Bug Hunt 실습 3 — 종합 [↑](#toc)

아래는 Copilot이 생성한 **할 일 목록 앱** 전체 코드입니다. 이 코드에는 **4개의 버그**가 숨겨져 있습니다. 모두 찾으세요.

```javascript
// Copilot이 생성한 todo.js
let todos = [];
let nextId = 1;

function addTodo(text) {
  if (text = '') return; // 버그 1
  todos.push({
    id: nextId,
    text: text,
    completed: false
  });
  nextId++;
}

function toggleTodo(id) {
  const todo = todos.find((t) => t.id === id);
  if (todo) {
    todo.completed = !todo.completed;
  }
}

function deleteTodo(id) {
  todos = todos.filter((t) => t.id != id); // 버그 2
}

function getCompletedCount() {
  let count = 0;
  for (let i = 0; i <= todos.length; i++) { // 버그 3
    if (todos[i].completed) {
      count++;
    }
  }
  return count;
}

function renderTodos() {
  const list = document.querySelector('#todoList');
  list.innerHTML = ''; // 목록 초기화

  todos.forEach((todo) => {
    const li = document.createElement('li');
    li.innerHTML = `
      <span>${todo.text}</span>
      <button onclick="deleteTodo(${todo.id})">삭제</button>
    `;
    if (todo.completed) {
      li.style.textDecoration = 'line-through';
    }
    li.addEventListener('click', () => toggleTodo(todo.id));
    list.appendChild(li);
  });

  // 완료 개수 표시
  const counter = document.querySelector('#counter');
  counter.textContent = `완료: ${getCompletedCount()} / ${todos.length}`; // 버그 4
}
```

<details>
<summary>버그 힌트 보기</summary>

- 버그 1: 빈 문자열 체크 방식이 잘못됨
- 버그 2: 타입 비교 방식이 일관성 없음
- 버그 3: 반복문 경계값
- 버그 4: `counter`가 null일 때 어떻게 됩니까?
</details>

<details>
<summary>정답 보기</summary>

**버그 1** (`text = ''`):
대입 연산자 `=`를 비교 연산자 `===`로 잘못 사용했습니다. `text = ''`는 항상 falsy(`''`이므로)이지만 의도한 동작이 아닙니다. 더 심각한 문제는 이게 `text`를 빈 문자열로 바꿔버린다는 것입니다.

```javascript
// 수정:
if (text === '') return;
// 또는 더 안전하게:
if (!text.trim()) return;
```

**버그 2** (`t.id != id`):
`===`/`!==`를 일관되게 써야 합니다. `!=`는 타입 변환을 허용하는 느슨한 비교입니다. `id`가 숫자인데 문자열로 전달되면 `===`에서는 실패하지만 `==`에서는 성공하는 불일치가 생깁니다. 코드 전반에서 `!==`를 사용해야 합니다.

```javascript
// 수정:
todos = todos.filter((t) => t.id !== id);
```

**버그 3** (`i <= todos.length`):
`<=`를 `<`로 바꿔야 합니다. `i`가 `todos.length`와 같을 때 `todos[i]`는 `undefined`이고, `undefined.completed`에서 에러가 납니다.

```javascript
// 수정:
for (let i = 0; i < todos.length; i++) {
```

더 나은 방법은 `filter`를 쓰는 것입니다:
```javascript
function getCompletedCount() {
  return todos.filter((t) => t.completed).length;
}
```

**버그 4** (`counter.textContent`):
`document.querySelector('#counter')`가 `null`을 반환할 수 있습니다. HTML에 `#counter` 요소가 없거나, DOM이 아직 로드되지 않은 상태라면 `null.textContent`에서 에러가 납니다.

```javascript
// 수정:
const counter = document.querySelector('#counter');
if (counter) {
  counter.textContent = `완료: ${getCompletedCount()} / ${todos.length}`;
}
```
</details>

## 6️⃣ 📖 **더 알아보기** — AI에게 디버깅 도움 요청하기 [↑](#toc)

버그를 직접 찾기 어려울 때 AI에게 도움을 요청할 수 있습니다. 하지만 **어떻게 질문하느냐**에 따라 답변의 품질이 크게 달라집니다.

### 나쁜 질문 vs 좋은 질문

**나쁜 질문 예시**:
```
이 코드에서 버그를 찾아줘.
```

AI가 무엇을 찾아야 할지 맥락이 없습니다. 어떤 동작을 기대했는지, 실제로 무슨 일이 일어났는지 모릅니다.

---

**좋은 질문 예시**:
```
이 코드가 버튼 클릭 시 카운터를 1씩 올려야 하는데,
버튼을 3번 클릭하면 3이 아니라 9가 됩니다.
왜 이런 문제가 생기는지 설명해줘.

[코드 붙여넣기]
```

좋은 디버깅 질문의 3요소:
1. **기대 동작**: "X를 해야 한다"
2. **실제 동작**: "Y가 일어난다"
3. **관련 코드**: 전체 코드 또는 관련 부분

---

### 좋은 질문 더 보기

```
// 질문 1 — 에러 메시지 포함
이 코드를 실행하면 "Cannot read properties of null (reading 'textContent')"
에러가 납니다. 어디서 null이 생기는지 찾아줘.

// 질문 2 — 환경 정보 포함
Node.js 20, Vitest 2.x 환경에서 이 async 함수 테스트가 실패합니다.
테스트 코드와 구현 코드를 같이 올릴게요.

// 질문 3 — 이미 시도한 것 포함
fetch 에러를 try/catch로 잡으려 했는데 여전히 콘솔에 Uncaught Error가 뜹니다.
await를 붙였고 try/catch 안에 있는데 왜 잡히지 않을까요?
```

### AI 디버깅 답변 검증하기

AI가 "이 줄이 문제입니다"라고 알려줘도, 그 답변을 바로 믿지 마세요. 직접 확인하는 습관을 기르세요.

```
1. AI의 설명을 이해하려고 노력한다
2. AI가 제안한 수정을 적용하기 전에 왜 그 수정이 맞는지 스스로 설명해 본다
3. 수정 후 테스트로 확인한다
4. 여전히 모르겠으면 MDN이나 공식 문서에서 관련 내용을 찾아본다
```

## 7️⃣ 실습 과제 [↑](#toc)

### 기본 과제 — AI 생성 코드에서 버그 3개 찾기

아래 코드는 "사용자 이름을 입력받아 목록에 추가하고, 중복을 허용하지 않는 기능"을 Copilot에게 요청해서 받은 코드입니다. 이 코드에는 버그가 3개 있습니다. 찾고 수정하세요.

```javascript
// Copilot이 생성한 코드
const users = [];

function addUser(name) {
  const trimmed = name.trim;  // 버그 1

  if (trimmed == '') return false; // 버그 2

  const isDuplicate = users.some((u) => u === trimmed);
  if (isDuplicate) return false;

  users.push(trimmed);
  return true;
}

function renderUsers() {
  const list = document.querySelector('#userList');
  list.innerHTML = '';

  for (let i = 0; i <= users.length; i++) { // 버그 3
    const li = document.createElement('li');
    li.textContent = users[i];
    list.appendChild(li);
  }
}
```

### 도전 과제 — 직접 버그 찾기 보고서

1. Copilot에게 다음을 요청하세요:

   ```
   JavaScript로 간단한 계산기를 만들어줘.
   두 숫자를 입력받아 더하기, 빼기, 곱하기, 나누기를 할 수 있어야 해.
   각 연산은 별도의 함수로 구현해줘.
   ```

2. 생성된 코드에 AI 코드 평가 체크리스트를 적용하세요.

3. 버그나 문제점을 발견하면 다음 형식으로 보고서를 작성하세요:

   ```
   발견한 버그/문제점:
   1. [위치]: [줄 번호 또는 함수명]
      [문제]: 무엇이 잘못됐는가
      [수정]: 어떻게 고쳐야 하는가

   2. ...
   ```
