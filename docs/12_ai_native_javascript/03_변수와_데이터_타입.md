## 1️⃣ 변수란? — 이름표가 붙은 상자 [↑](#toc)

### 상자 비유로 시작하기

프로그래밍에서 **변수(variable)**란 값을 담아두는 공간입니다.  
이것을 이해하는 가장 좋은 방법은 **이름표가 붙은 상자**로 생각하는 것입니다.

```
[ name: "김철수" ]   ← 상자 이름이 "name", 안에 "김철수"가 들어 있음
[ age: 25 ]          ← 상자 이름이 "age", 안에 25가 들어 있음
[ isStudent: true ]  ← 상자 이름이 "isStudent", 안에 true가 들어 있음
```

택배를 보낼 때 상자에 이름을 붙이듯, 변수에도 이름을 붙입니다.  
나중에 그 값이 필요할 때 이름만 적으면 바로 꺼낼 수 있습니다.

---

### let — 내용물을 교체할 수 있는 상자

`let`으로 선언한 변수는 나중에 **값을 바꿀 수 있습니다**.

```javascript
let score = 80;
console.log(score); // 출력: 80

score = 95;         // 값을 바꿈
console.log(score); // 출력: 95
```

마치 지우고 다시 쓸 수 있는 화이트보드 같습니다.  
오늘의 점수가 80점이었다가, 재시험을 봐서 95점이 되면 숫자를 고칠 수 있죠.

---

### const — 밀봉된 상자

`const`로 선언한 변수는 한 번 값을 넣으면 **바꿀 수 없습니다**.

```javascript
const PI = 3.14159;
console.log(PI); // 출력: 3.14159

PI = 3;          // 에러 발생!
// TypeError: Assignment to constant variable.
```

생년월일이나 주민등록번호처럼, 절대 변하지 않는 값에는 `const`를 씁니다.  
실수로 값을 바꾸는 것을 JavaScript가 막아줍니다.

---

### 언제 let을, 언제 const를?

실무 원칙: **일단 `const`로 시작하고, 값을 바꿔야 할 때만 `let`으로 바꾸세요.**

```javascript
// 이름은 변하지 않으니까 const
const userName = "이지은";

// 레벨은 플레이하면서 오르니까 let
let level = 1;
level = 2;
level = 3;

// 생년월일은 변하지 않으니까 const
const birthYear = 2000;
```

---

### var는 왜 쓰지 않나요?

`var`는 JavaScript의 오래된 변수 선언 방식입니다.  
현재는 **사용하지 않는 것이 권장됩니다**.

이유: `var`는 예상치 못한 방식으로 동작해서 버그를 만들기 쉽습니다.  
(나중에 스코프 챕터에서 자세히 배웁니다)  

간단한 규칙: `var`를 본다면 "오래된 코드구나"라고 생각하세요.  
새로 쓸 때는 `let`과 `const`만 사용합니다.

---

### 변수 이름 짓는 규칙

```javascript
// 좋은 이름 — 의미가 명확
const userName = "홍길동";
let studentScore = 95;
const isLoggedIn = true;

// 나쁜 이름 — 의미를 알 수 없음
const x = "홍길동";
let a = 95;
const flag = true;
```

규칙:
- 영문자, 숫자, `_`, `$` 사용 가능
- 숫자로 시작 불가 (`2name` X, `name2` O)
- camelCase 사용 권장: `userName`, `studentScore`, `isLoggedIn`
- 한국어도 기술적으로 가능하지만 실무에서는 영문을 씁니다

## 2️⃣ 데이터 타입 — 상자 안에 무엇이 들어 있나 [↑](#toc)

### 타입이 중요한 이유

상자 안에 무엇이 들어있냐에 따라 다르게 취급됩니다.  
"1" + 1의 결과가 2일까요, "11"일까요? 타입에 따라 달라집니다.

JavaScript의 기본 데이터 타입 세 가지를 먼저 배웁니다.

---

### 1. string — 글자 (문자열)

따옴표 안에 들어있는 모든 것은 문자열입니다.

```javascript
const name = "김민준";          // 큰따옴표
const city = '서울';             // 작은따옴표 (동일)
const greeting = `안녕하세요`;   // 백틱 (특별한 기능 있음, 아래에서 설명)

console.log(typeof name);      // 출력: string
console.log(typeof city);      // 출력: string
```

숫자처럼 생겨도 따옴표 안에 있으면 문자열입니다.

```javascript
const code = "01012345678";  // 전화번호 — 따옴표 안이므로 string
const num = 42;              // 따옴표 없으므로 number

console.log(typeof code); // 출력: string
console.log(typeof num);  // 출력: number
```

---

### 2. number — 숫자

따옴표 없이 쓴 숫자는 모두 number 타입입니다.

```javascript
const age = 25;
const price = 9900;
const temperature = -5;
const pi = 3.14;

console.log(typeof age);         // 출력: number
console.log(typeof temperature); // 출력: number
console.log(typeof pi);          // 출력: number
```

---

### 3. boolean — 참(true) / 거짓(false)

오직 두 가지 값만 가집니다: `true` 또는 `false`.

```javascript
const isAdult = true;
const hasTicket = false;

console.log(typeof isAdult);   // 출력: boolean
console.log(typeof hasTicket); // 출력: boolean
```

스위치 같은 것입니다. 켜져 있거나(true), 꺼져 있거나(false).  
"로그인 여부", "학생 여부", "구매 완료 여부" 같은 상태에 사용합니다.

---

### typeof 연산자 — 타입 확인하기

`typeof`를 변수 앞에 붙이면 해당 변수의 타입을 문자열로 돌려줍니다.

```javascript
const a = "안녕";
const b = 42;
const c = true;
const d = 3.14;

console.log(typeof a); // 출력: string
console.log(typeof b); // 출력: number
console.log(typeof c); // 출력: boolean
console.log(typeof d); // 출력: number  ← 소수도 number입니다
```

---

### null과 undefined — 값이 없는 두 가지 방법

헷갈리지만 중요한 개념입니다.

```javascript
let score;                      // 아직 값을 넣지 않음
console.log(score);             // 출력: undefined
console.log(typeof score);      // 출력: undefined

const selectedItem = null;      // 의도적으로 "없음"을 표현
console.log(selectedItem);      // 출력: null
console.log(typeof selectedItem); // 출력: object  ← JavaScript의 유명한 버그
```

- **undefined**: "아직 값을 넣지 않은 상태" — JavaScript가 자동으로 부여
- **null**: "값이 없음을 의도적으로 표현" — 개발자가 직접 씀

일상 비유:
- `undefined`: 수납장을 샀는데 아직 아무것도 안 넣은 상태
- `null`: 수납장을 비웠다는 것을 메모지에 "비어있음"이라고 써둔 상태

지금 단계에서는 "둘 다 없다는 표현이구나" 정도만 기억하세요.

## 3️⃣ 문자열 다루기 — 글자를 조작하는 법 [↑](#toc)

### 템플릿 리터럴 — 백틱의 마법

백틱(`` ` ``)을 사용하면 변수를 문자열 안에 직접 끼워 넣을 수 있습니다.

```javascript
const name = "이서연";
const age = 23;

// 예전 방법 — 연결 연산자 +
const msg1 = "안녕하세요, " + name + "님! 나이는 " + age + "살이군요.";

// 템플릿 리터럴 — 훨씬 읽기 쉬움
const msg2 = `안녕하세요, ${name}님! 나이는 ${age}살이군요.`;

console.log(msg1); // 출력: 안녕하세요, 이서연님! 나이는 23살이군요.
console.log(msg2); // 출력: 안녕하세요, 이서연님! 나이는 23살이군요.
```

`${ }` 안에는 변수뿐만 아니라 계산식도 쓸 수 있습니다.

```javascript
const price = 9900;
const quantity = 3;

console.log(`총 금액: ${price * quantity}원`);
// 출력: 총 금액: 29700원
```

---

### 문자열 연결

```javascript
const firstName = "김";
const lastName = "철수";

const fullName = firstName + lastName;
console.log(fullName); // 출력: 김철수

const sentence = "오늘은 " + "날씨가 " + "좋네요.";
console.log(sentence); // 출력: 오늘은 날씨가 좋네요.
```

### 문자열의 길이 — .length

```javascript
const word = "JavaScript";
console.log(word.length); // 출력: 10

const korean = "안녕하세요";
console.log(korean.length); // 출력: 5

const password = "mypassword123";
if (password.length < 8) {
  console.log("비밀번호가 너무 짧습니다.");
} else {
  console.log("비밀번호 길이가 적절합니다.");
}
// 출력: 비밀번호 길이가 적절합니다.
```

---

### 자주 쓰는 문자열 메서드

```javascript
const text = "Hello, World!";

// 대문자로
console.log(text.toUpperCase()); // 출력: HELLO, WORLD!

// 소문자로
console.log(text.toLowerCase()); // 출력: hello, world!

// 특정 문자 포함 여부
console.log(text.includes("World")); // 출력: true
console.log(text.includes("world")); // 출력: false  ← 대소문자 구분!

// 특정 위치의 문자
console.log(text[0]); // 출력: H
console.log(text[7]); // 출력: W

// 잘라내기
console.log(text.slice(0, 5)); // 출력: Hello
console.log(text.slice(7));    // 출력: World!

// 앞뒤 공백 제거 (사용자 입력 처리할 때 자주 씀)
const userInput = "   홍길동   ";
console.log(userInput.trim()); // 출력: 홍길동
```

## 4️⃣ 숫자 다루기 — 계산과 수학 함수 [↑](#toc)

### 사칙연산

```javascript
console.log(10 + 3);  // 출력: 13  (더하기)
console.log(10 - 3);  // 출력: 7   (빼기)
console.log(10 * 3);  // 출력: 30  (곱하기)
console.log(10 / 3);  // 출력: 3.3333333333333335  (나누기)
console.log(10 % 3);  // 출력: 1   (나머지)
console.log(2 ** 10); // 출력: 1024 (거듭제곱, 2의 10승)
```

**나머지 연산자 `%`** 는 특히 유용합니다.

```javascript
// 짝수/홀수 판별
const number = 7;
if (number % 2 === 0) {
  console.log("짝수");
} else {
  console.log("홀수");
}
// 출력: 홀수

// 3의 배수 찾기
console.log(9 % 3 === 0);  // 출력: true
console.log(10 % 3 === 0); // 출력: false
```

---

### Math 객체 — 수학 계산 도구모음

```javascript
// 반올림
console.log(Math.round(4.7)); // 출력: 5
console.log(Math.round(4.3)); // 출력: 4

// 올림 / 내림
console.log(Math.ceil(4.1));  // 출력: 5  (무조건 올림)
console.log(Math.floor(4.9)); // 출력: 4  (무조건 내림)

// 최대값 / 최소값
console.log(Math.max(3, 7, 2, 9, 1)); // 출력: 9
console.log(Math.min(3, 7, 2, 9, 1)); // 출력: 1

// 절대값
console.log(Math.abs(-5)); // 출력: 5
console.log(Math.abs(5));  // 출력: 5

// 제곱근
console.log(Math.sqrt(16)); // 출력: 4
console.log(Math.sqrt(2));  // 출력: 1.4142135623730951
```

---

### Math.random() — 랜덤 숫자

```javascript
// 0 이상 1 미만의 소수
console.log(Math.random()); // 출력: 0.7391749... (매번 다름)

// 1 ~ 6 사이의 정수 (주사위)
const dice = Math.floor(Math.random() * 6) + 1;
console.log(dice); // 출력: 1~6 중 하나

// 1 ~ 100 사이의 정수
const randomScore = Math.floor(Math.random() * 100) + 1;
console.log(randomScore); // 출력: 1~100 중 하나
```

### 문자열을 숫자로 — parseInt(), parseFloat(), Number()

사용자가 입력한 값은 항상 문자열입니다.  
계산하려면 숫자로 변환해야 합니다.

```javascript
const input = "42";           // 사용자가 입력한 텍스트
console.log(typeof input);    // 출력: string

const num = parseInt(input);  // 정수로 변환
console.log(typeof num);      // 출력: number
console.log(num + 1);         // 출력: 43  (문자열이었으면 "421"이 됐을 것)

const price = parseFloat("9900.5"); // 소수 포함 변환
console.log(price);           // 출력: 9900.5

// 숫자가 아닌 문자열은 NaN (Not a Number)
console.log(parseInt("hello")); // 출력: NaN
console.log(isNaN("hello"));    // 출력: true
```

---

### NaN이란?

NaN은 "Not a Number(숫자가 아님)"의 줄임말입니다.  
숫자 연산이 불가능할 때 JavaScript가 돌려주는 특수한 값입니다.

```javascript
console.log(0 / 0);          // 출력: NaN
console.log(parseInt("abc")); // 출력: NaN
console.log("hello" * 2);    // 출력: NaN

// NaN인지 확인할 때
const result = parseInt("abc");
console.log(isNaN(result));   // 출력: true  ← 이렇게 확인
console.log(result === NaN);  // 출력: false ← 이렇게 하면 안 됨! NaN은 자기 자신과도 같지 않음
```

### 도전 과제 — BMI 계산기

파일 `bmi.js`를 만들고 BMI를 계산하는 코드를 작성하세요.

BMI 공식: `체중(kg) ÷ (키(m) × 키(m))`

조건:
1. `const`로 이름, 키(cm), 체중(kg)을 저장
2. 키를 cm에서 m로 변환 (힌트: 100으로 나누기)
3. BMI를 계산하고 `Math.round()`로 소수점 첫째 자리까지 표시  
   (힌트: `Math.round(bmi * 10) / 10`)
4. 결과를 템플릿 리터럴로 출력

예상 출력:
```
이름: 홍길동
키: 175cm, 체중: 70kg
BMI: 22.9
```

도전을 더 하고 싶다면: BMI 기준으로 "저체중/정상/과체중/비만"도 출력해보세요.
(기준: 18.5 미만 저체중, 25 미만 정상, 30 미만 과체중, 30 이상 비만)
