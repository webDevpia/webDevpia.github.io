## ⭐ 핵심: 1️⃣ 함수란? — 레시피 비유

### 레시피를 생각해보세요

계란 프라이 레시피가 있다고 합시다.

```
[계란 프라이 레시피]
1. 프라이팬에 기름을 두른다
2. 계란을 깨서 넣는다
3. 소금을 뿌린다
4. 익으면 완성
```

이 레시피를 **한 번만 작성해두면** 아침마다 다시 생각할 필요 없이 실행할 수 있습니다.

함수도 마찬가지입니다.  
**자주 사용하는 코드 덩어리에 이름을 붙여서 언제든 "호출"할 수 있게 만드는 것**입니다.

---

### 함수 없이 쓴다면

```javascript
// 함수 없이 — 중복 코드
const score1 = 85;
const grade1 = score1 >= 90 ? "A" : score1 >= 80 ? "B" : score1 >= 70 ? "C" : "F";
console.log(`김민준: ${grade1}`);

const score2 = 92;
const grade2 = score2 >= 90 ? "A" : score2 >= 80 ? "B" : score2 >= 70 ? "C" : "F";
console.log(`이서연: ${grade2}`);

const score3 = 67;
const grade3 = score3 >= 90 ? "A" : score3 >= 80 ? "B" : score3 >= 70 ? "C" : "F";
console.log(`박지훈: ${grade3}`);
```

같은 로직이 3번 반복됩니다. 만약 기준이 바뀌면 3곳을 모두 수정해야 합니다.

---

### 함수로 묶으면

```javascript
// 함수 선언
function getGrade(score) {
  if (score >= 90) return "A";
  if (score >= 80) return "B";
  if (score >= 70) return "C";
  return "F";
}

// 함수 호출 — 훨씬 깔끔
console.log(`김민준: ${getGrade(85)}`);  // 출력: 김민준: B
console.log(`이서연: ${getGrade(92)}`);  // 출력: 이서연: A
console.log(`박지훈: ${getGrade(67)}`);  // 출력: 박지훈: F
```

기준이 바뀌면 함수 안만 고치면 됩니다. **한 번 작성, 여러 번 사용.**

---

### 함수 선언 기본 문법

```javascript
function 함수이름(매개변수1, 매개변수2) {
  // 함수 본체
  return 반환값;
}
```

함수를 **선언**하는 것만으로는 실행되지 않습니다.  
**호출** (`함수이름()`)해야 실행됩니다.

```javascript
// 선언 — 레시피를 적어두는 것
function sayHello() {
  console.log("안녕하세요!");
}

// 이 시점에서는 아무것도 출력되지 않음

// 호출 — 레시피를 실행하는 것
sayHello(); // 출력: 안녕하세요!
sayHello(); // 출력: 안녕하세요!
sayHello(); // 출력: 안녕하세요!
```

## ⭐ 핵심: 2️⃣ 매개변수와 반환값 — 재료와 완성품

> 💡 **실습 흐름**: 강사 시연 보기 → 함께 따라하기 → 혼자 변형해 보기

### 매개변수 — 함수에 넣어주는 재료

함수 밖에서 데이터를 받아서 사용할 때 **매개변수(parameter)**를 씁니다.

```javascript
function greet(name) {
  console.log(`안녕하세요, ${name}님!`);
}

greet("홍길동"); // 출력: 안녕하세요, 홍길동님!
greet("이순신"); // 출력: 안녕하세요, 이순신님!
greet("세종대왕"); // 출력: 안녕하세요, 세종대왕님!
```

`name`은 매개변수입니다. 호출할 때 넣어준 값(`"홍길동"`)이 `name`에 담깁니다.

---

### 여러 개의 매개변수

```javascript
function add(a, b) {
  console.log(a + b);
}

add(3, 5);  // 출력: 8
add(10, 20); // 출력: 30
add(0.1, 0.2); // 출력: 0.30000000000000004  ← 부동소수점 문제
```

---

### 반환값 — 함수가 결과물을 돌려줌

`return`으로 값을 돌려줄 수 있습니다.

```javascript
function multiply(a, b) {
  return a * b; // 계산 결과를 반환
}

const result = multiply(4, 5);
console.log(result); // 출력: 20

// 반환값을 바로 사용
console.log(multiply(3, 7) + multiply(2, 8)); // 출력: 37
```

`return`이 없는 함수는 `undefined`를 반환합니다.

```javascript
function printHello() {
  console.log("안녕");
}

const val = printHello(); // 출력: 안녕
console.log(val);          // 출력: undefined
```

---

### return은 함수를 즉시 종료합니다

```javascript
function checkAge(age) {
  if (age < 0) {
    return "올바르지 않은 나이입니다";  // 여기서 함수 종료
  }
  if (age < 18) {
    return "미성년자";  // 여기서 함수 종료
  }
  return "성인";  // 위 조건 모두 해당 없을 때
}

console.log(checkAge(-1)); // 출력: 올바르지 않은 나이입니다
console.log(checkAge(15)); // 출력: 미성년자
console.log(checkAge(20)); // 출력: 성인
```

---

### 기본값 매개변수

매개변수를 전달하지 않았을 때의 기본값을 설정할 수 있습니다.

```javascript
function greet(name = "방문자") {
  return `안녕하세요, ${name}님!`;
}

console.log(greet("김철수")); // 출력: 안녕하세요, 김철수님!
console.log(greet());         // 출력: 안녕하세요, 방문자님!
```

## 🚀 도전: 3️⃣ 화살표 함수 — 더 짧게 쓰기

> 여유가 있다면 도전해 보세요! 수업 후에 천천히 익혀도 좋습니다.

### 화살표 함수란?

ES6(2015년)에 도입된 짧은 함수 작성법입니다.  
`function` 키워드 대신 `=>`(화살표)를 씁니다.

```javascript
// 일반 함수
function add(a, b) {
  return a + b;
}

// 화살표 함수 — 동일한 동작
const add = (a, b) => {
  return a + b;
};

// 더 짧게 — 본체가 return 하나뿐이면
const add = (a, b) => a + b;

console.log(add(3, 4)); // 출력: 7
```

---

### 화살표 함수 단계별 줄이기

```javascript
// 1단계 — 일반 함수
function square(n) {
  return n * n;
}

// 2단계 — 화살표 함수 (중괄호와 return 있음)
const square = (n) => {
  return n * n;
};

// 3단계 — 매개변수가 1개면 괄호 생략 가능
const square = n => {
  return n * n;
};

// 4단계 — 본체가 return 하나뿐이면 중괄호와 return 생략
const square = n => n * n;

console.log(square(5)); // 출력: 25
```

---

### 실무에서 많이 보이는 패턴

```javascript
const double = n => n * 2;
const greet = name => `안녕, ${name}!`;
const isEven = n => n % 2 === 0;
const getFullName = (first, last) => `${last}${first}`;

console.log(double(5));           // 출력: 10
console.log(greet("민준"));       // 출력: 안녕, 민준!
console.log(isEven(4));           // 출력: true
console.log(getFullName("길동", "홍")); // 출력: 홍길동
```

## 📖 더 알아보기: 4️⃣ 함수는 값이다 — 콜백 맛보기

> 수업 후에 천천히 읽어보세요. Phase 2에서 콜백 개념이 다시 등장합니다.

### 함수를 변수에 저장할 수 있습니다

JavaScript에서 함수는 **값**입니다. 숫자나 문자열처럼 변수에 담고 전달할 수 있습니다.

```javascript
// 함수를 변수에 저장
const sayHi = function() {
  console.log("안녕!");
};

sayHi(); // 출력: 안녕!

// 화살표 함수도 동일
const sayBye = () => {
  console.log("잘 가!");
};

sayBye(); // 출력: 잘 가!
```

---

### 콜백 함수 — 함수를 함수에 전달하기

함수를 다른 함수의 **인자(argument)**로 전달할 수 있습니다.  
이렇게 전달된 함수를 **콜백(callback) 함수**라고 합니다.

```javascript
function doTwice(fn) {
  fn(); // 전달받은 함수를 실행
  fn(); // 한 번 더 실행
}

doTwice(() => console.log("실행!"));
// 출력:
// 실행!
// 실행!
```

---

### forEach — 배열을 순회하는 콜백의 첫 경험

배열의 `.forEach()` 메서드는 각 원소마다 함수를 실행합니다.

```javascript
const fruits = ["사과", "바나나", "딸기"];

// for...of 방식
for (const fruit of fruits) {
  console.log(fruit);
}

// forEach 방식 — 화살표 함수 콜백
fruits.forEach(fruit => {
  console.log(fruit);
});

// 더 짧게
fruits.forEach(fruit => console.log(fruit));

// 출력 (모두 동일):
// 사과
// 바나나
// 딸기
```

인덱스도 같이 받을 수 있습니다.

```javascript
const scores = [85, 92, 78];

scores.forEach((score, index) => {
  console.log(`${index + 1}번: ${score}점`);
});
// 출력:
// 1번: 85점
// 2번: 92점
// 3번: 78점
```

지금은 "배열.forEach(화살표 함수)"를 패턴으로 기억하세요.  
콜백의 전체 개념은 Phase 2에서 깊이 다룹니다.

## 📖 더 알아보기: 5️⃣ 스코프 — 함수 안의 비밀

> 수업 후에 천천히 읽어보세요. 코드가 복잡해질수록 중요해지는 개념입니다.

### 스코프란?

**스코프(scope)**는 변수가 "보이는 범위"입니다.

일상 비유: 회사 내부 서류는 직원만 볼 수 있고, 외부인은 볼 수 없습니다.  
함수 안에서 만든 변수도 함수 밖에서는 보이지 않습니다.

---

### 함수 스코프 예제

```javascript
function makeSecret() {
  const secret = "비밀이야"; // 함수 안에서만 존재
  console.log(secret);       // 출력: 비밀이야
}

makeSecret();
console.log(secret); // 에러! ReferenceError: secret is not defined
```

함수 바깥에서 `secret`에 접근하려 하면 에러가 납니다.  
`secret`은 `makeSecret()` 함수 안에서만 존재합니다.

---

### 전역 vs 지역

```javascript
const globalVar = "전역 변수"; // 어디서든 접근 가능

function myFunction() {
  const localVar = "지역 변수"; // 이 함수 안에서만 접근 가능
  console.log(globalVar); // 출력: 전역 변수  ← 전역 변수는 함수 안에서도 보임
  console.log(localVar);  // 출력: 지역 변수
}

myFunction();
console.log(globalVar); // 출력: 전역 변수
console.log(localVar);  // 에러! 지역 변수는 함수 밖에서 안 보임
```

---

### 블록 스코프 — let과 const

`let`과 `const`는 `{}` 블록 안에서만 유효합니다.

```javascript
if (true) {
  const blockVar = "블록 안에서 선언";
  console.log(blockVar); // 출력: 블록 안에서 선언
}

console.log(blockVar); // 에러! 블록 밖에서 접근 불가
```

```javascript
for (let i = 0; i < 3; i++) {
  // i는 이 for 블록 안에서만 유효
  console.log(i);
}

console.log(i); // 에러! for 블록 밖에서 i 접근 불가
```

---

### 왜 스코프가 중요한가?

1. **이름 충돌 방지**: 함수마다 같은 이름의 변수를 써도 서로 간섭 없음
2. **코드 안전성**: 의도치 않게 변수를 수정하는 것을 방지
3. **메모리 관리**: 함수가 끝나면 지역 변수는 사라짐

```javascript
function calculateScore() {
  const bonus = 10;      // 이 함수만의 bonus
  return 80 + bonus;
}

function calculatePrice() {
  const bonus = 5000;    // 다른 함수의 bonus — 충돌 없음
  return 50000 + bonus;
}

console.log(calculateScore()); // 출력: 90
console.log(calculatePrice()); // 출력: 55000
```

## 6️⃣ 실습 과제

### 기본 과제 — 온도 변환 함수

파일 `temperature.js`를 만들고 두 가지 변환 함수를 **직접** 작성하세요.

조건:
1. `celsiusToFahrenheit(celsius)` — 섭씨를 화씨로 변환  
   공식: `화씨 = 섭씨 × 9/5 + 32`
2. `fahrenheitToCelsius(fahrenheit)` — 화씨를 섭씨로 변환  
   공식: `섭씨 = (화씨 - 32) × 5/9`
3. 결과는 소수점 첫째 자리까지 표시 (`Math.round(val * 10) / 10`)
4. 화살표 함수로 작성하세요

예상 출력:
```
0°C = 32°F
100°C = 212°F
37°C = 98.6°F
98.6°F = 37°C
```

---

### 도전 과제 — 배열 평균 함수

파일 `average.js`를 만들고 다음 함수들을 작성하세요.

1. `sum(numbers)` — 배열의 합계를 반환
2. `average(numbers)` — 배열의 평균을 반환 (내부에서 `sum` 함수를 호출)
3. `getStats(numbers)` — 합계, 평균, 최대값, 최소값을 **객체**로 반환

```javascript
const scores = [85, 92, 78, 96, 61, 88, 74];

console.log(sum(scores));      // 출력: 574
console.log(average(scores));  // 출력: 82

const stats = getStats(scores);
console.log(stats);
// 출력: { total: 574, average: 82, max: 96, min: 61 }
```

힌트: 최대값과 최소값을 구할 때 `Math.max(...numbers)`, `Math.min(...numbers)`를 사용해도 됩니다.  
(스프레드 연산자 `...`는 다음 장에서 배웁니다)

## 7️⃣ 정리

### 이 장에서 배운 것

| 개념 | 핵심 |
|------|------|
| 함수 선언 | `function 이름(매개변수) { ... }` |
| 함수 호출 | `이름(인자)` — 호출해야 실행됨 |
| 매개변수 | 함수에 전달하는 입력값 |
| 반환값 | `return`으로 결과를 돌려줌. 없으면 `undefined` |
| 화살표 함수 | `const 이름 = (매개변수) => 반환값` |
| 콜백 함수 | 함수를 다른 함수의 인자로 전달 |
| `forEach` | 배열 각 원소에 함수를 실행 |
| 스코프 | 변수가 보이는 범위. 함수/블록 안의 변수는 밖에서 안 보임 |

### 자주 하는 실수

```javascript
// 실수 1: 함수 선언 후 호출을 잊음
function sayHi() {
  console.log("안녕!");
}
// sayHi(); ← 호출 안 하면 아무것도 출력 안 됨

// 실수 2: return 없이 반환값을 받으려 함
function add(a, b) {
  a + b;  // return 없음!
}
const result = add(3, 4);
console.log(result); // 출력: undefined

// 실수 3: 화살표 함수에서 return 생략 시 중괄호도 없애야 함
const double = n => { n * 2 };  // 틀림! 중괄호 있으면 return 써야 함
const double = n => n * 2;      // 맞음
const double = n => { return n * 2; };  // 이것도 맞음

// 실수 4: 지역 변수를 함수 밖에서 쓰려 함
function calc() {
  const result = 42;
}
calc();
console.log(result); // 에러!
```

### 다음 장 미리보기

06장에서는 **객체와 배열 심화**를 배웁니다.
