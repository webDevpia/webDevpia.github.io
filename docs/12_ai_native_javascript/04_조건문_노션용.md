## ⭐ 핵심: 1️⃣ 조건문이란? — 자판기 비유

### 자판기를 생각해 보세요

자판기 앞에 섰습니다. 음료를 뽑으려면 어떻게 되나요?

```
동전을 넣는다
  ↓
[조건 확인]: 잔액이 음료 가격 이상인가?
  ↓
  YES → 음료가 나온다
  NO  → "잔액 부족" 화면이 켜진다
```

이것이 조건문의 본질입니다.  
**어떤 조건이 참(true)이면 A를 하고, 거짓(false)이면 B를 한다.**

프로그램의 거의 모든 "결정"은 이렇게 조건문으로 구현됩니다.

- 로그인: 비밀번호가 맞으면 → 입장, 틀리면 → "비밀번호 오류"
- 쇼핑: 재고가 있으면 → "구매하기" 버튼 활성화, 없으면 → "품절"
- 게임: 체력이 0이면 → "게임 오버", 아니면 → 계속 플레이

## ⭐ 핵심: 2️⃣ if / else — 두 갈래 길

> 💡 **실습 흐름**: 강사 시연 보기 → 함께 따라하기 → 혼자 변형해 보기

### 기본 문법

```javascript
if (조건) {
  // 조건이 true일 때 실행
} else {
  // 조건이 false일 때 실행
}
```

### 첫 번째 예제 — 자판기

```javascript
const balance = 500;    // 현재 잔액
const price = 300;      // 음료 가격

if (balance >= price) {
  console.log("음료가 나옵니다. 잔액: " + (balance - price) + "원");
} else {
  console.log("잔액이 부족합니다. " + (price - balance) + "원 더 넣어주세요.");
}
// 출력: 음료가 나옵니다. 잔액: 200원
```

잔액을 200으로 바꾸면:

```javascript
const balance = 200;
const price = 300;

if (balance >= price) {
  console.log("음료가 나옵니다.");
} else {
  console.log("잔액이 부족합니다. " + (price - balance) + "원 더 넣어주세요.");
}
// 출력: 잔액이 부족합니다. 100원 더 넣어주세요.
```

---

### === 를 쓰세요, == 가 아니라

값을 비교할 때는 반드시 `===` (등호 세 개)를 씁니다.

```javascript
const input = "5";  // 사용자 입력 (문자열)
const answer = 5;   // 정답 (숫자)

console.log(input == answer);  // 출력: true  ← 타입이 달라도 같다고 함! (위험)
console.log(input === answer); // 출력: false ← 타입까지 비교 (안전)
```

`==`는 "느슨한 비교"로 타입이 달라도 억지로 맞추려 합니다.  
`===`는 "엄격한 비교"로 값과 타입이 **모두** 같아야 true입니다.

**규칙: 항상 `===`를 쓰세요.** `==`는 잊어버리세요.

---

### else 없이도 됩니다

조건이 거짓일 때 아무것도 안 해도 된다면 `else`를 생략합니다.

```javascript
const hour = new Date().getHours(); // 현재 시간 (0~23)

if (hour < 12) {
  console.log("좋은 아침입니다!");
}

// 12시 이후에는 아무것도 출력하지 않음
```

## ⭐ 핵심: 3️⃣ else if — 여러 갈래 길

### 조건이 세 가지 이상일 때

자판기 예제를 확장해봅시다. 이번엔 성적을 등급으로 변환합니다.

```javascript
const score = 85;
let grade;

if (score >= 90) {
  grade = "A";
} else if (score >= 80) {
  grade = "B";
} else if (score >= 70) {
  grade = "C";
} else if (score >= 60) {
  grade = "D";
} else {
  grade = "F";
}

console.log(`점수: ${score}점, 등급: ${grade}`);
// 출력: 점수: 85점, 등급: B
```

`else if`는 위에서부터 순서대로 확인합니다.  
처음 true인 조건을 만나면 그 블록을 실행하고 나머지는 건너뜁니다.

---

### 순서가 중요합니다

```javascript
const score = 95;

// 잘못된 순서 — 버그 발생
if (score >= 60) {
  console.log("D"); // 출력: D  ← 95점인데 D가 나옴!
} else if (score >= 70) {
  console.log("C");
} else if (score >= 80) {
  console.log("B");
} else if (score >= 90) {
  console.log("A");
}
```

95는 60보다 크니까 첫 번째 조건에서 바로 걸립니다.  
이렇게 조건문은 **위에서 아래로 순서대로** 확인하므로, 좁은 범위를 먼저 써야 합니다.

---

### 실생활 예제 — 배달 예상 시간

```javascript
const distance = 3.5; // km

let estimatedTime;

if (distance <= 1) {
  estimatedTime = "10분";
} else if (distance <= 3) {
  estimatedTime = "20분";
} else if (distance <= 5) {
  estimatedTime = "30분";
} else {
  estimatedTime = "30분 이상";
}

console.log(`배달 예상 시간: ${estimatedTime}`);
// 출력: 배달 예상 시간: 30분
```

## ⭐ 핵심: 4️⃣ 비교 연산자와 논리 연산자

### 비교 연산자 — 두 값을 비교하면 boolean이 나온다

| 연산자 | 의미 | 예시 | 결과 |
|--------|------|------|------|
| `===` | 같다 (타입 포함) | `5 === 5` | `true` |
| `!==` | 같지 않다 | `5 !== 3` | `true` |
| `>` | 크다 | `5 > 3` | `true` |
| `<` | 작다 | `5 < 3` | `false` |
| `>=` | 크거나 같다 | `5 >= 5` | `true` |
| `<=` | 작거나 같다 | `3 <= 5` | `true` |

```javascript
console.log(10 > 5);    // 출력: true
console.log(10 < 5);    // 출력: false
console.log(10 >= 10);  // 출력: true
console.log("a" === "a"); // 출력: true
console.log("A" === "a"); // 출력: false ← 대소문자 구분
```

---

### 논리 연산자 — 조건을 합치기

#### && (그리고 / AND)

두 조건이 **모두** true여야 true입니다.

```javascript
const age = 20;
const hasId = true;

if (age >= 18 && hasId) {
  console.log("입장 가능합니다.");
} else {
  console.log("입장할 수 없습니다.");
}
// 출력: 입장 가능합니다.
```

일상 예시: "나이가 18세 이상이고, 신분증이 있어야" 클럽에 들어갈 수 있다.

```javascript
// 18세 미만이면
const age2 = 16;
if (age2 >= 18 && hasId) {
  console.log("입장 가능합니다.");
} else {
  console.log("입장할 수 없습니다."); // ← 나이 조건 실패
}
// 출력: 입장할 수 없습니다.
```

---

#### || (또는 / OR)

두 조건 중 **하나라도** true면 true입니다.

```javascript
const isWeekend = false;
const isHoliday = true;

if (isWeekend || isHoliday) {
  console.log("오늘은 쉬는 날입니다!");
} else {
  console.log("평일입니다. 출근!");
}
// 출력: 오늘은 쉬는 날입니다!
```

일상 예시: "주말이거나 공휴일이면" 쉬는 날이다.

---

#### ! (아니다 / NOT)

조건을 뒤집습니다. true → false, false → true.

```javascript
const isRaining = false;

if (!isRaining) {
  console.log("우산 없이 나가도 됩니다.");
} else {
  console.log("우산을 챙기세요.");
}
// 출력: 우산 없이 나가도 됩니다.
```

---

### && 와 || 를 함께 쓸 때

```javascript
const score = 75;
const attendance = 80; // 출석률 (%)

// 점수 70 이상이고 출석률 70 이상이어야 통과
if (score >= 70 && attendance >= 70) {
  console.log("수료 조건을 충족했습니다.");
} else {
  console.log("수료 조건 미충족.");
}
// 출력: 수료 조건을 충족했습니다.
```

복잡한 조건은 괄호로 묶어서 명확하게 만드세요.

```javascript
const isVIP = false;
const couponCode = "SAVE10";

// VIP이거나, 쿠폰을 가지고 있어야 할인
if (isVIP || couponCode !== "") {
  console.log("10% 할인 적용됩니다.");
}
// 출력: 10% 할인 적용됩니다.
```

## 🚀 도전: 5️⃣ switch — 메뉴판 같은 조건문

> 여유가 있다면 도전해 보세요! 수업 후에 천천히 익혀도 좋습니다.

### switch는 언제?

`if/else if`가 "특정 변수가 정확히 이 값일 때"를 여러 개 비교할 때,  
`switch`가 더 읽기 쉬울 수 있습니다.

```javascript
const day = "화";
let message;

switch (day) {
  case "월":
    message = "한 주의 시작! 힘내세요.";
    break;
  case "화":
    message = "화요일, 한 고비 넘겼어요.";
    break;
  case "수":
    message = "수요일, 정중앙!";
    break;
  case "목":
    message = "목요일, 거의 다 왔어요.";
    break;
  case "금":
    message = "금요일! 불금이에요!";
    break;
  case "토":
  case "일":
    message = "주말입니다. 쉬세요!";
    break;
  default:
    message = "올바른 요일을 입력하세요.";
}

console.log(message);
// 출력: 화요일, 한 고비 넘겼어요.
```

**`break`를 빠뜨리면 다음 case로 흘러내려갑니다 (fall-through).** 항상 `break`를 씁니다.

---

### switch vs if/else — 언제 뭘 쓸까?

| 상황 | 추천 |
|------|------|
| 특정 값 하나를 여러 경우와 비교 | `switch` |
| 범위 비교 (크다/작다) | `if/else` |
| 복잡한 논리 조건 (`&&`, `\|\|`) | `if/else` |

```javascript
// 범위 비교 — if/else가 자연스러움
if (score >= 90) {
  console.log("A");
} else if (score >= 80) {
  console.log("B");
}

// 정확한 값 비교 — switch도 좋음
switch (status) {
  case "pending":
    console.log("처리 중");
    break;
  case "done":
    console.log("완료");
    break;
  case "error":
    console.log("오류 발생");
    break;
}
```

### 도전 과제 — 가위바위보 게임

파일 `rps.js`를 만들고 가위바위보 게임을 구현하세요.

- `Math.random()`과 `Math.floor()`로 컴퓨터의 선택을 랜덤으로 만들 수 있습니다


## 7️⃣ 정리

### 이 장에서 배운 것

| 개념 | 핵심 |
|------|------|
| `if / else` | 조건이 true/false에 따라 다른 코드 실행 |
| `else if` | 조건이 3개 이상일 때. 위에서 아래로 순서대로 확인 |
| `===` | 엄격한 같음 비교 (타입 포함). 항상 `===` 사용 |
| `!==` | 같지 않음 비교 |
| `&&` | 그리고 — 모두 true여야 true |
| `\|\|` | 또는 — 하나만 true여도 true |
| `!` | 아니다 — 결과를 뒤집음 |
| `switch` | 하나의 값을 여러 경우와 비교할 때 |

### 자주 하는 실수

```javascript
// 실수 1: = 와 === 혼동
if (score = 90) { ... }  // 이건 대입! 조건문이 아님
if (score === 90) { ... }  // 이게 비교

// 실수 2: == 사용 (타입이 달라도 true가 될 수 있음)
if ("5" == 5) { ... }  // true — 위험!
if ("5" === 5) { ... } // false — 안전

// 실수 3: switch에서 break 빠뜨리기
switch (x) {
  case 1:
    console.log("하나");
    // break 없으면 아래 case도 실행됨!
  case 2:
    console.log("둘");
}
```
