## 1️⃣ 왜 지금 AI-Native인가? [↑](#toc)

여러분은 코딩할 때 AI 도구를 써본 적 있나요? 아직 써보지 않았다면, 이미 84%의 개발자들과 다른 출발선에 서 있는 것입니다.

> AI는 "거의 맞는" 초안을 빠르게 만들지만,
> "정확히 맞는" 결과를 만드는 건 여전히 사람의 몫입니다.
> 생산성이 높아질수록, 검증 능력의 가치도 함께 올라갑니다.

### 지금 시장에서 일어나는 일

| 지표 | 수치 | 의미 |
|------|------|------|
| AI 도구 사용/계획 | 개발자의 **84%** (SO 2025) | AI가 실험 단계를 넘어 기본 개발 도구로 자리잡음 |
| "거의 맞는 답" 불만 | **66%** 가 반복 수정 경험 (SO 2025) | 생성 속도보다 검증·정제 역량이 핵심 |
| AI 역량 중심 채용 | 전체 채용은 약세, AI 언급 채용은 강화 (Indeed 2026) | "아무나"가 아니라 "AI 협업 역량" 기준으로 선별 |

### 이것이 의미하는 것

- AI-Native는 "코딩 자동화"가 아닙니다. 개발 운영체제가 교체되는 것입니다.
- 핵심 병목은 작성 속도가 아니라 **검증, 리뷰, 배포, 책임 설계**입니다.
- 개인 경쟁력은 "코드를 얼마나 빨리 쓰는가"에서 "AI와 협업해 얼마나 안정적으로 제품을 내는가"로 이동하고 있습니다.

## 2️⃣ 전통 개발 vs AI-Native 개발 [↑](#toc)

> 요리사가 직접 칼질하던 것에서,
> 자동 조리 기계의 설정과 품질 검수를 담당하는 것으로.
> 요리사가 사라지는 게 아니라, 역할이 바뀌는 겁니다.

### 개발 패러다임 비교

| 항목 | 전통 개발 | AI-Native 개발 |
|------|-----------|----------------|
| 요구사항 해석 | PM/개발자 문서 중심 | 자연어 + 구조화 컨텍스트 동시 활용 |
| 설계 | 선행 설계 후 구현 | 설계-구현-검증의 짧은 루프 반복 |
| 구현 | 인간이 직접 작성 | 인간 의도 + 에이전트 생성/수정/리팩터 |
| 테스트 | 구현 후 테스트 | 생성 단계부터 테스트/검증 동시 수행 |
| 코드 리뷰 | 사람 중심 PR 리뷰 | AI 1차 리뷰 + 사람 최종 승인 |
| 지식 관리 | 위키/문서 분절 | 프로젝트 컨텍스트 + 규칙 파일 기반 |
| **개발자 역할** | **구현자** | **오케스트레이터** |

### 핵심 역량의 이동

**Before**: 알고리즘과 문법 숙련 중심

**After**: 문제 분해, 컨텍스트 설계, 검증 자동화, 위험 통제, 협업 프로토콜 설계

이 과정에서 배우는 Custom Instructions, Prompt Files, Custom Agents는 바로 이 "오케스트레이터" 역할을 실습하는 도구입니다.

## 3️⃣ AI 코딩 에이전트 5종 비교 [↑](#toc)

2026년 상반기 기준 주요 AI 코딩 에이전트입니다. 이 과정에서는 **GitHub Copilot**을 사용합니다.

| 도구 | 포지셔닝 | 주요 인터페이스 | 이 과정과의 관련 |
|------|---------|----------------|----------------|
| **GitHub Copilot** | IDE/조직 정책 통합형 | VS Code, JetBrains, GitHub, CLI | **이 과정의 실습 도구** |
| Cursor | 에이전트 중심 IDE | IDE + Cloud Agents | 고성능 워크플로, 모델 선택 폭 넓음 |
| Claude Code | 터미널/에이전트 실행형 | Terminal, IDE, Slack | 멀티파일 편집, 이슈→PR 자동화 강점 |
| OpenAI Codex | ChatGPT 연계 에이전트 | Web, App, IDE, CLI | ChatGPT 계정 기반 연속 경험 |
| Google Antigravity | 에이전트 중심 IDE | IDE + Browser | 개인 무료 플랜, Google AI 연동 |

### GitHub Copilot을 선택한 이유

- GitHub 생태계와 가장 긴밀하게 통합
- 엔터프라이즈 정책과 감사(audit) 기능 지원
- VS Code 기반 학습 자료가 풍부
- 학생/교육자 무료 플랜 존재 (GitHub Student Developer Pack)

## 4️⃣ 비용 현실 [↑](#toc)

실제로는 초과 사용량(overage)이 총비용의 45~75%를 더 올릴 수 있습니다.

| Cursor Pro 통일 | $20 × 30명 × 4개월 | $2,400 |

**초과 사용량까지 포함했을 때 (Copilot Pro 기준)**:

## 6️⃣ 정리 [↑](#toc)

이번 장의 핵심 3줄:

1. **AI-Native는 도구 교체가 아니라 역할 교체입니다.** 개발자는 구현자에서 오케스트레이터가 됩니다.
2. **생산성 향상의 본질은 생성 속도가 아니라 검증 파이프라인의 성숙도입니다.** "거의 맞는 답"을 정확히 맞는 결과로 만드는 역량이 핵심입니다.
3. **이 과정이 다루는 Custom Instructions, Prompt Files, Custom Agents는 그 오케스트레이터 역할을 구체적으로 실행하는 방법입니다.**
