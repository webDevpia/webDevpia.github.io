# Qdrant 벡터 DB 검색

## 라이브러리 설치 및 준비

실습에 필요한 파이썬 라이브러리들을 설치합니다. `%pip install` 매직 명령을 사용하여 여러 패키지를 최신 버전으로 업그레이드하며 설치합니다:

* **`python-dotenv`**: `.env` 파일에서 환경 변수를 읽어오는 데 사용됩니다 (API 키 등을 코드에 직접 쓰지 않고 관리하기 위해 사용).
* **`qdrant-client`**: Qdrant 벡터 데이터베이스에 연결하고 조작하기 위한 파이썬 클라이언트 라이브러리입니다.
* **`llama-index`** 및 **`llama-index-vector-stores-qdrant`**: Llama Index(예전 이름: GPT Index) 라이브러리와 Qdrant 벡터스토어 연동을 위한 플러그인입니다. 텍스트 임베딩 생성과 벡터 DB 연계를 도와줍니다.
* **`pandas`**: 표 형식의 데이터를 다루는 라이브러리로, CSV 등의 데이터 파일을 읽고 데이터프레임으로 처리하기 위해 사용합니다.
* **`scikit-learn` (scikit-learn)**: 머신러닝 라이브러리로, 여기서는 TF-IDF 벡터화를 위해 사용됩니다 (`TfidfVectorizer` 제공).

코드 셀을 실행하면 위 패키지들이 설치되며, 설치 후 커널 재시작을 권장하는 메시지가 나올 수 있습니다. 이후 코드들이 해당 라이브러리를 임포트하여 사용하게 됩니다.

```python
%pip install -U python-dotenv qdrant-client llama-index llama-index-vector-stores-qdrant pandas scikit-learn
```

## 환경 변수 로드 (API 키 설정)

`.env` 파일에 저장된 **환경 변수**들을 불러옵니다. 외부 API 키나 민감한 정보를 코드에 직접 작성하지 않고 관리하기 위한 관용적인 방법입니다:

* `load_dotenv()` 함수를 통해 현재 작업 디렉토리에 있는 `.env` 파일을 읽어 환경 변수로 설정합니다.
* `os.getenv('KEY_NAME')`를 사용하여 환경변수 값을 가져옵니다. 여기서는 OpenAI API 키, Qdrant API 키, Qdrant 서버 URL을 불러와 각각 `OPENAI_API_KEY`, `QDRANT_API_KEY`, `QDRANT_URL` 변수에 저장합니다.
* 이러한 방식을 통해 API 키 등이 코드에 하드코딩되지 않고도 필요 시 불러와 사용할 수 있습니다. (예를 들어 OpenAI 임베딩 API를 사용하기 위해 `OPENAI_API_KEY`가 필요하고, Qdrant의 클라우드나 서버에 접속하기 위해 `QDRANT_URL`과 인증 키가 필요합니다.)


```python
from dotenv import load_dotenv
import os

load_dotenv()

OPENAI_API_KEY = os.getenv('OPENAI_API_KEY')
QDRANT_API_KEY = os.getenv('QDRANT_API_KEY')
QDRANT_URL = os.getenv('QDRANT_URL')
```

## Qdrant 클라이언트 연결

Qdrant에 연결하기 위한 클라이언트를 초기화합니다:

* `QdrantClient` 클래스를 임포트하고, `url`과 `api_key`를 지정하여 `client` 객체를 생성합니다. 이때 `QDRANT_URL`은 Qdrant 서버의 주소, `QDRANT_API_KEY`는 인증을 위한 키입니다. (예: Qdrant Cloud를 사용하거나 로컬 Qdrant 서버에 API 키 인증을 설정한 경우 활용)
* 이렇게 생성된 `client`는 Qdrant 벡터 데이터베이스와 통신할 수 있는 객체로, 이후 벡터 데이터의 등록, 검색 등의 작업에 사용됩니다.
* `client.get_collections()`로 현재 Qdrant 서버에 존재하는 컬렉션 목록을 불러옵니다. 컬렉션(collection)은 Qdrant에서 벡터들을 저장하는 **공간**이나 **테이블** 개념입니다.
* 처음 실행 시 아무 컬렉션도 없으므로 출력 결과는 `collections=[]`와 같이 빈 리스트가 출력됩니다. 이를 통해 현재 새로운 데이터베이스 상태임을 확인할 수 있습니다.


```python
from qdrant_client import QdrantClient

client = QdrantClient(
    url=QDRANT_URL,
    api_key=QDRANT_API_KEY,
)

print(client.get_collections())
```

## 데이터 읽어오기 (CSV 파일)

실습에 사용할 **데이터셋**을 불러옵니다. 데이터는 `01.data.csv`라는 CSV 파일에 저장되어 있으며, 뉴스 기사의 간략한 **제목/내용**과 **카테고리** 등이 포함되어 있다고 가정합니다:

* `pandas`를 사용하여 CSV 파일을 읽습니다. `pd.read_csv("01.data.csv", encoding="utf-8")`를 통해 CSV 파일 내용을 데이터프레임(`df_data`)으로 로드합니다. `encoding="utf-8"`은 한글 등이 깨지지 않도록 인코딩 형식을 지정한 것입니다.
* `df_data.to_dict(orient="records")`는 데이터프레임을 **리스트** 형태로 변환합니다. 각 행(row)을 하나의 파이썬 딕셔너리로 바꾸고, 이 딕셔너리들의 리스트를 `data` 변수에 저장합니다. `orient="records"` 옵션을 통해 `'id'`, `'category'`, `'text'` 등의 열을 키(key)로 하는 딕셔너리 리스트를 얻습니다.
* 마지막 줄에서 `data`를 그대로 적어주었기 때문에, Jupyter 노트북에서는 `data`의 내용을 출력해 줍니다. 이 리스트의 각 요소는 하나의 뉴스 항목을 나타내며, 예를 들면 다음과 같습니다:

  * `{'id': 1, 'category': '정치', 'text': '대통령, 신년 기자회견에서 경제 정책 발표'}`
  * `{'id': 2, 'category': '경제', 'text': '주식 시장, 연일 강세 지속'}`
  * *...* (총 20개 항목)
    이처럼 `id` (뉴스 번호), `category` (분류), `text` (뉴스 제목 또는 내용) 필드를 가진 데이터 목록을 확보했습니다.

이 데이터는 이후에 벡터로 변환되어 Qdrant에 저장되고, 유사도 검색의 대상이 될 것입니다.


```python
import pandas as pd

df_data = pd.read_csv("01.data.csv", encoding="utf-8")
data = df_data.to_dict(orient="records")
data
```

## OpenAI 임베딩 모델 설정

OpenAI의 텍스트 임베딩(embedding) 모델을 설정합니다. Llama Index 라이브러리를 통해 OpenAI 임베딩 API를 사용할 것입니다:

* `OpenAIEmbedding` 클래스를 임포트하여 사용합니다. 이 클래스는 OpenAI의 텍스트 임베딩 API를 간편하게 호출할 수 있도록 래핑(wrapping)되어 있습니다.
* `embedding_model = OpenAIEmbedding(...)`로 임베딩 모델 인스턴스를 생성합니다. `model="text-embedding-3-small"`는 사용하고자 하는 OpenAI의 임베딩 모델 이름을 지정한 것입니다. (OpenAI에서는 다양한 크기의 임베딩 모델을 제공하며, 여기서는 예시로 "text-embedding-3-small"을 사용했습니다.)
* `api_key=OPENAI_API_KEY`를 통해 앞서 불러온 OpenAI API 키를 제공하여 인증합니다. 이를 통해 `embedding_model` 객체를 통해 OpenAI의 임베딩 서비스에 요청을 보낼 수 있습니다.
* 이제 `embedding_model`을 사용하여 텍스트를 벡터로 변환(임베딩)할 준비가 되었습니다. (참고: OpenAI의 임베딩 모델을 호출하면 주어진 텍스트를 의미론적 공간의 고차원 벡터로 변환해 줍니다.)


```python
from llama_index.embeddings.openai import OpenAIEmbedding

embedding_model = OpenAIEmbedding(model="text-embedding-3-small", api_key=OPENAI_API_KEY)
```

## 텍스트 데이터 리스트 추출

임베딩을 생성하기 위해 **텍스트만 추출**하여 리스트를 만드는 작업을 합니다:

* `data` 리스트에서 각 항목의 `'text'` 값을 가져와서 새로운 리스트 `texts`에 담았습니다. 리스트 컴프리헨션 구문 `[item['text'] for item in data]`는 `data`의 각 딕셔너리 `item`에서 `'text'` 필드를 추출하여 `texts` 리스트를 만드는 파이썬 문법입니다.
* 이렇게 하면 방금 CSV에서 불러온 20개 뉴스 항목의 텍스트 부분만 모은 **문자열 리스트**가 생성됩니다.
* 마지막에 `texts`를 적어서 출력함으로써, 리스트에 담긴 뉴스 텍스트들을 확인합니다. 출력 결과는 예를 들어 다음과 같은 문자열들의 리스트입니다:

  ```
  ['대통령, 신년 기자회견에서 경제 정책 발표',
   '주식 시장, 연일 강세 지속',
   '한국 축구 대표팀, 월드컵 예선 승리',
   '새로운 AI 기술, 의료 진단에 활용',
   ...,
   '환경 보호 캠페인, 시민 참여 독려']
  ```

  모든 뉴스 기사들의 제목/문장이 잘 추출되어 리스트로 모인 것을 볼 수 있습니다.

이 `texts` 리스트를 사용하는 이유는, 임베딩 모델이나 TF-IDF 벡터화 등 텍스트 데이터를 입력으로 받는 함수에 편리하게 넣어주기 위함입니다.



```python
texts = [item['text']for item in data]
texts
```

## 텍스트 임베딩 생성 (배치 처리)

OpenAI 임베딩 모델을 이용해 방금 준비한 `texts` 리스트에 대한 **임베딩 벡터**를 한꺼번에 생성합니다. 또한 각 임베딩 벡터의 차원 수를 확인해 봅니다:

* `embedding_model.get_text_embedding_batch(texts)`를 호출하여 `texts` 리스트의 모든 문장에 대한 임베딩을 한꺼번에 요청합니다. \*\*배치(batch)\*\*로 처리를 함으로써 효율적으로 여러 텍스트의 벡터를 얻습니다. 결과는 각 텍스트에 대응되는 임베딩 벡터들의 리스트로 반환되며, 이를 `embeddings` 변수에 저장합니다.
* `embeddings`는 길이가 20인 리스트이며, 각 원소가 하나의 뉴스 텍스트를 표현하는 **벡터**입니다. 이 벡터들은 고차원 부동소수점 숫자들의 배열로 이루어져 있습니다.
* 이어지는 `for` 루프에서는 각 임베딩 벡터의 길이를 출력합니다. `len(vec)`를 통해 벡터의 차원(dimension)을 구하여 `문서 1의 임베딩 벡터 차원 수: 1536`와 같이 출력합니다. 모든 문서의 임베딩 차원 수는 동일하며, 실행 결과 각 벡터의 길이가 **1536**임을 확인할 수 있습니다.
* **1536차원**이라는 숫자는 사용한 OpenAI 임베딩 모델(`text-embedding-3-small`)의 출력 벡터 크기를 의미합니다. 이처럼 각 문장을 1536차원의 벡터로 변환함으로써, 텍스트를 수치적으로 표현할 수 있게 되었습니다.


```python
embeddings = embedding_model.get_text_embedding_batch(texts)
for idx, vec in enumerate(embeddings):
    print(f"문서 {idx+1}의 임베딩 벡터 차원 수: {len(vec)}")
```

## 임베딩 벡터 예시 출력

생성된 임베딩 벡터의 실제 값을 **일부** 살펴보고자, 이 셀에서는 첫 번째 문서의 임베딩 벡터에서 처음 10개 값만 출력해 봅니다:

* `embeddings[0]`는 첫 번째 뉴스 텍스트에 대한 임베딩 벡터입니다. 이 벡터는 1536개의 실수(float)로 구성되어 있습니다. `[:10]`을 추가하여 그 중 앞쪽 10개의 요소만 잘라서 가져왔습니다.
* Jupyter 노트북에서 이 값을 출력하면 예를 들어 다음과 같은 결과를 얻을 수 있습니다:

  ```
  [0.003245 ..., -0.01234 ..., 0.0201 ..., -0.00789 ..., ..., -0.038443874567747116]
  ```

  (실제로는 10개의 실수 값이 리스트로 출력됩니다. 위 값들은 예시이며, 실제로는 각 요소가 소수점 아래 여러 자리가 있는 부동소수점 숫자입니다.)
* 이 출력으로부터 임베딩 벡터가 어떤 형태인지 감을 잡을 수 있습니다. 벡터의 각 차원은 -0.03, 0.02 등의 값을 가지며, 이러한 수치들의 조합이 해당 텍스트의 \*\*의미적 표현(embedding)\*\*이 됩니다. 개별 숫자 하나하나에 큰 의미를 부여하기는 어렵지만, 1536차원 공간에서 이 벡터의 위치가 텍스트의 의미를 나타내고, 유사한 텍스트들은 인접한 벡터로 배치됩니다.

요약하면, 임베딩 벡터는 사람의 언어를 기계가 이해할 수 있도록 높은 차원 좌표로 변환한 결과이며, 위에서는 그 일부를 확인했습니다.


```python
embeddings[0][:10]
```

## TF-IDF 벡터화 및 정보 출력

**TF-IDF** 방법을 사용하여 같은 텍스트들에 대한 **희소 벡터** 표현을 생성합니다. 또한 생성된 행렬의 크기와 단어 개수를 출력합니다:

* `TfidfVectorizer`를 임포트하여 사용합니다. TF-IDF는 \*Term Frequency - Inverse Document Frequency(단어 빈도-역문서 빈도)\*의 약자로, 문서 내 단어의 빈도와 해당 단어가 전체 문서집합에서 얼마나 희귀한지를 반영한 가중치 표현입니다. 이는 **전통적인 문서 벡터화 방법**으로, 각 단어를 차원으로 간주하고 단어의 가중치를 값으로 가지는 **희소 벡터**를 생성합니다.
* `vectorizer = TfidfVectorizer()`로 벡터라이저 객체를 만들고, `vectorizer.fit_transform(texts)`에 우리 텍스트 리스트를 넣어 TF-IDF 행렬을 한꺼번에 얻습니다. `fit_transform`은 내부적으로 모든 텍스트에서 \*\*어휘집(vocabulary)\*\*을 학습하고, 각 문서를 해당 어휘 공간에서 TF-IDF 벡터로 변환합니다.
* 결과로 나온 `tfidf_matrix`는 SciPy의 **희소 행렬(sparse matrix)** 형태입니다. 각 행이 하나의 문서에 해당하고, 각 열은 하나의 단어(용어)에 해당하는 행렬입니다. `print(tfidf_matrix.shape)`를 통해 이 행렬의 크기를 출력합니다.

  * 출력된 모양이 `(20, 104)`라면, **20개의 문서**와 \*\*104개의 특징(단어)\*\*으로 이루어진 행렬이라는 의미입니다. 즉, 우리의 전체 텍스트 데이터에서 고유한 단어가 104개 존재함을 나타냅니다.
* `vectorizer.vocabulary_` 속성은 단어 사전을 `{단어: 인덱스}` 형태로 저장하고 있습니다. `len(vectorizer.vocabulary_)`를 출력한 결과도 104로서, 어휘 사전에 104개의 단어가 있음을 확인할 수 있습니다. (이 값은 행렬의 열 수와 동일해야 합니다.)
* 요약하면, TF-IDF를 통해 각 문서를 104차원의 희소 벡터로 표현했고, 대부분의 차원 값은 0이며 해당 문서에 등장하는 단어 위치에만 가중치 값이 존재합니다. 이 방식은 **단어의 빈도** 기반 벡터 표현이며, 앞서 생성한 OpenAI 임베딩과는 다르게 **해석 가능한(단어 기준)** 벡터입니다.

*(출력 예시: `(20, 104)` 그리고 다음 줄에 `104`가 출력됩니다.)*

```python
from sklearn.feature_extraction.text import TfidfVectorizer

vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(texts)
print(tfidf_matrix.shape)
print(len(vectorizer.vocabulary_))
```

## 희소 벡터 인덱스와 값 추출

Qdrant에 **희소 벡터**를 저장하기 위해, TF-IDF 결과 행렬에서 각 문서별로 **유의미한 차원(단어)의 인덱스와 값**을 추출합니다. Qdrant에 희소 벡터를 넣을 때는 이렇게 인덱스와 값 목록으로 넣을 수 있습니다:

* `tfidf_matrix.getrow(i)`를 통해 i번째 문서의 TF-IDF 결과를 가져옵니다. 이는 희소 행렬에서 해당 행만 뽑아온 것으로, 역시 희소 벡터 형식입니다.
* `row.indices` 속성은 그 행에서 값이 0이 아닌 **유효한 열 인덱스**들을 NumPy 배열로 제공합니다. (즉, 해당 문서에 등장한 단어들의 인덱스 목록입니다.)
* `row.data` 속성은 그 행에서 0이 아닌 값들, 즉 해당 인덱스들에 대응되는 **TF-IDF 가중치 값**들을 NumPy 배열로 제공합니다.
* `astype(int).tolist()` / `astype(float).tolist()`를 사용하여 NumPy 배열을 일반 파이썬 리스트로 변환합니다. 그 결과를 `indices`와 `values` 변수에 저장합니다.
* 그런 다음 문서별 인덱스 리스트를 `sparse_indices_list`에, 값 리스트를 `sparse_values_list`에 차례로 추가(`append`)합니다.
* 이 루프를 문서 개수만큼 반복한 후, `sparse_indices_list`와 `sparse_values_list`는 각각 **길이 20**인 리스트가 됩니다. 예를 들어 `sparse_indices_list[0]`에는 0번 문서의 단어 인덱스들이 들어있고, `sparse_values_list[0]`에는 해당 단어들의 TF-IDF 값들이 들어있는 식입니다.
* 이렇게 준비한 인덱스/값 리스트는 Qdrant에 희소 벡터를 저장할 때 활용할 것입니다. (Qdrant는 희소 벡터를 직접 저장할 수 있는데, `{ indices: [...], values: [...] }` 형태로 저장 가능합니다.)

이 과정은 희소 벡터 데이터를 다루기 쉽게 파싱한 것으로, Qdrant에 업로드하기 좋은 형태로 변환한 단계입니다.


```python
sparse_indices_list = []
sparse_values_list = []

for i in range(tfidf_matrix.shape[0]):
    row = tfidf_matrix.getrow(i)
    indices = row.indices.astype(int).tolist()
    values = row.data.astype(float).tolist()
    sparse_indices_list.append(indices)
    sparse_values_list.append(values)
```

## 희소 벡터 예시 확인

앞서 추출한 희소 벡터 중 첫 번째 문서에 대한 인덱스와 값 리스트를 출력하여 제대로 추출되었는지 확인해봅니다:

* `sparse_indices_list[0]`는 0번 문서(첫 번째 뉴스 텍스트)의 TF-IDF 벡터에서 **0이 아닌 값이 있는 위치들의 인덱스** 리스트입니다.
* `sparse_values_list[0]`는 그 위치들에 해당하는 **TF-IDF 값들**의 리스트입니다.
* 출력 예시는 다음과 같습니다 (가령 첫 번째 문서에 등장한 단어가 6개라고 한다면):

  ```
  [24, 50, 21, 12, 73, 33]
  [0.4343474634097237, 0.4343474634097237, 0.4343474634097237, 0.4343474634097237, 0.3817980687003446, 0.3155936609864683]
  ```

  첫 번째 리스트 `[24, 50, 21, ...]`는 이 문서에서 나타난 단어들의 **어휘 사전 인덱스**입니다. 두 번째 리스트는 각각의 단어에 대한 TF-IDF 가중치 값입니다.
* 예를 들어 인덱스 24에 해당하는 단어, 50에 해당하는 단어, ... 등이 이 문서에 등장했고, 그들의 중요도가 0.43 등으로 부여된 것을 볼 수 있습니다. (모든 값이 0.4 정도로 동일한 것은, 이 특정 문서에 나타난 여러 단어들의 TF-IDF 값이 우열 없이 비슷함을 의미할 수 있습니다. 마지막 값 0.315는 상대적으로 덜 중요한 단어일 수 있습니다.)
* 이 출력으로부터 우리가 구성한 `sparse_indices_list`와 `sparse_values_list`가 의도한 대로 잘 만들어졌음을 확인했습니다. 이제 0이 아닌 부분만 뽑은 희소 벡터 정보가 준비되었으므로, 이를 벡터 DB에 넣어서 **하이브리드 검색**(Dense + Sparse)을 실험할 수 있습니다.


```python
print(sparse_indices_list[0])
print(sparse_values_list[0])
```

## Qdrant 컬렉션 생성 및 설정

이제 벡터 검색을 위해 **Qdrant에 컬렉션을 생성**합니다. 컬렉션은 벡터와 관련 메타데이터를 저장하는 공간이며, 여기에 \*\*벡터 설정(스펙)\*\*을 정의해야 합니다:

* `qdrant_client`의 `models` 모듈을 임포트하여, 벡터 파라미터와 거리(metrics) 등을 설정하는 클래스들을 사용합니다.
* `collection_name = "news"`로 새 컬렉션 이름을 `"news"`로 지정했습니다. (뉴스 기사 데이터이므로 이렇게 명명)
* 이미 해당 이름의 컬렉션이 있을 경우를 대비해 `client.collection_exists()`로 존재 여부를 확인하고, 있다면 `client.delete_collection()`으로 삭제하여 초기 상태를 만듭니다. 이렇게 하면 이전에 같은 이름으로 저장된 내용이 있더라도 깨끗이 지우고 새로 만들 수 있습니다.
* `client.create_collection()`을 호출하여 컬렉션을 생성합니다. 주요 인자는 다음과 같습니다:

  * `collection_name`: 컬렉션의 이름 (여기서는 `"news"`).
  * `vectors_config`: 컬렉션에 저장할 **Dense 벡터** 필드들의 구성을 딕셔너리로 지정합니다.

    * `"text_dense"`라는 키로 벡터 필드를 하나 정의했고, 값으로 `models.VectorParams(...)`을 설정했습니다.
    * `distance=models.Distance.COSINE`는 벡터 유사도 측정 방법으로 **코사인 유사도**를 사용하겠다는 뜻입니다 (내부적으로는 코사인 거리 계산). 임베딩 벡터의 경우 코사인 유사도를 흔히 사용합니다.
    * `size=len(embeddings[0])`는 벡터의 차원 수를 지정합니다. 우리가 생성한 임베딩 벡터의 길이가 1536이었으므로 size를 1536으로 설정하게 됩니다. Qdrant는 이 정보를 통해 고정된 차원의 벡터를 다룹니다.
  * `sparse_vectors_config`: 컬렉션에 저장할 **Sparse 벡터** 필드들의 구성을 딕셔너리로 지정합니다.

    * `"text_sparse"`라는 이름의 희소 벡터 필드를 정의했고, `models.SparseVectorParams()`로 설정했습니다. (별도의 매개변수를 지정하지 않았는데, Qdrant의 희소 벡터는 유연한 차원을 가지며, `indices`와 `values` 쌍으로 입력되므로 추가 설정이 간단합니다.)
    * 이 설정을 함으로써, 각 데이터 포인트에 `"text_sparse"` 필드로 희소 벡터를 함께 저장할 수 있게 됩니다.
* `create_collection` 호출의 결과로 컬렉션이 성공적으로 생성되면, Jupyter에서는 `True` 등을 반환하여 출력합니다. (실제로 코드 출력 결과 `True`로 나타났다면 컬렉션 생성이 완료된 것입니다.)
* 이제 Qdrant에 `"news"` 컬렉션이 만들어졌고, 이 컬렉션은 두 종류의 벡터 필드를 갖습니다:

  1. **text\_dense**: 1536차원 Dense 임베딩 벡터 (코사인 유사도 기준)
  2. **text\_sparse**: 가변 길이 Sparse 벡터 (단어 인덱스 및 값 목록으로 저장)

이렇게 컬렉션 설정을 함으로써, Qdrant는 우리의 데이터에 맞는 형식으로 벡터를 저장하고 검색할 준비를 마쳤습니다.


```python
from qdrant_client import models

collection_name = "news"

if client.collection_exists(collection_name=collection_name):
    client.delete_collection(collection_name=collection_name)

client.create_collection(
    collection_name=collection_name,
    vectors_config={
        "text_dense": models.VectorParams(
            distance=models.Distance.COSINE,
            size=len(embeddings[0])
        )
    },
    sparse_vectors_config={
        "text_sparse": models.SparseVectorParams()
    }
)
```

## 데이터 포인트 업로드 (Upsert)

준비된 데이터를 Qdrant 컬렉션에 **업로드**합니다. 각 뉴스 기사의 임베딩 벡터와 희소 벡터, 그리고 부가 정보(payload)를 모아 \*\*포인트(Point)\*\*로 만들고, 이를 일괄 업서트(upsert)합니다:

* `points_to_upload`라는 리스트를 만들고, 여기에 각 데이터 포인트를 생성하여 담을 것입니다.
* `enumerate(data)`를 사용하여 `data` 리스트 (뉴스 항목 딕셔너리들의 리스트)를 순회합니다. `i`는 인덱스 (0부터 시작)이고, `item`은 각 뉴스 딕셔너리입니다.
* 루프 내부에서 `models.PointStruct(...)`를 이용해 하나의 포인트를 생성합니다. `PointStruct`는 Qdrant에 저장할 한 개의 데이터 포인트(벡터 및 속성)를 표현하는 구조체입니다. 인자로:

  * `id`: 포인트의 고유 ID를 설정합니다. 여기서는 CSV에 있던 `'id'` 값을 그대로 사용합니다. (예: 1, 2, 3, ... 20)
  * `vector`: 딕셔너리 형태로, 해당 포인트에 포함될 벡터 필드들을 지정합니다.

    * `"text_dense"` 키에는 해당 문서의 임베딩 벡터 `embeddings[i]`를 넣습니다.
    * `"text_sparse"` 키에는 해당 문서의 희소 벡터를 `{"indices": ..., "values": ...}` 형태로 넣습니다. `sparse_indices_list[i]`와 `sparse_values_list[i]`를 이용해 우리가 앞서 준비한 해당 문서의 희소 벡터 정보를 입력합니다.
  * `payload`: 딕셔너리 형태로, 벡터와 함께 저장할 메타데이터를 지정합니다. 여기서는 원본 데이터의 `'category'` (뉴스 분류)와 `'text'` (뉴스 텍스트 내용) 값을 넣었습니다. 이 payload는 검색 결과를 볼 때 부가정보로 활용할 수 있습니다.
* 이렇게 만든 `point` 객체를 `points_to_upload` 리스트에 추가합니다. 이 루프가 끝나면 `points_to_upload`에는 20개의 `PointStruct`가 들어있으며, 각 포인트에 dense 벡터, sparse 벡터, payload가 모두 포함되어 있습니다.
* `client.upsert(...)`를 호출하여 Qdrant에 포인트들을 업로드합니다.

  * `collection_name='news'`로 업로드 대상 컬렉션을 지정하고, `points=points_to_upload`로 업로드할 포인트들의 리스트를 전달합니다.
  * 업서트(upsert)는 insert or update의 의미로, 같은 ID가 존재하면 데이터를 갱신하고 없으면 새로 삽입합니다. 지금은 신규 삽입이므로 insert 동작이 이루어집니다.
* `upsert`의 결과(`result`)는 `status` 속성을 가지는데, 이를 출력해보면 **completed**라는 상태 메시지가 나옵니다. (출력 예시: `Upsert result: completed`)

  * `completed`는 모든 포인트가 정상적으로 저장되었음을 의미합니다.
* 이 단계까지 완료되면, Qdrant `"news"` 컬렉션에 20개의 데이터 포인트가 모두 저장되었습니다. 각 포인트는 1536차원 임베딩 벡터와 TF-IDF 희소 벡터를 가지고 있으며, 부가적으로 카테고리와 텍스트 내용이 payload로 들어가 있습니다.

이제 Qdrant에 데이터가 준비되었으므로, 다음 단계에서는 이 데이터에 대해 벡터 **유사도 검색**을 수행해볼 것입니다.



```python
points_to_upload = []

for i, item in enumerate(data):
    point = models.PointStruct(
        id=item['id'],
        vector={
            "text_dense": embeddings[i],
            "text_sparse": {
                "indices": sparse_indices_list[i],
                "values": sparse_values_list[i]
            }
        },
        payload={
            "category": item['category'],
            "text": item['text']
        }
    )
    points_to_upload.append(point)

result = client.upsert(
    collection_name='news',
    points=points_to_upload
)

print(f"Upsert result: {result.status}")
```

## 검색 쿼리 임베딩 생성

이 셀에서는 사용자가 검색하고자 하는 **쿼리(query)** 문장을 임베딩 벡터로 변환합니다. 우리가 저장한 데이터와 동일한 방식으로 쿼리도 임베딩해야, 같은 벡터 공간에서 유사도 검색을 할 수 있습니다:

* `query_text` 변수에 검색 질의로 사용할 문장을 지정했습니다. 예시로 `"소비자 정책 발표"`라는 문장을 사용하고 있습니다. (이 문장은 예컨대 "소비자와 관련된 어떤 정책 발표"에 대한 질의라고 볼 수 있습니다.)
* `embedding_model.get_text_embedding(query_text)`를 호출하여, 앞서 사용한 OpenAI 임베딩 모델을 통해 이 한 문장을 임베딩 벡터로 변환합니다. 결과를 `query_dense_vec` 변수에 저장합니다.

  * 배치 처리 메소드가 아니라 단일 텍스트에 대한 임베딩이므로 `get_text_embedding` 메소드를 사용했습니다. 내부적으로 OpenAI API를 호출하여 1536차원의 임베딩 벡터를 얻습니다.
* `len(query_dense_vec)`를 실행하여 쿼리 임베딩 벡터의 차원 수를 확인합니다. 우리가 기대한 대로 **1536**이 출력될 것입니다.

  * 이 출력은 곧, 쿼리도 데이터와 동일한 벡터 공간(1536차원)으로 표현되었음을 뜻합니다. (임베딩 모델이 동일하므로 당연한 결과지만, 확인 차원에서 찍어본 것입니다.)
* 이로써 검색 질의문을 나타내는 `query_dense_vec` 벡터가 준비되었습니다. 다음으로는 이 벡터를 사용하여 Qdrant에서 유사한 문서를 찾아볼 것입니다.

*(참고: 실제 응용에서는 질의에 대해서도 데이터와 동일한 전처리/임베딩 과정을 거쳐야 합니다. 여기서는 간단히 한 문장을 지정하여 그 임베딩을 구했습니다.)*


```python
query_text = "소비자 정책 발표"
query_dense_vec = embedding_model.get_text_embedding(query_text)
len(query_dense_vec)
```

## 벡터 유사도 검색 실행

앞서 만든 `query_dense_vec`를 이용해 Qdrant에서 **벡터 유사도 검색**을 수행합니다. 쿼리 벡터와 가장 **코사인 유사도가 높은 벡터**들을 찾아 상위 결과를 반환받습니다:

* `client.search(...)` 메소드를 호출하여 검색을 수행합니다. (※ 경고: 실행 시 DeprecationWarning이 나타날 수 있는데, 이는 `search` 메소드가 향후 `query_points`로 대체될 예정이라는 알림입니다. 여기서는 편의상 `search`를 사용했습니다.)
* 주요 파라미터:

  * `collection_name='news'`: 어떤 컬렉션에서 검색할지 지정합니다. 우리가 사용 중인 `"news"` 컬렉션에서 검색합니다.
  * `query_vector=models.NamedVector(...)`: 검색에 사용할 **쿼리 벡터**를 지정합니다.

    * Qdrant 컬렉션이 여러 개의 벡터 필드를 가질 수 있으므로, `NamedVector`를 통해 어느 벡터 필드를 쿼리에 사용할지 명시합니다.
    * 여기서는 `name="text_dense"`로 우리가 저장한 dense 임베딩 벡터 필드를 선택했고, `vector=query_dense_vec`로 해당 필드에 대한 쿼리 벡터 값을 넣었습니다.
    * 따라서 **임베딩 벡터 공간에서** 유사도 검색을 수행하게 됩니다. (희소 벡터 `"text_sparse"`를 함께 사용하는 쿼리도 가능하지만, 이 코드에서는 dense 벡터만으로 검색했습니다.)
  * `limit=3`: 상위 3개의 결과만 반환하도록 설정합니다. (즉, 코사인 유사도가 가장 높은 3개의 포인트를 가져옵니다.)
  * `with_payload=True`: 검색 결과에 벡터만이 아니라 우리가 저장했던 payload(메타데이터)도 함께 가져오도록 합니다. 이를 통해 검색된 뉴스의 카테고리와 텍스트 내용을 결과에서 확인할 수 있게 됩니다.
* 이 호출의 결과 `results`는 검색된 상위 3개 벡터 포인트의 리스트입니다. 각 결과에는 유사도 점수(score)와 저장된 payload, ID 등이 포함되어 있습니다.

  * 코사인 유사도의 경우, Qdrant의 `Distance.COSINE` 설정 하에서는 **score 값이 클수록 쿼리와 유사도가 높다**고 해석됩니다. (예: 1에 가까울수록 유사도가 높음)
* 이제 `results` 리스트에 검색 결과가 담겼으며, 다음 셀에서 이를 출력하여 어떤 문서들이 검색되었는지 살펴볼 것입니다.


```python
results = client.search(
    collection_name='news',
    query_vector=models.NamedVector(name="text_dense", vector=query_dense_vec),
    limit=3,
    with_payload=True
)
```

## 검색 결과 출력

마지막으로, 검색된 결과를 보기 쉽게 출력합니다. `results`에 들어있는 상위 3개 문서의 카테고리와 텍스트, 점수를 순위와 함께 표시합니다:

* `enumerate(results, start=1)`로 결과 리스트를 순회하며 순위를 매깁니다. `i`는 1부터 시작하는 순위 번호입니다.
* `res`는 개별 결과로서, `res.score`에는 쿼리와 해당 포인트 간의 **유사도 점수**가 들어있고, `res.payload`에는 우리가 저장했던 `'category'`와 `'text'`가 들어있습니다.
* `res.payload.get('text')`로 저장된 기사 **텍스트**를 가져오고, `res.payload.get('category')`로 **카테고리**를 가져옵니다. 딕셔너리의 키로 가져오는 방식을 사용했습니다.
* `print(f"{i}. {category}: {title} (score: {score})")`를 통해 순위, 카테고리, 기사 제목, 점수를 한 줄에 포맷팅하여 출력합니다.
* 예시 출력 (실제 예시로 들었던 `"소비자 정책 발표"` 질의를 넣었을 때의 예상 결과):

  ```
  1. 경제: 중소기업 지원 정책 강화 발표 (score: 0.5571874)
  2. 정치: 대통령, 신년 기자회견에서 경제 정책 발표 (score: 0.41099024)
  3. 경제: 주식 시장, 연일 강세 지속 (score: 0.38271013)
  ```

  이 결과를 해석해보면:

  * 1위 문서는 카테고리 "경제"의 \*\*"중소기업 지원 정책 강화 발표"\*\*라는 기사로, score 약 0.557로 쿼리와 가장 유사했습니다. (쿼리 "소비자 정책 발표"와 직접적으로 단어가 일치하지는 않지만, "정책 발표"라는 중요한 의미를 공유하고 있고 경제 분야의 발표라는 점에서 임베딩 상 유사도가 높았을 것입니다.)
  * 2위 문서는 카테고리 "정치"의 \*\*"대통령, 신년 기자회견에서 경제 정책 발표"\*\*로 score 0.411을 보입니다. 이 문장 역시 "정책 발표"란 표현이 있고 경제 정책에 대한 내용이어서 쿼리와 연관성이 있습니다.
  * 3위 문서는 카테고리 "경제"의 \*\*"주식 시장, 연일 강세 지속"\*\*으로 score 0.382입니다. 직접적인 단어 일치는 없지만, 경제 분야란 점에서 어느 정도 연관성을 가져 임베딩상 유사도로 선택된 것으로 볼 수 있습니다.
* 위 결과는 **벡터 임베딩 검색**의 위력을 보여줍니다. 단순 키워드 매칭이 아닌 **의미 기반 유사도**를 통해, 쿼리와 주제가 비슷한 문서들을 찾아낸 것입니다. (예: 쿼리에 "소비자"란 단어가 있었지만, 1위 결과는 "소비자"란 단어가 없어도 정책 발표라는 맥락의 유사성으로 선정되었음)
* 만약 **하이브리드 검색**을 활용했다면, sparse 벡터(키워드 매칭)까지 고려하여 더 정확한 결과를 얻을 수도 있습니다. 여기서는 dense 임베딩으로만 검색했지만, Qdrant에 sparse 벡터를 함께 저장한 것은 이처럼 키워드 일치와 임베딩 유사도를 함께 활용할 수 있다는 점을 염두에 둔 구성입니다.


```python
for i, res in enumerate(results, start=1):
    score = res.score
    title = res.payload.get('text')
    category = res.payload.get('category')
    print(f"{i}. {category}: {title} (score: {score})")
```

이로써, **Qdrant를 이용한 벡터 데이터베이스 검색 실습**이 완료되었습니다. 우리는:

1. 텍스트 데이터를 불러와 임베딩(dense vector)과 TF-IDF(sparse vector)를 생성했고,
2. 이를 Qdrant에 저장한 후,
3. 한 쿼리에 대해 임베딩 유사도 검색을 수행하여 의미적으로 유사한 문서를 찾는 과정을 거쳤습니다.

결과를 통해 벡터 DB를 사용한 **유사도 검색**의 개념과 효과를 체험할 수 있었습니다.
