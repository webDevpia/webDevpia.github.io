# AWS 클라우드 입문 — 1일 강의 계획서

---

## 과정 개요

| 항목 | 내용 |
|------|------|
| **과정명** | AWS 클라우드 입문 — EC2로 나만의 웹 서비스 배포하기 |
| **대상** | 비전공자 / 클라우드 처음 접하는 학습자 |
| **수업 시간** | **1일 7시간 (7교시 × 60분)** — 점심·휴식은 강사 재량 |
| **실습 환경** | 개인 노트북 + AWS 계정 (신규 가입 기준 — 2025-07-15 이후 정책) |

> **시간 운영:** 본 계획서는 **수업 7시간(420분)** 분량의 콘텐츠를 7교시로 구성한 것입니다. 점심·쉬는 시간은 강사가 현장 상황에 맞춰 자유롭게 운영합니다.

---

## 학습 목표

수업을 마치면 다음을 할 수 있습니다:

1. 클라우드와 AWS의 기본 개념을 설명할 수 있다
2. 신규 AWS 프리티어($200 크레딧 / 6개월) 모델을 이해하고 비용 알림을 설정할 수 있다
3. AWS EC2 인스턴스(t3.micro / Ubuntu 24.04)를 생성하고 접속할 수 있다
4. 리눅스 기본 명령어와 git 기초로 서버를 관리할 수 있다
5. Nginx 웹서버를 설치하고 동작을 확인할 수 있다
6. **GitHub 저장소의 정적 사이트를 EC2 서버에 배포**할 수 있다
7. 수업 후 AWS 리소스를 안전하게 정리(Terminate)하여 과금을 방지할 수 있다

---

|------|---|------|------|-----------|

| **2** | **02장** | AWS 계정·신 프리티어·비용·IAM | 강의+실습 | ✅ Free Plan 가입 + Zero-Spend Budget 설정 |
| **3** | **03장** | EC2 인스턴스 생성과 접속 | **실습** | ✅ Ubuntu 24.04 + t3.micro 서버 접속 |
| **4** | **04장** | 리눅스 기초 명령어 + git | **실습** | ✅ 파일/폴더 조작 + git clone 가능 |
| **5** | **05장** | Nginx 웹서버 설치 | **실습** | ✅ "Welcome to nginx!" 페이지 확인 |

## 예상 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| "인스턴스가 안 보여요" | 리전이 서울이 아님 | 우측 상단 리전 → ap-northeast-2 |
| "Free Plan/Paid Plan 화면이 달라요" | 가입 시점 정책 차이 | 신규 계정은 Free Plan 선택 권장 |
| "t2.micro가 안 보여요" | 2025-07-15 이후 신규 계정 | t3.micro 선택 (이 수업의 표준) |
| "연결 시간 초과" | 보안 그룹 SSH(22) 미오픈 | 보안 그룹 → 인바운드 규칙 편집 |
| "Nginx 페이지 안 보여요" | HTTP(80) 미오픈 또는 https 접속 | 보안 그룹 + http:// 확인 |
| "git clone이 안 돼요" | 인터넷 연결, 잘못된 URL | `ping github.com`, URL 재확인 |
| "내 사이트가 안 보여요" | `/var/www/html/index.html` 미교체 | `sudo cp -r ...` 재실행 |
| "Permission denied" | sudo 빠뜨림 | 명령어 앞에 `sudo` 추가 |
