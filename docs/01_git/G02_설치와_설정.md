### 설치 확인

설치 후 **Git Bash**를 열고 다음 명령어를 실행합니다.

```bash
git --version
```

실행 결과:
```
git version 2.49.0.windows.1
```

> 💡 실무 팁: 버전이 2.49 이상이면 이 강의의 모든 명령어가 정상 동작합니다.
> 더 낮은 버전이 나오면 git-scm.com에서 다시 최신 버전을 다운로드하세요.

## 2️⃣ Git 설치 (Mac) [↑](#toc)

### Homebrew 설치 (아직 없는 경우)

> Homebrew는 Mac에서 프로그램을 터미널로 설치하는 패키지 매니저입니다.
> "앱스토어의 터미널 버전"이라고 생각하면 됩니다.

터미널을 열고 (Spotlight 검색: `terminal`) 다음 명령어를 붙여넣기합니다.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

설치 중 Mac 암호를 물어보면 입력합니다 (입력해도 화면에 보이지 않는 것이 정상입니다).

---

### Git 설치

```bash
brew install git
```


### 설치 확인

```bash
git --version

```

> 버전 확인 후 2.49+가 나오지 않으면 터미널을 닫고 다시 열어 확인해보세요.

## 3️⃣ 터미널 기초 [↑](#toc)

> 터미널은 **글자로 명령하는 리모컨**입니다.
> TV 리모컨 버튼 대신 명령어를 입력해서 컴퓨터에게 작업을 시킵니다.
> 처음에는 낯설지만, 5개 명령어만 알아도 Git 작업에 문제없습니다.

---

### 터미널 열기

| 운영체제 | 방법 |
|---------|------|
| Windows | **Git Bash** 실행 (시작 메뉴에서 검색) |
| Mac | **터미널(Terminal)** 실행 (Spotlight: `Cmd+Space` → "terminal") |
| VS Code | `Ctrl+\`` (백틱) — 모든 OS에서 동일 |

---

### 명령어 1: `pwd` — 지금 어디에 있나요?

> **P**rint **W**orking **D**irectory — "현재 폴더의 위치를 출력합니다."

```bash
pwd
```

실행 결과:
```
/Users/sunny/Documents
```

---

### 명령어 2: `ls` — 이 폴더에 뭐가 있나요?

> **L**i**s**t — "현재 폴더 안의 파일과 폴더 목록을 보여줍니다."

```bash
ls
```

실행 결과:
```
Desktop  Documents  Downloads  Music  Pictures
```

더 자세히 보기 (숨김 파일 포함, 상세 정보):

```bash
ls -la
```

실행 결과:
```
total 0
drwxr-xr-x   6 sunny  staff   192 Apr 10 09:00 .
drwxr-xr-x  20 sunny  staff   640 Apr 10 08:00 ..
drwxr-xr-x   3 sunny  staff    96 Apr 10 09:00 Desktop
drwxr-xr-x   4 sunny  staff   128 Apr 10 09:00 Documents
drwxr-xr-x   2 sunny  staff    64 Apr 10 09:00 Downloads
```

---

### 명령어 3: `cd` — 다른 폴더로 이동

> **C**hange **D**irectory — "폴더를 이동합니다."

```bash
# Documents 폴더로 이동
cd Documents

# 한 단계 위 폴더로 이동
cd ..

# 홈 폴더로 바로 이동 (물결표시 = 홈 폴더)
cd ~
```

실행 결과 (pwd로 확인):
```
/Users/sunny/Documents
```

> 💡 실무 팁: `cd` 입력 후 폴더명의 첫 몇 글자를 치고 `Tab` 키를 누르면 자동 완성됩니다.
> 긴 폴더명을 다 입력하지 않아도 됩니다.

---

### 명령어 4: `mkdir` — 새 폴더 만들기

> **M**a**k**e **Dir**ectory — "새 폴더를 만듭니다."

```bash
# git-practice라는 폴더 만들기
mkdir git-practice

# 확인
ls
```

실행 결과:
```
git-practice
```

---

### 명령어 5: `rm` — 파일/폴더 삭제

> **R**e**m**ove — "파일이나 폴더를 삭제합니다."

```bash
# 파일 삭제
rm 파일명.txt

# 폴더 삭제 (폴더 안 내용물까지 전부 삭제)
rm -rf 폴더명
```

> ⚠️ `rm -rf`는 **복구가 불가능한 영구 삭제**입니다. 휴지통에 가지 않습니다.
> 폴더명을 반드시 두 번 확인하고 실행하세요.

---

### 자주 쓰는 터미널 단축키

| 단축키 | 기능 |
|--------|------|
| `Tab` | 폴더/파일명 자동 완성 |
| `↑` / `↓` | 이전에 입력한 명령어 불러오기 |
| `Ctrl + C` | 현재 실행 중인 명령어 강제 종료 |
| `Ctrl + L` | 터미널 화면 지우기 (clear) |
| `clear` | 터미널 화면 지우기 |

## 4️⃣ Git 초기 설정 [↑](#toc)

> Git을 처음 사용하기 전에, Git에게 "나는 누구입니다"를 알려줘야 합니다.
> 커밋에 이름과 이메일이 기록되기 때문에, 한 번만 설정하면 됩니다.

---

### 필수 설정 1: 사용자 이름

```bash
# "홍길동" 자리에 본인 이름(또는 GitHub 아이디)을 입력
git config --global user.name "홍길동"
```

---

### 필수 설정 2: 이메일

```bash
# GitHub 가입 시 사용한 이메일을 동일하게 입력
git config --global user.email "honggildong@example.com"
```

> ⚠️ GitHub 계정의 이메일과 **동일하게** 입력해야 합니다.
> 다르면 GitHub에서 커밋이 본인 계정과 연결되지 않습니다.

---

### 필수 설정 3: 기본 브랜치 이름

```bash
# 새 저장소 생성 시 기본 브랜치 이름을 main으로 설정
git config --global init.defaultBranch main
```

---

### 필수 설정 4: 기본 편집기

```bash
# VS Code를 Git의 기본 편집기로 설정
git config --global core.editor "code --wait"
```

> 💡 실무 팁: `--wait` 옵션은 "VS Code 창을 닫을 때까지 Git이 대기"를 의미합니다.
> 커밋 메시지를 VS Code에서 작성하고 저장 후 닫으면 Git이 계속 진행합니다.

---

### 필수 설정 5: 줄바꿈 처리 (Windows vs Mac)

Windows와 Mac은 줄바꿈 문자가 다릅니다. 팀 협업 시 이 차이가 의도치 않은 변경사항으로 나타날 수 있습니다.

**Windows 사용자:**

```bash
git config --global core.autocrlf true
```

**Mac/Linux 사용자:**

```bash
git config --global core.autocrlf input
```

| 설정값 | 동작 | 사용 환경 |
|--------|------|----------|
| `true` | 체크아웃 시 LF→CRLF 변환, 커밋 시 CRLF→LF 변환 | Windows |
| `input` | 커밋 시 CRLF→LF 변환 (반대 방향은 변환 없음) | Mac/Linux |
| `false` | 변환 없음 | 혼합 환경 (권장하지 않음) |

## 5️⃣ 설정 확인 [↑](#toc)

설정이 올바르게 적용되었는지 확인합니다.

```bash
git config --list
```

실행 결과:
```
user.name=홍길동
user.email=honggildong@example.com
init.defaultbranch=main
core.editor=code --wait
core.autocrlf=true
```

특정 항목만 확인:

```bash
git config user.name
```

실행 결과:
```
홍길동
```

---

### 설정 파일 위치

모든 `--global` 설정은 홈 디렉토리의 `.gitconfig` 파일에 저장됩니다.

```bash
# 설정 파일 직접 보기
cat ~/.gitconfig
```

실행 결과:
```
[user]
        name = 홍길동
        email = honggildong@example.com
[init]
        defaultBranch = main
[core]
        editor = code --wait
        autocrlf = true
```

VS Code에서도 Git이 정상 동작하는지 확인합니다.

### VS Code 터미널 열기

1. VS Code 실행
2. 상단 메뉴: **Terminal → New Terminal** 클릭
3. 또는 단축키: `Ctrl+\`` (Windows/Linux) / `Cmd+\`` (Mac)

```bash
git --version
```

```
git version 2.49.0
```

VS Code에서 확인: 왼쪽 사이드바의 **소스 제어(Source Control)** 아이콘(Y 모양 분기)을 클릭하면 Git 패널이 보입니다. 현재는 "열린 폴더에서 Git 저장소를 찾을 수 없습니다" 메시지가 나오면 정상입니다.

## 7️⃣ GitHub 계정 만들기 [↑](#toc)

> GitHub는 Git 저장소를 클라우드에 저장하고, 다른 사람과 협업하는 플랫폼입니다.
> Google Drive가 파일을 저장한다면, GitHub는 코드의 버전 기록을 저장합니다.

### 가입 방법

1. [https://github.com/](https://github.com/) 접속
2. **Sign up** 클릭
3. 이메일 주소 입력 (Git 설정과 **동일한 이메일** 사용)
4. 비밀번호 설정
5. 사용자명(username) 입력 — 이후 `github.com/사용자명`이 내 주소가 됩니다
6. 이메일 인증 코드 입력

> ⚠️ 가입 시 입력한 **이메일 인증**을 반드시 완료해야 합니다.
> 이메일 인증 없이는 저장소 생성 등 주요 기능을 사용할 수 없습니다.

---

### 사용자명 선택 팁

GitHub 사용자명은 이후에도 계속 사용됩니다.

| 권장 | 비권장 |
|------|--------|
| 실명 또는 약자 (예: `kimminsu`) | 숫자만 (예: `user12345`) |
| 기억하기 쉬운 이름 | 특수문자 많은 이름 |
| 직업/기술 관련 (예: `developer-kim`) | 임시 느낌의 이름 |

> 💡 실무 팁: 취업/포트폴리오 시 GitHub 주소를 이력서에 적습니다.
> 처음부터 전문적인 이름을 선택하면 좋습니다.

### 실습 과제

**기본**: 터미널에서 다음 순서로 작업해보세요.
1. `cd ~` (홈 폴더로 이동)
2. `mkdir git-practice` (새 폴더 생성)
3. `cd git-practice` (새 폴더로 이동)
